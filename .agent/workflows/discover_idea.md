# アイデア発見ワークフロー

AIエージェント（Antigravity）が自律的にビジネスアイデアを発見・生成するワークフロー。

## 入力・出力

| 項目 | 内容 |
|------|------|
| **入力** | `five_perspectives_analysis.md`（オプション・推奨） |
|         | `demand_discovery.md`（オプション・推奨）【NEW v2.2】 |
| **出力** | `documents/2_discovery/business_idea.md` |
| **次のWF** | `/create_lean_canvas` |

> [!TIP]
> **v2.2改善**: `/discover_demand` の生ログ分析結果を参照することで、実際のユーザー発言に基づいた精度の高いアイデア発見が可能になります。


## 前提条件

- `startup_science_knowledge.md` を参照可能
- Web検索（search_web）が利用可能
- 出力先ディレクトリが存在する

## 制約条件（自動確認）

以下の制約を満たすアイデアのみを対象とする：
- ✅ **デジタル完結**: 物理製品・物理オペレーション不要
- ✅ **Antigravity実装可能**: LP/Webアプリ/コンテンツ系
- ✅ **法令遵守**: 違法・公序良俗に反するものは除外
- ✅ **個人スケール**: 1人で開始可能な規模

## STEP 1: 市場トレンド調査【自動実行】

### 使用ツール
`search_web`

### 実行手順
// turbo
以下のクエリを順次実行（各クエリで3-5件の結果を取得）：

```
クエリ1: "2025年 成長市場 SaaS 個人開発"
クエリ2: "AIエージェント ビジネス活用 事例 2024"
クエリ3: "副業 デジタルビジネス 需要 トレンド"
クエリ4: "ノーコード 収益化 成功事例"
クエリ5: "課題解決 ツール 不満 足りない"
```

### 期待出力
- 成長市場・トレンドのリスト
- ユーザーの課題・不満点
- 既存サービスの限界

### エラー対応
| エラー | 対応 |
|-------|------|
| 検索結果0件 | クエリを言い換えて再検索 |
| API制限 | 30秒待機後リトライ（最大3回） |
| 関連性低い結果 | 別のクエリセットに切り替え |

**代替クエリセット**:
```
"起業アイデア 2025 デジタル"
"オンラインビジネス 隙間市場"
"フリーランス ツール 不足"
```

## STEP 3: アイデア候補生成【自動実行】

### 使用ツール
内部生成（ツール不要）

### 生成ルール
- 発見した課題ごとに1-2個のアイデアを生成
- 合計5-10個のアイデア候補を作成
- 各アイデアは以下の形式で記述

```markdown
## アイデア候補 #1: [アイデア名]

| 項目 | 内容 |
|------|------|
| **ターゲット** | [誰] |
| **課題** | [何に困っている] |
| **ソリューション** | [どう解決する] |
| **提供形態** | [LP/Webアプリ/SaaS/コンテンツ] |
| **収益モデル** | [サブスク/買い切り/フリーミアム/広告] |
| **デジタル完結** | ✅可能 / ❌不可 |
| **競合** | [既存サービス名 or なし] |
| **差別化ポイント** | [何が違うか] |
```

## STEP 4: アイデアスコアリング【自動実行】

### スコアリング基準

`startup_science_knowledge.md` の3U + デジタル適合性 + 差別化で評価：

| 評価項目 | 配点 | 評価基準 |
|---------|:----:|---------|
| **Unsolved**（未解決） | 0-10 | 10=競合なし, 5=競合あるが不十分, 0=十分解決済み |
| **Urgent**（緊急） | 0-10 | 10=今日解決したい, 5=今月中, 0=いつでもいい |
| **Expensive**（高コスト） | 0-10 | 10=月1万円以上払う価値, 5=数千円, 0=無料希望 |
| **デジタル完結** | 0-10 | 10=100%デジタル, 5=一部手動, 0=物理必須 |
| **差別化可能性** | 0-10 | 10=唯一無二, 5=少し差別化, 0=コピー |

**合計50点満点**

### スコアリング結果テーブル

```markdown

|---|-----------|---|---|---|---------|-------|------|-----|
| 1 | [名前] | X | X | X | X | X | XX | X |
| 2 | [名前] | X | X | X | X | X | XX | X |
...
```

## STEP 5: 最終選定【自動判定】

### 選定ロジック

```python
# 擬似コード
sorted_ideas = sort_by_score(ideas, descending=True)

if sorted_ideas[0].score >= 35:
    selected = sorted_ideas[0]
elif len([i for i in sorted_ideas if i.score >= 30]) >= 2:
    # 同点付近の場合、デジタル完結スコアで決定
    selected = max(sorted_ideas[:3], key=lambda x: x.digital_score)
else:
    # スコア不足 → STEP 1に戻り別市場を調査
    return retry_with_different_market()
```

### 判定基準
| 条件 | アクション |
|------|-----------|
| 最高スコア ≥ 35点 | そのアイデアを自動選定 |
| 30-34点が2つ以上 | デジタル完結スコアで決定 |
| 全アイデア < 30点 | 別市場で再調査（最大2回） |
| 3回失敗 | Human-in-the-Loop へ |
