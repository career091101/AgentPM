# CPF達成判定ワークフロー

課題検証の結果に基づき、CPF（顧客課題適合）の達成状況を判定する。

> [!IMPORTANT]
> **v2.3 厳格化**: 仮想インタビュー結果とWeb課題検証結果を必須入力とし、未達成時はHuman-in-the-Loop必須に変更。

## 前提条件
- `lean_canvas.md` が作成済み
- `persona.md` が作成済み
- `problem_validation.md` が作成済み
- `problem_research.md` が作成済み【NEW v2.3】
- `interview_simulation.md` が作成済み【必須】
- `startup_validation_checklist.md` を参照可能

## ステップ

### 1. CPFチェックリストの確認

`startup_validation_checklist.md` のCPFセクションを確認し、各項目を評価：

```markdown
## CPFチェックリスト評価

### 顧客理解
| チェック項目 | 達成 | エビデンス |
|-------------|:----:|-----------|
| ターゲット顧客を明確に定義しているか？ | ✅/❌ | [persona.mdリンク] |
| 顧客インタビューを最低20人実施したか？ | ✅/❌/N/A | [※Phase1はAI検証でN/A、Phase2では必須] |
| 仮想インタビューを実施したか？【NEW】 | ✅/❌ | [interview_simulation.mdリンク] |
| ペルソナを作成したか？ | ✅/❌ | [persona.mdリンク] |

### 課題検証（3U検証）
| チェック項目 | 達成 | スコア |
|-------------|:----:|--------|
| 課題は「Unsolved（未解決）」か？ | ✅/❌ | [/10] |
| 課題は「Urgent（緊急）」か？ | ✅/❌ | [/10] |
| 課題は「Expensive（高コスト）」か？ | ✅/❌ | [/10] |
| 顧客が「お金を払ってでも解決したい」課題か？ | ✅/❌ | [判断根拠] |
| 課題が頻繁に発生するものか？ | ✅/❌ | [発生頻度] |

### 仮想インタビュー検証【NEW】
| チェック項目 | 達成 | スコア |
|-------------|:----:|--------|
| UVP刺さり度 | ✅/⚠️/❌ | [/40] |
| 最も深刻な課題が特定されているか？ | ✅/❌ | [課題名] |
| 購買意向が確認されているか？ | ✅/❌ | [金額帯] |

### Web課題検証【NEW v2.3】
| チェック項目 | 達成 | スコア |
|-------------|:----:|--------|
| Web上に課題投稿が存在するか？ | ✅/❌ | [X件発見] |
| 投稿の感情が強いか？（不満・怒り等） | ✅/❌ | [感情スコア/10] |
| 解決要望・支払い意向があるか？ | ✅/❌ | [有/無] |
| Web投稿スコア | ✅/⚠️/❌ | [X/50] |

### CPF達成基準
| 基準 | 達成 | 結果 |
|------|:----:|------|
| 3U検証で24点以上/30点 | ✅/❌ | [実際のスコア] |
| 仮想インタビューUVP刺さり度24/40以上 | ✅/❌ | [実際のスコア] |
| Web投稿スコア20/50以上【NEW】 | ✅/❌ | [実際のスコア] |
```

### 2. 達成状況の総合評価

```markdown
## CPF達成判定

### スコアサマリー
- 顧客理解: [X/4項目達成]
- 課題検証: [X/5項目達成]
- 仮想インタビュー: [X/3項目達成]
- 達成基準: [達成/未達成]

### 総合判定
| 判定 | 説明 |
|------|------|
| ✅ **CPF達成** | 80%以上の項目を達成。PSF検証へ進む |
| ⚠️ **条件付き達成** | 60-79%達成。警告を記録しPSF検証へ |
| ❌ **未達成** | 60%未満。**Human-in-the-Loop必須** |

**今回の判定**: [✅/⚠️/❌]
```

### 3. ギャップ分析

```markdown
## ギャップ分析

### 未達成項目
| 項目 | 現状 | 必要な対応 |
|------|------|-----------|
| [未達成項目1] | [現状の状態] | [改善アクション] |
| [未達成項目2] | [現状の状態] | [改善アクション] |

### リスク評価
| リスク | 影響度 | 対策 |
|--------|:------:|------|
| [リスク1] | 高/中/低 | [対策] |
```

### 4. 次のステップ決定【厳格化】

```markdown
## 次のステップ

### ✅ CPF達成の場合
→ `/diagnose_psf` でPSF検証へ進む

### ⚠️ 条件付き達成の場合
1. 警告内容を記録
2. PSF検証で追加注意を払う
3. Phase2では実インタビュー必須

### ❌ 未達成の場合【Human-in-the-Loop必須】

> [!CAUTION]
> CPF未達成は「課題そのものが顧客にとって重要でない可能性」を示唆します。
> 自動リトライせず、根本的な見直しを検討してください。

**ユーザーへの質問**:
以下から次のアクションを選択してください：

1. **課題・顧客を見直して再実行**
   - `/create_lean_canvas` を更新（課題・顧客セグメントを見直し）
   - `/define_persona` を更新
   - `/validate_problem` を再実行
   - 再度 `/diagnose_cpf` を実行

2. **ピボットを検討**
   - `/decide_pivot` を実行してピボット判断

3. **リスク承知で強行突破**
   - CPF未達成を承知でPSF検証に進む（非推奨）
```
