## エラーハンドリング

| エラー | 対応 |
|--------|------|
| 記事URLがアクセス不可 | 警告付きでスキップ、次の対象へ |
| Xアカウントが凍結/削除 | 代替ソースで調査継続 |
| 3回リトライ失敗 | 「⚠️要確認」フラグを付けて継続 |
| 品質基準未達（200行未満） | 自動再調査トリガー |
| ファクトチェックNG | WARNフラグを付けて記録 |

---

## 中断・再開機能

オーケストレーションは中断しても再開可能：

1. **進捗の永続化**: 各調査完了時に `research_progress.md` を更新
2. **再開時の動作**: 🔲ステータスの対象から自動的に継続
3. **手動中断**: ユーザーが停止した場合、次回は中断地点から再開

---

## Human-in-the-Loop ポイント

以下の状況では**確認を求める**：

| ポイント | 条件 | 選択肢 |
|---------|------|--------|
| Tier完了時 | 各Tier終了後 | 継続 / 一時停止 / 中止 |
| 大量エラー発生 | 1バッチで3件以上失敗 | リトライ / スキップ / 中止 |
| 50%完了時 | 中間チェックポイント | 継続 / レビュー要求 |
| AI著名人調査前 | オプション調査の確認 | 実行 / スキップ |

## 品質基準（v3.1準拠）

各事例ファイルは以下の基準を満たすこと：

| 項目 | 基準 |
|------|------|
| **行数** | 約200行以上（001_wilson_senja.md相当） |
| **セクション数** | 12セクション（基本情報〜参考リンク） |
| **ソース数** | 最低3ソース（X、個人サイト、記事等） |
| **ファクトチェック** | PASS（またはWARN+理由記載） |
| **日本市場スコア** | 定量評価（5段階×重み付け） |

※品質未達の場合は自動再調査がトリガーされる

## バッチ終了時省略チェック★v1.1追加

各バッチ完了時、および全体完了時に以下のチェックを必ず実行：

### 必須ステップチェックリスト

各ケースファイルに対して以下のセクションが存在するか確認：

| チェック項目 | 検索キーワード | 省略判定 |
|--------------|------------------|----------|
| X(Twitter)調査 | "X/Twitter", "@", "フォロワー" | セクションなし=省略 |
| IndieHackers調査 | "IndieHackers", "indiehackers" | セクションなし=省略 |
| Product Hunt調査 | "Product Hunt", "producthunt" | セクションなし=省略 |
| 本人サイト調査 | "URL", "公式サイト", "プロダクト" | セクションなし=省略 |
| 日本市場評価 | "日本市場", "適用性" | セクションなし=省略 |
| ファクトチェック | "ファクトチェック", "PASS", "WARN" | セクションなし=省略 |

### 省略発見時の対応フロー

```text
1. 省略されたステップをリスト化
   例: "014_danny_postma.md: IndieHackers省略, Product Hunt省略"

2. 各省略ステップについて再調査を実行:
   - IndieHackers省略 → search_webでIndieHackersプロフィールを検索
   - Product Hunt省略 → search_webでProduct Huntを検索
   - X(Twitter)省略 → browser_subagentでXプロフィールを確認

3. 再調査結果をファイルに追記
   - 既存セクションに情報を追加
   - または新セクションを作成

4. 再調査完了後、再度省略チェック
```

### 省略許容ケース

以下の場合のみ省略を許容（「調査済み・情報なし」）：

| ステップ | 許容条件 | 記載例 |
|--------|----------|--------|
| IndieHackers | 検索したがプロフィールなし | "IndieHackers: プロフィールなし" |
| Product Hunt | 検索したが未ローンチ | "Product Hunt: 未ローンチ" |
| YouTube | 検索したが該当なし | "YouTube: 該当動画なし" |

> **重要**: 「調査していない」と「調査したが情報がなかった」は別。
> 前者は省略として再調査必須、後者は許容。
