# BtoB特有検証ワークフロー

起業大全に基づき、BtoB事業特有のPMF検証とエンタープライズ展開可能性を評価する。

> [!IMPORTANT]
> **起業大全の教え**: BtoBでは「導入=成功」ではない
> 複数のステークホルダー（決裁者、管理者、利用者）全員がサクセスしていることが重要

## 入力・出力

| 項目 | 内容 |
|------|------|
| **入力** | `persona.md`, 顧客データ, 導入事例 |
| **出力** | `documents/5_monitoring/btob_validation.md` |
| **対象** | BtoB/SaaS事業のみ |

---

## STEP 1: BtoB特有の構造理解【自動実行】

```markdown
## BtoB特有の構造

### ステークホルダー構造
| ステークホルダー | 関心事 | 意思決定への影響 |
|----------------|--------|-----------------|
| **決裁者** | ROI、戦略適合性、リスク | 最終承認 |
| **管理者** | 運用負荷、管理効率、セキュリティ | 導入可否 |
| **利用者** | 使いやすさ、業務効率、学習コスト | 日常利用 |
| （IT部門）| 技術要件、統合、セキュリティ | 技術承認 |
| （購買部門）| 価格、契約条件 | 契約条件 |

### 購買プロセス
| フェーズ | 主な活動 | 関与者 |
|---------|---------|--------|
| 認知 | 課題認識、情報収集 | 利用者、管理者 |
| 検討 | 比較検討、トライアル | 管理者、IT部門 |
| 決裁 | ROI算出、稟議 | 決裁者、購買 |
| 導入 | 設定、移行、トレーニング | IT、利用者 |
| 活用 | 日常利用、定着 | 利用者 |
| 拡大 | 他部署展開、アップセル | 管理者、決裁者 |
```
