# 10倍優位性検証ワークフロー

ソリューションが既存の代替案と比較して10倍以上優れているかを詳細検証する独立ワークフロー。

> [!IMPORTANT]
> **起業の科学の教え**: 「10倍優れていなければ、顧客は乗り換えない」
> スイッチングコストを超える価値がなければ、市場で勝てません。

## 入力・出力

| 項目 | 内容 |
|------|------|
| **入力** | `lean_canvas.md`, `problem_validation.md` |
| **出力** | `documents/2_discovery/10x_validation.md` |
| **次のWF** | `/diagnose_psf` または `/build_lp` |

## 前提条件

- `lean_canvas.md` が作成済み（ソリューション定義済み）
- `problem_validation.md` が作成済み（課題・競合把握済み）
- `startup_science_knowledge.md` を参照可能

## STEP 2: 比較軸の設定【自動実行】

### 標準5軸

| 軸 | 説明 | 評価基準 |
|----|------|---------|
| **時間効率** | タスク完了にかかる時間 | 少ないほど優位 |
| **コスト** | 金銭的負担 | 低いほど優位 |
| **使いやすさ** | 学習コスト・操作の簡便さ | 簡単なほど優位 |
| **成果・効果** | 得られる結果の質・量 | 高いほど優位 |
| **導入障壁** | 始めるまでのハードル | 低いほど優位 |

### カスタム軸（オプション）

課題に応じて追加：
- 信頼性・安定性
- カスタマイズ性
- サポート品質
- データセキュリティ
- モバイル対応
