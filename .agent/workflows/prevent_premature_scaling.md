# プレマチュア・スケーリング診断ワークフロー

起業の科学に基づき、PMF達成前の過剰投資（プレマチュア・スケーリング）を防止する。

> [!CAUTION]
> **起業の科学の教え**: セブンドリーマーズ（ランドロイド）の事例
> 技術は凄いがPMF前に巨額投資し倒産。「ムラ」= 成長スピードと経営陣の成長スピードの不一致

## 入力・出力

| 項目 | 内容 |
|------|------|
| **入力** | 事業データ、財務状況、組織状況 |
| **出力** | `documents/5_monitoring/premature_scaling_check.md` |
| **実施タイミング** | 投資実行前、四半期ごと |

## STEP 6: 適正投資ガイドライン【自動実行】

```markdown
## フェーズ別適正投資ガイドライン

### CPF/PSFフェーズ（PMF前）
| カテゴリ | 適正範囲 | 重点 |
|---------|---------|------|
| チーム | 5-10名、コア人材のみ | Founder + 開発 + 初期営業 |
| マーケ | 〜100万/月、実験的 | 仮説検証に特化 |
| 開発 | コア機能のみ | MVP品質でOK |
| オフィス | 最小限/リモート | 固定費抑制 |

### PMF達成後
| カテゴリ | 適正範囲 | 重点 |
|---------|---------|------|
| チーム | 成長に合わせ段階的 | 再現性ある採用 |
| マーケ | LTV/CAC > 3を維持 | チャネル多様化 |
| 開発 | スケーラビリティ投資 | 技術負債解消 |
| オフィス | チーム規模に応じて | 採用競争力も考慮 |
```
