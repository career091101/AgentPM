# MVV定義ワークフロー

起業大全に基づき、企業のMission/Vision/Valueと戦略ストーリーを定義する。

## 前提条件
- `lean_canvas.md` が作成済み
- `pmf_diagnosis.md` でPMF達成またはPMF達成間近
- `startup_science_knowledge.md` を参照可能

## ステップ

### 1. Mission（存在意義）の定義

```markdown
## Mission定義

**質問**: なぜこの事業は存在するのか？

### ミッションステートメント
[1-2文で企業の存在意義を表現]

### ミッション構成要素
| 要素 | 内容 |
|------|------|
| 誰のために | [ターゲット] |
| 何を | [提供する価値] |
| なぜ | [社会的意義] |

### チェック
- [ ] 社員が朝起きる理由になるか
- [ ] 顧客に共感されるか
- [ ] 10年後も変わらないか
```

### 2. Vision（実現したい未来）の定義

```markdown
## Vision定義

**質問**: 5-10年後、どんな世界を実現したいか？

### ビジョンステートメント
[具体的で測定可能な未来像]

### ビジョン構成要素
| 要素 | 内容 |
|------|------|
| 時間軸 | [いつまでに] |
| 到達点 | [何を達成するか] |
| 影響範囲 | [誰にどんな影響を与えるか] |

### チェック
- [ ] ワクワクするか
- [ ] 具体的にイメージできるか
- [ ] 挑戦的だが達成可能か
```

### 3. Value（行動指針）の定義

```markdown
## Value定義

**質問**: どんな姿勢で事業を運営するか？

### コアバリュー（3-5個）
| # | バリュー | 具体的な行動例 |
|---|---------|--------------|
| 1 | [バリュー名] | [具体的にどう行動するか] |
| 2 | [バリュー名] | [具体的にどう行動するか] |
| 3 | [バリュー名] | [具体的にどう行動するか] |
| 4 | [バリュー名] | [具体的にどう行動するか] |
| 5 | [バリュー名] | [具体的にどう行動するか] |

### チェック
- [ ] 採用・評価の基準になるか
- [ ] 意思決定の指針になるか
- [ ] 覚えやすいか
```

### 4. MVV整合性チェック

```markdown
## MVV整合性チェック

| チェック項目 | 達成 |
|-------------|:----:|
| MissionとVisionが一貫している | ✅/❌ |
| ValueがMission達成を支援する | ✅/❌ |
| リーンキャンバスと整合している | ✅/❌ |
| ターゲット顧客に響く | ✅/❌ |
| 競合との差別化になる | ✅/❌ |
```

### 5. 唯一無二のストーリー（Only One Story）

```markdown
## 戦略ストーリー構築

起業大全「唯一無二のストーリー」に基づき、事業の独自性を言語化：

### ストーリーの構成要素
| 要素 | 内容 |
|------|------|
| **きっかけ（Why）** | [なぜこの事業を始めたか] |
| **発見（Insight）** | [どんな気づきがあったか] |
| **解決策（How）** | [どう解決するか] |
| **未来（Where）** | [どこに向かうか] |

### エレベーターピッチ（30秒版）
[上記を30秒で説明できる形に]

### ストーリーの独自性
他社にはない、自社だけのストーリー要素：
1. [独自要素1]
2. [独自要素2]
3. [独自要素3]
```

### 6. MVVの活用計画

```markdown
## MVV活用計画

### 社内浸透
- [ ] 全メンバーへの共有
- [ ] オンボーディングへの組み込み
- [ ] 評価制度への反映

### 社外発信
- [ ] Webサイトへの掲載
- [ ] 採用ページでの訴求
- [ ] SNS・広報での活用

### 意思決定への活用
- [ ] 新機能開発の判断基準
- [ ] パートナーシップの判断基準
- [ ] ピボット判断の軸
```
