# ペルソナ定義ワークフロー

ビジネスアイデアのターゲット顧客を具体的なペルソナとして定義する。

## 前提条件
- `business_idea.md` が作成済み
- `lean_canvas.md` が作成済み（顧客セグメント情報）

## ステップ

### 1. 顧客セグメントの確認
`lean_canvas.md` から以下を確認：
- アーリーアダプターの特徴
- 想定市場サイズ
- 課題の深刻度

### 2. デモグラフィック情報の設定

```markdown
## 基本情報
- **名前**: [架空の名前]
- **年齢**: [具体的な年齢]
- **性別**: [該当する場合]
- **職業**: [具体的な職種・役職]
- **年収**: [収入レンジ]
- **居住地**: [都市部/地方等]
- **家族構成**: [必要に応じて]
```

### 3. サイコグラフィック情報の設定

```markdown
## 心理特性
- **価値観**: [何を大切にしているか]
- **ライフスタイル**: [日常の行動パターン]
- **情報収集方法**: [SNS/検索/口コミ等]
- **意思決定スタイル**: [即断型/熟考型]
- **テクノロジーリテラシー**: [高/中/低]
```

### 4. 課題・ゴールの定義

```markdown
## 課題（Pains）
1. **最大の課題**: [最も困っていること]
   - 発生頻度: [毎日/週/月]
   - 深刻度: [1-10]
   - 現在の対処法: [どうしのいでいるか]

2. **2番目の課題**: [...]
3. **3番目の課題**: [...]

## ゴール（Gains）
1. **理想の状態**: [どうなりたいか]
2. **期待する成果**: [何を得たいか]
3. **成功の定義**: [何をもって成功とするか]
```

### 5. 行動パターンの定義

```markdown
## 典型的な1日
- 6:00 [朝の行動]
- 9:00 [業務開始]
- 12:00 [昼]
- 18:00 [業務終了]
- 22:00 [夜の行動]

## 課題に直面するタイミング
- いつ: [具体的な時間・状況]
- どこで: [場所・環境]
- なぜ: [トリガーとなる出来事]
```

### 6. 購買行動の分析

```markdown
## 購買行動
- **予算感**: [この課題解決にいくら払えるか]
- **決裁権**: [自分で決められるか]
- **購入障壁**: [何が購入を妨げるか]
- **購入決定要因**: [何があれば買うか]
```

### 7. エンパシーマップの作成

```markdown
## エンパシーマップ

### 見ていること（See）
- [目に入る情報・環境]

### 聞いていること（Hear）
- [周囲からの声・情報]

### 考えていること・感じていること（Think & Feel）
- [内心の思い・不安・願望]

### 言っていること・やっていること（Say & Do）
- [外に表す言動]

### 痛み（Pains）
- [恐れ・フラストレーション・障壁]

### 得たいもの（Gains）
- [欲しいもの・成功の尺度]
```

## 出力

成果物を以下のパスに保存：

```

```
