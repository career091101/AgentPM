# リーンキャンバス作成ワークフロー

ビジネスアイデアを「起業の科学」の9要素に基づいてリーンキャンバスに落とし込む。

## 前提条件
- `business_idea.md` が作成済みであること
- `startup_science_knowledge.md` を参照可能
- `startup_validation_checklist.md` を参照可能

## ステップ

### 1. ビジネスアイデアの確認
`business_idea.md` から以下を確認：
- 選定されたアイデア
- ターゲット顧客
- 課題
- ソリューション概要

### 2. リーンキャンバス9要素の作成
`startup_science_knowledge.md` の「3. リーンキャンバスとビジネスモデル」を参照し、以下の順序で記入：

#### 2.1 課題（Problem）
```markdown
### 課題
1. [最重要の課題]
2. [2番目の課題]
3. [3番目の課題]

**既存の代替案**: [現在ユーザーがどう対処しているか]
```

#### 2.2 顧客セグメント（Customer Segments）
```markdown
### 顧客セグメント
**アーリーアダプター**: [最初に採用する具体的なユーザー像]
- 属性: [年齢、職業、行動特性]
- 規模: [想定市場サイズ]
```

#### 2.3 独自の価値提案（Unique Value Proposition）
```markdown
### 独自の価値提案（UVP）
**一言で**: [競合との差別化を含む価値提案]

- なぜ注目に値するか:
- どう違うか:
- なぜ今か:
```

#### 2.4 ソリューション（Solution）
```markdown
### ソリューション
各課題に対応する機能：
1. [課題1への解決策]
2. [課題2への解決策]
3. [課題3への解決策]
```

#### 2.5 チャネル（Channels）
```markdown
### チャネル
**顧客へのリーチ方法**:
- 認知: [どう知ってもらうか - SNS/SEO/広告等]
- 獲得: [どう使い始めてもらうか]
- 維持: [どう継続利用してもらうか]
```

#### 2.6 収益の流れ（Revenue Streams）
```markdown
### 収益の流れ
**マネタイズモデル**: [サブスク/買い切り/フリーミアム等]
- 価格帯: [想定価格]
- 課金タイミング: [いつ課金するか]
- LTV目標: [顧客生涯価値の目標]
```

#### 2.7 コスト構造（Cost Structure）
```markdown
### コスト構造
**固定費**:
- [サーバー/ドメイン等]

**変動費**:
- [ユーザー増加に伴うコスト]

**初期投資**:
- [MVP構築にかかるコスト]
```

#### 2.8 主要指標（Key Metrics）
```markdown
### 主要指標（KPI）
**北極星指標**: [最も重要な1つの指標]

**フェーズ別KPI**:
- CPF: [課題検証の指標]
- PSF: [ソリューション検証の指標]
- PMF: [NPS, リテンション等]
```

#### 2.9 圧倒的な優位性（Unfair Advantage）
```markdown
### 圧倒的な優位性（MOAT）
**模倣困難な要素**:
- [コピーできない強み - データ/ネットワーク効果/専門知識等]

**注意**: 初期段階では「なし」でも可。PMF後に構築することを明記。
```

### 3. 整合性チェック
リーンキャンバス全体の整合性を確認：

| チェック項目 | 確認 |
|-------------|------|
| 課題とソリューションが対応しているか | [ ] |
| 顧客セグメントが具体的か | [ ] |
| UVPが一言で説明できるか | [ ] |
| 収益モデルが持続可能か | [ ] |
| コスト構造が現実的か | [ ] |
| KPIが測定可能か | [ ] |

### 4. 仮説の明確化
リーンキャンバスの各要素を「検証すべき仮説」として整理：

```markdown
## 検証すべき仮説

### 最重要仮説（課題）
- 仮説: [ターゲット]は[課題]に困っている
- 検証方法: [どう確認するか]
- 成功基準: [何をもって検証済みとするか]

### ソリューション仮説
- 仮説: [ソリューション]は[課題]を解決する
- 検証方法: [どう確認するか]
- 成功基準: [何をもって検証済みとするか]
```
