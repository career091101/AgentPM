# ソロプレナー収益計算ワークフロー

個人開発者・副業起業家向けの収益シミュレーションを実行し、現実的な目標設定を支援する。

## 概要

月収目標から必要ユーザー数・価格設定を逆算し、ソロプレナー事例をベンチマークとして参照する。

## 入力

| パラメータ | 必須 | 説明 |
|-----------|------|------|
| `target_mrr` | ○ | 目標月収（円） |
| `product_type` | ○ | `saas` / `tool` / `course` / `newsletter` |
| `pricing_model` | - | `freemium` / `subscription` / `one-time` |

## 出力

`{IDEA_FOLDER}/documents/3_planning/solo_economics.md`

## 実行手順

### STEP 1: ベンチマーク参照
// turbo
```text
ツール: /reference_solo_cases category=revenue
処理:
  1. Solo_App事例から同規模の成功事例を検索
  2. 収益モデル・価格帯・ユーザー数を抽出
  3. ベンチマーク表を作成
```

### STEP 2: 収益シミュレーション
// turbo
```text
処理:
  目標MRRから以下を逆算:
  
  【サブスクモデル】
  - 必要有料ユーザー数 = 目標MRR ÷ 月額単価
  - 必要無料ユーザー数 = 有料ユーザー ÷ 転換率(5%)
  - 必要PV/月 = 無料ユーザー ÷ 登録率(10%)
  
  【買い切りモデル】
  - 必要販売数/月 = 目標MRR ÷ 単価
  - 必要PV/月 = 販売数 ÷ CVR(2%)
  
  【フリーミアムモデル】
  - 無料ユーザー = 目標MRR ÷ ARPU
  - 必要PV/月 = 無料ユーザー ÷ 登録率(10%)
```

### STEP 3: 価格感度分析
// turbo
```text
処理:
  3つの価格帯でシミュレーション:
  - 低価格帯（$9-19/月）
  - 中価格帯（$29-49/月）
  - 高価格帯（$99-199/月）
  
  各価格帯での:
  - 必要有料ユーザー数
  - 平均的な転換率（ベンチマーク参照）
  - 達成難易度評価
```
