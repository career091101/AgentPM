# 仮想ペルソナインタビュー ワークフロー

AIエージェントが定義したペルソナに対して仮想インタビューを実施し、課題仮説を検証する。

## 入力・出力

| 項目 | 内容 |
|------|------|
| **入力** | `persona.md`, `lean_canvas.md` |
| **出力** | `documents/2_discovery/interview_simulation.md` |
| **次のWF** | `/diagnose_cpf` |

## 目的

- 実際の顧客インタビューが困難な場合の代替検証
- ペルソナ視点での課題深堀り
- UVPの刺さり方を事前検証
- AIエージェントによる完全自律検証を実現

---

## STEP 1: インタビュー対象の設定【自動実行】

### 使用ツール
`view_file`

// turbo
`persona.md` から以下を抽出：
- ペルソナ名
- 属性（年齢、職業、役職）
- 主要な課題3つ
- 目標・ゴール
- 行動パターン
- 価値観

---

## STEP 2: インタビューシナリオ設計【自動実行】

### インタビュー構成

```markdown
## インタビューシナリオ

### オープニング（自己紹介・目的説明）
1分程度

### パート1: 現状把握（5分）
- 普段の業務/生活について
- 1日の流れ
- 使用しているツール

### パート2: 課題深堀り（10分）
- 課題に直面するタイミング
- その時の感情
- 現在の対処法
- 理想と現実のギャップ

### パート3: ソリューション提示（5分）
- UVPの説明
- 反応の確認
- 懸念点の聴取

### パート4: 購買意向（3分）
- 価格感
- 導入障壁
- 意思決定プロセス

### クロージング（1分）
```

| 条件 | 判定 | アクション |

| UVP刺さり度 32/40以上 | ✅ 良好 | CPF診断へ進む |
