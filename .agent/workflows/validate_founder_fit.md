# Founder-Issue-Fit検証ワークフロー

創業者とその解決しようとする課題の相性を検証し、PMFの土台を確認する。

> [!IMPORTANT]
> **起業参謀の教科書より**: 「なぜ他の誰でもなく、あなたがこの課題を解決すべきなのか？」
> Founder-Issue-FitはPMFパルテノン神殿の「土台」に位置する重要な概念です。

## 入力・出力

| 項目 | 内容 |
|------|------|
| **入力** | `business_idea.md`, `lean_canvas.md`（任意） |
| **出力** | `documents/1_initiating/founder_fit.md` |
| **次のWF** | `/create_mvv` または `/create_lean_canvas` |

## 前提条件

- ビジネスアイデアが明確化されている
- 創業者（またはプロジェクトオーナー）の情報が把握できる
- `startup_science_knowledge.md` を参照可能

## STEP 2: 課題との適合性検証

// turbo

6つの軸で創業者と課題の適合性を評価：

```markdown
## Founder-Issue-Fit 6軸検証

| # | 軸 | 質問 | 回答 | スコア |
|---|-----|------|------|:------:|
| 1 | **経験** | この課題に関連する経験があるか？ | [回答] | /10 |
| 2 | **動機** | なぜこの課題を解決したいのか？個人的な原体験はあるか？ | [回答] | /10 |
| 3 | **情熱** | 10年以上取り組める情熱があるか？ | [回答] | /10 |
| 4 | **共感** | 顧客の痛みを深く理解しているか？顧客視点で語れるか？ | [回答] | /10 |
| 5 | **スキル** | 解決に必要なスキルを持っているか？コア技術を保有しているか？ | [回答] | /10 |
| 6 | **ネットワーク** | 関連業界との繋がりがあるか？初期顧客にリーチできるか？ | [回答] | /10 |

### スコアリング基準

| スコア | 基準 |
|:------:|------|
| 8-10 | 非常に強い（明確なエビデンスあり） |
| 5-7 | 中程度（ある程度の根拠あり） |
| 2-4 | 弱い（限定的な根拠） |
| 0-1 | ほぼなし |
```

## STEP 3: 総合判定

```markdown
## Founder-Issue-Fit 総合判定

### スコアサマリー
| 軸 | スコア |
|----|:------:|
| 経験 | /10 |
| 動機 | /10 |
| 情熱 | /10 |
| 共感 | /10 |
| スキル | /10 |
| ネットワーク | /10 |
| **合計** | **/60** |

### 判定基準

| 判定 | 条件 | 推奨アクション |
|------|------|---------------|
| ✅ **強いFit** | 50/60以上 | そのまま進行 |
| ⚠️ **部分Fit** | 30-49/60 | 不足領域を補う共同創業者を検討 |
| ❌ **弱いFit** | 29/60以下 | 別の課題/創業者変更を検討 |

### 今回の判定
**判定**: [✅/⚠️/❌]
**合計スコア**: [X/60]
```

## STEP 5: Founder-Issue-Fitが弱い場合のリスク確認

```markdown
## リスク確認

> [!WARNING]
> Founder-Issue-Fitが弱い場合、以下のリスクがあります：

| リスク | 影響 | 対策 |
|--------|------|------|
| モチベーション維持困難 | 困難な局面で諦めやすい | 原体験を深掘り |
| 顧客理解の浅さ | 的外れなソリューション | 顧客インタビュー強化 |
| 業界知識不足 | 判断ミス | アドバイザー採用 |
| 長期戦での耐久力不足 | 資金切れ・燃え尽き | チーム強化 |
```
