# PSF達成判定ワークフロー

ソリューション検証の結果に基づき、PSF（課題解決適合）の達成状況を判定する。

> [!IMPORTANT]
> **v2.0 強化**: 10倍優位性検証を詳細化し、未達成時はHuman-in-the-Loop必須に変更。

## 前提条件
- `cpf_diagnosis.md` でCPF達成済み
- `lean_canvas.md` （ソリューション定義済み）
- `interview_simulation.md` （UVPへの反応確認済み）
- `startup_validation_checklist.md` を参照可能

## ステップ

### 1. PSFチェックリストの確認

`startup_validation_checklist.md` のPSFセクションを確認し、各項目を評価：

- 価値提案: [X/3項目達成]

### 総合判定
| 判定 | 説明 |
|------|------|
| ✅ **PSF達成** | ソリューションが課題を解決することを確認。MVP構築へ進む |
| ⚠️ **条件付き達成** | MVPで検証が必要。LP構築で検証を継続 |
| ❌ **未達成** | ソリューションの再設計が必要。**Human-in-the-Loop必須** |

**今回の判定**: [✅/⚠️/❌]

### 5. 次のステップ決定【厳格化】

```markdown
## 次のステップ

### ✅ PSF達成の場合
→ `/build_lp` でLP構築へ進む
→ `/deploy_mvp` でMVP公開へ

### ⚠️ 条件付き達成の場合
1. `/build_lp` でオファー型MVPを構築
2. ユーザー反応を測定して検証継続
3. 事前登録30人以上で本格開発へ

### ❌ 未達成の場合【Human-in-the-Loop必須】

> [!CAUTION]
> PSF未達成は「ソリューションが課題を解決できていない可能性」を示唆します。
> 10倍優位性がないソリューションは市場で勝てません。

**ユーザーへの質問**:
以下から次のアクションを選択してください：

1. **ソリューションを再設計**
   - `/create_lean_canvas` のソリューション欄を更新
   - 再度 `/diagnose_psf` を実行

2. **ピボットを検討**
   - `/decide_pivot` を実行してピボット判断
   - 課題自体は正しいが解決策が間違っている可能性

3. **10倍優位性を強化**
   - 特定の軸に集中して10倍以上の改善を目指す
   - 機能削減・特化戦略を検討

4. **リスク承知で強行突破**
   - PSF未達成を承知でLP構築に進む（非推奨）
```
