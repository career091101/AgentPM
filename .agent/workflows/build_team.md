# チームビルディング・CXO設計ワークフロー

起業参謀の教科書に基づき、創業チームの構成と各CXOの役割・成長計画を策定する。

> [!IMPORTANT]
> **起業参謀の教え**: フェーズが進むにつれ、創業者は「プレイヤー」から「経営者」へ進化する必要がある。
> CXOの専門性だけでなく「全体最適」の視点を持てない「エセCXO」は組織を壊す。

## 入力・出力

| 項目 | 内容 |
|------|------|
| **入力** | 現在のチーム情報、`mvv.md`（推奨） |
| **出力** | `documents/3_planning/team_design.md` |
| **実施タイミング** | Seed調達前、Series A前、組織拡大時 |

## 前提条件
- リーンキャンバスが作成済み
- フェーズ（CPF/PSF/PMF/Scale）が明確

## STEP 3: CXO役割定義【自動実行】

```markdown
## CXO役割定義

### CEO（Chief Executive Officer）
| 責任領域 | 詳細 |
|---------|------|
| **ビジョン策定** | 会社の方向性・戦略を決定 |
| **資金調達** | 投資家との関係構築、ピッチ |
| **最終意思決定** | 重要事項の最終判断 |
| **全社最適** | 部門間の調整、リソース配分 |
| **対外コミュニケーション** | メディア、パートナー対応 |

### CTO（Chief Technology Officer）
| 責任領域 | 詳細 |
|---------|------|
| **技術戦略** | アーキテクチャ、技術選定 |
| **プロダクト開発** | 開発ロードマップ、品質管理 |
| **エンジニア組織** | 採用、育成、チームビルディング |
| **技術的意思決定** | Build vs Buy、技術負債管理 |

### COO（Chief Operating Officer）
| 責任領域 | 詳細 |
|---------|------|
| **事業開発** | 新規事業、パートナーシップ |
| **オペレーション** | 業務プロセス構築・最適化 |
| **実行責任** | 戦略の実行、KPI管理 |
| **組織運営** | 採用、人事制度、カルチャー |

### CMO（Chief Marketing Officer）
| 責任領域 | 詳細 |
|---------|------|
| **マーケティング戦略** | ブランディング、市場ポジショニング |
| **顧客獲得** | チャネル戦略、CAC最適化 |
| **市場との対話** | 顧客インサイト、市場調査 |
| **PR・広報** | メディアリレーション |

### CFO（Chief Financial Officer）
| 責任領域 | 詳細 |
|---------|------|
| **財務戦略** | 資本政策、資金調達支援 |
| **管理会計** | 予実管理、コスト管理 |
| **IPO準備** | 内部統制、監査対応 |
| **ガバナンス** | コンプライアンス、リスク管理 |
```

| フェーズ | 創業者の主な役割 | 必要な成長 |
|---------|----------------|-----------|

| Series A | マネージャー | チームを通じて成果を出す |
| Series B+ | 経営者 | ビジョン・戦略に集中 |

### CXO採用基準

| 評価項目 | 重要度 | チェックポイント |
|---------|:------:|----------------|
| 専門スキル | 高 | [具体的な基準] |
| 全体最適思考 | 高 | 部門最適でなく全社最適を考えられるか |
| カルチャーフィット | 高 | MVVへの共感度 |
| 成長意欲 | 中 | フェーズの変化に適応できるか |
| 実行力 | 高 | 過去の成果・実績 |

> [!CAUTION]
> **エセCXO警告サイン**:
> - 専門性はあるが全体最適を考えられない
> - 前職のやり方をそのまま持ち込もうとする
> - 成長フェーズの変化に適応できない
