## 概要

ニュースレター調査を完全自動化するマスターオーケストレーター。
以下の順序で116件以上の記事を自動処理します：

```text
┌─────────────────────────────────────────────────────────────────────┐
│  Newsletter Research Orchestration Flow v2.0                        │
├─────────────────────────────────────────────────────────────────────┤

│           └─ newsletter_articles_list.md を読み込み                 │
│                                                                     │
│  STEP 1:  📚 戦略ノウハウ記事調査（24件）                            │
│           └─ 5件/バッチ × 5バッチ                                   │

│           └─ 【バッチ検証】バッチ完了時に一括再検証                  │
│                                                                     │
│  STEP 2:  🛠️ ツール・AI活用記事調査（12件）                          │
│           └─ 5件 + 7件 の2バッチ                                    │
│           └─ 【個別検証】【バッチ検証】同上                          │
│                                                                     │
│  STEP 3:  📋 高収益事例調査（80件+）                                 │
│           └─ 5件/バッチ × 16+バッチ                                 │
│           └─ 【個別検証】【バッチ検証】同上                          │

│  STEP 4:  完了レポート生成                                          │

```

|--------|------|

## 中断・再開機能

オーケストレーションは中断しても再開可能：

1. **進捗の永続化**: 各バッチ完了時に `newsletter_articles_list.md` を更新
2. **再開時の動作**: 🔲ステータスの記事から自動的に継続
3. **手動中断**: ユーザーが停止した場合、次回は中断地点から再開

---

## Human-in-the-Loop ポイント

以下の状況では**確認を求める**：

| ポイント | 条件 | 選択肢 |
|---------|------|--------|
| カテゴリ完了時 | 各カテゴリ終了後 | 継続 / 一時停止 / 中止 |
| 大量エラー発生 | 1バッチで3件以上失敗 | リトライ / スキップ / 中止 |
| 50%完了時 | 中間チェックポイント | 継続 / レビュー要求 |
| 品質FAIL多発 | 1バッチで2件以上品質FAIL | 確認要求 |

## 実行コマンド

このワークフロー全体を開始するには：
```
/orchestrate_newsletter
```

個別カテゴリのみ実行する場合：
```
/orchestrate_newsletter 戦略のみ
/orchestrate_newsletter ツールのみ
/orchestrate_newsletter 事例のみ
```

進捗確認のみ：
```
/orchestrate_newsletter 進捗確認
```
