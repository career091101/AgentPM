# Web課題発見ワークフロー

顧客ヒアリングの代替として、ブラウザエージェントとWeb検索を活用してSNS・ブログ・Reddit等から課題の投稿を収集・分析し、課題仮説を裏付ける。

## 入力・出力

| 項目 | 内容 |
|------|------|
| **入力** | `lean_canvas.md`, `persona.md` |
| **出力** | `documents/2_discovery/problem_research.md` |
| **次のWF** | `/simulate_interview` → `/diagnose_cpf` |

## 目的

- 実在する課題の裏付け（エビデンス収集）
- 3U検証の精度向上
- CPF達成判定の信頼性向上
- 顧客インタビュー困難時の代替検証

---

## STEP 1: 検索クエリ生成【自動実行】

### 使用ツール
内部生成（ツール不要）

// turbo
`lean_canvas.md` と `persona.md` から課題仮説とペルソナを読み込み、最適な検索クエリを生成：

```markdown
## 検索クエリ生成

### 入力情報
- **課題仮説**: [lean_canvas.mdから抽出]
- **ターゲット**: [persona.mdから抽出]

### 生成クエリ

#### 日本語クエリ（5-10個）
1. "[課題キーワード] 困っている"
2. "[課題キーワード] 使いにくい"
3. "[課題キーワード] 不満"
4. "[ターゲット] [課題] 解決したい"
5. "[ツール/サービス名] 問題点"

#### 英語クエリ（5-10個）
1. "[problem keyword] frustration site:reddit.com"
2. "[problem keyword] struggling"
3. "[problem keyword] problem site:twitter.com"
4. "[target] [problem] hate"
5. "why [tool/service] sucks"

#### プラットフォーム別クエリ
- Reddit: "site:reddit.com [keyword]"
- X/Twitter: "site:twitter.com [keyword]"
- note: "site:note.com [keyword]"
- Qiita: "site:qiita.com [keyword]"
```

## STEP 2: Web検索実行【自動実行】

### 使用ツール
- `search_web`: 一般的なWeb検索
- `browser_subagent`: 詳細なページ内容をスクレイピング（必要に応じて）

// turbo
生成した各クエリで検索を実行：

### 検索対象プラットフォーム

| プラットフォーム | 特徴 | 言語 | 優先度 |
|----------------|------|:----:|:------:|
| **X (Twitter)** | リアルタイムの不満・課題 | 日/英 | ⭐⭐⭐ |
| **Reddit** | 詳細な課題説明・議論 | 英 | ⭐⭐⭐ |
| **note** | 日本の課題記事・体験談 | 日 | ⭐⭐ |
| **日本語ブログ** | 体験談・レビュー | 日 | ⭐⭐ |
| **Quora** | 質問形式の課題 | 英 | ⭐ |
| **Yahoo!知恵袋** | 日本の悩み相談 | 日 | ⭐ |
| **Product Hunt** | ツールへの不満コメント | 英 | ⭐ |
| **Qiita** | 技術系の課題 | 日 | ⭐ |

### 検索実行ルール
- 各クエリで3-5件の結果を取得
- 最低5つのプラットフォームから結果を収集
- 合計20-50件の投稿を目標

### エラー対応
| エラー | 対応 |
|-------|------|
| 検索結果0件 | クエリを言い換えて再検索 |
| API制限 | 30秒待機後リトライ（最大3回） |
| 関連性低い結果 | 別のキーワードで再検索 |

## STEP 3: 課題投稿の抽出【自動実行】

### 使用ツール
内部処理（必要に応じて `browser_subagent` で詳細取得）

// turbo
検索結果から課題に関する投稿を抽出し、以下の形式で記録：

```markdown
## 課題投稿一覧

### 投稿 #1
| 項目 | 内容 |
|------|------|
| **ソース** | [プラットフォーム名（例: Reddit r/ChatGPT）] |
| **URL** | [リンク] |
| **日付** | [投稿日または発見日] |
| **言語** | 日本語 / 英語 |
| **要約** | [1-2行での要約] |
| **原文（引用）** | > "実際の投稿内容をここに引用" |
| **感情分析** | 😤 不満 / 😢 困惑 / 😓 諦め / 😡 怒り / 🤔 疑問 |
| **深刻度** | ⭐⭐⭐⭐☆ (4/5) |
| **関連課題** | [lean_canvasのどの課題に対応するか] |

### 投稿 #2
...

### 投稿 #N
...
```

### 抽出基準
- ✅ 課題仮説に関連する投稿のみ
- ✅ 感情が読み取れる投稿を優先
- ✅ 具体的な状況説明がある投稿を優先
- ✅ 最近（1年以内）の投稿を優先
- ❌ 宣伝・スパム投稿は除外
- ❌ 単純な質問（課題ではない）は除外

## STEP 4: 課題のスコアリング【自動実行】

// turbo
収集した投稿を基に課題をスコアリング：

```markdown
## Web投稿スコアリング

### スコアリング基準

| 評価項目 | 配点 | 評価基準 |
|---------|:----:|---------|
| **言及頻度** | 0-10 | 10=50件以上, 7=20-49件, 5=10-19件, 3=5-9件, 0=0-4件 |
| **感情の強度** | 0-10 | 10=怒り/深刻, 7=強い不満, 5=不満, 3=軽い不満, 0=軽微 |
| **最近性** | 0-10 | 10=直近1ヶ月, 7=3ヶ月以内, 5=6ヶ月以内, 3=1年以内, 0=1年以上前 |
| **具体性** | 0-10 | 10=詳細な状況説明, 7=具体例あり, 5=概要のみ, 3=抽象的, 0=不明確 |
| **解決要望** | 0-10 | 10=お金払いたい発言, 7=強い解決希望, 5=解決希望, 3=愚痴のみ, 0=諦め |

**合計50点満点**

### スコアリング結果

| 課題 | 言及頻度 | 感情強度 | 最近性 | 具体性 | 解決要望 | 合計 |
|------|:-------:|:-------:|:-----:|:-----:|:-------:|:----:|
| [課題1] | X/10 | X/10 | X/10 | X/10 | X/10 | XX/50 |
| [課題2] | X/10 | X/10 | X/10 | X/10 | X/10 | XX/50 |
| [課題3] | X/10 | X/10 | X/10 | X/10 | X/10 | XX/50 |

### 総合判定

| スコア | 判定 | 意味 |
|:------:|:----:|------|
| 35-50 | ✅ 強い裏付け | 課題は市場に明確に存在する |
| 20-34 | ⚠️ 中程度の裏付け | 課題は存在するが追加検証推奨 |
| 0-19 | ❌ 裏付け不足 | 課題仮説の見直しが必要 |
```
