# AARRR ダッシュボードワークフロー

Dave McClure提唱のAARRR（Pirate Metrics）に基づき、プロダクト成長の各ステージを計測・可視化する。

> [!IMPORTANT]
> **PMF検証の必須指標**: AARRRはPMF達成度を測る重要なフレームワークです。
> 特にRetention（継続率）はPMFの最重要指標です。

## 入力・出力

| 項目 | 内容 |
|------|------|
| **入力** | 実績データ（アナリティクス等） |
| **出力** | `documents/5_monitoring/aarrr_dashboard.md` |
| **次のWF** | `/diagnose_pmf` または `/startup_scorecard` |

## 前提条件

- MVP/プロダクトがリリース済み
- 何らかの計測ツール（GA, Mixpanel等）が導入されている
- または手動での計測データがある

---

## STEP 1: AARRR概念の確認

```markdown
## AARRRとは

**Pirate Metrics**（海賊指標）とも呼ばれるスタートアップ成長指標。

```
A → A → R → R → R
獲得 → 活性化 → 継続 → 紹介 → 収益
```

| ステージ | 目的 |
|---------|------|
| **A**cquisition（獲得） | ユーザーをサイト/アプリに呼び込む |
| **A**ctivation（活性化） | 初回体験で価値を感じさせる |
| **R**etention（継続） | 繰り返し使ってもらう |
| **R**eferral（紹介） | 他者に紹介してもらう |
| **R**evenue（収益） | 課金してもらう |
```
