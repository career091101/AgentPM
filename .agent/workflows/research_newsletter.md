## 記事の種類と処理分岐

jabbaニュースレターには2種類の記事がある：

| 種類 | 説明 | テンプレート |
|------|------|-------------|
| **📋 事例調査** | 特定のニュースレター・運営者の詳細分析 | `newsletter_case_study_template.md` |

**判定基準**:

### STEP 9: ファクトチェック（5フェーズ強化版 v3.2）

> [!CAUTION]
> このステップは **省略禁止** です。戦略記事でも事例記事でも必ず実行してください。
> ファクトチェック結果セクションには以下の5つのサブセクションを全て含める必要があります。

```text
必須サブセクション:
  1. 元記事確認テーブル
  2. 引用事例の検証テーブル
  3. 数値データの検証テーブル
  4. URL/リンク確認テーブル
  5. 総合判定（確認済み項目数、未確認項目、推奨アクション）

判定基準:
  - ✅ PASS: 2ソース以上で確認完了
  - ⚠️ 一部確認: 1ソースのみ、または自己申告値
  - ❌ FAIL: URL無効、データ乖離20%以上、確認不可
```

#### 9-1. 収益データ照合

// turbo-mandatory
1. 収集した収益データを一覧化（ソース別）
2. 最大値と最小値の乖離率を計算
3. **乖離が20%以上**の場合は NG 判定

```text
計算式: 乖離率 = (最大値 - 最小値) / 最大値 × 100
例: ソースA: $50,000, ソースB: $38,000
    乖離率 = (50000 - 38000) / 50000 × 100 = 24% → NG
```

#### 9-2. URL存在確認

// turbo
1. `read_url_content` または `browser_subagent` でニュースレターURLにアクセス
2. 404エラー、ドメイン失効の場合は NG 判定
3. 正常にアクセスできれば PASS

#### 9-3. Xアカウント確認

// turbo
1. Xプロフィールにアクセス
2. アカウント凍結・削除の場合は NG
3. 1年以上投稿がない場合は WARN
4. アクティブな場合は PASS

#### 9-4. 判定とログ記録

| 判定 | 条件 | アクション |
|------|------|------------|
| ✅ **PASS** | 全必須項目が2ソース以上で確認、乖離20%未満 | STEP 10 へ |
| ⚠️ **WARN** | 1-2項目が単一ソースのみ | 警告付きでSTEP 10へ |
| ❌ **NG** | 乖離20%以上 or URL/Xに問題あり | **STEP 9.5（再調査）へ** |

// turbo
1. 判定結果を事例ファイル内の「ファクトチェック履歴」に記録
2. `factcheck_log.md` にも詳細ログを追記

3. 不足セクションが追加されているか
4. ファイルが正しく更新されているか

```text

```

## エラーハンドリング

| エラー | 対応 |
|--------|------|
| jabba記事が有料で読めない | 無料部分のみ抽出 + 他ソースで補完 |
| X アカウントが見つからない | Web 検索で `"[人物名]" twitter` を試行 |
| X がブロック/凍結 | 他ソースを優先して調査続行 |
| ニュースレターが閉鎖済み | 「⚠️ 閉鎖済み」と記録、過去情報のみ記載 |
| Substack/beehiiv でブロック | 直接URLアクセスを試行 |
| 情報が極端に少ない | 調査困難としてスキップ、別件へ |
| 英語ページの翻訳が必要 | AI要約で対応 |

## 注意事項

- 1件あたり約30分程度の調査時間を想定
- jabbaニュースレターの有料部分は読み込み不可（無料部分 + 他ソースで補完）
- ニュースレター特有のKPI（開封率、解約率等）を重視
- 収益データは最新のものを優先
- 著作権に配慮し、情報源を明記
- **事例と戦略は別ディレクトリに保存**
