# スタートアップ検証チェックリスト

> **出典**: 田所雅之 著「起業の科学」「起業大全」「起業参謀の教科書」
> **目的**: プロジェクトのステージに応じた検証項目の確認
> **最終更新**: 2025-12-25

---

## ステージ判定

現在のステージを特定してください：

- [ ] **Idea**: アイデア段階
- [ ] **CPF**: 顧客課題の検証中
- [ ] **PSF**: ソリューションの検証中
- [ ] **PMF**: プロダクト市場適合の検証中
- [ ] **Scale**: スケール段階

---

## 1. Idea（アイデア検証）チェック

### 1.1 アイデアの質
- [ ] 自分が欲しいと思うプロダクトか？
- [ ] 課題が具体的に言語化されているか？
- [ ] 既存の解決策の問題点を理解しているか？
- [ ] Founder-Issue-Fit があるか？（自分がこの課題を解くべき理由）

### 1.2 避けるべきアイデア（7つのパターン）
- [ ] ❌ 誰の何を解決するか不明確なアイデアではない
- [ ] ❌ ソリューションありきのアイデアではない
- [ ] ❌ 自分が欲しくないアイデアではない
- [ ] ❌ 分析からのみ生まれたアイデアではない
- [ ] ❌ 激しい競争市場への参入ではない
- [ ] ❌ 一言で説明できる明確さがある
- [ ] ❌ 既存サービスのコピーではない

---

## 2. CPF（Customer Problem Fit）チェック

### 2.1 顧客理解
- [ ] ターゲット顧客を明確に定義しているか？
- [ ] 顧客インタビューを最低20人実施したか？
- [ ] ペルソナを作成したか？

### 2.2 課題検証（3U検証）
- [ ] 課題は「**Unsolved**（未解決）」か？
- [ ] 課題は「**Urgent**（緊急）」か？
- [ ] 課題は「**Expensive**（高いコストを払っている）」か？
- [ ] 顧客が「お金を払ってでも解決したい」課題か？
- [ ] 課題の深刻度を1-10でスコアリングしたか？
- [ ] 課題が頻繁に発生するものか？

### 2.3 CPF達成基準
- [ ] 「この課題を解決したいですか？」に80%以上が「はい」

---

## 3. PSF（Problem Solution Fit）チェック

### 3.1 ソリューション検証
- [ ] ソリューションが課題を解決するか検証したか？
- [ ] ペーパープロトタイプまたはMVPを作成したか？
- [ ] 顧客からフィードバックを得たか？
- [ ] 既存の代替案より**10倍優れている**か？

### 3.2 MVP種類の検討（作る前に売る）
- [ ] **オファー型MVP**: LP事前登録で需要検証したか？
- [ ] **コンシェルジュ型MVP**: 手動でサービス提供を試したか？
- [ ] **オズの魔法使い型MVP**: 裏側人力システムを検討したか？

### 3.3 価値提案
- [ ] 独自の価値提案（UVP）を1文で説明できるか？
- [ ] 競合との差別化ポイントが明確か？

### 3.4 PSF達成基準
- [ ] 「このソリューションを使いたいですか？」に70%以上が「はい」
- [ ] 「お金を払ってでも使いたい」と言う顧客がいるか？

---

## 4. PMF（Product Market Fit）チェック

### 4.1 PMFパルテノン神殿チェック
- [ ] 土台: Vision / Founder-Issue-Fit が確立しているか？
- [ ] 1階: CPF (Customer Problem検証) 完了か？
- [ ] 2階: PSF (Value Proposition検証) 完了か？
- [ ] 3階: 熱狂の再現性、チャネル検証ができているか？

### 4.2 定量指標
- [ ] NPS（Net Promoter Score）が40以上か？
- [ ] 継続利用率が目標を達成しているか？
- [ ] 口コミによる自然流入があるか？
- [ ] リテンションレートが平坦化しているか？（40%ルール）

### 4.3 ショーン・エリスのPMFテスト
- [ ] 「このプロダクトが使えなくなったらどう思いますか？」
  - 「非常に残念」が40%以上か？

### 4.4 BtoB特有チェック
- [ ] 導入企業のチェンジマネジメントができているか？
- [ ] 複数ステークホルダー（決裁者・管理者・利用者）全員がサクセスしているか？

### 4.5 ユニットエコノミクス
- [ ] LTV（顧客生涯価値）を算出したか？
- [ ] CAC（顧客獲得コスト）を算出したか？
- [ ] LTV > CAC × 3 を達成しているか？

### 4.6 PMF達成基準
- [ ] 月次成長率が一定以上を維持
- [ ] 解約率（Churn Rate）が低水準

---

## 5. Scale チェック

### 5.1 成長準備
- [ ] 再現可能なセールスプロセスがあるか？
- [ ] オペレーションがスケール可能か？
- [ ] チーム体制が整っているか？

### 5.2 フライホイール
- [ ] 成長のフライホイールを構築しているか？
- [ ] ネットワーク効果を活用しているか？

### 5.3 DEARモデル（カスタマーサクセス）
- [ ] **D**eployment: オンボーディング完了率を計測しているか？
- [ ] **E**ngagement: DAU/MAU、NPS、ログイン頻度を追跡しているか？
- [ ] **A**doption: 重要機能の利用率を計測しているか？
- [ ] **R**OI: 顧客の成果・費用対効果を可視化しているか？

## 6. ステージ別KPI参照表

| フェーズ | 注目KPI | 主なアクション |
|---------|---------|---------------|
| **CPF/PSF** (Pre-Seed) | 顧客インサイト獲得度、ソリューションの刺さり方 | 定性インタビュー、課題深刻度検証 |
| **PMF検証** (Seed) | AARRR、解約率、NPS | プロダクト磨き込み、熱狂的ファン獲得 |
| **Scale前夜** (Series A) | LTV/CPA、MRR成長率 | 再現性チャネル構築、組織化 |
| **Scale期** (Series B~) | GMV/売上成長率、バリュエーション | 大規模マーケ、権限委譲、M&A |

## 7. ファイナンスチェック

### 7.1 資本政策
- [ ] 持分比率を適切に設計しているか？（創業者持分の死守）
- [ ] ランウェイ18ヶ月分を確保しているか？

### 7.2 フェーズ別ファイナンスCSF

| フェーズ | 重要成功要因 |
|---------|------------|
| Pre-seed | 最初の資本政策で失敗しない |
| Seed | PMFを見据えたランウェイ確保 |
| Series A | ユニットエコノミクスの健全化証明 |
| Series B~ | ガバナンス、予実管理精度 |

### 7.3 バリュエーション要素
- [ ] 技術的優位性/知財があるか？
- [ ] 独自データ/Data Driven経営か？
- [ ] ネットワーク効果を構築しているか？
- [ ] 第一想起ブランドを獲得しているか？
- [ ] 経営陣の経験と実行力が十分か？

## 9. ピボット判断チェック

### 9.1 ピボットを検討すべきサイン
- [ ] 3ヶ月以上、主要指標が改善していない
- [ ] 顧客の反応が継続的に悪い
- [ ] ユニットエコノミクスが成り立たない

### 9.2 ピボットの10類型（検討用）
1. Zoom-in（機能切り出し）
2. Zoom-out（機能統合）
3. Customer Segment（顧客変更）
4. Customer Need（課題変更）
5. Platform（プラットフォーム化）
6. Business Architecture（B2B ↔ B2C）
7. Value Capture（収益モデル変更）
8. Engine of Growth（成長エンジン変更）
9. Channel（販路変更）
10. Technology（技術変更）

### 9.3 ピボット実行前確認
- [ ] チーム全員がピボットに合意しているか？
- [ ] ビジョンは維持されているか？
- [ ] これまでの学びを活かしたピボットか？

## 10. 「ムリ・ムダ・ムラ」排除チェック

- [ ] **ムリ**: リソースに見合わない目標設定をしていないか？
- [ ] **ムダ**: 顧客が欲しがらないものを作っていないか？
- [ ] **ムラ**: 成長スピードと経営陣の成長が一致しているか？
- [ ] プレマチュア・スケーリング（PMF前の大量採用等）をしていないか？
