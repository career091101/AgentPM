# Note アルゴリズム解説

## 概要

Noteは、日本独自の長文コンテンツプラットフォームです。クリエイターが2,000-5,000文字の記事を投稿し、有料note（500-2,000円/記事）やクリエイターサポート（投げ銭）で収益化できるのが特徴です。

本ドキュメントは、425ソースの分析から抽出された36件のNote関連情報（アルゴリズム6件、BP10件、収益化8件、成功事例12件）を基に、日本市場特化のアルゴリズム活用方法を解説します。

**総合評価**: 70/100 (Good)
**カバー率**: アルゴリズム60%、BP70%、収益化85%、成功事例75%

---

## Noteプラットフォーム特性

### Noteの特徴

**1. 日本語圏限定**
- 海外競合ゼロ
- 日本文化に最適化
- 有料note購入文化が定着

**2. 長文コンテンツ優位**
- 2,000-5,000文字が推奨
- LinkedInより長い、ブログ形式
- じっくり読む文化

**3. 収益化機能が充実**
- 有料note（記事単位販売）
- クリエイターサポート（投げ銭）
- 定期購読マガジン（月額制）

**4. SNS連携が容易**
- X/Instagramとのクロスプロモーション
- note記事のティザーをSNSで拡散
- 外部流入がメイン

### 主要成功者プロフィール

**kensu**:
- 戦略: note × X併用、多角的収益化
- エンゲージメント率: 2.5%（X/Note併用）
- 収益源: Product Sales, Community, Consulting, NFT
- 特徴: 専門性発信 → 信頼構築 → 各収益源への導線

**yukos**:
- 戦略: Instagram/X/YouTube/note マルチプラットフォーム
- 投稿頻度: 20投稿/週を複数媒体に分散
- note活用法: SNSコンテンツを深掘り、長文記事で専門性アピール、有料noteで収益化

**ikehaya氏**:
- 領域: NFT/Web3
- 戦略: noteでNFTノウハウ発信
- 特徴: コミュニティ × note × NFT連携

**ゴールデンタイム**:

**信頼度**: MEDIUM

#### 評価指標の優先順位（重要度順）

**1. 読了率（Finish Rate）** - 最重要
- どれだけ最後まで読まれたか
- 重み: ★★★★★

**2. 滞在時間**
- 記事内での滞在時間の長さ
- 重み: ★★★★★

**3. クリック率（CTR）**
- タイトル・サムネイルのクリック率
- 重み: ★★★★☆

**4. スキ率（Like Rate）**
- 読者のうち何%が「スキ」を押したか
- 重み: ★★★★☆

**5. 外部流入**
- 外部サイトからnoteへの流入
- 重み: ★★★☆☆

**信頼度**: MEDIUM
**ソース**: noteで人気順に表示される条件とは？、noteのアルゴリズム完全解析

### エンゲージメントの相対価値（信頼度: MEDIUM）

**1コメント = 100スキ**:
- クリエイター間で広く認識されている相対価値
- 理由: コメントは記事を実際に読み、考えて書く必要があるため

**ランキングへの影響**:
- **人気順**: スキ数が多ければ少なくとも人気順には表示される
- **急上昇**: フォロワー数に対するスキ比率が重要（少フォロワーでも可能）
- **コメント**: 感情的価値は高いが、アルゴリズム上の直接的影響は不明確

**信頼度**: MEDIUM
**ソース**: noteの1コメントは100スキくらいうれしい、なぜこのnoteだけ200スキ？

### ハッシュタグの効果（信頼度: MEDIUM-HIGH）

**公式推奨**:
- **推奨個数**: **3-5個**のハッシュタグ
- **信頼度**: HIGH
- **ソース**: note公式ヘルプセンター

**効果的な使い方**:
- **3-4個の高品質タグ**: 記事テーマを正確に伝えるタグを厳選
- **一般タグと専門タグのバランス**: 幅広いリーチと特定ニッチのバランス
- **公式トピックタグ**: 特に2週間以内に作成されたトピックタグが効果的
- **募集タグ**: note編集部が見つけやすい

**注意点**:
- **20個程度のタグ**: 「人気」「急上昇」タブから除外される可能性
- **単純なタグ設定だけでは効果薄**: アクセス増加には直結しない

**検証結果**:
- ハッシュタグは主に「新着」タブでの発見性を高める
- 人気キーワードでは「新着」にしか表示されない可能性が高い
- ニッチなキーワードの方が上位表示されやすい

**信頼度**: MEDIUM
**ソース**: noteでハッシュタグをつけて投稿すると全体ビューにどれだけ貢献するのか、保存版:Noteのハッシュタグ攻略ガイド

## 長文記事戦略

### 推奨文字数と構造

**最適文字数**: 2,000-5,000文字

**理由**:
- noteの文化として長文が受け入れられる
- LinkedInより長い、ブログ形式
- 有料noteは2,000文字以上が推奨

**プラットフォーム別文字数比較**:
| プラットフォーム | 推奨文字数 | 特徴 |
|----------------|----------|------|
| X (Twitter) | 100-280文字 | 短文、リアルタイム |
| LinkedIn | 1,200-2,000文字 | 長文、B2B特化 |
| Note | 2,000-5,000文字 | 超長文、ブログ形式 |

### 記事構造の最適化（推定）

**注意**: 記事構造の詳細は不明（gap_analysis.mdで「High Priority Gap」として指摘）。以下は推定です。

**推奨構造**:

**1. 導入（300-500文字）**
- 問題提起・課題設定
- 読者の共感を得る
- 記事の価値提案（この記事で何が得られるか）

**2. 本論（1,500-4,000文字）**
- 見出し構造（H2, H3の使い方）
- 画像挿入（何文字ごとに1枚？→ 不明）
- 引用・リンクの配置
- ステップバイステップ解説

**3. 結論（200-500文字）**
- キーポイントの再確認
- アクションを促すCTA
- クリエイターサポート誘導

**視覚的要素**:
- 画像挿入頻度: 500-1,000文字ごとに1枚（推定）
- 見出し構造: H2（大見出し）、H3（小見出し）で階層化
- 引用: 重要なポイントを引用形式で強調
- 箇条書き: 読みやすさ向上

**スマホ vs PC**:
- noteはスマホ閲覧が多い（推定）
- 短い段落（3-4行）で改行
- 視覚的な区切りを多用

**推奨調査方法**（gap_analysis.mdより）:
```
成功事例分析: 高スキ記事の構造パターン
A/Bテスト: 画像多め vs 少なめ記事
```

## クロスプラットフォーム連携戦略

### note × SNS連携導線

**標準パターン**:
```
X/Instagram投稿（ティザー）
    ↓ 注目喚起、価値提案
noteリンク誘導
    ↓ 長文記事で価値提供
有料note/クリエイターサポート
    ↓ 収益化
Community/Consulting誘導
    ↓ 長期的関係構築
```

**適用事例**: kensu, yukos

### ティザー投稿の設計

**X (Twitter) ティザー**:
- 投稿内容: noteのキーポイントを1ツイートで要約
- フック: "詳細はnoteで"
- リンク配置: リプライ or 引用RT
- 投稿頻度: note公開直後 + 翌日リマインド

**Instagram ティザー**:
- 投稿内容: ビジュアル（noteの見出し画像）+ キャプション
- フック: "プロフィールリンクから全文読めます"
- ストーリー活用: リンクスタンプで直接誘導
- 投稿頻度: note公開当日

### マルチプラットフォーム戦略の効果

**メリット**:
- リーチの最大化（各媒体の強みを活用）
- リスク分散（単一プラットフォーム依存回避）
- クロスプロモーション効果

**デメリット**:
- 時間投資大（各媒体のコンテンツ制作）
- 一貫性の維持が困難
- 分析・最適化の複雑化

**推奨フェーズ**:
- 初期（0-6ヶ月）: X + note（2媒体集中）
- 成長（6-12ヶ月）: X + Instagram + note（3媒体展開）
- 成熟（12ヶ月~）: 全媒体最適化（kensu, yukos型）

## 有料note戦略

### 有料note vs 無料noteの比率

**現状**: 最適な有料/無料比率は不明（gap_analysis.mdで「High Priority Gap」として指摘）。

**推定パターン**:

**パターンA: 7:3（無料7割、有料3割）**
- 戦略: 無料記事で信頼構築 → 有料note販売
- 適用: 初期~成長フェーズ
- メリット: 無料記事でリーチ拡大
- デメリット: 収益化まで時間

**パターンB: 5:5（無料5割、有料5割）**
- 戦略: 無料と有料をバランス
- 適用: 成長~成熟フェーズ
- メリット: 収益と認知のバランス
- デメリット: 有料noteの価値提供が課題

**パターンC: 無料記事でコミュニティ → 有料noteは深掘り**
- 戦略: 無料記事は導入・概要、有料noteは詳細・実践
- 適用: 全フェーズ
- メリット: 無料記事で興味喚起 → 有料noteで収益化
- デメリット: 両方の質維持が困難

### 価格設定戦略

**価格帯**: 500-2,000円/記事

**価格別の特徴**:

**500円**:
- 購入ハードル低
- 初回購入に適合
- ボリューム: 2,000-3,000文字（推定）

**1,000円**:
- 標準的な価格
- バランス型
- ボリューム: 3,000-5,000文字（推定）

**2,000円**:
- 高価格・高価値
- 専門性・希少性が必要
- ボリューム: 5,000文字以上（推定）

**推定購入率**:
- 500円: 3-5%（推定）
- 1,000円: 2-3%（推定）
- 2,000円: 1-2%（推定）

**注意**: 購入率の詳細データは不明（gap_analysis.mdで調査推奨）。

**推奨調査方法**（gap_analysis.mdより）:
```
実践者インタビュー: 有料note販売者
データ分析: 価格別の購入率

```

## 日本市場への適用可能性

### 日本市場での再現性

**再現性: 極めて高い**

**理由**:
1. **日本独自プラットフォーム**
