### 出力形式

- ファイル: `chunks/all_chunks.jsonl`
- フォーマット: JSONL（1行1チャンク）
- チャンク単位: セマンティックチャンキング（見出し単位）

### チャンクメタデータ

各チャンクには以下のメタデータが付与されています：

```json
{
  "content": "チャンク本文",
  "heading": "見出しテキスト",
  "heading_level": 2,
  "source_file": "blog/instagram/...",
  "title": "記事タイトル",
  "url": "記事URL",
  "category": "blog"
}
```

## 💻 使用方法

### 1. コンテンツの閲覧

```bash
# ブログ記事一覧
ls -lh blog/instagram/
ls -lh blog/twitter/
ls -lh blog/tiktok/

# 対談動画一覧
ls -lh interviews/

# ZOOMコンサル一覧
ls -lh zoom_consult/

# PDF資料
ls -lh resources/pdf/
```

### 2. RAGチャンクの読み込み

```python
import json

# チャンクを読み込み
chunks = []
with open('chunks/all_chunks.jsonl', 'r', encoding='utf-8') as f:
    for line in f:
        chunks.append(json.loads(line))

print(f"総チャンク数: {len(chunks)}")

# カテゴリ別フィルタリング
instagram_chunks = [c for c in chunks if 'instagram' in c.get('source_file', '')]
twitter_chunks = [c for c in chunks if 'twitter' in c.get('source_file', '')]
tiktok_chunks = [c for c in chunks if 'tiktok' in c.get('source_file', '')]

print(f"Instagram: {len(instagram_chunks)}チャンク")
print(f"Twitter: {len(twitter_chunks)}チャンク")
print(f"TikTok: {len(tiktok_chunks)}チャンク")
```

### 3. 再取得・更新

コンテンツを再取得する場合：

```bash
cd scripts
source venv/bin/activate

# 全コンテンツ取得
python scraper.py

# RAG用チャンク分割

```

## 🛠️ スクリプト一覧

### scraper.py

メインスクレイパー。ホームページから以下を取得：
- ブログ記事（Instagram/Twitter/TikTok）
- 対談動画（YouTube URL + テキスト）
- ZOOMコンサル（YouTube URL）

### fetch_missing_content.py

特定URLリストから追加コンテンツを取得：
- 未取得ブログ記事
- PDF資料のダウンロード

### verify_completeness.py

完全性検証ツール：
- サイト上の総コンテンツURL収集
- 取得済みURLとの差分検出
- 未取得コンテンツのリスト化

### youtube_transcriber.py

YouTube文字起こしツール：
- YouTube URLから動画IDを抽出
- yt-dlpで字幕取得（利用可能な場合）
- Whisper APIで音声文字起こし

### chunker.py

RAGチャンク分割ツール：
- Markdownファイルをセマンティックチャンキング
- 見出し単位でチャンク分割
- メタデータ付与してJSONL形式で出力
