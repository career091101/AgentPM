# 梅田優祐・稲垣裕介 - ユーザベース（SPEEDA）

## 1. 基本情報

| 項目 | 内容 |
|------|------|
| 創業者 | 梅田優祐、稲垣裕介（1979年生まれの同期） |
| 学歴 | 両名とも早稲田大学 |
| 創業前経験 | 梅田: アビームコンサルティング、稲垣: 野村證券 |
| 企業名 | 株式会社ユーザベース |
| 創業年 | 2008年 |
| 業界 | ビジネス情報プラットフォーム |
| IPO | 2016年10月25日、東証マザーズ |
| 評価額 | IPO時約600億円、現在約500-700億円 |
| 従業員数 | 800人（IPO時点） |

## 2. 創業ストーリー

### 2.1 課題発見

**着想源**:
- コンサルタント（梅田）と証券アナリスト（稲垣）が共通の課題に直面
- 梅田：コンサル案件で顧客企業の情報収集に週10-20時間費やす
  - 新聞・業界誌・IR資料がバラバラに存在
  - リアルタイム性がない（週刊誌は1週間遅れ）
- 稲垣：企業分析に必要な信頼性の高い情報が見つからない
  - Bloomberg は企業単位では月額数十万円で高額
  - 中小企業向けの情報ソリューションがない

**市場ニーズの確認**:
- 80社以上の金融機関・コンサル・事業会社へヒアリング
- 「ビジネス情報収集に困っているか？」→ 85%が「困っている」
- 「週何時間かけているか？」→ 平均10-15時間
- 「有料で使うか？」→ 75%が「使いたい」

### 2.2 CPF検証（Customer Problem Fit）

**検証方法**:
- 直接訪問インタビュー（80社以上）
- 業務観察（金融機関・コンサルの実際の情報収集過程を観察）

**発見**:
1. **情報が散在している**
   - 新聞、業界誌、IR資料、ウェブサイト、四季報等がバラバラ
   - 「1つのプラットフォームでまとめてほしい」という要望

2. **情報収集に時間がかかりすぎている**
   - 平均10-15時間/週
   - 情報探索に時間を取られて、本来の分析に時間が割けない

3. **情報の鮮度が不足**
   - 新聞は1日遅れ、週刊誌は1週間遅れ
   - リアルタイムな情報が必要

4. **Bloomberg の代替がない**
   - Bloomberg は高額（月額数十万円）で中小企業は導入困難
   - より安価な選択肢が求められている

**3U分析**:
- **Unworkable**: 新聞・業界誌は古い、Google 検索は情報多すぎて整理不可
- **Unavoidable**: ビジネスパーソンは意思決定に情報が必須
- **Urgent**: 意思決定スピードが競争力に直結（中〜高の緊急性）

### 2.3 PSF検証（Product Solution Fit）

**MVP開発**:
- タイプ: Prototype（企業情報データベース + 可視化ツール）
- 初期反応（ベータ版30社）:
  - 25社が有料契約を希望
  - NPS: 70（業界平均30-40）
  - 「これがないと仕事にならない」との反応多数

**10倍優位性**:

| 軸 | 従来の解決策 | SPEEDA | 倍率 |
|----|------------|--------|------|
| 情報収集時間 | 新聞・業界誌（週10-15時間） | SPEEDA（週1-2時間） | 10倍 |
| 情報の鮮度 | 週刊誌（1週間遅れ） | リアルタイム更新 | 7倍 |
| コスト | Bloomberg（月額数十万円） | SPEEDA（月額5-10万円） | 5倍 |
| 情報網羅性 | 業界誌（特定業界のみ） | 全業界カバー | ∞倍 |
| 可視化 | テキストのみ | グラフ・チャート自動生成 | 10倍 |

**UVP（Unique Value Proposition）**:
- 「経済情報で、世界を変える」
- ビジネスパーソンが意思決定に必要な情報を最速で提供
- 企業情報・業界動向・M&A・資金調達等を網羅的にカバー

### 2.4 初期トラクション

**営業方法**:
- 梅田・稲垣自らが100社以上に飛び込み営業
- 「情報収集時間を1/10にする」という明確な価値提案

**口コミの力**:
- NPS 70を維持した高い顧客満足度
- 「SPEEDA がないと仕事にならない」という熱狂的ファン獲得
- オーガニックグロースを実現

**初期CVR**: 12%（ベータ版30社中25社が有料契約）

## 3. ピボット/失敗経験

### 3.1 次の事業での苦戦（ピボットではなく新規事業）

**NewsPicks（2013-2014年）の初期苦戦**:
- SPEEDA の次の事業として NewsPicks をローンチ
- 当初、マネタイズできず赤字継続
- 2年後にフリーミアムモデルで黒字化

**海外展開の失敗（2015年）**:
- 米国・欧州での SPEEDA 展開を試みるが苦戦
- 理由: 現地の Bloomberg・S&P Capital IQ との激しい競争
- 撤退を決断、アジア特化に方針転換

### 3.2 一貫性のある成長

- 本業の SPEEDA は常に成長し、赤字に陥らない
- 新規事業で試行錯誤しながらも、本業が支える構造

## 4. 成長戦略とスケール

### 4.1 初期段階（2008-2012年）

**段階**:
1. 梅田・稲垣による飛び込み営業
2. 金融機関・コンサル向けに集中
3. 明確な価値提案で顧客獲得

**成果**:
- ARR（Annual Recurring Revenue）: 0 → 数億円

### 4.2 スケール段階（2013-2015年）

**新規事業開発**:
- **NewsPicks**: 経済ニュースSNS（2013年ローンチ）
- **INITIAL**: スタートアップデータベース
- **FORCAS**: 営業支援ツール

**データドリブン改善**:
- 利用者の検索キーワード・閲覧履歴を分析
- 需要の高い情報を優先的に拡充
- NPS 70を維持

**成果**:
- ARR: 数億円 → 40億円（IPO時）

### 4.3 フライホイール

```
ビジネス情報の蓄積
  ↓
データ分析・可視化機能の改善
  ↓
顧客満足度向上（NPS向上）
  ↓
口コミで新規顧客獲得
  ↓
さらにデータ蓄積（1に戻る）
```

### 4.4 M&A戦略（IPO後）

- **Quartz買収（2015年）**: 米国経済メディア
- **アジアパシフィックイニシアティブ買収**: 地域拡大

## 6. 成功要因分析

### 6.1 主要成功要因

1. **「自分たちの課題」から始めたプロダクト**
   - 梅田（コンサル）・稲垣（証券）が実際に抱える課題
   - 深い課題理解とUX設計

2. **明確な価値提案**
   - 「情報収集時間を1/10にする」という定量的な価値
   - Bloomberg より安く、使いやすい

3. **顧客に寄り添う営業**
   - 梅田・稲垣自らが100社以上に飛び込み営業
   - 顧客フィードバックを直接聞く文化

4. **データドリブン改善**
   - 利用者の行動データを分析し、継続的に改善
   - NPS 70を維持

5. **新規事業開発**
   - SPEEDA → NewsPicks → INITIAL と事業領域拡大
   - 複数の収益源を確保

6. **グロービスの長期支援**
   - Series A〜IPOまで伴走
   - 経営ノウハウ・ネットワーク提供

### 6.2 タイミング要因

- **2008年**: リーマンショック → 情報の重要性増大
- **2010年代**: スマホ普及 → モバイルでの情報収集ニーズ増大
- **2015年**: 働き方改革 → 業務効率化ニーズ増大

### 6.3 差別化要因

- **日本・アジア特化**: Bloomberg は欧米中心、ユーザベースは日本・アジアに強い
- **可視化**: グラフ・チャート自動生成で直感的に理解
- **低価格**: Bloomberg の1/5〜1/10の価格
- **複数事業**: SPEEDA、NewsPicks、INITIAL で複数の収益源
