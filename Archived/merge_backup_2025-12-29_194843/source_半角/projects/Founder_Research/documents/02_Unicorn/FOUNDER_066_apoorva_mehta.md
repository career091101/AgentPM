# Apoorva Mehta - Instacart

## 1. 基本情報

| 項目 | 内容 |
|------|------|
| 創業者 | Apoorva Mehta |
| 生年 | 1986年（インド・ジョードプル生まれ） |
| 国籍 | インド系カナダ人 |
| 学歴 | University of Waterloo（電気工学、2008年卒業） |
| 創業前経験 | BlackBerry、Qualcomm、Amazon（サプライチェーンエンジニア、2008-2010年） |
| 企業名 | Instacart |
| 創業年 | 2012年 |

## 2. 創業ストーリー

### 2.1 課題発見（需要発見）

**着想源**:
- 2012年夏、サンフランシスコのアパートで冷蔵庫を開けると、中にはスリラチャソースのボトルしかなかった
- 車を持っておらず、忙しい生活の中で食料品を買いに行く時間がなかった
- カナダで育った時代、バスで食料品店に行く不便さを経験していた

**需要検証方法**:
- 「2012年時点で、ほぼすべてのものをオンラインで注文できたが、食料品だけはオフラインのままだった」という市場ギャップに着目
- スマートフォンの普及と、消費者がアプリベースの取引に慣れてきたタイミングを認識

### 2.2 CPF検証（Customer Problem Fit）

**インタビュー/顧客検証**:
- 実施数: 正式なインタビューよりも自己体験に基づく検証
- 手法: 自分自身が最初の顧客として、買い物から配達まで全プロセスを体験
- 発見した課題の共通点: 忙しいプロフェッショナルにとって、食料品の買い物は大きな時間的負担

**3U検証**:
- Unworkable（現状では解決不可能）: 既存の食料品デリバリーは品揃えが限定的で、好みの店舗から買えない
- Unavoidable（避けられない）: 人は食料品を定期的に購入する必要がある（週1-2回）
- Urgent（緊急性が高い）: 忙しい都市部の消費者にとって時間は最も貴重なリソース

**支払い意思（WTP）**:
- 確認方法: 実際のサービス提供を通じた検証
- 結果: 忙しいプロフェッショナルは配達料を支払う意思があることを確認

### 2.3 PSF検証（Problem Solution Fit）

**10倍優位性**:

| 軸 | 従来の解決策 | 自社ソリューション | 倍率 |
|---|------------|-----------------|------|
| 時間 | 店舗往復1-2時間 | アプリで数分で注文、1時間配達 | 10x |
| 利便性 | 自分で店舗に行く必要 | 自宅で受け取り | 5x |
| 品揃え | 1店舗の商品のみ | 複数店舗から選択可能 | 3x |
| 柔軟性 | 営業時間内のみ | 早朝から深夜まで対応 | 2x |

**MVP**:
- タイプ: Concierge MVP
- 初期反応: 3週間でアプリの初期バージョンを開発。最初の注文は自分自身に配達して検証
- 創業者自身が買い物と配達を行い、顧客体験を深く理解

**UVP（独自の価値提案）**:
- 好きな地元の店舗から、1時間以内に食料品を届ける

**競合との差別化**:
- 既存小売店とのパートナーシップ（自社在庫を持たないアセットライトモデル）
- パーソナルショッパーによる代替品の相談機能
- 1,400以上の小売ブランド、80,000店舗以上との提携

## 3. ピボット/失敗経験

### 3.1 初期の失敗

Instacart創業前、MehtaはAmazonを退職後2年間で約20のスタートアップを立ち上げ、すべて失敗した：

- **弁護士向けソーシャルネットワーク**: 1年間開発したが、弁護士の日常業務に関心が持てず、情熱を維持できなかった
- **ゲーム会社向け広告プラットフォーム**: 技術的には機能したが、市場のニーズと合致しなかった
- その他多数のアイデアを試行

**失敗から学んだこと**:
- 「失敗は予想していた。最初の会社が成功するとは思っていなかった」
- 自分が本当に気にかける問題を解決することの重要性
- 迅速なプロトタイピング、ビジネス戦略、起業家としての心理管理を習得

### 3.2 Y Combinator参加のストーリー

Instacartは特異な方法でY Combinatorに参加した：

1. 2012年6月、応募締め切りを2ヶ月以上過ぎていた
2. YCパートナーのGarry Tanから「遅すぎる」と断られた
3. 最後の手段として、InstacartアプリでビールをY Combinator本部のGarry Tanに配達
4. 30分後にGarryから電話があり「これは何だ？」と質問
5. 「これがInstacartです！」と答え、翌日面接の機会を獲得
6. 4人のYCパートナーと1時間近く面接し、史上初めて締め切り後に参加を認められた

## 4. 成長戦略

### 4.1 初期トラクション獲得

- **口コミマーケティング**: 初期はマーケティング部門を持たず、口コミのみで成長
- **サンフランシスコ集中**: 地理的に限定してサービスを最適化
- **プレミアム小売パートナーシップ**: Whole Foodsなど高品質店舗との提携で信頼性を確立

### 4.2 フライホイール

```
より多くのショッパー → 配達時間短縮 → 顧客満足度向上
       ↑                                    ↓
    収益増加 ← より多くの注文 ← より多くの顧客
       ↑                                    ↓
より多くの小売パートナー ← 販売チャネル拡大 ←
```

### 4.3 スケール戦略

**資金調達**:
- Y Combinator: $150,000（2012年）
- シード: $230万（2012年）
- シリーズA: $850万（2013年）
- COVID-19前後で急成長、2021年には$39Bの評価額

**地理的拡大**:
- 2012年: サンフランシスコ
- 2013年: シカゴ、ボストンへ拡大
- 現在: 全米50州、5,500以上の都市

**COVID-19による急成長（2020年）**:
- 売上: 104%増の$15億
- ユーザー数: 550万→960万（ほぼ倍増）
- 注文量: 300%増
- ショッパー数: 13万→50万人に拡大
- 2020年4月、初の月間黒字化（$1,000万の利益）

## 6. 成功要因分析

### 6.1 主要成功要因

1. **自分自身の問題を解決**: 本当に気にかける問題に取り組んだ
2. **アセットライトモデル**: 既存小売店の在庫を活用し、初期投資を抑制
3. **タイミング**: スマートフォン普及とオンデマンド経済の成長期に参入
4. **Amazonでの経験**: サプライチェーンの専門知識を活用
5. **失敗からの学習**: 20回の失敗で得た知見を活用

### 6.2 タイミング要因

- 2012年はスマートフォンが普及し、消費者がアプリでの購入に慣れ始めた時期
- Uberなどオンデマンドサービスの成功が消費者行動を変革
- オンライン注文できないのは食料品だけという市場ギャップ

### 6.3 差別化要因

- 既存小売店との協業（競合ではなくパートナー）
- パーソナルショッパーによる高品質なサービス
- リテールメディア事業による高収益モデル

## 8. orchestrate-phase1への示唆

### 8.1 需要発見（/discover-demand）

- **自分自身の問題から出発する**: Mehtaは自分が本当に困っている問題を解決した
- **市場のギャップを見つける**: 「オンラインで買えないもの」という視点
- **タイミングを読む**: 技術と消費者行動の成熟度を見極める

### 8.2 CPF検証（/validate-cpf）

- **Concierge MVPで始める**: 自分が最初の配達員となり、顧客体験を深く理解
- **3週間でMVPを構築**: 過度に作り込まず、迅速に市場投入
- **失敗を恐れない**: 20回の失敗は「予想通り」と捉える

### 8.3 PSF検証（/validate-10x）

- **時間の10倍改善**: 店舗往復1-2時間→数分で注文完了
- **既存インフラの活用**: 自社で在庫を持たず、既存小売店と協業
- **パーソナライズ**: ショッパーとのコミュニケーションで差別化

## 9. 事業アイデア候補

この事例から着想を得られる日本向けビジネスアイデア:

1. **シニア向け買い物代行サービス**: 高齢化社会における買い物困難者向け、パーソナルショッパーモデル
2. **地方スーパーのオンライン化支援**: 小規模スーパーのDX支援＋配達ネットワーク提供
3. **専門店デリバリーマーケットプレイス**: 精肉店、鮮魚店など専門店からの配達サービス
