# Elon Musk - Tesla/SpaceX

## 1. 基本情報

| 項目 | 内容 |

| 生年 | 1971年 |

## 2. 創業ストーリー

### 2.1 課題発見（需要発見）

**着想源（SpaceX）**:
- 2000年にPayPal CEO解任後、宇宙探査に興味を持つ
- NASAのウェブサイトを確認し、火星への具体的な計画がないことに驚愕
- Mars Societyの年次大会で人類を惑星間種にすることへの支持を表明（2001年8月）

**着想源（Tesla投資）**:
- 電気自動車の可能性に長年興味を持っていた
- Tesla創業者Martin Eberhard、Marc TarpenningとMars Societyのイベントで出会う
- 共通の宇宙探査への情熱で意気投合

**需要検証方法**:
- 宇宙カンファレンスへの参加
- Mars Society、Ansari X Prizeへの資金提供
- ロシアでのロケット購入交渉（失敗→自社開発を決意）

### 2.2 CPF検証（Customer Problem Fit）

**課題の明確化**:
- 宇宙打ち上げコストが高すぎる → 人類の火星移住が不可能
- 既存自動車産業がEVに本気で取り組んでいない

**3U検証**:
- **Unworkable**: 既存のロケットは使い捨てでコスト削減に限界
- **Unavoidable**: 人類の長期存続には多惑星種になる必要
- **Urgent**: 気候変動・資源枯渇で時間的制約あり

**支払い意思（WTP）**:
- NASA Commercial Orbital Transportation Services契約獲得
- 各国政府・民間企業からの打ち上げ受注

### 2.3 PSF検証（Problem Solution Fit）

**10倍優位性**:

| 軸 | 従来の解決策 | SpaceXソリューション | 倍率 |
|---|------------|-----------------|------|
| コスト | $150M/打ち上げ | $15M（将来目標）| 10倍 |
| 再利用性 | 使い捨て | Falcon 9再利用 | 100倍 |
| 製造速度 | 年数機 | 高頻度製造 | 5倍+ |

**MVP**:
- Falcon 1: 最初の商用ロケット（3回失敗後、4回目で成功、2008年）
- 反復的・漸進的開発プロセス（ウォーターフォールではなくアジャイル）

**UVP（独自の価値提案）**:
- SpaceX: 「宇宙を民間が利用可能な価格に」
- Tesla: 「持続可能なエネルギーへの移行を加速」

**競合との差別化**:
- 垂直統合（ほぼすべてを自社製造）
- 反復開発・高速イテレーション
- First Principles思考

## 3. 初期の失敗と学び

### 3.1 Zip2での学び

- 1995年創業、1999年に$307Mで売却
- 最初の資金調達で株式を大幅に希薄化 → 取締役会がCEO交代を決定
- **学び**: 「会社のコントロールを失わないよう、できるだけ自己資金を投入」

### 3.2 X.com/PayPalでの学び

- 2000年に新婚旅行中にCEO解任（Peter Thielが後任）
- PayPalは「1999年最悪のビジネスアイデア」の1つに選ばれた時期も
- **学び**: 「コントロールの重要性」→ 以後の会社では筆頭株主を維持

### 3.3 SpaceX初期の失敗

- Falcon 1: 3回連続で打ち上げ失敗（燃料漏れ、ロケットステージ衝突等）
- 4回目で成功（2008年）
- 「会社には1回分の資金しか残っていなかった」
- **学び**: 粘り強さ、失敗から学ぶ文化

## 4. 成長戦略

### 4.1 初期トラクション獲得

**SpaceX**:
- NASA Commercial Orbital Transportation Services契約（安定した顧客確保）
- 政府契約で開発資金を確保しながら技術を磨く

**Tesla**:
- 高級スポーツカー（Roadster）から開始 → 富裕層をターゲット
- 段階的に大衆車へ展開（Model S → Model 3）

### 4.2 フライホイール

```
SpaceX:
打ち上げ成功 → 信頼性向上 → 受注増加 → 収益増加 → 開発投資 → 技術向上 → コスト削減 → 競争力強化

Tesla:
高級EV販売 → ブランド構築 → 量産技術開発 → コスト削減 → 大衆車展開 → 規模拡大 → さらなるコスト削減
```

### 4.3 スケール戦略

- 垂直統合による品質・コスト管理
- ギガファクトリーによる製造規模拡大
- ソフトウェアOTAアップデートによる継続的改善

## 6. 成功要因分析

### 6.1 主要成功要因

1. **First Principles思考**: 既存の常識を疑い、物理法則から再設計
2. **リスクテイク**: 全財産を投入する覚悟
3. **粘り強さ**: 連続失敗でも諦めない
4. **長期ビジョン**: 火星移住という100年規模のビジョン

### 6.2 タイミング要因

- リチウムイオン電池技術の成熟
- 気候変動への関心の高まり
- NASA民間委託政策の変化

### 6.3 差別化要因

- 競合が「不可能」と考えることに挑戦
- 垂直統合による競争優位
- ソフトウェア・ハードウェア両方の強み
