# プロジェクト憲章
**プロジェクト名**: Autonomous Founder Agent - 自律創業AIエージェント（Phase1）

## 1. ビジョン
**AIエージェント（Antigravity）が完全自律的にデジタルビジネスを創業・経営・サービス提供まで行う世界を実現する。**

## 2. 背景と目的

### 2.1 市場背景
2024年は「**AIエージェント元年**」と呼ばれ、自律型AIが実用化段階に入った。

| 指標 | 数値 |
|------|------|
| グローバルAIエージェント市場（2024年） | $51億～$58億 |
| 2030年予測 | $470億超（CAGR 44%） |
| 国内市場（2025年度予測） | 152億円（前年比232%増） |
| 企業導入率（2024年） | 79%が何らかの形で導入済み |
| 期待ROI | 平均171% |

**トレンド**: 単なる応答型の生成AIから、目標を与えると自ら計画・実行・改善する**自律型エージェントAI**へのパラダイムシフトが進行中。Microsoft、Google、NVIDIAなど大手テック企業が相次いで製品を発表し、実用化が加速している。

### 2.1.1 OpenAI: Level 5 Organizer と本プロジェクトの位置づけ

OpenAIは**AGI（汎用人工知能）達成への5段階ロードマップ**を公開している。本プロジェクト「Autonomous Founder Agent」は、このロードマップの**Level 3〜5の能力を創業領域で実証する先駆的プロジェクト**である。

| レベル | 名称 | 能力 | **本プロジェクトでの実装** |
|--------|------|------|------|
| Level 3 | Agents | 長時間自律動作、タスク実行・意思決定 | 市場調査→MVP構築→SNS運用を自律実行 |
| Level 4 | Innovators | 画期的なアイデア・ソリューションを開発 | 事業アイデアの自己発見・ピボット判断 |
| **Level 5** | **Organizer** | **組織全体を運営** | **創業から経営まで一貫した自律運営** |

> **本プロジェクトの技術的意義**:
> Level 5 Organizerが「組織を運営するAI」を目指すのに対し、本プロジェクトは**「事業をゼロから創業し運営するAI」**を実現する。これは単なる既存業務の自動化ではなく、**意思決定・創造・実行のすべてをAIが担う**という点で、Level 5の本質的な実証プロジェクトである。
>
> OpenAIが2025年にリリースした「**Operator**」は複雑なタスクの自律実行を可能にするが、本プロジェクトはその先を行き、**連続した事業創出ワークフロー全体をAIで完結**させることを目指す。

### 2.1.2 Macrohard構想：本プロジェクトの参照モデル

イーロン・マスクのxAIが発表した「**Macrohard**」構想は、**本プロジェクトと同じビジョンを大規模に追求する事例**であり、重要な参照モデルである。

| 項目 | Macrohard構想 | **本プロジェクト（Founder Agent）** |
|------|------|------|
| **コンセプト** | AIのみで「純粋なAIソフトウェア企業」を運営 | AIエージェントが自律的にデジタルビジネスを創業・経営 |
| **スコープ** | 大規模ソフトウェア企業の全機能 | スタートアップの創業〜PMF達成 |
| **対象** | エンタープライズ規模 | 個人・スモールビジネス規模 |
| **共通点** | ✅ 物理製品なし・デジタル完結 | ✅ 物理製品なし・デジタル完結 |

> **本プロジェクトの差別化**:
> Macrohardが「既存の大企業機能をAIで代替」するトップダウンアプローチであるのに対し、本プロジェクトは**「ゼロからの事業創出」というボトムアップアプローチ**を採用する。
>
> これにより、**起業家の意思決定プロセス自体をAIで再現・検証**できる。Macrohardが達成困難な規模のリソースを必要とするのに対し、本プロジェクトは**個人レベルで実験・検証可能なスケール**で同様のビジョンを追求する。

### 2.1.3 2025年：AIエージェントの本格実用化

上記の動向から、**2025年は「AIエージェントがビジネスの主役になる年」**と言える。

| キーポイント | 説明 |
|-------------|------|
| **完全自律型オペレーション** | Level 5 Organizerが示すように、AIが組織全体を運営する世界が現実味を帯びている |
| **ソフトウェア企業のAI化** | Macrohardが目指す「AIだけで構成されるソフトウェア企業」は、本プロジェクトの延長線上にある |
| **技術競争の加速** | OpenAI、xAI、Google、Microsoft、Anthropic が激しく競争し、AIエージェントの能力向上が急加速 |
| **創業の自動化** | 本プロジェクトは、これらの先端技術動向を活用し、**AIエージェントが自律的に事業を創業・運営する先駆的ユースケース**を構築する |

### 2.2 社会的意義
本プロジェクトは、以下の社会課題解決に寄与する可能性を持つ。

| 意義 | 詳細 |
|------|------|
| **労働力不足への対応** | 日本の人口減少・労働力不足に対し、AIエージェントによる事業創出・運営の自動化がソリューションとなりうる |
| **起業の民主化** | 技術スキル・資金・人材がなくても、アイデアを持つ誰もがAIを通じて事業を創出可能になる未来を切り拓く |
| **24時間365日の事業運営** | 人間のファウンダーの体力・時間制約を超えた継続的な事業創出・改善・運営が可能に |
| **データドリブン意思決定** | 人間の認知バイアスを排除し、データに基づく客観的・迅速な事業判断を実現 |
| **イノベーション加速** | 反復的作業をAIが代行することで、人間はより創造的・戦略的な活動に集中できる |
| **グローバル競争力強化** | 日本発のAIエージェント技術・ユースケースを確立し、国際競争力の向上に貢献 |

### 2.3 プロジェクト目的
本プロジェクトでは、AIエージェント自身がファウンダーとなり、以下を**人間の介入なしに**実行することを目指す：

| # | 目的 | 説明 |
|---|------|------|
| 1 | **事業アイデアの自己発見・生成** | 市場調査・トレンド分析に基づくビジネスアイデアの創出 |
| 2 | **仮説検証（CPF→PSF→PMF）** | 「起業の科学」に基づく段階的検証プロセス |
| 3 | **MVP（LP等）の自動構築・公開** | 最小限の機能を持つ製品の開発・デプロイ |
| 4 | **SNSを通じた初期ユーザー獲得** | ソーシャルメディアマーケティングによる顧客獲得 |
| 5 | **顧客フィードバックの自動収集・分析** | ユーザー行動・声の収集とセンチメント分析 |
| 6 | **ピボット判断・反復改善の自律実行** | 仮説検証結果に基づく方向転換・製品改善 |
| 7 | **収益化（マネタイゼーション）機能の実装** | 課金・決済機能による持続可能な事業モデル構築 |
| 8 | **PMF達成度の自動測定・レポート** | NPS・リテンションカーブ等による客観的評価 |
| 9 | **事業計画書の自動生成** | 上記を統合した包括的事業計画ドキュメント |

> **重要制約**: 
> - 物理世界・人間の関与が必須な領域は対象外。Antigravityが完結できるデジタルビジネスに限定する。
> - **法令遵守**: 日本国憲法、民法、商法、特定商取引法、個人情報保護法等の関連法令を遵守する。違法・公序良俗に反するビジネスは対象外。

## 4. スコープ

### 含む（In Scope）
- **自律ワークフロー設計**: `.agent/workflows/` に創業プロセスを実装
  - **Phase 1: アイデア発見**
    - `discover_idea.md`: 市場調査・アイデア発見・生成
  - **Phase 2: アイデア検証**
    - `create_lean_canvas.md`: リーンキャンバス作成
    - `validate_problem.md`: 課題仮説の検証
  - **Phase 3: CPF検証**
    - `define_persona.md`: ペルソナ定義
    - `simulate_interview.md`: 仮想ペルソナインタビュー
    - `diagnose_cpf.md`: CPF達成判定
  - **Phase 4: PSF検証**
    - `design_solution.md`: ソリューション設計
    - `select_mvp_type.md`: MVPタイプ選定
    - `diagnose_psf.md`: PSF達成判定
  - **Phase 5: MVP構築**
    - `build_lp.md`: LP構築
    - `build_webapp.md`: Webアプリ構築
    - `deploy_mvp.md`: MVPデプロイ
  - **Phase 6: ユーザー獲得**
    - `create_sns_content.md`: SNSコンテンツ作成
    - `post_sns.md`: SNS投稿実行
    - `analyze_engagement.md`: エンゲージメント分析
  - **Phase 7: 課金機能**
    - `setup_stripe.md`: Stripe連携設定
    - `implement_payment.md`: 決済機能実装
  - **Phase 8: フィードバック分析**
    - `collect_feedback.md`: ユーザーフィードバック収集
    - `analyze_feedback.md`: センチメント分析・課題抽出
    - `decide_pivot.md`: ピボット判断
  - **Phase 9: PMF達成**
    - `measure_pmf.md`: PMF達成度測定（NPS・リテンション等）
    - `report_pmf.md`: PMFレポート生成
  - **Phase 10: 完了**
    - `generate_business_plan.md`: 事業計画書Markdown生成
- **知識ベース**: `startup_science_knowledge.md` の活用・拡張
- **Antigravity標準機能利用**: Browser、Search、画像生成、コード生成

### 含まない（Out of Scope）
- 物理的な製品・サービス
- 人間の関与が必要なオペレーション
- 実際の法人登記・銀行口座開設
- 本番環境の高可用性・SLA保証

## 8. リスクと対策

| リスク | 対策 |
|--------|------|
| 2週間で全ワークフロー実装が困難 | Phase1（MVP公開まで）を優先。Phase2以降は別途計画 |
| デジタル完結ビジネスの制約 | SaaS / 情報提供 / コンテンツ系に特化 |
| MVP公開の技術的障壁 | GitHub Pages / Vercel 等の無料ホスティング活用 |
| **PMF未達成の長期化** | PMF達成度を定量的に測定し、早期にピボット判断を実施 |
| **フィードバック収集の困難** | 複数チャネル（SNS、フォーム、分析ツール）を併用し収集手段を多様化 |
| **責任の所在の不明確化** | AIの行動に対する最終責任はスポンサー（Yuichi）に帰属。各ワークフローにログ・監査証跡を残す |
| **知的財産権侵害** | 生成コンテンツの著作権チェック、既存著作物の無断利用禁止をワークフローに組み込む |
| **個人情報保護** | ユーザーデータ収集時はプライバシーポリシー明示、GDPR/個人情報保護法準拠を設計に組み込む |
| **バイアス・差別** | サービス設計時に公平性チェックを実施、特定属性への差別的な扱いを禁止 |
| **セキュリティリスク** | APIキー・認証情報の安全管理、プロンプトインジェクション対策 |
| **AIの幻覚・誤情報生成** | 重要な事実・数値は外部ソースで検証、生成コンテンツのファクトチェックをワークフローに組み込む |

## 9. ガバナンス・倫理原則

### 9.1 責任体制
- **最終責任者**: Yuichi（スポンサー）
- **実行主体**: Antigravity（Founder Agent）
- AIエージェントは法的主体ではないため、全ての行動・結果の責任はスポンサーに帰属する

### 9.2 倫理原則
| 原則 | 説明 |
|------|------|
| **透明性** | AIの判断プロセスを可視化し、説明可能性を確保する |
| **公平性** | 特定の属性（性別、人種、年齢等）に対する差別的な扱いを禁止する |
| **プライバシー尊重** | 個人情報の最小限収集、目的外利用の禁止、適切な同意取得 |
| **安全性** | ユーザーや社会に害を与えるサービスの提供を禁止する |
| **誠実性** | 虚偽の情報提供、詐欺的行為、ユーザーを欺く行為を禁止する |
| **正確性** | AIによる幻覚・誤情報生成リスクを認識し、重要情報のファクトチェックを実施する |

### 9.3 Human-in-the-Loop ポイント
完全自律を目指すが、以下のポイントでは人間（スポンサー）による確認・承認を推奨：
- **重大な方針決定**: ビジネスドメインの最終選定
- **ピボット判断前**: 大きな方向転換を行う前の確認
- **課金開始前**: 決済機能の本番稼働前
- **法的リスク判断**: 契約・利用規約の最終確認
- **緊急停止**: 問題発生時のサービス停止判断

### 9.4 継続的モニタリング
- 各ワークフロー実行時のログ記録
- 定期的な成果物・サービス品質の監査
- ユーザーフィードバックの収集と対応
