# ケーススタディ 091: 自治体E - 議事録自動生成AI

## 概要
- **組織**: 自治体E（人口50万都市）
- **課題**: 会議議事録作成に月100時間の人的作業
- **AI解決策**: 会議音声の自動文字起こし＋AIによる要約・項目分類
- **定量効果**: 事務作業60%削減、議事録作成時間から月80時間削減

## 導入背景
市議会・行政会議は月20回以上開催。各会議60-120分の議事録作成に事務職員が2-3時間要費。音声がアナログで手作業リスクが高かった。

**課題点**：
- 議事録作成工数が月100時間
- 記録漏れ・誤字リスク
- 原稿作成から配信まで1週間要すること

## AI解決策
**システム構成**：
1. **音声認識**: Google Cloud Speech-to-Text（日本語精度98%）で会議音声をリアルタイム文字起こし
2. **AIサマリー**: Claude 3.5 Sonnetで以下を自動抽出
   - 議題ごとの要点
   - 決定事項・合意事項
   - 担当者・期限の自動抽出
3. **フォーマット統一**: 既存の議事録テンプレートに自動整形

**操作性**: 会議開始時にクラウド連携ボタンを押すだけで文字起こし開始→AIが自動で整形→10分以内に下書き完成

## 成果・効果
| 指標 | 効果 |
|------|------|
| **議事録作成時間** | 120分 → 40分（66%削減） |
| **月間削減工数** | 月100時間 → 月40時間（60%削減） |
| **誤字・脱落** | 手作業時比較で99.2%削減 |
| **配信リードタイム** | 1週間 → 翌営業日 |
| **職員満足度** | 82%（事務作業軽減に好評） |

**経済効果**：年間1,200時間削減 = 年間300万円のコスト削減相当

## 技術スタック
- **音声認識**: Google Cloud Speech-to-Text API
- **LLM**: Claude 3.5 Sonnet（要約・項目分類）
- **インフラ**: Google Cloud Functions + Cloud Storage
- **フロントエンド**: Chrome拡張 + Slack連携

## インサイト
**成功要因**：
1. **ユーザー参加型設計** - 事務職員参画でUXを最適化
2. **既存ワークフロー尊重** - 従来の議事録形式を踏襲
3. **品質チェック機構** - AIサマリー後の人間確認プロセス構築

**展開効果**：
- 同自治体の他部門（福祉部・環境部等12部）への横展開を検討中
- 類似自治体への共有化で関連市町村10団体が導入予定
