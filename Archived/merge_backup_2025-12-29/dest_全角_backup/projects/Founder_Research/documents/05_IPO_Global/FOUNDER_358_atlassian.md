# Mike Cannon-Brookes & Scott Farquhar - Atlassian（Bootstrap to IPO）

## 1. 基本情報

| 項目 | 内容 |
|------|------|
| 創業者 | Mike Cannon-Brookes & Scott Farquhar |
| 生年 | 両名とも1979年生まれ |
| 国籍 | オーストラリア |
| 学歴 | University of New South Wales（コンピューターサイエンス学士、2002年） |
| 創業前経験 | 大学同級生、卒業後即起業 |
| 企業名 | Atlassian |
| 創業年 | 2002年 |
| 業界 | エンタープライズソフトウェア / 開発者ツール / SaaS |
| 現在の状況 | NASDAQ上場 |
| IPO日 | 2015年12月10日 |
| IPO時評価額 | $4.37B |
| 現在時価総額 | $50B+（2024年） |

## 2. 創業ストーリー

### 2.1 大学卒業直後の起業（2002年）

**起業の動機**:
- Mike Cannon-BrookesとScott Farquharは大学同級生
- 2002年、University of New South Wales卒業
- 「普通の就職では年収$40,000程度、自分たちなら$48,000稼げるはず」
- 目標: 年収$48,000（週$1,000）を自分たちで稼ぐ

**初期資金**:
- $10,000のクレジットカード借入のみ
- VC資金調達なし
- シドニーのアパートで創業

**課題発見**:
- 企業のソフトウェア開発チームが使うツールが高価で複雑
- JIRA、Bugzilla等のバグトラッキングツールは$10,000-$50,000
- 中小企業や個人開発者には手が届かない

### 2.2 JIRA誕生（2002年）

**製品コンセプト**:
- 開発者が実際に使いたいバグトラッキングツール
- 低価格（$1,200〜）
- セルフサービス（営業なし）
- 簡単にセットアップ可能

**初期販売**:
- オンラインで販売開始
- 無料トライアル提供
- 開発者コミュニティからの口コミ
- 初年度売上: $150,000

### 2.3 ブートストラップ期（2002-2010年）

**営業チームなしの成長**:
- 営業担当者ゼロ
- セルフサービスモデル
- プロダクト主導成長（PLG）
- 開発者からの口コミ

**製品ラインナップ拡大**:
- **2004年**: Confluence（Wiki/コラボレーションツール）リリース
- **2006年**: Bamboo（継続的インテグレーション）
- **2010年**: Bitbucket買収（Git/Mercurialホスティング）

**財務状況**:
- 黒字経営を継続
- キャッシュフローポジティブ
- 再投資による成長
- 2010年時点で年間売上$59M

**顧客数**:
- 2010年: 18,000社以上
- Fortune 500の多数が利用
- NASA、LinkedIn、Twitter等

### 2.4 初めての資金調達（2010年）

**Accel Partnersから$60M調達**:
- 創業8年目、初めての外部資金調達
- ポストマネー評価額: $400M
- 目的: 成長加速、国際展開、買収

**調達理由**:
- 成長資金が必要だったわけではない
- 「いつか資金調達が必要になるかもしれない」という保険
- Accelの経験・ネットワークへのアクセス
- Mike: 「正直、何に使うか明確な計画はなかった」

**調達後の使途**:
- 国際展開（米国、欧州）
- M&A（Bitbucket、HipChat等）
- プロダクト開発投資

## 3. IPOまでの成長（2010-2015年）

### 3.1 製品ポートフォリオの拡大

**主要製品**:
1. **JIRA**: バグトラッキング、プロジェクト管理
2. **Confluence**: Wiki、ナレッジマネジメント
3. **Bitbucket**: Git/Mercurialホスティング
4. **HipChat**: チームチャット（2012年買収）
5. **Bamboo**: 継続的インテグレーション
6. **Crucible**: コードレビュー
7. **FishEye**: リポジトリブラウザ

**プラットフォーム戦略**:
- Atlassian Marketplace開設
- サードパーティアプリ1,000+
- エコシステム構築

### 3.2 IPO前の財務指標（2015年）

| 項目 | 数値 |
|------|------|
| 年間売上 | $320M |
| YoY成長率 | 40%+ |
| 顧客数 | 50,000+ |
| 従業員数 | 1,600人 |
| 粗利率 | 85% |
| 純利益 | 黒字 |

**ユニークな特徴**:
- 営業チームゼロ（カスタマーサクセスチームのみ）
- セルフサービスモデル
- CAC（顧客獲得コスト）が極めて低い
- LTV（顧客生涯価値）が高い

## 4. IPO詳細（2015年12月）

### 4.1 IPO条件

| 項目 | 詳細 |
|------|------|
| IPO日 | 2015年12月10日 |
| 取引所 | NASDAQ |
| ティッカーシンボル | TEAM |
| 公募価格 | $21/株 |
| 公募株数 | 10,500,000株 |
| 調達額 | $462M |
| IPO時評価額 | $4.37B |
| 主幹事 | Morgan Stanley、Goldman Sachs、Allen & Company |

**S-1ハイライト**:
- 営業チームゼロを強調
- プロダクト主導成長（PLG）
- 高い顧客満足度（NPS 60+）
- Fortune 500の85%が顧客

### 4.2 初日取引

**初値・終値**:
- 初値: $27.50（公募価格比+31.0%）
- 初日終値: $27.78（公募価格比+32.3%）
- 初日時価総額: $5.8B

**投資家の反応**:
- セルフサービスモデルへの注目
- 営業コストゼロの効率性
- 高い粗利率（85%）
- 安定した成長率

### 4.3 Direct Listingスタイル

**特徴**:
- 伝統的なIPOプロセスを踏襲
- しかし、ロックアップ期間なし（創業者も初日から売却可能）
- 価格発見メカニズムの透明性
- 2019年のSpotify、Slack等のDirect Listing先駆け的要素

## 5. IPO成功要因分析

### 5.1 ビジネスモデルの独自性

**営業チームゼロのセルフサービス**:
- CAC（顧客獲得コスト）が極めて低い
- スケーラブルな成長モデル
- プロダクトの品質が全て
- 顧客からの口コミが成長ドライバー

**プロダクト主導成長（PLG）**:
- 無料トライアル
- 開発者コミュニティからの拡散
- Bottom-Up採用（開発者 → チーム → 組織全体）

**ランド＆エクスパンド**:
- 小規模チームから開始
- 使用量・ユーザー数に応じて課金
- エンタープライズへの拡大

### 5.2 財務健全性

**高い粗利率**:
- 粗利率85%（SaaS業界トップクラス）
- 営業コストが低い
- キャッシュフロー効率が高い

**黒字経営**:
- IPO時点で黒字
- キャッシュフローポジティブ
- 成長と利益のバランス

### 5.3 開発者ファーストの文化

**開発者への信頼**:
- 開発者が意思決定者
- 使いやすさ、カスタマイズ性を重視
- オープンなエコシステム

**コミュニティ**:
- Atlassian Community（ユーザーフォーラム）
- Atlassian Summit（年次カンファレンス）
- 開発者アドボカシー

### 5.4 ブートストラップの利点

**資本効率**:
- 無駄な支出なし
- 顧客収益のみで成長
- 高い資本効率

**創業者支配**:
- Mike & Scottが過半数の議決権
- 長期的意思決定が可能
- 四半期決算に左右されない

## 6. IPO後の成長戦略

### 6.1 クラウドシフト（2016年〜）

**オンプレミスからクラウドへ**:
- JIRA Cloud、Confluence Cloud
- サブスクリプションモデル
- リカーリング収益の増加

**クラウド移行の課題**:
- 既存顧客の移行
- オンプレミス版のサポート終了（2024年2月）
- 一時的な収益減少

### 6.2 M&A戦略

**主要買収**:
- **2017年**: Trello買収（$425M）
- **2019年**: AgileCraft買収（JIRA Alignへ）
- **2021年**: Chartio、Halp買収
- **2022年**: Percept.ai買収（AI強化）

### 6.3 新製品開発

**Jira Work Management**:
- 非技術者向けのプロジェクト管理
- ターゲット市場の拡大

**Atlassian Intelligence**:
- AIアシスタント機能
- 生成AI統合

### 6.4 株価推移
