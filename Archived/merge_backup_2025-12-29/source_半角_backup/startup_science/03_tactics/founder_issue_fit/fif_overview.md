# Founder-Issue-Fit (FIF)

> **出典**: 田所雅之「起業の科学」STEP 1
> **関連概念**: [[CONCEPT_IDEA_001]], [[CONCEPT_CPF_001]], [[FRAMEWORK_MVV_001]]

---

## 1. 定義

**Founder-Issue-Fit (FIF)** とは、「創業者がその課題に取り組むべき理由・適性がある状態」を指す。

スタートアップは長く困難な道のり。創業者と課題の間に強い結びつき（Fit）がなければ、途中で諦めてしまう。

**FIFの本質**:
- 課題への**情熱** (Passion)
- 課題領域の**専門性** (Expertise)
- 解決への**執念** (Obsession)

Y Combinatorの格言:
> "Make something people want. But first, make sure YOU want to make it for 10+ years."

---

## 2. なぜ重要か

スタートアップの創業者が直面する現実：
- 平均7年かかる（PMF達成まで平均3-4年、Exit/IPOまで10年）
- 90%以上が失敗する
- 資金繰り・採用・競合対応で毎日ストレス
- 孤独な意思決定の連続

**FIF不足だと起こること**:
- ❌ 最初の困難で諦めてしまう
- ❌ 情熱が続かず、プロダクト開発が中途半端
- ❌ 専門性不足で顧客に刺さらない
- ❌ 「なんとなく儲かりそう」で始めて失敗

**FIF達成の効果**:
- ✅ 困難に直面しても諦めない
- ✅ 10年以上取り組める持続力
- ✅ 専門性を活かして独自の洞察を得られる
- ✅ 創業者自身が最初のユーザー（Eat your own dog food）

---

## 3. 6つの検証項目

### 3.1 課題への情熱 (Passion)

**質問**:
- この課題に10年以上取り組めるか?
- お金がなくても、成功が保証されなくても続けたいか?
- 朝起きたとき、この課題のことを考えてワクワクするか?

**判定基準**:
- ✅ YES: 寝ても覚めても考えてしまう、人に話したくなる
- ❌ NO: 「儲かりそう」「流行ってる」が主な動機

**事例**:
- **Brian Chesky (Airbnb)**: デザイナーとして「旅の体験」に情熱。自らホストとして泊まり、デザインを磨き続けた
- **Drew Houston (Dropbox)**: 大学時代からファイル同期に困り続け、自分のために作った

---

### 3.2 専門性 (Expertise)

**質問**:
- この課題領域で5年以上の経験があるか?
- 業界の専門家として認められているか?
- 顧客よりも深く課題を理解しているか?

**判定基準**:
- ✅ YES: 業界経験、学術研究、趣味での深い知識
- ❌ NO: 外野から「良さそう」と思っているだけ

**専門性のレベル**:
| レベル | 説明 | 例 |
|--------|------|-----|
| Lv1: 初心者 | 本やネットで学んだだけ | NG |
| Lv2: 経験者 | 1-3年の実務経験 | 微妙 |
| Lv3: 専門家 | 5年以上の実務経験、業界で認知 | ✅ OK |
| Lv4: 権威 | 10年以上、著書・講演・業界リーダー | ✅ 理想 |

**事例**:
- **Eric Yuan (Zoom)**: Cisco WebEx開発で10年以上、ビデオ会議の専門家
- **Marc Benioff (Salesforce)**: Oracle幹部としてエンタープライズソフトウェア13年

---

### 3.3 課題の深さ（当事者性）

**質問**:
- 自分自身がその課題に困っているか（"I have this problem"）?
- 単なる「誰かの課題」ではなく「自分の課題」か?
- 既存ソリューションに不満を感じているか?

**判定基準**:
- ✅ YES: 自分が最初のユーザー、日常的に困っている
- ❌ NO: 「こういう人がいるらしい」という伝聞

**Paulg (Y Combinator) の名言**:
> "The best startup ideas come from the founders' own experiences."

**事例**:
- **Tony Hsieh (Zappos)**: 自分が靴のオンライン購入で不満 → 返品無料で解決
- **Travis Kalanick (Uber)**: パリで雪の日にタクシーが捕まらず激怒 → Uber創業

---

### 3.4 解決への執念 (Obsession)

**質問**:
- 何があっても解決したいと思うか?
- 失敗しても別の方法で挑戦し続けるか?
- この課題が解決されないことに怒りを感じるか?

**判定基準**:
- ✅ YES: 「誰もやらないなら自分がやる」という使命感
- ❌ NO: 「できればいいな」程度の願望

**執念の兆候**:
- 夜中でも思いついたアイデアをメモしてしまう
- 関連ニュースを毎日チェックしている
- 競合サービスを使い込んで不満点をリスト化している

**事例**:
- **Elon Musk (Tesla)**: 化石燃料依存に憤り、電気自動車普及に執念。何度も倒産危機を乗り越えた
- **Jeff Bezos (Amazon)**: 「地球上で最も顧客中心の企業」に執念、24年黒字化遅延も耐えた

---

### 3.5 タイミング (Timing)

**質問**:
- **今**この課題に取り組むべきか?
- 技術・市場・規制の環境は整っているか?
- 5年前・5年後ではダメな理由は?

**判定基準**:
- ✅ YES: 技術的実現可能性、市場の成熟度、規制緩和が揃っている
- ❌ NO: 早すぎる or 遅すぎる

**タイミングチェックリスト**:
- [ ] 必要な技術が実用レベルに達しているか?
- [ ] 市場が十分に成熟しているか（早すぎない）?
- [ ] まだ競合が少ない状態か（遅すぎない）?
- [ ] 規制・社会的受容が整っているか?

**事例**:
- **Uber (2009年創業)**:
  - ✅ スマホGPS普及（2007年iPhone）
  - ✅ クレカオンライン決済普及
  - ✅ シェアリングエコノミー受容
  - → **タイミング完璧**

- **Webvan (1999年創業、2001年倒産)**:
  - ❌ ネット普及率低い（1999年: 約40%）
  - ❌ オンライン決済未成熟
  - ❌ 物流インフラ未整備
  - → **早すぎた** (同じモデルで2012年Instacart成功)

---

### 3.6 スキルセット (Skills)

**質問**:
- 課題解決に必要なスキルを持っているか?
- 不足スキルを補える共同創業者がいるか?
- 学習可能な範囲か?

**必要スキル例（プロダクト系スタートアップ）**:
| スキル | 必要度 | 代替案 |
|--------|--------|--------|
| プロダクト開発 | 必須 | エンジニア共同創業者 |
| マーケティング | 必須 | マーケター共同創業者 |
| セールス | 高 | B2Bなら創業者が兼務 |
| ファイナンス | 中 | 外部CFO、アドバイザー |
| デザイン | 中 | フリーランス、ツール |

**判定基準**:
- ✅ YES: チーム全体で主要スキルをカバーできる
- ❌ NO: クリティカルなスキルが欠如、採用も困難

**事例**:
- **Google (Larry Page & Sergey Brin)**:
  - スキル: 検索アルゴリズム、分散システム（Stanford Ph.D.）
  - 不足: ビジネス → Eric Schmidt CEO招聘で補完

- **Airbnb (Brian Chesky & Joe Gebbia)**:
  - スキル: デザイン、UX
  - 不足: エンジニアリング → Nathan Blecharczyk共同創業者で補完

## 4. 判定基準

### 4.1 スコアリング

| 項目 | 質問 | YES/NO |
|------|------|--------|
| 1. 情熱 | 10年以上取り組めるか? | |
| 2. 専門性 | 5年以上の経験・深い知識があるか? | |
| 3. 当事者性 | 自分自身が困っているか? | |
| 4. 執念 | 何があっても解決したいか? | |
| 5. タイミング | 今取り組むべき理由があるか? | |
| 6. スキル | 必要スキルを持つ/補えるか? | |

**合格基準**:
- **5-6個 YES**: ✅ FIF達成、起業推奨
- **3-4個 YES**: ⚠️ 一部不足、共同創業者で補完検討
- **0-2個 YES**: ❌ FIF未達成、別の課題を探す

---

### 4.2 レッドフラグ（危険信号）

以下に当てはまる場合は要注意:
- ❌ 「儲かりそう」「流行ってる」が主な動機
- ❌ 自分は課題を感じていない（「誰かの課題」止まり）
- ❌ 業界経験ゼロ、専門性ゼロ
- ❌ 1年以内に飽きそう
- ❌ 競合が既に巨大、後発で勝てる根拠なし

---

## 5. 実践ステップ

### ステップ1: 自己分析（1時間）

1. 過去10年の経験を棚卸し
   - どんな仕事・プロジェクトをしてきたか?
   - 何に情熱を感じたか?
   - どんな課題に直面したか?

2. 自分が困っている課題をリスト化
   - 日常的に不満を感じること
   - 既存ツール・サービスの不満点
   - 「こうなればいいのに」と思うこと

3. 専門性の棚卸し
   - 5年以上経験がある領域
   - 人に教えられるレベルのスキル
   - 業界で認められている実績

### ステップ2: 6項目チェック（30分）

上記の6項目に対して正直に YES/NO を答える。

### ステップ3: 不足項目の補完計画（1時間）

**パターン1: スキル不足**
- → 共同創業者を探す
- → アドバイザーを招聘
- → 3-6ヶ月学習・経験を積む

**パターン2: 専門性不足**
- → 6ヶ月業界に飛び込む（転職・インターン）
- → 顧客インタビュー50人で知識補完

**パターン3: タイミング不適**
- → 5年後に再挑戦する計画を立てる
- → 今できる小さな一歩から始める

### ステップ4: 共同創業者との対話（必須）

- FIFチェックをチーム全体で実施
- 「なぜこの課題に取り組むのか」をすり合わせ
- 足りないスキル・専門性を補完し合えるか確認

---

## 6. よくある間違い

| 間違い | 症状 | 対策 |
|--------|------|------|
| **流行に飛びつく** | 「AI/Web3が流行ってるから」で選ぶ | 自分の情熱・専門性と合致するか確認 |
| **他人の課題を想像** | 「高齢者が困ってるはず」（未検証） | 自分が当事者、または徹底的にインタビュー |
| **スキル過信** | 「プログラミングできるから何でもいける」 | 課題領域の専門性も必要 |
| **甘いタイミング判断** | 「いつかは実現する」で早すぎる市場に参入 | 技術・市場の現状を冷静に分析 |
| **FIF無視** | 「とりあえずやってみる」で始める | 最低3-4項目はYESでないと危険 |

---

## 7. 事例

### 7.1 FIF達成例: Pieter Levels (PhotoAI)

| 項目 | 内容 |
|------|------|
| 情熱 | ノマドライフ・AI技術への10年以上の情熱 |
| 専門性 | 12年以上のプログラミング、40+プロダクト構築 |
| 当事者性 | 自分がプロフィール写真に困った経験 |
| 執念 | Build in Publicで毎日改善 |
| タイミング | 2022年Stable Diffusion登場、生成AI革命 |
| スキル | フルスタック開発、マーケティング、デザイン全て可能 |

**結果**: 6/6 FIF達成 → 月商$170K達成

### 7.2 FIF不足例: Theranos (Elizabeth Holmes)

| 項目 | 内容 |
|------|------|
| 情熱 | ❓ 本当の情熱か、名声欲か不明 |
| 専門性 | ❌ 医療・血液検査の専門性ゼロ（Stanford中退） |
| 当事者性 | ❌ 自分が血液検査に困った経験なし |
| 執念 | ✅ 詐欺的手法を使ってでも続けた（逆方向） |
| タイミング | ❓ 技術的実現不可能なのに強行 |
| スキル | ❌ 必要な科学・医療スキルなし |

**結果**: 1/6（タイミングも微妙）→ 詐欺で崩壊
