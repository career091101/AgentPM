# Flywheel Design (フライホイール設計)

> **出典**: 田所雅之「起業大全」第2章、Jim Collins "Good to Great"
> **関連**: [[FRAMEWORK_MVV_001]], [[CONCEPT_PMF_001]]

---

## 1. 定義

**Flywheel（フライホイール）** とは、一度回り始めると勢いが増し、自己強化的に成長し続ける好循環の仕組み。

物理学の「はずみ車」に由来。最初は重くて回らないが、一度回り始めると慣性で加速していく。

**ビジネスにおけるFlywheel**:
- 顧客満足 → 口コミ → 新規顧客 → さらなる改善 → さらなる満足 → ...
- この循環が勝手に回り続ける状態を設計する

**Jim Collinsの定義**:
> "Build momentum through consistency, not dramatic transformation."
> （劇的な変革ではなく、一貫性によって勢いを築く）

---

## 2. なぜ重要か

### 2.1 Flywheelの威力

**線形成長 vs 指数関数的成長**:
- ❌ 線形成長: 広告費を2倍にすると売上が2倍
- ✅ Flywheel: 最初は遅いが、加速すると止まらない

**複利効果**:
- ネットワーク効果: ユーザーが増えるほど価値が増す
- データ効果: データが増えるほどプロダクトが改善
- ブランド効果: 口コミが口コミを呼ぶ

### 2.2 スタートアップに不可欠な理由

**資金効率**:
- 広告依存（プッシュ型）: CAC高騰、持続不可能
- Flywheel（プル型）: 口コミで自然成長、CAC低下

**競争優位性（Moat）**:
- 簡単には真似できない
- 時間とともに差が開く

**投資家へのストーリー**:
- 単発の施策 < 自己強化する成長エンジン

## 3. Flywheelの構成要素

### 3.1 基本構造（4要素）

```
     ┌──────────────────┐
     │  ① Input         │ 何を投入するか
     │  （投入）         │
     └────────┬─────────┘
              │
              ▼
     ┌──────────────────┐
     │  ② Process       │ どう変換するか
     │  （プロセス）     │
     └────────┬─────────┘
              │
              ▼
     ┌──────────────────┐
     │  ③ Output        │ 何を生み出すか
     │  （産出）         │
     └────────┬─────────┘
              │
              ▼
     ┌──────────────────┐
     │  ④ Feedback      │ 次のInputを増やす
     │  （フィードバック）│
     └────────┬─────────┘
              │
              └──────────┐
                        │
              ┌─────────▼──────────┐
              │  再び①へ（加速）   │
              └────────────────────┘
```

### 3.2 良いFlywheelの条件

- [ ] **自己強化する**: Output が次の Input を自動的に増やす
- [ ] **加速する**: 回るたびに速度が上がる
- [ ] **シンプル**: 3-5ステップで表現できる
- [ ] **測定可能**: 各ステップの指標が明確
- [ ] **持続可能**: 外部投入（広告費等）に依存しない

## 4. 代表的なFlywheel事例

### 4.1 Amazon Flywheel

**Jeff Bezosがナプキンに描いた有名な図**:

```
低価格 → 顧客体験向上 → トラフィック増加 → 出品者増加
   ↑                                             ↓
   └───────── 商品選択肢増加 ←──────────────────────┘
               ↓
          さらなる低価格化（規模の経済）
               ↓
         【Flywheel加速】
```

**詳細**:
1. **低価格**: 顧客に低価格で提供
2. **顧客体験向上**: 配送速度・品揃え改善
3. **トラフィック増加**: 満足した顧客が再訪・口コミ
4. **出品者増加**: トラフィック多い→出品者が集まる
5. **商品選択肢増加**: 出品者増→品揃え増
6. **さらなる低価格化**: 規模の経済でコスト削減
7. **①に戻る（加速）**

**結果**: 1994年創業、2023年売上$575B（約80兆円）

---

### 4.2 Airbnb Flywheel

**両面市場のFlywheel**:

```
ゲスト満足 → レビュー投稿 → ホスト信頼性向上 → ホスト増加
   ↑                                             ↓
   └──────── リスティング増加 ←───────────────────┘
               ↓
          ゲスト選択肢増加
               ↓
         【Flywheel加速】
```

**詳細**:
1. **初期ゲスト獲得**: イベント時の宿不足で利用
2. **良い体験**: 安い・ローカル体験
3. **レビュー投稿**: 5つ星レビュー
4. **ホスト増加**: レビューで信頼→新ホスト参入
5. **リスティング増加**: 部屋の選択肢が増える
6. **さらなるゲスト**: 選択肢多い→新ゲスト来る
7. **①に戻る（加速）**

**結果**: 2008年創業、2023年時価総額$80B

---

### 4.3 LinkedIn Flywheel

**プロフェッショナル・ネットワークのFlywheel**:

```
プロフィール作成 → 検索表示 → つながり増加 → データ蓄積
   ↑                                         ↓
   └────── レコメンド精度向上 ←────────────────┘
               ↓
          新規ユーザー招待
               ↓
         【Flywheel加速】
```

**詳細**:
1. **初期ユーザー**: プロフィール作成
2. **検索表示**: Google検索で上位表示
3. **つながり増加**: 同僚・知人がつながる
4. **データ蓄積**: 職歴・スキルデータ蓄積
5. **レコメンド精度向上**: AIが最適なつながりを提案
6. **ネットワーク効果**: つながりが多い人ほど価値が高い
7. **新規ユーザー招待**: 既存ユーザーが招待
8. **①に戻る（加速）**

**結果**: 2003年創業、2016年Microsoftが$26Bで買収

## 5. Flywheel設計プロセス

### ステップ1: 顧客の成功を定義（30分）

**質問**:
- 顧客は何を達成したら「成功」と感じるか?
- その成功体験を最大化する要素は?

**例（Uber）**:
- 顧客成功 = 5分以内に車が来る、安全に目的地到着

### ステップ2: 成功体験の次に起こることを考える（30分）

**質問**:
- 顧客が成功したら、次に何をするか?
- その行動はビジネスにどうプラスになるか?

**例（Uber）**:
- 満足した顧客 → 友人に勧める → 新規ユーザー増加

### ステップ3: ループを描く（1時間）

**ワークショップ形式**:
1. ホワイトボードに円を描く
2. 時計回りに各ステップを配置
3. 各ステップが次をどう加速するか矢印で示す

**チェック**:
- [ ] ループが閉じているか?（最後が最初に戻る）
- [ ] 外部投入（広告等）なしで回るか?
- [ ] 回るたびに加速するか?

### ステップ4: 各ステップの指標を定義（30分）

**測定可能にする**:
- ステップ1: NPS（顧客満足度）
- ステップ2: 口コミ率（K-factor）
- ステップ3: 新規ユーザー獲得数（Organic）
- ステップ4: リピート率

### ステップ5: ボトルネックを特定（継続的）

**質問**:
- どのステップが最も遅い?
- どこを改善すれば加速するか?

**優先順位**:
1. 最も効果が大きいステップ
2. 最も改善しやすいステップ
3. 最もコストが低いステップ

## 6. Flywheel vs Funnel（従来型）

| 観点 | Funnel（従来型） | Flywheel（新型） |
|------|-----------------|-----------------|
| **構造** | 一方向（直線） | 循環（円） |
| **成長** | 線形 | 指数関数的 |
| **外部投入** | 常に必要（広告費） | 初期のみ |
| **顧客** | 使い捨て | 資産（口コミ源） |
| **加速** | しない | 時間とともに加速 |
| **持続性** | 広告止めたら終了 | 自走し続ける |

**Funnelの限界**:
```
広告 → 認知 → 興味 → 購入 → （終了）
```
- 毎回広告費が必要
- CAC（顧客獲得コスト）が上がり続ける
- 競合との広告費競争

**Flywheelの優位性**:
```
顧客満足 → 口コミ → 新規顧客 → データ蓄積 → 改善 → さらなる満足 → ...
```
- 口コミで自然成長
- CACが下がり続ける
- 競合が真似しにくい

---

## 7. よくある間違い

| 間違い | 症状 | 対策 |
|--------|------|------|
| **ループが閉じていない** | Output が Input に戻らない | 最後のステップが最初に戻るか確認 |
| **広告依存** | 広告なしで回らない | Organic成長の仕組みを組み込む |
| **複雑すぎる** | 10ステップ以上 | 3-5ステップに単純化 |
| **測定不可能** | 各ステップの指標がない | KPIを明確に定義 |
| **短期思考** | すぐ結果を求める | 最初は遅い、長期視点で |

## 8. Flywheel加速のための戦術

### 8.1 ネットワーク効果を組み込む

**定義**: ユーザーが増えるほど価値が増す仕組み

**種類**:
- **Direct Network Effect**: Facebook（友達が多いほど価値高い）
- **Two-sided Network Effect**: Uber（ドライバー↔乗客）
- **Data Network Effect**: Google（検索データ蓄積→精度向上）

### 8.2 バイラル係数（K-factor）を上げる

**定義**: 1人のユーザーが何人を招待するか

**計算式**:
```
K-factor = (招待数) × (招待受諾率)
```

**目標**:
- K > 1.0: 指数関数的成長（バイラル）
- K = 0.5-1.0: 健全な成長
- K < 0.5: 口コミ不足

**向上策**:
- Dropbox: 紹介で追加容量プレゼント
- Uber: 紹介で$10クーポン

### 8.3 リテンションを最優先

**なぜ**:
- Flywheel = 顧客が戻ってくる × 友人を連れてくる
- リテンション低い = 穴の空いたバケツに水を注ぐ

**目標値**:
- Day 1: 40%以上
- Day 7: 20%以上
- Day 30: 10%以上

## 9. 実践：自社Flywheel設計ワークショップ

### 準備（15分）

**参加者**: 創業者 + 初期メンバー3-5人
**用意するもの**: ホワイトボード、付箋、ペン

### ステップ1: 顧客成功の定義（15分）

**各自で付箋に書く**:
- 顧客が「最高！」と思う瞬間は?
- 顧客が達成したいゴールは?

**共有・グルーピング**:
- 共通項を抽出

### ステップ2: 次のアクション想像（15分）

**質問**:
- その後、顧客は何をする?
- 友人に話す? SNSに投稿? リピート?

### ステップ3: ループを描く（30分）

**ホワイトボードに円を描く**:
1. 顧客成功（例: 5分で配車）
2. 満足・口コミ
3. 新規ユーザー
4. ドライバー増加
5. 配車時間短縮
6. ①に戻る（加速）

### ステップ4: 指標定義（15分）

各ステップのKPIを決める:
- ステップ1: 平均配車時間
- ステップ2: NPS
- ステップ3: Organic成長率
- ステップ4: ドライバー数
- ステップ5: 配車成功率

### ステップ5: ボトルネック特定（15分）

**現状分析**:
- どのステップが最も遅い?
- どこを改善すべき?
