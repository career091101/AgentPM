# Problem Solution Fit (PSF)

> **出典**: 田所雅之「起業の科学」STEP 3
> **関連概念**: [[CONCEPT_CPF_001]], [[CONCEPT_PMF_001]]

---

## 1. 定義

**Problem Solution Fit (PSF)** とは、「特定の課題に対して適切なソリューションを見つけられた状態」。

CPF（課題検証）の次のステップ。「何を作るべきか」が明確になった状態。

**PSFの核心**:
- **Validated Problem（検証済み課題）**: CPFで確認済み
- **10x Solution（10倍ソリューション）**: 既存の10倍良い解決策
- **Early Adopter Fit（アーリーアダプター適合）**: 初期ユーザーが熱狂

**PMFパルテノン神殿の2階部分**:
```
        ┌─────────────────────────────────┐
        │    PMF（真のプロダクト適合）     │ 屋根
        └─────────────────────────────────┘
        ┌─────────────────────────────────┐
        │  PSF（ソリューション検証） ← ★  │ 2階
        └─────────────────────────────────┘
        ┌─────────────────────────────────┐
        │  CPF（課題検証）                 │ 1階
        └─────────────────────────────────┘
```

---

## 2. なぜ重要か

### 2.1 PSF未達成のリスク

**課題は正しいが、ソリューションが間違っている**:
- ❌ 顧客が本当に欲しいものと違う
- ❌ 複雑すぎる・シンプルすぎる
- ❌ 価格が高すぎる・安すぎる
- ❌ 既存ツールと大差ない（10倍がない）

**結果**:
- プロダクトを作ったが誰も使わない
- 時間とリソースの無駄
- ピボットが遅れる

### 2.2 PSF達成の効果

**プロダクト開発の方向性が明確**:
- ✅ 何を作るべきか分かる
- ✅ 優先順位が付けられる
- ✅ 初期ユーザーが熱狂する

**PMFへの道が開ける**:
- ✅ MVP構築の土台
- ✅ リテンション率向上
- ✅ 口コミが起き始める

## 3. 構成要素

### 3.1 Validated Problem（検証済み課題）

**CPF達成の確認**:
- [ ] 20人以上にインタビュー済み
- [ ] 70%以上が同じ課題を挙げる
- [ ] 3U検証済み（Unworkable/Unavoidable/Urgent）
- [ ] ペルソナが具体的に定義されている

参照: [[CONCEPT_CPF_001]], [[TACTIC_3U_VALIDATION_001]]

---

### 3.2 10x Solution（10倍ソリューション）

**既存解決策の10倍良いソリューション**:
- 時間が1/10
- コストが1/10
- 品質が10倍
- 体験が「魔法のよう」

**判定方法**:
- 顧客に実際に使ってもらう
- 「既存と比べて何倍良いか?」を聞く
- 目標: 平均8倍以上

参照: [[TACTIC_10X_VALIDATION_001]]

---

### 3.3 UVP（Unique Value Proposition）

**30秒で説明できる独自価値**:

フォーマット:
```
[ターゲット顧客]が
[解決したい課題]を解決するために
[プロダクト名]を使うと
[得られる便益]が得られます。
なぜなら[差別化要因]だからです。
```

**例（Slack）**:
```
リモートチームが
メール地獄とコミュニケーション混乱を解決するために
Slackを使うと
リアルタイムで透明性の高いコミュニケーションが実現します。
なぜなら、チャンネル単位で整理され、全履歴が検索可能だからです。
```

参照: [[TACTIC_UVP_CANVAS_001]]

---

### 3.4 MVP（Minimum Viable Product）

**最小限の機能で価値を証明**:
- 完璧なプロダクトではなく、学習ツール
- 10倍を体感できる最小機能のみ
- 1-4週間で構築できる範囲

**MVPタイプ**:
1. **Concierge MVP**: 手動で1人ずつサポート
2. **Wizard of Oz MVP**: 裏で人間が手動対応（自動に見せる）
3. **Landing Page MVP**: ページだけ作って反応を見る
4. **Prototype MVP**: 動くプロトタイプ

参照: [[TACTIC_MVP_TYPES_001]]

---

## 4. 判定基準・指標

### 4.1 定量指標

| 指標 | PSF達成基準 | 測定方法 |
|------|-----------|----------|
| **10倍検証** | 8倍以上 | 顧客に実測してもらう |
| **MVP登録数** | 30人以上 | ウェイトリスト登録 |
| **初期コンバージョン率** | 10%以上 | 登録→利用開始 |
| **アーリーアダプター熱狂度** | 80%以上 | 「友人に勧めるか?」Yes率 |
| **支払い意思** | 50%以上 | 「有料でも使うか?」Yes率 |

### 4.2 定性指標

- [ ] アーリーアダプターが「これを待っていた！」と言う
- [ ] 既存ツールから乗り換える意思がある
- [ ] 友人に勧めている（口コミ発生）
- [ ] 改善要望が具体的（使い込んでいる証拠）
- [ ] プロダクトなしでは困ると言う

## 5. 実践ステップ

### ステップ1: CPF達成確認（前提）

**チェックリスト**:
- [ ] 20人以上インタビュー完了
- [ ] 課題が3U検証済み
- [ ] ペルソナ定義済み
- [ ] 既存解決策の不満点明確

参照: [[CONCEPT_CPF_001]]

---

### ステップ2: ソリューション仮説設定（1-2日）

**質問リスト**:
1. 既存解決策の何が不満か?（インタビューで確認済み）
2. どうすれば10倍良くなるか?
3. 技術的に実現可能か?
4. ビジネスモデルとして成立するか?

**仮説フォーマット**:
```
[既存解決策]は[不満点]があるが、
[我々のソリューション]は[差別化要因]により、
[10倍改善の内容]を実現する。
```

**例（Zoom）**:
```
Skypeは接続が不安定で使いにくいが、
Zoomは独自の映像圧縮技術により、
安定した接続とワンクリック参加を実現する。
```

---

### ステップ3: UVP作成（2-3時間）

**ワークショップ形式**:
1. ターゲット顧客を明確化（ペルソナ）
2. 課題を1文で表現
3. ソリューションを1文で表現
4. 差別化要因を1文で表現
5. 30秒で説明できるか確認

**チェック**:
- [ ] 小学生でも理解できるか?
- [ ] 競合との違いが明確か?
- [ ] 顧客が「欲しい」と思うか?

参照: [[TACTIC_UVP_CANVAS_001]]

---

### ステップ4: MVP構築（1-4週間）

**MVPタイプ選択**:

| タイプ | 期間 | コスト | 学習効果 |
|--------|------|--------|----------|
| Landing Page | 1-3日 | 低 | 中 |
| Concierge | 1週間 | 中 | 高 |
| Wizard of Oz | 2週間 | 中 | 高 |
| Prototype | 2-4週間 | 高 | 高 |

**目標**:
- 10倍を体感できる最小機能
- 1-4週間で完成
- コスト最小化

参照: [[TACTIC_MVP_TYPES_001]]

---

### ステップ5: MVP検証（1-2週間）

**テストユーザー**:
- アーリーアダプター10-30人
- CPFインタビューした人を優先

**検証質問**:
```
Q1. 「既存ツールと比べて、何倍良くなりましたか?」
   → 10倍検証

Q2. 「このプロダクトが使えなくなったらどう思いますか?」
   → Sean Ellisテスト

Q3. 「有料でも使いたいですか? いくらまで?」
   → 支払い意思確認

Q4. 「友人に勧めますか?」
   → 口コミ可能性

Q5. 「一番良かった点は? 改善して欲しい点は?」
   → フィードバック収集
```

---

### ステップ6: PSF達成判定（1日）

**定量データ集計**:
- 10倍検証: 平均X倍（目標8倍以上）
- 支払い意思: XX%（目標50%以上）
- 推奨率: XX%（目標80%以上）

**定性データ分析**:
- 顧客の生の声を整理
- 共通する賞賛・不満を抽出

**判定**:
- **✅ PSF達成**: PMF検証へ進む
- **⚠️ 一部達成**: ソリューション調整
- **❌ 未達成**: ピボット検討

---

## 6. よくある失敗パターン

| パターン | 症状 | 対策 |
|---------|------|------|
| **CPFスキップ** | 課題検証せずソリューション開発 | 必ずCPF先行 |
| **10倍がない** | ちょっと良い程度（2倍） | 10倍を死守 |
| **完璧主義** | 完璧なプロダクト作ろうとする | MVPで十分 |
| **少なすぎる検証** | 5人だけテスト | 最低10-30人 |
| **アーリーアダプター以外** | 一般消費者にテスト | アーリーアダプターに絞る |
| **フィードバック無視** | 自分の理想を追求 | 顧客の声を最優先 |

## 7. 事例

### 7.1 成功事例: Dropbox

**CPF（課題）**:
- ターゲット: テックに詳しい人
- 課題: ファイル同期が面倒、USBメモリ紛失リスク

**PSF（ソリューション）**:
- 10倍: ファイル同期が自動（手動の1/100の手間）
- UVP: 「あなたのファイルがどこでも、どのデバイスでも」
- MVP: 動画デモ（Explainer Video）
  - プロダクト未完成
  - 動画だけで7万人ウェイトリスト登録
  - PSF達成を確認

**結果**: PMF達成、2018年IPO（時価総額$10B）

---

### 7.2 成功事例: Airbnb

**CPF（課題）**:
- ターゲット: イベント時の旅行者
- 課題: ホテルが高い・満室

**PSF（ソリューション）**:
- 10倍: 価格が1/3、ユニークな体験
- UVP: 「地元の人の家に、ホテルより安く泊まれる」
- MVP: 3人の創業者の部屋だけ掲載
  - 2007年デザインカンファレンス時にテスト
  - 3人の宿泊客獲得 → PSF検証

**ピボット**:
- PSF検証で「写真品質が重要」と判明
- プロカメラマン派遣サービス追加
- 予約率2-3倍向上

**結果**: PMF達成、2020年IPO（時価総額$100B）

---

### 7.3 失敗事例: Color Labs

**CPF（課題）**:
- 想定: 写真共有のニーズ
- 実態: 課題が不明確（CPF未達成）

**PSF（ソリューション）**:
- 10倍: なし（Instagramと大差ない）
- UVP: 曖昧（「近くの人と写真共有」）
- MVP: $41M調達後にフル機能アプリ開発
  - 複雑すぎて使われない
  - アーリーアダプターが熱狂しない

**結果**: $41M調達も1年で終了、Appleに売却

**教訓**:
- CPF未達成でPSF進むと失敗
- 10倍がないソリューションは刺さらない
- MVPで小さく検証すべきだった

## 8. PSFチェックリスト

### CPF達成（前提）
- [ ] 20人以上インタビュー済み
- [ ] 70%以上が同じ課題
- [ ] 3U検証済み
- [ ] ペルソナ定義済み

### ソリューション設計
- [ ] 既存解決策の10倍良いか?
- [ ] UVP が30秒で説明できるか?
- [ ] MVPタイプを選定したか?
- [ ] 1-4週間で構築可能か?

### 検証実施
- [ ] 10-30人のアーリーアダプターにテストしたか?
- [ ] 10倍検証で8倍以上達成したか?
- [ ] 支払い意思50%以上か?
- [ ] 推奨率80%以上か?

### PSF達成判定
- [ ] 定量指標すべて達成か?
- [ ] 顧客が熱狂しているか?
- [ ] 口コミが起き始めているか?
