# Persona Creation (ペルソナ作成)

> **出典**: 田所雅之「起業の科学」STEP 2
> **関連**: [[CONCEPT_CPF_001]], [[TACTIC_CUSTOMER_INTERVIEW_001]]

---

## 1. 定義

**Persona（ペルソナ）** とは、ターゲット顧客を具体的な1人の人物像として定義したもの。

「30代男性」といった抽象的なセグメントではなく、名前・顔写真・日常・悩みまで具体化する。

**目的**:
- チーム全員が同じ顧客像を共有
- プロダクト開発・マーケティングの意思決定の基準
- 「誰のために作るか」を明確にする

**重要な原則**:
> "Sell to everyone, and you sell to no one."
> （全員に売ろうとすると、誰にも売れない）

---

## 2. なぜ重要か

### 2.1 ペルソナがないと起こる問題

**意思決定のブレ**:
- 機能A: 「若い人向け」
- 機能B: 「ビジネスマン向け」
- → 結果: 誰にも刺さらないプロダクト

**マーケティングの失敗**:
- 「誰でも使えます」→ 誰の心にも響かない
- 広告が曖昧 → CTR低下、CAC高騰

**チーム内の認識ズレ**:
- エンジニア: 「技術者向けだと思ってた」
- マーケター: 「一般消費者向けだと思ってた」

### 2.2 ペルソナがあると得られるもの

**明確な意思決定基準**:
- 新機能追加すべきか? → ペルソナが喜ぶか?
- どのチャネルで広告? → ペルソナはどこにいる?

**刺さるメッセージ**:
- 抽象的: 「効率化ツール」
- ペルソナ向け: 「忙しいママが夕食準備を30分短縮できる」

**チームの一体感**:
- 全員が同じ人物をイメージできる
- 「田中さんは喜ぶか?」で議論できる

## 3. ペルソナの構成要素

### 3.1 基本情報

| 項目 | 内容 | 例 |
|------|------|-----|
| **名前** | 仮名でOK | 田中健太郎 |
| **年齢** | 具体的に | 35歳 |
| **性別** | | 男性 |
| **職業** | 具体的に | IT企業のプロダクトマネージャー |
| **年収** | | 800万円 |
| **家族構成** | | 妻・子供2人（5歳・3歳） |
| **居住地** | | 東京都世田谷区 |
| **顔写真** | フリー素材 | [写真] |

**重要**: 実在する友人・知人をモデルにすると具体性が増す

---

### 3.2 日常行動（A Day in the Life）

**目的**: ペルソナの1日を理解し、プロダクト利用シーンを想像

**フォーマット**:
```
6:00  起床、朝食準備
7:00  子供を保育園へ送る
8:30  通勤（電車でニュースアプリ）
9:00  出社、メール確認
10:00 会議（プロダクトロードマップ議論）
12:00 昼食（同僚とランチ）
13:00 開発チームとミーティング
15:00 ユーザーデータ分析（← ここで課題発生！）
18:00 退社
19:00 帰宅、夕食
21:00 子供寝かしつけ
22:00 自己学習（オンライン講座）
24:00 就寝
```

**ポイント**:
- どこで課題に直面するか特定
- いつプロダクトを使うか想像
- どんな時間制約があるか理解

---

### 3.3 課題・ペイン（Problems & Pains）

**トップ3の課題**:
1. **課題1（最重要）**: ユーザーデータ分析に時間がかかりすぎる
   - 詳細: SQL書いて、集計して、グラフ作成で2-3時間
   - 頻度: 週3回
   - 感情: イライラする、他の仕事が圧迫される

2. **課題2**: データが散らばっていて統合が面倒
   - 詳細: GA、Mixpanel、Amplitudeなど5つのツール行き来
   - 頻度: 毎日
   - 感情: 非効率さにストレス

3. **課題3**: チームメンバーに共有するレポート作成が面倒
   - 詳細: PowerPoint作成に1時間
   - 頻度: 週1回
   - 感情: 本質的でない仕事だと感じる

**3U検証との紐付け**:
- Unworkable（切実）: ✅ データ分析できないと意思決定できない
- Unavoidable（不可避）: ✅ PMの仕事として毎週必須
- Urgent（今すぐ）: ✅ 週次レポート締切が迫る

参照: [[TACTIC_3U_VALIDATION_001]]

---

### 3.4 ゴール・願望（Goals & Desires）

**短期ゴール（3ヶ月）**:
- データ分析時間を半分にしたい
- チームにインサイトを素早く共有したい

**長期ゴール（1-3年）**:
- データドリブンな組織を作りたい
- プロダクトマネージャーとしてキャリアアップ

**理想の状態**:
- 朝来たら自動でレポートが出来上がっている
- データを見てすぐ意思決定できる
- 分析作業ではなく、戦略思考に時間を使える

---

### 3.5 現在の解決策（Current Alternatives）

**使っているツール**:
1. Google Analytics（Web解析）
2. Mixpanel（イベントトラッキング）
3. SQL + BigQuery（データ集計）
4. Excel / Google Sheets（レポート作成）
5. PowerPoint（プレゼン資料）

**現在のワークフロー**:
```
1. BigQueryでSQL書く（30分）
2. データをSpreadsheetにエクスポート（10分）
3. グラフ作成（30分）
4. PowerPointに貼り付け（30分）
5. レビュー・修正（30分）
合計: 2時間10分
```

---

### 3.6 既存解決策への不満（Frustrations）

| 不満 | 詳細 | 強度（1-10） |
|------|------|--------------|
| **時間がかかる** | 毎回2時間以上 | 9 |
| **ツールが分散** | 5つのツールを行き来 | 8 |
| **再現性がない** | 毎回ゼロから作成 | 7 |
| **リアルタイムでない** | 昨日のデータしか見れない | 6 |
| **共有が面倒** | メール添付、更新の度に再送 | 7 |

**「理想のツールがあればいくら払う？」**:
- 月額5万円までなら即決
- 理由: 週6時間削減 = 月24時間 = 時給換算で十分ペイする

---

### 3.7 情報源・メディア（Information Sources）

**どこで情報を得るか**:
- **ニュース**: TechCrunch, Product Hunt
- **SNS**: Twitter（プロダクト界隈フォロー）、LinkedIn
- **コミュニティ**: Product Manager Slack, Reddit r/ProductManagement
- **学習**: Udemy, Coursera
- **イベント**: ProductCamp Tokyo（年2回参加）

**意思決定プロセス**:
1. Twitter で新ツールを知る
2. Product Hunt でレビュー確認
3. 無料トライアル試す
4. 同僚・コミュニティで評判聞く
5. 導入判断（上司承認必要）

**広告反応**:
- Facebook広告: ✗ 見ない
- Google検索広告: ✅ 課題で検索する
- Twitter広告: △ たまに見る
- 口コミ: ✅ 最重視

---

### 3.8 価値観・動機（Values & Motivations）

**大切にしていること**:
- **効率性**: 無駄な作業が嫌い
- **データドリブン**: 勘ではなくデータで判断
- **学習意欲**: 常に新しいツール・手法を学びたい
- **家族との時間**: 残業せず19時には帰宅したい

**行動の動機**:
- **恐怖**: データ分析できず間違った判断をすること
- **願望**: チームから「データの達人」と認められたい
- **承認欲求**: 上司・経営陣に評価されたい

**性格**:
- アーリーアダプター（新ツール積極的に試す）
- 論理的思考
- せっかち（すぐ結果が欲しい）

## 4. ペルソナ作成プロセス

### ステップ1: 顧客インタビュー（最重要）

**インタビュー数**: 最低10-20人

**質問例**:
- 1日のスケジュールを教えてください
- 一番困っていることは何ですか?
- 今どうやって解決していますか?
- 理想のツールがあれば何ができますか?
- いくらなら払いますか?

参照: [[TACTIC_CUSTOMER_INTERVIEW_001]]

### ステップ2: データ整理（1-2時間）

**共通パターン抽出**:
- 10人中8人が「時間がかかる」と回答 → 最重要課題
- 職業: PM、マーケター、データアナリストが多い
- 年齢: 30-40代中心

### ステップ3: ペルソナ作成（2-3時間）

**テンプレート使用**:
- 1枚のA4またはスライドにまとめる
- 写真・名前・基本情報
- 課題・ゴール・日常行動

**チーム全員で作成**:
- 創業者全員参加
- 初期メンバーも参加
- 議論しながら合意形成

### ステップ4: 検証（継続的）

**リアル顧客と比較**:
- ペルソナ通りの顧客が実際にいるか?
- 想定と違う点は?

**更新**:
- 3ヶ月ごとに見直し
- 新しいインタビュー結果を反映

## 5. よくある間違い

| 間違い | 症状 | 対策 |
|--------|------|------|
| **抽象的すぎる** | 「30代男性、会社員」止まり | 名前・顔写真・日常まで具体化 |
| **複数ペルソナ作成** | 5人のペルソナ作成 | 最初は1人に絞る |
| **理想のペルソナ** | 実際にいない架空の人物 | 実在する人をモデルに |
| **インタビューなし** | 想像だけで作成 | 必ず10人以上インタビュー |
| **作って終わり** | 壁に貼って放置 | 毎週の会議で言及 |
| **全員に当てはまる** | 「誰でも使える」前提 | 尖ったペルソナを作る |

## 6. 事例

### 6.1 成功事例: Slack

**ペルソナ名**: Emily（エミリー）

| 項目 | 内容 |
|------|------|
| 基本情報 | 32歳、女性、スタートアップのプロジェクトマネージャー |
| 日常 | リモートワーク、Zoom会議多数、メール地獄 |
| 課題 | メールが多すぎて重要な情報を見逃す、プロジェクト進捗が把握しづらい |
| ゴール | チームとリアルタイムでコミュニケーション、情報を一元管理 |
| 現在の解決策 | Email, Skype, Googleドライブ |
| 不満 | ツールが分散、リアルタイムでない、検索しづらい |
| 情報源 | Product Hunt, Twitter, TechCrunch |

**結果**: Emilyのような「リモートワークで情報過多に悩むPM」に刺さった → 2014年ローンチ、2019年時価総額$20B

---

### 6.2 失敗事例: Quibi

**ペルソナ想定**: 通勤中にスマホで短編動画を見る人

**問題点**:
- ❌ 実際の顧客インタビュー不足
- ❌ 「短編動画需要がある**はず**」という思い込み
- ❌ 実在する具体的な人物像が不明

**実際の市場**:
- 通勤中は既に YouTube, TikTok で満たされている
- わざわざ月額課金する理由がない

**結果**: $1.75B調達も6ヶ月で終了
