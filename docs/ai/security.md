# セキュリティ・機密情報管理

## 基本原則

1. **機密情報はコードにハードコードしない**
2. **.env ファイルはGit管理外**
3. **公開リポジトリに機密データを含めない**

## .env 管理

```bash
# .env ファイル作成
cp .env.example .env
```

`.gitignore` に必ず含める：
```
.env
.env.local
*.pem
*.key
```

## APIキー管理

| サービス | 環境変数名 | 用途 |
|----------|-----------|------|
| OpenAI | `OPENAI_API_KEY` | GPT利用 |
| Google | `GOOGLE_API_KEY` | 検索・カレンダー |
| Anthropic | `ANTHROPIC_API_KEY` | Claude利用 |

## 機密ドキュメント

機密性の高いドキュメントは：
- `Flow/` 内で作業（公開前）
- `Stock/` 移動前にレビュー
- 必要に応じてプライベート領域に配置

## 注意事項

- プロジェクト憲章に機密情報を含めない
- ステークホルダー情報は匿名化を検討
- Web検索結果の出典を必ず記載
