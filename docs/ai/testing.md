# テスト方針

## 基本方針

- ドキュメント生成後は必ず内容を確認
- 「確定反映して」前に完全性をチェック
- スクリプト実行前にパラメータを検証

## バックログ検証

```bash
python scripts/validate_backlog_yaml.py
```

YAMLの構文エラー、必須フィールド欠落をチェック。

## ドキュメント検証チェックリスト

### 一般
- [ ] タイトルが適切
- [ ] 必須セクションが存在
- [ ] 日付が正しい
- [ ] パスが正しい

### プロジェクト憲章
- [ ] 目的が明確
- [ ] スコープが定義
- [ ] ステークホルダーが特定

### ペルソナ
- [ ] 名前・属性が設定
- [ ] ニーズ・課題が記載
- [ ] 行動パターンが定義

### WBS
- [ ] 階層構造が適切
- [ ] 全タスクが網羅
- [ ] 依存関係が明確
