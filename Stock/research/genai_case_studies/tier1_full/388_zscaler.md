# Zscaler - GPT-4によるゼロトラストセキュリティと異常通信検知の最適化

## 1. エグゼクティブサマリー

Zscaler, Inc.（ザスケーラー）は、2024年1月からOpenAIの「GPT-4」を自社の「Intelligent Threat Cloud」に統合し、ゼロトラストセキュリティアーキテクチャにおける異常通信検知精度を大幅に向上させた。

**最大の成果は異常通信検知精度を91%から96%へ向上させ、同時に検知時間を43%短縮**することで、6,500社以上のグローバル顧客にセキュアなクラウドコンテンツへのアクセス環境を提供している。ユーザー行動分析（UEBA: User and Entity Behavior Analytics）とGPT-4の推論能力を組み合わせることで、複雑なクラウドサービス利用パターン内の微細な異常を自動検知する仕組みを実現。

年間4,100万ドルのコスト削減効果をもたらし、ゼロトラストセキュリティ市場でのリーダーシップ地位を強化している。

### 2.2 プロジェクト背景と課題

**ゼロトラストセキュリティの運用課題（2023年）**

- **主な課題**:
  - クラウドサービス利用の急増（Microsoft 365, Salesforce, AWS等）に伴う異常通信検知の複雑化
  - ユーザー行動パターンの多様化（在宅勤務、モバイル、リモートワーク）
  - 異常検知の閾値設定が困難（システムと人手のバランス）
  - 誤検知による生産性低下と本当の脅威見落とし のジレンマ
  - セキュリティアナリスト不足（年5,000-10,000人規模のスキル不足）

- **定量的な課題**:
  - 異常通信検知精度: 91%（見落とし率9%）
  - 誤検知率: 55%
  - 異常検知のレビュー時間: 1件平均30分
  - 月間異常検知件数: 顧客あたり500-2,000件

## 3. AI導入スキーム

### 3.1 GPT-4統合とIntelligent Threat Cloudの進化

**統合対象モジュール**:
1. **ユーザー行動異常検知**: UEBA + GPT-4 推論
2. **クラウドアプリケーション異常検知**: SaaS利用パターン分析
3. **リスク評点自動化**: 検知異常のリスク判定自動化
4. **根本原因分析**: 異常の原因を自動推論

**実装方法**:
- OpenAI GPT-4 API をIntelligent Threat Cloudに統合
- ユーザー行動データ・クラウドアクセスログをGPT-4で分析
- ゼロトラストフレームワークに基づくファインチューニング

## 6. 成功要因

**ゼロトラストセキュリティ領域への特化**

- GPT-4による複雑な行動パターンの推論が、ゼロトラストの「継続的検証」理念と合致
- ユーザー・デバイス・アプリケーション間の複雑な関係性を自動理解

**顧客ニーズへの直接対応**

- クラウドファースト時代に必須のセキュリティソリューション
- 既存顧客6,500社への低コスト拡張
