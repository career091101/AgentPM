# Planet Labs - 生成AI導入事例

## 1. 基本情報

### 企業概要
| 項目 | 内容 |
|------|------|
| 企業名（日本語） | Planet Labs |
| 企業名（英語） | Planet Labs PBC |
| 本社所在地 | 米国・カリフォルニア州サンフランシスコ |
| 業界 | 衛星画像・地球観測 |
| 従業員数 | 約800名 |
| 設立年 | 2010年 |
| AI導入開始年 | 2023年 |
| 株式公開 | NYSE上場（PL） |

### 事業概要
Planet Labsは、地球観測衛星の設計・製造・運用を行う宇宙テクノロジー企業です。200機以上の小型衛星（Dove）を運用し、毎日地球全体を撮影する能力を持ち、農業、林業、都市計画、環境モニタリング、災害対応など幅広い分野にデータを提供しています。

---

## 2. 導入背景

### 導入前の課題
1. **膨大な画像データ処理**: 毎日数百万枚の衛星画像を処理する必要性
2. **分析速度のボトルネック**: 専門アナリストによる手動分析では需要に追いつかない
3. **顧客サポート負荷**: 技術的な問い合わせへの対応工数増大
4. **データ活用のハードル**: 非専門家にとって衛星画像分析の敷居が高い

### 導入目的
- 衛星画像解析の自動化と精度向上
- 環境変化検知のリアルタイム化
- 顧客への洞察提供の迅速化
- データアクセシビリティの向上

### AI選定理由
**Claude（Anthropic）を選定**
- 複雑な技術文書の理解・生成能力
- 大量データ処理における安定性
- 環境・サステナビリティ分野への親和性
- セキュリティとプライバシーへの配慮

## 3. 技術詳細

### 使用AIツール/LLM
| ツール | 用途 |
|--------|------|
| Claude | 画像解析レポート生成、顧客サポート |
| カスタムML | 衛星画像の変化検知、オブジェクト認識 |
| Computer Vision | 土地利用分類、森林被覆解析 |

### 統合方法
- **API統合**: Claude APIを社内システムに統合
- **画像パイプライン**: ML前処理→Claude解析→レポート生成の自動化
- **ダッシュボード連携**: 顧客ポータルへの洞察自動配信

### データソース
- 毎日撮影される衛星画像（約300万km²/日）
- 過去10年分のアーカイブデータ
- 気象データ、地理空間データとの統合

### インフラ構成
- クラウドベースの画像処理パイプライン（AWS/GCP）
- エッジコンピューティングによる初期処理
- 分散ストレージシステム

### セキュリティ対策
- 衛星画像データの暗号化
- 政府・防衛関連顧客向けの高セキュリティ環境
- SOC2 Type II準拠

**中程度** - 日本でも農業、災害対応、都市計画での衛星画像活用ニーズは高い

### 注意点
- 日本固有の土地利用パターンへのモデル調整が必要
- 政府調達における国産技術優先との兼ね合い
- 衛星画像の解像度・頻度要件の確認
