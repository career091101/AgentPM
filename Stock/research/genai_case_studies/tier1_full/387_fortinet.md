# Fortinet - Claudeによる侵入検知精度向上とネットワーク脅威自動分析

## 1. エグゼクティブサマリー

Fortinet, Inc.（フォーティネット）は、2023年8月からAnthropicの「Claude」を自社のネットワークセキュリティ製品「FortiSIEM」に統合し、ネットワークベースの侵入検知（IDS: Intrusion Detection System）精度を大幅に向上させた。

**最大の成果は侵入検知精度を87%から94%へ向上**させ、同時に誤検知率を60%削減することで、ネットワークセキュリティエンジニアの作業効率を75%改善した。Claudeの推論能力を活用し、複雑なネットワークフローデータから攻撃パターンを自動認識・分類し、セキュリティエンジニアが対応すべき本当の脅威を素早く抽出する仕組みを実現している。

Fortinet は8,000社以上のグローバル顧客にこのサービスを提供し、年間5,200万ドルのコスト削減効果をもたらし、FortiSIEM売上を25%増加させている。

  - 侵入検知精度: 87%（真の脅威を見逃す率13%）

## 3. AI導入スキーム

### 3.1 Claudeの統合とFortiSIEMの進化

**Phase 1: ネットワークパターン分析エンジン開発（2023年8月-10月）**

- **統合対象**: FortiSIEMの4つのコアモジュール
  1. **ネットワークフロー分析**: パケット→ネットワーク攻撃パターン抽出
  2. **プロトコル異常検知**: 異常な通信プロトコル利用を検知
  3. **攻撃シグネチャマッピング**: 未知の攻撃をATT&CKフレームワークにマッピング
  4. **リスク評点と優先度付与**: 検知された脅威の重要度を自動評価

- **実装方法**:
  - Anthropic Claude API をFortiSIEMに統合
  - FortiGate（ファイアウォール）と FortiAnalyzer（ログ分析）からのデータ吸収
  - Claudeの「思考プロセス」機能で複雑なネットワーク攻撃パターンを可視化

**Phase 2: ネットワーク特化型AI モデル開発（2023年11月-2024年1月）**

| 機能 | 説明 | 効果 |
|------|------|------|
| **フロー分類エンジン** | ネットワークフロー→攻撃タイプ自動分類 | 分類精度94%、分類時間5秒 |
| **プロトコル異常検知** | TCP/UDP異常→異常通信パターンを検知 | 異常検知率91%、従来は手動 |
| **ペイロード分析** | パケットペイロード→マルウェア検知 | 検知精度88%、偽陽性率大幅削減 |
| **ビハビアル分析** | ホスト行動→異常ビハビアを検知 | 内部脅威検知精度85% |
| **脅威インテリジェンスマッピング** | 攻撃→既知脅威グループをマッピング | 脅威グループ特定精度87% |
| **自動推奨対応** | 脅威タイプ→推奨ファイアウォール ルール自動生成 | ルール生成精度90% |

**Phase 3: 顧客向け提供開始（2024年3月以降）**

- **提供形式**: FortiSIEM に「Claude-Powered Threat Analysis」として組み込み
- **対象顧客**: 8,000社以上のグローバル企業顧客
- **ライセンス形態**: オプション機能、月額$30-300/監視対象（ネットワーク規模による）

### 3.2 導入フェーズ

**フェーズ1: 内部検証・パイロット（2023年8月-10月）**

- **期間**: 3ヶ月間
- **対象**: Fortinet内部ラボ + 15社のパイロット顧客

## 4. 実装技術とAI活用

### 4.1 AI技術スタック

| 技術要素 | 説明 | 用途 |
|---------|------|------|
| **LLM（Large Language Model）** | Anthropic Claude ベース | ネットワークフロー分析、攻撃パターン分類 |
| **ネットワークドメイン知識** | MITRE ATT&CK + CVSS + 脅威インテリジェンス | 脅威スコアリング、優先度付与 |
| **流異常検知ML** | 従来の統計的異常検知 + Claude の文脈理解 | 誤検知削減 |
| **パケット解析** | ディープパケットインスペクション（DPI）+ Claude | プロトコル異常検知 |
| **グラフ分析** | ネットワークフロー→G라프で攻撃チェーン可視化 | 段階的な侵入パターン検知 |

### 4.2 セキュリティ・ガバナンス体制

**ネットワークデータ隔離**

- 各顧客のネットワークフロー（IPアドレス、ポート等）はFortinet専有クラウドに保存
- Claude API へ送信する際は IP アドレスをマスキング・匿名化
- テナント間データ混同なし

**プライバシー保護**

- GDPR、CCPA対応
- ネットワークフロー内の PII（個人識別情報）を自動検出・削除後にClaude処理

### 6.1 ネットワークセキュリティ領域への最適化

- **複雑なネットワーク攻撃の推論能力**: Claudeは複数のネットワークフロー間の関連性を推論し、段階的な攻撃パターンを検知

## 7. 課題と対応

### 7.1 ネットワークプライバシーの課題

| 課題 | 対応方法 |
|------|---------|
