## 1. エグゼクティブサマリー

東京都は2023年8月23日、生成AI（ChatGPT）を全庁の職員5万人が利用できる体制を整備し、日本の自治体における最大規模の生成AI全庁導入を実現しました。Microsoft Azure OpenAI Serviceを採用した安全な利用環境の構築、職員向けガイドラインの策定（2024年4月にVersion 2.0に更新）、および34の実践的活用事例の公開により、生成AIを組織規模で安全かつ効果的に運用するモデルケースを確立しています。

本事例は、従業員規模が大きな公的機関における生成AI導入の手法、ガイドライン策定プロセス、職員教育体制、セキュリティ対策など、他の自治体や民間企業の参考となる重要な知見を提供しています。

---

## 2. 組織概要と背景

### 2.1 組織の基本情報

東京都は日本最大規模の自治体として、東京都知事小池百合子の下、デジタル変革を重点施策として推進しています。特に、デジタルサービス局の設置により、行政のデジタル化を組織的に推進してきました。

**主な特性:**
- 職員数：約5万人（全庁）
- 部局数：28局
- 管轄地域：東京都全域（1,300万人以上の住民）
- デジタル化推進体制：デジタルサービス局が中核機関

### 2.2 導入前の課題と背景

東京都が生成AI導入を決定した背景には、以下のような行政組織が直面する課題がありました。

**業務効率化への需要:**
- 大量の文書作成業務（企画書、報告書、議事録など）
- 定型的なQ&A作成や説明資料の作成時間
- 職員の事務作業負担の軽減

**デジタル変革の推進:**
- AI技術の急速な進展に対応した組織体制整備
- 職員のスキル向上とリテラシー養成
- テクノロジーを活用した行政サービス品質の向上

**先行事例の検証:**
- 横須賀市でのパイロットプログラム（2023年初期）において、ChatGPTの活用により業務時間の短縮が確認されていました
- 先行自治体の成功事例に基づく全庁展開の判断

## 3. AI導入の戦略と計画

### 3.1 導入方針

東京都は、生成AIの導入にあたり、以下の3つの基本方針を掲げました。

**安全性の確保:**
- 個人情報や機密情報の漏えいリスクを最小化
- Azure OpenAI Serviceの採用により、入力データが学習目的で利用されないこと、サーバー側での保存が行われないことを確認

**職員への啓発と教育:**
- 導入前の十分な情報提供と研修
- ガイドラインの策定と周知徹底
- 段階的な運用開始と改善

**効果的な活用支援:**
- 実践的なプロンプト事例の提供
- 各部局のユースケース発掘支援
- 継続的な改善と新しい活用機会の探索

### 3.2 導入スケジュール

**2023年4月:** デジタルサービス局にプロジェクトチーム立ち上げ
- ChatGPTの活用可能性に関する検討開始

**2023年8月23日:** 全庁導入を発表
- 文章生成AI利活用ガイドライン（Version 1.0）の公開
- Azure OpenAI Serviceの運用開始

**2023年10月:** 職員利用状況アンケート実施
- 導入効果の初期測定
- 課題と改善点の抽出

**2023年11月～2024年1月:** アイデアソン・勉強会開催
- 全28局対象の複数回開催
- 職員からのプロンプト、ユースケース案の集約
- 約600個のアイデアを収集、103個を抽出

**2024年2月:** 文章生成AI活用事例集の公開
- 34の実践的な活用事例を掲載

**2024年4月:** 文章生成AI利活用ガイドライン Version 2.0の策定
- プラグインサービス対応
- 文章生成以外の生成AI活用可能性の検討項目追加

## 4. 技術選定と実装方式

### 4.1 採用した技術プラットフォーム

**ChatGPT（OpenAI）**
- 生成AIの中核サービス
- テキスト生成、要約、編集、質問応答機能

**Azure OpenAI Service（Microsoft）**
- 安全で安心な企業向け展開環境
- データプライバシー保護機能搭載

### 4.2 セキュリティ対策

東京都が採用したAzure OpenAI Serviceには、以下の重要なセキュリティ機能が含まれています。

**データ保護:**
1. 入力データが学習目的で利用されないこと
2. サーバー側でのデータ保存が行われないこと
3. 個人情報・機密情報漏えいリスクの最小化

**アクセス管理:**
- 職員の個別認証
- 部局別のアクセス権限管理
- 利用ログの記録と監視

**運用ガイドラインの徹底:**
- 個人情報を含む情報のプロンプト入力禁止
- 機密情報の取り扱い基準の明示
- 違反時の対応プロセスの定義

### 4.3 インフラストラクチャ

**展開方式:** クラウド型（Azure環境）
**利用者アクセス:** 都内ネットワークおよび許可されたリモートアクセス
**サポート体制:** デジタルサービス局を中核とした継続的支援

## 7. ガイドラインと運用規定

### 7.1 文章生成AI利活用ガイドラインの概要

東京都は、生成AI導入と同時に「文章生成AI利活用ガイドライン」を策定しました。このガイドラインは、職員が安全かつ効果的にChatGPTを利用するための包括的なルールを定めています。

**ガイドラインの構成:**

**第1部：生成AIの特性と活用の考え方**
- 生成AIの仕組みと限界（幻覚現象、学習データのバイアスなど）
- 行政業務での活用に適した場面と不適切な場面
- リスク評価フレームワーク

**第2部：セキュリティとコンプライアンス**
- 入力禁止情報（個人情報、機密情報の定義）
- データ漏えい防止対策
- 違反時の対応プロセス

**第3部：実践的活用方法**
- プロンプト設計の基本原則
- 効果的なプロンプト例（34事例）
- よくある質問と回答

**第4部：職員向けチェックリスト**
- 生成AI利用前の確認事項
- 出力内容の検証方法
- ドキュメント化と承認プロセス

### 7.2 Version 2.0での改定内容（2024年4月）

2024年4月に策定されたVersion 2.0では、初版導入後の成果と課題を踏まえ、以下の改定が加えられました。

**拡張内容:**
1. **プラグインサービス対応** - ChatGPTのプラグイン機能の活用ガイド
2. **文章生成以外の生成AI** - 画像生成AI、音声処理AIなど新しいモダリティへの対応
3. **ファインチューニング・カスタマイズ** - 部局独自のモデル カスタマイズの可能性検討
4. **倫理的配慮** - AIの透明性、バイアス対策、人間の判断の重要性強調

**改定の背景:**
- ChatGPT技術の急速な進化への対応
- 職員からの新しい活用ニーズの集約
- 他自治体の導入事例への対応

### 7.3 禁止事項と注意事項

**明確に禁止される入力:**
- 氏名、住所、電話番号などの個人識別情報
- 顧客ID、納税者番号などの機密ID
- 医療記録、法的文書、機密会議の議事
- 個人の評価・人事情報

**注意が必要な領域:**
- 著作権保護情報（許可なく利用された可能性）
- 判例や法令解釈（確実な法的アドバイスではない）
- 統計データや予測情報（精度検証が必要）

## 8. 課題と制限事項

### 8.1 認識された課題

**精度と信頼性の問題:**
- 先行自治体（横須賀市）で調査した職員の約50%が、ChatGPTの回答精度に不満を報告
- 幻覚（ハルシネーション）：存在しない情報を事実のように生成する現象

**プロンプト設計の難しさ:**
- 効果的なプロンプト作成には経験とスキルが必要
- 職員からの「プロンプトサポート」への要望が多い

**組織的な活用のばらつき:**
- 部局によって活用程度に大きな差異
- デジタルリテラシーの差に基づく活用格差の発生

**セキュリティ意識の維持:**
- 職員教育の継続的な実施が必要
- 違反事例への迅速な対応体制整備

### 8.2 今後の改善計画

**プロンプト支援の強化:**
- 部局別の専門プロンプト集作成
- ChatGPT活用教室の開催（初級者～上級者向け段階化）

**精度向上への取り組み:**
- ファインチューニングによる東京都固有の法令・施策知識の学習
- 回答検証プロセスの標準化

**新技術への対応:**
- 画像生成AI、音声処理AIなどへの対応検討
- マルチモーダルAI活用の準備

## 11. 実装上の教訓と成功要因

### 11.1 成功の鍵となった要素

**1. 明確なガイドラインの策定**
- 導入と同時期にガイドラインを策定し、職員に安心感を提供
- セキュリティとコンプライアンスに対する懸念を払拭

**2. 職員の積極的な巻き込み**
- アイデアソン・勉強会による「ボトムアップ」の活用事例発掘
- 職員の創意工夫を認め、フィードバック機会の提供

**3. 段階的な拡大と継続的改善**
- Version 1.0 → Version 2.0への改定により、技術進化に対応
- 定期的なアンケート実施による改善点の抽出

**4. 外部専門家の活用**
- Algomaticなどの外部エキスパートの知見を活用
- プロンプト設計の質的向上

**5. セキュリティに対する真摯な取り組み**
- Azure OpenAI Serviceの選定により、データ保護を実現
- 禁止事項の明確化と啓発

### 11.2 実装上の教訓

**実装前の準備が重要:**
- 十分なプロジェクト検討期間（4ヶ月）を確保
- リスク分析と対応策の事前準備

**職員教育の継続実施:**
- 導入直後だけでなく、継続的な教育が必要
- デジタルリテラシーのばらつきへの対応

**プロンプト設計支援の提供:**
- 単なるツール提供ではなく、活用支援が効果を大きく左右
- 段階的なレベル別研修の実施

**セキュリティと利便性のバランス:**
- 過度な制限は利活用を阻害
- リスク基準を明確にし、判断基準を統一

**組織規模での導入は段階的に:**
- 全庁同時導入でも、実質的には段階的な成熟化が必要
- 早期段階での無理な拡大より、質的向上を優先

## 14. まとめと結論

### 14.1 事例の総括

東京都の生成AI（ChatGPT）全庁導入は、日本における自治体・大規模組織のAI導入における「ケーススタディ」として高い価値を持っています。以下の成果を達成しました。

**実装的成果:**
- 5万職員規模での全庁ChatGPT導入を実現
- Azure OpenAI Serviceによる安全で安心な利用環境の構築
- セキュリティとコンプライアンスを両立させた運用体制の確立

**組織的成果:**
- 職員向けガイドラインの策定と継続的改定（Version 1.0 → 2.0）
- 600個のアイデアから103個の有望事例を抽出、34事例を事業化
- 約10分/日の業務時間短縮を達成（先行自治体実績に基づく）

**社会的成果:**
- 他自治体への情報提供と指導的立場の確保
- 民間企業のAI導入に対するベストプラクティスの提示
- デジタル行政への市民信頼の向上

### 14.2 重要な学習ポイント

本事例から得られた主要な学習ポイント：

1. **規模が大きいほど、ガイドラインと教育が重要**
   - 5万人の職員に対して、統一されたルールと継続的教育が不可欠

2. **セキュリティを標準機能として組み込む**
   - 事後対応ではなく、プラットフォーム選定時点でセキュリティを確保

3. **職員の創意工夫を引き出すプロセスが成功の鍵**
   - トップダウンではなく、ボトムアップの活用事例発掘が高い利用率を実現

4. **技術進化に対応した段階的な拡張計画**
   - Version 2.0への改定により、プラグイン対応や新モダリティへの柔軟性確保

5. **効果測定と透明性の重要性**
   - 定期的なアンケート実施と効果の可視化が、継続的な改善を推進

### 14.3 今後の展望と課題

**東京都の今後の方向性:**
- 他自治体の模範となる先進的な取り組み継続
- 全国自治体へのガイドライン・ベストプラクティスの共有
- 中央政府との連携による国レベルのAI活用基準の検討

**業界全体への貢献:**
- 生成AIの公共部門での活用モデルの確立
- セキュリティとコンプライアンスの両立方法の提示
- 職員教育とスキル向上の具体的手法の提供

**残る課題:**
- 精度向上に向けた継続的な取り組み（幻覚現象への対処）
- 部局間の活用格差の是正
- 市民向けサービスへの安全な展開方法の検討
