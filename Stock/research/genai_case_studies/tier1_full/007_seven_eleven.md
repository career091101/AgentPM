# セブン-イレブン・ジャパン：AI発注数提案システム導入事例

## エグゼクティブサマリー

セブン-イレブン・ジャパンは、全国21,000店舗を擁する日本最大級のコンビニチェーンです。2020年より段階的に導入開始したAI発注数提案システムにより、**発注業務にかかる時間を約40%削減**することに成功しました。野村総合研究所（NRI）との共同開発、Google Cloudへの移行により、2023年3月から全店舗での本格運用が実現。2025年春には固定型ストアコンピューターからタブレット・スマートフォンへの移行を予定し、さらなる効率化を進めています。

---

## 1. プロジェクト背景と課題

### 1.1 セブン-イレブン・ジャパンの概要

セブン-イレブン・ジャパンは、1974年の設立以来、日本国内で最大規模のフランチャイズコンビニエンスストアチェーンとして、全国21,000店舗を超える規模で営業しています。営業時間は24時間、取扱商品は約2,500～3,000品目と、小売業の中でも特に多品目・多店舗運営の複雑さを抱える企業です。

### 1.2 従来の発注システムの課題

従来の発注方法は「**設定発注**」方式でした：

- **手動設定**: 店舗従業員が手動で各商品の基準在庫数を設定
- **受動的対応**: 在庫が一定数以下に減ってから、ストア・コンピューター（SC）が発注を提案
- **時間負担**: 発注担当者が約2,800商品の販売動向を確認しながら設定作業を実施
- **品切れリスク**: 在庫がなくなる前に発注できないため、欠品による販売機会ロスが頻発
- **属人的判断**: 従業員の経験やバイアスが反映されやすく、店舗ごとの最適化が困難

### 1.3 発注業務の具体的な負担

- **作業時間**: 1店舗あたり1日約80分を発注作業に費やしていた
- **スケールの問題**: 21,000店舗 × 80分 = 年間約100万時間（推定）
- **機会損失**: 品切れによる販売機会ロス、不適切な在庫による廃棄増加
- **スタッフ疲弊**: 日々の業務に加え、定期的な在庫設定作業が負担

### 1.4 物流・労働市場の環境変化

- **2024年問題**: 物流業界における労働時間上限規制の導入
- **人手不足**: コンビニ業界全体での人材確保の困難化
- **消費者期待**: 24時間、常に品揃え豊富な状態の維持要求

## 2. AI発注システムの構想と開発体制

### 2.1 開発方針

セブン-イレブン・ジャパンは、以下の方針でAI発注システムを構想しました：

1. **外部専門機関との連携**: 野村総合研究所（NRI）をパートナーとして選定
2. **クラウド基盤の活用**: スケーラビリティと高速処理を実現するためGoogle Cloudを採用
3. **段階的導入**: パイロット → 拡大 → 全店舗展開の3段階アプローチ
4. **人とAIの協業**: AIの提案に対し、人的判断を挟む仕組みの構築

### 2.2 技術パートナーとしてのGoogle Cloud

Google Cloudの選択理由：

- **BigQuery**: 大規模データの分析・機械学習に最適化
- **予測精度**: 毎月数百パターンのAIモデルを自動生成可能
- **スケーラビリティ**: 21,000店舗 × 2,800商品のデータ処理に対応
- **機械学習**: ベース部分は機械学習に任せ、開発コストを削減

### 2.3 開発体制

| 役割 | 機関 | 責務 |
|------|------|------|
| **事業責任** | セブン-イレブン・ジャパン | 要件定義、事業戦略、全店舗展開 |
| **システム開発** | 野村総合研究所（NRI） | AI・機械学習モデル開発、システム統合 |
| **クラウド基盤** | Google Cloud | インフラ、BigQuery、スケーラビリティ |

## 3. AI発注システムの仕組みと技術

### 3.1 従来方式と新方式の比較

#### 従来の「設定発注」方式
```
販売 → 在庫減少 → 基準在庫以下 → SC自動提案 → 従業員確認・発注
（受動的、後追い型）
```

#### AI発注方式
```
販売データ + 天候 + 曜日 + 季節 + リードタイム
→ AI需要予測 → 基準在庫の自動算出 → 品切れ前に発注提案
（予測的、先制型）
```

### 3.2 AI発注システムが考慮する要素

AIモデルは以下のファクターを多次元的に分析：

| 要素 | 説明 | 重要性 |
|------|------|--------|
| **過去販売実績** | 過去3ヶ月～1年の販売トレンド | 高 |
| **天候・気温** | 各地域の気象データ、季節変動 | 高 |
| **曜日特性** | 平日・週末・祝日の購買パターン | 中 |
| **商品特性** | カテゴリ、賞味期限、季節品 | 中 |
| **納品リードタイム** | 発注から到着までの日数 | 中 |
| **売場レイアウト** | 実際の陳列面積、在庫保持可能数 | 低 |

### 3.3 技術スタック

```
データ入力層：
├─ POS（販売データ）
├─ 在庫管理システム
├─ 気象API
└─ 店舗マスタ

処理層（Google Cloud）：
├─ BigQuery（データウェアハウス）
├─ Vertex AI（機械学習）
├─ Cloud Functions（自動化処理）
└─ Python（カスタムロジック）

出力層：
├─ Web UI（マネージャー向け）
├─ タブレット/スマートフォン（店舗向け）
└─ SC連携（レガシー環境対応）
```

### 3.4 予測精度向上のプロセス

1. **ベースモデル構築**: 過去販売データから初期モデルを構築
2. **地域・店舗別調整**: 各地域の特性に応じたパラメータ調整
3. **商品カテゴリ最適化**: 商品ごとに異なるモデルを構築
4. **月次モデル更新**: 毎月数百パターンのモデルを自動検証・更新
5. **準拠率フィードバック**: 従業員の採用/非採用データをモデルに反映

## 4. 段階的導入戦略

### 4.1 Phase 1: パイロット導入（2020年1月～2021年）

**対象**: 千葉県内のセブン-イレブン約1,100店舗

**実施内容**：
- 対象商品: 日用品など約2,500品目
- 対象カテゴリ: 初期段階では安定需要の日用品・食品に限定
- 期間: 12ヶ月の検証運用

**初期成果**：
- 発注時間: **80分/日 → 45分/日**（約44%削減）
- AI提案準拠率: **約55%** （スタート当初）
- 欠品率: わずかに改善傾向

**学習点**：
- AI提案への信頼構築が重要
- 商品カテゴリによって精度が大きく異なる
- リアルタイム需要変動への対応が課題

### 4.2 Phase 2: Google Cloud移行と精度向上（2022年7月）

**転機**: システム基盤をGoogle Cloudに移行

**改善内容**：
- 毎月数百パターンのAIモデルを自動生成
- BigQueryによる大規模データ分析
- 機械学習のベース部分の自動化

**成果**：
- 予測精度が大幅向上
- ソフトドリンク・お菓子などの精度が特に向上
- 従業員からの信頼が増加

### 4.3 Phase 3: 全店舗展開（2023年3月～現在）

**展開規模**: 全国21,000店舗

**実施方法**：
- 段階的なロールアウト（地域ごと）
- 研修・サポート体制の充実
- フィードバック収集メカニズムの構築

**現在の状況（2025年1月時点）**：
- AI提案準拠率: **70～75%以上**
- 対象商品拡大: 常温品からソフトドリンク、お菓子へ
- 準拠率が高い地域・カテゴリの欠品率が顕著に低下

## 6. AI準拠率の進化：信頼構築の過程

### 6.1 準拠率の推移

```
スタート当初（2020年1月）
├─ 準拠率: 約55%
├─ 背景: AI提案への懐疑的態度
└─ 课题: 「機械学習が本当に信頼できるか」

中間段階（2021年末）
├─ 準拠率: 約60-65%
├─ 背景: パイロット成果の周知、実感
└─ 変化: ソフトドリンク・お菓子の高精度を確認

Google Cloud移行後（2022年末）
├─ 準拠率: 約70%
├─ 背景: 予測精度の顕著な向上
└─ 達成: 目標値（70%）達成

現在（2025年1月）
├─ 準拠率: 約70-75%（一部地域で75%超）
├─ 背景: 信頼の定着、AIの説明責任向上
└─ 次目標: 商品カテゴリ全体での75%以上達成
```

### 6.2 準拠率向上の要因

1. **成功事例の共有**: 高準拠率の店舗・地域の成功ノウハウの水平展開
2. **説明責任の充実**: 「なぜこの提案か」をAIが説明する機能の追加
3. **地道な改善**: 準拠率が低い商品カテゴリの精度向上に注力
4. **人とAIの協業**: AIの提案に人的判断を挟む仕組みの定着

### 6.3 準拠率と欠品率の相関

```
準拠率が高い ⟹ AI提案の質が高い ⟹ 在庫が適正 ⟹ 欠品率低下
準拠率が低い ⟹ 独自判断が多い ⟹ 在庫が不規則 ⟹ 欠品率上昇
```

この相関が明確化することで、現場スタッフのAIへの信頼が加速度的に向上しました。

## 7. 技術基盤の進化：Google Cloud移行の影響

### 7.1 移行前後の比較

#### 移行前（オンプレミス/従来クラウド）
- 予測モデル: 月1-2回の手動更新
- データ分析: 数日単位の遅延
- スケーラビリティ: 店舗増加に伴う処理能力の瓶頸
- 開発サイクル: 3-6ヶ月単位の大きなリリース

#### Google Cloud移行後
- 予測モデル: **毎月数百パターンの自動生成・検証**
- データ分析: リアルタイム処理（秒単位）
- スケーラビリティ: 21,000店舗を無制限に処理可能
- 開発サイクル: 週単位の継続的改善

### 7.2 BigQueryの活用

BigQueryの導入により実現した革新：

1. **超高速分析**: 数百万行のデータを秒単位で分析
2. **複雑なクエリ実行**: 複数の条件を組み合わせた予測が可能
3. **機械学習の統合**: BigQuery内で直接ML.CREATEで予測モデル生成
4. **コスト効率化**: サーバーレスのため、使用量に応じた課金

### 7.3 実装例：月次モデル自動更新

```python
# BigQuery SQL（疑似コード）
CREATE OR REPLACE MODEL `seveneleven.ai_forecasting_model`
OPTIONS(
  model_type='linear_reg',
  input_label_cols=['sales']
) AS

  sales,
  past_sales_3m,
  weather_temp,
  is_weekday,
  is_holiday,
  lte_leadtime
FROM `seveneleven.training_data`
WHERE date BETWEEN DATE_SUB(CURRENT_DATE(), INTERVAL 90 DAY) AND CURRENT_DATE()
UNION ALL
SELECT ... (過去3ヶ月分の学習データ)
WHERE date BETWEEN DATE_SUB(CURRENT_DATE(), INTERVAL 90 DAY) AND DATE_SUB(CURRENT_DATE(), INTERVAL 365 DAY);

-- 推論スコアリング
SELECT
  store_id,
  product_id,
  predicted_sales,
  confidence_interval
FROM ML.PREDICT(MODEL `seveneleven.ai_forecasting_model`, (
  SELECT * FROM `seveneleven.scoring_input`
));
```

2. **複数店舗オーナーの利便性向上**:
   - 複数店舗のダッシュボード
   - 統一的なレポート作成

3. **スタッフのデジタル適応**:
   - モバイル世代の自然な操作
   - トレーニングコストの削減

### 8.3 2025年8月の生成AI基盤「AIライブラリー」本格展開

**対応業務範囲**：
- 商品企画

- 在庫管理（発注システムとの統合）

#### 発注システムとの統合期待

- AIライブラリーの複数モデルと連携

## 9. 今後の展望と拡張計画

### 9.1 対象商品の拡大

#### 現在のステータス
- **導入済み**: 常温品、ソフトドリンク、お菓子（精度高）
- **検証中**: 日用品、食品

- **生鮮食品**（おにぎり、からあげ棒、サンドイッチ）

  - 課題: 気温・時間帯による需要変動が急峻

  - 期待効果: フードロス削減

**中期（2025-2026年）**：
- **酒類、タバコ**
  - 課題: 法的規制、年齢確認の手動プロセス
  - 対策: 新しい販売データの収集、規制対応

### 9.2 品揃え最適化への進化

現在: **「発注数」の提案段階**
↓
最終目標: **「店舗に最適な商品品揃え」の提案段階**

#### 実現方法

1. **売上ポテンシャル分析**:
   - 各店舗の立地、客層に基づく最適品揃え
   - 新商品導入時の売上予測

2. **品ぞろえ最適化**:
   - 限られた売場面積の中での品目数決定
   - シーズン変動への自動対応

3. **顧客セグメント分析**:
   - 客層（年代、性別、ライフスタイル）に応じた品揃え
   - 地域特性を反映した商品ポートフォリオ

#### 期待効果
- 売上: +2-5%
- 廃棄率: -30-50%
- 顧客満足度: 向上

3. **継続的学習**:
   - 従業員からのフィードバック → AIモデルに反映
   - Good Practice（優れた判断）の共有

## 11. 課題と解決方法

### 11.1 克服した課題

#### 課題1: AI提案への信頼構築（準拠率55% → 70%）

| 段階 | 課題 | 解決方法 | 結果 |
|------|------|---------|------|
| パイロット | 「機械学習は信じられない」 | 成功事例の可視化 | 準拠率60% |
| 拡大期 | 「カテゴリによって精度がまちまち」 | カテゴリ別モデルの最適化 | 準拠率70% |
| 全社展開 | 「なぜこの提案か説明してほしい」 | UI/説明機能の充実 | 準拠率75%+ |

#### 課題2: 複雑な商品カテゴリの精度向上

- **生鮮食品**: 日中の気温変化、イベント開催などに即座に対応
  - 解決: リアルタイムデータの活用、時間帯別モデル

- **季節商品**: 正月、お盆、クリスマスなど変則的な需要
  - 解決: カレンダー情報の組み込み、過去5年の季節トレンド学習

#### 課題3: リアルタイム需要変動への対応

- **大雨**: 来店客数が激減
- **野球中継**: スナック・飲料の売上急増
- **政府給付金**: 一時的な購買力向上

解決策:
- 外部データ連携: 天気予報、イベント情報、ニュースデータ
- 回帰分析: 過去の「非日常」の販売データから学習
- リアルタイム修正: 当日のPOS実績から即座にモデルを補正

### 11.2 残存課題と対策

#### 残存課題1: 低準拠率地域への対応

- **地域によって準拠率が60-65%の地域も存在**
- **原因**: スタッフの高齢化、経験主義の定着

対策:
- 高準拠率地域（75%+）のベストプラクティス展開
- 若いスタッフへの集中研修
- インセンティブプログラム（準拠率目標の設定）

#### 残存課題2: 新店舗オープン時の初期精度

- **初期段階**: 販売データが1ヶ月しかない
- **結果**: AI精度が低い → 準拠率低下 → スタッフの信頼喪失

対策:
- 類似店舗のデータ転用（「Clone Learning」）
- 最初の1ヶ月は手動設定との併用
- 2ヶ月目以降、段階的にAI信頼度を上げる

### 14.1 セブン-イレブン事例の汎用性

セブン-イレブン・ジャパンのAI発注システム導入は、以下の点で**業界横断的な示唆**を持っています：

#### 1. 段階的導入の重要性
- **パイロット**: リスク最小化、学習獲得
- **拡大**: 安定性確認、スケール検証
- **全社展開**: 運用成熟度の確保

#### 2. 人とAIの協業の実現
- AIの自動化機能 + 人的判断の融合
- 「完全自動化」ではなく「意思決定支援」
- 信頼構築の段階的プロセス（55% → 75%+）

#### 3. クラウド基盤の活用
- スケーラビリティ（21,000店舗への対応）
- 柔軟性（毎月数百モデルの更新）
- 経済性（初期投資と運用コストのバランス）

### 14.2 今後の発展方向

#### 短期（2025年内）
- 生鮮食品カテゴリへの拡大
- タブレット・スマートフォン新システムの安定運用
- AIライブラリーの全社員展開

#### 中期（2025-2026年）
- 品揃え最適化への進化
- 価格最適化AIとの統合
- 顧客体験の向上

#### 長期（2027年以降）
- AI基盤の業界標準化
- 供給チェーン全体の最適化（ベンダー含む）
- 店舗DXの完成（店員負担→ほぼゼロへ）

### 14.3 最終評価

セブン-イレブン・ジャパンのAI発注システムは：

✅ **ビジネス的成功**: 40%の業務効率化、推定¥18-31億円の年間効果
✅ **技術的成功**: Google Cloud基盤で毎月数百モデルの自動更新
✅ **社会的成功**: 40万時間の労働時間削減、廃棄ロス削減
✅ **スケール**: 21,000店舗、全国展開完了
✅ **持続性**: 2025年春タブレット移行、2025年8月AIライブラリー全展開

**結論**: 日本企業による大規模AI活用の代表的成功事例として、他業種・他企業へのロールモデルとなるポテンシャルを持つ。
