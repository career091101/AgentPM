## 2. AI導入サマリー

| 項目 | 内容 |
|------|------|
| AIツール名 | JAL-AI、JAL-AI Report |
| ベンダー | アバナード、富士通（Kozuchi）、ヘッドウォータース、Microsoft（Phi-4） |
| 導入時期 | 2024年1月〜（継続展開） |
| 対象範囲 | 間接部門100%、客室乗務員 |
| 主要技術 | RAG、小規模言語モデル（SLM）、オンデバイスAI |

## 3. 定量的効果

| 指標 | 数値 | 備考 |
|------|------|---------|
| 間接部門利用率 | 実質100% | JAL-AI |
| 従来レポート作成時間 | 1時間以上/件 | 削減対象 |
| マーケティング購買率向上 | 3.0% | AIバーチャル顧客活用 |

## 4. 導入背景・課題

- 2021〜2025年度中期経営計画でのDX戦略推進
- 社内ナレッジの検索・活用効率化
- 客室乗務員のレポート作成負担軽減
- マーケティング施策の高度化

## 5. ソリューション詳細

### JAL-AI（社内向け生成AIツール）
- 2023年4月にワーキンググループ立ち上げ
- 当初RAG精度向上に課題が発生
- アバナードをパートナーに選定し2024年1月再スタート
- 4つのテーマで開発:
  1. 社内ナレッジの検索・活用
  2. 他システムとのAPI連携
  3. 議事録の自動生成
  4. 整備部門向けマニュアル文書検索・活用
- 2024年度には間接部門の実質100%が利用

### JAL-AI Report（客室乗務員向け）
- キーワード・フレーズ入力とチェックボックス選択で引き継ぎレポート生成
- MicrosoftのAzure AI FoundryとPhi-4 SLMを活用
- クラウド接続不要のデバイス上動作が可能
- 富士通がKozuchiでPhi-4をファインチューニング
- ヘッドウォータースがオフライン対応アプリ開発
- 2025年1月末〜3月初頭に環境構築
- リーダークラスCA 4名で実証実験
- 現在、全CA向け展開の第2フェーズ進行中

### AIバーチャル顧客（JALカード）
- NTTデータのLITRON Multi Agent Simulationを活用
- JALカード利用傾向から複数のAIバーチャル顧客を作成
- AIバーチャル顧客同士で会話・グループディスカッション
- マーケティング施策に有効な示唆を抽出
- 従来ターゲットより購買率3.0%向上

## 7. 成功要因分析

| 要因 | 詳細 |
|------|------|
| パートナー変更の決断 | 課題発生時にアバナードへ変更 |
| オンデバイスAI | 機内のオフライン環境に対応 |
| SLM活用 | Phi-4による軽量・高性能の両立 |
| マルチパートナー | Microsoft、富士通、ヘッドウォータースの協業 |

## 8. 課題・対策

| 課題 | 対策 |
|------|------|
| RAG精度 | パートナー変更で解決 |
| 機内オフライン | SLMによるオンデバイス動作 |
| CA業務負担 | レポート自動生成で1時間/件削減 |

## 9. 今後の展開

- JAL-AI Reportの全客室乗務員展開
- 整備部門向けAI機能の強化
- AIバーチャル顧客のマーケティング活用拡大
- オンデバイスAI技術の他業務への適用
