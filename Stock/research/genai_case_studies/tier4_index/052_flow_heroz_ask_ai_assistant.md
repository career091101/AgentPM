## 2. AI導入サマリー

| 項目 | 内容 |

| AIツール名 | HEROZ ASK |

| 対象範囲 | 法人向けAIアシスタントSaaS |
| 主要技術 | Azure OpenAI Service、Claude Opus4.5対応 |

## 3. 定量的効果

| 指標 | 数値 | 備考 |
|------|------|------|
| 導入企業数 | 100社以上 | 2024年11月時点 |
| 対応業界 | 建設業界、製造業界他 | 大手から中小企業まで |
| 対応言語モデル | Claude Opus4.5 | 2025年12月対応発表 |

## 4. 導入背景・課題

- 企業におけるセキュリティを担保した生成AI活用ニーズの高まり
- PoCにとどまらない生成AIの事業化加速の必要性
- 社内データの探索・要約・翻訳等の業務効率化ニーズ
- 会話データの学習リスクへの懸念

## 5. ソリューション詳細

### HEROZ ASK の主要機能
- 社内データの探索・要約・翻訳
- 音声の言語化（議事録作成等）
- Azure OpenAI Service リファレンスアーキテクチャ準拠
- 会話・やりとりは全て学習されない設定

### 2025年新機能
- **Meeting Assist**（2025年秋リリース予定）: AIエージェント統合による会議・商談効率化
- **JOINT iPaaS連携**（2025年4月）: ストラテジット社との連携アプリ作成支援

## 6. 導入プロセス

| フェーズ | 時期 | 内容 |
|---------|------|------|
| クローズドβ版 | 2023年9月 | 無料提供開始 |

## 7. 成功要因分析

| 要因 | 詳細 |
|------|------|
| セキュリティ設計 | Azure OpenAI Service準拠、学習データ利用なし |
| ブランドコンセプト | 「とりあえずASKに聞いてみよう」 |
| 業界横断展開 | 建設・製造等、幅広い業界への対応 |
| 継続的進化 | 最新LLMへの迅速な対応（Claude Opus4.5等） |

## 8. 課題・対策

| 課題 | 対策 |
|------|------|
| 企業固有データ活用 | RAG機能の強化 |
| 業務フロー自動化 | AIエージェント機能の追加（Meeting Assist） |
| 他システム連携 | iPaaS連携による拡張性確保 |

## 9. 今後の展開

- AIエージェント統合による業務フロー自動化の推進
- Meeting Assist機能による会議効率化（2025年秋）
- 外部サービスとの連携拡大（JOINT iPaaS等）
- 最新LLMへの継続的対応

## 10. 他社への示唆

1. **セキュリティ最優先設計**: エンタープライズ向けAIでは学習データ利用なしの設計が必須
2. **シンプルなコンセプト**: 「とりあえず聞いてみよう」という低ハードルなUX設計
3. **継続的進化**: 最新モデルへの迅速対応で競争力維持
