## 4. 導入背景・課題

### 4.1 警察業務の現状課題

警視庁は、以下の重大課題に直面していました：

- **サイバー犯罪の急増**による捜査リソースの逼迫
- **膨大なログデータ**の手作業分析による時間消費
- **手書き調書・証拠資料**のデジタル化コスト
- **映像証拠の効率的活用**（ドライブレコーダー、防犯カメラ等）
- **DX推進の遅れ**（警察組織の構造的課題）

### 4.2 具体的な業務課題

#### 課題1：サイバー犯罪捜査の時間コスト

- **通信ログ分析**: 1件の事件で数千～数百万行のログを手作業確認
- **容疑者特定**: 不正アクセス痕跡から容疑者を見つけるまで数週間
- **人員不足**: サイバー犯罪専門官の育成が追いつかない

#### 課題2：従来型捜査資料の管理

- **調書作成**: 供述記録を手書き → 手入力でデジタル化（1件5～10時間）
- **データ検索**: キーワード検索に手作業が必要
- **情報共有**: 関連事件との照合に時間がかかる

#### 課題3：映像証拠の活用

- **ドライブレコーダー映像**: 長時間映像から容疑者・車を抽出するのに数日
- **防犯カメラ映像**: 複数カメラの映像を人手で追跡
- **手がかり見落とし**: 膨大な映像データから有用な証拠を逃す可能性

#### 課題4：DXツール導入の遅れ

- 予算制約による最先端技術導入の困難
- 警察組織の保守的文化（新技術導入抵抗感）
- セキュリティ規制による制限（公務用システム）

### 4.3 既存施策の限界

- RPA導入は事務作業（給与計算など）に限定
- AIの活用は試験的段階で組織的導入なし
- **捜査の高度化・知的作業支援が不足**

## 5. ソリューション詳細

### 5.1 Geminiを選定した理由

| 特性 | 警視庁での活用意義 |
|------|----------------------|
| **高度な画像認識能力** | 映像から人物・車の特徴抽出、顔認識補助 |
| **テキスト処理精度** | 調書、供述記録の自動分類・検索対応 |
| **ログデータ分析** | サーバーログから異常検知パターン抽出 |
| **日本語対応** | 日本語供述記録、報告書の完全対応 |
| **拡張性** | カスタマイズで警察固有業務へ対応可能 |

### 5.2 4つの主要活用領域

#### 領域1：ドライブレコーダー映像解析

```
入力: 交通事故・当て逃げのドライブレコーダー映像
処理: 映像フレーム抽出 → 人物・車の検出 → 特徴情報抽出
出力: 容疑者顔画像、逃げ車のナンバープレート、走行ルート
削減時間: 従来8時間 → 1時間（87.5%削減）
活用例: 2025年11月の当て逃げ事件で容疑者特定を3日短縮
```

#### 領域2：サーバーログファイル分析

```
入力: 被害企業のサーバーセキュリティログ（100万行）
処理: ログ自動分類 → 不正アクセス痕跡抽出 → 攻撃ルート再現
出力: 疑わしいIPアドレス、アクセスタイムライン、侵害箇所
削減時間: 従来40時間 → 16時間（60%削減）
活用例: 不正送金に至るまでの16時間の侵害経路を自動追跡
```

#### 領域3：手書き資料デジタル化・分類

```
入力: 供述調書、捜査報告書等の手書き文書（画像化）
処理: OCR自動認識 → テキスト正規化 → 内容分類
出力: デジタルテキスト、キーワードインデックス、関連文書リンク
削減時間: 従来10時間 → 1時間（90%削減）
活用例: 過去事件との関連文言を自動抽出（容疑者リンク発見）
```

#### 領域4：防犯カメラ映像の容疑者追跡

```
入力: 駅・商業施設の複数防犯カメラ映像
処理: 映像フレーム同期 → 人物検出 → 時系列追跡
出力: 容疑者の移動ルート、 立ち寄り施設、同行者情報
削減時間: 従来20時間 → 4時間（80%削減）
活用例: 盗難事件で容疑者を施設内12ヶ所から自動追跡
```

### 5.3 導入形態

- **アクセス**: Google Cloud 専用テナント（警視庁オンプレミス接続）
- **セキュリティ**: 特定の捜査データのみ、秘匿情報はマスキング
- **対象部門**: サイバーセキュリティ対策本部、刑事部門（試行）
- **ユーザー**: 捜査官、情報技術部門職員（約50名）
- **トレーニング**: 「サイバーセキュリティ有識者研修」での説明・実行デモ

## 8. 課題・対策

### 8.1 直面する課題

#### 課題1：捜査への AI出力の法的証拠性

**現象**: AI分析結果の証拠としての採用可能性が明確でない

**対策案**:
- 検察庁との協議で証拠採用可能性を検証
- 最終的な容疑者特定は人間の判断を優先
- 「AI分析補助ツール」としての位置づけ明確化

#### 課題2：秘匿情報・個人情報保護

**現象**: サーバーログに被害者情報、捜査情報が含まれる

**対策案**:
- クラウド送信前の自動マスキング機能実装
- Google Cloudとの契約で情報管理を厳格化
- 月1回のセキュリティ監査、ログ監視

#### 課題3：出力品質のばらつき

**現象**: 映像解析の誤認識、ログ分析でのノイズ検出

**対策案**:
- 警察データでのファインチューニング検討
- 複数AIツールの比較検証（Gemini vs 他社）
- 人間による最終判断体制の徹底

#### 課題4：操作官のリテラシー格差

**現象**: AI活用スキルのある捜査官とない人の格差

**対策案**:
- 段階的な操作研修（基礎→実務実習）
- メンター制度、経験者によるOJT
- 定期的なスキルアップセミナー

### 8.2 対策の実装状況

| 課題 | 対策 | 実装状況 | 予定時期 |
|------|------|---------|----------|
| 法的証拠性 | 検察庁協議 | 進行中 | 2026年3月末 |
| 個人情報保護 | 自動マスキング機能 | 計画中 | 2026年2月中旬 |
| 出力品質 | ファインチューニング検討 | 検討段階 | 2026年4月 |
| リテラシー格差 | 研修プログラム | 開始済み | 継続中 |

## 10. 他機関への示唆

### 10.1 公共・自治体への教訓

#### ✅ 適用可能なポイント

1. **セキュリティと効率化の両立**
   - 個人情報保護を前提にAI導入設計
   - 警視庁の「マスキング義務化」は全自治体で応用可能

2. **試行的導入による信頼構築**
   - 実務課題で実証実験 → 成功事例の積み重ね
   - 他県警、自治体で即応用可能

3. **高度なAI活用の可能性証明**
   - 映像解析、ログ分析は公共業務に汎用性高い
   - 医療（放射線画像解析）、社会福祉、教育等へ展開可能

4. **組織文化への適応**
   - 保守的な公務員組織でも新技術導入は可能
   - 専門家会議での認知、段階的導入がカギ

#### ⚠️ 注意すべきポイント

1. **法的枠組みの明確化必須**
   - AI出力の法的証拠性、採用可能性を事前検証
   - 顧問弁護士の関与が不可欠

2. **秘匿情報・個人情報のリスク**
   - クラウド利用による漏洩リスク（=信頼失墜）
   - データマスキング、監査の徹底が必須

3. **職員教育の過小評価**
   - 技術導入だけでなく、捜査官育成に同等投資が必要
   - AI活用スキル教育は組織的カリキュラム化が不可欠
