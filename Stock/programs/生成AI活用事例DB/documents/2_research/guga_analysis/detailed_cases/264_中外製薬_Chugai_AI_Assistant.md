# 事例調査:中外製薬(製薬)

## 1. 基本情報

- **企業名**: 中外製薬株式会社
- **業界**: 製薬
- **従業員数**: 約5,000名以上(Chugai AI Assistant利用者数)
- **AI導入開始時期**: 2020年頃からITインフラ基盤整備開始、2024年5月にChugai AI Assistant全社リリース
- **主要AI製品/サービス**: Chugai AI Assistant、MALEXA(抗体AI)、AI創薬プラットフォーム
- **対象業務領域**: 全社業務効率化、創薬研究開発(基礎研究〜承認)

## 2. AI導入サマリー

中外製薬は2024年5月に全社向け生成AIチャットボット「Chugai AI Assistant」をリリース。社員5,000人以上が利用し、そのうち1,000〜1,500人が毎日利用している。

Amazon Bedrock Claude 3、Azure OpenAI GPT-4/GPT-4o、Google Gemini Pro、医療分野特化MedLMなど複数の大規模言語モデルを自由に選択可能。RAG(検索拡張生成)により社内文書のナレッジを活用した回答生成を実現。

創薬分野では、AIのポテンシャルとして基礎研究から承認までの期間を13年から9年へと4年短縮、費用を約1,200億円から約560億円へと640億円削減、成功確率を0.004%から0.04%へと10倍に高められると試算。

2025年1月には、ソフトバンク、SB Intuitionsと3社で生成AIを活用した臨床開発業務の革新に向けた共同研究の基本合意を締結した。

## 3. 定量的効果

### 3.1 利用状況

- **利用者数**: 社員5,000人以上
- **毎日利用者数**: 1,000〜1,500人
- **生成AIアイディア収集**: 約900件(2024年初頭〜8月)
- **PoC/本番開発**: PoC中・済9件、本番開発中・実装済14件(2024年8月時点)

### 3.2 創薬AI期待効果(試算)

- **開発期間短縮**: 13年 → 9年(4年短縮)
- **開発費削減**: 約1,200億円 → 約560億円(640億円削減)
- **成功確率向上**: 0.004% → 0.04%(10倍)

### 3.3 その他効果

- **MALEXA(抗体AI)**: 抗体のアミノ酸配列予測を高精度化
- **RPA累計削減時間**: 23万時間(組織風土改革の一環)

### 3.4 開発サイクル

- **機能リリース頻度**: 週次ペースで新機能リリース

## 4. 導入背景・課題

### 4.1 ビジネス課題

1. **創薬期間の長期化**: 基礎研究から承認まで平均13年
2. **開発コストの高騰**: 1つの医薬品に約1,200億円
3. **低い成功確率**: 0.004%という極めて低い成功率
4. **臨床開発業務の負担**: コスト・期間の両面で最もインパクトが大きい

### 4.2 業務課題

- 膨大な科学論文・データからの知見抽出の困難性
- 臨床試験に必要な多数の文書作成の煩雑さ
- 疾患情報・業界規制・社内手順などの情報収集の非効率性

### 4.3 組織課題

- 「石橋をたたいて渡らない」組織文化からの脱却
- デジタル起点での意思決定への転換
- AI活用に対する社員の心理的ハードルの克服

## 5. ソリューション詳細

### 5.1 Chugai AI Assistant

**技術構成**:
- **基盤**: AWSとともに開発した独自生成AIアプリ
- **LLM選択肢**:
  - Amazon Bedrock Claude 3
  - Azure OpenAI GPT-4
  - Azure OpenAI GPT-4o
  - Google Gemini Pro
  - MedLM(医療分野特化)
- **RAG(検索拡張生成)**: 社内文書のナレッジを検索して回答生成

**主要機能**:
- 複数LLMの自由選択
- 社内文書検索(RAG)
- 業務に役立つ社内文書へのアクセス
- 週次ペースでの機能追加

**開発体制**:
- AWS連携
- 全社公開後も週次ペースで機能リリース継続
- New Relic(オブザーバビリティプラットフォーム)を採用しユーザー体験向上

**今後の展望**:
- 自律的に問題解決に取り組むエージェント的機能の実装
- 生成AI以外のAI機能も統合したスーパーアプリへの発展

### 5.2 創薬AI

**MALEXA**:
- 抗体のアミノ酸配列を予測するAI
- 2020年頃から開発

**Preferred Networks(PFN)との連携**:
- 包括的なパートナーシップ契約締結
- 最先端の深層学習技術と中外製薬の知見・技術・データを融合

**FRONTEOとの連携**:
- 創薬支援AIシステムのライセンス契約
- 自然言語解析AI「Concept Encoder」を用いた論文探索AIシステム「Amanogawa」
- 疾病メカニズム可視化「Cascade Eye」

### 5.3 ソフトバンク・SB Intuitionsとの共同研究(2025年1月)

**目的**: 臨床開発業務の革新、新薬開発のスピードアップ

**開発内容**:
- 臨床開発業務を自律的に実行できるAIエージェント
- 製薬産業に特化した大規模言語モデル(LLM)

**段階的導入計画**:
1. **第1段階**: 治験文書の自動生成、疾患情報・業界規制・社内手順の情報収集、探索的データ解析等の特定タスクへのAIエージェントプロトタイプ開発
2. **第2段階**: 対象タスクを臨床開発業務全体に拡大、複数AIエージェントが連携するマルチエージェントシステムへ発展

**各社の役割**:
- **中外製薬**: 臨床開発特化LLMの学習用データ提供、AIエージェントの企画・評価
- **ソフトバンク**: AI計算基盤提供、臨床開発特化LLM・AIエージェントの社会実装推進
- **SB Intuitions**: 臨床開発特化LLM・AIエージェント研究開発

2. **パイロット版リリース**: 一部部門での試行

3. **継続的機能追加**: 週次ペースでの機能リリース
4. **アイディア収集**: 約900件の生成AIアイディアを収集(2024年8月時点)
5. **PoC・本番開発**: 9件がPoC中・済、14件が本番開発中・実装済(2024年8月時点)

## 8. 課題・対策

### 8.1 組織文化の変革

**課題**: 「石橋をたたいて渡らない」保守的文化

**対策**:
- RPA等のデジタル施策で小さな成功体験を積み重ね
- 経営層からのメッセージ発信
- 生成AI CoE設立によるボトムアップのアイディア収集

**結果**: 約900件のアイディアが集まる活発な文化へ

### 8.2 複数LLMの使い分け

**課題**: どのLLMをどのタスクに使うべきか判断困難

**対策**:
- 複数LLMを並列提供し、ユーザーが自由に試せる環境
- 社内ベストプラクティスの共有
- 週次ペースでの機能改善

### 8.3 ユーザー体験の継続的向上

**課題**: 利用者のニーズが多様

**対策**:
- New Relic(オブザーバビリティプラットフォーム)導入
- ユーザーフィードバックの可視化
- 週次ペースでの機能追加

### 8.4 臨床開発業務のAI化

**課題**: 臨床試験は規制が厳しく、AI化が困難

**対策**:
- ソフトバンク・SB Intuitionsとの共同研究
- 段階的導入(特定タスク → 全体 → マルチエージェント)
- 製薬産業特化LLMの開発
