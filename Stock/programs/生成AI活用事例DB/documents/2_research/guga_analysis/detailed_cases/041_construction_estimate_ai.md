### 効果の詳細

- **見積作成時間87%削減**: 従来は建築仕様書をベースに、複数の単価表を参照しながら手作業で見積をまとめていたが、生成AIが建築仕様から材料・工事項目を自動抽出し、直近の単価データベースから自動計算することで、大幅な時間短縮を実現

- **見積精度6.8pp向上**: AI生成の初期見積精度は86%だったが、学習用に実装される過去1年の確定見積データを学習させることで、94%まで改善。人間の修正がほぼ不要に

- **営業提案作成時間67%削減**: 見積完成 → 提案資料作成のプロセスを、AIが「見積データ + 顧客情報 + 過去提案事例」から自動でドラフト作成、営業が内容確認・調整のみで対応

- **受注成功率6pp向上**: 迅速な見積対応（3日 → 1日）により、顧客の意思決定ウィンドウ内での提案が可能に。競合他社との比較時間を減らせ、提案品質向上で成約率向上

- **見積修正率78%削減**: 初期見積の精度向上に伴い、顧客からの修正依頼が激減。営業社員の修正対応時間から解放

- **顧客満足度7点向上**: 「見積回答が早い」「提案内容が充実している」というフィードバック

## 4. 導入背景・課題

### 導入前の課題

1. **見積作成プロセスの属人性と時間浪費**
   - 建築仕様書（PDF、Word、時には手書き）から必要情報を手作業で抽出
   - 複数の単価表（紙・Excel）を参照しながら計算
   - チェックリストに基づいた確認作業が多く、各項目で誤漏がないか確認に時間を要する
   - ベテラン営業社員でも5時間以上の時間をかけている
   - 新人社員は10時間以上かかる場合も

2. **見積精度のばらつき**
   - 営業社員によって見積精度が異なる（85% ～ 91%）
   - 外部の特殊工事項目（基礎工事、防水工事など）の原価計算誤りが多発
   - 顧客との契約後に大幅な修正が発生 → コスト増加
   - 過度な見積値の場合、受注機会喪失
   - 過低見積の場合、利益率が低下する悪循環

3. **営業提案品質の不安定性**
   - 見積完成後、提案資料をゼロから作成する営業が多い
   - 過去の類似案件のテンプレート・提案資料を活用していない営業も存在
   - 提案資料のクオリティが、営業スキルに大きく依存
   - 顧客側が「競合他社との比較」を短期間で実施したい場合に対応できない

4. **顧客対応リードタイムの長さ**
   - 見積依頼から見積提示まで平均3日（内勤務日2日半）
   - 他の建設企業は同日対応をしている場合も多く、競争力が低下
   - 急ぎの案件では対応できず、受注機会を喪失

5. **営業リソースの限定性**
   - 営業社員の見積作成業務に年間3,000時間以上を費やしている（全営業社員合計）
   - 本来の営業活動（顧客開拓、商談対応）に割く時間が減少
   - 見積修正対応で、すぐに営業社員が呼び出される状況

6. **データ活用の未成熟さ**
   - 過去見積データが営業社員個別のファイルやメールに分散
   - 過去の類似案件の参照が困難
   - 原価分析や利益管理が、見積時点では不十分

## 5. ソリューション詳細

| 項目 | 内容 |
|------|------|
| **プロダクト名** | AI見積作成システム「EstiMate-Genius」（仮） |
| **プロバイダー** | 日本対応の生成AIサービスプロバイダー |
| **主要機能** | 建築仕様自動抽出、工事項目自動展開、単価自動計算、見積レイアウト自動生成 |
| **技術スタック** | 大規模言語モデル（LLM）、光学文字認識（OCR）、データベース連携 |
| **カスタマイズ** | 建設企業AF の単価表・見積フォーマットに合わせたプロンプト・テンプレート設定 |

### 技術的特徴

#### 1. **マルチフォーマット建築仕様書の自動解析**
   - **入力フォーマット対応**: PDF、Word、画像（写真・スキャン）、手書きメモ
   - **自動抽出項目**: 建物用途、延床面積、階数、工事範囲、特殊要求仕様
   - **精度**: 95% の抽出成功率（1回目の自動処理で適切な項目抽出）

#### 2. **工事項目の自動展開**
   - **ロジック**: LLMが「建築仕様 + 過去見積データ」から、必要な工事項目を自動推定
   - **出力**: 「基礎工事 → 杭打ち、土留め、打設...」のように階層的に展開
   - **カスタマイズ**: 建設企業AF独自の「施工フロー」「工事区分」に対応

#### 3. **動的原価計算エンジン**
   - **単価データベース**:
     - 社内の過去実績（3年分、1,500件以上の確定見積）
     - 外部の建築統計（日本建設情報総合センター、建設物価情報など）
     - リアルタイム資材価格（鋼材・セメント・木材など）
   - **計算ロジック**:
     - 基本単価 × 施工条件係数（高層度、地域補正、季節補正）
     - 規模効果係数（大規模案件で5～10%割引など）

#### 4. **提案資料の自動生成**
   - **入力**: 完成した見積 + 顧客情報（過去案件、担当者、予算感度）
   - **出力**: PowerPoint形式の提案資料（スライド10～15枚）
     - タイトル、施工実績、工事スケジュール、安全管理体制、見積概要
   - **質**: 過去の優秀営業サンプルをベースに学習したテンプレート

#### 5. **継続学習メカニズム**
   - **フィードバックループ**: 確定見積（実績値）を自動的に学習データに追加
   - **月次学習**: 新たに30～50件の確定見積を学習に組み込み
   - **精度向上**: 初期 86% → 3ヶ月後 94% → 6ヶ月後 96%の見込み

### 導入形態

- **SaaS型**: クラウドベースのシステム（セキュリティはエンタープライズグレード対応）
- **オンプレミス連携**: 見積システム、CRM、単価データベースとのAPI連携
- **初期構築**: 2025年6月〜7月（2ヶ月の導入期間）
- **本格運用**: 2025年8月より

## 10. 他社への示唆

### 1. **ホワイトカラー業務の生成AI活用**

建設企業の営業見積作成は「ナレッジ依存」「時間浪費」「属人性高い」の典型的なホワイトカラー業務である。

- **示唆**: 生成AIは、こうしたドキュメント生成・データ処理業務で最大の効果を発揮
- **適用業種**: 金融（ローン審査書類生成）、保険（査定レポート生成）、コンサル（提案資料生成）

### 2. **見積精度向上の仕組み**

本事例では、初期精度 86% → 最終 94% への改善の主な要因は「モデル改善」よりも「データ品質向上」だった。

- **示唆**: AI導入前に、既存データ（過去見積）の品質監査・整備に時間をかけることが重要
- **実例**: 3年分の過去見積1,500件を整理・タグ付け、単価表の最新化

### 3. **営業プロセス改善の順序**

従来の営業プロセス改善は「営業スキル研修」「営業ツール導入」という順序だったが、本事例では「業務自動化」が先行した。

- **示唆**: 属人的な定型業務は、人材育成より自動化を優先すべき
- **効果**: 見積作成5時間30分 → 45分で「営業の時間・脳力」を高度な営業活動に配分可能

### 4. **生成AI vs 従来型AI**

従来の工事見積システムは「決定木」「線形回帰」など統計的モデルであり、複雑な建築仕様の解釈に弱かった。

- **示唆**: テキスト・自然言語処理が必要なタスクは、生成AI（LLM）が適切
- **例**: 仕様書解析、提案文章生成、顧客対応メール自動作成

### 5. **組織の抵抗感への対処**

「AIが営業スキル低下を招く」という懸念に対して、本事例では「AIは提案サポート、見積責任は営業」という明確な位置づけをした。

- **示唆**: 技術導入時に「人間の役割・責任」を再定義することが重要
- **実例**: 営業社員を「見積作成者」から「提案コンサルタント」へ職務転換

### 6. **競争優位性の確保**

本事例の成果（見積対応を3日 → 1日に短縮）は、営業現場で直接的な競争優位となった。

- **示唆**: AI導入効果は、内部効率化（コスト削減）だけでなく、顧客体験向上を含めて設計すべき
- **実例**: 「見積対応が早い」を新規営業の訴求ポイントに
