# Customer Problem Fit (CPF)

> **出典**: 田所雅之「起業の科学」STEP 2
> **関連概念**: [[CONCEPT_IDEA_001]], [[CONCEPT_PSF_001]]

---

## 1. 定義

**Customer Problem Fit (CPF)** とは、「顧客が本当に解決したいと思っている課題」を特定できた状態。

スタートアップ最大の失敗原因「市場にニーズがなかった」(42%) を防ぐための最初のステップ。

**CPFの核心**:
- **Who（誰が）**: ターゲット顧客を具体的に特定
- **What（何を）**: 顧客が抱える課題を明確化
- **Why（なぜ）**: その課題を解決したい理由

## 2. なぜ重要か

CPFを達成せずにプロダクト開発を始めると:
- ❌ 誰も欲しがらないものを完璧に作ってしまう
- ❌ 時間とリソースを無駄にする
- ❌ ピボットが遅れて資金が尽きる

**CPF達成の効果**:
- ✅ 顧客が本当に解決したい課題に集中できる
- ✅ プロダクト開発の方向性が明確になる
- ✅ 初期ユーザー獲得が容易になる

## 3. 構成要素

### 3.1 3U検証

課題が解決する価値があるかを判定する3つの基準：

| 要素 | 説明 | 質問例 |
|-----|------|-------|
| **Unworkable (切実)** | 課題が解決されないと困る | 「今この課題を解決しないと何が起こるか?」 |
| **Unavoidable (不可避)** | 必ず直面する課題である | 「ターゲットの何%がこの課題に直面するか?」 |
| **Urgent (今すぐ)** | すぐに解決したい | 「今日解決したいか、来年でいいか?」 |

**+1: Underserved (代替なし)**
既存の解決策がない/不十分

### 3.2 ペルソナ定義

具体的なターゲット顧客像:

```
┌─────────────────────────────────────────────────────────┐
│ ペルソナ名: [名前]                                      │
├──────────────────────┬──────────────────────────────────┤
│ 年齢・性別           │ [例: 35歳男性]                   │
├──────────────────────┼──────────────────────────────────┤
│ 職業・役職           │ [例: IT企業マネージャー]          │
├──────────────────────┼──────────────────────────────────┤
│ 日常                 │ [1日の行動パターン]              │
├──────────────────────┼──────────────────────────────────┤
│ 課題・ペイン         │ [最も困っていること]             │
├──────────────────────┼──────────────────────────────────┤
│ ゴール               │ [達成したいこと]                 │
├──────────────────────┼──────────────────────────────────┤
│ 現在の解決策         │ [今使っているツール・方法]        │
├──────────────────────┼──────────────────────────────────┤
│ 不満点               │ [既存解決策への不満]             │
└──────────────────────┴──────────────────────────────────┘
```

## 4. 判定基準・指標

### 4.1 定量指標

| 指標 | CPF達成基準 | 測定方法 |
|------|-----------|----------|
| **インタビュー実施数** | 20人以上 | 実施したインタビュー数 |
| **課題共通率** | 70%以上 | 同じ課題を挙げた人の割合 |
| **支払い意思** | 50%以上 | 「お金を払ってでも解決したい」と答えた割合 |
| **緊急性スコア** | 平均7/10以上 | 「どれくらい急いで解決したいか」10段階評価 |

### 4.2 定性指標

- [ ] 20人以上の顧客インタビューを実施した
- [ ] 70%以上の顧客が同じ課題を挙げた
- [ ] 顧客が「お金を払ってでも解決したい」と言っている
- [ ] 課題の切実さ（Urgency）が確認できた
- [ ] 既存代替品とその問題点が明確になった
- [ ] ペルソナが具体的に定義できた

## 5. 実践ステップ

### ステップ1: 課題仮説の立案

1. **ターゲット顧客の仮説**: 誰が困っているか?
2. **課題の仮説**: 何に困っているか?
3. **既存代替品**: 今どうやって解決しているか?
4. **既存代替品の問題点**: なぜ不満があるか?

### ステップ2: 顧客インタビュー

**良い質問の例**:

| カテゴリ | 質問例 |
|---------|-------|
| 現状把握 | 「〇〇について、普段どうされていますか?」 |
| 課題発見 | 「一番困っていることは何ですか?」 |
| 解決意欲 | 「もしこの課題が解決されたら、いくら払いますか?」 |
| 既存代替 | 「今はどうやって解決していますか?」 |
| 頻度確認 | 「この課題にどのくらいの頻度で直面しますか?」 |

**悪い質問（避けるべき）**:
- ❌ 「こういうプロダクトがあったら使いますか?」（誘導的）
- ❌ 「いくらなら買いますか?」（架空の状況）
- ❌ 「便利だと思いますか?」（抽象的）

**インタビュー数の目安**:
- 最低: 15-20人
- 理想: 30人以上
- パターンが見えるまで: 同じ回答が70%以上

### ステップ3: インサイト分析

1. インタビュー内容を整理
2. 共通パターンを抽出
3. 3U検証を実施
4. ペルソナを定義

### ステップ4: CPF達成判定

上記の定量・定性指標をすべて満たすことを確認。

## 6. よくある失敗パターン

| パターン | 症状 | 対策 |
|---------|------|------|
| **誘導的質問** | 「こういうのあったら使いますか?」と聞く | 現状の行動を聞く（過去形で） |
| **少なすぎるサンプル** | 5人だけインタビューして確信 | 最低20人、できれば30人以上 |
| **仮説に固執** | 最初の課題仮説に縛られる | インタビューで見つかった課題を優先 |
| **友人だけに聞く** | 身内の意見だけで判断 | ターゲット顧客（赤の他人）に聞く |
| **架空の話** | 「もしあったら」を聞く | 「今どうしているか」を聞く |

## 7. 事例

### 7.1 成功事例

- **Airbnb**:
  - ターゲット: イベント時の宿泊難民
  - 課題: ホテルが満室で泊まれない
  - 3U検証: Unsolved（ホテルがない）、Urgent（イベント当日）、Expensive（高い）
  - インサイト: ホスト側も収入欲しい
  - 参照: [[CASE_AIRBNB_001]]

- **Zoom**:
  - ターゲット: リモートワーカー、企業
  - 課題: ビデオ会議が不安定・使いにくい
  - 3U検証: Unsolved（既存ツールが不満）、Urgent（日常的に必要）、Expensive（時間コスト大）
  - 参照: [[CASE_ZOOM_001]]

### 7.2 失敗事例

- **Quibi**:
  - 仮説: 短編動画コンテンツへのニーズ
  - 実態: 顧客インタビュー不足、課題が存在しなかった
  - 結果: $1.75B調達するも6ヶ月で終了
  - 参照: [[CASE_QUIBI_001]]
