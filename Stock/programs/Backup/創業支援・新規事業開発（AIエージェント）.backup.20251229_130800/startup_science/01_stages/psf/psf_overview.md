# Problem Solution Fit (PSF)

> **出典**: 田所雅之「起業の科学」STEP 3
> **関連概念**: [[CONCEPT_CPF_001]], [[CONCEPT_PMF_001]]

---

## 1. 定義

**Problem Solution Fit (PSF)** とは、「顧客の課題に対するソリューションが、既存の代替案と比較して圧倒的に（10倍）優れている」ことを検証した状態。

CPFで「解決すべき課題」を特定した後、PSFで「どう解決するか」を検証する。

**PSFの核心**:
- **What（何を）**: 既存代替案と比較して10倍優れたソリューション
- **How（どう）**: MVPで実際に顧客に使ってもらい、反応を検証
- **Why（なぜ）**: UVP（独自の価値提案）が顧客に刺さるか

## 2. なぜ重要か

PSFを達成せずにプロダクト開発を本格化すると:
- ❌ 「既存ツールで十分」と言われる
- ❌ 使ってもらえても継続されない
- ❌ 10倍の価値がないため、スイッチングコストを超えられない

**PSF達成の効果**:
- ✅ 顧客が既存ツールから乗り換える理由ができる
- ✅ MVP段階で収益化の目処が立つ
- ✅ PMF達成への道筋が明確になる

## 3. 構成要素

### 3.1 10倍検証

課題解決の「圧倒的優位性」を検証するフレームワーク。

| 比較軸 | 説明 | 10倍の例 |
|-------|------|---------|
| **時間効率** | タスク完了時間 | 10時間 → 1時間 |
| **コスト** | 金銭的負担 | $1000/月 → $100/月 |
| **使いやすさ** | 学習コスト | 1週間 → 5分 |
| **成果・効果** | 得られる結果の質 | 成功率10% → 100% |
| **導入障壁** | 始めるまでのハードル | 契約書10枚 → ワンクリック |

**判定基準**:
- 最低2軸で10倍以上の改善
- または1軸で100倍以上の改善

**注意**: 定量的に測定困難な場合、定性的な圧倒的優位性（体験、感情）でもOK

### 3.2 MVP（Minimum Viable Product）

**MVPの目的**: 最小の労力で「PSFを検証」すること

| MVP類型 | 説明 | 例 |
|---------|------|-----|
| **Landing Page** | 説明ページ+メール登録 | Dropbox |
| **Concierge** | 手動でサービス提供 | Food on the Table |
| **Wizard of Oz** | 自動化のフリ | Zappos |
| **Prototype** | 動くモックアップ | Instagram（Burbn） |
| **Single Feature** | 1機能のみ実装 | Twitter（140文字投稿のみ） |
| **Piecemeal** | 既存ツール組み合わせ | Groupon（WordPressブログ） |
| **Video** | デモ動画 | Dropbox |
| **Crowdfunding** | 事前注文 | Pebble Watch |
| **Email** | メール1本 | Tim Ferriss「4時間」 |
| **Pre-order** | 先行販売 | Tesla Model 3 |

**選定基準**:
- 1-4週間で作成可能
- 顧客の反応が測定できる
- コストが最小限

### 3.3 UVP（Unique Value Proposition）

**UVPの定義**: 「なぜ既存代替案ではなく、このプロダクトを選ぶべきか」を1文で表現したもの

**UVPキャンバス5要素**:

| 要素 | 説明 | 質問 |
|------|------|------|
| **ターゲット** | 誰のための製品か | 最も刺さる顧客セグメントは？ |
| **課題** | 何を解決するか | 最も深刻な課題は？ |
| **ソリューション** | どう解決するか | 独自の解決方法は？ |
| **便益** | 何が得られるか | 顧客が得る具体的価値は？ |
| **差別化** | なぜ他ではダメか | 競合にはない独自性は？ |

**テンプレート**:
```
[ターゲット]が[課題]を解決するために、
[ソリューション]を使うと[便益]が得られます。
なぜなら[差別化要因]だからです。
```

## 4. 判定基準・指標

### 4.1 定量指標

| 指標 | 目標値 | 測定方法 |
|------|--------|----------|
| **10倍達成軸数** | 2軸以上 | 5軸（時間/コスト/使いやすさ/成果/導入障壁）で比較 |
| **MVP反応率** | CVR 5%以上 | LP→登録、テスト→継続使用、等 |
| **UVP刺さり度** | 8/10以上 | 顧客インタビューでの評価 |

### 4.2 定性指標

- [ ] 顧客が「これは欲しい」と即座に反応する
- [ ] 既存代替案への不満を完全に解消している
- [ ] 10倍の価値を1文で説明できる
- [ ] MVPでも継続使用したいと言われる
- [ ] 価格設定に対して「高すぎる」と言われない

## 5. 実践ステップ

### STEP 1: 競合・代替案の特定（2時間）

1. **直接競合**: 同じ課題を同じ方法で解決
2. **間接競合**: 同じ課題を別の方法で解決
3. **既存の対処法**: ツールを使わずに解決

各項目について:
- URL、主な特徴、価格帯、規模を記録
- 顧客が実際に何を使っているかヒアリング

### STEP 2: 10倍検証の実施（4時間）

5軸（時間/コスト/使いやすさ/成果/導入障壁）で比較:

| 軸 | 既存最良 | 当ソリューション | 改善倍率 |
|----|---------|----------------|:--------:|
| 時間 | [X時間] | [Y時間] | [Z倍] |
| ... | ... | ... | ... |

**判定**:
- 2軸以上で10倍達成 → PSF候補
- 1軸のみ → ソリューション強化
- 0軸 → 根本的見直し

### STEP 3: MVP選定・作成（1-4週間）

1. **MVP類型の選定**: 上記10類型から最適なものを選択
2. **作成**: 1-4週間で完成
3. **テスト**: 最低10人の顧客に実際に使ってもらう

**完了条件**:
- 動くプロトタイプが存在
- 顧客が実際に操作できる
- 反応（CVR、継続率等）が測定できる

### STEP 4: UVP明確化（2時間）

UVPキャンバスを埋める:
1. ターゲット: ペルソナから最も刺さるセグメントを選定
2. 課題: CPF検証結果から最重要課題を選定
3. ソリューション: 10倍達成した軸を中心に表現
4. 便益: 顧客が得る具体的価値を明記
5. 差別化: 競合にはない独自要素を3つ

**検証**: 顧客に1文で説明し、反応を確認

### STEP 5: PSF判定（1時間）

| 項目 | 達成 | 判定 |
|------|:----:|:----:|
| 10倍達成（2軸以上） | ✅/❌ | - |
| MVP反応率 5%+ | ✅/❌ | - |
| UVP刺さり度 8/10+ | ✅/❌ | - |

**総合判定**:
- 3つすべて✅ → PSF達成
- 2つ✅ → 条件付き達成（未達成項目を改善）
- 1つ以下 → PSF未達成（ソリューション見直し or ピボット）

## 6. よくある失敗パターン

| パターン | 症状 | 対策 |
|---------|------|------|
| **機能追加で10倍を目指す** | 複雑化して使いにくい | 1軸に集中し、シンプルに |
| **MVPが完璧すぎる** | 時間とコストがかかりすぎ | 最低限の機能で検証 |
| **UVPが曖昧** | 「便利そう」止まり | 1文で差別化を明確に |
| **競合分析が浅い** | 「競合なし」と誤認 | 顧客が実際に何を使っているか調査 |
| **10倍が自己満足** | 顧客には響かない | 顧客視点での価値を検証 |

## 7. 事例

### 7.1 成功事例

- **Dropbox**:
  - 10倍: ファイル同期の手間が1/10（手動コピー → 自動同期）
  - MVP: デモ動画（1日で作成）
  - UVP: 「どのデバイスからでも同じファイルにアクセス」

- **Airbnb**:
  - 10倍: 宿泊費が1/10（ホテル$200 → Airbnb$20）
  - MVP: 自宅を貸し出す簡易サイト
  - UVP: 「ローカルの家に泊まる体験」

- **Uber**:
  - 10倍: 配車時間が1/10（タクシー会社電話 → アプリタップ）
  - MVP: SMS配車サービス
  - UVP: 「3分でタクシーが来る」

### 7.2 失敗事例

- **Juicero**:
  - 10倍主張: ジューサー不要（実際は手で絞れる）
  - 問題: 顧客が10倍価値を感じなかった
  - 結果: $120M調達も1.5年で破綻

- **Quibi**:
  - MVP: フル機能アプリ（$1.8B投資）
  - 問題: MVPが大きすぎて検証できず
  - 結果: ローンチ6ヶ月で破綻
