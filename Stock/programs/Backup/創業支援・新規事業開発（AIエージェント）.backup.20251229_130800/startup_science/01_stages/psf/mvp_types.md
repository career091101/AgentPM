# MVP 10 Types（MVP 10類型）

> **出典**: 田所雅之「起業の科学」STEP 3、Eric Ries「The Lean Startup」
> **関連**: [[CONCEPT_PSF_001]], [[FRAMEWORK_LEAN_001]]

---

## 1. 定義

**MVP (Minimum Viable Product)** とは、「最小の労力で仮説検証できる製品」。

**MVPの目的**:
- ❌ 完璧な製品を作ること
- ✅ 最速で学習すること
- ✅ PSFを検証すること

**Eric Riesの教え**:
> 「MVPは顧客から最大の学びを得るために必要最小限の機能を備えた製品バージョン」

## 2. MVP選定の基準

| 基準 | 説明 | 目安 |
|------|------|------|
| **作成期間** | どれだけ早く作れるか | 1-4週間 |
| **コスト** | 開発・運用費用 | $0-$5,000 |
| **学習量** | どれだけ学べるか | 顧客の反応が測定可能 |
| **リスク** | 失敗時のダメージ | 小さいほど良い |

## 3. MVP 10類型

### 3.1 Landing Page（LP型）

| 項目 | 内容 |
|------|------|
| **説明** | サービス説明ページ+メール登録フォーム |
| **作成期間** | 1-3日 |
| **コスト** | $0-$100 |
| **測定指標** | CVR（訪問→登録）、登録者数 |
| **適用場面** | 需要があるか確認したい |

**手順**:
1. サービスの価値提案をLPで説明
2. 「事前登録」ボタンを設置
3. 広告でトラフィックを流す
4. CVRを測定（目標5%以上）

**成功事例**:
- **Dropbox**: デモ動画+LP で1日で75,000人登録
- **Buffer**: LPのみで初期100人獲得

---

### 3.2 Concierge（コンシェルジュ型）

| 項目 | 内容 |
|------|------|
| **説明** | 完全手動でサービスを提供 |
| **作成期間** | 即日 |
| **コスト** | $0（人件費のみ） |
| **測定指標** | 継続率、満足度、WTP |
| **適用場面** | サービスの価値を確認したい |

**手順**:
1. 顧客にサービス価値を説明
2. 初回は完全手動で提供
3. フィードバックを収集
4. 自動化の優先順位を決定

**成功事例**:
- **Food on the Table**: 創業者が毎週手動でレシピ・買い物リストを作成
- **Zappos**: 最初は靴を買いに行って発送

---

### 3.3 Wizard of Oz（オズの魔法使い型）

| 項目 | 内容 |
|------|------|
| **説明** | 自動化されたフリをして手動で動かす |
| **作成期間** | 3-7日 |
| **コスト** | $0-$500 |
| **測定指標** | 継続率、自動化の必要性 |
| **適用場面** | 自動化前に需要を確認したい |

**手順**:
1. 自動化されたUIを作成
2. 裏側は人間が手動で処理
3. 顧客の使い方を観察
4. 自動化の優先順位を決定

**成功事例**:
- **Zappos**: オンラインストアのフリ、実際は店舗で購入して発送
- **Aardvark**: AI検索のフリ、実際は人間が回答

---

### 3.4 Prototype（プロトタイプ型）

| 項目 | 内容 |
|------|------|
| **説明** | 動くモックアップ |
| **作成期間** | 1-2週間 |
| **コスト** | $0-$1,000 |
| **測定指標** | 使用時間、タスク完了率 |
| **適用場面** | UXを検証したい |

**手順**:
1. Figma/Sketch等でプロトタイプ作成
2. ユーザビリティテスト（5-10人）
3. 改善点を特定
4. 本開発前にUXを最適化

**成功事例**:
- **Instagram**: Burbnから「写真投稿」機能のみ抽出してプロトタイプ
- **Twitter**: 初期は「140文字投稿」のみの簡易プロトタイプ

---

### 3.5 Single Feature（単一機能型）

| 項目 | 内容 |
|------|------|
| **説明** | 1機能のみ実装 |
| **作成期間** | 2-4週間 |
| **コスト** | $500-$5,000 |
| **測定指標** | DAU/MAU、リテンション |
| **適用場面** | コア機能を検証したい |

**手順**:
1. コア機能1つに絞る
2. その機能だけを完璧に作る
3. ユーザーに使ってもらう
4. 継続率を測定

**成功事例**:
- **Twitter**: 初期は「140文字投稿」のみ
- **Facebook**: 初期は「大学生プロフィール」のみ

---

### 3.6 Piecemeal（寄せ集め型）

| 項目 | 内容 |
|------|------|
| **説明** | 既存ツールを組み合わせる |
| **作成期間** | 1-3日 |
| **コスト** | $0-$100 |
| **測定指標** | 需要、使用頻度 |
| **適用場面** | 低コストで検証したい |

**手順**:
1. 既存ツール（WordPress, Airtable等）を組み合わせ
2. 最低限の機能を実現
3. 顧客に使ってもらう
4. 本開発の優先順位を決定

**成功事例**:
- **Groupon**: WordPressブログ+PDF発行
- **Stripe（初期）**: PayPal API+簡易UI

---

### 3.7 Video（動画型）

| 項目 | 内容 |
|------|------|
| **説明** | デモ動画で価値を伝える |
| **作成期間** | 1-3日 |
| **コスト** | $0-$500 |
| **測定指標** | 動画視聴数、登録数 |
| **適用場面** | 複雑な製品を説明したい |

**手順**:
1. 3分以内のデモ動画作成
2. 製品の価値を実演
3. YouTube, SNSで公開
4. 反応を測定

**成功事例**:
- **Dropbox**: デモ動画で1日75,000人登録
- **Dollar Shave Club**: バイラル動画で1日12,000人登録

---

### 3.8 Crowdfunding（クラウドファンディング型）

| 項目 | 内容 |
|------|------|
| **説明** | 事前注文を募る |
| **作成期間** | 1-2週間（ページ作成） |
| **コスト** | $0（手数料5%程度） |
| **測定指標** | 支援額、支援者数 |
| **適用場面** | 製品開発前に需要を確認したい |

**手順**:
1. Kickstarter/Indiegogo等でページ作成
2. 製品コンセプトを説明
3. 目標金額を設定
4. 達成→本開発、未達→ピボット

**成功事例**:
- **Pebble Watch**: $10Mを69,000人から調達
- **Oculus Rift**: $2.4Mを9,500人から調達

---

### 3.9 Email（メール型）

| 項目 | 内容 |
|------|------|
| **説明** | メール1本で価値提供 |
| **作成期間** | 即日 |
| **コスト** | $0 |
| **測定指標** | 開封率、返信率、登録率 |
| **適用場面** | 情報提供型サービスを検証したい |

**手順**:
1. 価値ある情報をメールで提供
2. 継続購読を募る
3. 反応を測定
4. 自動化・プロダクト化

**成功事例**:
- **Tim Ferriss「4-Hour Workweek」**: ブログ+メールで検証後に書籍化
- **Morning Brew**: メールニュースレターから$75M調達

---

### 3.10 Pre-order（事前販売型）

| 項目 | 内容 |
|------|------|
| **説明** | 製品開発前に販売 |
| **作成期間** | 1-7日（LP作成） |
| **コスト** | $0-$100 |
| **測定指標** | 購入率、購入者数 |
| **適用場面** | WTP（支払い意思）を確認したい |

**手順**:
1. 製品説明ページ+購入ボタン
2. 「X日後に配送」と明記
3. 購入率を測定（目標1%以上）
4. 目標達成→本開発開始

**成功事例**:
- **Tesla Model 3**: $1000デポジットで375,000台事前注文
- **iPhone（初代）**: 発売前に予約開始で需要確認

## 4. MVP選定フローチャート

```
質問1: 製品は複雑か？
  YES → Video型
  NO → 質問2へ

質問2: WTP（支払い意思）を確認したいか？
  YES → Pre-order型 or Crowdfunding型
  NO → 質問3へ

質問3: サービス提供が手動可能か？
  YES → Concierge型 or Wizard of Oz型
  NO → 質問4へ

質問4: コア機能1つに絞れるか？
  YES → Single Feature型 or Prototype型
  NO → 質問5へ

質問5: 既存ツールで実現可能か？
  YES → Piecemeal型
  NO → Landing Page型
```

## 5. よくある失敗パターン

| パターン | 症状 | 対策 |
|---------|------|------|
| **MVPが完璧すぎる** | 3ヶ月かけて作る | ✅ 1-4週間で作れるものに |
| **機能を詰め込む** | 学びたいことが曖昧 | ✅ 1つの仮説に集中 |
| **顧客に見せない** | 内部で完成度を上げ続ける | ✅ 早期に顧客に見せる |
| **1種類のみ** | リスク分散できない | ✅ 複数のMVP類型を試す |

## 6. 事例比較表

| 企業 | MVP類型 | 期間 | 成果 |
|------|---------|:----:|------|
| Dropbox | Video + LP | 1日 | 75,000人登録 |
| Airbnb | Concierge + Photo | 1週間 | 初の3人ゲスト獲得 |
| Zappos | Wizard of Oz | 即日 | 需要を確認 |
| Twitter | Single Feature | 2週間 | 継続率70% |
| Groupon | Piecemeal | 1日 | 初回20人が購入 |

## クイックリファレンス

```
選定基準:
- 作成期間: 1-4週間
- コスト: $0-$5,000
- 学習量: 測定可能な指標あり

最速: Email, Concierge（即日）
低コスト: Piecemeal, LP（$0-$100）
高精度: Prototype, Single Feature（2-4週間）

次のステップ: PSF判定（CVR 5%+）
```
