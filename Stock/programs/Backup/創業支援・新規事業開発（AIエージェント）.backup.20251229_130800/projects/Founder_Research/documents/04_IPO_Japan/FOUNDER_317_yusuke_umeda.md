## 2. 創業ストーリー

### 2.1 課題発見（需要発見）

**着想源**:
- 梅田（コンサル）・稲垣（証券）双方が「ビジネス情報収集の非効率」に悩む
- コンサルタント時代:
  - クライアント企業の情報収集に膨大な時間（週10-20時間）
  - 情報が散在（新聞、業界誌、IR資料、ウェブ等）
  - リアルタイム性に欠ける（週刊誌は1週間遅れ）
- 証券アナリスト時代:
  - 企業分析に必要な情報が見つからない
  - Bloomberg は高額（月額数十万円）で中小企業は導入困難
- 「ビジネスパーソンが意思決定に必要な情報を、最速で提供するプラットフォーム」を構想

**需要検証方法**:
- 金融機関・コンサル・事業会社80社にヒアリング
- 「ビジネス情報収集に困っているか？」→ 85%が「困っている」
- 「情報収集に週何時間かけているか？」→ 平均10-15時間
- 「有料サービスを使うか？」→ 75%が「使いたい」

### 2.2 CPF検証（Customer Problem Fit）

**インタビュー/顧客検証**:
- 実施数: 80社以上（金融機関・コンサル・事業会社）
- 手法: 訪問インタビュー、業務観察
- 発見した課題の共通点:
  - **情報収集の非効率**: 「情報が散在」「探すのに時間がかかる」「最新情報が手に入らない」
  - **Bloomberg の高額**: 「月額数十万円は高い」「中小企業には導入困難」
  - **意思決定の遅延**: 「情報収集に時間を取られ、意思決定が遅れる」

**3U検証**:
- Unworkable（現状では解決不可能）:
  - 新聞・業界誌は情報が古い（週刊誌は1週間遅れ）
  - Google 検索は情報が多すぎて整理できない
  - Bloomberg は高額で中小企業は導入困難
- Unavoidable（避けられない）:
  - ビジネスパーソンは意思決定に情報が必須
  - 経営者・投資家・コンサルは常に情報収集が必要
- Urgent（緊急性が高い）:
  - 意思決定のスピードが競争力に直結（中〜高の緊急性）
  - 投資判断は情報の鮮度が命

**支払い意思（WTP）**:
- 確認方法: 有料ベータ版を30社に提供
- 結果:
  - 月額5万円で25社が契約
  - 年間契約率80%（継続意思の確認）
  - 「Bloomberg より安くて使いやすい」との声が多数

### 2.3 PSF検証（Problem Solution Fit）

**10倍優位性**:

| 軸 | 従来の解決策 | 自社ソリューション | 倍率 |
|---|------------|-----------------|------|
| 情報収集時間 | 新聞・業界誌（週10-15時間） | SPEEDA（週1-2時間） | 10倍 |
| 情報の鮮度 | 週刊誌（1週間遅れ） | リアルタイム更新 | 7倍 |
| コスト | Bloomberg（月額数十万円） | SPEEDA（月額5-10万円） | 5倍 |
| 情報の網羅性 | 業界誌（特定業界のみ） | 全業界カバー | ∞倍 |
| 可視化 | テキストのみ | グラフ・チャート自動生成 | 10倍 |

**MVP**:
- タイプ: Prototype（企業情報データベース + 可視化ツール）
- 初期反応:
  - ベータ版30社、25社が有料契約
  - NPS（推奨度）: 70（業界平均30-40）
  - 「これがないと仕事にならない」との声多数
- CVR: 初期12% → 改善後25%（2年後）

**UVP（独自の価値提案）**:
- 「経済情報で、世界を変える」
- ビジネスパーソンが意思決定に必要な情報を最速で提供
- 企業情報・業界動向・M&A・資金調達等を網羅的にカバー
- グラフ・チャート自動生成で可視化

**競合との差別化**:
- Bloomberg との差別化:
  - 圧倒的低価格（月額5-10万円 vs 数十万円）
  - 日本企業・アジア企業に強い
  - UI/UX が日本人向け
- 新聞・業界誌との差別化:
  - リアルタイム更新（新聞は1日遅れ、週刊誌は1週間遅れ）
  - 可視化（グラフ・チャート自動生成）
  - 検索性（キーワード・業界・企業で瞬時に検索）

## 3. ピボット/失敗経験

### 3.1 初期の失敗

- **メディア「NewsPicks」の苦戦（2013-2014年）**:
  - SPEEDA の次の事業として NewsPicks をローンチ
  - 当初、マネタイズできず赤字継続
  - 2年後にフリーミアムモデルで黒字化

- **海外展開の困難（2015年）**:
  - 米国・欧州での SPEEDA 展開を試みるが苦戦
  - 現地の Bloomberg・S&P Capital IQ との競争激化
  - 撤退を決断、アジア特化に方針転換

### 3.2 ピボット（該当する場合）

- ピボットは実施していない
- 一貫して「ビジネス情報プラットフォーム」を追求

## 4. 成長戦略

### 4.1 初期トラクション獲得

- **金融機関・コンサル向け営業（2009-2012年）**:
  - 梅田・稲垣自らが100社以上に飛び込み営業
  - 「情報収集時間を1/10にする」という明確な価値提案

- **口コミ戦略**:
  - 「SPEEDA がないと仕事にならない」という熱狂的ファンを獲得
  - NPS 70を維持し、オーガニックグロース

- **コンテンツマーケティング**:
  - 「業界動向レポート」「M&A事例」等のコンテンツ発信
  - SEO/SEM 集中投資

### 4.2 フライホイール

1. **ビジネス情報の蓄積**
   → 2. **データ分析・可視化機能の改善**
   → 3. **顧客満足度向上（NPS向上）**
   → 4. **口コミで新規顧客獲得**
   → 5. **さらにデータ蓄積**
   → 1に戻る

- **データドリブン改善**:
  - 利用者の検索キーワード・閲覧履歴を分析
  - 需要の高い情報を優先的に拡充

### 4.3 スケール戦略

- **新規事業開発（2013年〜）**:
  - NewsPicks（経済ニュースSNS）
  - INITIAL（スタートアップデータベース）
  - FORCAS（営業支援ツール）

- **M&A戦略（2016年〜）**:
  - Quartz（米国経済メディア）買収
  - アジアパシフィックイニシアティブ買収

- **海外展開（2015年〜）**:
  - アジア特化（中国・東南アジア）
  - 現地企業情報の拡充

### 4.4 バリューチェーン

```
情報ソース（新聞、IR、プレスリリース等）
  → ユーザベース（データ収集・分析・可視化）
  → SPEEDA（顧客に提供）
  → 顧客（意思決定に活用）
```

## 6. 成功要因分析

### 6.1 主要成功要因

1. **「自分たちの課題」から始めたプロダクト**:
   - コンサル・証券出身で、情報収集の課題を深く理解
   - 深い課題理解とUX設計

2. **明確な価値提案**:
   - 「情報収集時間を1/10にする」という定量的な価値
   - Bloomberg より安く、使いやすい

3. **データドリブン改善**:
   - 利用者の行動データを分析し、継続的に改善
   - NPS 70を維持

4. **新規事業開発**:
   - SPEEDA → NewsPicks → INITIAL と事業領域拡大
   - 複数の収益源を確保

5. **グロービスの長期支援**:
   - Series A〜IPOまで伴走
   - 経営ノウハウ・ネットワーク提供

### 6.2 タイミング要因

- **2008年**: リーマンショック → 情報の重要性増大
- **2010年代**: スマホ普及 → モバイルでの情報収集ニーズ増大
- **2015年**: 働き方改革 → 業務効率化ニーズ増大

### 6.3 差別化要因

- **日本・アジア特化**: Bloomberg は欧米中心、ユーザベースは日本・アジアに強い
- **可視化**: グラフ・チャート自動生成で直感的に理解
- **低価格**: Bloomberg の1/5〜1/10の価格
- **複数事業**: SPEEDA、NewsPicks、INITIAL で複数の収益源
