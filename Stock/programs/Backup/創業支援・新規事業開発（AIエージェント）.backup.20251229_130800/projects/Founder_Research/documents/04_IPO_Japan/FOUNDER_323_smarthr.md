## 2. 創業ストーリー

### 2.1 課題発見（需要発見）

**着想源**:
- 2013年、株式会社KUFU（現SmartHR）を共同創業者の内藤研介氏と設立
- 当初は明確な課題が見つからず、2年間で12回ものピボットを経験
- 受託開発で資金を繋ぎながら、複数の自社サービスを開発するもヒットせず
- 「世の中の課題が見つけられない」状態が継続

**転機: Open Network Lab参加（2015年）**:
- デジタルガレージグループ主催のインキュベーションプログラム「Open Network Lab（Onlab）」第10期に参加
- 3ヶ月後のDemoDayで優勝できなければ会社を畳む覚悟で臨む
- プログラム開始から「3日サイクル高速検証」を実践:
  - 1日目: 課題発見
  - 2日目: ソリューション開発
  - 3日目: 5人にヒアリング検証
- この高速サイクルを繰り返す中で、「社会保険・雇用保険の入退社手続き」という課題に到達

**需要検証方法**:
- 社労士事務所・企業人事部へ100社以上ヒアリング
- 「入退社手続きに困っているか？」→ 100%が「困っている」
- 「ペーパーレス化できたら使うか？」→ 95%が「使いたい」
- 定量データ:
  - 入社手続きに要する時間: 平均2-3時間/人
  - 手続きミス率: 30%（差し戻し経験あり）
  - 年間手続き件数: 中小企業で平均10-50件、大企業で数百〜数千件

### 2.2 CPF検証（Customer Problem Fit）

**インタビュー/顧客検証**:
- 実施数: 100社以上（社労士・企業人事部）
- 手法: 訪問インタビュー、オフィス観察、実際の手続き業務同行
- 発見した課題の共通点:
  - **企業人事**: 「手続きが煩雑」「時間がかかる」「ミスが多い」「従業員への説明が面倒」
  - **社労士**: 「顧問先の手続き代行が負担」「紙のやり取りが非効率」「マイナンバー対応が困難」
  - **従業員**: 「何枚も紙に記入するのが面倒」「個人情報を会社に渡すのが不安」

**3U検証**:
- Unworkable（現状では解決不可能）:
  - 手書きの書類記入は時間がかかり、ミスが多発
  - 役所への郵送・持参は非効率
  - 従業員から情報を集めるのが困難（Excelメール添付、紙記入等）
  - マイナンバー管理が複雑で漏洩リスクあり
- Unavoidable（避けられない）:
  - 社会保険・雇用保険の入退社手続きは法的義務（全企業・全従業員）
  - 手続き漏れは罰則・従業員トラブルに直結
  - 日本の法制度が続く限り永続的に発生
- Urgent（緊急性が高い）:
  - 入社日の当日〜5日以内に手続き必須
  - 手続き遅延は従業員の社会保険未加入につながる
  - マイナンバー制度導入（2016年1月）で手続きがさらに複雑化

**支払い意思（WTP）**:
- 確認方法: Open Network Lab期間中にMVP提供、有料ベータ版テスト
- 結果:
  - 初期顧客30社、全社が月額契約（無料トライアル後）
  - 継続率99%以上（現在まで維持）
  - 「これがないと人事業務が回らない」との声多数
  - 社労士からも「顧問先に推奨したい」との反応

### 2.3 PSF検証（Problem Solution Fit）

**10倍優位性**:

| 軸 | 従来の解決策 | SmartHRソリューション | 倍率 |
|---|------------|-----------------|------|
| 作業時間 | 手書き記入（2-3時間/人） | Web入力（5-10分/人） | 20倍 |
| ミス率 | 手書き（ミス率30%） | 自動チェック（ミス率0.5%未満） | 60倍 |
| コスト | 社労士委託（5-10万円/年） | SmartHR（1-5万円/年） | 2-10倍 |
| 従業員負担 | 紙記入・押印・郵送 | スマホ・PCで完結 | 10倍 |
| データ管理 | 紙・Excel分散 | クラウド一元管理・検索可能 | ∞倍 |

**MVP**:
- タイプ: Prototype（Web入力フォーム + e-Gov API連携）
- 初期反応（Open Network Lab DemoDay）:
  - 最優勝賞受賞
  - 複数VCから投資オファー
  - 社労士・企業からの問い合わせ殺到
- CVR: 初期20% → 改善後40%（2年後）
- NPS（推奨度）: 85（業界平均20-30の約3倍）

**UVP（独自の価値提案）**:
- ミッション: 「Well-working（よく働ける）を、全ての人へ」
- コアバリュー:
  1. **完全ペーパーレス化**: 入退社手続きを100%デジタル化
  2. **従業員セルフサービス**: 従業員が自分でWeb/スマホから情報入力
  3. **電子申請対応**: e-Gov API連携で役所への申請も自動化
  4. **ワンストップ**: 社会保険・雇用保険・年金の手続きを一元管理
  5. **マイナンバー管理**: 安全な収集・保管・利用

**競合との差別化**:

1. **vs 社労士（従来型）**:
   - 圧倒的低価格（月額数千円 vs 月額数万円）
   - リアルタイム（社労士は月1回訪問）
   - セルフサービス（待ち時間なし）
   - ただし社労士とは競合せず、提携モデル（後述）

2. **vs 既存人事システム（オービック、PCA、奉行等）**:
   - クラウド型（インストール不要、自動アップデート）
   - 電子申請対応（既存システムは紙前提が多い）
   - 従業員向けUI（既存システムは人事部門専用）
   - モダンなUX/UI（スマホ対応）

3. **vs freee人事労務・マネーフォワードクラウド**:
   - **SmartHR**: 労務管理・タレントマネジメント特化、人事データ活用に強み
   - **freee**: 会計連携重視のオールインワン型、freee会計ユーザー向け
   - **マネーフォワード**: バックオフィス全体統合、経理〜労務の一体化
   - SmartHRの差別化: 労務・人材領域の専門性とマルチプロダクト展開

## 3. ピボット/失敗経験

### 3.1 初期の失敗と12回のピボット

**創業〜2015年（約2年間）**:
- 株式会社KUFU創業後、明確な課題が見つからず苦悩
- 自社サービスを複数開発するもヒットせず、全てクローズ
- 受託開発で何とか資金を繋ぐ日々
- 「世の中の課題を見つけられない」状態が継続

**Open Network Lab参加決断**:
- 2回目のサービスクローズと同時期にOnlabアクセラレータープログラムに参加
- 「3ヶ月後のDemoDayで優勝できなければ会社を畳む」覚悟
- 家族・共同創業者の内藤氏への責任を感じての決断

**3日サイクル高速検証**:
- プログラム開始から徹底した仮説検証を実践
- サイクル:
  1. 1日目: 課題発見（インタビュー、観察）
  2. 2日目: ソリューション開発（MVP作成）
  3. 3日目: 5人にヒアリング検証
- このサイクルを3ヶ月間で約30回繰り返す
- 11回目のピボットで「社会保険手続き」に到達
- 12回目の検証で圧倒的な手応えを確認

**Onlab DemoDay**:
- 最優勝賞受賞
- 複数VCから投資オファー
- 社労士・企業から問い合わせ殺到
- この成功が現在のSmartHRの起点

### 3.2 その後の困難

**1. マイナンバー制度対応（2016年1月）**:
- SmartHRリリース直後にマイナンバー制度が開始
- 法改正に伴う大幅な仕様変更が必要
- エンジニアリソース不足で対応に苦慮
- 結果: エンジニア採用を加速、マイナンバー管理機能を追加開発

**2. e-Gov API連携の壁（2016年）**:
- 政府の電子申請システム（e-Gov）との連携が困難
- API仕様が不明確で、試行錯誤に3ヶ月
- 政府との折衝・技術的ハードル
- 結果: 粘り強い開発で連携成功、電子申請対応が差別化要因に

**3. 創業者の健康問題**:
- 宮田氏は過去に難病「ハント症候群」を患う
- リハビリを経て完治
- この経験が「Well-working」の思想に影響

**4. 2025年: 創業者の退任**:
- 2025年3月26日、宮田氏はSmartHRの取締役を退任
- 新会社「Nstock株式会社」を設立（SmartHRのグループ会社から独立）
- SmartHRは芹澤雅人氏がCEOとして継続
- 宮田氏は10億円を神山まるごと高専に寄付（2025年1月）

### 3.3 ピボット成功の学び

**宮田氏の言葉**:
> 「大切なのは、机上の空論からビジネスを作るのをやめて、実際のユーザーの課題をもとにビジネスを作るということ」

> 「諦めないことが大事。12回ピボットしても諦めなかった」

**重要な洞察**:
1. **課題が明確であること**: 誰が困っているか、なぜ困っているかが明確
2. **課題が大きいこと**: 市場規模が十分（全企業380万社が対象）
3. **ソリューションが適切**: 10倍優位性がある解決策
4. **トラクションがある**: ユーザー数・売上が伸びる兆し

## 4. 成長戦略

### 4.1 初期トラクション獲得（2015-2017年）

**1. Open Network Lab効果**:
- DemoDay最優勝賞でメディア露出
- VC・エンジェル投資家からの注目
- 初期顧客獲得のきっかけ

**2. 社労士パートナー戦略**:
- 社労士事務所との提携プログラム開始
- 社労士が顧問先にSmartHRを推奨
- Win-Winモデル:
  - 社労士: 手続き代行業務が効率化、顧問先満足度向上
  - SmartHR: 信頼できる専門家経由で顧客獲得
- 初期顧客の80%は社労士経由
- 代表的提携: SATO社会保険労務士法人（顧客5,400社）、PSRネットワーク

**3. フリーミアムモデル**:
- 従業員30名以下は完全無料（¥0プラン）
  - 2019年10月に従業員10名→30名に拡大
- 導入障壁を極限まで下げる
- 小規模企業でトライアル → 成長後に有料プラン移行
- 口コミ効果: 「無料で使えるのに高機能」

**4. 口コミ・バイラル成長**:
- NPS 85の圧倒的な顧客満足
- 「入社手続きが5分で終わった！」という体験が拡散
- 継続率99%超で解約がほぼゼロ
- オーガニックグロースが主要チャネル

### 4.2 フライホイール

```
1. 簡単なUI/UXで中小企業獲得
   ↓
2. 高いNPS（85）で口コミ拡散
   ↓
3. 社労士との提携拡大
   ↓
4. データ蓄積で機能改善・自動化向上
   ↓
5. さらに使いやすくなる（UX向上）
   ↓
6. 大企業にも導入拡大
   ↓
1に戻る（加速）
```

**ネットワーク効果**:
- 利用企業増加 → 電子申請の普及促進 → 行政手続きのデジタル化加速
- 社労士提携増加 → 顧問先への導入増加 → 社労士業界でのデファクトスタンダード化
- データ蓄積 → 自動チェック精度向上 → ミス率低下 → さらに信頼性向上

### 4.3 スケール戦略（2018年〜現在）

**1. 大企業進出（2018年〜）**:
- 従業員1,000名以上の大企業向けプラン
- セキュリティ強化（ISO27001取得、プライバシーマーク）
- カスタマイズ対応・専任サポート
- 大企業シェアNo.1達成

**2. マルチプロダクト戦略（2019年〜）**:
- **給与計算** (2025年6月提供開始予定): クラウド人事給与基幹システムへ進化
- **年末調整**: 従業員が自分で入力、自動計算
- **従業員サーベイ**: エンゲージメント測定
- **タレントマネジメント**: 配置シミュレーション、スキル管理、キャリア台帳
  - タレントマネジメント領域でARR約50億円（2025年5月）

## 6. 成功要因分析

### 6.1 主要成功要因

**1. 12回のピボットと諦めない姿勢**:
- 2年間の試行錯誤を経て真の課題に到達
- Open Network Labでの3日サイクル高速検証
- 「諦めない」ことが最大の成功要因

**2. 完璧な3U（Unworkable, Unavoidable, Urgent）**:
- 現状では解決不可能な非効率性
- 全企業に避けられない法的義務
- 入社日から5日以内という緊急性
- この3要素が揃った稀有な市場

**3. タイミングの良さ（法制度変更）**:
- 2016年1月: マイナンバー制度導入 → 手続き複雑化
- 2020年: 電子申請義務化（大企業） → 追い風
- 2020年: コロナ禍 → リモートワーク普及、クラウド化加速
- 2021年〜: DX推進・人的資本経営 → バックオフィスDXニーズ急増

- 社労士を競合と見なさず、パートナーとして連携
- 初期顧客の80%は社労士経由
- Win-Winの関係構築が成長の鍵

**5. フリーミアムによる導入障壁ゼロ化**:
- 従業員30名以下は完全無料
- 小規模企業でトライアル → 成長後に有料化
- 70,000社という圧倒的な登録企業数を実現

**6. トップティアVCの支援**:
- Sequoia、KKR等、グローバルトップティアVCから支援

- 豊富な資金調達（累計358億円）でアグレッシブな成長投資

**7. 圧倒的な10倍優位性**:
- 作業時間20倍、ミス率60倍の改善
- 競合が追いつけないレベルの差別化

**8. マルチプロダクト戦略**:
- 労務管理から人的資本経営プラットフォームへ進化
- 顧客あたり売上（ARPU）を継続的に拡大

## 8. orchestrate-phase1への示唆

### 8.1 需要発見（/discover-demand）

**SmartHRの手法**:
- **100社以上のインタビュー実施**:
  - 社労士事務所・企業人事部に直接ヒアリング
  - 実際の業務に同行、観察
  - 課題の共通性95%を確認

- **市場規模の定量化**:
  - 日本の企業数: 約380万社
  - 年間入退社者数: 約1,000万人
  - TAM（総市場規模）: 労務管理市場 数千億円
  - 人的資本経営まで含めると兆円規模

- **3日サイクル高速検証**:
  - 1日目: 課題発見
  - 2日目: ソリューション開発
  - 3日目: 5人検証
  - このサイクルを30回繰り返し

**orchestrate-phase1への適用**:
- `/discover-demand` 実行時、3日サイクル検証を推奨
- 最低50-100社のインタビューを実施
- 課題共通性90%以上を目標

### 8.2 CPF検証（/validate-cpf）

**SmartHRの手法**:
- **完璧な3U検証**:
  - Unworkable: 手書きは時間とミス多発
  - Unavoidable: 法的義務で全企業必須
  - Urgent: 入社日から5日以内
  - 3Uが全て揃った稀有な事例

- **支払い意思の事前確認**:
  - MVP提供後、全ての初期顧客が有料契約
  - 継続率99%超で解約ほぼゼロ
  - 「これがないと業務が回らない」レベルの必須性

**orchestrate-phase1への適用**:
- `/validate-cpf` 実行時、3U全てを厳密にチェック
- 1つでも欠けている場合は再検討
- 支払い意思は「使いたい」ではなく「実際に契約」で確認

### 8.3 PSF検証（/validate-10x）

**SmartHRの手法**:
- **10倍優位性の明確化**:
  - 作業時間20倍、ミス率60倍、コスト2-10倍の改善
  - 複数軸で10倍以上を達成
  - 単一軸ではなく、多軸での優位性

- **MVP検証**:
  - Prototype型MVP（実際に動くWeb入力 + 電子申請）
  - Open Network Lab DemoDay最優勝
  - CVR 20%でローンチ、後に40%まで改善

**orchestrate-phase1への適用**:
- `/validate-10x` 実行時、最低3軸で10倍優位性を確認
- MVP型は Prototype 推奨（特にB2B SaaS）
- CVR 15-20%以上をPMF基準に設定
