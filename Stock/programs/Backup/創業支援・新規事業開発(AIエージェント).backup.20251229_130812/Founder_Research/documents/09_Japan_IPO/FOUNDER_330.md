# FOUNDER_330: 寺田親弘 - Sansan

## 1. 創業の経緯・課題認識

### 原体験・課題との出会い

寺田親弘（てらだ ちかひろ）は1976年12月29日生まれ。父は在英日本大使館や国際企業での要職を歴任した寺田縉太郎。このエリート家系の中で育ち、慶應義塾高等学校を経て、慶應義塾大学環境情報学部を卒業した。

1999年4月に三井物産株式会社に入社。情報産業部門に配属され、ビジネスの本質を学ぶ。特に2001年5月の米国シリコンバレーでの勤務（Mitsui Comtek社）を通じて、ベンチャー企業の急速な成長と日本とのギャップを目の当たりにした。

三井物産時代（2002年8月帰任後）、寺田は社内ベンチャーとしてデータベースソフトウェア事業部門を立ち上げたが、組織の大きさや経営意思決定の遅さに直面。さらに2006年2月の三井物産セキュアディレクション出向時代に、企業間の「人と人のつながり」の価値が圧倒的に見過ごされていることに気付いた。

**課題認識**: ビジネスシーンにおいて、営業活動や新規事業開発の際に、「誰が誰と関係しているか」「どの企業のどの部門がどのような課題を抱えているか」といった情報が紙の名刺のままで、デジタル化・活用されていない。この「出会いの価値」を最大化する仕組みが存在しない。

### 課題の定義

- **問題1: 名刺情報の散在化** - 営業員が持つ数千枚の名刺が個人レベルで管理され、企業資産として活用されていない
- **問題2: 機会損失** - 同じ顧客に複数部門からアプローチしたり、重複営業が発生。逆に他部門が解決できるニーズに気付かない
- **問題3: データ化の困難性** - 手書き名刺や外国語名刺の正確なデータ化に膨大な時間・コストがかかる
- **問題4: セキュリティと管理** - 個人管理の名刺は紛失や漏洩リスクが高い

### 創業チーム

- **寺田親弘 (CEO/創業者)** : 三井物産出身、シリコンバレー経験者、ビジネスプラットフォーム構想者
- **富岡氏 (共同創業者・事業部長)** : 技術・事業推進の専門家

## 2. ソリューション・事業内容

### プロダクト/サービス概要

**Sansan**: 企業向けのクラウドベースの営業・出会い管理プラットフォーム

**コアサービス**:
1. **名刺デジタル化** - スキャナやスマートフォンカメラで撮影した名刺を、独自のOCRと人手を組み合わせて正確にデータ化
2. **出会い管理プラットフォーム** - デジタル化された名刺情報を企業全体で共有・活用
3. **営業支援機能** - AI推薦による提案対象企業の抽出、過去接触履歴の可視化
4. **ビジネスインテリジェンス** - データ分析による顧客セグメンテーション、営業施策の最適化

**付随するサービス**:
- **Eight** - 個人向けの名刺管理アプリ（法人向けSansanの個人版）
- **Sansan Data Intelligence** (2025年11月新サービス) - 企業が保有する顧客データの名寄せ、正規化、補完、AI活用前提のデータ基盤構築を支援

### 差別化ポイント

1. **独自のOCR技術** - 手書き・外国語対応の精度の高いOCRにより、正確なデータ化を実現
2. **企業向けプラットフォーム** - 個人向けではなく法人組織全体での活用を前提設計
3. **出会いという視点** - 単なる名刺管理ではなく、「人と人のつながり」から生まれるビジネス機会を最大化する思想
4. **データベース資産化** - 日本のビジネスシーン における「人とつながり」のマスターデータを保有
5. **AI・機械学習の継続的投資** - スマートレコメンデーション、営業支援機能の高度化

## 3. 市場環境・競合分析

### 市場規模

- **日本のCRM/営業支援市場**: 年間数千億円規模
- **名刺管理市場**: 機能単体では小規模だが、営業DX全体では大規模成長市場
- **デジタル化率**: 日本企業の営業プロセスのデジタル化率は欧米に比べ低く、未開拓市場

### 競合状況

**直接競合**:
- **Eight** (自社提供・個人向け) - 無料で個人ユーザーに広がるも、法人向けではないため異なる市場セグメント
- **LinkedIn Sales Navigator** (海外大手) - グローバルネットワークは強いが、日本語・日本企業データには限定あり
- **Sky（株）SkyPCE** - 営業支援ツールとしての競合
- **インターパーク サスケLead** - 営業DX系ツール

**間接競合**:
- **Salesforce** - CRM/SFA大手。SansanはSalesforceとの連携を強化し、補完関係を構築
- **kintone** - ノーコードCRM
- その他営業支援ツール（HubSpot等）

**市場シェア**: Sansan は国内シェア82%の圧倒的領域を獲得

### 参入障壁

1. **OCR精度** - 名刺デジタル化の核となる技術で、高い参入障壁
2. **データベース資産** - 数年かけて蓄積した日本企業のビジネスデータが競争優位性
3. **大企業向けセールスエンジニアリング** - フォーチュン500企業への営業組織構築に時間・コスト
4. **組織的導入の難しさ** - 営業プロセス改革を伴うため、単なるツール導入では不十分

## 4. 成長プロセス

### 初期段階 (2007-2012)

2007年6月に三井物産を退職後、わずか1ヶ月で会社設立（三三株式会社→後にSansan）。初期段階は法人営業に絞り、大手企業への導入を目指した。2007-2012年は赤字続きながら、着実に顧客数を増加させた。

### 成長初期 (2012-2015)

- 企業向けSansanの累積導入社数が増加
- 個人向けサービスEightの立ち上げ (※Eightは後にLINE傘下へ一部出資を受ける可能性も検討)
- 顧客基盤の拡大により営業組織の強化

### スケール期 (2015-2019 IPO前)

- **2015年前後** : 年間売上50億円レベルに到達
- **2016-2018年** : 大型資金調達（Series D/E）による事業拡大

  - Pre-IPO企業価値は100億円を超え、ユニコーン企業に
- **2019年6月** : 東証マザーズ上場、上場時時価総額142.5億円

## 6. 技術・イノベーション

### コア技術

**1. OCR (光学文字認識)**:
- 独自開発したOCRエンジンで、手書き名刺、外国語名刺、印字不鮮明な名刺の判読
- AWS SageMaker ProcessingとSageMaker Trainingを活用した機械学習パイプライン
- 継続的な改善サイクル: ユーザーのフィードバック → 学習データ増加 → モデル精度向上

**2. 機械学習基盤**:
- **スマートレコメンデーション**: ユーザーの過去接触履歴、企業属性から、最適な提案対象企業を提示
- **名寄せ・正規化**: 複数の同一企業・顧客情報を正確にマッピング
- **ビジネスインテリジェンス**: 業界セグメント、規模別の営業成果分析

**3. データベース・データ基盤**:
- 正確にデータ化された「出会いのデータ」を核とした、日本最大のビジネス人脈データベース
- 2024年新サービス「Sansan Data Intelligence」で、外部企業の顧客データ補完・正規化に対応
- AWS、GCP、Kubernetes等で高可用性を実現

**4. クラウドアーキテクチャ**:
- AWS (Batch, Lambda, SageMaker) + GCP (AppEngine, Kubernetes) のハイブリッド
- マイクロサービスアーキテクチャによるスケーラビリティ
- 2025年「Data Hub リアーキテクチャ」により、ドメイン分離と標準化を推進

### 技術的優位性

1. **日本企業データへの特化** - 日本の企業構造、部門、役職体系の深い理解
2. **OCR精度** - 数年の学習データ蓄積と継続改善による業界トップレベルの精度
3. **API連携** - Salesforce、Slack等との連携により、既存ワークフローに統合
4. **リアルタイム更新** - スマートフォン撮影から数分以内にデータベース反映

### R&D方針

- **継続的AI投資** : スマートレコメンデーション、自動営業分析等の高度化
- **業界別ソリューション** : 製造、金融、サービス業等、業界ニーズに合わせたカスタマイズ
- **国際展開への基盤構築** : 海外市場への進出を見据えた多言語OCR開発
- **MCP (Model Context Protocol) の活用** : Claude等の最新LLMをデータベース接続で活用

## 8. 課題と解決アプローチ

### 直面した主な課題

**課題1: IPO前の赤字期間の長期化**
- **背景**: 2007-2012年（約5年間）、高い開発投資と営業体制構築により営業赤字継続
- **解決策**:
  - 投資家（特にIncubate Fundの赤浦氏）による継続的な信任と資金支援
  - 長期ビジョン（「日本のビジネス人脈のプラットフォーム化」）への確信
  - 段階的な収益化モデルの構築（SaaS型に移行）

**課題2: 大企業向けセールスエンジニアリングの難しさ**
- **背景**: 初期段階では、フォーチュン500企業への導入に時間がかかった。組織的な営業導入には抵抗が大きい
- **解決策**:
  - 大手企業での導入実績を積み重ねることで信用構築
  - 業界別ソリューション開発による課題解決型営業
  - 営業組織の強化（2015年以降、営業チーム拡大）

**課題3: 技術力との両立**
- **背景**: OCR精度向上には継続的な機械学習投資が必要だが、営業成長と並行させる必要
- **解決策**:
  - R&D部門の独立・強化
  - 大学との研究連携（NII情報学研究データリポジトリへのSansanデータセット寄与）
  - 採用強化（2024年現在、研究員・エンジニア募集強化）

**課題4: 国際展開への障壁**
- **背景**: 日本固有のビジネスモデル（人脈の重要性、営業文化）が海外で通用するか不確実
- **解決策**:
  - 国内市場の成熟度を高める（82%シェア達成）
  - 多言語OCRの開発
  - 将来的な海外展開の基盤整備

### 現在の課題 (2024年以降)

1. **飽和市場への対応** : 国内名刺管理市場では82%シェアを獲得したため、新たな成長軸の構築が必要
2. **データ利活用の拡大** : Sansan Data Intelligenceなど、データベース資産の有効活用
3. **AI時代への対応** : 生成AI、LLMと連携したスマート営業支援機能の高度化

## 9. 学び・洞察

### 成功要因

1. **明確で大きなビジョン** : 単なる「名刺管理ツール」ではなく、「出会いからイノベーション」という社会的インパクトを掲げた
2. **実務経験の活用** : 三井物産での企業営業経験が、営業組織の課題理解につながった
3. **投資家との信頼関係** : 赤浦氏（Incubate Fund）との創業前からのつながりが、長期的な支援を可能にした
4. **テクノロジーへの執着** : OCR精度の継続改善により、競争優位性を維持
5. **忍耐強い経営** : 赤字期間5年を乗り切ったレジリエンス
6. **データベース資産化** : 単なるツール企業ではなく、日本最大のビジネス人脈データベース保有企業へ

### 重要な意思決定

1. **企業向けに特化** : 個人向けではなく、法人向けに絞ることで、高価格・高契約価値の事業モデルを実現
2. **OCR自社開発** : 外部ツール利用ではなく、データ化の品質向上に投資
3. **IPO実施** : Pre-IPO段階のユニコーン企業価値（100億円以上）から、IPOで企業価値142.5億円へ
4. **Data Intelligenceの新サービス化** : 内部向けのデータ化機能を、外部顧客向けサービスに転換

### 失敗・ピボット

- **大きな失敗は報告されていない** が、Eightの位置づけについては、個人向けビジネスの独立や別ブランド化など、戦略の見直しがあった可能性
