# プロジェクト憲章
**プロジェクト名**: SNS

## 1. 背景と目的

### 背景

複数のSNS（Facebook, Instagram, X, LinkedIn, YOUTRUST, Threads）への投稿管理が煩雑化しており、運用コストが増大している。

**現状実績（2026年1月、11日間実測値）**:
- **月間総imp**: 30-50万 imp（目標100万の30-50%、-50-70万 imp不足）
- **プラットフォーム別貢献度**（11日間実績、括弧内は月間換算）:
  - Facebook: 223,804 imp（約61万 imp/月、57.6%貢献）
  - LinkedIn: 87,150 imp（約24万 imp/月、22.5%貢献）
  - X: 77,290 imp（約21万 imp/月、19.9%貢献）
  - Threads: 14,170 imp（計測中、Late API対応）
- **投稿頻度**: 約435件/月（LinkedIn 27, X 318, Threads 63, Facebook 27）
- **運用効率**: 修正工数不明（AIエージェント生成→Late API整形→予約投稿フローで修正点が発生）
  - 自動化構成: Claude Code Skill（sns-automation、52-77分/実行）
  - 課題: 修正点（手直し）が多く、完全自動化に至っていない

**参考: 過去実績（2025年10月-12月、90日間）**:
- 月間総imp: 346,766 imp
- Facebook: 172,261 imp/月（49.7%）、LinkedIn: 159,264 imp/月（45.9%）、X: 15,242 imp/月（4.4%）
- 運用効率: 47.5分/投稿

**市場トレンド（社内ナレッジより）**:
- **Build in Public**: 透明性戦略でコミュニティ売上30%達成事例
- **PLG (Product-Led Growth)**: 製品がマーケティングツール化、SaaS月間MRR $32K成功事例
- **属人性経済**: Instagram事例で、属人性あり（月300-500万円）vs なし（月5-10万円）= **50-100倍の収益差**
- **プラットフォーム特化**: 各SNSアルゴリズム最適化が必須。横展開では各媒体のポテンシャル1/3しか発揮できない

**課題の構造（詳細は別紙レポート参照）**:
1. **コンテンツ品質**: フック不足（Top/Bottom 60倍差）、情報解像度不足、プラットフォーム最適化欠如
2. **戦略・方向性**: トピック偏り（OpenAI 55.8%）、属人性欠如、収益化導線不在
3. **プロセス・体制**: 再現性欠如、KPI管理不備、投稿時間帯集中（深夜0時100%）

### 目的

**フェーズ1（実名アカウント×生成AI）**:
クロスポストやAI活用によるコンテンツ生成フローを構築し、投稿品質とエンゲージメント率を最大化しながら、月間100万インプレッションを達成する。

**フェーズ2構想（匠名アカウント展開）**:
フェーズ1で確立した自動化ノウハウを活用し、恋愛・キャリア・副業などの収益性の高いジャンルで匠名アカウントを展開。インプレッション拡大と商品購入誘導により収益化を実現する。

**改善目標（フェーズ1: 2026年6月30日まで）**:
1. **総インプレッション**: 30-50万 → **100万 imp/月**（+100-233%成長）
2. **エンゲージメント率**: 0.63% → **1.5%**（+138%改善）← 最優先改善領域
3. **自動化フロー精度**: 修正工数不明 → **修正作業ゼロ化**（完全自動化）
4. **プラットフォーム最適化**:
   - LinkedIn ER: 0.66% → **2.50%**（高野氏ベンチマーク接近）
   - X 平均imp: 661回 → **2,000回**（目標達成レベル）
   - Facebook 閲覧数維持: 月間61万 imp（現状達成水準を維持）
5. **再現性向上**: Top/Bottom投稿のimp差 → **10倍以内**（品質バラツキ抑制）

**フェーズ2展開予定（2026年9月以降）**:
- タイミング: フェーズ1で月間100万imp達成後、数ヶ月安定運用してから
- 対象ジャンル: 恋愛・マッチング、キャリア・転職、副業・起業
- 運用形態: 匠名アカウント（ジャンル特化）
- マネタイズ: note商品、コンサル、有料コミュニティ、アフィリエイト

## 2. スコープ

### 含む

- **フェーズ1（実名アカウント×生成AI）の目標達成施策**:
  1. **エンゲージメント率向上**（最優先）: 投稿品質改善、高野式7パターン適用、問いかけ終結
  2. **自動化フロー改善**: AIエージェント生成精度向上、修正点の洗い出しと解消
  3. **LinkedIn品質改善**: ER 0.66% → 2.50%、問いかけ終結率向上、高野氏7パターン適用
  4. **X パフォーマンス改善**: 平均imp 661回 → 2,000回、投稿数削減と品質重視
  5. **Facebook 閲覧数維持**: 月間61万 imp水準の維持、インタラクション向上
  6. **運用効率化**: 自動化フロー修正点ゼロ化、完全自動化の実現
  7. **再現性確保**: トップ/ボトム分析、チェックリスト導入、PDCA構築
- **修正点の洗い出しと改善計画策定**（今後の課題として明記）

### 含まない

- **フェーズ2（匠名アカウント展開）の詳細計画**（2026年9月以降の将来フェーズ）
- **収益化機能の実装**（フェーズ1は認知・エンゲージメント獲得が主眼）
- 新規プラットフォームへの展開（Instagram, TikTok等）
- 投稿スケジュールの手動調整（Late API自動予約投稿を使用）
- 各個別のSNSへの手動返信（AIアシストは検討するが、全自動化は範囲外）
- 有料広告運用の最適化
- 長尺動画コンテンツの制作（テキスト・画像メイン、短尺Reelsのみ含む）
- インプレッション分析ダッシュボードの導入（既存の週次レポートで代替）
- **個人ブランド確立の全体戦略**（属人性向上は重要だが、本PJ期間内での完全確立は困難）

## 5. リスクと対策

| # | リスク | 影響 | 発生確率 | 対策 |
|---|--------|------|---------|------|
| 1 | **エンゲージメント率改善が進まない** | 100万imp目標未達 | 中 | - 高野氏7パターンテンプレート適用<br>- 問いかけ終結を習慣化（チェックリスト導入）<br>- 週次A/Bテスト実施<br>- 投稿品質重視（X投稿数削減等） |
| 2 | **自動化フロー修正点が解消されない** | 手作業継続、運用効率低下 | 高 | - 修正点の洗い出し（課題調査を最優先実施）<br>- AIエージェント生成精度向上<br>- LateAPI整形ロジック改善<br>- プラットフォーム別最適化強化 |
| 3 | **X パフォーマンス改善の長期化** | X平均imp 661回のまま停滞 | 高 | - 投稿数削減（318件/月 → 50件/月）<br>- 品質重視（成功パターン活用）<br>- LinkedIn高パフォ投稿のX転用 |
| 4 | **Facebook閲覧数の低下** | 主要imp源の喪失 | 中 | - 現状の投稿スタイル維持<br>- インタラクション改善（コメント誘発型）<br>- 投稿時間帯最適化 |
| 5 | **匠名アカウント展開のタイミング判断ミス** | フェーズ2失敗、リソース浪費 | 中 | - フェーズ1で100万imp達成を厳守<br>- 数ヶ月安定運用してから展開<br>- ノウハウ蓄積を優先 |
| 6 | **PDCA未構築によるバラツキ継続** | Top/Bottom imp差改善なし | 中 | - 週次Top 5/Bottom 5分析を習慣化<br>- チェックリスト導入（5桁数字、企業名3社、問いかけ終結）<br>- テンプレート適用率80%達成 |

**重大リスク（発生時の緊急対応）**:
- **データ矛盾の再発**: LinkedIn impが0になる等のデータ異常 → 即座に原因調査（CSVエクスポート、API確認）
- **投稿時間帯最適化の副作用**: 時間帯変更でERが逆に低下 → 即座にロールバック
- **法令違反リスク**: ステマ規制、著作権侵害等 → 該当投稿を即座に削除、再発防止策策定
- **自動化フロー完全停止**: システム障害等で投稿が完全停止 → 手動投稿への切り替え、原因究明

## 6. 自動化フロー改善計画

### 現在の自動化構成

**使用ツール**: Claude Code Skill（sns-automation、sns-automation-v2）

**実行フロー**:
- Phase 1: データ収集（12-22分、5並列最適化版）
  - X タイムライン収集（200件）
  - Top 10 ツイート抽出
  - ツイート詳細取得（リンク・リプライ）
- Phase 2: 分析・調査（20-30分）
  - コンテンツ抽出（記事・YouTube・PDF、LLM判定ワンパス化）
  - リプライ分析 + Web調査（並列実行）
- Phase 3: 投稿生成（15-30分）
  - 高野式7パターン適用
  - LinkedIn、X、Threads向け生成
- Phase 4: 予約投稿（5-15分）
  - Late API経由でマルチプラットフォーム予約投稿
  - LinkedIn 1案/日、X 3投稿/日、Threads 2-3投稿/日

**所要時間**: 52-97分/実行

### 現在の課題

1. **修正点（手直し）の発生**: 具体的な内容は未調査
   - 推定される問題:
     - 文章のトーン調整（AI特有の硬さ、不自然な言い回し）
     - ハッシュタグ・絵文字・CTAの調整
     - プラットフォーム別最適化不足（文字数、フォーマット等）
     - 画像選定・差し替え
     - 事実誤認・情報の古さ

2. **運用効率**: 修正工数が不明、完全自動化に至っていない

### 改善目標

- **修正作業ゼロ化**: AIエージェント生成→Late API投稿まで手作業なし
- **所要時間短縮**: 52-97分 → 40-60分（Phase 3高速化）
- **投稿品質向上**: エンゲージメント率 0.63% → 1.5%

### 実施予定の改善策

1. **修正点の洗い出し**（最優先）:
   - 過去の投稿データと修正履歴を分析
   - 修正頻度の高い項目を特定
   - カテゴリ別に課題を分類

2. **AIエージェント生成精度向上**:
   - プロンプト改善（トーン、スタイル、フォーマット指定）
   - 高野式7パターンの適用精度向上
   - プラットフォーム別最適化ロジック強化

3. **LateAPI整形ロジック改善**:
   - 文字数制御の精度向上
   - ハッシュタグ・絵文字の自動最適化
   - CTAの自動挿入

4. **品質評価とフィードバックループ**:
   - 投稿後のパフォーマンスデータを自動収集
   - 高パフォーマンス投稿の特徴を分析
   - 生成ロジックに反映

## 7. フェーズ別ロードマップ

### フェーズ1: 実名アカウント×生成AI（2026年1-6月）

**目標**: 月間100万インプレッション達成

**重点施策**:
1. エンゲージメント率向上（0.63% → 1.5%）
2. 自動化フロー修正点ゼロ化
3. プラットフォーム別最適化（LinkedIn ER 2.50%、X平均imp 2,000回）

**マイルストーン**:
- 2026年3月末: 月間70万imp、ER 1.0%
- 2026年6月末: 月間100万imp、ER 1.5%

### フェーズ1.5: 安定運用・ノウハウ蓄積（2026年7-8月）

**目標**: 月間100万imp水準を安定維持

**実施内容**:
- 2-3ヶ月間の安定運用
- 自動化フローの最終調整
- 成功パターンのドキュメント化
- フェーズ2展開の準備（ジャンル選定、収益化商品設計）

### フェーズ2: 匠名アカウント展開（2026年9月以降）

**目標**: インプレッション拡大+収益化

**対象ジャンル**（優先順位順）:
1. 恋愛・マッチング系（note商品、コンサル）
2. キャリア・転職系（キャリア相談、有料コミュニティ）
3. 副業・起業系（教材、オンラインサロン）

**運用形態**:
- ジャンル別に匠名アカウント新規作成
- フェーズ1で確立した自動化フローを活用
- ジャンル特化の投稿コンテンツ生成

**マネタイズ戦略**:
- SNS投稿でインプレッション獲得
- 商品購入への誘導（note、アフィリエイト、コンサル等）
- 「ニーズがあり、お金が払われる領域」で発信
