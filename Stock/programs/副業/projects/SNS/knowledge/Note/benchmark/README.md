# Note記事ベンチマーク収集

## 概要

AIエージェント開発の学習データとして、Note記事のベンチマークを収集するプロジェクト。

## 対象

- **ジャンル**: テクノロジー・AI、ビジネス・起業、クリエイター・副業（3ジャンル横断）
- **収集規模**: 120記事以上（各ジャンル40記事）
- **対象時期**: 過去1年以内（2025年1月〜2026年1月）
- **有料記事**: 含む（タイトル・概要・メタデータのみ、本文は無料記事のみ）

## ディレクトリ構造

```
benchmark/
├── README.md                    # 本ファイル
├── raw/                         # 生データ
│   ├── tech_ai/                # テクノロジー・AI（40記事）
│   ├── business/               # ビジネス・起業（40記事）
│   └── creator/                # クリエイター・副業（40記事）
├── metadata/
│   ├── article_mapping.yaml    # 記事-テーママッピング
│   ├── engagement_stats.yaml   # エンゲージメント統計
│   └── quality_scores.yaml     # 品質評価スコア
└── analysis/
    ├── pattern_analysis.md     # 成功パターン分析
    └── training_dataset.json   # AI学習用データセット
```

## 収集データ項目

| 項目 | 説明 | 必須 |
|------|------|------|
| article_id | Note記事ID | ✓ |
| title | 記事タイトル | ✓ |
| url | 記事URL | ✓ |
| author | 著者名 | ✓ |
| published_at | 公開日時 | ✓ |
| genre | ジャンル分類 | ✓ |
| hashtags | ハッシュタグ一覧 | ✓ |
| is_paid | 有料記事フラグ | ✓ |
| like_count | スキ数 | ✓ |
| comment_count | コメント数 | |
| word_count | 文字数 | ✓ |
| body_text | 本文テキスト | ✓ |

## 品質評価基準

| 指標 | 最適値 | 配点 |
|------|-------|------|
| 文字数 | 2,000-5,000字 | 20点 |
| ハッシュタグ数 | 3-5個 | 15点 |
| 投稿時刻 | 19:00前後 | 10点 |
| スキ率 | 上位20% | 25点 |
| コメント数 | 1以上 | 10点 |
| 構造化度 | 見出し3個以上 | 20点 |
