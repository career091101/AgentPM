# Threads ベストプラクティス

## 概要

本ドキュメントは、Threadsのベストプラクティスをまとめたものです。Facebookの兄弟プラットフォームとして、短文・カジュアルトーン・会話重視の特性を持ちます。Late API PostAnalyticsスキーマの`views`フィールドに対応し、views指標でのパフォーマンス測定をサポートします。

---

## Threadsの特性とポジショニング

### 他SNSとの違い

| 特性 | Threads | X (Twitter) | Instagram |
|------|---------|------------|-----------|
| **主要ユーザー層** | 20-40代 | 20-40代 | 20-35代 |
| **主要目的** | カジュアルな会話 | 情報発信・議論 | ビジュアル共有 |
| **文字数制限** | 500文字 | 280文字（有料:4,000） | 2,200文字 |
| **コンテンツ寿命** | 6-12時間 | 15-30分 | 48時間 |
| **主要指標** | Views | Impressions | Impressions |

### Threadsが最適なユースケース

1. **カジュアルな意見表明**: 硬すぎないトーン、軽い会話
2. **Instagramフォロワーとの連携**: Metaアカウント統合によるクロスプラットフォーム展開
3. **短文テキスト投稿**: 画像・動画よりもテキスト中心
4. **リアルタイム性低**: X（旧Twitter）のような速報性は不要

## コンテンツ戦略

### コンテンツフォーマット別特性

| フォーマット | Views | エンゲージメント | 推奨用途 |
|------------|-------|----------------|---------|
| **短文テキスト（200-300文字）** | 高 | 高 | 意見表明、問いかけ |
| **長文テキスト（400-500文字）** | 中 | 中 | ストーリー、詳細解説 |
| **画像+テキスト** | 最高 | 最高 | ビジュアル補完 |
| **動画（短尺）** | 高 | 高 | 5-15秒の短い動画 |
| **スレッド（連投）** | 中 | 低 | 長いストーリー（非推奨） |

### 推奨コンテンツミックス

**週10投稿の場合の内訳**:
- 短文テキスト: 60%（週6投稿）
- 画像+テキスト: 30%（週3投稿）
- 動画: 10%（週1投稿）

**重要**: スレッド連投は避ける（エンゲージメント分散）

## Views指標の特殊性

### Threads独自の指標: Views

**Late API PostAnalyticsスキーマ対応**:
```yaml
PostAnalytics:
  properties:
    views:
      type: integer
      description: Number of times a post was viewed
```

**views vs impressions**:
- **views**: 投稿がユーザーの画面に表示された回数（Threads）
- **impressions**: 投稿がフィードに表示された回数（X, LinkedIn）

**views=0の場合の対応**:
- 計測不可として扱う
- エンゲージメント絶対数（likes + comments + shares）のみで評価
- エンゲージメント率は計算しない

### KPI目標値（週次）

| 指標 | 目標値 | 評価基準 |
|------|--------|---------|
| **投稿あたりViews** | 100 views/post | ✅≧100、⚠️80-99、❌<80 |
| **投稿あたりエンゲージメント** | 5 engagement/post | ✅≧5、⚠️4、❌<4 |
| **エンゲージメント率** | 0.8-1.5% | views>0の場合のみ計算 |

## トーンとスタイル

### Threadsのトーン指針

**カジュアル重視**:
- 硬すぎる専門用語は避ける
- 絵文字を適度に使用（2-3個/投稿）
- 「です・ます」調よりも「だ・である」調
- 「〜かも」「〜って感じ」などの口語表現OK

**例（X vs Threads）**:

| プラットフォーム | 投稿例 |
|------------|------|
| **X** | 「AI開発において、RAG（Retrieval-Augmented Generation）は検索拡張生成と呼ばれ、LLMの精度向上に寄与します。」 |
| **Threads** | 「RAGって最近よく聞くけど、要は検索機能をLLMに追加して賢くするやつ💡 これでハルシネーション減るんだよね〜」 |

### 投稿構成のベストプラクティス

**3パート構成**:
1. **フック（1行目）**: 注目を引く一文
2. **本文（2-4行）**: 簡潔な説明・意見
3. **クロージング（最終行）**: 問いかけ・行動喚起

**例**:
```
AI開発、実はそんなに難しくない説 🤔

Claude APIとかOpenAI APIを使えば、
コード数行でチャットボット作れちゃう。
むしろ「何を作るか」のアイデア勝負になってきてる感。

みんな、最初に作ったAIツールって何？
```

## エンゲージメント最適化

### コメント誘導テクニック

**効果的な問いかけ**:
- 「あなたはどっち派？」（2択質問）
- 「〇〇で困ったことある人いる？」（共感喚起）
- 「みんなの〇〇教えて！」（情報収集型）

**エンゲージメント信号の重み**（推定）:
- コメント: 最高（Metaアルゴリズム共通）
- いいね: 高
- シェア（リポスト）: 高
- Views: 中

### 返信戦略

**24時間以内の返信**:
- コメントには24時間以内に返信
- 会話の継続でエンゲージメント倍増
- 質問形式で返信すると再度コメントが来やすい

**返信例**:
```
コメント: 「この方法、初心者でもできますか？」
返信例: 「できるよ！プログラミング経験ゼロから始めた人も何人もいる。どの言語使ってる？」
```

## 投稿頻度と時間帯

### フェーズ別推奨頻度

| フェーズ | 投稿頻度 | 理由 |
|---------|---------|------|
| **初期（0-1,000フォロワー）** | 日1-2投稿 | アルゴリズム学習、フォロワー構築 |
| **成長（1,000-10,000）** | 日2-3投稿 | エンゲージメント最大化 |
| **成熟（10,000+）** | 日1-2投稿 | 品質重視、バーンアウト防止 |

### 最適時間帯（日本市場）

**曜日別推奨時間**:

| 曜日 | 最適時間1 | 最適時間2 | 理由 |
|------|----------|----------|------|
| **平日** | 7:00-9:00 | 19:00-21:00 | 通勤時間、帰宅後 |
| **土日** | 10:00-12:00 | 20:00-22:00 | 朝のんびり時間、夜のリラックスタイム |

**避けるべき時間帯**:
- 深夜1:00-6:00（リーチ激減）
- 平日13:00-17:00（仕事中、スマホ見ない）

## Meta統合戦略

### InstagramとのクロスポストCross-post戦略

**Threads → Instagram Stories**:
- Threads投稿をInstagram Storiesでシェア
- 両プラットフォームのフォロワーにリーチ
- Metaアカウント統合でシームレス

**トーン調整の必要性**:
- Instagram: ビジュアル重視、洗練されたトーン
- Threads: テキスト中心、カジュアルトーン
- 同じ内容でもトーン調整が必要

### Metaアルゴリズムの共通点

**Facebook/Instagram/Threads共通のアルゴリズム原則**:
1. **コメント最優先**: コメント数が最も重要なエンゲージメント信号
2. **滞在時間**: 投稿を見ている時間が長いほど評価
3. **シェア重視**: シェアはバイラル性の指標
4. **外部リンク抑制**: プラットフォーム内滞在を優先

## 避けるべき失敗パターン

### ❌ 失敗パターン vs ✅ 成功パターン

| 失敗パターン | 理由 | 成功パターン |
|-----------|------|------------|
| ❌ スレッド連投（5投稿以上） | エンゲージメント分散、読みにくい | ✅ 単一投稿で完結（最大2-3連投） |
| ❌ X風の硬いトーン | Threadsの文化に合わない | ✅ カジュアル・会話的トーン |
| ❌ 外部リンク多用 | アルゴリズム抑制 | ✅ プロフィールにリンク集約 |
| ❌ ハッシュタグ乱用（5個以上） | スパム判定リスク | ✅ ハッシュタグ1-3個 |
| ❌ 自動投稿ツール多用 | エンゲージメント率低下 | ✅ 手動投稿中心、会話重視 |

## 日本市場への適応

### 日本特有の文化的配慮

**謙虚さの表現**:
- 「〜と思います」「〜かもしれません」などの控えめ表現
- 断定を避ける（ただし、過度に弱腰にならない）

**絵文字文化**:
- 日本ユーザーは絵文字を多用
- 2-3個/投稿が最適（過剰は避ける）
- 顔文字（^_^、(^^;）も好まれる

**礼儀正しさ**:
- 批判的意見も丁寧な表現で
- 「〜ですよね？」「〜じゃないですか？」などの同意求め表現

## 次のステップ

### 測定と改善

**週次レビュー項目**:
1. **Views数トレンド**: 増加/横ばい/減少の判定
2. **エンゲージメント率**: views>0の投稿のみで計算
3. **トップ5投稿分析**: 共通パターンの抽出
4. **フォロワー増減**: 純増数の確認

### さらなる最適化

1. **A/Bテスト**: トーン・長さ・時間帯を変えて比較
2. **コメント分析**: どんなコメントが来るか分析
3. **競合ベンチマーク**: 同ジャンルの成功アカウント分析
