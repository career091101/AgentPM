## 3. ペルソナの心理・行動特性

### 🧠 価値観・信念

```
「変化を恐れるな。ただし、根拠のない変化は避けよ」
```

- **テクノロジーへの姿勢**: 積極的だが慎重。導入前に十分な検証を求める
- **情報の信頼基準**: 数字・データで裏付けられた情報を好む
- **意思決定**: 「誰が言ったか」を重視（権威性）
- **リスク許容度**: 中程度。失敗しても学びがあれば許容

### 😤 フラストレーション（Pain Points）

| Pain Point | 詳細 |
|------------|------|
| **情報過多** | AI関連ニュースが多すぎて、何が本当に重要かわからない |
| **判断の遅れ** | 競合がAI導入を進める中、自社の意思決定が遅れている不安 |
| **社内理解** | 役員や社員にAIの重要性を説明するのが難しい |
| **投資判断** | AIに投資すべきか、待つべきか判断材料が不足 |

### ✨ 願望（Gains）

| Gain | 詳細 |
|------|------|
| **要約された情報** | 忙しい中でも5分で業界動向を把握したい |
| **判断の材料** | 「うちも導入すべきか」を判断できる具体的な事例 |
| **語れるネタ** | 役員会議や商談で「最新のAI動向」を語りたい |
| **先見性** | 周囲より一歩先の情報を持っておきたい |

## 4. コンテンツ設計への示唆

### 📝 刺さるコンテンツの特徴

| 要素 | 具体的な設計 |
|------|-------------|
| **冒頭** | 衝撃的な数字で注意を引く（「1兆円」「100万人」） |
| **権威性** | 著名CEO/専門家の発言を引用（サム・アルトマン、イーロン・マスク） |
| **具体性** | 抽象論を避け、具体的な企業名・金額・期限を明示 |
| **示唆** | 「経営者として何を考えるべきか」を最後に提示 |
| **長さ** | 2〜3分で読める長さ（500〜800字） |

### 🎨 ビジュアル設計

| 要素 | 推奨 |
|------|------|
| **画像** | データグラフ、インフォグラフィック、関係図 |
| **色調** | ビジネスライク（青、グレー、白基調） |
| **避けるべき** | カジュアルすぎるミーム、極端な煽り表現 |

## 7. 投稿テンプレート（ペルソナ対応版）

### テンプレートA: CEO向け（田中ペルソナ）
```markdown
【見出し】
[著名CEO名]は「[衝撃的発言]」と断言した。

【本文】
1. 発言の背景と数字（具体的な金額・規模）
2. 業界への影響（競合動向、市場変化）
3. 経営者が今考えるべきこと（アクションの示唆）

【締め】
あなたの会社では、この変化にどう対応しますか？
```

### テンプレートB: 企画担当向け（鈴木ペルソナ）
```markdown
【見出し】
[企業名]がAI導入で[成果]を達成。その方法とは？

【本文】
1. 導入前の課題
2. 具体的な施策（ツール名、期間、コスト）
3. 達成した成果（数字で）
4. 自社への応用ポイント

【締め】
詳細なケーススタディは[リンク]で公開中
```
