# SNSデータ分析要件 ヒアリングシート

## 📋 プロジェクト背景
- **プロジェクト名**: SNS生産性向上プロジェクト
- **主要KPI**: 月間インプレッション 100万imp
- **時短目標**: 1投稿あたり15分以内
- **対象期間**: 3ヶ月（持続可能性）
- **重点SNS**: Facebook, LinkedIn, X (Twitter)

---

## 🎯 Section 1: 分析のゴールとKPI

### Q1. この分析で最も知りたいことは何ですか？（複数選択可）

**選択肢:**
- [ ] A. どのトピック/テーマが最もインプレッションを獲得しているか
- [ ] B. どの投稿フォーマット（テキスト、画像、動画）が効果的か
- [ ] C. 最適な投稿時間帯・曜日
- [ ] D. エンゲージメント率の高い投稿パターン
- [ ] E. プラットフォーム間の特性の違い

**💡 推奨回答**: 
```
A, D, E を選択
理由: 
- A: 「勝ちトピック」を特定し、投稿テーマの再現性を高める
- D: インプレッションだけでなく、質の高いエンゲージメントも重要
- E: 各SNSに最適化された投稿戦略を構築するため
```

---

### Q2. 最も重視する指標は何ですか？（優先順位をつけてください）

**選択肢:**
1. インプレッション数（リーチ）
2. エンゲージメント率（いいね、コメント、シェア）
3. クリック数・保存数
4. フォロワー増加数
5. 動画視聴時間（動画投稿の場合）

**💡 推奨回答**: 
```
1位: インプレッション数（目標100万impに直結）
2位: エンゲージメント率（質の担保）
3位: 保存数・シェア数（将来的なリーチ拡大）

理由: プロジェクトKPIと直接連動する順序
```

---

## 📊 Section 2: 分析の深さと範囲

### Q3. データをどのレベルで分析しますか？

**選択肢:**
- [ ] A. 投稿全体の傾向分析（マクロ視点）
- [ ] B. トップ10%の高パフォーマンス投稿の詳細分析（ベストプラクティス抽出）
- [ ] C. ボトム10%の低パフォーマンス投稿の分析（失敗パターン把握）
- [ ] D. 投稿タイプ別のセグメント分析

**💡 推奨回答**: 
```
A, B, D を実施

優先順位:
1. B (トップ10%分析) → 勝ちパターンを即座に特定
2. A (全体傾向) → ベースライン理解
3. D (セグメント分析) → SNS別・トピック別の最適化

C（失敗分析）は時間があれば実施
```

---

### Q4. 比較軸として重視するのは？

**選択肢:**
- [ ] A. プラットフォーム間比較（FB vs LinkedIn vs X）
- [ ] B. 投稿内容のカテゴリ比較（AIニュース vs 企業分析 vs 投資情報など）
- [ ] C. 投稿フォーマット比較（テキスト vs 画像 vs 動画）
- [ ] D. 時系列変化（月ごとのトレンド）

**💡 推奨回答**: 
```
A, B を最優先

理由:
- A: 各SNSに最適化された投稿戦略を立てるため必須
- B: 「勝ちトピック」を特定し、投稿テーマの方向性を決定

C, D は二次分析として実施
```

## 🎨 Section 3: アウトプット・成果物

### Q5. 分析結果のアウトプット形式は？（複数選択可）

**選択肢:**
- [ ] A. レポート（Markdown / PDF形式）
- [ ] B. ダッシュボード（データの可視化）
- [ ] C. 投稿テンプレート（勝ちパターンの型化）
- [ ] D. アクションプラン（具体的な次の一手）

**💡 推奨回答**: 
```
A, C, D を作成

優先順位:
1. C (投稿テンプレート) → 即実行可能な成果物
2. D (アクションプラン) → 次の3ヶ月の投稿戦略
3. A (レポート) → 分析根拠の記録

理由: プロジェクトゴールは「投稿案の作成」なので、
実務に直結する成果物を優先
```

---

### Q6. 「勝ちパターン」として定義したい要素は？

**選択肢（複数選択可）:**
- [ ] A. トピック（例: OpenAI最新情報）
- [ ] B. 投稿の構成（例: 結論→データ→考察）
- [ ] C. 文字数・文章スタイル
- [ ] D. タイトル/見出しのパターン
- [ ] E. 画像・ビジュアルの有無
- [ ] F. 投稿時間帯

**💡 推奨回答**: 
```
A, B, D, E を定義

理由: これらは再現性が高く、即座に実践可能
- A: コンテンツの方向性
- B: 投稿フォーマットのテンプレート化
- D: インプレッションを最大化するタイトル法則
- E: ビジュアル戦略

C, F は補足情報として記録
```

## ⏱️ Section 4: 実施スケジュール

### Q7. 分析結果の提出期限は？

**選択肢:**
- [ ] A. 即日（簡易分析）
- [ ] B. 3日以内（詳細分析）
- [ ] C. 1週間以内（包括的分析 + テンプレート作成）

**💡 推奨回答**: 
```
B (3日以内) を推奨

スケジュール案:
- Day 1: データクリーニング + 基礎統計分析
- Day 2: 高パフォーマンス投稿の詳細分析 + パターン抽出
- Day 3: テンプレート作成 + アクションプラン策定

理由: 年末までに戦略を固め、年明けから新戦略で投稿開始
```

## 🚀 Section 5: 最終確認

### Q8. 分析で最も避けたいリスクは？

**選択肢:**
- [ ] A. データ量が多すぎて分析が終わらない
- [ ] B. 分析結果が実務に活かせない（理論倒れ）
- [ ] C. SNS特性を無視した一般論になる

**💡 推奨回答**: 
```
B を最優先で回避

対策:
- 分析の各ステップで「これは投稿に使えるか？」を確認
- テンプレート化できない分析結果は優先度を下げる
- 「勝ちパターン」は具体的な投稿例とセットで提示

理由: プロジェクトゴールは「使える投稿案の作成」
```
