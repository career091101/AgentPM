# 月刊アプリマーケティング記事抽出計画

## プロジェクト概要

**目的**: 月刊アプリマーケティングマガジンの記事から、落合式リサーチ手法を適用して知識を体系的に抽出・蓄積する

**対象マガジン**: [月刊アプリマーケティング](https://markelabo.com/m/mc375c9b46464)

**記事数**: 14記事（2026年1月1日時点）

**参考手法**: 落合式リサーチ（Deep Research to Note）を記事用にカスタマイズ

## 実行手順

### 1. 入力受付と初期設定

#### 必須パラメータ
- `article_source`: 記事URL（note記事）またはマガジンURL
- `output_format`: サマリー形式（`markdown` / `notion` / `csv`）

#### オプションパラメータ
- `research_theme`: 自分の研究テーマ（副業・アプリマーケティング・グロースハック等）デフォルト: "アプリマーケティング"
- `reading_mode`: 読解モード
  - `quick` - 10分速読（要点のみ抽出）
  - `standard` - 20分標準（6つの質問に回答）デフォルト
  - `deep` - 30分精読（詳細分析+実践プラン）
- `language`: 出力言語（`ja`）デフォルト: `ja`
- `existing_articles`: 既読記事リスト（重複チェック用）デフォルト: 空リスト
- `next_article_count`: 推薦する関連記事数 デフォルト: 3
- `image_count`: サマリーに含める画像数 デフォルト: 1-3

#### 使用ツール
- ブラウザ自動化（Claude in Chrome）
- パラメータバリデーション

#### 注意点
- 記事URLが無効な場合はエラーハンドリング
- デフォルト値の適切な設定（読解モード: standard、出力形式: Markdown）
- マガジン全体を指定した場合は記事一覧を取得してから個別処理

---

### 2. 記事構造解析

#### 処理内容
- note記事のHTMLをフェッチ
- 記事構造の特定（タイトル、見出し、本文、画像、リンク等の自動検出）
- 画像の抽出と保存
- 参考文献・外部リンクの抽出
- 記事メタデータの取得（著者、公開日、いいね数、コメント数等）

#### 使用ツール
- WebFetch / ブラウザ自動化
- HTML/DOMパーサー
- 画像ダウンロードツール

#### 注意点
- note記事の構造は比較的統一されているが、埋め込みコンテンツ（YouTube、Twitter等）に注意
- 有料部分がある場合は購読済み前提で全文を取得
- 画像抽出失敗時はテキストのみで継続
- 外部リンクは後で参照できるよう記録

---

### 3. 戦略的読解

**論文との違い**: 論文は逆順（Abstract→Conclusion→Results）が効率的だが、note記事は順読みが基本。ただし、**要点抽出重視**の姿勢は踏襲。

#### 読解プロセス
1. **タイトル解析**: 記事の主題と期待される内容の把握
2. **概要・導入部解析**: 記事の背景・問題提起の理解
3. **本文解析**:
   - 事例・データの抽出
   - 施策の具体的内容
   - 結果・インパクトの確認
4. **まとめ・結論解析**: 重要ポイントの再確認

#### 使用ツール
- LLM（Claude/ChatGPT）による自然言語理解
- セマンティック抽出
- キーフレーズ抽出

#### 注意点
- だらだら読まず、**1記事20分以内**を目標
- 自分の副業・アプリマーケティング実践にとっての意義を常に問う
- 重要記事と判断した場合のみ深掘り（deepモード）を適用

### 4. 6つの質問への回答生成（記事用カスタマイズ版）

#### 質問1: **「どんなもの？」**
- 記事のテーマと主要メッセージを3行要約
- 扱っているプロダクト/サービス/施策の概要
- 対象読者と記事の目的

**出力例**:
```
アプリのペイウォール画面に「お得になる金額」を具体的に表示したら、
年間プラン選択率が20％向上した事例を紹介。
アプリマーケティング担当者向けのCVR改善施策の知見。
```

---

#### 質問2: **「既存知識と比べてどこがユニークか？」**
- 一般的な施策との違い
- 新規性・独自性のポイント
- 他の記事や書籍との差別化要素

**出力例**:
```
従来は「割引率（30% OFF）」を強調するのが一般的だったが、
この事例では「年間で節約できる具体的な金額（¥3,600お得）」を表示。
金額の具体性が心理的インパクトを高め、長期プラン選択を促進。
```

---

#### 質問3: **「施策・手法のキモはどこ？」**
- 実装された具体的な施策・デザイン・コピー
- なぜその施策が効果的だったのか（心理学的背景等）
- 重要な図表・スクリーンショット（1-3点）の抽出

**出力例**:
```
【施策詳細】
- ペイウォール画面に「年間プランで¥3,600お得」と金額明示
- 月額プランとの比較表を視覚化
- 「今だけ限定」の緊急性訴求は行わず、価値訴求に徹した

【効果的だった理由】
- 損失回避バイアス: 抽象的な「30% OFF」より具体的な「¥3,600」が刺さる
- アンカリング効果: 月額プランを基準に年間プランの価値を強調
```

---

#### 質問4: **「どうやって検証した？効果は？」**
- ABテストの実施方法
- 測定した指標（CVR、売上、リテンション等）
- 結果の数値（具体的なパーセンテージや金額）
- 統計的有意性の有無

**出力例**:
```
【検証方法】
- ABテスト実施（A: 従来の割引率表示、B: 具体的金額表示）
- 測定期間: 4週間
- サンプルサイズ: 各グループ5,000ユーザー

【結果】
- 年間プラン選択率: 20％向上（A: 25% → B: 30%）
- 統計的有意性: p < 0.01
- 売上への影響: 月次MRRが15％増加
```

---

#### 質問5: **「議論・限界点・応用可能性は？」**
- 施策の限界点（適用できないケース等）
- 今後の改善余地
- 他のプロダクトへの応用可能性
- 批判的視点（本当に再現可能か？）

**出力例**:
```
【限界点】
- 価格帯が高い（月額3,000円以上）プロダクトでは効果が薄い可能性
- ブランド認知が低い段階では、価格訴求よりも価値訴求が優先されるべき

【応用可能性】
- SaaSのペイウォール、サブスク型ECにも適用可能
- 「時間的価値」の訴求（例: 年間100時間節約）にも転用できる

【批判的視点】
- 記事では成功事例のみ紹介。失敗事例や条件の詳細が不明
- ユーザー属性（年齢層、所得層）による効果の違いは不明
```

---

#### 質問6: **「次に読むべき記事は？」**
- マガジン内の関連記事（3-5本）
- 外部リンク先の重要記事
- 推薦理由の明記

**出力例**:
```
1. 「不安解消テキストでCVRが2倍」（https://markelabo.com/n/n6ec19f3b24a5）
   → ペイウォールのコピー改善に関する補完的知見

2. 「ユーザー登録画面に『メリット箇条書き』を入れたらCVRが6％伸びた」（https://markelabo.com/n/n6347cb5bc5af）
   → 価値訴求の別パターン事例

3. 「プロダクトのマーケティング良施策まとめ20事例」（https://markelabo.com/n/ne843ab198ccc）
   → 複数施策を俯瞰して学ぶ
```

---

#### 使用ツール
- LLMプロンプトチェーン
- 批判的思考モデル
- 関連記事推薦アルゴリズム（セマンティック類似度計算）

#### 注意点
- 各質問への回答は簡潔に（1-3段落以内）
- 必ず批判的視点（限界点、応用可能性）を含める
- 自分の副業実践にどう活かせるかを常に考える

### 7. 次の記事推薦

#### 推薦ロジック
1. **マガジン内の関連記事**:
   - テーマ（CVR改善、リテンション、グロース等）の類似度
   - 参照リンクの共通性
   - セマンティック類似度計算
2. **外部リンク先の重要記事**:
   - 記事内で言及されている外部記事
   - 著者の他の記事
3. **既読記事リストとの照合**: 重複除外

#### 推薦数
- デフォルト: 3記事
- オプションで変更可能

#### 使用ツール
- セマンティック検索（埋め込みベクトル比較）
- 推薦アルゴリズム
- 既読リスト管理

#### 注意点
- 次の記事候補は必ずマガジン内または外部リンク先から選定
- 推薦理由は具体的に（「参考になる」ではなく「CVR改善施策の別パターン事例」）
- 既読リストを活用して重複を避ける

## 効率化の原則

### 1. 戦略的読解の徹底
- だらだら読まず、**1記事20分以内**を目標
- 要点抽出重視：全文を丁寧に読むのではなく、キーポイントをスキャン
- 重要記事（実践優先度が高い）と判断した場合のみ深掘り

### 2. アウトプット駆動の学習
- **必ずサマリーを作成**: 読むだけでは定着しない
- **批判的視点の組み込み**: 単なる要約ではなく、限界点・応用可能性を必ず含める
- **実践アイデアの追記**: 自分の副業にどう活かすかを必ず考える

### 3. 自分軸の読解
- **著者の意図通りに読まない**: 自分の副業実践にとっての意義を問い続ける
- **問題解決軸での判断**: この記事が自分の課題解決にどう役立つかを常に考える
- **次のアクションを明確化**: 次に読むべき記事、次に試すべき施策を必ず特定

### 4. データベース管理の徹底
- **CSV/Notionで一元管理**: タグ付け、全文検索、関連記事の可視化
- **次に読むべき記事を明記**: 知識のネットワークを辿る連続性を保つ
- **実践アイデアを記録**: 読んだだけで満足せず、実践につなげる

### 5. 完璧主義の回避
- **全ての記事を完璧に理解しようとしない**: 6つの質問に答えられればOK
- **美しいサマリーを作ろうとしない**: 情報が整理されていれば十分
- **フィードバックループを作る**: 完璧主義を避け、反復改善を重視

## LLM活用の注意

### ハイブリッドアプローチ
- **LLM自動要約をベースに、人間が批判的視点を追加**
- LLMの出力をそのまま鵜呑みにせず、必ず自分で検証
- 人間の役割: 要約作成 → 批判的評価・自分のテーマとの関連付け

### LLMの活用場面
- 記事の構造解析
- 要点の抽出
- 関連記事の推薦
- タグの自動生成

### 人間が担うべき部分
- 批判的視点の追加（限界点、応用可能性）
- 自分の副業実践への応用アイデア
- 実践優先度の判断
- 次に読むべき記事の最終判断
