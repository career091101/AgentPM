## 全体構成

スタートアップの成功に至る道のりを「20のステップ」に分解し、それを5つの大きなステップに分類。

```
STEP 0: Starting Point（なぜ失敗するのか）
    ↓
STEP 1: アイデアの検証
    ↓
STEP 2: CPF（Customer Problem Fit）- 課題の質を上げる
    ↓
STEP 3: PSF（Problem Solution Fit）- ソリューションの検証
    ↓
STEP 4: PMF（Product Market Fit）- 人が欲しがるものを作る
    ↓
STEP 5: スケール - 事業を拡大する
```

---

## STEP 0: Starting Point

### なぜスタートアップは失敗するのか

スタートアップの失敗原因トップ20（CB Insights調査より）:

| 順位 | 失敗原因 | 割合 |
|-----|---------|------|
| 1 | 市場にニーズがなかった | 42% |
| 2 | 資金が尽きた | 29% |
| 3 | チームが悪かった | 23% |
| 4 | 競合に負けた | 19% |
| 5 | 価格設定の問題 | 18% |

> **重要**: 最大の失敗原因は「誰も欲しがらないものを作った」こと。これを避けるためにPMFを目指す。

### スタートアップ・メタ原則

1. **「作れば売れる」は幻想** - 誰も欲しくないものを完璧に作っても意味がない
2. **Plan A は必ず間違っている** - 仮説検証のサイクルを高速で回す
3. **正しい順序で進める** - 各ステップをスキップしない

## STEP 1: アイデアの検証

### 良いアイデアの条件

#### 3U + 1 の法則

アイデアが解決しようとしている課題が以下を満たすか確認する:

| 要素 | 説明 | 質問例 |
|-----|------|-------|
| **Unworkable** (切実) | 課題が解決されないと困る | 「今この課題を解決しないと何が起こるか?」 |
| **Unavoidable** (不可避) | 必ず直面する課題である | 「ターゲットの何%がこの課題に直面するか?」 |
| **Urgent** (今すぐ) | すぐに解決したい | 「今日解決したいか、来年でいいか?」 |
| **+1: Underserved** (代替なし) | 既存の解決策がない/不十分 | 「既存代替品の問題点は何か?」 |

### スタートアップのアイデア着想法

1. **自分が欲しいものを作る** - Founder-Problem Fit
2. **既存市場の「非効率」を見つける** 
3. **新しいテクノロジーの応用を考える**
4. **規制変化を先取りする**
5. **異なる業界のベストプラクティスを持ち込む**

### アイデア検証チェックリスト

- [ ] 自分が本当に欲しいと思うプロダクトか?
- [ ] 一言（30秒）で説明できるか?
- [ ] 既存の解決策は何か? その問題点は?
- [ ] ターゲット顧客は誰か（具体的に）?
- [ ] 市場規模は十分か（最低$1B）?

## STEP 2: CPF（Customer Problem Fit）

### 目的

「顧客が本当に解決したいと思っている課題」を特定する。

### 課題仮説キャンバス

```
┌─────────────────────────────────────────────────────────────┐
│ 課題仮説キャンバス                                            │
├──────────────────────┬──────────────────────────────────────┤
│ ターゲット顧客        │ [具体的なペルソナを記載]              │
├──────────────────────┼──────────────────────────────────────┤
│ 課題                 │ [顧客が抱える主要な課題]              │
├──────────────────────┼──────────────────────────────────────┤
│ 既存代替品           │ [顧客が現在使っている解決策]          │
├──────────────────────┼──────────────────────────────────────┤
│ 既存代替品の問題点    │ [なぜ不満があるか]                   │
├──────────────────────┼──────────────────────────────────────┤
│ 検証方法             │ [どうやって仮説を検証するか]          │
└──────────────────────┴──────────────────────────────────────┘
```

### 顧客インタビューの方法

#### インタビュー数の目安

- **最低**: 15-20人
- **理想**: 30人以上
- **パターンが見えるまで**: 同じ回答が70%以上

#### 良い質問の例

| カテゴリ | 質問例 |
|---------|-------|
| 現状把握 | 「〇〇について、普段どうされていますか?」 |
| 課題発見 | 「一番困っていることは何ですか?」 |
| 解決意欲 | 「もしこの課題が解決されたら、いくら払いますか?」 |
| 既存代替 | 「今はどうやって解決していますか?」 |
| 頻度確認 | 「この課題にどのくらいの頻度で直面しますか?」 |

#### 悪い質問の例（避けるべき）

- ❌ 「こういうプロダクトがあったら使いますか?」（誘導的）
- ❌ 「いくらなら買いますか?」（架空の状況）
- ❌ 「便利だと思いますか?」（抽象的）

### ペルソナ作成

#### ペルソナシート

```
┌─────────────────────────────────────────────────────────────┐
│ ペルソナ名: [名前]                                          │
├──────────────────────┬──────────────────────────────────────┤
│ 年齢・性別           │ [例: 35歳男性]                       │
├──────────────────────┼──────────────────────────────────────┤
│ 職業・役職           │ [例: IT企業マネージャー]              │
├──────────────────────┼──────────────────────────────────────┤
│ 日常                 │ [1日の行動パターン]                  │
├──────────────────────┼──────────────────────────────────────┤
│ 課題・ペイン         │ [最も困っていること]                 │
├──────────────────────┼──────────────────────────────────────┤
│ ゴール               │ [達成したいこと]                     │
├──────────────────────┼──────────────────────────────────────┤
│ 現在の解決策         │ [今使っているツール・方法]            │
├──────────────────────┼──────────────────────────────────────┤
│ 不満点               │ [既存解決策への不満]                 │
└──────────────────────┴──────────────────────────────────────┘
```

### CPF達成の判断基準

以下の条件を「すべて」満たすこと:

- [ ] 20人以上の顧客インタビューを実施した
- [ ] 70%以上の顧客が同じ課題を挙げた
- [ ] 顧客が「お金を払ってでも解決したい」と言っている
- [ ] 課題の切実さ（Urgency）が確認できた
- [ ] 既存代替品とその問題点が明確になった

## STEP 3: PSF（Problem Solution Fit）

### 目的

課題に対する「適切なソリューション」を見つける。

### UVP（Unique Value Proposition）の定義

#### UVPキャンバス

```
[ターゲット顧客] が
[解決したい課題] を解決するために
[プロダクト名] を使うと
[得られる便益] が得られます。
なぜなら [差別化要因] だからです。
```

#### UVPの例

> 「時間がない起業家が、顧客のニーズを素早く検証するために、AIインタビューシミュレーターを使うと、リアルな課題インサイトが得られます。なぜなら、AIが数百人分のペルソナ視点を提供するからです。」

### MVPの種類

| MVP種類 | 説明 | 適用場面 |
|--------|------|---------|
| **ペーパープロトタイプ** | 紙やスライドでの説明 | 最初期の検証 |
| **ランディングページ** | 簡単なWebページ+登録フォーム | 興味関心の測定 |
| **コンシェルジュMVP** | 人力でサービス提供 | サービス系 |
| **オズの魔法使いMVP** | 裏で人が対応、表では自動に見せる | 技術的に難しいもの |
| **プレトタイプ** | 存在しないプロダクトを「売る」 | 需要検証 |

### 10倍改善の原則

既存代替品より**10倍良い**必要がある:

- 10倍速い
- 10倍安い
- 10倍便利
- 10倍楽しい

> **注意**: 2-3倍の改善では、顧客は乗り換えない。

### PSF達成の判断基準

- [ ] UVPが一文で説明できる
- [ ] MVP（何らかの形）を作成した
- [ ] 顧客がMVPを見て「欲しい」と言った
- [ ] 既存代替品より10倍良い点が明確
- [ ] 初期ユーザーが獲得できそう

## STEP 4: PMF（Product Market Fit）

### PMFとは

「顧客が本当に欲しがるプロダクトを作れた状態」

### PMF達成の指標

#### Sean Ellisテスト

> 「このプロダクトが使えなくなったらどう思いますか?」

| 回答 | 割合 | 判定 |
|-----|------|------|
| 非常に残念 | **40%以上** | ✅ PMF達成 |
| 少し残念 | - | - |
| 残念ではない | - | - |

#### その他の指標

| 指標 | PMF達成基準 |
|-----|------------|
| 月次成長率 | 10%+/月 |
| NPS | 50以上 |
| Churn Rate | 5%以下 |
| LTV/CAC | 3倍以上 |

### ピボットの判断

#### ピボットすべきサイン

- 顧客インタビューで「いいね」と言われるが購入されない
- 成長率が3ヶ月連続で下がっている
- チームのモチベーションが下がっている
- 資金が6ヶ月以内に尽きる見込み

#### ピボットの種類

1. **ズームイン** - 機能の一部に特化
2. **ズームアウト** - より大きな課題へ拡大
3. **顧客セグメント変更** - 別のターゲットへ
4. **課題ピボット** - 別の課題を解決
5. **チャネルピボット** - 販売経路を変更

## STEP 5: スケール

### ユニットエコノミクス

#### LTV（顧客生涯価値）

```
LTV = ARPU × 粗利率 × 平均継続月数
```

#### CAC（顧客獲得コスト）

```
CAC = (マーケティング費用 + 営業費用) / 新規顧客数
```

#### 健全性の目安

| 指標 | 目標値 |
|-----|-------|
| LTV/CAC | **3倍以上** |
| CACの回収期間 | **12ヶ月以内** |

### グロースハック

#### AARRRフレームワーク（パイレーツメトリクス）

| 段階 | 説明 | 主要KPI |
|-----|------|---------|
| **Acquisition** | 獲得 | サイト訪問数、登録数 |
| **Activation** | 活性化 | アクティベーション率 |
| **Retention** | 継続 | 継続率、Churn Rate |
| **Referral** | 紹介 | K-factor、NPS |
| **Revenue** | 収益 | ARPU、LTV |

### フライホイールの構築

成長が次の成長を生む「好循環」を作る:

```
ユーザー増加
    ↓
データ蓄積
    ↓
プロダクト改善
    ↓
満足度向上
    ↓
口コミ増加
    ↓
ユーザー増加（循環）
```
