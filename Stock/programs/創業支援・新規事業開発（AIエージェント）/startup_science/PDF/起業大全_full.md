## Framework 1: ミッション・ビジョン・バリュー（MVV）

### 目的

会社の「存在意義」と「向かう方向」を定義し、全社員の行動指針とする。

### MVVの定義

| 要素 | 定義 | 質問 |
|-----|------|------|
| **Mission** | 会社の存在意義・使命 | 「なぜこの会社は存在するのか?」 |
| **Vision** | 実現したい未来像 | 「10年後、どんな世界を作りたいか?」 |
| **Value** | 大切にする価値観・行動指針 | 「どのように行動すべきか?」 |

### MVV策定のポイント

1. **創業者の原体験** - なぜこの事業を始めたか
2. **顧客への価値** - 誰のために存在するか
3. **社会課題との接続** - どんな課題を解決するか
4. **差別化要素** - なぜ自分たちがやるのか

### MVVテンプレート

```
【Mission】
私たちは [ターゲット顧客] のために
[解決する課題] を解決し
[提供する価値] を創造します。

【Vision】
[実現したい未来の状態]

【Value】
1. [価値観1]: [具体的な行動例]
2. [価値観2]: [具体的な行動例]
3. [価値観3]: [具体的な行動例]
```

## Framework 2: 戦略

### 目的

競争優位を構築し、持続可能なビジネスを作る。

### 戦略キャンバス

| 項目 | 説明 |
|-----|------|
| **ターゲット市場** | どの市場で戦うか |
| **顧客セグメント** | 誰を顧客とするか |
| **競合** | 誰と競争するか |
| **差別化要因** | なぜ選ばれるか |
| **参入障壁** | 模倣されにくい理由 |

### ポーターの5フォース

```
            新規参入の脅威
                 ↓
    ┌───────────────────────────┐
    │                           │
供給者の ←    業界内の競争    → 買い手の
交渉力                           交渉力
    │                           │
    └───────────────────────────┘
                 ↑
            代替品の脅威
```

### 競争戦略の3類型

| 戦略 | 説明 | 適した企業 |
|-----|------|-----------|
| **コストリーダーシップ** | 最も安く提供 | 規模の経済が働く |
| **差別化** | 独自の価値を提供 | プレミアム市場 |
| **集中** | 特定セグメントに特化 | ニッチ市場 |

## Framework 3: 人材

### 目的

優秀な人材を採用・育成・維持し、強いチームを作る。

### 採用の原則

#### 「A人材」を採用する

| 人材タイプ | 特徴 | 採用基準 |
|-----------|------|---------|
| A人材 | 自走できる、成果を出す | ✅ 積極採用 |
| B人材 | 指示があれば動ける | △ 限定的に |
| C人材 | 成果を出せない | ❌ 採用しない |

#### 採用で見るべきポイント

1. **スキルフィット** - 必要なスキルを持っているか
2. **カルチャーフィット** - 価値観が合うか
3. **成長ポテンシャル** - 学習意欲があるか
4. **リーダーシップ** - 周囲を巻き込めるか

### 組織設計

#### スタートアップの組織進化

```
シード期: ファウンダー中心（5人以下）
    ↓
アーリー期: 機能別組織（10-30人）
    ↓
グロース期: 事業部制（50-100人）
    ↓
レイター期: マトリクス組織（100人以上）
```

### カルチャー形成

- [ ] Valueを明文化している
- [ ] 採用時にカルチャーフィットを確認している
- [ ] 定期的にカルチャーを振り返っている
- [ ] 経営陣がValueを体現している

## Framework 4: プロダクト

### 目的

顧客に価値を提供する「プロダクト」を継続的に改善する。

### プロダクトマネジメントの役割

| 役割 | 責任 |
|-----|------|
| **What** | 何を作るか決める |
| **Why** | なぜ作るか説明する |
| **When** | いつ作るか優先順位付け |

### プロダクトロードマップ

```
Q1: [新機能A] - 目的: 獲得率向上
Q2: [改善B] - 目的: 継続率向上
Q3: [新機能C] - 目的: 収益向上
Q4: [インフラD] - 目的: スケーラビリティ
```

### 機能優先順位付けフレームワーク

#### ICEスコア

```
ICE = Impact × Confidence × Ease
```

| 項目 | 説明 | スケール |
|-----|------|---------|
| Impact | 効果の大きさ | 1-10 |
| Confidence | 確信度 | 1-10 |
| Ease | 実装の容易さ | 1-10 |

### プロダクト開発サイクル

```
アイデア → 仮説 → 開発 → リリース → 計測 → 学習 → 改善
                                        ↑                ↓
                                        └────────────────┘
```

## Framework 5: UX（ユーザーエクスペリエンス）

### 目的

顧客が「使いやすい」「心地よい」と感じる体験を設計する。

### UXデザインの5要素

```
       表層
        ↑
      骨格
        ↑
      構造
        ↑
      範囲
        ↑
      戦略
```

| レイヤー | 内容 |
|---------|------|
| 戦略 | ユーザーニーズ、ビジネス目標 |
| 範囲 | 機能要件、コンテンツ要件 |
| 構造 | 情報アーキテクチャ、インタラクション |
| 骨格 | ナビゲーション、UI配置 |
| 表層 | ビジュアルデザイン |

### カスタマージャーニーマップ

```
認知 → 興味 → 検討 → 購入 → 利用 → 推薦
 ↓      ↓      ↓      ↓      ↓      ↓
[タッチポイント・感情・課題を記載]
```

### UX改善のポイント

1. **ユーザビリティテスト** - 実際に使ってもらう
2. **ヒートマップ分析** - クリック・スクロール分析
3. **A/Bテスト** - 仮説を検証する
4. **NPS調査** - 満足度を定量化

## Framework 6: マーケティング

### 目的

ターゲット顧客に「認知」され、「興味」を持ってもらう。

### マーケティングファネル

```
           認知 (Awareness)
              ↓
           興味 (Interest)
              ↓
           検討 (Consideration)
              ↓
           購入 (Purchase)
              ↓
           推薦 (Advocacy)
```

### チャネル戦略

| チャネル | 特徴 | 適した段階 |
|---------|------|-----------|
| コンテンツマーケ | 長期的、信頼構築 | 認知〜興味 |
| SNS | 拡散力、双方向 | 認知〜推薦 |
| 広告 | 即効性、スケール | 認知〜検討 |
| SEO | 継続的、コスト効率 | 認知〜検討 |
| イベント | 深い関係構築 | 興味〜購入 |

### ブランディング

#### ブランドの3要素

1. **ブランドアイデンティティ** - 何者か
2. **ブランドポジショニング** - 他との違い
3. **ブランドパーソナリティ** - どんな人格か

## Framework 7: セールス

### 目的

見込み顧客を「購入」に導く。

### セールスプロセス

```
リード獲得 → 商談 → 提案 → クロージング → 契約
```

### セールスの種類

| 種類 | 説明 | 適した商材 |
|-----|------|-----------|
| **フィールドセールス** | 対面営業 | 高単価、複雑 |
| **インサイドセールス** | 電話・オンライン | 中単価 |
| **セルフサーブ** | 自己完結 | 低単価、シンプル |

### 営業指標

| 指標 | 計算式 |
|-----|-------|
| コンバージョン率 | 成約数 / 商談数 |
| 平均契約単価 | 売上 / 契約数 |
| 営業サイクル | 初回接触〜成約の日数 |

## Framework 8: カスタマーサクセス

### 目的

既存顧客の「成功」を支援し、継続利用と拡大を促す。

### カスタマーサクセスとカスタマーサポートの違い

| 項目 | カスタマーサポート | カスタマーサクセス |
|-----|------------------|------------------|
| 姿勢 | 受動的（問い合わせ対応） | 能動的（先回り） |
| 目的 | 問題解決 | 成功実現 |
| 指標 | 対応時間、解決率 | NPS、継続率、LTV |

### 顧客ライフサイクル

```
オンボーディング → 定着 → 拡大 → 更新
```

### ヘルススコア

顧客の「健康状態」を可視化:

| 指標 | 良好 | 要注意 | 危険 |
|-----|------|-------|------|
| ログイン頻度 | 毎日 | 週1 | 月1以下 |
| 機能利用率 | 80%+ | 50%+ | 30%以下 |
| NPS | 9-10 | 7-8 | 6以下 |

### チャーン（解約）対策

1. **早期警告システム** - ヘルススコア監視
2. **オンボーディング強化** - 最初の30日が重要
3. **定期的なチェックイン** - QBR（四半期ビジネスレビュー）
4. **成功事例の共有** - ベストプラクティス

## Framework 9: ファイナンス

### 目的

資金を調達・管理し、持続可能な経営を実現する。

### 資金調達のステージ

| ラウンド | 調達額目安 | 主な投資家 |
|---------|-----------|-----------|
| シード | 数百万〜数千万 | エンジェル、シードVC |
| シリーズA | 1〜5億 | VC |
| シリーズB | 5〜20億 | VC、CVC |
| シリーズC+ | 20億〜 | グロースファンド、PE |

### ユニットエコノミクス（再掲）

```
LTV = ARPU × 粗利率 × 平均継続月数
CAC = (マーケ費用 + 営業費用) / 新規顧客数

目標: LTV/CAC ≥ 3
```

### バーンレート

```
ランウェイ = 手元資金 / 月間バーンレート
```

| ランウェイ | 状態 |
|-----------|------|
| 18ヶ月以上 | 余裕あり |
| 12-18ヶ月 | 調達検討開始 |
| 6-12ヶ月 | 要注意 |
| 6ヶ月以下 | 危機的 |

### KPIダッシュボード

| カテゴリ | KPI |
|---------|-----|
| 成長 | MRR、ARR、成長率 |
| 効率 | LTV/CAC、CAC回収期間 |
| 健全性 | チャーン率、NPS |
| 財務 | バーンレート、ランウェイ |
