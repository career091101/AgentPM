# Lean Canvas (リーンキャンバス)

> **出典**: 田所雅之「起業の科学」第3章、Ash Maurya「Running Lean」
> **関連**: [[CONCEPT_CPF_001]], [[CONCEPT_PSF_001]]

---

## 1. 定義

**Lean Canvas (リーンキャンバス)** は、Ash Maury が Business Model Canvas をスタートアップ向けに改良したフレームワーク。

**1ページでビジネスモデル全体を俯瞰**できるツール。

ビジネスモデルキャンバスとの違い:
- より「課題」と「ソリューション」にフォーカス
- 「リスク」を可視化しやすい
- 素早く書き換え可能（アジャイル対応）

## 3. 各要素の詳細

### ① 課題 (Problem)

**最も重要な3つの課題**を特定する。

- 上位1つだけでも明確になっていればOK
- CPF検証で確認した課題を記載
- 顧客インタビューで得た生の声を反映

**質問**:
- 顧客が本当に困っている課題は何か?
- 3U（Unsolved/Urgent/Expensive）を満たすか?

### ② 顧客セグメント (Customer Segments)

**アーリーアダプター**を具体的に定義。

- ペルソナを記載
- 「誰でも」は禁止。具体的に絞る
- B2Bなら企業規模・業種・役職も明記

**質問**:
- 最初に狙うべき顧客は誰か?
- その顧客はどこにいるか?

### ③ 独自の価値提案 (Unique Value Proposition)

**30秒で説明できる差別化要因**。

フォーマット:
```
[ターゲット顧客] が
[解決したい課題] を解決するために
[プロダクト名] を使うと
[得られる便益] が得られます。
なぜなら [差別化要因] だからです。
```

**質問**:
- 既存の代替案とどう違うか?
- なぜ10倍良いのか?

### ④ ソリューション (Solution)

課題ごとの**最小限の機能**。

- 課題①に対する機能A
- 課題②に対する機能B
- 課題③に対する機能C

**注意**: 詳細な機能リストではなく、本質的な解決策のみ

### ⑤ チャネル (Channels)

顧客に**どうやって届けるか**。

- 獲得チャネル: SEO, 広告, SNS, イベント
- 販売チャネル: 直販, 代理店, オンライン
- 配信チャネル: Web, アプリ, 店舗

**質問**:
- 顧客はどこで情報を得ているか?
- どのチャネルが最も効率的か?

### ⑥ 収益の流れ (Revenue Streams)

**どうやって儲けるか**。

- サブスクリプション ($X/月)
- 従量課金
- 広告モデル
- マーケットプレイス手数料

**質問**:
- 顧客は何にお金を払うか?
- LTV（顧客生涯価値）はいくらか?

### ⑦ コスト構造 (Cost Structure)

**何にお金がかかるか**。

- 固定費: 人件費、オフィス、サーバー
- 変動費: マーケティング費用、CAC

**質問**:
- 主要なコストドライバーは何か?
- ユニットエコノミクス（LTV/CAC）は健全か?

### ⑧ 主要指標 (Key Metrics)

**ビジネスの健全性を測る指標**。

ステージ別:
- CPF: インタビュー数、課題共通率
- PSF: MVP登録数、コンバージョン率
- PMF: リテンション率、NPS、Sean Ellisテスト
- Scale: MRR成長率、LTV/CAC

### ⑨ 圧倒的な優位性 (Unfair Advantage)

**簡単には真似できない強み** (Moat)。

- ネットワーク効果
- データ蓄積
- ブランド
- 独自技術・特許
- チームの専門性

**注意**: 「情熱」「頑張り」は優位性ではない

## 4. 使い方

### 4.1 準備

- A4用紙またはホワイトボード
- 付箋（色分け推奨）
- チームメンバー全員参加

### 4.2 実施手順

1. **ステップ1: 課題から埋める**
   - CPF検証で得た課題を記載
   - 上位3つに絞る

2. **ステップ2: 顧客セグメント**
   - ペルソナを具体的に記載

3. **ステップ3: UVP作成**
   - 30秒で説明できるように磨く

4. **ステップ4: ソリューション**
   - 課題に対応する機能を記載

5. **ステップ5: チャネル**
   - 顧客にどう届けるか考える

6. **ステップ6-9: 残りを埋める**
   - 収益、コスト、指標、優位性

### 4.3 更新頻度

- **初期**: 毎週更新
- **CPF/PSF検証中**: 顧客インタビューごとに見直し
- **PMF後**: 四半期ごと

## 6. よくある間違い

| 間違い | 正しい使い方 |
|--------|------------|
| 全部埋めようとする | 不明な項目は空欄でOK。仮説として記載 |
| 一度書いて終わり | 継続的に更新するリビングドキュメント |
| 機能リスト化 | 本質的なソリューションのみ記載 |
| 抽象的な記述 | 具体的な数値・名称を使う |
| ソリューションから埋める | 必ず「課題」から埋める |

## 7. 事例

### 7.1 Airbnb のリーンキャンバス例

| 要素 | 内容 |
|------|------|
| 課題 | ホテルが高い、満室で泊まれない、現地感がない |
| 顧客セグメント | イベント時の旅行者、ホスト（収入欲しい人） |
| UVP | 「地元の人の家に、ホテルより安く泊まれる」 |
| ソリューション | 部屋検索、予約、レビュー、決済 |
| チャネル | SEO, イベント参加者向けマーケ, 口コミ |
| 収益 | ホスト手数料3%、ゲスト手数料6-12% |
| コスト | エンジニア、カスタマーサポート、マーケティング |
| KPI | 予約数、リピート率、レビュー数 |
| 優位性 | 両面ネットワーク効果、信頼のレビューシステム |
