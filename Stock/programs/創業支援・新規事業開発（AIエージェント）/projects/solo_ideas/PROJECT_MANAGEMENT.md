**作成日**: 2025-12-31
**プロジェクトタイプ**: Solopreneurアイデアインデックス・検証管理プロジェクト
**エージェント**: SolopreneurAgent

---

## 📋 プロジェクト概要

### 目的
個人開発者・副業起業家向けのマイクロSaaS・ツール開発アイデアを管理し、軽量な検証プロセスで実装可能性を評価する。

### 位置づけ
- **プロジェクト種別**: インデックスプロジェクト（個別アイデアは番号付きサブプロジェクトとして管理）
- **管理対象**: Solopreneur向け小規模ビジネスアイデア（目標月収1万〜10万円）
- **検証フレームワーク**: Lean Startup (軽量版) + マイクロSaaS戦略

### ideasプロジェクトとの違い

| 項目 | ideas | solo_ideas |
|------|-------|-----------|
| ターゲット | 副業者・スタートアップ | 個人開発者・Solopreneur |
| 目標規模 | 月収10万円〜 | 月収1万〜10万円 |
| 検証期間 | 3-6ヶ月 | 1-2ヶ月 |
| 初期投資 | 50万円以内 | 10万円以内 |
| チーム | 1-3人 | 1人 |

### プロジェクト憲章の必要性
**不要**
理由: このプロジェクトは個別のアイデア（001, 002, ...）を管理するインデックスであり、各アイデアが独自の憲章を持つ。全体を統括する憲章ではなく、本ドキュメント（運用ガイドライン）で管理方針を定義する。

## 🔄 アイデアのライフサイクル

### ステータス定義

| ステータス | 説明 | 達成基準 |
|-----------|------|---------|
| 🆕 Discovery | アイデア発見・検証中 | ペルソナ・課題定義完了 |
| 🟡 CPF Complete | Customer Problem Fit達成 | 5人以上の課題ヒアリング完了（軽量化） |
| ✅ PSF Complete | Problem Solution Fit達成 | プロトタイプで2人以上が価値認識（軽量化） |
| 🚀 MVP Deployed | MVP公開済み | 本番環境公開（Vercel/Heroku等） |
| 💰 First Revenue | 初収益達成 | 1円以上の売上発生 |
| 📈 Scaling | スケーリング中 | 月収1万円以上達成 |
| 📦 Archived | アーカイブ済み | 検証終了・中止 |

### フロー

```
新規アイデア
  ↓
Discovery (ペルソナ・課題定義) - 1週間
  ↓
CPF検証 (5人ヒアリング) - 2週間
  ↓
PSF検証 (プロトタイプ検証) - 2週間
  ↓
MVP開発・公開 - 4週間
  ↓
初収益 → スケーリング / Archive
```

## 🚀 新規アイデア追加プロセス

### 1. アイデア番号の採番
- 3桁の連番を採用（001, 002, ...）
- index.mdで最新番号を確認

### 2. ディレクトリ作成
```bash
cd /Users/yuichi/AIPM/aipm_v0/Stock/programs/創業支援・新規事業開発（AIエージェント）/projects/solo_ideas

# 新規ディレクトリ作成
mkdir -p 001_[idea-name]/documents/1_initiating
mkdir -p 001_[idea-name]/mvp
```

### 3. 個別憲章作成
- PMBOKトリガー「プロジェクト憲章」を使用
- 憲章パス: `001_[idea-name]/documents/1_initiating/project_charter.md`
- 必須項目:
  - プロジェクト名
  - 背景・目的
  - ターゲットペルソナ（Solopreneur特化）
  - 課題仮説
  - ソリューション仮説
  - 収益モデル（サブスク/買い切り/広告等）
  - 成功指標（目標月収、ユーザー数）

### 4. index.md更新
```markdown
| ID | 名称 | ステータス | 目標月収 | 作成日 |
|----|------|----------|---------|--------|
| 001 | [アイデア名](./001_[idea-name]/) | 🆕 Discovery | 3万円 | 2025-XX-XX |
```

### 5. SolopreneurAgentで検証開始
- Solopreneur_Researchの事例研究を参照
- 軽量な検証プロセスで実装可能性を評価
- MVP開発は1人で完結できる範囲に限定

## 📊 管理ルール

### index.mdの更新タイミング
- 新規アイデア追加時
- ステータス変更時
- 初収益達成時
- 月収目標達成時

### 個別アイデアの責任範囲
- **個別憲章**: 各アイデアディレクトリ配下で管理
- **検証ドキュメント**: documents/配下に格納
- **MVPコード**: mvp/配下に格納
- **収益データ**: documents/6_monitoring/revenue_report.md

### アーカイブ基準
- CPF検証で課題が深刻でないと判明
- PSF検証でソリューションが機能しない
- 実装が1人で完結できない（技術的難易度が高すぎる）
- MVP公開後3ヶ月で初収益に到達しない

## 💡 Solopreneur戦略

### 成功パターン
- **Solo_App事例**: 単機能ツール、週末開発、サブスク月額500円
- **Solo_SNS戦略**: Twitter/YouTube発信、オーディエンス構築
- **Solo_Newsletter**: メール配信、有料会員化

### 開発スタック推奨
- **フロントエンド**: Next.js + Vercel（無料デプロイ）
- **バックエンド**: Supabase（無料プラン）
- **決済**: Stripe（手数料3.6%）
- **認証**: Clerk / Supabase Auth

### マーケティング戦略
- **初期**: Twitter/ProductHunt/Hacker News
- **成長**: SEO、コンテンツマーケティング
- **収益化**: フリーミアム → サブスク
