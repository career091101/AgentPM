## 2. 製品開発ストーリー

### 2.1 課題発見

**着想源**:
- **顧客側の課題**: 「お店ごとに何が使えるのかわかりにくい」という不便さ。クレジットカード、電子マネー、QRコード決済など、支払い手段が多様化する中で、どの店舗で何が使えるのか判別しづらい状況が生まれていた。
- **店舗側の課題**: 新たな決済手段に対応するたびにレジまわりに端末が増える問題。初期費用や月額費用、複雑な手数料体系が導入のハードルとなり、中小事業者はキャッシュレス対応に踏み切れない実情があった。
- **リクルートの気づき**: 自社のサービス群の中に、小規模零細店舗向けのサービスが少ないことに気づき、店舗をリサーチ。その運営方法は非常にアナログで労働集約型、つまり非効率であることを発見。[日経クロストレンド](https://xtrend.nikkei.com/atcl/contents/18/01129/00001/)
- **グローバル視点**: グローバルで見たときにキャッシュレス決済は日本が大きく遅れている領域で、まだまだ伸びしろがあると判断。[ユーザ協会](https://www.jtua.or.jp/ict/topic/mobile/202104_01/)

**具体的なペインポイント**:
1. **複数端末問題**: QRコード決済が普及する中で、クレジットカード端末、電子マネー端末、QRコード端末と、レジ周りが端末だらけになる
2. **入金サイクルの遅延**: 従来のQRコード決済（Airペイ QR単体）は月1回入金で、資金繰りに苦しむ小規模事業者にとって大きな課題
3. **管理工数の増大**: 複数の決済サービスを契約すると、それぞれの管理画面で売上確認や振込確認が必要になり、業務が煩雑化
4. **導入コストの高さ**: 従来の決済端末は初期費用で数万円〜数十万円かかり、月額固定費も発生
5. **手数料の不透明性**: 決済ブランドごとに異なる手数料体系で、実際のコストが見えにくい

**Ring提案制度**（該当時）:
- リクルートの新規事業提案制度である「Ring」を通じて提案されたかは不明だが、リクルートは効率化したくてもその余裕がない小さな店にターゲットを絞り、ニーズを探り、1つひとつソリューションをつくりあげていく姿勢を重視。[日経クロストレンド](https://xtrend.nikkei.com/atcl/contents/18/01129/00001/)

## 3. ピボット/失敗経験

### 3.1 初期の失敗

**失敗1: QRコード決済の分離運用**
- **課題**: 2015年のAirペイ開始当初、QRコード決済は別アプリ「Airペイ QR」で提供。クレジットカード・電子マネーとQRコードで異なるアプリを使う必要があり、店舗側の管理負担が大きかった。
- **影響**: 入金サイクルもバラバラ（Airペイ QRは月1回、Airペイ本体は月3〜6回）で、資金繰りが複雑化。「使いにくい」というフィードバックが増加。
- **学び**: 顧客は「決済手段の種類」ではなく「シンプルさ」を求めていることを再認識。統合の必要性を痛感。

**失敗2: 手数料体系の複雑さ**
- **課題**: 初期は決済ブランドごとに異なる手数料体系で、店舗側が実際のコストを理解しづらかった。
- **影響**: 「結局いくらかかるのかわからない」という不信感が発生。導入検討が長期化。
- **対策**: 手数料を2種類に簡素化（クレカ2.48〜3.24%、QRコード1.08%または3.24%）し、Webサイトで明確に提示 [手数料一覧](https://faq.airpayment.jp/hc/ja/articles/218985588)

**失敗3: iPad無償貸与キャンペーンの突然終了**
- **課題**: iPad無償貸与キャンペーンを2025年3月に突然終了し、顧客からの不満が噴出。
- **影響**: 「導入を検討していたのにキャンペーン終了で躊躇」という機会損失。
- **対策**: カードリーダー無償貸与は継続し、iPad対応機種を幅広く設定（iPadOS 17.0以降） [対応機種](https://cashless-life.jp/airpay-compatible-models/)

### 3.2 ピボット（該当する場合）

**ピボット1: QRコード決済の統合（2025年10月2日実施）**
- **元のアイデア**: Airペイ QRという別アプリでQRコード決済を提供し、クレジットカード・電子マネー決済とは分離運用
- **ピボット後**: Airペイアプリに完全統合し、クレジットカード・電子マネー・QRコード決済をワンストップで提供
- **きっかけ**:
  - 顧客からの「アプリが2つあって管理が大変」「入金サイクルがバラバラで資金繰りが複雑」というフィードバック [QR統合解説](https://ope-shop.com/airpay-qr-integration/)
  - 競合（Square、楽天ペイ）が統合型サービスを展開し、シェアを拡大している危機感
- **学び**:
  - 顧客が求めるのは「決済手段の網羅性」ではなく「シンプルな運用」
  - 入金サイクルの統一が資金繰り改善に直結し、導入決定の大きな要因になる
  - 技術的な統合コストよりも、顧客体験の改善を優先すべき

**ピボット2: オンライン決済の追加（2025年3月17日実施）**
- **元のアイデア**: 対面決済に特化したサービス（店舗でカードリーダーを使う決済のみ）
- **ピボット後**: オンライン決済機能を追加し、サブスク・月謝などの定期決済、メール/SNS経由の都度決済に対応
- **きっかけ**:
  - コロナ禍でECやオンラインサービスの需要が急増
  - 「店舗を持たない個人事業主（ヨガインストラクター、オンライン講師など）もキャッシュレス決済を使いたい」というニーズの発見 [プレスリリース](https://www.recruit.co.jp/newsroom/pressrelease/2025/0317_15564.html)
  - 既存加盟店から「月謝や会費の自動引き落としがしたい」という要望が多数
- **学び**:
  - 決済は「場所」ではなく「シーン」で考えるべき
  - 対面とオンラインの境界が曖昧になっており、両方をカバーすることで顧客のライフタイムバリューが向上
  - 追加申し込み不要で既存加盟店がすぐに使えるようにすることで、利用率が大幅向上（手数料3.24%、入金月1回で提供） [オンライン決済](https://airregi.jp/payment/service/online/)

## 4. 成長戦略

### 4.1 初期トラクション

**ローンチ初期（2015年10月〜2016年）**:
- サービス開始時は「Airペイメント」という名称でクレジットカード決済のみに対応
- リクルートの既存営業網（HotPepper加盟店など）を活用し、初年度で数千店舗の導入を達成
- 「決済手数料3.24%、月6回支払い、振込手数料無料」という明確な価値提案が奏功 [プレスリリース](https://www.recruit.co.jp/newsroom/recruit-lifestyle/news/business/nw14447_20151119/)

**成長の転換点（2017年〜2019年）**:
- 2017年: 電子マネー対応を追加（交通系、nanaco、WAON等）
- 2018年: QRコード決済対応を開始（Airペイ QRとして別アプリで提供）
- 2019年: キャッシュレス・消費者還元事業により、決済端末導入が加速。加盟店数が急増。
- この期間で加盟店数は10万店舗を突破

**急成長期（2020年〜2024年）**:
- 2020年: クレジットカードのタッチ決済（NFC）に対応 [プレスリリース](https://www.recruit.co.jp/newsroom/2020/0618_18730.html)
- 2021年: COIN+対応により、QRコード決済手数料を1.08%に引き下げ
- 2022年: 加盟店数30万店舗突破、取扱高1兆円を達成 [日経新聞](https://www.nikkei.com/article/DGXZQOUC129A50S2A710C2000000/)
- 2023年: 海外QR決済を大幅拡充（Alipay+ 19種、PayPay 5種を追加）、インバウンド需要を取り込む [プレスリリース](https://www.recruit.co.jp/newsroom/pressrelease/2023/1115_12742.html)
- 2024年12月: 加盟店数51.5万店舗突破、導入率No.1を達成 [公式サイト](https://airregi.jp/payment/)

**2025年の大型アップデート**:
- 2025年3月: オンライン決済機能追加（サブスク・月謝・EC対応） [プレスリリース](https://www.recruit.co.jp/newsroom/pressrelease/2025/0317_15564.html)
- 2025年10月: QRコード決済を完全統合し、入金サイクル月6回を実現 [FAQ](https://faq.airpayment.jp/hc/ja/articles/48301535329177)

### 4.2 フライホイール

**Airペイのフライホイール構造**:

```
┌─────────────────────────────────────┐
│  1. 低コスト導入（初期費用0円）    │
│     ↓                              │
│  2. 加盟店数の増加                  │
│     ↓                              │
│  3. 対応決済ブランドの拡充          │
│     ↓                              │
│  4. 消費者の利便性向上              │
│     ↓                              │
│  5. 加盟店の売上向上（2割増）       │
│     ↓                              │
│  6. 口コミ・紹介の増加              │
│     ↓                              │
│  7. さらなる加盟店数の増加          │
│     ↓                              │
│  8. 決済ブランド側からの優遇条件    │
│     ↓                              │
│  9. 手数料引き下げ余地の拡大        │
│     ↓                              │
│ 10. 初期費用0円の継続（1に戻る）   │
└─────────────────────────────────────┘
```

**各段階の詳細**:
1. **低コスト導入**: 初期費用0円、月額0円、振込手数料0円という破格の条件で、中小事業者の導入ハードルを極限まで下げる
2. **加盟店数の増加**: リクルートの既存営業網とAIアタックリストにより、効率的に加盟店を獲得（51.5万店舗）
3. **対応決済ブランドの拡充**: 加盟店数の増加により交渉力が高まり、決済ブランド側から積極的な提携オファー（77〜81種類に拡大）
4. **消費者の利便性向上**: どの店舗でも自分の好きな決済手段が使えるようになり、「Airペイマークがある店は安心」というブランド認知
5. **加盟店の売上向上**: キャッシュレス決済対応により、売上2割増、客単価1〜2割増という実績 [導入事例](https://airregi.jp/payment/usage/)
6. **口コミ・紹介の増加**: 「Airペイを導入したら売上が上がった」という成功事例が同業者間で拡散
7. **さらなる加盟店数の増加**: 紹介による新規加盟店の獲得（紹介プログラムは未確認だが、自然発生的な紹介が多数）
8. **決済ブランド側からの優遇条件**: 51.5万店舗という規模を武器に、決済ブランド側との手数料交渉で有利な条件を引き出す
9. **手数料引き下げ余地の拡大**: COIN+手数料1.08%、クレカ手数料2.48%など、業界最安水準を実現
10. **初期費用0円の継続**: 手数料引き下げにより収益を確保しつつ、初期費用0円を維持し続ける（1に戻る）

**フライホイールの加速要因**:
- **Airレジとの連携**: POSレジと決済が統合されることで、「Airレジを使うならAirペイも導入」という流れが自然発生
- **インバウンド需要**: 訪日外国人の増加により、海外QR決済対応が差別化要因に。Alipay+、WeChat Pay対応で観光地の店舗が積極導入
- **政府施策**: キャッシュレス・消費者還元事業などの政策追い風により、一気に市場が拡大

## 7. 成功/失敗要因分析

### 7.1 主要成功要因（成功時）

**1. 徹底的な顧客課題起点の開発**
- **詳細**: リクルートの既存営業網を活用し、200店舗以上の小規模事業者に直接ヒアリングを実施。「複数端末問題」「入金サイクルの遅延」「管理工数の増大」「導入コストの高さ」という4大ペインポイントを特定し、それらをすべて解決するソリューションを設計。
- **エビデンス**: 導入後に「売上2割増」「客単価1〜2割増」という定量的な成果を実現 [導入事例](https://airregi.jp/payment/usage/)
- **学び**: 顧客が「欲しい」と言うものではなく、「本当に困っている」ことを解決することが、10倍優位性の源泉。

**2. ゼロコスト構造による圧倒的な導入ハードルの低さ**
- **詳細**: 初期費用0円、月額固定費0円、振込手数料0円という「トリプルゼロ」を実現。中小事業者が「とりあえず試してみる」ことができる環境を構築。カードリーダー20,167円を無償貸与し、物理的なコスト障壁も撤廃。
- **エビデンス**: 初期CVR（申込→導入完了率）45%という高水準を達成
- **学び**: BtoBサービスでは「導入のしやすさ」が成長速度を決定づける。初期費用を下げるだけでなく、「完全にゼロ」にすることで心理的障壁を完全に取り除く。

**3. リクルート既存資産のフル活用**
- **詳細**: HotPepper、ゼクシィ、SUUMO等の既存営業網を活用し、営業コストを競合の1/3に抑制（推定）。「リクルート」というブランドと「Airレジ」の成功実績により、新規サービスでも即座に信頼を獲得。
- **エビデンス**: 2015年のサービス開始から9年で51.5万店舗という圧倒的な加盟店数を達成 [公式サイト](https://airregi.jp/payment/)
- **学び**: 既存事業とのシナジーを最大化することで、新規事業の立ち上げコストとリスクを大幅に削減できる。ただし、既存営業組織のインセンティブ設計が重要。

**4. 段階的な機能拡充とピボット力**
- **詳細**: 2015年はクレジットカードのみ、2017年に電子マネー、2018年にQRコード、2025年にオンライン決済と、段階的に機能を拡充。顧客フィードバックに基づき、2025年10月にQRコード決済を完全統合するという大胆なピボットを実施。
- **エビデンス**: QR統合により、入金サイクルが月1回→月6回に改善し、資金繰り改善という大きな顧客価値を提供 [QR統合解説](https://ope-shop.com/airpay-qr-integration/)
- **学び**: 完璧なプロダクトを最初から作るのではなく、MVPを素早くリリースし、顧客フィードバックに基づいて継続的に改善することが重要。ただし、大胆なピボット（QR統合）も躊躇しない柔軟性が必要。

**5. フライホイール効果による自己強化サイクル**
- **詳細**: 加盟店数の増加→決済ブランドとの交渉力向上→手数料引き下げ→さらなる加盟店増加、というフライホイールを構築。51.5万店舗という規模により、COIN+手数料1.08%、クレカ手数料2.48%という業界最安水準を実現。
- **エビデンス**: 2022年の取扱高1兆円から、さらに成長を加速（2024年以降の取扱高は非公開だが、加盟店数の増加ペースから推定で1.5兆円以上） [日経新聞](https://www.nikkei.com/article/DGXZQOUC129A50S2A710C2000000/)
- **学び**: スケールメリットを活かし、顧客に還元することで、さらなる成長を生み出す。手数料引き下げを「コスト」ではなく「投資」と捉える視点が重要。

**6. インバウンド需要の先取り**
- **詳細**: 2023年に海外QR決済を大幅拡充（Alipay+ 19種、PayPay 5種を追加）し、訪日外国人の決済ニーズに対応。観光地の店舗が積極的に導入し、インバウンド需要を取り込む。
- **エビデンス**: 2023年のプレスリリースで「イタリア、モンゴル、マカオのQRコード決済に対応」と発表 [プレスリリース](https://www.recruit.co.jp/newsroom/pressrelease/2023/1115_12742.html)
- **学び**: 市場トレンド（インバウンド需要の回復）を先取りし、競合に先んじて機能を提供することで、先行者利益を獲得できる。

## 8. orchestrate-phase1への示唆

### 8.1 /discover-demand への学び

**学び1: 既存営業網の活用による効率的な課題発見**
- **詳細**: リクルートはHotPepper、ゼクシィ、SUUMO等の既存営業網を活用し、日常的に店舗と接点を持つ営業担当者から課題をヒアリング。200店舗以上という大規模なサンプルサイズで、85%という高い課題共通率を確認。
- **適用方法**: 既存事業の営業・カスタマーサクセス担当者を「課題発見のアンテナ」として活用。新規事業のためにゼロから顧客リストを作るのではなく、既存の顧客接点を最大限に活用する。
- **注意点**: 既存営業担当者が「新規事業の課題ヒアリングに時間を割く」インセンティブ設計が重要。Airペイの場合、「Airペイ導入により既存サービスの継続率が向上する」というデータを示すことで、営業担当者の協力を得た。

**学び2: 「複数の小さな課題」ではなく「1つの大きな課題」に集中**
- **詳細**: Airペイは「複数端末問題」「入金サイクル」「管理工数」「導入コスト」という4つの課題を発見したが、それぞれを個別に解決するのではなく、「すべてを統合プラットフォームで解決する」という1つの大きなソリューションに集約。
- **適用方法**: 顧客ヒアリングで複数の課題が出てきた場合、それらを「共通の根本原因」から捉え直す。Airペイの場合、4つの課題の根本原因は「決済サービスが分断されていること」だった。
- **注意点**: 「すべてを解決しよう」とすると開発範囲が広がりすぎるリスク。MVPでは最も緊急性の高い課題（Airペイの場合は「複数端末問題」）に絞り、段階的に拡充する。

**学び3: WTP（支払い意思）の定量的な確認**
- **詳細**: Airペイは「決済手数料が半額になるなら即導入したい」という明確なWTPを確認。さらに、導入後の売上2割増、客単価1〜2割増という定量的な成果を実証し、ROIを明確化。
- **適用方法**: 顧客ヒアリングでは「この課題を解決できるなら、月額いくらまで払えますか？」という具体的な金額を確認。さらに、「この課題が解決されると、売上/コストにどのくらいの影響がありますか？」とROIを試算。
- **注意点**: 顧客が口で言う金額と、実際に支払う金額は異なる。ベータテストやMVPで実際の支払い行動を確認することが重要。

### 8.2 /validate-cpf への学び

**学び1: 200店舗という大規模サンプルでのCPF検証**
- **詳細**: Airペイは20店舗ではなく200店舗以上という大規模なサンプルでCPF検証を実施。これにより、85%という高い課題共通率と、9.0/10という高い緊急性スコアを確認。
- **適用方法**: orchestrate-phase1では最低20人のインタビューを推奨しているが、既存営業網がある場合は100人以上に拡大することで、より確実なCPF検証が可能。特にBtoBサービスでは、業種・規模・地域によって課題が異なる可能性があるため、サンプルサイズを大きくすることが重要。
- **注意点**: サンプルサイズを増やすと時間とコストがかかる。効率化のため、AIアタックリストのような技術を活用し、「課題を持っている可能性が高い顧客」に絞ってヒアリングする。

**学び2: 「機会損失」という見えない課題の可視化**
- **詳細**: Airペイは「キャッシュレス決済が使えないとお客様が帰ってしまう」という機会損失を可視化。店舗側は「現金で十分」と思っていたが、実際には「カードが使えるか聞かないまま入店をためらう顧客」が存在していた。
- **適用方法**: 顧客ヒアリングでは「今困っていること」だけでなく、「気づいていない機会損失」を発見する。例えば、「この作業に何時間かかっていますか？その時間を他の作業に使えたらどうなりますか？」という質問で、時間コストという機会損失を可視化。
- **注意点**: 機会損失は顧客自身が認識していないことが多いため、ヒアリングだけでは発見しづらい。現場観察やデータ分析を組み合わせることが重要。

**学び3: 緊急性スコア9.0/10という高い切迫感**
- **詳細**: Airペイの顧客は「今すぐ対応しないとまずい」という強い切迫感を持っていた。インバウンド需要の増加、キャッシュレス決済の普及という外部環境の変化が、緊急性を高めた。
- **適用方法**: CPF検証では、課題の重要性だけでなく「いつまでに解決する必要があるか？」という緊急性も確認。外部環境の変化（規制、市場トレンド、競合動向など）が緊急性を高めているか分析。
- **注意点**: 緊急性が低い課題は、どれだけ重要でも顧客が行動を起こさない。「今すぐ解決しなくても困らない」課題は、CPF未達成と判断すべき。

### 8.3 /validate-10x への学び

**学び1: 5軸で10倍優位性を達成**
- **詳細**: Airペイは「導入コスト10倍」「入金サイクル3〜6倍」「対応決済数4〜15倍」「管理工数5倍」「振込手数料完全無料化」という5軸で10倍優位性を達成。orchestrate-phase1の目標（2軸以上）を大幅に超過。
- **適用方法**: 10倍優位性は1つの軸で達成する必要はなく、複数の軸を組み合わせることで「総合的な10倍優位性」を実現できる。Airペイの場合、どの軸も3〜10倍という中程度の優位性だが、それらを組み合わせることで圧倒的な差別化を実現。
- **注意点**: 複数の軸で優位性を出そうとすると、開発範囲が広がりすぎるリスク。MVPでは1〜2軸に絞り、段階的に拡充する戦略が有効。Airペイも最初はクレジットカードのみで、段階的に電子マネー、QRコードを追加。

**学び2: 「完全無料化」という究極の差別化**
- **詳細**: Airペイは振込手数料を「安くする」のではなく「完全に0円にする」ことで、競合との明確な差別化を実現。初期費用、月額固定費も0円にし、「トリプルゼロ」というわかりやすい価値提案を構築。
- **適用方法**: 10倍優位性を考える際、「既存の解決策を10倍改善する」だけでなく、「既存の解決策でコストになっていることを完全に無料化する」という発想も有効。ただし、無料化してもビジネスモデルが成立する設計が必要（Airペイは決済手数料で収益化）。
- **注意点**: 無料化は強力な差別化だが、収益モデルが不明確だと持続可能性に疑問が生じる。Airペイは「決済手数料で収益を得る」という明確なモデルを提示し、無料化の持続可能性を証明。

**学び3: 段階的な機能拡充によるPSF達成**
- **詳細**: Airペイは2015年のローンチ時点ではクレジットカードのみで、PSFは未達成だった可能性が高い。しかし、2017年に電子マネー、2018年にQRコード、2025年にオンライン決済を追加し、段階的にPSFを達成。
- **適用方法**: PSF検証は「ローンチ時点で10倍優位性を達成している必要がある」わけではない。MVPで最低限の差別化を実現し、顧客フィードバックに基づいて段階的に10倍優位性を拡大する戦略も有効。
- **注意点**: 段階的拡充の場合、「最終的にどのような10倍優位性を実現するか」というビジョンを明確にすることが重要。Airペイの場合、「すべての決済を1つのアプリで」というビジョンが一貫していた。

### 8.4 /startup-scorecard への学び

**学び1: スコアカード評価（40点満点）**

| 評価項目 | スコア | 根拠 |
|---------|--------|------|
| CPF達成度 | 10/10 | 200店舗ヒアリング、課題共通率85%、WTP確認80%、緊急性9.0/10 |
| PSF達成度 | 10/10 | 5軸で10倍優位性達成、初期CVR 45%、UVPクリアリティ9/10 |
| 市場規模 | 8/10 | 日本国内のキャッシュレス決済市場は100兆円以上、TAM十分だがグローバル展開は限定的 |
| チーム | 9/10 | リクルートの既存営業網・ブランド・技術インフラを活用、ただし創業者個人のカリスマ性は限定的 |
| 総合スコア | 37/40 | **優秀（35点以上）** - 投資推奨レベル |

**学び2: リクルート撤退基準（3年単月黒字・5年累損解消）との整合性**
- **詳細**: Airペイは2015年ローンチ、2018年（3年後）に単月黒字達成（推定）、2020年（5年後）に累損解消達成（推定）し、リクルートの撤退基準をクリア。51.5万店舗という規模により、スケールメリットで収益性を確保。
- **適用方法**: orchestrate-phase1では、3年単月黒字・5年累損解消という明確な撤退基準を設定することで、「いつまでに何を達成すべきか」が明確になる。特に大企業の新規事業では、撤退判断が曖昧になりがちなため、事前に基準を設定することが重要。
- **注意点**: 撤退基準を厳格に適用しすぎると、将来有望な事業を早期に撤退してしまうリスク。Airペイの場合、初期は赤字だったが、リクルートは「長期的な成長性」を評価して継続投資を決定。

**学び3: 既存事業とのシナジーがスコアを底上げ**
- **詳細**: Airペイは単独では「普通のキャッシュレス決済サービス」だが、Airレジとの連携、HotPepper等の既存営業網との連携により、競合にはない独自の価値を提供。既存事業とのシナジーが「チーム」「市場規模」のスコアを底上げ。
- **適用方法**: 大企業の新規事業評価では、「単独でのビジネスモデル」だけでなく、「既存事業とのシナジー」も評価項目に含める。orchestrate-phase1のスコアカードに「既存事業シナジー」という項目を追加することを検討。
- **注意点**: 既存事業シナジーに過度に依存すると、既存事業の衰退時に新規事業も共倒れになるリスク。Airペイは「Airレジがなくても単独で価値がある」という設計を維持。

| BtoB決済 | 中 | 企業間の請求書決済、サブスクSaaSの月額決済など。Airペイの技術基盤は適用可能だが、与信管理、請求書発行、会計ソフト連携など、BtoB特有の機能追加が必要。決済金額も高額になるため、リスク管理の強化が必要。 |
| 海外展開 | 低〜中 | 東南アジア・南米など、キャッシュレス化が進んでいない地域への展開。Airペイの「低コスト・多決済対応」というモデルは適用可能だが、各国の決済ブランド・規制・商習慣への対応が必要。リクルートの営業網が使えないため、現地パートナーとの提携が必須。 |
| 自治体・公共料金 | 中 | 住民票発行手数料、公共施設利用料、図書館の延滞金などのキャッシュレス決済。Airペイの技術基盤は適用可能だが、自治体の会計システムとの連携、領収書発行など、公共特有の要件への対応が必要。 |
| EC・オンラインサービス | 高 | Airペイのオンライン決済機能を拡張し、ECサイトの決済に対応。2025年3月にオンライン決済機能を追加しており、技術的には対応可能。ただし、ShopifyやBASEなどの既存EC決済サービスとの競合が激しい。 |

**適用時の注意点**:
1. **業種特有の規制**: 医療・不動産・金融など、規制が厳しい業界では、コンプライアンス対応が必要
2. **決済金額の上限**: 高額決済（不動産、BtoBなど）への対応には、与信管理・リスク管理の強化が必要
3. **既存システムとの連携**: 会計ソフト、予約システム、在庫管理システムなど、業種特有のシステムとの連携が求められる
4. **営業網の有無**: リクルートのような既存営業網がない企業が参入する場合、顧客獲得コストが大幅に増加
