## 3. エージェント実行の詳細仕様

### 3.1 各エージェントのタスク構造

**Input**:
1. 既存Research Note v2ファイル（research_notes_68/）
2. v3テンプレート（_templates/corporate_product_research_v3_template.md）

4. 品質スコアリング基準（_templates/quality_scoring_criteria.md）

**Process**:
1. **既存データ読み込み** (5分)
   - Research Note v2の10セクション読み込み
   - 既存ソース10-11件の確認

2. **追加調査** (45分)
   - リクルート公式サイト・プレスリリース（Tier 1）
   - IR資料・決算説明会資料（Tier 2）
   - 第三者メディア・比較記事（Tier 3）
   - Wayback Machine（Withdrawn製品、Tier 5）
   - 追加ソース5-7件収集、合計15-17件

3. **YAML作成** (20分)
   - 基本情報（14フィールド）
   - M&A情報（6フィールド、該当時）
   - 撤退情報（9フィールド、該当時）
   - orchestrate-phase1検証（12フィールド）
     - CPF: interview_count（推定）、problem_commonality（推定）、wtp_confirmed、urgency_score
     - PSF: ten_x_axes（2-5軸、multiplier定量化）、mvp_type、competitive_advantage
     - Pivot: occurred、pivot_count、pivot_trigger
   - リクルート資産（3フィールド）
   - クロスリファレンス（3フィールド）
   - 品質管理（3フィールド）

4. **コンテンツ作成** (25分)
   - セクション1: 基本情報（既存活用）
   - セクション2: 製品開発ストーリー（CPF/PSF検証含む）
   - セクション3: ピボット/失敗経験
   - セクション4: 成長戦略（リクルート資産活用）
   - セクション5: M&A統合戦略（該当時）

- 品質スコア: 70-100点
- 行数: 450-600行

### 3.2 推定値の適用ルール

**CPF.interview_count**:
- B2B SaaS（Airシリーズ等）: 20-30
- B2C（ホットペッパー等）: 50-100
- 既存事業派生（SUUMO等）: 10-20
- **根拠**: 業界標準値、リクルートの製品開発プロセスから推定

**CPF.problem_commonality**:
- B2B生産性向上（Airレジ等）: 70-80%
- B2Cマッチング（ゼクシィ等）: 60-70%
- ニッチB2B（サロンボード等）: 50-60%
- **根拠**: 市場調査データ、競合状況から推定

**PSF.ten_x_axes.multiplier**:
- コスト破壊: 10-100倍（例: Airペイ初期費用10万円→0円）
- 時間短縮: 5-15倍（例: SUUMO物件検索、手作業→即座）
- アクセス革命: 3-10倍（例: スタディサプリ、全国どこでも受講可能）
- **根拠**: 競合サービスとの定量比較

**Unit Economics**:
- LTV/CAC比率: 3-7倍（業界平均）
- Churn Rate: 15-30%/年（SaaS業界平均）
- NRR: 100-120%（成功SaaSの基準）
- **根拠**: SaaS業界ベンチマーク、IR資料の断片的情報

**推定値の明記**:
```yaml
cpf:
  interview_count: 30  # 推定値（B2B SaaS業界標準）
  problem_commonality: 75  # 推定値（市場調査データから類推）
```

```markdown
### 2.2 CPF検証（orchestrate-phase1基準）
- **Interview Count**: 推定30店舗（B2B SaaS製品の業界標準値、Airレジユーザーヒアリングから類推）
- **Problem Commonality**: 推定75%（中小店舗の決済手段不足は共通課題、市場調査から類推）
```

### 3.3 データ取得困難時のフォールバック

**Tier 1-3ソース取得困難** → **Tier 5活用**:
1. **Wayback Machine**: Withdrawn製品の公式サイト復元
2. **ユーザーレビュー**: 価格.com、ITreview、App Store等
3. **業界レポート**: 矢野経済研究所、MM総研、Gartner

**それでもデータ不足** → **保守的推定**:
- 推定値を業界標準の下限値で設定
- 品質スコアを保守的評価（Tier B: 70-79点）
- ファクトチェック表で「推定」明記
