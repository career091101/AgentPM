## 1. プロジェクト概要

### 1.1 プロジェクト名
Founder_Agent_ForRecruit（採用特化型創業支援AIエージェント）

### 1.2 プロジェクトの目的
リクルートグループなど大企業内での新規事業立ち上げに特化した、企業内起業家（イントレプレナー）向けAI支援システムの構築。リクルート製品31件（成功16製品、撤退15製品）の実証分析に基づき、CPF/PSF検証フレームワークを実装し、社内承認プロセス最適化とRing制度準拠を実現する。

### 1.3 ビジネス価値
- **企業内新規事業成功率の向上**: CPF検証スコア50%以上、PSF検証で1軸以上の10倍優位性確保により、成功確率を従来比3倍向上
- **社内承認プロセスの効率化**: Ring制度準拠チェック機能により、承認フロー期間を30-50%短縮
- **失敗コストの削減**: 早期撤退基準（1.5-6年）の明確化により、無駄なリソース投入を60%削減
- **知見の体系化**: リクルート31製品分析から抽出した成功パターン・失敗要因を新規事業立ち上げに適用可能

## 2. プロジェクトの背景

### 2.1 市場機会
- 大企業における新規事業立ち上げの成功率は約10-15%と極めて低い
- リクルートRing制度など社内起業制度は増加傾向だが、体系的な支援フレームワークが不足
- 既存の創業支援ツールは外部スタートアップ向けが中心で、企業内新規事業の特殊性（社内リソース活用、承認プロセス、既存事業とのシナジー等）に対応していない

### 2.2 課題認識
- **CPF検証不足**: 市場ニーズの過大評価により5製品が撤退（リクルートDMPフォロー、じゃらんナビ札幌等）
- **PSF構築の弱さ**: 競合優位性欠如により5製品が撤退（エリクラ、CODE.SCORE等）
- **撤退判断の遅れ**: 6年間実証実験レベル継続（エリクラ）など、早期撤退判断の基準不明確
- **社内リソース活用不足**: リクルート資産3種以上活用製品は成功率100%だが、活用方法が体系化されていない

### 2.3 プロジェクト発足の経緯
- 2025年12月: リクルート製品31件のCPF/PSF検証パターン分析完了
- 分析成果: 統合分析レポート、PSFパターン6種、CPFパターン6種、撤退理由分類、品質レポート複数
- 次フェーズ: 分析知見をAIエージェントとして実装し、実用化を目指す

## 3. スコープ

### 3.1 含む（In-Scope）

#### 3.1.1 主要機能
1. **CPF検証支援**
   - User Research計画（目標30回以上）
   - Problem Commonality測定（目標70%以上）
   - 既存顧客基盤活用戦略
   - 自社先行導入計画

2. **PSF検証支援**
   - 10倍優位性分析（最低1軸）
   - Competitive Advantage Axes評価（目標3軸以上）
   - リクルート資産活用戦略（営業網、データ資産、ブランド等）
   - エコシステム連携設計

3. **社内承認支援**
   - ステークホルダー分析・合意形成戦略
   - 社内リソース棚卸し（顧客基盤、技術、ブランド）
   - 承認プレゼンテーション作成
   - Ring制度準拠チェック（Ring 1-3対応）

4. **撤退判断支援**
   - 早期撤退基準評価（1.5-6年）
   - Unit Economics健全性チェック（LTV/CAC比5倍以上）
   - 市場変化モニタリング
   - ピボット判断支援

#### 3.1.2 分析データベース
- リクルート製品31件の詳細分析データ（272ファイル）
- PSFパターン6種（コスト破壊型、時間革命型、アクセス革命型、情報革命型、更新速度革命型、マッチング精度革命型）
- CPFパターン6種（アナログ業務非効率性、情報非対称性解消、地域経済格差解消、マッチング効率革命、時間場所制約解放、意思決定支援）
- 撤退理由分類（5類型）
- 品質レポート（品質スコア平均88.5点）

### 3.2 含まない（Out-of-Scope）

- 外部スタートアップ向け資金調達支援（ForStartup版に分離）
- VC向けピッチデッキ作成（ForStartup版に分離）
- 実際の製品開発・実装支援（Executing/Monitoring フェーズで別途計画）
- リクルート以外の企業事例分析（将来的な拡張予定）
- 海外市場特化機能（現時点では国内企業内新規事業に限定）

## 6. 前提条件

### 6.1 技術的前提
- Founder Agent Originのコア機能が稼働していること
- 既存分析データ（272ファイル）がアクセス可能であること
- LLM API（Claude/ChatGPT等）が利用可能であること

### 6.2 組織的前提
- リクルートRing制度の仕様が公開されていること
- 社内承認プロセスの標準フローが明文化されていること
- イントレプレナーへのアクセス（ユーザー調査、ベータテスト）が可能であること

### 6.3 データ前提
- リクルート製品31件の分析データが最新（2025年12月時点）であること
- CPF/PSFパターン分析が品質スコア85点以上であること
- 撤退理由分類が5類型で網羅的であること
