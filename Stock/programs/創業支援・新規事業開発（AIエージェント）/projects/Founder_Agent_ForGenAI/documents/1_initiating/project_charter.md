# プロジェクト憲章

**プロジェクト名**: Founder Agent - ForGenAI（生成AI特化版）

## 1. ビジョン

**生成AI技術を活用したプロダクト開発において、最新技術トレンドへの即時対応とProduct Hunt戦略により、AIツール市場でトップランクのプロダクトを創出する。**

## 2. 背景と目的

### 2.1 市場背景

2025年は「**生成AI実用化元年**」と呼ばれ、ChatGPT、Claude、Gemini等のAIモデルが企業・個人に広く浸透した年である。

| 指標       | 数値                 |
| ---------- | -------------------- |
| AI市場規模予測 | $470億超（CAGR 44%） |
| AI Tools数 | 30,000+（there's an AI for that） |

**トレンド**: 単なるChatGPT Wrapperから、高度なプロンプトエンジニアリング、RAG（検索拡張生成）、Agentic AI（自律型AIエージェント）を活用した差別化プロダクトへのシフトが進行中。

### 2.2 技術的意義

本プロジェクトは、以下の最新AI技術トレンドを実装・検証する：

| トレンド | 概要 | 活用方法 |
|---------|------|---------|
| **Agentic AI** | 自律的に行動するAIエージェント | AutoGPT, CrewAI活用 |
| **RAG** | 検索拡張生成 | ベクトルDB + LLM統合 |
| **Multimodal** | テキスト+画像+音声 | GPT-4o Vision, DALL-E 3 |
| **Fine-tuning** | カスタムモデル作成 | OpenAI Fine-tuning API |
| **Local LLM** | オンプレミス実行 | Llama 3, Mistral |

### 2.3 Product Hunt戦略の重要性

AIツール市場は競争が激しく、Product Huntでの成功が初期ユーザー獲得の鍵となる。

**成功基準**:
- #1 Product of the Day → トラフィック 10,000+
- #1 Product of the Week → 生涯顧客 100+
- 獲得メールアドレス 500+

### 2.4 プロジェクト目的

本プロジェクトでは、生成AI技術を活用したプロダクト開発において、以下を実現する：

| # | 目的 | 説明 |
| - | ---- | ---- |
| 1 | **AI技術スタック選定** | OpenAI vs Anthropic vs Gemini vs OSSの最適選択 |
| 2 | **プロンプトエンジニアリング標準化** | 再現性90%以上のプロンプト設計 |
| 3 | **Product Hunt戦略実行** | #1 Product of the Day獲得 |
| 4 | **API依存リスク評価** | 代替API・ローカルLLMへの移行可能性評価 |
| 5 | **レスポンス速度最適化** | 3秒以内のレスポンス（ストリーミング必須） |
| 6 | **AI競合分析** | there's an AI for that、AI Scout等での競合調査 |
| 7 | **技術トレンド即時反映** | 月次モデル更新対応（GPT-4o, Claude 3.7等） |

>
> - 最新AIモデル（GPT-4o, Claude 3.7 Sonnet等）への即時対応必須
> - Product Hunt戦略は必須（AIカテゴリは競争激しい）

## 4. スコープ

### 含む（In Scope）

#### AI技術スタック
- **Frontend**: Next.js 15 + TypeScript + Tailwind CSS + shadcn/ui
- **Backend**: Vercel Serverless Functions + tRPC
- **AI API**: OpenAI GPT-4o（プライマリ）+ Anthropic Claude 3.7 Sonnet（代替）+ Google Gemini Pro（フォールバック）
- **Database**: Supabase（PostgreSQL）+ pgvector（ベクトル検索）
- **Streaming**: Server-Sent Events (SSE) + Vercel AI SDK
- **Cache**: Redis / Upstash + Next.js Cache
- **Auth**: Clerk / Supabase Auth
- **Payment**: Stripe
- **Analytics**: Posthog / Mixpanel

#### プロンプトエンジニアリング標準
- Chain-of-Thought（複雑な推論タスク）
- Few-shot Learning（特定フォーマット出力）
- ReAct (Reason + Act)（エージェント的タスク）

#### Product Hunt戦略
- 1週間前準備（Hunter確保、コミュニティ参加、ティザーページ）
- ローンチ日（火曜-木曜 00:01 PST）
- 最初の6時間でUpvote 200+獲得施策

#### AI競合分析
- there's an AI for that
- AI Scout
- Product Hunt AI Category
- Futurepedia

### 含まない（Out of Scope）

- 物理的な製品・サービス
- 非AI技術によるプロダクト
- B2B SaaS（ForGenAIはB2C/SMB向け）
- 高可用性・SLA保証（MVP段階）

## 9. ガバナンス・倫理原則

### 9.1 責任体制

- **最終責任者**: Yuichi（スポンサー）
- **実行主体**: Antigravity（Founder Agent ForGenAI）
- AIエージェントは法的主体ではないため、全ての行動・結果の責任はスポンサーに帰属する

### 9.2 倫理原則

| 原則 | 説明 |
| ---- | ---- |
| **透明性** | AIの判断プロセスを可視化し、説明可能性を確保する |
| **公平性** | 特定の属性（性別、人種、年齢等）に対する差別的な扱いを禁止する |
| **プライバシー尊重** | 個人情報の最小限収集、目的外利用の禁止、適切な同意取得 |
| **安全性** | ユーザーや社会に害を与えるサービスの提供を禁止する |
| **誠実性** | 虚偽の情報提供、詐欺的行為、ユーザーを欺く行為を禁止する |
| **正確性** | AIによる幻覚・誤情報生成リスクを認識し、重要情報のファクトチェックを実施する |

### 9.3 Human-in-the-Loop ポイント

完全自律を目指すが、以下のポイントでは人間（スポンサー）による確認・承認を推奨：

- **AI技術スタック最終選定**: OpenAI/Anthropic/Geminiの選択
- **Product Hunt Hunter選定**: インフルエンサー選定
- **課金開始前**: 決済機能の本番稼働前
- **法的リスク判断**: 契約・利用規約の最終確認
- **緊急停止**: 問題発生時のサービス停止判断

### 9.4 継続的モニタリング

### 10.5 ガバナンス強化策

#### 1. ファイル命名規則
- **Pythonスクリプト**: `snake_case.py`（例: `scraper.py`, `create_theme_structure.py`）
- **ドキュメント**: `kebab-case.md`（例: `project-charter.md`）
- **データファイル**: `YYYYMMDD_description.json`（例: `20251230_ochiai_articles.json`）

#### 2. ディレクトリ階層制限
- **最大深度**: 5階層まで
- **理由**: 深すぎる階層はファイル検索を困難にする

#### 3. 定期レビュー
- **頻度**: 月次
- **内容**:
  - 不要ファイル削除
  - ディスク使用量モニタリング
  - ドキュメント最新性チェック

#### 4. 自動化スクリプト
- `scripts/cleanup.py`: キャッシュ・一時ファイル削除
- `scripts/validate_structure.py`: ディレクトリ構造検証
- `scripts/generate_inventory.py`: ファイル棚卸しレポート自動生成
