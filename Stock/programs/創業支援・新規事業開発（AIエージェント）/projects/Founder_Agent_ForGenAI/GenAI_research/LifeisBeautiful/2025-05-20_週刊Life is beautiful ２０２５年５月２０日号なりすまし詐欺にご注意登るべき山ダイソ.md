<h2>なりすまし詐欺にご注意</h2>
<p>先日、以下のようなメッセージが、XのDMで入って来ました。</p>
<blockquote>
<p>突然のご連絡を失礼いたします。
実は、私の父がメルマガを通じてLINEを紹介され資産家のグループに参加していると言い、中島聡様の勧めで「DOGZというアメリカ株」への投資を勧められ購入しております。失礼ながら事実確認をさせていただきたくご連絡いたしました。
中島様ご本人が、一般の個人に対してこのような銘柄を推奨されたり、LINE等で個別に勧誘されることはございますでしょうか？</p>
</blockquote>
<p>明らかに「成りすまし投資詐欺」なので、すぐにその旨を伝えました。私はこのメルマガで、自分が株を持っている会社の話をしたり、（株を持つ・持たないに関係なく）企業の評価を書いたりはしますが、決して、特定の株や投資ファンドを勧めることはありません。もし、私の名で特定の会社やファンドへ投資を勧めるメッセージを受け取ったり見かけた場合、それは１００％「成りすまし投資詐欺」なので、ご注意ください。</p>
<h2>登るべき山</h2>
<p>ここ数週間ほど、寝る間も惜しんでコードを書いています。エクササイズのためにプールで泳いでいる時も、常にプロジェクトのことを考え続けているし、朝も4時前には頭の中がアイデアで一杯で目が覚めて、すぐにコーディングにかかっています。</p>
<p>MulmoCastというプロジェクトは、まさに「私が全力投球すべきプロジェクト」であり、こんな機会を絶対に逃してはいけないと感じるからです。</p>
<p>この感覚をどう説明したら良いか考えたのですが、ひとことで言えば「目の前に私が登るべき山があるから」という感覚です。</p>
<p>有名な登山家が、山に登る理由を「そこに山があるから」と答えた話は有名ですが、私の場合、単にそこに山があるだけではなく、それは「私にしか登れない山」であるどころか、「私にしか、そこにあることが見えない山」「私にしか、登る価値が見出せない山」なのです。</p>
<p>そんな山に出会えることは一生に何回もないので、そんな時は、全力投球をするしかないのです。常日頃から、プログラミングという「天職」に出会えてラッキーだったと感じていますが、特に、こんなモードに入れた時は、格別です。</p>
<p>今回は、私自身が盛り上がっているし、ビジョンにも筋が通っているので、資金集めは簡単に出来そうですが、とりあえずは私自身がコードを書くことが何よりも大切なので、まずはモノづくりに集中します。</p>

<p>ちなみに、ダイソンは未上場企業で株を購入することは出来ませんが、もし上場していたら、応援する意味でも、是非とも株を持ちたい会社です。</p>

<h2>英語うんちく</h2>
<p>私はたまにしか料理をしませんが、料理をする時に難しいと思うのが塩加減です。少なすぎると味がはっきりしないし、でも入れ過ぎてしまうと台無しにです。「塩梅」という言葉は、塩加減以外でも使う言葉ですが、とても良い言葉だと思います。塩に限らず、加減の難しい物事に汎用的に使えるとても便利な言葉です。「食」を重視する、日本ならではの言い回しです。</p>
<p>「塩梅」のような言葉は英語にもあるのかと考えて、最初に頭に浮かんだのは「on the same page」という慣用句です。「書物の同じページを見ている」という意味ですが、ビジネスの場面などで、「互いに見ているものは同じ」という意味で使います。</p>
<p>必ずしも「意見が同じ」である必要はありません。何らかの問題を解決しようとしている際に、まずは全員に「問題は何か」を理解してもらう必要がありますが、そんな場面で「Are we on the same page?（みんな、ここまでは分かってくれたよな？）」のように使います。</p>
<h2>Youtubeチャンネルの立ち上げについて</h2>
<p>このメルマガ「週間Life is beautiful」のYoutubeチャンネルを立ち上げる予定だということは、今年の初め頃に書きましたが、ようやく準備が整いつつあります。</p>
<p>何をどう投稿すべきかが一番重要で、色々と考えて来ましたが、メルマガと同様に、IT系の時事ネタや、エンジニアが知っておくべき重要な技術の解説などを投稿する予定です。それであれば、メルマガの執筆と同時に行うことが可能だし、メルマガとの親和性も良いからです。メルマガから映像へのリンクを貼れば、メルマガの文章量を増やさずに内容をリッチにすることが可能になります。</p>
<p>映像の生成に関しては、現在開発中のMulmoCast（ai-podcasterを進化させたもの）を使います。これを使うと、ほぼ自動で映像が作れるので、毎日どころか、一日数回の投稿さえ可能になります。私自身がユーザーであるという点は、製品の開発においてとても重要です。</p>
<p>映像を出す場所としては、YoutubeとXの両方にします。Youtubeはこれからですが、Xの方は10万人のフォロワーがいるので、多くの人に見てもらえます。ものによっては、Youtubeのみに出して、Xからはリンクを貼るだけにするケースもあると思いますが、せっかく始めるのであれば、Youtubeで10万フォロワーを目指します。</p>
<p>実際のプレゼンの仕方ですが、まずは、あえて話すのは英語で、日本語の字幕付き、というものにしようと考えています。試作していた先生と太郎くんの会話も悪くないのですが、既に似たようなものも多く出ているので、「私らしさ」を出すことが難しいし、日本語音声のクオリティもまだ十分とは言えません。</p>
<p>喜ぶ人は限られてしまうと思いますが、あえて英語で作ってしまうことにより、同時に英語を勉強したい人にとって価値の高いものにするのも悪くないと考えています。6月の頭にはスタートする予定なので、よろしくお願いします。</p>
<h2>「AI使い」になるということ</h2>
<p>MulmoCastは各所でAIを活用していますが、AIは使いよう（バカとハサミは使いよう、の使いようです）だとつくづく思います。</p>
<p>ChatGPTと会話するぐらいならば誰にでも出来ますが、APIを上手に使いこなすのは簡単ではありません。そんな中でつくづく思うのは、AIに関しても人と同じで、「AIに得意なこと」をさせるのが一番だ、ということです。</p>
<p>MulmoCastは、ひとことで言えば、AIを活用したプレゼンツールですが、AIに対して単に「プレゼン資料を作れ」と言っても、必ずしも期待通りのものを作ってはくれません。</p>
<p>そこで、適切なテンプレートやサンプルを用意し、こちらが期待する通りのものを生成するように導いてあげる必要があります。</p>
<p>簡単ではありませんが、適切な導きを与えると、こちらの予想を超えた驚くような仕事をしてくれるので、それに成功した時は、「猛獣使い」ならぬ「AI使い」になったような気がします。</p>
<p>先日も、LLMにチャートを描いてもらうにはどうしたら良いかと考えていた時に、JavaScriptのライブラリに<a href="https://www.chartjs.org">chart.js</a>というものがあることを知り、そのライブラリに渡す、下のような、chartDataというJSONデータだけをLLMに生成させるようにしたら、とても良い仕事をするようになりました。</p>

<h2>Cursorを使い始めました</h2>
<p>開発用のテキスト・エディターにAIを組み込み、それにコード生成をさせながらアプリを開発することをバイブ・コーディングと呼ぶそうですが、これまで私は、エディターに関しては、AIの活用を避けて来ました。</p>
<p>MicrosoftのGithub Copilotを一度試したことがあるのですが、どれがAIにより書かれたコードで、どれが自分が書いたコードなのかがすぐに分からなくなってしまう体験が嫌いで、すぐに使わなくなってしまいました。</p>
<p>ChatGPTやClaudeに依頼してコードを書いてもらうことはしばしばですが、最終的にはそれをコピペして、自分の開発環境で実際に動かすので十分だと感じていたのです。</p>
<p>しかし、あまりにも周りに<a href="https://www.cursor.com">Cursor</a>を使う人が増えているので、先週からチャレンジしています。使い慣れているVisual Studioベースだというのも悪くない点です。</p>
<p>使い始めて、まだ一週間も経ちませんが、とても気に入っています。Chat画面は使わず、単にオート・コンプリート機能を使っているだけですが、ちゃんとコンテキストを理解して正しいコードを提案してくれるので、かなりの時間の節約になっています。</p>
<p>開発の初期では、私はしばしば激しいリファクタリング（コードの改善）を行いますが、そんな時にCursorのオート・コンプリート機能は大活躍しています。</p>
<p>先日も、MulmoScriptというタイプの<code>reference:string</code>というプロパティを```references:MulmoReference[]｀``という配列に変更するリファクタリングを行ったのですが、それを使っているサンプルデータが２０個以上あり、普段であれば、うんざりするような単純作業を繰り返す必要があります。</p>
<p>しかし、cursorは、私が１つ目のファイルで行なった変更の意味をちゃんと理解し、２つ目以降のファイルでは、当該箇所にカーソルを持っていくだけで、私が何をしたいのかを理解し、新しいデータ構造を提案し、私はそれに（タブキーを押して）OKを出すだけでした。</p>
<p>ちなみに、以下の変更履歴を見ていただければ、Cursorがかなり賢いことをしていることが分かります。</p>

</code></pre>

    &quot;title&quot;: &quot;AI-2027&quot;,

<h2>ChatGPTが書いたエレガントなコード</h2>
<p>私たちエンジニアの間でコード（＝プログラム）に対する一番の褒め言葉は「エレガントなコード」です。余分な複雑さを一切持たず、与えられた仕事をしっかりとこなす、明瞭、簡潔なコードのことです。</p>
<p>先日、ChatGPTがそんなエレガントなコードを書いて来て驚かされました。</p>
<p>TypeScriptでは、文字の置き換えをする際には、しばしば、</p>
<pre><code>const data = { name: &quot;Satoshi&quot;, id: &quot;12345&quot; };
const output = `Hello, ${data.name}! Your ID is ${data.id}.`;
</code></pre>
<p>のように書きますが、これはインタープリタの機能なので、&quot;<code>Hello, ${data.name}! Your ID is ${data.id}.</code>&quot;をテンプレートとして外から読み込むことが出来ません。</p>
<p>そこでそんなライブラリがあるかとChatGPTに尋ねたところ、ライブラリ名は教えてくれたものの、同時に、こんな感じに書くことも出来る、と提案してくれたのが、下の１行です。</p>
<pre><code>const output = template.replace(/\$\{(.*?)\}/g, (_, key) =&gt; data[key.trim()] ?? &quot;&quot;);
</code></pre>
<p>レギュラー・エクスプレッション（/${(.*?)}/g）を使って置き換えるべき場所を探し、それぞれについて無名関数を使ってdataから引っ張って来て置き換えるという、とてもシンプルなソリューションです。</p>
<p>こんなエレガントなコードを書けるエンジニアの割合はそもそも少ないのですが、それが一瞬で出来てしまうところが最近のLLMの凄いところです。これまで１００人の開発者が必要だったソフトウェアが、わずか数人で出来てしまう。まさにそんな時代に突入しつつあることを実感させてくれる事例でした。</p>
<h2>Tesla Model X から Model Y へ</h2>
<p>シアトルに戻ってModel Xを運転しようとすると、カーナビが表示される側の画面が真っ暗で、（スピードなどが表示される）ダッシュボードには、&quot;Center Display unavailable&quot;と表示され、運転すら出来ません。</p>
<p>仕方がないので、Teslaに出張サービスをリクエストしたところ、出張サービスではこの症状には対応できず、車をレッカー車でTeslaのサービス・センターまで持って来る必要があると言われました。仕方がないので、AAAに加入し（加入すると、無料でレッカー移動が出来るようになります）、すぐにサービス・センターに届けました。</p>
<p>するとTeslaからすぐに連絡があり、コンピュータ(MCU1)の故障であり、新しいコンピュータ(MCU2)に買い換える必要があると言われました。値段は2000ドル弱でしたが、MCU1であるが故に最新のFSD（自動運転システム）にアップデート出来なかったり、Apple Musicにアクセスできないなどの難点があったので、悪くないと思い、アップデートをお願いしました。</p>
<p>すると翌日には交換が終わり、引き取ることが出来ました。早速、最新のFSDにアップデートして試したのですが、少しがっかりしました。ハワイで運転しているModel YのFSDと比べて、かなり精度が落ちるのです。</p>
<p>FSDをつかさどるコンピュータは、MCUとは異なるハードウェアで、私のModel Xに搭載されているのは、一世代古いHW3と呼ばれるものです。2016年モデルでFSDが動くこと自体が奇跡のような話ですが、最新のHW4と比べると処理能力は3分の１ぐらいしかなく、精度が大きく劣るのです。</p>
<p>この問題に関しては、Teslaも認識しており、「HW4への無料アップデートを提供する予定だ」というアナウンスはありましたが、いつになるかは不明です。</p>
<p>そんな時に、Teslaから届いたのは、「今、Tesla車を買えば、Model XのFSDをトランスファーすることが可能です」というオファーです。Model XのFSDは購入時に6000ドルで購入していますが、それをトランスファー出来るのであれば、1万ドルの節約になります。</p>
<p>さらに家のガレージが、私のModel Xと妻のe-Tronの両方を入れるには狭すぎて、彼女からは「もっと小さな車に買い替えてほしい」とリクエストされていたのです。</p>
<p>買うならば、ハワイの家と同じ、Model Yのロングレンジ一択なので、色だけ選んで購入の手続きを始めました。Model Xには2万ドルの下取り価格がついたし、1万ドル相当のFSDはトランスファー出来るし、政府からの7500ドルのEV補助金も付くので、かなりお得なお買い物となりました。</p>
<p>これでModel Yをシアトルとハワイの両方で持つことになりました。ハワイと違って、シアトルでは街中を運転することも多いので、FSDのテストには絶好の環境です。FSDの様子を撮影するためのアタッチメントを新たに買い直す必要がありますが、今後は、ビデオ付きのFSDレポートをMulmoCastを使って作りたいと思います。</p>
<h2>百科事典を読んで育った子供、ChatGPTから言葉を教わる子供</h2>
<p>私は子供の頃は百科事典が大好きで、暇があれば読んでさまざまな知識を仕入れていました。今時の子供はそれがWikipediaであり、これからの子供達はChatGPTなのだろうと思います。</p>

<p>アシモフのロボット工学三原則
1.	第一条：ロボットは人間に危害を加えてはならない。また、その危険を看過することによって人間に危害を及ぼしてはならない。
2.	第二条：ロボットは人間に与えられた命令に従わなければならない。ただし、その命令が第一条に反する場合は、この限りでない。
3.	第三条：ロボットは前記第一条および第二条に反しない限り、自己を守らなければならない。</p>
