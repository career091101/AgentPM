## 1. API・データ取得の制約

### 制約1.1: 公式API の不在

**制約内容**
- note は公開されている「公式 API」を提供していない
- クリエイターが自分の記事データを API 経由で取得する仕組みなし
- 第三者向けのデータ連携 API も存在しない

**影響範囲**
- 自動データ収集・分析ツールの開発が困難
- 他プラットフォーム（Zapier 等）との直接連携不可
- note の売上・PV データを自動で外部ツールに流し込めない

**代替案**
- [✓] note 内の CSV ダウンロード機能を使用（公式サポート）
- [✓] 非公式 API（逆エンジニアリング）の使用（リスク有）
- [✓] Web スクレイピング（robots.txt との兼ね合いを確認）
- [✓] note からの手動エクスポート + 自社システムへのインポート

**リスク評価**: 🔴 **HIGH** - API 仕様変更時に無警告で動作不可になる

---

### 制約1.2: 非公式 API のレート制限

**制約内容**
- 非公式 API 使用時にレート制限（Rate Limiting）が発生
- 1分間に 10 リクエスト程度が上限目安
- 「429」エラーが返される場合がある

**具体的な制限値**

| 制限タイプ | 上限 | 超過時の処理 |
|----------|------|----------|
| **1分間のリクエスト数** | 約 10 件 | 429 エラーで拒否 |
| **スキ操作の集中** | 50 を超える1度の操作 | 403 エラーで拒否 |
| **連続アクセス時間** | 長時間連続は検出 | IP制限の可能性 |

**影響範囲**
- 大量データ収集（1万件以上）に数時間〜数日要する
- リアルタイムデータの更新が困難
- バッチ処理の実行計画が複雑化

**代替案**
- [✓] リクエスト間に 2 秒待機を設定（note_benchmark_collector.py での実装例）
- [✓] 深夜のレート制限が低い時間帯を狙う
- [✓] 複数 IP アドレスからの分散実行（プロキシ使用）
- [✓] 公式 CSV ダウンロードで大量データを一括取得

**リスク評価**: 🟡 **MEDIUM** - 通常利用では問題なし、高頻度アクセスは危険

---

### 制約1.3: スクレイピングの法的・技術的リスク

**制約内容**
- note の利用規約では「プログラムによる自動情報収集の禁止」は明記されていない（2024年10月時点）
- ただし robots.txt の更新により、いつ制限される可能性もある
- note が違反と判断した場合、アカウント停止の可能性

**法的リスク**

| リスク項目 | リスク度 | 法的根拠 |
|----------|--------|--------|
| **著作権侵害** | 低 | 記事は著作権保護対象だが、取得のみでは違法でない |
| **利用規約違反** | 中 | 今後の規約更新で禁止される可能性 |
| **不正アクセス** | 中 | サーバーに過度な負荷をかけると該当 |
| **個人情報取得** | 低 | note のプロフィールは公開情報 |

**技術的リスク**
- note のページ構造が変更されると、スクレイピングコードが動作しなくなる
- 過去事例: 2023年に note の HTML 構造が大幅変更された結果、複数の自動化ツールが停止

**代替案**
- [✓] note 公式 CSV ダウンロード機能の使用（最も安全）
- [✓] 低頻度なスクレイピング（週 1-2 回程度）の実施
- [✓] User-Agent を正当に設定（ボット判定を避ける）
- [✓] サーバー負荷を最小化（リクエスト間隔を長めに設定）
- [✓] note にスクレイピング許可を事前申請（非公式、成功率低）

**リスク評価**: 🔴 **HIGH** - アカウント停止リスクあり

## 2. HTML・構造化データの制約

### 制約2.1: カスタム HTML 編集の限定性

**制約内容**
- 無料版: HTML 編集機能なし（完全に固定）
- note Pro 版: 「カスタム HTML」機能あり（推定）
  - ただし実装範囲は限定的
  - `<head>` 内への任意 HTML 挿入は不可（推定）
  - `<body>` 内の特定セクションのみ編集可能（推定）

**影響範囲**
- スキーママークアップ（JSON-LD）の `<head>` 挿入不可（推定）
- トラッキングコード（Google Analytics等）の自由な配置が困難
- カスタム JavaScript の `<head>` 挿入が不可（推定）

**具体的な制約（推定）**

| 要素 | 編集可能 | 編集不可 | 備考 |
|------|--------|--------|------|
| **記事内の HTML** | △（制限あり） | - | マークダウン形式がメイン |
| **カスタム CSS** | ? | - | Pro版で部分的に可能（推定） |
| **スクリプトタグ** | △ | - | セキュリティ上の制限あり（推定） |
| **メタタグ** | △ | `<head>` 内への自由編集 | title, description のみ（推定） |
| **OGP タグ** | △ | 完全カスタマイズ | 自動生成、部分編集のみ（推定） |

**代替案**
- [✓] note Pro版での実装テスト（契約して確認）
- [✓] マークダウン内での HTML タグ埋め込み（セキュリティチェック後）
- [✓] 外部の「Web フォント配信」や「CDN」を活用
- [✓] メタタグは自動生成に頼る

**確認方法**
1. note Pro版を試契約（月額 500 円）
2. 実際にカスタム HTML セクションでテスト実装
3. `<script>` タグの埋め込みテスト
4. スキーママークアップ JSON-LD の挿入テスト

**リスク評価**: 🟡 **MEDIUM** - Pro版での実装可否次第で大きく変わる

## 3. メタタグ・URL 構造の制約

### 制約3.1: メタタグのカスタマイズ限定性

**制約内容**
- **編集可能**: タイトル、本文（自動的に description 生成）
- **編集不可**: robots タグ、canonical タグ、その他のカスタムメタタグ
- **OGP タグ**: 自動生成のみ、カスタマイズ不可（推定）

**具体的な制限**

| メタタグ | カスタマイズ可 | 自動生成 | 手動設定 |
|---------|--------------|--------|--------|
| **title** | ○ | - | 記事タイトル |
| **description** | △ | ○ | 本文最初の 160 字が自動 |
| **og:title** | ✗ | ○ | - |
| **og:description** | ✗ | ○ | - |
| **og:image** | △ | ○ | サムネイル画像指定可（推定） |
| **og:url** | ✗ | ○ | - |
| **twitter:card** | ✗ | ○ | - |
| **robots** | ✗ | ○ | - |
| **canonical** | ✗ | ○ | - |

**影響範囲**
- 検索結果表示時の description テキストが固定（本文の最初160字）
- SNS シェア時の表示内容が固定（og: タグ自動生成）
- キーワード最適化が本文に依存

**代替案**
- [✓] 記事タイトルを 60-70 文字で キーワード最適化（title タグの制御）
- [✓] 本文の最初 160 字を description として最適化
- [✓] サムネイル画像を戦略的に選択（OGP image の最適化）
- [✓] 本文内での キーワード頻度最適化（タグによる制御がないため）

**リスク評価**: 🟡 **MEDIUM** - title のみ制御可能だが、description 最適化は工夫が必要

---

### 制約3.2: URL 構造のカスタマイズ不可

**制約内容**
- URL は note が自動生成（記事ID ベース）
- クリエイターが URL をカスタマイズできない
- URL パーマリンクの変更はできない（永続的に記事ID固定）

**URL の一般的なパターン**
```
https://note.com/{creator_id}/n/{article_id}
```

**SEO への影響**

| 項目 | impact | 説明 |
|------|--------|------|
| **キーワード露出** | 低 | URL に キーワードが含まれない |
| **URL 可読性** | 低 | 記事内容が URL から判断できない |
| **ソーシャルシェア** | 低 | URL が長く、見栄えが悪い |
| **クリック率** | 低 | 検索結果での CTR が低下（推定） |

**代替案**
- [✓] タイトルで キーワードを露出（URL の不足を補う）
- [✓] 短縮 URL サービス（note.mu）を使用（ただし SEO 効果は限定的）
- [✓] SNS シェア時は独自の短縮 URL を生成（bitly 等）
- [✓] 被リンク時にアンカーテキストでキーワード最適化を依頼

**リスク評価**: 🔴 **HIGH** - URL 制御がないため SEO 効果を減少

---

### 制約3.3: slug（スラッグ）のカスタマイズ不可

**制約内容**
- 日本語タイトルは note.com の自動翻訳により数文字エンコード
- タイトルが URL に反映されない
- 個人ブログのような「slug」設定機能なし

**具体例**
```
デザインのコツ 5選
      ↓ (note の自動変換)
https://note.com/{id}/n/n9d4fdaaa00v
```

**他プラットフォームとの比較**

| プラットフォーム | URL 例 | slug カスタマイズ |
|------------|--------|----------------|
| **note** | `/n/n9d4fdaa...` | ✗ 不可 |
| **はてなブログ** | `/entry/2026/01/10/design` | ○ 可 |
| **Medium** | `/@user/design-tips-5` | ○ 可 |
| **Qiita** | `/items/xxxxxxx` | △ 部分的 |
| **WordPress** | `/design-tips-5/` | ○ 完全自由 |

**代替案**
- [✓] 本文内での キーワード配置を強化
- [✓] 見出し（H2-H3）で キーワードを露出
- [✓] SNS での タイトル修正（X/Instagram での投稿時に キーワード追加）

**リスク評価**: 🔴 **HIGH** - slug 制御がないため SEO の自由度が大きく低下

## 4. JavaScript 実行の制約

### 制約4.1: カスタム JavaScript の実行不可

**制約内容**
- note プラットフォーム内では、カスタム JavaScript の実行が不可能
- `<script>` タグの埋め込みが制限またはサニタイズされる
- クリエイター側で JavaScript による動的要素の追加が不可

**影響範囲**
- インタラクティブな要素（タブ、スライダー等）の追加が不可
- 動的なテーブルの作成が不可
- クライアント側での データ処理ができない
- トラッキングスクリプト（Google Analytics等）の カスタム実装が困難

**実装テスト結果（推定）**

| JavaScript 機能 | 実行可 | 理由 |
|----------|--------|------|
| **Google Analytics** | △ | note が自動挿入している可能性 |
| **カスタムスクリプト** | ✗ | セキュリティ上の制限 |
| **外部 iframe** | △ | 特定ドメインのみ許可か |
| **localStorage/sessionStorage** | ? | 不明 |
| **DOM 操作** | ✗ | コンテンツセキュリティポリシー（CSP）制限 |

**代替案**
- [✓] note の分析ダッシュボードを使用（Google Analytics の代わり）
- [✓] 外部サービス（Calendly、Typeform等）を iframe で埋め込む
- [✓] 動的要素は note マークダウン内での HTML で静的に表現
- [✓] インタラクティブ要素が必要な場合は、リンク先の外部ページに誘導

**リスク評価**: 🔴 **HIGH** - JavaScript の自由な実行ができないため、高度なインタラクティビティが不可能

## 5. デザイン・カスタマイズの制約

### 制約5.1: CSS カスタマイズの限定性

**制約内容**
- 無料版: CSS カスタマイズ機能なし
- note Pro版: CSS カスタマイズ機能あり（推定、詳細不明）
- テーマ選択は複数用意（ただし数は限定的）

**無料版での制約**
- プロフィール背景色: デフォルトから選択可（カスタムカラー設定不可）
- フォント: note のデフォルトフォントのみ（変更不可）
- レイアウト: 固定レイアウト（カラム数等の変更不可）
- ウィジェット: note の定義済みウィジェットのみ

**代替案**
- [✓] note Pro版での CSS 編集テスト
- [✓] マークダウン内での `<style>` タグ埋め込み（セキュリティ確認後）
- [✓] テーマの「公式オプション」から最も好みのものを選択
- [✓] デザイン統一は note のデフォルトに合わせ（視認性を優先）

**リスク評価**: 🟡 **MEDIUM** - デザインの差別化は困難だが、note のデフォルトも高品質

---

### 制約5.2: ブランディング要素の制限

**制約内容**
- カスタムロゴ、ファビコン（favicon）の設定不可
- header/footer のカスタマイズ不可
- プロフィール画像・背景は設定可、ただし容量制限あり

**影響範囲**
- 個人ブランドの「視覚的統一」が困難
- 企業の CI（Corporate Identity）を note 内では実装できない
- note のロゴが常に表示される（削除不可）

**代替案**
- [✓] プロフィール欄での テキストベースの ブランディング
- [✓] プロフィール画像を工夫（ロゴ化、統一テンプレート化）
- [✓] note の ロゴを活用した プロモーション（「note で発信中」という前提で設計）
- [✓] 外部ウェブサイト（note 経由の誘導先）で 完全な ブランディングを実施

**リスク評価**: 🟡 **MEDIUM** - ブランディングの自由度が低いが、note の ブランド力を活用できる

## 10. まとめ：技術的制約の整理

### 制約の大分類

**1. 自動化・連携の制約（修復困難）**
- 公式 API 不在
- スクレイピングのリスク
- JavaScript 実行不可
→ **対応**: 手動処理 + 安全な非公式 API 使用

**2. SEO 最適化の制約（工夫で対応可）**
- URL カスタマイズ不可
- メタタグ部分的カスタマイズのみ
- スキーママークアップ実装不確実
→ **対応**: タイトル・本文の最適化で補完

**3. デザイン・UI の制約（許容可）**
- HTML/CSS カスタマイズ限定
- ブランディング要素の制限
→ **対応**: note デフォルト + 外部ウェブでの補完

### 結論

note.com は「クリエイター向けのシンプルなプラットフォーム」として、技術的なカスタマイズを意図的に制限しています。これは以下のメリットをもたらします：

1. **ユーザーフレンドリー**: 複雑な技術知識不要
2. **品質の統一**: デザイン/セキュリティの一貫性
3. **スピード**: セットアップから発信まで数時間

一方、技術的な制約がある以上、最適化戦略は「note のシステム内での工夫」に集中すべきです：
- **タイトル・本文**: 完全制御可能 → 最重要
- **メタタグ**: 部分制御 → 次点
- **構造化データ**: 確認後対応 → 余力があれば
