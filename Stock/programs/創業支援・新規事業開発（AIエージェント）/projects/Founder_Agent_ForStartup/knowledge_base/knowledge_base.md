# ForStartup Edition 共通ナレッジベース

## Overview

ForStartup Editionは、VC調達を目指すスタートアップ向けの創業支援AIエージェントです。ForRecruitよりも厳格な基準（CPF 70%、10倍優位性3軸）を適用し、ピッチデッキ作成やVC Meeting準備など、資金調達に特化した機能を提供します。

## ForStartup Edition の特徴

### 基本理念

- **ターゲット**: VC調達を目指すスタートアップ
- **目標**: $1M-10M Seed/Series A調達成功
- **差別化**: ピッチデッキ、VC Meeting準備、ユニットエコノミクス厳格検証

### ForRecruitとの差異

| 項目 | ForStartup | ForStartup |
|------|-----------|-----------|
| **CPFスコア基準** | 50% | **70%**（厳格化） |
| **10倍優位性** | 2軸必須 | **3軸必須**（スケーラビリティ重視） |
| **市場規模** | TAM $100M | **TAM $1B以上** |
| **成長率** | 月次10% | **月次20%以上** |
| **ユニットエコノミクス** | LTV/CAC 3.0 | **LTV/CAC 5.0以上** |
| **CAC回収期間** | 18ヶ月 | **12ヶ月以内** |
| **追加スキル** | スタートアップ評価基準分析 | **ピッチデッキ、VC Meeting準備** |

## ユニットエコノミクス詳細

### 主要指標

| 指標 | 最低基準 | 理想基準 | 計算式 |
|------|---------|---------|-------|
| **LTV/CAC** | 5.0以上 | 7.0以上 | LTV ÷ CAC |
| **CAC回収期間** | 12ヶ月以内 | 6ヶ月以内 | CAC ÷ (ARPU × Gross Margin) |
| **Gross Margin** | 70%以上 | 80%以上 | (Revenue - COGS) / Revenue |
| **Net Revenue Retention** | 100%以上 | 120%以上 | (MRR期末 - Churn + Expansion) / MRR期初 |
| **Magic Number** | 0.75以上 | 1.0以上 | Net New ARR / Sales & Marketing費用 |

### 計算プロセス

#### Step 1: CAC計算

**CAC（Customer Acquisition Cost）**:
```
CAC = (Sales費用 + Marketing費用) / 新規顧客数
```

**例**:
- Sales費用: $50K/月
- Marketing費用: $30K/月
- 新規顧客数: 100人/月
- **CAC = $80K / 100 = $800/顧客**

#### Step 2: LTV計算

**LTV（Lifetime Value）**:
```
LTV = ARPU × Gross Margin ÷ Churn Rate
```

**例**:
- ARPU: $100/月
- Gross Margin: 80%
- Churn Rate: 2%/月
- **LTV = $100 × 0.8 / 0.02 = $4000**

#### Step 3: LTV/CAC計算

```
LTV/CAC = $4000 / $800 = 5.0
```

**判定**: ✅ 最低基準クリア（5.0以上）

#### Step 4: CAC回収期間計算

```
CAC回収期間 = CAC / (ARPU × Gross Margin)
            = $800 / ($100 × 0.8)
            = 10ヶ月
```

**判定**: ✅ 基準クリア（12ヶ月以内）

### ユニットエコノミクス成功事例

| 企業 | LTV/CAC | CAC回収期間 | 特徴 |
|------|---------|------------|------|
| **Shopify** | 7.5 | 5ヶ月 | 低CAC、高リテンション |
| **Atlassian** | 9.0 | 3ヶ月 | プロダクト主導成長 |
| **Zoom** | 6.8 | 4ヶ月 | バイラル成長 |
| **Slack** | 8.2 | 6ヶ月 | ネットワーク効果 |

## ピッチデッキ成功パターン

### デザイン原則

1. **1スライド1メッセージ**: 詰め込みすぎない
2. **ビジュアル優先**: テキストは最小限、グラフ・画像を活用
3. **ストーリー**: 課題→解決→成長のストーリーライン
4. **データドリブン**: すべての主張を数値で裏付け

### スライド別成功パターン

#### Problem Slide

- **定量データ**: 「年間$10B損失」「10M人が困っている」
- **ストーリー**: 実際のユーザー事例
- **悪い例**: 「多くの人が困っている」（曖昧）
- **良い例**: 「米国だけで年間$10B損失、10M中小企業が手作業で経費精算」

#### Solution Slide

- **10倍優位性**: 3軸（速度・コスト・品質）を明示
- **デモ**: スクリーンショット、動画
- **Before/After**: 具体的な改善数値
- **良い例**: 「従来10時間→10分（60倍高速化）」

#### Market Slide

- **TAM/SAM/SOM**: 具体的な市場規模
- **成長率**: 市場の年率成長率20%以上
- **良い例**: 「TAM $50B、年率25%成長、当社はSAM $5Bをターゲット」

#### Traction Slide

- **成長曲線**: 月次成長率20%以上のグラフ
- **主要指標**: MRR、ARR、ユーザー数
- **良い例**: 「MRR $100K→$500K（5ヶ月で5倍）、月次成長率35%」

### ピッチデッキ成功事例

- **Airbnb**: "Book rooms with locals, rather than hotels"のシンプルなメッセージ
- **Uber**: "Tap a button, get a ride"の明確な価値提案
- **LinkedIn**: ネットワーク効果の可視化（グラフで成長曲線）

## VC Meeting準備

### VC Meeting前の準備（7-14日前）

#### Phase 1: VC調査（7日前）

1. **投資家プロフィール**:
   - 過去の投資先（類似企業）
   - 投資フェーズ（Seed、Series A等）
   - 投資額レンジ
   - ポートフォリオ企業

2. **パートナー調査**:
   - LinkedIn、Twitter、ブログ
   - 最近の発言・関心領域
   - 類似企業の有無（競合 or シナジー）
   - 投資タイミング（シード、シリーズA等）

#### Phase 2: 想定Q&A準備（Meeting 3日前）

##### 典型的なVC質問（30個）

**市場・競合**:
1. 「なぜ今、この市場なのか？」
2. 「GoogleやAmazonがこれをやったらどうなる？」
3. 「競合Xとの違いは何か？」
4. 「市場規模$1Bの根拠は？」

**プロダクト・技術**:
5. 「10倍優位性の証明は？」
6. 「技術的参入障壁は何か？」
7. 「スケーラビリティの検証は？」
8. 「プロダクトロードマップは？」

**ビジネスモデル**:
9. 「LTV/CACの計算根拠は？」
10. 「CAC回収期間が12ヶ月の理由は？」
11. 「Churn率の改善計画は？」
12. 「価格設定の根拠は？」

**トラクション**:
13. 「月次成長率20%をどう達成した？」
14. 「どのチャネルが最も効果的か？」
15. 「ユーザーリテンションは？」
16. 「有料転換率は？」

**チーム**:
17. 「なぜあなたがこの課題を解決できるのか？」
18. 「共同創業者のスキルセットは？」
19. 「採用計画は？」
20. 「過去の失敗から学んだことは？」

**財務**:
21. 「調達額の使途は？」
22. 「ランウェイ（資金が持つ期間）は？」
23. 「次のマイルストーンは？」
24. 「Unit Economicsの前提条件は？」

**戦略**:
25. 「Go-to-Market戦略は？」
26. 「最大のリスクは何か？」
27. 「Exit戦略は？」
28. 「なぜ今、調達するのか？」

**その他**:
29. 「最悪のシナリオは？」
30. 「5年後、どうなっていたいか？」

#### Phase 3: デモ準備（Meeting 2日前）

1. **デモシナリオ作成**:
   - 3-5分のデモストーリー
   - 10倍優位性を示すユースケース
   - エラーハンドリング（ネットワーク切断等）

2. **デモ環境チェック**:
   - 動作確認（3回以上）
   - バックアップ環境準備
   - スクリーンショット・動画のバックアップ

#### Phase 4: Meeting当日

1. **タイムマネジメント**:
   - ピッチ: 10-15分
   - Q&A: 30-45分
   - 余裕を持った時間配分

2. **持ち物チェック**:
   - ノートPC（充電済み）
   - ピッチデッキ（PDF、USB）
   - 名刺
   - デモ環境（オフライン対応）

3. **メモ取り**:
   - VCのフィードバック記録
   - 質問内容の記録
   - Next Stepの確認

#### Phase 5: フォローアップ（Meeting後24時間以内）

1. **お礼メール**:
   - 面談への感謝
   - 質問への追加回答
   - 次のステップ確認

2. **追加資料送付**:
   - デモ動画
   - 詳細財務モデル
   - ユーザー事例

### VC Meeting成功パターン

- **Stripe**: シンプルなAPIデモで7分間に投資決定
- **Airbnb**: 写真の質改善で成長率向上の実演
- **Notion**: オールインワンの価値をライブデモ

### VC Meeting失敗パターン

- **過度な技術詳細**: VCは技術より市場機会を重視
- **競合軽視**: 「競合なし」は市場理解不足の証拠
- **数字の曖昧さ**: ユニットエコノミクスを即答できない
