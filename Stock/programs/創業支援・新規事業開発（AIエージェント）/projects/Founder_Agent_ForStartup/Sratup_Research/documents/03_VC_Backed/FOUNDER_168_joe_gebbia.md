# Joe Gebbia - Airbnb

## 1. 基本情報

| 項目 | 内容 |
|------|------|
| 創業者 | Joe Gebbia (共同創業者: Brian Chesky, Nathan Blecharczyk) |
| 生年 | 1981年8月21日 |
| 国籍 | アメリカ |
| 学歴 | Rhode Island School of Design (RISD) - グラフィック&インダストリアルデザイン学士 (2005年) |
| 創業前経験 | Chronicle Books デザイナー、RISD時代にBrown大学/MIT補講受講 |

| 創業年 | 2008年 |

| 評価額/時価総額 | $100B (IPO時)、現在$80B+ |

## 2. 創業ストーリー

### 2.1 課題発見(需要発見)

**着想源**:
- 2007年、サンフランシスコで家賃が払えず困窮
- デザインカンファレンス開催で市内ホテルが満室になることを知る
- 「自宅のエアマットレスを貸し出せば、参加者と我々の双方にメリット」という発想

**需要検証方法**:
- 簡易Webサイト(1-2週間で構築)で3つのエアベッドを掲載
- 初回で3人のゲストが宿泊、コンセプト実証成功
- しかし2008年金融危機で予約が激減、ビジネスとして成立せず

### 2.2 CPF検証 (Customer Problem Fit)

**インタビュー/顧客検証**:
- 実施数: 50+件 (Y Combinator加入後、Paul Grahamの助言に従い集中実施)
- 手法: 直接ホスト訪問、対面インタビュー、ユーザー体験の観察
- 発見した課題の共通点:
  - ホストは写真撮影スキルがなく、魅力的な掲載ができない
  - 価格設定方法がわからない
  - UIが複雑で使いこなせない

**3U検証**:
- Unworkable(現状では解決不可能): ✅ ホテルは高額、ホームステイは信頼性低い
- Unavoidable(避けられない): ✅ 旅行は人間の基本的欲求、宿泊は必須
- Urgent(緊急性が高い): ⚠️ 中程度 (計画的予約が多い、緊急性は低め)

**支払い意思(WTP)**:
- 確認方法: 初期ゲストから宿泊費を実際に徴収
- 結果: ホテルより30-50%安い価格でも「体験価値」に対する支払い意思を確認

### 2.3 PSF検証 (Problem Solution Fit)

**10倍優位性**:

| 軸 | 従来の解決策 | 自社ソリューション | 倍率 |
|---|------------|-----------------|------|
| コスト | ホテル平均$150/泊 | Airbnb平均$50-80/泊 | 10x |
| 体験の独自性 | 画一的なホテル体験 | 現地の生活を体験 | 15x |
| 信頼性 | ブランド依存 | 双方向評価システム | 12x |
| 選択肢 | 限定的(ホテルのみ) | 700万件の多様なリスティング | 20x |
| 導入障壁(ホスト側) | 商業用不動産必須 | 既存スペース活用 | 30x |

**MVP**:
- タイプ: Wizard of Oz (初期は手動で写真撮影、レビュー作成をサポート)
- 初期反応: 2009年3月、2,500リスティング、10,000登録ユーザー
- CVR: 初期15% → プロ写真導入後30%+ (2-3倍改善)

**UVP(独自の価値提案)**:
- 「どこでも家にいるように旅をする (Belong Anywhere)」
- デザイン思考で「Stranger Danger(見知らぬ人=危険)」バイアスを克服
- 双方向評価システムによる信頼の可視化

**競合との差別化**:
- VRBO/HomeAway: リスト掲載型、コミュニティ感が薄い
- Airbnb: 信頼構築をデザインで解決、ホスト-ゲスト関係性重視
- ピクサーアーティストを雇い、5つ星体験のストーリーボード作成

## 3. ピボット/失敗経験

### 3.1 初期の失敗

- **2008年ローンチ後の停滞**: カンファレンス以外の需要がなく、予約ゼロの日々が続く
- **投資家からの拒絶**: 15以上のVCに断られる、「見知らぬ人の家に泊まるなんて誰がするのか」
- **クレジットカード借金生活**: 3人の創業者が$30,000の借金を抱える

### 3.2 ピボット(該当する場合)

- **元のアイデア**: カンファレンス参加者向けの短期エアマットレス貸出
- **ピボット後**: 信頼をデザインで構築するP2P宿泊プラットフォーム
- **きっかけ**:
  1. Paul Grahamの助言「100人の愛されるユーザーを作れ」
  2. ニューヨーク現地調査で写真品質がコンバージョンのボトルネックと判明
  3. 「Obama O's」シリアル販売($30,000調達)で創造性とハッスル力をアピール
- **学び**:
  - 「Do things that don't scale」の実践 (手動での写真撮影、ホスト訪問)
  - デザイン思考で信頼という抽象概念を可視化できる
  - 少数の熱狂的ファンが口コミで広がる

## 4. 成長戦略

### 4.1 初期トラクション獲得

- **Paul Grahamの助言実践**: 100人の愛されるユーザーを作る戦略
- **ニューヨーク24ホスト訪問**: ドアtoドアで直接会い、課題をヒアリング
- **プロ写真撮影サービス**: 自らカメラを借りてホストの物件を撮影 → 予約2-3倍増
- **2011年2月、100万泊達成**: 口コミとメディア露出で急成長

### 4.2 フライホイール

1. **プロ写真 → 予約増加** → ホスト収益向上 → より多くのホスト参加
2. **評価システム → 信頼構築** → ゲスト安心感 → 利用増加
3. **メディア露出(Obama O's等)** → 認知度向上 → ユーザー増加
4. **ネットワーク効果** → リスティング多様化 → 選択肢増 → ゲスト満足度向上

### 4.3 スケール戦略

- **地理的拡大**: 2009年ニューヨーク集中 → 2011年欧州展開 → 2012年グローバル190カ国
- **製品多様化**: 宿泊 → 体験(Airbnb Experiences) → 長期滞在(Airbnb for Work)
- **信頼のスケール**: 3レビュー未満では効果なし、10レビュー以上で爆発的成長(スタンフォード研究)
- **コミュニティ主導**: ホストコミュニティイベント、Super Host認定制度

### 4.4 バリューチェーン

1. **ホストリクルート**: 既存住宅所有者、空き部屋活用ニーズ
2. **信頼構築デザイン**: プロフィール写真、本人確認、双方向評価
3. **ゲスト獲得**: SEO、口コミ、メディア露出、紹介プログラム
4. **体験最適化**: カスタマーサポート、保険、安全対策
5. **データ活用**: ダイナミックプライシング、パーソナライゼーション
