# 笠原健治 - MIXI (mixi)

## 1. 基本情報

| 項目 | 内容 |
|------|------|
| 創業者 | 笠原健治 |
| 生年 | 1975年12月6日 |
| 国籍 | 日本 |
| 学歴 | 大阪府立北野高等学校 → 東京大学経済学部（2001年卒業） |
| 創業前経験 | 大学在学中にFind Job!を運営 |
| 企業名 | 株式会社MIXI（旧：有限会社イー・マーキュリー → 株式会社ミクシィ） |
| 創業年 | 1999年（法人化）/ 1997年（個人事業開始） |
| 業界 | インターネット・SNS・デジタルエンターテインメント |
| 現在の状況 | 東証プライム上場（証券コード: 2121） |
| 評価額/時価総額 | 約2,200億円（2025年3月時点） |

## 2. 創業ストーリー

### 2.1 課題発見（需要発見）

**着想源**:
- 大学3年時（1997年）、日経新聞・日経産業新聞を読む中で、インターネット関連企業が次々と設立されていることに気づく
- PC産業が1980年代に立ち上がったように、「インターネット産業」という新しい産業が生まれつつあると感じた
- 大学時代にアルバイトを探す際、紙の求人情報誌を見て応募する非効率さを体感

**需要検証方法**:
- NetAge社長・西川潔氏のオフィスで事業を開始し、ITベンチャー向け求人サービスの需要を確認
- 求人サイト「Find Job!」を1997年11月に開始、ITベンチャー企業からの求人ニーズを実証

### 2.2 CPF検証（Customer Problem Fit）

**インタビュー/顧客検証**:
- 実施数: 明確な数値は不明
- 手法: 自身の経験（求人情報誌の非効率さ）+ ユーザー行動観察
- 発見した課題の共通点: ITベンチャー企業の人材採用難、求職者の情報アクセス困難

**3U検証（mixi開発時）**:
- Unworkable（現状では解決不可能）: 当時の日本にはリアルな人間関係をオンラインで再現するサービスがなかった
- Unavoidable（避けられない）: コミュニケーション欲求は普遍的ニーズ
- Urgent（緊急性が高い）: 米国でFriendsterが流行し、日本でも同様のニーズがあると判断

**支払い意思（WTP）**:
- 確認方法: 無料サービスとして開始し、広告収入モデルで収益化
- 結果: ユーザー数増加に伴い広告収入が拡大

### 2.3 PSF検証（Problem Solution Fit）

**10倍優位性**:

| 軸 | 従来の解決策 | 自社ソリューション | 倍率 |
|---|------------|-----------------|------|
| 信頼性 | 匿名掲示板（2ちゃんねる等） | 招待制・実名ベースの安心空間 | 10x |
| つながり維持 | メール・電話 | 足跡機能・日記・コミュニティ | 5x |
| コミュニケーション頻度 | 年賀状・時々の連絡 | 日常的な近況共有 | 8x |
| 興味関心での出会い | オフライン限定 | コミュニティ機能 | 10x |
| プライバシー | 完全公開 or 非公開 | 友達の友達までの公開範囲設定 | 5x |

**MVP**:
- タイプ: prototype（社内開発者Batara Kesuma氏が米国SNS「Friendster」を参考に開発）
- 初期反応: 開発スタッフが知人を招待し、口コミで拡大
- CVR: 招待された人の高い登録率（招待制のため）

**UVP（独自の価値提案）**:
- 「リアルな人間関係をオンラインで再現し、日常的なコミュニケーションを促進する」
- 招待制により安心・安全な空間を提供
- 日記・足跡・コミュニティによる日本人のコミュニケーション習慣に合った設計

**競合との差別化**:
- 米国Friendster: 人間関係を可視化するだけ → mixi: 日記・コミュニティなどコミュニケーション機能を追加
- 2ちゃんねる: 匿名・荒れやすい → mixi: 招待制で安心

## 3. ピボット/失敗経験

### 3.1 初期の失敗

- Find Job!からmixiへの事業多角化は成功したが、mixi自体は2010年代に大きく衰退
- **足跡機能の廃止（2011年6月）**: 26万人以上が反対コミュニティに参加、1.7万人の署名が集まる
- **招待制廃止（2006年頃）**: スパムアカウントや業者が流入し、サービスの特別感が薄れた
- **他社模倣**: Facebookの「いいね」ボタン、Twitterの「つぶやき」機能を取り込み、独自性を喪失
- **リアルグラフへのシフト（2011-2012年）**: 親しい友人間の繋がりを重視する方針転換により、Facebookとの差別化が困難に

### 3.2 ピボット（該当する場合）

**ピボット1: Find Job! → mixi（2004年）**
- 元のアイデア: ITベンチャー向け求人情報サイト
- ピボット後: SNS「mixi」
- きっかけ: 社内インターンのBatara Kesuma氏がFriendsterを見て「留学生の間でSNSが流行している。開発したい」と提案
- 学び: 新規サービスへの挑戦が成長をもたらす

**ピボット2: mixi衰退 → モンスターストライク（2013年）**
- 元の状態: mixiのユーザー数・PV急減
- ピボット後: スマホゲーム「モンスターストライク」
- きっかけ: mixiチームのゲーム開発への挑戦
- 学び: SNSで培ったコミュニケーション設計のノウハウをゲームに活用

**ピボット3: 新規事業 → みてね（2015年）**
- 元のアイデア: 様々な新規事業検討
- ピボット後: 家族向け写真共有アプリ「みてね」
- きっかけ: 笠原氏自身の子育て経験（子どもの写真が急増し、家族との共有が困難）
- 学び: 自分自身のペインポイントから優れたサービスが生まれる

## 4. 成長戦略

### 4.1 初期トラクション獲得

- **招待制**: 開発スタッフが知人を招待し、信頼性の高いユーザーベースを構築
- **口コミ拡散**: 招待された人が自分の友人を招待する連鎖
- **足跡機能**: 誰が自分のページを訪問したかわかることで、新たな繋がりが生まれやすい
- **コミュニティ**: 共通の興味関心でユーザー同士が繋がる

### 4.2 フライホイール

```
招待制による信頼性
    ↓
ユーザーが安心して日記を書く
    ↓
足跡機能でリアクションがわかる
    ↓
コミュニケーションが活性化
    ↓
コミュニティで新たな出会い
    ↓
友達を招待したくなる
    ↓
（繰り返し）
```

### 4.3 スケール戦略

- 2004年2月: mixi開始
- 2006年7月: 登録ユーザー500万人突破
- 2006年9月: 東証マザーズ上場（時価総額約2,000億円）
- 2007年7月: 登録ユーザー1,000万人突破
- 2008年1月: 登録ユーザー1,300万人突破
- 2013年10月: モンスターストライク リリース
- 2015年: みてね リリース
- 2024年12月: mixi2 リリース（1週間で120万ユーザー突破）

## 6. 成功要因分析

### 6.1 主要成功要因

1. **日本市場に最適化されたSNS設計**: 招待制・足跡・日記・コミュニティなど、日本人のコミュニケーション習慣に合った機能
2. **連続的なヒット創出能力**: mixi → モンスターストライク → みてね → mixi2と、継続的に新サービスを開発
3. **自身の課題からの事業創出**: Find Job!（アルバイト探しの不便さ）、みてね（子どもの写真共有の不便さ）

### 6.2 タイミング要因

- 2004年: 日本でブロードバンドが普及し、SNS利用の土壌が整った
- 2013年: スマホゲーム市場の急成長期にモンスト投入
- 2024年: X（旧Twitter）の混乱期にmixi2投入

### 6.3 差別化要因

- **クローズドな安心空間**: 招待制により荒らしやスパムを排除
- **日本語でのきめ細やかなUX**: 足跡機能など日本人向けの設計
- **継続的なイノベーション**: 1つのサービスに固執せず、新規事業に挑戦

## 8. orchestrate-phase1への示唆

### 8.1 需要発見（/discover-demand）

- **自身の日常体験から課題を発見**: 笠原氏はアルバイト探しの不便さ（Find Job!）、子どもの写真共有の困難さ（みてね）から事業を創出
- **グローバルトレンドの日本適用**: 米国Friendsterを見てmixiを着想。海外で成功したモデルを日本市場に最適化

### 8.2 CPF検証（/validate-cpf）

- **ユーザー行動の観察**: 足跡廃止時の26万人の反対コミュニティ形成は、機能の価値を証明
- **WTPより利用頻度を重視**: 無料SNSのため、課金よりもDAU/MAUを重視した検証

### 8.3 PSF検証（/validate-10x）

- **10倍優位性の軸**:
  - 信頼性（招待制による安心感）
  - コミュニケーション頻度（日記・足跡機能）
  - 興味関心での出会い（コミュニティ機能）
- **独自機能の重要性**: 足跡機能廃止による衰退は、独自機能が競争優位性の源泉であることを示す

## 9. 事業アイデア候補

この事例から着想を得られる日本向けビジネスアイデア:

1. **特定コミュニティ向けクローズドSNS**: ママ友、趣味仲間、同窓会など特定グループ向けの招待制SNS
2. **世代間コミュニケーションアプリ**: 「みてね」のように祖父母世代も簡単に使える家族向けサービス
3. **リアルイベント連動型SNS**: 足跡機能のようなリアルタイムのインタラクションを重視したサービス
