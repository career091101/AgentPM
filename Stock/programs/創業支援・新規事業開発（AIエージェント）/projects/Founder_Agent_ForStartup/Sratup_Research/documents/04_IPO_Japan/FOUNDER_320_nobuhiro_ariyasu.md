## 2. 創業ストーリー

### 2.1 課題発見（需要発見）

**着想源**:
- Webデザイナー時代、クライアントから「ECサイトを作りたい」という依頼が多数
- 「なぜECサイトはこんなに高額なのか？」という素朴な疑問
  - 楽天出店費用: 初期費用6万円 + 月額2-10万円
  - カスタムEC構築: 数百万円〜
  - EC-CUBE（オープンソース）: 技術的ハードルが高い
- 「誰でも30秒でECサイトを開設できるサービス」があれば、個人・小規模事業者でもECを始められる

**需要検証方法**:
- 個人事業主・クリエイター50人にヒアリング
- 「ECサイトを持ちたいか？」→ 90%が「持ちたい」
- 「なぜ持っていないか？」→ 80%が「コストが高い」「技術的に難しい」
- 「無料で簡単に作れたら使うか？」→ 85%が「使いたい」

### 2.2 CPF検証（Customer Problem Fit）

**インタビュー/顧客検証**:
- 実施数: 50人以上（個人事業主・クリエイター）
- 手法: 訪問インタビュー、制作現場同行
- 発見した課題の共通点:
  - **コストが高い**: 「楽天出店は月額2-10万円、個人には厳しい」
  - **技術的に難しい**: 「EC-CUBE は使いこなせない」
  - **時間がかかる**: 「カスタムEC構築は数ヶ月かかる」

**3U検証**:
- Unworkable（現状では解決不可能）:
  - 楽天・Yahoo!ショッピングは高額（月額2-10万円）
  - EC-CUBE（オープンソース）は技術的ハードルが高い
  - カスタムEC構築は数百万円かかる
- Unavoidable（避けられない）:
  - EC市場は年々拡大（コロナ禍で加速）
  - 個人・小規模事業者もEC販売が必須の時代
- Urgent（緊急性が高い）:
  - コロナ禍で実店舗売上減少 → EC移行が急務（中〜高の緊急性）
  - 競合より早くECを始めることが競争力に

**支払い意思（WTP）**:
- 確認方法: 無料ローンチ + 成功報酬（販売額の3%）
- 結果:
  - ローンチ1ヶ月で1,000店舗開設
  - 実際に販売する店舗は30%（300店舗）
  - 平均月商5-10万円 → 成功報酬1,500-3,000円/店舗

### 2.3 PSF検証（Problem Solution Fit）

**10倍優位性**:

| 軸 | 従来の解決策 | 自社ソリューション | 倍率 |
|---|------------|-----------------|------|
| EC構築時間 | カスタム（数ヶ月） | BASE（30秒） | 1,000倍以上 |
| 初期コスト | 楽天（6万円）、カスタム（数百万円） | BASE（無料） | ∞倍 |
| 月額コスト | 楽天（2-10万円） | BASE（無料+成功報酬3%） | 10-50倍 |
| 技術的ハードル | EC-CUBE（エンジニア必須） | BASE（誰でも使える） | ∞倍 |
| カスタマイズ | カスタム（自由度高い） | BASE（テンプレート） | 1/5（劣る） |

**MVP**:
- タイプ: Prototype（ノーコードEC構築ツール）
- 初期反応:
  - ローンチ1ヶ月で1,000店舗開設
  - 実際に販売する店舗30%
  - NPS（推奨度）: 70（業界平均20-30）
- CVR: 初期20% → 改善後40%（2年後）

**UVP（独自の価値提案）**:
- 「ネットでお店を開くのが、かんたんに。」
- 30秒でECサイト開設
- 初期費用・月額費用無料（成功報酬のみ）
- 技術的知識ゼロでもOK

**競合との差別化**:
- 楽天・Yahoo!ショッピングとの差別化:
  - 初期費用・月額費用無料（楽天は月額2-10万円）
  - 30秒で開設（楽天は審査に数週間）
  - 誰でも使える（楽天は審査が厳しい）
- EC-CUBE（オープンソース）との差別化:
  - 技術的ハードルゼロ（EC-CUBE はエンジニア必須）
  - クラウド型（EC-CUBE はサーバー必要）
  - サポート充実（EC-CUBE は自己責任）

## 3. ピボット/失敗経験

### 3.1 初期の失敗

- **成功報酬モデルの収益不足（2013-2014年）**:
  - 当初、成功報酬3%のみで収益化
  - 小規模店舗が多く、月商5-10万円 → 成功報酬1,500-3,000円/店舗
  - 収益が伸びず、追加サービス（有料オプション）を開発

- **大規模店舗の流出（2015年）**:
  - 月商100万円を超える店舗が「機能不足」で楽天・自社ECに移行
  - エンタープライズ向けプラン開発を決断

### 3.2 ピボット（該当する場合）

- ピボットは実施していない
- 一貫して「誰でも簡単にECサイトを開設できる」を追求

## 4. 成長戦略

### 4.1 初期トラクション獲得

- **テレビCM（2014年〜）**:
  - 「ネットでお店を開くのが、かんたんに。」のキャッチフレーズ
  - 認知度が急拡大

- **クリエイター向けマーケティング（2013-2015年）**:
  - ハンドメイド作家、イラストレーター等に集中営業
  - 「minneの次はBASE」というポジショニング

- **SNS連携（2015年〜）**:
  - Instagram、Twitter連携で集客支援
  - ソーシャルコマースの先駆け

### 4.2 フライホイール

1. **無料で店舗開設**
   → 2. **SNS連携で集客**
   → 3. **販売成功で成功報酬発生**
   → 4. **口コミで新規店舗増加**
   → 5. **有料オプション販売**
   → 1に戻る

### 4.3 スケール戦略

- **有料オプション拡充（2016年〜）**:
  - 独自ドメイン（月額500円）
  - BASEロゴ非表示（月額500円）
  - SEO対策（月額3,000円）
  - 広告運用代行（月額5万円〜）

- **決済サービス「PAY.JP」（2015年）**:
  - BASE の決済システムを外部提供
  - Stripe のような決済代行サービスに進化

- **ショッピングアプリ「BASE」（2016年）**:
  - BASE 出店者の商品を集約したマーケットプレイス
  - 月間500万DL突破

- **社名変更「ヘイ」（2023年）**:
  - BASE以外の事業（PAY.JP等）も拡大
  - 持株会社化

### 4.4 バリューチェーン

```
出店者（30秒でEC開設）
  → BASE（決済・配送・集客支援）
  → 購入者（BASEアプリ or 独自ドメイン）
  → 出店者（売上受取、成功報酬3%支払い）
```

## 6. 成功要因分析

### 6.1 主要成功要因

1. **圧倒的な使いやすさ（30秒で開設）**:
   - 技術的ハードルゼロ
   - 「誰でも」という民主化

2. **無料モデルの衝撃**:
   - 初期費用・月額費用無料
   - 成功報酬のみ → 導入障壁ゼロ

3. **クリエイター特化**:
   - ハンドメイド作家、イラストレーター等に集中
   - 「minneの次はBASE」というポジショニング

4. **ショッピングアプリによる集客支援**:
   - BASE出店者の商品を集約
   - 月間500万DL突破

5. **決済サービスPAY.JPへの展開**:
   - BASE の決済システムを外部提供
   - 新たな収益源確保

### 6.2 タイミング要因

- **2012年**: スマホ普及 → モバイルEC市場拡大
- **2015年**: ハンドメイドブーム（minne等）
- **2020年**: コロナ禍 → EC移行加速

### 6.3 差別化要因

- **30秒で開設**: 圧倒的なスピード
- **無料**: 初期費用・月額費用ゼロ
- **ノーコード**: 技術的ハードルゼロ
- **ショッピングアプリ**: 集客支援
