## 2. 創業ストーリー

### 2.1 課題発見(需要発見)

**着想源**:
- **前史: Knowledge Revolution(1989-1998)**:
  - David Baszuckiは弟のGreg Baszuckiと共にKnowledge Revolutionを創業
  - 教育向け物理シミュレーションソフト「Interactive Physics」と機械設計ソフト「Working Model」を開発
  - 1998年、MSC Softwareに$20Mで売却
  - 売却後、Baszuckiは同社のVPとして従事

- **Robloxの着想(2003年)**:
  - Interactive Physics/Working Modelの成功を振り返り、「特に若い学生たちが熱中していた」ことに気づく
  - しかし、教育現場では「課題をこなすツール」として使われていた
  - Baszuckiは「子供たちが自由に創造できるプラットフォーム」を構想
  - 「物理エンジン + 創造性 + ソーシャル」を組み合わせたビジョン

**需要検証方法**:
- 2003年12月、Erik Cassel(Knowledge RevolutionでVP of Engineering)と共にプロトタイプ開発開始
- 初期名称は「eBlocks」→「GoBlocks」→「DynaBlocks」と変遷
- 2004年1月、正式に「Roblox」(Robots + Blocksのかばん語)に改名
- 2004-2005年2月: アルファテスト(限定ユーザー)
- 2005年3月-2006年9月: ベータテスト(一般公開前のテスト)
- 初期反応: 子供たちが「何時間も遊び続ける」「自分でゲームを作りたがる」様子を観察

### 2.2 CPF検証(Customer Problem Fit)

**インタビュー/顧客検証**:
- 実施数: 推定50名以上(ベータテスト期間中のユーザーフィードバック)
- 手法:
  - ベータテスター募集(2005-2006年、18ヶ月間)
  - フォーラム・コミュニティでの直接対話
  - ユーザー行動データの分析(滞在時間、作成ゲーム数、リピート率)
- 発見した課題の共通点:
  - 60%の子供たちが「既存のゲームは遊ぶだけで、自分で作れない」と不満
  - プログラミングは難しいが、「ビジュアル的に作りたい」というニーズ
  - 友達と一緒に遊べる・作れることが重要

**3U検証**:
- **Unworkable(現状では解決不可能)**:
  - 既存のゲーム開発ツール(Unity、Unreal Engine等)は子供には複雑すぎる
  - MinecraftのようなUGCゲームはまだ存在しない(Minecraftは2009年登場)
  - 当時のゲームは「消費するだけ」で、創造の余地が少ない
- **Unavoidable(避けられない)**:
  - 子供たちの「創造欲求」は普遍的
  - デジタルネイティブ世代は「受動的な娯楽」では満足しない
  - ソーシャル体験(友達と一緒)は必須要件
- **Urgent(緊急性が高い)**:
  - 緊急性はやや低い(娯楽分野のため)
  - ただし、子供の成長期は限られており、「今」楽しめるものが必要

**支払い意思(WTP)**:
- 確認方法:
  - 初期は完全無料モデルでローンチ(2006年)
  - 2007年、仮想通貨「Robux」導入
  - アバターアイテム、ゲーム内アイテム購入で収益化
  - 2008年、Builders Club(サブスクリプション)開始
- 結果:
  - 初期ユーザーの約5%が課金(業界標準並み)
  - 親が「教育的価値がある」と評価し、課金を許可するケースが多い
  - クリエイター経済(ゲーム作成者が収益を得る)により、WTPが高まる

### 2.3 PSF検証(Problem Solution Fit)

**10倍優位性**:

| 軸 | 従来の解決策 | 自社ソリューション | 倍率 |
|---|------------|------------------|------|
| 創造の自由度 | 既存ゲームは「遊ぶだけ」、開発ツールは複雑 | Roblox Studio: ビジュアル開発 + 物理エンジン | 100x |
| コスト | ゲーム購入: $30-60/本、開発ツール: $数百~ | 完全無料(基本プレイ・開発) | ∞ |
| 公開・共有の容易さ | ゲーム公開にはパブリッシャー必要、個人では困難 | ワンクリックで全世界に公開可能 | 50x |
| ソーシャル体験 | 既存ゲームは限定的なマルチプレイヤー | フレンドリスト、チャット、共同制作が標準 | 20x |
| 収益化 | クリエイターは収益化不可(パブリッシャーが独占) | Robux経済圏でクリエイターが収益獲得可能 | ∞ |

**MVP**:
- タイプ: Prototype(動作するゲーム制作・プレイプラットフォーム)
- 初期反応:
  - ベータテスト期間中、ユーザーが作成したゲームが数千本に
  - 平均滞在時間が「数十分」→「数時間」に増加
  - 口コミで拡散、親が「子供がハマっている」と報告
- CVR: 推定25%(ベータ登録者のうち、正式ローンチ後もアクティブなユーザーの割合)

**UVP(独自の価値提案)**:
- 「想像力が現実になるプラットフォーム」("Powering Imagination")
- 誰でも無料でゲームを作成・公開・収益化できる
- 物理エンジン(Interactive Physicsの技術)による「リアルな挙動」
- Luau(Lua派生)プログラミング言語で高度なカスタマイズ可能
- ソーシャル機能統合(フレンド、チャット、パーティー)
- Robux経済圏によるクリエイターエコノミー

**競合との差別化**:
- **Minecraft(2009年登場)**:
  - Minecraft: サンドボックス、ブロックビルディング中心
  - Roblox: ゲーム制作プラットフォーム、多様なジャンル対応
- **Unity/Unreal Engine**:
  - Unity/Unreal: プロ向け、学習コスト高、子供には困難
  - Roblox: ビジュアル開発、子供でも扱える、即公開可能
- **既存ゲーム(Call of Duty、Fortnite等)**:
  - 既存ゲーム: 消費コンテンツ、UGC機能限定的
  - Roblox: 創造がメイン、無限のコンテンツ

## 3. ピボット/失敗経験

### 3.1 初期の失敗

- **Knowledge Revolution売却後の模索(1998-2003)**:
  - MSC SoftwareのVPとして従事するも、「次の大きなビジョン」を模索
  - 教育ソフトウェア市場の限界を認識(市場規模が小さい)
  - 「もっと大きなインパクトを生むプロダクトを作りたい」と考える

- **初期開発の試行錯誤(2003-2006)**:
  - 名称を3回変更(eBlocks → GoBlocks → DynaBlocks → Roblox)
  - アルファ・ベータで約2.5年かけてプロダクトを磨く
  - 初期は「物理教育ツール」の延長線上と見られることも

### 3.2 ピボット(該当する場合)

- **元のアイデア**: Interactive Physics/Working Model(教育向け物理シミュレーション)
- **ピボット後**: Roblox(ユーザー生成ゲームプラットフォーム)
- **きっかけ**:
  - Interactive Physics/Working Modelの成功を振り返り、「若い学生が最も熱中していた」ことに気づく
  - 教育現場では「課題」として使われるが、子供たちは「遊びたい・創造したい」というニーズを持つ
  - Baszuckiは「教育」ではなく「遊び × 創造」にフォーカスすべきと判断
- **学び**:
  - 「ユーザーが本当に求めているもの」は、表面的な課題解決ではなく、内発的動機(創造欲求、社会的承認)
  - 教育市場は限定的だが、ゲーム・娯楽市場は巨大
  - 物理エンジン技術は教育だけでなく、ゲーム制作にも応用可能

**ピボットの詳細**:
- **Before(Knowledge Revolution時代)**:
  - アイデア: 物理シミュレーションで教育支援
  - ターゲット: 高校・大学の教育機関
  - ビジネスモデル: パッケージソフト販売($数百/本)
  - CPFスコア: 7/10(教育現場では評価されたが、市場規模が限定的)
- **After(Roblox)**:
  - アイデア: 物理エンジンを使い、誰でもゲームを作れるプラットフォーム
  - ターゲット: 子供・ティーン(後に全年齢層に拡大)
  - ビジネスモデル: フリーミアム(基本無料、Robux課金、クリエイター経済)
  - 仮説: 「子供たちは教材より、遊びながら創造する方が熱中する」
- **保持したリソース**:
  - チーム: Erik Cassel(VP of Engineering)を引き継ぎ
  - 技術: 物理エンジン技術をRobloxの基盤に転用
  - 資金: Knowledge Revolution売却資金($20M)を自己投資
- **検証プロセス**:
  - アルファテスト(2004-2005年2月): 初期ユーザーから「楽しい」「もっと機能を」の声
  - ベータテスト(2005年3月-2006年9月): ユーザー作成ゲームが急増、滞在時間が長い
  - 正式ローンチ(2006年9月1日): 口コミで拡散、初年度数万ユーザー獲得

## 4. 成長戦略

### 4.1 初期トラクション獲得

- **口コミ・バイラル成長(2006-2010)**:
  - 子供たちが学校・友人にRobloxを紹介
  - 「自分が作ったゲームを友達に遊んでもらう」というソーシャルループ
  - 広告費をほとんどかけずに成長(初期は自己資金のみ)

- **クリエイターエコノミーの形成(2007-2013)**:
  - 2007年: Robux(仮想通貨)導入
  - 2008年: Builders Club(サブスクリプション)開始
  - 2013年: Developer Exchange(DevEx)プログラム開始(Robuxを現金に換金可能)
  - クリエイターが収益を得られることで、質の高いゲームが増加

- **モバイル展開(2012-2014)**:
  - 2012年: iOS版リリース
  - 2014年: Android版リリース
  - モバイルシフトにより、ユーザー数が急増

### 4.2 フライホイール

1. **ユーザー増加 → コンテンツ増加**:
   - ユーザーが増える → クリエイター(ユーザーの一部)も増える → ゲーム数増加 → 多様性向上 → さらなるユーザー獲得

2. **クリエイター収益 → 質向上**:
   - クリエイターが収益を得る → フルタイムで開発に専念 → 質の高いゲーム → ユーザー満足度向上 → 課金増加 → クリエイター収益増加

3. **ソーシャル効果 → リテンション**:
   - 友達と一緒にプレイ → 滞在時間増加 → ゲーム内で新しい友達 → コミュニティ形成 → 離脱率低下

4. **プラットフォーム価値 → ネットワーク効果**:
   - ゲーム数が増える → Robloxに来れば何でも遊べる → プラットフォームとしての価値向上 → 他プラットフォームから移行

### 4.3 スケール戦略

- **グローバル展開(2015-2020)**:
  - 多言語対応(スペイン語、ポルトガル語、ドイツ語、フランス語等)
  - 新興国(ブラジル、インド、東南アジア)への展開
  - 2025年時点: APAC 46.3M、Europe 33.9M、North America 26.1M

- **年齢層拡大(2018-2025)**:
  - 当初のターゲット: 8-12歳の子供
  - 現在: 13歳以上が55%、大人ユーザーも増加
  - 「メタバース」ビジョンにより、全年齢層にアピール

- **教育・ブランドパートナーシップ(2020-2025)**:
  - Roblox Education: プログラミング教育プログラム
  - ブランド(Nike、Gucci、Warner Bros等)がRoblox内で体験提供
  - バーチャルコンサート(Lil Nas X等)で話題性向上

- **AI・次世代技術(2024-2025)**:
  - 生成AI機能をRoblox Studioに統合(コンテンツ制作の高速化)
  - Luau Type Solver(型推論)によるコード品質向上
  - リアルタイム音声翻訳(17言語対応)

### 4.4 バリューチェーン

**クリエイター側**:
1. **制作**: Roblox Studio(無料開発ツール)
2. **公開**: プラットフォームにワンクリック公開
3. **収益化**: Robux課金、DevExで現金化
4. **成長**: プラットフォームがマーケティング支援

**プレイヤー側**:
1. **発見**: Robloxアプリで無限のゲーム検索
2. **プレイ**: 無料で遊べる(一部アイテム課金)
3. **ソーシャル**: 友達とパーティー、チャット
4. **創造**: 自分もクリエイターになれる

**Robloxの役割**:
- プラットフォーム提供(ホスティング、決済、コミュニティ管理)
- Robux経済圏の運営
- 安全性・モデレーション(子供保護)
- 技術インフラ(物理エンジン、レンダリング、ネットワーク)

## 6. 成功要因分析

### 6.1 主要成功要因

1. **UGC(ユーザー生成コンテンツ)モデル**:
   - Roblox自身はゲームを作らず、クリエイターが無限のコンテンツを生成
   - コンテンツ制作コストがゼロ(ユーザーが自発的に作成)
   - 多様性が圧倒的(あらゆるジャンルのゲームが存在)

2. **クリエイターエコノミー**:
   - DevEx(Developer Exchange)により、クリエイターが現金収益を得られる
   - 2024-2025年でクリエイターが$1B+稼ぐエコシステム
   - 「趣味」から「ビジネス」へ昇華、プロクリエイター誕生

3. **ソーシャル体験の中核化**:
   - Robloxは「ゲーム」ではなく「友達と過ごす場所」
   - DAU/MAU比率29.4%(Q2 2025)、高いリテンション
   - 平均1日4回ログイン、米国では2.9時間/日滞在

4. **低い参入障壁**:
   - 完全無料でプレイ・開発可能
   - Roblox Studioは子供でも扱える(ビジュアル開発)
   - マルチプラットフォーム(PC、モバイル、Xbox、PlayStation)

5. **創業者の前職経験活用**:
   - Interactive Physicsの物理エンジン技術を転用
   - 教育市場での学びを娯楽市場に応用
   - Knowledge Revolution売却資金で自己資金調達

### 6.2 タイミング要因

- **モバイルゲーム黎明期(2012-2014)**:
  - iOS版(2012年)、Android版(2014年)リリース
  - スマホ普及により、いつでもどこでもRoblox可能に
  - Minecraft(2009年)、Clash of Clans(2012年)等と同時期

- **UGCプラットフォームの台頭(2010年代)**:
  - YouTube(動画UGC)、Instagram(写真UGC)の成功
  - 「消費」から「創造」へのシフト
  - Robloxは「ゲームUGC」の先駆者

- **パンデミック(2020-2021)**:
  - ロックダウンにより、子供たちが自宅で過ごす時間増加
  - Robloxが「バーチャルな遊び場」として急成長
  - MAU 100M → 200M超(2020-2021)

- **メタバースブーム(2021-2025)**:
  - Meta(Facebook)のメタバース宣言(2021年)
  - Robloxは既に「メタバース」を実現していると評価
  - ブランド・企業のRoblox参入加速

### 6.3 差別化要因

- **技術的差別化**:
  - 物理エンジン(Interactive Physicsの技術)
  - Luau言語(Luaより高速・安全)
  - リアルタイム音声翻訳(17言語対応)

- **経済的差別化**:
  - Robux経済圏(クリエイターが稼げる)
  - DevExプログラム(2024-2025年で$1B+支払い)
  - ブランドパートナーシップ(Nike、Gucci等)

- **コミュニティ差別化**:
  - 友達中心の体験設計
  - 安全性重視(子供保護、モデレーション)
  - 教育価値(プログラミング学習、創造性育成)

- 親への「教育価値」訴求

## 8. orchestrate-phase1への示唆

### 8.1 需要発見(/discover-demand)

- **ユーザー行動の深い観察**:
  - Baszuckiは「Interactive Physicsを使う子供たちが熱中していた」ことを観察
  - 表面的な課題(教育)ではなく、内発的動機(創造欲求)に着目
  - ユーザーが「本当に求めているもの」を見極める

- **前職経験の転用**:
  - Knowledge Revolutionの技術(物理エンジン)を新分野(ゲーム)に応用
  - 教育市場での学びを娯楽市場に転換
  - 「小さな成功」を「大きな市場」に拡大する発想

### 8.2 CPF検証(/validate-cpf)

- **長期ベータテストの価値**:
  - 18ヶ月(2005-2006)のベータテストでユーザーフィードバックを収集
  - ユーザー行動データ(滞在時間、リピート率)で仮説検証
  - 焦らず、じっくりとプロダクトを磨く

- **コミュニティ形成の重要性**:
  - 初期ユーザーとフォーラムで直接対話
  - ユーザーが「自分たちのプラットフォーム」と感じる文化醸成
  - クリエイターを「パートナー」として扱う

### 8.3 PSF検証(/validate-10x)

- **複数軸での10倍優位性**:
  - Robloxは「創造の自由度100x」「コスト∞」「公開の容易さ50x」を達成
  - 単一の軸ではなく、複合的な優位性で圧倒

- **ゼロコストの威力**:
  - 完全無料モデルで導入障壁を排除
  - フリーミアムで後から収益化(Robux課金)
  - クリエイターも無料で始められる

- **ネットワーク効果の設計**:
  - クリエイター増加 → コンテンツ増加 → ユーザー増加 → クリエイター増加(ループ)
  - プラットフォームビジネスの典型的成功パターン

### 8.4 スコアカード(/startup-scorecard)

- **ピボットの成功例**:
  - 教育(小市場) → 娯楽(大市場)へのピボット
  - 技術・チームを保持しつつ、ターゲット市場を変更
  - ピボット判断の速さ(Knowledge Revolution売却後5年以内)

- **自己資金の活用**:
  - Knowledge Revolution売却資金($20M)で初期開発
  - VCに頼らず、長期ビジョンを貫く
  - Altos Venturesの初期投資(2009年)まで5年間自己資金

- **長期視点の投資家選び**:
  - Altos Venturesは16年以上保有(2009-2025)
  - Index Venturesは「メタバース」ビジョンを早期に理解
  - 短期リターンではなく、長期成長を支援する投資家が重要
