# Amazon - 赤字許容のスケール戦略

## 概要

Amazonの初期段階（IPO後）における「戦略的赤字運営」によるスケール戦略。利益より成長を優先し、市場シェア獲得を実現した事例。

## バーンレート推移

### Phase 1: IPO直後（1997年）
- **IPO調達**: $54,000,000
- **年次支出**: $6,000,000（初期インフラ）
- **年次収益**: $150,000（初期ユーザー）
- **年間赤字**: $5,850,000
- **Runway**: 9年以上

### Phase 2: グローバル拡張（1999-2000年）
- **調達額**: $1,250,000,000（複数ラウンド）
- **年次支出**: $50,000,000（国際展開）
- **年次収益**: $10,000,000（初期手数料）
- **年間赤字**: $40,000,000
- **Runway**: 31年以上（十分すぎる）

### Phase 3: ドットコム崩壊後（2001-2003年）
- **年次支出**: $100,000,000（インフラ投資継続）
- **年次収益**: $50,000,000（段階的成長）
- **年間赤字**: $50,000,000（継続）
- **Runway**: 不確定（資金調達不可）

### Phase 4: 黒字化へ（2003年以降）
- **年次支出**: $100,000,000（抑制）
- **年次収益**: $100,000,000+（成長継続）
- **年間赤字**: $0 → 黒字化
- **評価額**: $100B+ （市場評価による自動調達）

## 「赤字を恐れない」VC投資家の支援

### 赤字継続の正当化

```
Amazon の場合:
- 1997-2003年: 6年連続赤字
- 累積赤字: 数億ドル
- でも投資家は支援継続

理由:
1. 市場シェア獲得が最優先
2. 「顧客満足」重視で、長期収益性を予測
3. スケーラブルなビジネスモデル（インターネット）
```

## 「顧客中心主義」がバーンレート管理を超越

### Bezos の経営哲学

```
利益最大化 → 短期的
顧客満足最優先 → 長期的

Amazon は顧客満足の為なら、
短期赤字も許容
→ 長期的には最大利益へ
```

## バーンレート監視への教訓

1. **業界によってはバーンレート管理の優先度が低い**
   - インターネット黎明期（1990年代）
   - 市場シェア獲得が最重要
   - 赤字は「投資」

2. **ランウェイが十分なら、赤字戦略も可能**
   - Amazon: 30年以上のランウェイ
   - 7-8年の赤字運営も可能

3. **VC投資家の支援が鍵**
   - 赤字継続でも、投資家が次ラウンド調達を支援
   - Bezos のビジョンと戦略性が評価された
