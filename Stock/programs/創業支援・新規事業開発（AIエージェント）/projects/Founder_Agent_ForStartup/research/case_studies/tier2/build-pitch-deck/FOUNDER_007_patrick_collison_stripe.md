# Stripe: ピッチデッキ成功事例

## 企業基本情報
- **企業名**: Stripe
- **創業**: 2010年
- **調達**: Series A-H → Valuation $95B（2024年プライベート）
- **創業者**: Patrick Collison (Co-founder & CEO) & John Collison

## ピッチデッキ構成の成功ポイント

### Problem スライド
- **課題**: オンライン決済が複雑（複数のプロバイダー、複雑な統合）
- **ターゲット**: スタートアップ、オンラインストア、SaaS
- **3Uスコア**: Urgency 9/10（e-commerce 爆発の波）

### Solution スライド
- **UVP**: 「30行のコードで決済機能を実装」
- **Developer-first**: API のシンプルさが競争優位（Stripe の本質）
- **Before/After**: 複雑な決済統合 vs 数行のコード

### Market Size スライド
- **TAM**: グローバル e-commerce 市場 $5T+
- **決済処理市場**: $100B+ のレベニュー規模
- **Why Now**: e-commerce 成長 + API-first 開発への転換

### Traction スライド
- **初期成長**: Series A 以降、ユーザー数が指数増
- **年間処理額（TPV）**: 初年 $1M → 成長に応じて指数増
- **顧客セグメント**: Lyft, Uber, DoorDash, Slack など超大型顧客獲得
- **チャーン率**: 極めて低い（スイッチングコスト高）

### Business Model スライド
- **Commission-based**: トランザクション手数料 2.9% + $0.30
- **Expansion**: 高 LTV のため ARR が自動増加（顧客の売上増 = Stripe 収益増）
- **B2B2C**: Stripe が間接的に顧客の顧客と接する（スケーラビリティ）

### Competition スライド
- **競合**: PayPal, Square, Adyen
- **10倍優位性**:
  1. **Developer Experience**: API が 10倍シンプル
  2. **統合速度**: 数時間 vs 数週間（PayPal比）
  3. **グローバル**: 100+ 国サポート（多くの競合は限定的）
- **参入障壁**: Developer Ecosystem + Network Effect

## ピッチデッキスコア: 117/130点（優秀）

### 強み
- **Developer-first 戦略**: スタートアップコミュニティからのウイルス成長
- **シンプルさの実証**: 「30行のコード」という具体例
- **大型顧客獲得**: Lyft, Uber, Slack 等の実績が説得力を与える

### VC評価ポイント
- **Patrick Collison の起業家スピリット**: 若い才能が作る elegant solution
- **TAM の大きさ**: 決済市場の永続的需要
- **Network Effect**: Developer が増える → API がより価値的 → さらに Developer が増える

## このスキルでの活用方法

- **Developer-first アプローチ**: 技術者向けの複雑な価値提案を明確化
- **統合速度**: 導入時間を競合と比較した図表
- **B2B2C モデル**: Stripe が直接タッチしない顧客をスケール
- **グローバル展開**: 多国家対応による市場拡張
