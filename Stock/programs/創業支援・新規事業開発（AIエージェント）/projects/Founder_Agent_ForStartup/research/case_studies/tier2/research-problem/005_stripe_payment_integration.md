# Stripe: オンライン決済の複雑性発見

**企業**: Stripe
**創業者**: Patrick Collison, John Collison
**業界**: フィンテック / 決済処理
**CPFスコア**: 81%
**実行時期**: 2009-2010年

## 課題発見プロセス

### 1. 定性リサーチ手法
- **手法**: スタートアップ・ECサイト企業への直接インタビュー + Developer ネットワーク調査
- **対象数**: 70+企業（Shopify、O2O サービス、SaaS企業等）
- **期間**: 1年間（2009年秋-2010年秋）
- **背景**: Collison 兄弟は大学時代に決済システム構築を試みた経験あり

### 2. 発見した課題

**スタートアップ経営者の課題**:
```
"オンライン決済を導入したいが、複数の決済処理業者と契約が必要。
各社ドキュメントがバラバラ、API も異なる。統合に3ヶ月かかる"
→ 欲求: 統一されたAPI、簡単な統合、複数決済手段の一元管理
```

**デベロッパーの課題**:
```
"Authorize.net、PayPal の API を比較してみたが、ドキュメント不親切。
PCI コンプライアンスの対応も不明確"
→ 欲求: 分かりやすいドキュメント、開発者体験（DX）を重視したAPI
```

**ECサイト運営者の課題**:
```
"顧客が支払い方法を選べない。手数料も不透明。
返金処理も手作業で管理"
→ 欲求: 複数支払い方法対応、自動返金、透明な手数料
```

### 3. 3Uスコアリング（Stripeの例）

| 軸 | スコア | 根拠 |
|---|--------|------|
| **Unworkable** | 19/20 | 決済統合ができないと、ECサイトの売上がゼロ。致命的な課題 |
| **Unavoidable** | 18/20 | 既存の決済処理業者（PayPal, Authorize.net）は複雑。複数統合は必須 |
| **Urgent** | 17/20 | サイト立ち上げには決済機能が必須。遅延は許されない |
| **合計** | **54/60** | CPFスコア 90% ✅ |

### 4. 顧客ニーズの定量化

**インタビューから抽出したデータ**:

| 指標 | データ | 根拠 |
|------|--------|------|
| **支払意思（WTP）** | 2.9% + $0.30 / 取引 | 業界標準（PayPal 比は低コスト） |
| **統合時間削減** | 3ヶ月 → 1週間 | デベロッパー時間の削減効果 |
| **開発コスト削減** | $30K → $5K | マネージドサービスの効果 |
| **TAM** | グローバルデジタル決済市場 = **$2T** |
| **SAM** | USA EC市場 × 決済処理 = **$500B** |

### 5. 既存代替案への不満点

**競合分析（インタビュー結果ベース）**:

| 選択肢 | 統合難度 | API品質 | 複数決済 | 手数料 | 不満点 |
|--------|---------|--------|---------|--------|--------|
| **PayPal** | 高い | 古い | ❌ | 3.5% | ドキュメント悪い、API複雑 |
| **Authorize.net** | 高い | 古い | △ | 2.9% | セットアップ大変 |
| **2Checkout** | 中程度 | 中程度 | ✅ | 4.5% | 高手数料 |
| **Square** | 低い（カード専用） | 良い | △ | 2.75% | クレジットカードのみ |

### 6. 課題の本質理解

**表面的な課題**: 「決済処理が複雑」
↓
**本質的な課題**: 「決済処理業界は 20 年旧態依然。複数決済手段の一元化、デベロッパー体験の改善がない」

→ **洞察**: **開発者中心設計 + シンプルなAPI + 複数決済の一元化** で、デベロッパー体験を 10 倍向上させられる

### 7. マーケットセグメント

**優先顧客**:
1. **テック系スタートアップ** （API 重視、成長速度重視）
2. **SaaS 企業** （サブスクリプション決済）
3. **マーケットプレイス** （複数決済手段必須）

## スキル実行時の活用方法

### このスキルで活用されるポイント

1. **ペインポイントの具体化**:
   - 統合期間: 3ヶ月 → 1週間（効果: $30K削減）
   - 既存ツール（PayPal）の具体的な不満（ドキュメント、API複雑性）

2. **定量データの層別分析**:
   - 決済額別（$1M → $100M）でのニーズ差異
   - スタートアップ vs 大規模EC での優先度の違い

3. **既存代替案の詳細比較**:
   - PayPal（業界標準）との比較表作成
   - なぜ既存業者は改善できないのか（レガシーシステム、組織体質）

4. **市場規模推定**:
   - グローバルデジタル決済市場 $2T
   - USA EC 市場での Stripe のシェア拡張可能性

## 定量データサマリー

- **インタビュー数**: 70+企業（スタートアップ / EC / SaaS混在）
- **3Uスコア合計**: 54/60（CPF 90%）
- **WTP確認**: 2.9% + $0.30 / 取引（PayPal比で低い）
- **統合時間削減効果**: 3ヶ月 → 1週間（開発コスト $30K削減）
- **推定市場（TAM）**: $2T（グローバルデジタル決済）
