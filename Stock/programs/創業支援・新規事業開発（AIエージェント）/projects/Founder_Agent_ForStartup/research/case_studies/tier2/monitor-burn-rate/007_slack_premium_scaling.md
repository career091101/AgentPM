# Slack - SaaS規模での高バーンレート管理

## 概要

SlackのSaaS段階（Series B以降）における高バーンレート管理の実例。月次成長率20-30%を維持しながら、月数千万円の支出を管理した戦略。

## バーンレート推移

### Phase 1: Series A（2013年）
- **調達額**: $15,000,000（Accel Partners）
- **月次支出**: $200,000
- **月次収益**: $100,000（初期MRR）
- **Net Burn Rate**: $100,000/月
- **Runway**: 150ヶ月
- **判定**: ✅ 安全

### Phase 2: Series B（2014年）
- **調達額**: $100,000,000（Sequoia Capital, Google Capital）
- **月次支出**: $1,000,000（営業・マーケティング強化）
- **月次収益**: $500,000（成長加速）
- **Net Burn Rate**: $500,000/月
- **Runway**: 200ヶ月
- **判定**: ✅ 安全

### Phase 3: Series C-F（2015-2018年）
- **累積調達**: $700,000,000
- **月次支出**: $5,000,000（エンタープライズ営業）
- **月次収益**: $10,000,000+（ARR $500M+）
- **Net Burn Rate**: -$5,000,000（✅ 黒字）
- **Runway**: 無限大（キャッシュフロー黒字）

### Phase 4: IPO前（2018年）
- **ARR**: $300,000,000+
- **月次支出**: $10,000,000
- **月次收益**: $25,000,000+（黒字継続）
- **月次成長率**: 20%+（継続）
- **Runway**: 無限大

## SaaS規模での高バーンレート管理

### 支出増加の階段

```
2013年: 月$200K → ARR $1.2M
2014年: 月$1M → ARR $6M
2015年: 月$2M → ARR $20M+
2016年: 月$5M → ARR $100M+
2017年: 月$8M → ARR $300M+
2018年: 月$10M → ARR $500M+

パターン: 月次支出増加 = ARR成長率に相関
```

### LTV/CAC の長期的最適化

```
2013年（初期）:
- CAC: $5,000（エンタープライズ営業開始）
- LTV: $100,000（年間契約 × 3-5年）
- LTV/CAC: 20x

2018年（スケール）:
- CAC: $10,000（複雑なセールスサイクル）
- LTV: $500,000+（年間 × 長期契約）
- LTV/CAC: 50x+

→ スケーラビリティの証明
```

## Series C-F調達の連続実施

### 複数回調達が必要な理由

```
Serie B（2014年）: Runway 200ヶ月（十分）
         ↓
でも、グローバル展開・エンタープライズ営業強化
         ↓
Series C-Fで複数回調達

理由: ランウェイ管理というより、
      「成長加速の資金需要」が重要
```

## VC投資基準との対比

### Sequoia Series B投資判断（2014年）

| 項目 | Slack | VC基準 |
|------|----------|---------|
| **月次成長率** | 20-30% | 20%以上 |
| **LTV/CAC** | 20x | 5.0以上 |
| **Churn Rate** | < 5% | 5%以下 |
| **ARR** | $6M | 実績豊富 |

**投資判断**: ✅ Series B $100M → 2019年IPO $23B+

## バーンレート監視への教訓

1. **SaaS段階での高バーンレート許容**
   - Slackの月次支出 $1M-10M（高い）
   - でも LTV/CAC 20x+ なら許容

2. **複数回調達の戦略性**
   - ランウェイが十分でも、成長加速のため調達継続
   - **VC基準**: ランウェイ > 24ヶ月でも、次ラウンド計画を立てる

3. **キャッシュフロー黒字化への道**
   ```
   Series B: Net Burn $500K/月
   Series C: 黒字化開始
   → 黒字化まで約1-2年のランウェイ必要
   ```
