# Slack: チームコミュニケーション課題の発見

**企業**: Slack Technologies
**創業者**: Stewart Butterfield
**業界**: エンタープライズコミュニケーション
**CPFスコア**: 82%
**実行時期**: 2011-2012年

## 課題発見プロセス

### 1. 定性リサーチ手法
- **手法**: 自社プロダクト開発チーム内での課題観察 + スタートアップ企業インタビュー
- **対象数**: 60+チーム（50-200人規模）
- **期間**: 1年間（内部テスト含む）
- **背景**: Flickr / Yahoo! での経験から、大規模組織のコミュニケーション課題を熟知

### 2. 発見した課題

**スタートアップチームの課題**:
```
"メールは返信忘れが多い。クリティカルなメッセージが埋もれる。
チャットツール（IRC）はテック企業向け、非テック層が使えない"
→ 欲求: 非テック層も使える、検索可能な、チーム全体が見える永続的なログ
```

**分散チームの課題**:
```
"リモートワーカーと時間差がある。非同期でのやり取りが必要だが、
メールだと重要な情報が埋もれる"
→ 欲求: スレッド化された会話、コンテキスト保持が必要
```

**部門間コラボレーション**:
```
"営業、マーケティング、エンジニアが情報を共有できない。
社内Wiki では情報が古くなる"
→ 欲求: リアルタイム情報共有、部門横断型の検索
```

### 3. 3Uスコアリング（Slackの例）

| 軸 | スコア | 根拠 |
|---|--------|------|
| **Unworkable** | 18/20 | コミュニケーション遅延が意思決定を阻害。スタートアップでは時間が命 |
| **Unavoidable** | 17/20 | メール（返信漏れ）+ IRC（非テック層不可）+ Wiki（情報陳腐化）。完全な解決手段なし |
| **Urgent** | 18/20 | リアルタイム対応が必要。特に分散チームでは即座性が重要 |
| **合計** | **53/60** | CPFスコア 88% ✅ |

### 4. 顧客ニーズの定量化

**インタビューから抽出したデータ**:

| 指標 | データ | 根拠 |
|------|--------|------|
| **支払意思（WTP）** | $8-15/ユーザー/月 | エンタープライズスタートアップの年間IT予算 |
| **課題頻度** | 1日8時間以上 | 常時コミュニケーション |
| **メール負荷削減** | 1日2時間削減 | 対象チーム50人 × $50/時間 = $100K/年削減 |
| **TAM** | グローバル企業 3,000万社 × $10 = **$300B** |
| **SAM** | USA企業 500万社 × $10 = **$5B** |

### 5. 既存代替案への不満点

**競合分析（インタビュー結果ベース）**:

| 選択肢 | 永続ログ | 検索 | リアルタイム | UI | 不満点 |
|--------|---------|------|------------|-----|--------|
| **メール** | △ | △ | ❌ | 標準 | 返信漏れ、スレッド化されない |
| **IRC** | ❌ | ❌ | ✅ | 複雑 | テック向け、ログ保持なし |
| **Microsoft Teams** | ✅ | ✅ | ✅ | 複雑 | 高い、セットアップ大変 |
| **Yammer** | ✅ | ○ | ○ | 複雑 | 企業向け、導入困難 |

### 6. 課題の本質理解

**表面的な課題**: 「メールが遅い」
↓
**本質的な課題**: 「スタートアップは非同期 + リアルタイム両立が必要。メールは非同期向けだが、即座性を失う。IRC は即座だがログが残らない」

→ **洞察**: **永続的なログ + リアルタイム性 + シンプルなUI** を両立する選択肢がない

### 7. セグメント分析

**優先顧客**:
1. **テック系スタートアップ** （IRC ユーザー経験者）
2. **成長企業** （メール負担が高い）
3. **リモート優先企業** （分散チームの需要高い）

## スキル実行時の活用方法

### このスキルで活用されるポイント

1. **課題の深掘りインタビュー**:
   - 「どの程度の頻度で課題が発生するか」（1日8時間 → 常時課題）
   - 「既存ツールで代替できない理由」（メール + IRC の両立不可）

2. **定量データの具体化**:
   - メール対応時間の削減効果（1日2時間 × $50 × 250日 = $250K）
   - チーム規模別の市場セグメント（50-200人スタートアップ → 5,000人+エンタープライズ）

3. **既存代替案の詳細分析**:
   - メール、IRC、Teams、Yammer の比較表作成
   - 各選択肢が失敗している理由を明確化

4. **市場規模推定の精度**:
   - グローバル企業数 3,000万社 × $10 = TAM $300B
   - USA スタートアップ 100万社 × $10 = SAM 初期段階
