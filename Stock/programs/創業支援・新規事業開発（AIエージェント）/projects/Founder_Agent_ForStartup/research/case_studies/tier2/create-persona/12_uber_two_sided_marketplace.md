## ペルソナ1: ライダー側 - Alex（都市通勤者）

### 基本情報
| 項目 | 内容 |
|------|------|
| 名前 | Alex（仮名） |
| 年齢 | 29歳 |
| 職業 | アイテー企業オフィスワーカー |
| 月額支払い | $200-300 |
| 課題 | 通勤・外出時の移動（タクシー高額） |

- **支払い実績**: Uber 平均$3-8/乗車 × 月30乗車 = $90-240/月

  - 従来タクシー: $10-15 vs Uber $5-8（50%削減）
  - 透明性: 事前に価格提示（タクシーメーター不安なし）
  - 利便性: アプリで呼ぶだけ、現金不要

## ペルソナ2: ドライバー側 - James（追加収入求職者）

### 基本情報
| 項目 | 内容 |
|------|------|
| 名前 | James（仮名） |
| 年齢 | 45歳 |
| 職業 | 元タクシー運転手 → Uberドライバー |
| 月額収入 | $2,000-3,500 |
| 動機 | 自由な勤務時間 + 従来より高い時給 |

### WTP検証
- **支払い実績**: Uber手数料 25% + プロモーション
- **支払い根拠**:
  - 従来タクシー: 保証給$1,500 + 売上50% vs Uber 売上75%
  - 自由性: 勤務時間自由、休息可能
  - **月額手数料 削減**: 従来タクシー配車 + ガソリン + 保険 = 固定費$1,200 vs Uber 変動費25%
  - **LTV**: 月$2,500売上 × 75% = $1,875利益 × 36月（3年）= **LTV $67,500**

### このスキルでのポイント
**両面ペルソナの独立WTP形成**:
- ライダー: 「利便性 + 低価格」で WTP $3-8
- ドライバー: 「自由性 + 高時給」で WTP 25%手数料容認

この相反する需要を **ネットワーク効果** で両立させることが両面マーケットプレイスのカギ。

### フライホイール設計
```
ドライバー増 → 待機時間減 → ライダー満足度↑
→ ライダー需要↑ → ドライバー時給↑ → ドライバー増
```

クリティカルマス: ドライバー500人程度で都市内フライホイール自動始動。
