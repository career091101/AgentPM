# Airbnb: 100人以上の直接訪問による課題発見

**企業**: Airbnb
**創業者**: Brian Chesky, Joe Gebbia, Nathan Blecharczyk
**業界**: ホームシェアリング / 旅行
**CPFスコア**: 85%
**実行時期**: 2008-2009年

## 課題発見プロセス

### 1. 定性リサーチ手法
- **手法**: ニューヨークへの直接訪問、ゲストとホストへのインタビュー
- **対象数**: **100人以上**（ゲスト50人、ホスト50人+）
- **期間**: 6ヶ月間（2008年春-秋）
- **方法**: 実際に Airbnb でリスティングをホストし、ゲストと対面インタビュー

### 2. 発見した課題

**ゲスト側の課題**:
```
"ホテルは高すぎる。特にイベント時は満室で値段も2倍以上"
→ 欲求: 安価で、ユニークな体験ができる宿泊

"見知らぬ人の家に泊まるのが不安。写真とレビューがほしい"
→ 欲求: 信頼できる情報、ホストからのレビュー
```

**ホスト側の課題**:
```
"空いている部屋を有効活用したい"
→ 欲求: 簡単に部屋を貸し出せる仕組み、支払いを自動化

"短期客の損害賠償リスクが怖い"
→ 欲求: 信頼できるゲストの選定、保険カバー
```

### 3. 3Uスコアリング（Airbnbの例）

| 軸 | スコア | 根拠 |
|---|--------|------|
| **Unworkable** | 19/20 | イベント時、ホテル満室で対応不可。旅行計画そのものが崩壊。年間$10B+の売上機会喪失 |
| **Unavoidable** | 18/20 | ホテル供給には限界。Craigslist個人売買は信頼性なし。避けられない課題 |
| **Urgent** | 20/20 | 旅行日程は固定。当日対応必須。時間的制約が厳格 |
| **合計** | **57/60** | CPFスコア 95% ✅ |

### 4. 顧客ニーズの定量化

**インタビューから抽出した定量データ**:

| 指標 | データ | 根拠 |
|------|--------|------|
| **支払意思（WTP）** | $50-100/泊 | 100人インタビューから平均値抽出（ホテルの50%） |
| **課題頻度** | 月1-2回 | 定期的に旅行する顧客セグメント |
| **課題期間** | 2-4週間前 | 旅行計画期間 = 急速に需要が高まる |
| **市場規模** | 100人 × 月12回 × 年間 = **14,400泊/年（100人）** |

### 5. 課題の本質理解

**表面的な課題**: 「ホテルが高い」
↓
**本質的な課題**: 「旅行者は『本物のローカル体験』を求めているが、ホテルは『画一的な快適さ』しか提供しない」

→ **洞察**: ホテル = 安全性/快適さ だが、Airbnb = 体験/文化交流

### 6. 既存代替案への不満点

**競合分析（インタビュー結果ベース）**:
- **ホテル**: 高価（$150-300/泊）、つまらない、予約困難（満室）
- **Craigslist**: 詐欺リスク高、検証手段なし
- **VRBO**: 長期滞在向け、短期には不便
- **Couchsurfing**: 無料だが信頼性なし、ホスト側の条件が厳しい

## スキル実行時の活用方法

### このスキルで強化される検証ポイント

1. **定性リサーチの深さ**:
   - 100人以上の直接インタビューで、表面的ニーズと本質的ニーズを分離
   - オンライン調査では得られない、顧客の本当の困りごと

2. **課題の定量化**:
   - インタビュー結果から WTP、頻度、市場規模を直接抽出
   - 「困っている」ではなく「$100支払える」レベルの具体的データ

3. **既存代替案の徹底分析**:
   - なぜホテルか、なぜ Craigslist か を理解することで、
   - 自社サービスの10倍優位性（信頼性 + 体験 + 価格）を導出

4. **市場規模推定の精度向上**:
   - 100人 × 12回/年 = 推定年間市場規模算出
   - イベント（音楽祭、カンファレンス）のピークシーズンから TAM を推定

## 定量データサマリー

- **インタビュー数**: 100+人（定性の最高品質）
- **3Uスコア合計**: 57/60（CPF 95%）
- **WTP確認**: $50-100/泊（ホテル比50%）
- **推定市場**: 14,400泊/年 × $75（平均） = $1.08M（100人規模）
- **スケール可能性**: ニューヨーク → 全米 → 世界
