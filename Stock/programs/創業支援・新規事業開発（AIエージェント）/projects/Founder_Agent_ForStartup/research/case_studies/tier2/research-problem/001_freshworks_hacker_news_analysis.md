# Freshworks: Hacker Newsコメント分析による課題発見

**企業**: Freshworks（旧Freshdesk）
**創業者**: Girish Mathrubootham
**業界**: カスタマーサポートソフトウェア
**CPFスコア**: 80%
**実行時期**: 2010年（創業前調査）

## 課題発見プロセス

### 1. 生ログ収集方法
- **プラットフォーム**: Hacker News
- **収集期間**: 6ヶ月間
- **収集数**: 1,000件以上のコメント分析
- **キーワード**: "Zendesk", "customer support", "SaaS pricing"

### 2. 発見した課題

**市場の声（生ログより抽出）**:
```
"Zendeskが価格を倍に引き上げた。本当に困っている。移行先を探している"
（Upvote: 500+、リプライ: 100+）

"中小企業にとってZendeskは高すぎる。年間$5,000以上かかる"
（スレッド深さ: 50以上のコメント）
```

### 3. 3Uスコアリング（Freshworksの例）

| 軸 | スコア | 根拠 |
|---|--------|------|
| **Unworkable** | 18/20 | Zendesk価格改定により顧客流出が加速。カスタマーサポートが止まると売上に直結 |
| **Unavoidable** | 16/20 | SMB企業は他に選択肢がない（Salesforce高価、オンプレ導入は複雑） |
| **Urgent** | 19/20 | 契約更新時に即座に対応が必要。遅延は不可能 |
| **合計** | **53/60** | CPFスコア 88% ✅ |

### 4. 市場規模推定

**課題を抱えるユーザー数推定**:
- Hacker Newsコメント: 1,000件
- 氷山の一角理論（推定倍率: 100倍）
- 推定ユーザー数: **100,000人** (Zendesk不満層)

**市場規模（TAM/SAM/SOM）**:
- TAM: カスタマーサポートソフトウェア市場 **$10B**
- SAM: SMB向けセグメント（100-1000従業員企業） **$2B**
- SOM: Zendesk不満顧客の切替市場 **$100-300M**

### 5. 既存代替案分析

**競合分析結果**:
- **Zendesk**: $19-99/エージェント/月（高価、複雑）
- **Salesforce Service Cloud**: $100+/ユーザー/月（エンタープライズ向け）
- **Zoho Desk**: $20-40/ユーザー/月（安いが機能限定）
- **手作業対応**: メール管理、スプレッドシート（属人的、スケール不可）

**ギャップ**: SMB向けで「シンプル + 低価格」の選択肢がない

## スキル実行時の活用方法

### このスキルで活用される要素

1. **生ログ収集戦略**:
   - Hacker Newsのような高品質コミュニティを優先（定量化しやすい）
   - Upvote数 > 50の投稿を厳選
   - コメントスレッドの深さから課題の深刻度を判定

2. **3Uスコアリング適用**:
   - Hacker Newsのコメント数 = 課題を抱える人数の代理指標
   - Upvote数が多い = 共感度が高い = Unavoidableの根拠

3. **市場規模推定**:
   - Web上の声 × 氷山の一角理論で推定人口を算出
   - 既存ツール（Zendesk）のユーザーベースから SAM を推定

4. **既存代替案の不満点**:
   - 高価さ + 複雑さが市場のペインポイント
   - これが Freshworks の10倍優位性（低価格 + シンプルUI）へ繋がる
