# プロジェクト憲章

**プロジェクト名**: Founder Agent - Origin（オリジン版）

## 1. ビジョン

**「起業の科学」の体系的フレームワークをベースとした、ビジネスアイデア検証の標準的なAIエージェントを構築し、CPF→PSF→PMFの段階的検証プロセスを自動化する。**

## 2. 背景と目的

### 2.1 市場背景

2025年は「AIエージェント元年」と呼ばれ、自律型AIが実用化段階に入った。スタートアップエコシステムにおいても、体系的なフレームワークに基づく仮説検証の重要性が再認識されている。

| 指標       | 数値                 |
| ---------- | -------------------- |
| AIエージェント市場（2030年予測） | $470億超（CAGR 44%） |
| スタートアップ失敗率（国内） | 約90%（PMF未達成が主因） |

**トレンド**: 田所雅之氏の「起業の科学」は、スタートアップの失敗率を下げるための体系的フレームワークとして広く認知されている。CPF（Customer Problem Fit）→ PSF（Problem Solution Fit）→ PMF（Product Market Fit）の段階的検証プロセスは、多くの起業家・VCに支持されている。

### 2.2 社会的意義

本プロジェクトは、以下の社会課題解決に寄与する可能性を持つ。

| 意義                             | 詳細                                                                                                   |
| -------------------------------- | ------------------------------------------------------------------------------------------------------ |
| **起業の民主化**                 | フレームワークを理解していない初心者でも体系的に検証できる環境を提供                                   |
| **失敗コスト削減**               | PMF達成前の早期ピボット判断により、時間・資金の無駄を削減                                              |
| **起業家教育の標準化**           | 「起業の科学」19フレームワークを実践的に学べるプラットフォーム                                         |
| **データドリブン意思決定**       | 3Uスコア、10倍優位性、UVP等の定量的指標による客観的判断                                                |

### 2.3 プロジェクト目的

本プロジェクトでは、「起業の科学」の19フレームワークを統合し、Phase1（アイデア検証）の自動化を実現する：

| # | 目的                                             | 説明                                                 |
| - | ------------------------------------------------ | ---------------------------------------------------- |
| 1 | **19フレームワーク統合**                         | FIF、3U、10倍検証、MVP10類型等の体系的な実装         |
| 2 | **Phase1全自動実行**                             | 12ステップ（需要発見→CPF→PSF→最終評価）の一気通貫実行 |
| 3 | **ステージゲート管理**                           | CPF 60%、PSF 80%等の明確な通過基準設定               |
| 4 | **スコアカード評価**                             | 40点満点での総合評価（32点以上でPhase1完了）         |
| 5 | **MVP構築支援**                                  | LP（ランディングページ）の自動生成                   |
| 6 | **ドメイン特化版の基盤提供**                     | ForStartup、ForRecruit等の派生バリアント開発基盤     |

> **重要制約**:
>
> - 「起業の科学」のフレームワーク体系を忠実に再現する
> - Phase1（アイデア検証）のみをスコープとし、Phase2以降は別プロジェクトとする
> - 初めて起業する方でも理解できる平易な説明・ガイドを提供

## 4. スコープ

### 含む（In Scope）

- **19フレームワーク統合**:
  - **Idea**: FIF（Founder-Issue-Fit）、5つの眼、MVV（Mission/Vision/Value）
  - **CPF**: ペルソナ、3U（Unworkable/Unavoidable/Urgent）、顧客インタビュー
  - **PSF**: 10倍検証、MVP10類型、UVPキャンバス
  - **PMF**: NPS、Retention、AARRR（Phase2以降で詳細化）
  - **Scale**: Unit Economics、DEAR、Balance Scorecard（Phase2以降で詳細化）
- **Phase1全自動実行スキル**:
  - `/orchestrate-phase1` - 12ステップ一気通貫実行
  - 需要発見リサーチ → MVV早期定義 → リーンキャンバス → フライホイール → Web課題発見 → 仮想ペルソナインタビュー → CPF診断 → 10倍優位性検証 → LP構築 → PSF診断 → SNSコンテンツ作成 → 最終評価
- **個別検証スキル**:
  - `/validate-cpf` - CPF検証（60%以上）
  - `/validate-psf` - PSF検証（80%以上）
  - `/validate-pmf` - PMF検証（Phase2詳細化）
- **成果物管理**:
  - `documents/` 配下の体系的ドキュメント生成
  - `mvp/lp/` 配下のLP自動生成
  - `mvp/sns_contents/` 配下のSNS投稿案作成
- **システムプロンプト**:
  - `system_prompts/01_orchestrator_prompt.md` - 統括エージェント
  - `system_prompts/02_executor_prompt.md` - 実行エージェント
  - `system_prompts/03-05_reviewer_*_prompt.md` - レビューエージェント（Peter Thiel, YC, 起業の科学）
  - `system_prompts/06_gatekeeper_prompt.md` - ステージゲート判定
  - `system_prompts/07_updater_prompt.md` - ドキュメント更新

### 含まない（Out of Scope）

- Phase2以降の詳細実装（PMF検証、収益化、スケーリング等）
- ドメイン特化版（ForStartup、ForRecruit等）の個別実装
- 実際のVC調達支援機能（ForStartupで実装）
- 法人登記・財務管理等の法務・会計業務

## 8. リスクと対策

| リスク                             | 対策                                                                                         |
| ---------------------------------- | -------------------------------------------------------------------------------------------- |
| 19フレームワーク実装の複雑性       | 段階的実装、Phase1重要フレームワークを優先                                                   |
| ステージゲート通過基準の妥当性     | 「起業の科学」の推奨値を参照、テスト検証で調整                                               |
| 自動生成コンテンツの品質           | レビューエージェント（Peter Thiel, YC, 起業の科学）による多角的評価                          |
| **CPF検証の客観性**                | 仮想ペルソナインタビューの精度向上、実際のユーザーインタビューへの移行推奨                   |
| **10倍優位性の定量化困難**         | 明確な比較軸（工数、コスト、精度等）を設定、主観評価を排除                                   |
| **MVPスコープの肥大化**            | MVP10類型から最適タイプを選定、最小機能に絞る                                                |
| **ドメイン特化版との不整合**       | domain_config.yamlで差分管理、Origin版をベースとした継承関係を明確化                         |

## 9. ガバナンス・倫理原則

### 9.1 責任体制

- **最終責任者**: yuichi（スポンサー）
- **開発主体**: Claude Code / Antigravity
- **品質保証**: 実証テストによる検証、多角的レビューエージェント評価

### 9.2 倫理原則

| 原則                       | 説明                                                                         |
| -------------------------- | ---------------------------------------------------------------------------- |
| **透明性**                 | フレームワーク適用プロセスを可視化、判断根拠を明示                           |
| **公平性**                 | 特定業界・ビジネスモデルへの偏りを排除、多様なアイデアに対応                 |
| **正確性**                 | 「起業の科学」のフレームワークを忠実に再現、独自解釈を避ける                 |
| **誠実性**                 | 検証結果が不合格でも客観的に提示、安易な合格判定を避ける                     |

### 9.3 Human-in-the-Loop ポイント

完全自律を目指すが、以下のポイントで人間確認を推奨：

- **ビジネスドメイン最終選定**: FIF（Founder-Issue-Fit）の主観的評価
- **CPFステージゲート通過判断**: 60%は基準、80%推奨であることの理解確認
- **PSFステージゲート通過判断**: 10倍優位性2軸は最低基準、3軸推奨の理解確認
- **MVP公開前レビュー**: 実際にインターネット公開する前の最終確認

### 9.4 継続的モニタリング

- 実証テスト結果の記録（test_idea_001等のスコアカード）
- ステージゲート通過率の測定（CPF/PSF合格率）
- ピボット回数の記録（検証プロセスの効率性評価）
