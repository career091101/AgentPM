# プロジェクト憲章

**プロジェクト名**: Founder Agent - ForSolo（ソロプレナー特化版）

## 1. ビジョン

**1人で事業を運営するソロプレナーが、コスト最小化と利益率重視により、$100K+ ARRを達成し、経済的自由と時間的自由を両立する。**

## 2. 背景と目的

### 2.1 市場背景

2025年は「**ソロプレナー・インディーハッカー元年**」と呼ばれ、1人で$100K+ ARRを達成する起業家が急増している。

| 指標 | 数値 |
| ---- | ---- |
| ソロプレナー成功事例 | 85件以上（Solopreneur_Research） |
| 平均ARR | $100K-500K（トップ事例は$3.5M） |
| 共通戦略 | Build in Public + SEO + Boilerplate/Micro-SaaS |

**トレンド**: チームを作らず、1人で実行可能なビジネスモデル（Micro-SaaS、Boilerplate販売、コンテンツビジネス）が注目され、X/TwitterでのBuild in Public戦略により透明性を重視したファン獲得が主流となっている。

### 2.2 ソロプレナー成功パターン分析

| 名前 | プロダクト | ビジネスモデル | ARR | 戦略 |
|------|-----------|---------------|-----|------|
| **Marc Lou** | ShipFast | Boilerplate販売 | $500K+ | Build in Public + X/Twitter |
| **Tony Dinh** | BlackMagic.so | Micro-SaaS | $500K+ | Build in Public + 複数プロダクト |
| **Pieter Levels** | Nomad List | Community + SaaS | $3.5M | SEO + Build in Public |

**共通パターン**:
- Build in Public（X/Twitterで透明性）
- SEO重視（ブログ記事50+本）
- Boilerplate/Template販売
- 複数プロダクト運営（ポートフォリオ戦略）

### 2.3 ソロプレナーの優位性

| 優位性 | 説明 |
|--------|------|
| **意思決定スピード** | チーム調整不要、即断即決 |
| **利益率最大化** | 人件費ゼロ、粗利率80%以上 |
| **時間的自由** | 働く場所・時間を自由に選択 |
| **経済的自由** | $10K MRRで生活費を賄える |
| **スケール不要** | ニッチ市場でも十分な収益 |

### 2.4 プロジェクト目的

本プロジェクトでは、ソロプレナーが1人で事業を運営するために、以下を実現する：

| # | 目的 | 説明 |
| - | ---- | ---- |
| 1 | **ソロプレナーFIT検証** | スキル充足度、時間確保可能性、動機評価 |
| 2 | **1人実行可能性評価** | コーディング、デザイン、マーケティング、サポート |
| 3 | **Build in Public戦略** | X/Twitterでの透明性、フォロワー獲得 |
| 4 | **コスト最小化** | 初期投資$0-1,000、月次コスト$0-50 |
| 5 | **利益率最大化** | 粗利率80%以上、LTV/CAC比率10.0以上 |
| 6 | **Micro-SaaS収益モデル** | 月額$10-50、MRR積み上げ |
| 7 | **SEO戦略** | ブログ記事50本、月間訪問1万PV |
| 8 | **MRR $10K達成** | フルタイム転換、経済的自由 |

> **重要制約**:
>
> - 1人実行可能性を最優先。チーム拡大を目指す場合は他のバリアントへ移行
> - 利益率80%以上を目標とするため、広告費ゼロ前提
> - Build in Publicは必須戦略（透明性でファン獲得）

## 4. スコープ

### 含む（In Scope）

#### ソロプレナーFIT評価
- **スキル評価**: コーディング（フルスタック or No-Code）、デザイン（Figma or Tailwind CSS）、マーケティング（SEO or SNS）、カスタマーサポート
- **時間確保**: 週20時間以上確保可能、サイドプロジェクトとして6ヶ月継続可能
- **動機**: なぜソロでやるのか、チームを作らない理由、利益率重視の理由

#### コスト最小化戦略
- **初期投資**: $0-1,000（ドメイン$10-20、ホスティング$0-50）
- **月次コスト**: $0-50（Vercel無料枠、Supabase無料枠）
- **マーケティング**: $0（SEO、X/Twitter、Product Hunt）
- **開発**: 自分で実装（Boilerplate活用で時間短縮）

#### Build in Public戦略
- **プラットフォーム**: X/Twitter（プライマリ）、LinkedIn、IndieHackers、Product Hunt（セカンダリ）
- **コンテンツ戦略**: 週3-5回投稿（開発進捗、収益報告）、透明性重視（失敗も共有）
- **フォロワー目標**: 3ヶ月で500、6ヶ月で1,000、12ヶ月で5,000、24ヶ月で10,000

#### 収益化ロードマップ（Micro-SaaS）
- **Phase0**: 1-2ヶ月、MRR $0（MVP構築、LP公開）
- **Phase1**: 3-6ヶ月、MRR $1K（Product Hunt、初期ユーザー100人）
- **Phase2**: 6-12ヶ月、MRR $5K（SEO、コンテンツマーケ）
- **Phase3**: 12ヶ月以降、MRR $10K+（フルタイム転換）

#### 技術スタック
- **Frontend**: Next.js + Tailwind CSS
- **Backend**: Vercel Serverless
- **Database**: Supabase（無料枠）
- **Auth**: Clerk / Supabase Auth
- **Payment**: Stripe
- **Analytics**: Posthog（無料枠）

### 含まない（Out of Scope）

- チーム拡大（1人固定）
- 広告費（$0前提）
- 外部資金調達（VC調達不要）
- 高速スケールアップ（ニッチ市場OK）
- B2B SaaS（SMB/個人向けに限定）

## 8. リスクと対策

| リスク | 対策 |
| ------ | ---- |
| 1人実行の限界 | スキル不足は外注・Boilerplate活用、時間確保は優先順位明確化 |
| モチベーション維持困難 | Build in Publicでコミュニティ参加、透明性でファン獲得 |
| 収益化の遅延 | SEO + Product Hunt + Twitter施策、早期課金導入 |
| 孤独感・燃え尽き | IndieHackerコミュニティ参加、定期的な休息 |
| 技術スキル不足 | Boilerplate（ShipFast, LaunchFast）活用、No-Code検討 |
| マーケティング失敗 | Build in Public戦略徹底、SEO記事50本作成 |
| 競合との差別化困難 | ニッチ市場選定、1軸の明確な優位性 |
| 収益が$10Kに届かない | 複数プロダクト運営（ポートフォリオ戦略） |

## 9. ガバナンス・倫理原則

### 9.1 責任体制

- **最終責任者**: Yuichi（スポンサー）
- **実行主体**: Antigravity（Founder Agent ForSolo）
- AIエージェントは法的主体ではないため、全ての行動・結果の責任はスポンサーに帰属する

### 9.2 倫理原則

| 原則 | 説明 |
| ---- | ---- |
| **透明性** | Build in Publicによる開発進捗・収益・失敗の公開 |
| **誠実性** | ユーザーを欺かない、虚偽の情報提供禁止 |
| **公平性** | 特定の属性への差別的扱いの禁止 |
| **持続可能性** | 燃え尽きを防ぐための適切な休息、時間管理 |
| **コミュニティ貢献** | IndieHacker/ソロプレナーコミュニティへの知見共有 |

### 9.3 Human-in-the-Loop ポイント

完全自律を目指すが、以下のポイントでは人間（スポンサー）による確認・承認を推奨：

- **ニッチ市場最終選定**: ターゲット顧客の最終決定
- **課金開始前**: 決済機能の本番稼働前
- **フルタイム転換判断**: MRR $10K達成時の会社員退職判断
- **複数プロダクト運営判断**: ポートフォリオ戦略の最終決定
- **緊急停止**: 問題発生時のサービス停止判断

### 9.4 継続的モニタリング

- MRR・ARRの週次監視
- Xフォロワー数の定期追跡
- SEO順位の月次チェック
- 利益率・LTV/CAC比率の評価
- 自身のモチベーション・健康状態の確認
