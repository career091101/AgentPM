## 6. 失敗・ピボット履歴

### Manuel Frigerioの失敗歴

#### Event Ninja（2014-2017）の失敗

| 項目 | 詳細 |
|------|------|
| プロダクト | EventBrite向けイベント主催者向け顧客データ分析ツール |
| 問題点 | イベント主催者はデータにあまり関心を持たなかった |
| 共同創業者 | Anthony Francis |
| 結果 | プロジェクトを断念 |

#### ReferralHeroへのピボット（2015-2021）

| 時期 | 出来事 |
|------|--------|
| 2015年クリスマス翌朝 | 「ウェイティングリストツール」のアイデアを着想 |
| 4日間 | プロトタイプを構築 |
| 2016年 | Product Huntでローンチ、**2,000以上のアップボート獲得** |

#### ReferralHeroの技術的課題

| 問題 | 詳細 |
|------|------|
| スケーリング | キャンペーンがバイラルになると、1万件→50万件に急増 |
| インフラ | トラフィックとデータストレージの処理が困難 |
| ダウンタイム | ダッシュボードが数日間ダウンすることも |
| 期間 | 「最初の4-6ヶ月は本当に苦痛だった」（Frigerio談） |
| 解決策 | より優秀な開発者を雇用して問題を修正 |

#### ReferralHero売却の決断

| 要因 | 詳細 |
|------|------|
| SparkLoopの成長 | SparkLoopがReferralHeroより成功し始める |
| 個人事情 | 妻の妊娠と同時期 |
| 決断 | 2つのビジネスを運営することが困難に |
| 売却 | 2021年にFE Internationalを通じて**6桁（6-figure）で売却** |

### SparkLoopの初期課題

#### 小規模ニュースレターでの効果の限界

| 問題 | 詳細 |
|------|------|
| 規模 | 200人程度の購読者ではリファラル獲得が困難 |
| 必要条件 | 基本的な要素（オーディエンス、コンテンツ品質）が必要 |
| 現実 | 5-10件のリファラルを獲得するのも困難 |

#### Partner Programの初期課題

| 問題 | 詳細 |
|------|------|
| 運営者の傾向 | 「コントロールフリーク・モード」に陥る |
| 典型的な失敗 | Recommendations HubやUpscribeウィジェット経由リファラルを禁止 |
| データ結果 | これらの制限をかけるとプログラムはほぼ確実に失敗 |
| 影響 | リファラル数が低下するだけでなく、質も悪化 |

#### リファラル報酬設計の課題

| 問題 | 詳細 |
|------|------|
| 過度な制限 | 厳格な地理的・年齢・エンゲージメント制限 |
| 実現困難 | 年齢や国の条件を満たす購読者を送っても、高エンゲージメント保証は困難 |

### B2Bマーケティングの手法

#### Louis Nichollsの「Sales for Founders」アプローチ

| 原則 | 詳細 |
|------|------|
| 哲学 | 「セールスとカスタマーサービスに違いはない」 |
| 定義 | 顧客の目標を理解し、達成を支援することがセールス |

#### 主要なB2B戦略

| 戦略 | 詳細 |
|------|------|
| 共有アクションプラン | 「開始したが今は適切な時期じゃない」顧客を減少 |
| 顧客視点の重視 | 「多くの創業者は自己中心的」という問題を解決 |
| バリデーション優先 | 「製品を作ってからマーケ」ではなく先にバリデーション |

### 二面マーケットプレイス戦略

| 側面 | 内容 |
|------|------|
| 収益化側 | ニュースレターを推薦して報酬を得る |
| 成長側 | 他のニュースレターから推薦を受けて購読者を獲得 |
| 参加形態 | どちらか一方、または両方に参加可能 |

### 品質保証によるリスク軽減

| 施策 | 詳細 |
|------|------|
| 手動審査 | ネットワーク内のすべての推薦ニュースレターを審査 |
| カスタム基準 | エンゲージメント基準を満たす購読者のみに課金 |
| リスクフリー期間 | 28日間で購読者の質をテスト |

### パイロットフェーズ制度

| 項目 | 詳細 |
|------|------|
| 対象 | 新パートナー |
| テスト範囲 | 最初の50件の有料リファラル |
| 判定 | 確認率に基づいて自動的に承認または除外 |
| メリット | ローリスク・ローエフォートで多くのパートナーをテスト可能 |

## 9. 主要な教訓

1. **プラットフォームパートナーシップ**: 早期にConvertKitと連携し、買収に至る
2. **B2Bに特化**: クリエイター向けツールとして確立
3. **Exit戦略**: SaaSとして売却可能な形態で構築
4. **過去の失敗を活用**: Event Ninja、ReferralHeroの経験がSparkLoopに活きた
5. **二面マーケット**: 成長と収益化の両方を支援するWin-Winモデル
