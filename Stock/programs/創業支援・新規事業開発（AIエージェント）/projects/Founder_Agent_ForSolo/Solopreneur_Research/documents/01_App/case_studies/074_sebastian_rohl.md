## 🚀 プロダクト情報

### HabitKit (メインプロダクト)

| 項目 | 詳細 |
|------|------|
| **カテゴリ** | Productivity / Habit Tracking |
| **ターゲット** | 習慣化したい全ての人(特にエンジニア・ギーク層) |
| **課題** | 習慣トラッキングアプリは多いが、視覚的モチベーションが続かない |
| **ソリューション** | **GitHub Contributions Graph風UI**で習慣を可視化 |
| **コア機能** | ①草(グリッド)表示 ②シンプルな記録 ③美しいビジュアル ④月次ビュー |
| **価格** | サブスク$2.99-$4.99/月 + ライフタイム$29.99-$49.99 |
| **プラットフォーム** | iOS + Android |
| **技術** | Flutter (クロスプラットフォーム) |
| **データ保存** | ローカルSQLite (プライバシー重視) |
| **URL** | [HabitKit](https://www.habitkit.net/) |

### 主要機能詳細

1. **GitHub Grass UI**
   - 1日1マス、継続すると緑色のグリッドが埋まる
   - エンジニアに馴染み深いビジュアルメタファー
   - スクショしてSNSシェアしたくなるデザイン

2. **シンプル記録**
   - タップ1回で記録完了
   - 複雑な入力不要
   - 摩擦ゼロの習慣記録

3. **月次ビュー(2024年追加)**
   - 月単位での習慣振り返り
   - 長期トレンド可視化
   - ユーザー要望で実装

4. **ローカルファースト**
   - データはデバイス内SQLiteに保存
   - プライバシー完全保護
   - オフライン完全動作

## 📖 失敗と教訓

### 1. Liftbear の失敗(2022年5-11月)

| 失敗内容 | 詳細 | 影響 |
|---------|------|------|
| **6ヶ月開発** | 市場検証なしで突っ走る | 時間の無駄 |
| **週10DL** | 全くトラクションなし | 収益ゼロ |
| **Build in Public未実施** | 誰にも知られずローンチ | ファンゼロ |

**教訓**: 「作る前に検証。Build in Publicで早期フィードバック」

### 2. 第1回独立の挫折(2023年初頭)

| 失敗内容 | 状況 | 兆候 |
|---------|------|------|
| **孤独感** | 一人開発、社会との接点喪失 | メンタル悪化 |
| **収益不足** | HabitKit初期、まだ食べていけない | 焦り・不安 |
| **再就職決断** | 「もう無理」 | 一見、失敗に見える |

**教訓**: 「独立だけが正解ではない。再就職も賢明な選択」

### 3. 精神的安定の重要性

| 時期 | 状況 | 生産性 |
|------|------|--------|
| **専業独立時** | 孤独、不安、焦り | 低い |
| **副業時** | 給与安定、社会的繋がり | **高い** |
| **永久独立後** | 経済的余裕、自信 | 最高 |

**教訓**: 「メンタルヘルスは生産性に直結。安定基盤の上で挑戦せよ」

### 4. Build in Public の遅延(Liftbear時代)

| 失敗 | Liftbear | HabitKit |
|------|---------|----------|
| **Build in Public** | なし | あり |
| **初投稿いいね** | - | 800 |
| **ローンチ前ファン** | 0 | 数百人 |
| **初週DL** | 10 | 数千 |

**教訓**: 「Build in Publicは開発初日から。ファンと一緒に作れ」

### 5. 季節性の過小評価(2024年夏)

| 季節 | MRR | 予想 | 現実 |
|------|-----|------|------|
| **冬** | $30K | 維持 | - |
| **夏** | $15K | $25K | $15K(-50%) |

**教訓**: 「習慣化アプリは季節性強い。冬の収益を当てにするな」

### 6. 本人の言葉

> 「Liftbearの失敗がなければ、HabitKitの成功はなかった。Build in Publicを学び、市場検証の重要性を知った」

## 🎯 分析者コメント

### 「失敗と再就職を恥じない勇気」

Sebastian Röhlの物語は、「会社を辞めて独立する」ことだけがゴールではなく、「会社員をしながら副業で稼ぐ」ことの精神的安定性と最強さを証明した貴重な事例である。

### 3つの革命的学び

1. **再就職は失敗ではない**
   - Liftbear失敗 → 再就職 → HabitKit成功
   - 安定基盤の上で挑戦することの強さ
   - リスクヘッジしながらホームランを打つ

2. **Build in Public の威力**
   - Liftbear(BIP無し): 週10DL
   - HabitKit(BIP有り): 初投稿800いいね → 120K DL
   - 開発初日からファンと一緒に作る

3. **Flutter × 一人開発の最適解**
   - HabitKitは2ヶ月で開発完了
   - iOS + Android 同時リリース
   - 一人で$15K-$30K MRR達成

### 日本の個人開発者への示唆

- **「会社を辞めなくても稼げる」**: 副業で月100万円超は現実的
- **「失敗を恐れるな」**: Liftbear失敗がHabitKit成功の礎
- **「GitHub Grass UIは日本でも刺さる」**: エンジニア文化は世界共通
- **「Build in Publicは必須」**: 透明性が信頼とファンを生む
- **「ASO最適化で98%」**: 広告費ゼロで月数千DL可能

### 2025年以降のトレンド

Sebastianは現在、HabitKitの成功を「Substack → YouTube → コンサル」という Creator Economy 戦略で再現しつつある。日本の個人開発者も、プロダクト開発と並行してZenn・note・YouTubeで Build in Public を実践し、Sebastian 型の「副業 → 永久独立」パスを目指すべきである。

## 分析者コメント

Sebastian Röhlの物語は「再就職は失敗ではない」という革命的メッセージを示している。Liftbear失敗後の再就職は、精神的安定と経済的余裕をもたらし、副業でHabitKitを$15K-$30K MRRまで育てる基盤となった。GitHub Grass UIという視覚的フックとBuild in Publicの徹底（初投稿800いいね、週次レポート2年継続）が、120K DL・9,845サブスクライバーを生み出した。Flutter×一人開発で2ヶ月MVP完成、iOS+Android同時リリースという効率性も成功要因だ。ASO最適化で98%が直接DL、広告費ゼロで成長した点も学びになる。

日本市場への適用可能性は非常に高い。日本は習慣化文化が強く（みんチャレ・継続の母等）、「継続は力なり」「三日坊主克服」というメッセージが刺さる。GitHub Grass UIは未開拓であり、「習慣草」「推し活草」「学習草」「筋トレ草」「副業草」等のニッチ特化版は高需要だ。可視化好き・グラフ好き国民性と相性抜群で、X・Instagramでのスクショシェア文化も追い風になる。Flutter人気と日本のインディー開発者コミュニティの活発さも有利だ。

再現可能性の観点では、「会社員しながら$5K MRRまで育て、$10K MRRで独立検討」という堅実パスが学べる。Build in Publicは開発初日から実践し、失敗も公開して人間味を出す。季節性ビジネス（習慣化アプリは冬ピーク・夏低迷）の理解も重要だ。Flutterによる高速開発とクロスプラットフォーム対応は、一人開発の最適解である。
