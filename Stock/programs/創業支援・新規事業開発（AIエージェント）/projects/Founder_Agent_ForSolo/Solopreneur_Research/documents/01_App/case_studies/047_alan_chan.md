| **カテゴリー**     | Visual Personal Knowledge Management (PKM) Tool                                                                |

### 主要機能

| 機能                     | 説明                                                                                                           |
| ------------------------ | -------------------------------------------------------------------------------------------------------------- |
| **ビジュアルボード**     | 無限のホワイトボード上にカード（メモ）を配置し、視覚的に整理                                                   |
| **カード間の関係線**     | カード同士を線で繋ぎ、情報の関連性を可視化（Knowledge Graph）                                                  |
| **マルチレイヤー構造**   | ボード内にボードを入れ子にでき、階層的思考を表現                                                               |
| **双方向リンク**         | Roam Research / Obsidian風の[[リンク]]で情報を相互参照                                                          |
| **ローカルファースト**   | オフラインでも全機能動作、データは自分のPCに保存                                                               |
| **PDF/Web注釈**          | PDF・Web記事に直接ハイライト→自動でカード化                                                                    |
| **タグ・フィルター**     | カードにタグ付け、高度なフィルタリングで必要な情報を即座に抽出                                                 |
| **検索機能**             | 全文検索、関連カード自動提案                                                                                   |
| **エクスポート**         | Markdown, JSON形式でエクスポート可能                                                                           |

### 競合との差別化ポイント

| 競合               | Heptabaseの優位性                                                                                              |
| ------------------ | -------------------------------------------------------------------------------------------------------------- |
| **Notion**         | Notionは「リスト・表形式」中心で階層的。Heptabaseは「視覚的繋がり」を重視し、非線形思考に最適。               |
| **Roam Research**  | Roamは「テキストベースの双方向リンク」。Heptabaseは「ビジュアルボード+双方向リンク」のハイブリッド。          |
| **Obsidian**       | Obsidianは「ファイルベースMarkdown」。Heptabaseは「カードベース+ビジュアル」で直感的。                        |
| **Miro/Mural**     | Miro/Muralは「コラボレーション」重視。Heptabaseは「個人の深い思考」に特化。                                    |

1. **学術研究**: 論文・文献を読みながらカード化→関連性を視覚化→論文執筆
2. **エンジニアリング**: 技術ドキュメント整理→アーキテクチャ図作成
3. **ライティング**: アイデア発散→構造化→執筆
4. **プロジェクト管理**: タスク・リソースをボード上で視覚的に管理

### アーキテクチャ思想

1. **ローカルファースト設計**: データは基本的にローカルに保存、クラウドは同期のみ→オフライン動作、高速レスポンス。
2. **関数型プログラミング**: ClojureScriptでImmutableデータ構造→バグ減少、デバッグ容易。
3. **マイクロサービス化**: 機能ごとに分離→スケーラビリティ向上。
4. **パフォーマンス最優先**: 「思考を止めないUI」実現のため、描画最適化に徹底的にこだわり。

## 📖 8. 失敗と教訓

### 失敗1: 国立台湾大学中退 - 周囲の反対を押し切った決断

- **状況**: 学年3位の成績なのに「ここでは学べない」と中退決意→両親は猛反対
- **失敗**: 当初は「自分が間違っているのでは」と自信喪失、1年間フリーターで悶々
- **学び**: **「多数派が正しいとは限らない。自分の直感を信じる勇気」**
- **引用**: 「僕は成績優秀だったけど、心の中では『これじゃない』とずっと思ってた。大学を辞めた時、周りは『人生終わった』と言ったけど、今振り返ればあれが最高の決断だった。」

### 失敗2: Minerva大学2度目の中退 - 「また逃げるのか」という葛藤

- **状況**: 理想の大学に入学も、また「ツールが必要」と感じ中退を検討→周囲は「2度目の中退は終わり」
- **失敗**: 自己肯定感の低下、「自分は何をやっても続かないのでは」という不安
- **学び**: **「自分が本当にやりたいことを見つけるまで試行錯誤を続けることの重要性」**
- **引用**: 「2回目の中退は本当に怖かった。でも『学位』より『自分の学びを加速するツール』の方が重要だと悟った。Heptabaseはその決断の結晶。」

### 失敗3: 初期MVP - 機能詰め込みすぎで使いにくい

- **状況**: 2021年7月の最初のMVPは「あれもこれも」と機能を詰め込んだ→ユーザーは混乱
- **失敗**: 初期Discordユーザーから「何がしたいツールか分からない」とフィードバック
- **学び**: **「Less is More - 一つの明確な価値提案に集中」**
- **引用**: 「最初のバージョンは最悪だった。機能が多すぎて誰も使えなかった。全部削ぎ落として『カードを繋ぐ』だけに集中したら、ユーザーが『これだ!』と言ってくれた。」

### 失敗4: YC参加当初 - 急成長プレッシャーへの抵抗

- **状況**: YCは「月次20-30%成長」を推奨→Alanは「長期的価値重視」と対立
- **失敗**: メンターから「成長が遅すぎる」と指摘され、一時的に無理な施策を試行→ユーザー体験悪化
- **学び**: **「自分のペースで持続可能な成長を優先」**
- **引用**: 「YCは素晴らしい経験だったけど、『急成長至上主義』には違和感があった。僕らは10倍成長より、10年続くプロダクトを作りたい。」

### 失敗5: 初期マーケティング - 広告費をかけても成果なし

- **状況**: 2022年初頭、Google Ads・Facebook Adsで$5,000投資→ROI マイナス
- **失敗**: 「PKMツール」はキーワード検索では発見されにくい→広告効果薄い
- **学び**: **「コミュニティ・口コミが最強のマーケティング」**
- **引用**: 「$5,000をGoogle Adsに使って、得たのは10人の無料ユーザー。同じ期間にDiscordで対話したら、100人が課金ユーザーになった。広告は二度とやらない。」
