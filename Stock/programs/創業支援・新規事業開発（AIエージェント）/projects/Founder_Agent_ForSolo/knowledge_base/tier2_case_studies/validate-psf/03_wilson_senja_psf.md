# Tier 2 Case Study: Wilson - Senja (競合弱点突破型PSF)

**Tier 2専用フォーカス**: validate-psf スキル用ケーススタディ
**PSFスコア**: 88/100点（ソリューション適合90点、MVP速度85点、初期FB 85点）

---

## 1. PSF概要（3行まとめ）

- **課題発見**: 既存テスティモニアルツール「Testimonial」は表示フォーマットが少ない（3種類のみ）
- **解決策の検証**: 数百種類のウィジェットで差別化 → Freemiumで初月$200獲得
- **PSF達成根拠**: 競合の明確な弱点 + ニッチ市場の需要確認

## 2. PSF評価（6軸詳細分析）

### 2.1 ソリューション適合度（90/100点）

**競合分析から発見した課題**:
- **Testimonial（競合）**: 表示フォーマット3種類のみ
- **ユーザーの不満**: 「同じデザインばかりで差別化できない」（Reddit投稿多数）
- **市場機会**: LP作成者、SaaS事業者のニーズ未充足

**ソリューションの適合**:
- ✅ **差別化明確**: 数百種類のウィジェット → 競合の10倍以上
- ✅ **ターゲット明確**: LP制作者、マーケター、SaaS事業者
- ✅ **価格設定妥当**: Freemium（15件まで無料）→ $29/月

**検証方法**:
```
Step 1: 競合Testimonial使用 → 不満発見
  「ウィジェットが3種類しかない」

Step 2: Reddit/Twitter調査
  → 同じ不満を持つユーザー多数

Step 3: MVP開発（数百種類のウィジェット）

Step 4: ローンチ
  → 初月$200（10人有料化）
```

**-10点の理由**:
- テスティモニアル市場は競合多数（差別化が必須）
- ニッチ過ぎる懸念（TAM限定）

### 2.2 MVP実装速度（85/100点）

**開発期間**: **数週間**（2022年1-2月、17歳で開発）

**Week 1-2: コア機能開発**
- 口コミ収集フォーム（Typeform風UI）
- データベース設計（Firebaseリアルタイム）

**Week 3-4: ウィジェット開発**

**Week 5: ローンチ準備**

**合計開発時間**: 約150-200時間（5週間、パートタイム）
