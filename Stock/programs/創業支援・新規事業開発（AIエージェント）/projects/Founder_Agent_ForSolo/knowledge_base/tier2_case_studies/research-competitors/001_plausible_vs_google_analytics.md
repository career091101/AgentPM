# Tier 2ケーススタディ: Plausible Analytics - 競合調査スキル

## 概要

**Uku Taht**が共同創業した**Plausible Analytics**は、Google Analyticsの「プライバシー侵害」「重い」「複雑」という3大弱点を突いて$3.1M ARRを達成したプライバシー重視のWeb解析ツール。**Cookie不要、1KB未満の軽量スクリプト、オープンソース**という明確な差別化で、GDPR時代のアナリティクス市場に新風を巻き起こした。

---

## 競合特定方法

### どうやって競合を見つけたか

1. **直接競合（Direct）**
   - **Google Analytics**: 圧倒的シェア、全ての代替品が戦う相手
   - **Matomo**: オープンソース×プライバシー重視の先行者
   - **Fathom Analytics**: プライバシー重視の有料代替（カナダ）

2. **間接競合（Indirect）**
   - **Simple Analytics**: よりシンプル化を追求
   - **Umami**: 軽量オープンソース代替
   - **Cloudflare Web Analytics**: 無料×プライバシー重視

3. **競合発見の手法**
   - **Hacker News検索**: "Google Analytics alternative" で上位20製品を調査
   - **Reddit r/webdev**: プライバシー重視ツールのディスカッション
   - **Product Hunt**: "analytics" タグで最新ツール把握
   - **GitHub Stars順**: オープンソースアナリティクスのランキング

4. **市場セグメンテーション**
   ```
   Web Analytics市場
   ├── エンタープライズ層（Google Analytics 360, Adobe Analytics）
   ├── 中小企業層（Google Analytics無料版）← Plausibleのターゲット
   ├── プライバシー重視層（Matomo, Fathom, Plausible）← ここで戦う
   └── 開発者層（自己ホスト型、Umami, Ackee）
   ```

## 差別化ポイント（4軸）

## 競合調査から得た教訓

### 1. 競合の弱点を3軸で整理せよ

Plausibleが見つけたGoogle Analyticsの3大弱点：
1. **プライバシー侵害** → Cookie不要で解決
2. **重い** → 1KB未満で解決
3. **複雑** → 1画面UIで解決

**学び**: 競合の弱点は「顧客の不満」に直結する。レビューサイト、Reddit、Hacker Newsで不満を収集せよ。

### 2. 差別化は「捨てる勇気」

Plausibleが捨てた機能：
- 詳細なユーザー行動分析
- 広告コンバージョン計測
- カスタムレポート

**学び**: 全機能で勝とうとせず、1つのニッチで圧倒的に勝つ。

### 3. オープンソースは信頼の証

プライバシーツールで「本当にプライバシー守ってる？」の疑念を払拭するには、コード公開が最強。

**学び**: オープンソースは「マーケティング戦略」でもある。

### 4. 競合の価格を30%下回れ

Matomo €19、Fathom $14に対し、Plausible $9で参入。

**学び**: 「ちょっと安い」（10%）では不十分。「明らかに安い」（30%+）でなければ乗り換えない。

### 5. ポジショニングは「対極」を狙え

Google Analyticsの対極（プライバシー重視）に立つことで、明確なメッセージが生まれた。

**学び**: 競合と同じ土俵で戦わず、真逆のポジションを取れ。

### 文化的適応
日本市場で本事例を適用する際の調整ポイント:

1. **言語・ローカライゼーション**
   - UI/UX: 日本語対応必須（機械翻訳ではなくネイティブレビュー）
   - カスタマーサポート: 日本語対応（英語のみは避ける）
   - マーケティング: 日本語コンテンツ作成（X/Twitter、ブログ等）

2. **決済・価格設定**
   - クレジットカード: JCB対応追加
   - 価格表示: 税込表示（消費税10%）
   - 決済手段: 銀行振込、コンビニ決済の検討

3. **マーケティングチャネル**
   - SNS: X/Twitter中心（70%）、LinkedIn補完（20%）、Facebook/Instagram（10%）
   - Build in Public: 日本では慎重（失敗の公開に抵抗あり）
   - コミュニティ: Slack/Discord活用（日本語チャンネル）

4. **タイムライン調整**
   - フォロワー獲得: 米国の1.5倍時間（日本市場は保守的）
   - PMF達成: 米国の1.2-1.5倍時間
   - MRR成長: 米国比80%ペース（市場規模差）
