# Tier 2 Case Study: Senja - CPF検証事例

**スキル**: validate-cpf
**創業者**: Wilson Wilson
**プロダクト**: Senja (口コミ・Testimonial収集ツール)
**CPFスコア**: 65% (ForSolo基準達成)
**作成日**: 2026-01-02
**元データ**: APP_001_wilson_senja.md

## 1. 課題の特定

### 1.1 発見した課題
**課題**: 口コミ（Testimonial）の収集と表示が面倒

**具体的なペインポイント**:
- 顧客に口コミを依頼しても返信率が低い
- 集まった口コミをWebサイトに綺麗に表示するのが手間
- 複数のフォーマット（テキスト、動画、Twitter埋め込み等）に対応できない
- 口コミの管理が煩雑（スプレッドシートやメールに散在）
- 既存ツール（Testimonial等）は表示フォーマットが少ない

**発見方法**:
- **競合調査**: 既存ツール「Testimonial」の弱点（表示フォーマットが数種類のみ）を発見
- **IndieHackers観察**: LP制作者、SaaS事業者の「口コミ表示に困っている」投稿を収集
- **自己体験**: 自分のプロダクト（Senja開発前の小規模プロジェクト）で口コミ管理に苦労

### 1.2 課題の深刻さ（3Uスコア）

| 軸 | スコア | 評価根拠 |
|---|---|---|
| **Unworkable（回避不可能）** | 7/10 | LP・SaaS事業者は社会的証明が必須 |
| **Unavoidable（代替困難）** | 6/10 | 手動管理やスプレッドシートで代替可能だが非効率 |
| **Urgent（緊急性）** | 7/10 | 口コミなしではコンバージョン率が大幅低下 |
| **総合** | 6.7/10 | 中程度〜高い深刻度 |

**ForSolo評価**: 7点以上が理想だが、6.7点でもForSolo基準では許容範囲 → ✅ 合格（緩和基準）

## 2. 課題の共通性（CPF検証）

### 2.1 インタビュー・検証プロセス

**検証方法**:
- **競合ユーザー調査**: Testimonial（競合）のレビューサイトで200+件の不満コメント分析
- **IndieHackers投稿分析**: 「testimonial management」「social proof」等のキーワードで100+件の投稿収集
- **Product Hunt観察**: LP制作ツールへのフィードバックで「口コミ表示の不満」を確認
- **自己体験**: 自分のプロダクト開発で実際に口コミ管理に苦労

**検証規模**:
- 直接インタビュー: 0人（ForSolo許容）
- 競合レビュー分析: 200+件
- SNS投稿分析: 100+件

**ForSolo基準**: インタビュー10-30人不要、SNS観察・競合レビュー分析で代替可能 → ✅ 許容

### 2.2 共通率の算出

**課題共通率**: **約65%**

| セグメント | 共通率 | 母数 | 根拠 |
|---|---|---|---|

## 3. 支払い意思の確認（WTP検証）

### 3.1 価格設定の根拠

**価格**: フリーミアム（15件まで無料）→ Starter $29/月、Pro $59/月

**価格妥当性の検証**:
- **コンバージョン率向上**: 口コミ表示でCVR 1-3%向上
- **月間売上インパクト**: 月商$10K → CVR 2%向上 → 追加$200-600/月
- **ROI**: $29-59で$200-600のリターン → **7-20倍**
- **競合比較**: Testimonial $49/月より安い

**競合比較**:
| プロダクト | 価格 | フォーマット数 |
|---|---|---|
| Senja | $29-59/月 | 数百種類 |
| Testimonial | $49/月 | 数種類 |
| Trustpilot | $199/月〜 | 限定 |
| 手動管理 | 無料（時間コスト大） | 自由 |

### 3.2 実績ベース検証

**初月実績**: 不明（ただし2022年2月ローンチ後に急成長）

**継続収益**:
- MRR $50,000（2024年10月、Wilson在籍時の最高値）
- 推定ユーザー数: 1,000-2,000社（フリープラン含む）
- 有料転換率: 推定10-15%

**リピート購入**:
- Starter → Pro移行率: 推定20%
- 年間契約割引あり

**ForSolo基準**: 実績ベースまたは50%以上の意思表明 → ✅ 実績ベースで合格

## 5. 成功要因分析

### 5.1 CPF検証の成功ポイント

1. **競合の弱点を突く**: Testimonialの「フォーマット少ない」問題を発見
2. **差別化の明確化**: 「数百種類のウィジェット」で圧倒的優位性
3. **Build in Public**: 開発過程をX（3.51万フォロワー）で公開し、初期ユーザー獲得
4. **フリーミアム戦略**: 15件無料でユーザー獲得 → 有料転換

### 5.2 ForSolo固有の学び

- **ナイジェリアからの挑戦**: 不安定な電力・インターネット環境でもStarlink、発電機で対応
- **Paddle + Payoneer**: Stripe非対応国でも決済・送金が可能
- **1人運営**: サポートもFAQ + チャットボットで自動化

## 7. CPF検証の教訓

### 7.1 「競合の弱点」発見プロセス

**Wilsonのアプローチ**:
1. **競合ツール調査**: Testimonialを実際に使ってみる
2. **レビューサイト分析**: Capterra、G2等で200+件の不満レビューを収集
3. **弱点の定量化**: 「フォーマットが少ない」が全体の40%を占める
4. **差別化戦略**: 「数百種類のフォーマット」で圧倒的優位性

**教訓**: CPF検証は「自分の課題」だけでなく「競合ユーザーの不満」からも発見できる

### 7.2 ForSolo向けのCPF検証プロセス

1. **競合レビュー分析**: Capterra、G2、Product Huntで200+件の不満を収集
2. **弱点の定量化**: 不満の種類を分類し、最も多い課題を特定
3. **差別化を明確化**: 競合が解決していない課題に特化
4. **Build in Public**: 開発過程を公開し、初期ユーザーの反応を確認

## 8. 「環境を言い訳にしない」精神

### 8.1 ナイジェリアからの挑戦

**制約条件**:
- 不安定な電力（停電が頻繁）
- 不安定なインターネット（光回線なし）
- Stripe非対応（決済手段が限定）
- PayPal/Wise非対応（送金手段が限定）

**Wilsonの解決策**:
- **電力**: 発電機を購入
- **インターネット**: Starlinkを導入
- **決済**: Paddle（Stripe代替）を利用
- **送金**: Payoneer（PayPal代替）を利用

**教訓**: 「環境のせいにしない」。制約を創意工夫で突破する。

**ForSolo向けの学び**: リソースが限られていても、工夫次第で成功できる。

## 9. 「離脱」という選択

### 9.1 Senjaからの離脱

**背景**:
- Wilsonは2024年にSenjaの共同創業者から離脱（"prev cofounder @senjahq"）
- 新プロダクト **Ferndesk**（AIヘルプセンター構築ツール）を開発中
- Ferndesk MRR $1,021/月（2024年11月、ローンチ3ヶ月）

**離脱の理由（推測）**:
- 共同創業者との方向性の違い
- Senjaが成熟期に入り、新しい挑戦を求めた
- 次のCPF高い課題（AIヘルプセンター）を発見

**教訓**: CPF検証は1回きりではない。次の課題を見つけて新しいプロダクトに挑戦し続ける。
