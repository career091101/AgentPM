# Tier 2 Case Study: Pieter Levels - NomadList (Spreadsheet-to-Product Validation)

**Tier 2専用フォーカス**: discover-demand スキル用ケーススタディ
**ニーズ発見手法**: 自分の課題 + スプレッドシート検証 + コミュニティ観察

---

## 1. 概要（Demand Discovery視点の3行まとめ）

- **Googleスプレッドシート→プロダクト**: 最初はスプレッドシートで都市情報を整理して公開 → バズった → 48時間でWebサイト化。
- **リモートワークトレンドの先取り**: 2014年時点でデジタルノマドという概念はニッチ → 2020年COVID-19で爆発的需要。
- **コミュニティ主導のデータ収集**: ユーザーが都市データを追加・編集する仕組み → 700万会員、38万チェックインからリアルタイムデータ。

---

## 2. Demand Discovery詳細（6軸評価）

### 2.1 課題の明確性（10/10点）

**課題発見のプロセス**:

**2013年: アジア放浪開始**:
- Pieter Levelsはノートパソコンとバックパック1つでアジア放浪
- 各都市でWi-Fi、物価、治安、気候をチェック

**2014年1月: 12 Startups in 12 Monthsチャレンジ開始**:
- 月1個のプロダクトをローンチする自主チャレンジ
- 1-3作目: 失敗（Play My Inbox、Go Fucking Do It、Tubelytics）

**2014年7月: 4作目のアイデア**:
- 自分が困った課題: 「次にどの都市に行くべきかわからない」
- 情報がバラバラ: Reddit、ブログ、フォーラムに散在
- 比較が困難: 都市ごとの情報フォーマットが違う

**課題の具体性**:

| 課題 | 具体例 | 影響 |
|------|-------|------|
| **情報収集に時間がかかる** | 1都市調べるのに3-5時間 | 旅の計画が進まない |
| **比較が困難** | バンコクとチェンマイ、どっちが安い？ | 意思決定できない |
| **リアルタイム性がない** | ブログ記事は1年前の情報 | 現地に行ったら状況が違う |
| **主観的すぎる** | 「最高の都市！」だけで定量データなし | 自分に合うか判断不可 |

**顧客の言葉で表現した課題（Pieter本人の言葉）**:
```
「旅をしながら次にどの都市に行くべきか、毎回Googleで調べるのが面倒だった。
 Wi-Fi速度、物価、気候、治安を一覧で比較できるサイトが欲しかった。
 ないから自分で作った。」
```

**課題の緊急度**:
- **高**: デジタルノマドは1-3ヶ月ごとに移動 → 毎回同じ課題に直面
- **低**: 一般旅行者は1年に1-2回の旅行 → 緊急度低い

**課題の頻度**:
- **高**: デジタルノマドは年4-12回の移動 → 毎回都市情報を調べる
- **低**: 一般旅行者は年1-2回

### 2.2 顧客アクセス（9/10点）

**顧客との距離**:
- **自分が顧客**: Pieter Levels自身がデジタルノマド → 自分の課題 = 顧客の課題
- **コミュニティ所属**: Reddit r/digitalnomad、Hacker News、Product Huntコミュニティに参加

**顧客と話す方法**:

**1. Reddit観察（2014年1-6月）**:
- **r/digitalnomad**: 登録者数5000人（2014年時点）
- **よくある質問**:
  - 「バンコクとチェンマイ、どっちが住みやすい？」
  - 「Wi-Fi速度が速い都市は？」
  - 「月10万円で暮らせる都市は？」

**2. 自分の経験**:
- **訪問都市**: 2013-2014年に20都市以上訪問
- **困った経験**: 毎回Google検索で3-5時間
- **確信**: 「俺が困ってる = 他のノマドも困ってる」

**3. Hacker News観察**:
- **トピック**: "Where to live as a remote worker?"
- **コメント**: 200件以上（情報需要の高さ）

**顧客アクセスの証拠**:
- Reddit投稿: 週10件以上（デジタルノマド関連）
- Hacker News: 月5-10トピック（リモートワーク関連）
- 自分の旅: 20都市訪問（一次情報）

**-1点の理由**:
- 2014年時点のデジタルノマド人口は少ない（推定1-2万人）
- ニッチすぎて顧客アクセスが限定的

### 2.3 支払い意思（8/10点）

**支払い意思の検証方法**:

**Phase 1: 無料公開（2014年7月）**:
- 理由: まず需要を確認したい
- 手法: Googleスプレッドシートで無料公開
- 結果: バズった（Product Hunt #1、Hacker News #1）

**Phase 2: 有料化検討（2014年9月）**:
- Reddit質問: 「このサイト、$5払ってもいい？」
- 反応: 80%が「払ってもいい」
- 判断: 有料化決定

**Phase 3: スパム対策として$5導入（2014年10月）**:
- 理由: 無料だとスパム投稿が増えた
- 価格: $5（スパム対策のため、利益目的ではない）
- 結果: スパム激減、質の高いユーザーのみ残る

**Phase 4: 段階的値上げ（2015-2024年）**:
- $5 → $25 → $50 → $65 → $75/年
- 戦略: **離脱率を見ながら徐々に値上げ**
- 結果: 離脱率は各値上げで5-10%のみ

**価格設定の根拠**:

**時間価値の計算**:
```
デジタルノマドの都市情報調査時間: 1都市3-5時間
年間移動回数: 4-12回
年間調査時間: 12-60時間

ノマドの時給: $20-$50（平均$35）
時間価値: 12-60時間 × $35 = $420-$2,100

Nomad List価格: $75/年
価値/価格比: $420 / $75 = 5.6倍
→ 十分な価値提供
```

**支払い意思が高い理由**:
1. **時間節約**: 都市情報調査が3-5時間 → 5分に短縮
2. **コミュニティ**: フォーラム、チャット、イベントへのアクセス
3. **ライフタイムオプション**: $149-199で永久アクセス（サブスク疲れ対策）

**-2点の理由**:
- 最初は無料 → 支払い意思の検証に時間がかかった
- ライトユーザー（年1-2回旅行）は$75払わない

### 2.4 市場規模（8/10点）

**TAM（Total Addressable Market）**:

**2014年時点（ローンチ時）**:
```
世界のデジタルノマド人口: 推定1-2万人
× 年間支払い: $75
= TAM: $75万-$150万（約1.1-2.2億円）
→ 小規模市場
```

**2020年時点（COVID-19後）**:
```
世界のデジタルノマド人口: 推定1000万人
× 年間支払い: $75
= TAM: $750M（約1,125億円）
→ 中規模市場
```

**2024年時点（現在）**:
```
世界のデジタルノマド人口: 推定3500万人
× 年間支払い: $75
= TAM: $2.6B（約3,900億円）
→ 大規模市場
```

**SAM（Serviceable Addressable Market）**:

**2024年時点**:
```
デジタルノマド: 3500万人
× アクティブユーザー比率: 20%（年4回以上移動）
= 700万人

700万人 × $75 = $525M（約787億円）
```

**SOM（Serviceable Obtainable Market）**:

**2024年実績**:
```
有料会員数: 2.9万人
年間収益: $1.656M（約2.5億円）
市場シェア: 2.9万 / 700万 = 0.41%
```

**市場成長性**:
- **2014-2019年**: 年15%成長（デジタルノマド人口）
- **2020-2021年**: 年100%成長（COVID-19でリモートワーク爆発）
- **2022-2024年**: 年20%成長（安定成長期）

**-2点の理由**:
- 2014年ローンチ時は超ニッチ市場（TAM $75万のみ）
- VC基準では投資不可能な市場規模（ForSolo基準では十分）

### 2.5 競合ギャップ（10/10点）

**2014年7月（ローンチ時）の競合状況**:

| 競合 | カテゴリ | 問題点 |
|------|---------|--------|
| **TripAdvisor** | 旅行レビュー | 旅行者向け、ノマド向けデータなし（Wi-Fi速度、物価等） |
| **Lonely Planet** | 旅行ガイド | 情報が古い、更新遅い、デジタルノマド向けではない |
| **Reddit r/digitalnomad** | コミュニティ | 情報がスレッドに散在、検索困難、定量データなし |
| **競合なし** | - | **デジタルノマド向けデータベースは存在しない** |

**Nomad Listの差別化ポイント（ローンチ時）**:
1. **定量データ**: Wi-Fi速度、物価、気候を数値化
2. **リアルタイム性**: ユーザーチェックインで最新情報
3. **比較機能**: 複数都市を一覧で比較
4. **コミュニティ**: フォーラム、チャット、イベント統合
5. **無料**: 最初は完全無料（後にフリーミアム）

**競合ギャップの発見プロセス**:

**2014年1-6月: 既存サービス調査**:
- TripAdvisor使用: 「観光情報は豊富だけど、Wi-Fi速度データなし」
- Lonely Planet読破: 「情報が1年前で古い」
- Reddit r/digitalnomad観察: 「毎日同じ質問が投稿される」

**結論**:
- デジタルノマド向けデータベースは存在しない
- 作れば確実に需要がある

**競合ギャップの持続可能性**:

**2014-2019年**:
- 競合ほぼゼロ → Nomad Listが独占
- 認知度: デジタルノマドの80%が知ってる

**2020年以降**:
- 競合登場: Remote Year、SafetyWing等
- Nomad Listの優位性: 先行者利益、コミュニティの強さ、データ量

## 4. リモートワークトレンドの先取り

### 4.1 2014年時点の市場環境

**デジタルノマド人口**:
- 推定: 1-2万人（世界）
- 認知度: 低い（「デジタルノマドって何？」）
- メディア露出: ほぼゼロ

**リモートワーク環境**:
- Slack: 未登場（2013年8月ローンチ、まだ普及前）
- Zoom: 未登場（2013年1月ローンチ、まだ普及前）
- 企業のリモートワーク制度: ほぼゼロ

**Pieterの判断**:
- 「今は小さい市場だけど、絶対伸びる」
- 「リモートワークは未来のトレンド」
- 「先行者利益を取る」

### 4.2 市場成長の予測と実績

**2014年予測（Pieterの推測）**:
```
デジタルノマド人口: 1万人 → 10万人（2020年）
年間成長率: 約40%
理由: リモートワークツールの普及、企業の柔軟化
```

**実績**:
```
2014年: 1-2万人
```

- 都市ごとのレビュー投稿
