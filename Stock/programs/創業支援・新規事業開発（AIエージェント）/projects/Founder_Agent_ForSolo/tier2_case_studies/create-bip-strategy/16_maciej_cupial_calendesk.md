# Tier 2 Case Study: Maciej Cupial - Calendesk

**ID**: BIP_TIER2_016 | **Founder**: Maciej Cupial | **Product**: Calendesk | **Revenue**: 破産 → 復活（推定MRR $50K+） | **Study Date**: 2026-01-02

---

## 1. Build in Public の核心: 破産からの復活物語

### プロフィール
- **国籍**: ポーランド
- **製品**: 予約管理SaaS
- **特徴**: **破産を公開し、そこからの復活プロセスを完全透明化**

---

## 2. 超透明性BIP戦略

### 破産公開（2020年）
```
X投稿: "Calendesk is bankrupt. I failed. Here's what happened..."
- 負債額: €50K
- 失敗理由: キャッシュフロー管理の甘さ、過剰投資
- 心境: 恥ずかしいが、隠さない
```

### 復活プロセス公開（2020-2023）
```
Month 1: "破産後、再起動を決意。まずは負債返済計画"
Month 6: "副業で€10K稼ぎ、負債返済開始"
Month 12: "Calendesk v2 開発中。今度は失敗しない"
Month 24: "MRR $10K 復活！負債完済まであと€20K"
Month 36: "MRR $50K。破産から完全復活しました"
```

---

## 3. コミュニティからの支援

- **共感**: 失敗を公開 → 多くの起業家から励まし
- **アドバイス**: 財務管理、マーケティング戦略のアドバイス殺到
- **顧客獲得**: "応援したい"という動機での契約

---

## 4. ForSolo スキル適用

### 破産・失敗公開BIP戦略
```markdown
## Maciej Cupial式 超透明性BIP

### 失敗公開の3原則
1. **隠さない**: 負債額、失敗理由を具体的に公開
2. **学びを共有**: "次は何をするか"を明示
3. **復活プロセス**: 月次進捗を継続公開

### 投稿例
- "破産しました。負債€50K。でも諦めません"
- "負債返済計画: 月€2K返済、24ヶ月で完済予定"
- "復活プロセス: 現在MRR $10K、負債残€30K"
```
