# 本田謙 - フリークアウト（Freak Out）

## 1. 基本情報

| 項目 | 内容 |
|------|------|
| 創業者 | 本田謙（ほんだ けん） |
| 生年 | 1976年 |
| 学歴 | 立教大学経済学部 |
| 創業前経験 | 電通（広告営業）、Google（AdWords営業） |
| 企業名 | フリークアウト・テクノロジーズ株式会社 |
| 創業年 | 2006年 |
| 業界 | AdTech（広告配信プラットフォーム） |
| IPO | 2014年6月26日、東証マザーズ |
| 評価額 | IPO時約150億円、現在約200-300億円 |
| 従業員数 | 200人（IPO時点） |

## 2. 創業ストーリー

### 2.1 課題発見

**着想源**:
- 本田謙：電通で広告営業、その後 Google でAdWords営業
- 電通時代の課題認識:
  - 広告主は「ターゲットユーザーに確実にリーチしたい」がメディアの在庫は限定的
  - メディア側は「空き在庫を活用したい」が広告主とのマッチングが非効率
  - 人手で広告配信を最適化するには限界がある

- Google AdWords での気づき:
  - リアルタイム入札（RTB: Real Time Bidding）の可能性
  - キーワード単位でオークション形式で配信を最適化
  - 「広告配信にも同じアプローチが使える」と直感

### 2.2 CPF検証（Customer Problem Fit）

**検証方法**:
- 電通での営業経験を活かして、広告主・媒体社にヒアリング（50社以上）

**発見**:
1. **広告主の課題**: 「限られた予算で最大のROIを得たい」
   - 媒体ごとに個別で枠を購入 → 非効率
   - 「最適なユーザーにリーチできているか不確実」

2. **媒体社の課題**: 「売上を最大化したい」
   - 広告主の直接営業では在庫が埋まらない
   - 空き在庫を効率的に販売する仕組みがない

3. **市場全体の課題**: 「デジタル広告の効率化がまだ手作業」
   - 人手で広告配信を最適化
   - スケールに限界

**3U分析**:
- **Unworkable**: 人手では広告配信を最適化できない
- **Unavoidable**: デジタル広告は増加するトレンド必至
- **Urgent**: ROI向上は急務（高い緊急性）

**支払い意思（WTP）**:
- 広告主は配信効率向上で予算を増加させたい意向
- 媒体社は在庫活用で売上増加を期待

### 2.3 PSF検証（Product Solution Fit）

**初期アプローチ**:
- 当初は「ネット広告配信の最適化ツール」として開発開始
- しかし 2010年頃、米国で RTB（Real Time Bidding）が台頭

**ピボット（2010年）**:
- RTB ベースの DSP（Demand Side Platform）へ方針転換
- 「リアルタイム入札で自動的に最適な広告配信を実現」という戦略に転換

**MVP**:
- タイプ: Prototype（RTB連動型DSP）
- 初期反応:
  - 大手広告主 3-5社が試験導入
  - CVR: 8%（初期段階）
  - 「配信効率が 2-3倍向上した」との反応

**10倍優位性**:

| 軸 | 従来の方法 | フリークアウト DSP | 倍率 |
|----|-----------|-----------------|------|
| 配信効率 | 人手で週単位で最適化 | リアルタイムで自動最適化 | 10倍 |
| レスポンス | 1日以上 | ミリ秒単位 | 15倍 |
| スケーラビリティ | 手作業では限界 | 数百万ユーザーに対応 | ∞倍 |
| コスト削減 | 媒体ごとに交渉 | 自動入札で最適価格 | 5倍 |
| 透明性 | ブラックボックス | データドリブンな配信 | 3倍 |

**UVP（Unique Value Proposition）**:
- 「リアルタイム入札で、最適な広告配信を実現」
- 広告主: 最小コストで最大のリーチ
- 媒体社: 在庫を最大化して売上拡大

### 2.4 初期トラクション

**営業方法**:
- 本田謙が大手広告主・媒体社に直接営業
- 「配信効率を 2-3倍にする」という明確な価値提案

**パイロット顧客**:
- 大手広告主（ユニクロ、LINEなど）との試験導入
- 効果測定を厳密に実施
- 「効果が出た」という成功事例を構築

**初期CVR**: 8% → 改善後 15%（1年後）

## 3. ピボット経験

### 3.1 RTB へのピボット（重要な転換点）

**背景**:
- 2010年頃、米国で RTB（Real Time Bidding）が急速に台頭
- Google、Yahoo が RTB 機能を強化

**ピボット前の状況**:
- 従来のプログラマティック配信では、週単位での最適化
- リアルタイム入札の時代へ対応が急務

**ピボット内容**:
- 既存の配信ロジック → RTB ベースの自動入札ロジックへ転換
- 在庫データベース → RTB 対応の高速マッチング機能へ

**成功の理由**:
- 本田謙が RTB のポテンシャルを早期に認識
- エンジニアチーム（Y Combinator 出身の技術者を採用）が実装
- 市場のトレンドに先手を打つことができた

**成長への影響**:
- ピボット後、ARR が急速に増加
- 2011-2013年で 15倍成長を実現

## 4. 成長戦略とスケール

### 4.1 初期段階（2006-2009年）

**フェーズ**:
- 手作業による広告配信最適化サービス
- 大手広告主・媒体社との直接取引

**成果**:
- ARR: 0 → 数千万円

### 4.2 ピボット・スケール段階（2010-2013年）

**RTB 導入と自動化**:
- リアルタイム入札でスケーラビリティを実現
- エンジニアチームの強化（Y Combinator 出身者の採用）

**プロダクト進化**:
- DSP の機能拡充（オーディエンスターゲティング、クリエイティブ最適化）
- API 整備で媒体社との連携強化

**営業展開**:
- 大手広告主への営業（ユニクロ、LINEなど）
- 媒体社への営業（Yahoo Japan、Ameblo など）

**成果**:
- ARR: 数千万円 → 20-30億円（IPO時）

### 4.3 フライホイール

```
広告主からの需要増加
  ↓
媒体社の在庫データが増加
  ↓
マッチングの精度向上
  ↓
配信効率が改善（ROI向上）
  ↓
さらに広告主が利用（1に戻る）
```

### 4.4 プロダクト戦略

**コア機能**:
- リアルタイム入札エンジン
- オーディエンスターゲティング
- クリエイティブ最適化
- レポーティング・ダッシュボード

**差別化要素**:
- 独自のマシンラーニングアルゴリズム
- 高速な入札応答時間（ミリ秒単位）
- 透明性のあるデータレポーティング

## 6. 成功要因分析

### 6.1 主要成功要因

1. **電通・Google でのキャリア**
   - 広告業界の深い理解
   - 大手顧客とのネットワーク

2. **RTB トレンドへの早期適応**
   - 2010年の時点で RTB へのピボット判断
   - 市場のトレンドに先手を打つ

3. **エンジニアチームの質**
   - Y Combinator 出身者の採用
   - 高速で反復的な開発文化

4. **大手顧客との信頼関係**
   - ユニクロ、LINE などの大型案件獲得
   - 「効果が出た」という成功事例の構築

5. **データドリブンなマーケティング**
   - 自社の DSP を自社で活用（dogfooding）
   - マーケティング効率が高い

### 6.2 タイミング要因

- **2008年**: リーマンショック → 広告効率化ニーズ増大
- **2010年前後**: RTB 技術の台頭（米国から）
- **2012-2013年**: プログラマティック広告の成長加速

### 6.3 差別化要因

- **独自の入札アルゴリズム**: 配信効率が競合より優れている
- **高速レスポンス**: ミリ秒単位の応答時間
- **透明性**: データドリブンな報告で信頼獲得
- **大手顧客基盤**: ユニクロ、LINE など業界を代表する企業の信頼

- 2006年のフリークアウト創業時: 日本のネット広告市場約 2,000億円
