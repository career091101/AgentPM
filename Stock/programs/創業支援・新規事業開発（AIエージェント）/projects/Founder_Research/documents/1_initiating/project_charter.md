# プロジェクト憲章: Founder_Research

**プロジェクト名**: Founder_Research（創業者リサーチ・分析プロジェクト）
**プログラム**: 創業支援・新規事業開発（AIエージェント）
**承認日**: 2025-12-31
**バージョン**: 1.0

---

## 1. プロジェクト概要

### 1.1 プロジェクト名
Founder_Research（創業者リサーチ・分析プロジェクト）

### 1.2 プロジェクトの目的
世界的起業家500人の実証データを収集・分析し、Founder Agentの判定基準を実証的に検証するとともに、AIエージェントが参照可能な高品質ナレッジベースを構築する。

### 1.3 ビジネス価値
- **Founder Agentの精度向上**: CPF60%、10倍2軸等の閾値が実際の成功事例で妥当かを検証
- **ユーザー説得力の強化**: 「この判定は世界500人の起業家データに基づいています」というエビデンス提供
- **AIエージェントのナレッジ強化**: orchestrate-phase1スキル実行時に成功・失敗パターンを参照可能に
- **再現性の向上**: 成功パターンの体系化により、新規創業者への実践的アドバイス精度が向上

## 2. プロジェクトの背景

### 2.1 現状の課題
- Founder Agentの判定基準（CPF60%、10倍2軸等）は理論ベースで、実証データによる裏付けが不足
- ユーザーから「なぜこの基準なのか？」の質問に対し、成功事例を提示できない
- AIエージェントが参照できる構造化されたケーススタディDBが存在しない

### 2.2 プロジェクト発足の背景
- Founder Agent Phase1（/orchestrate-phase1スキル）が実装され、実運用フェーズに移行
- ユーザーからの信頼性向上のため、判定基準のエビデンス整備が急務
- Solopreneur_Research（517件）の成功により、大規模データ収集の実績とノウハウを獲得

| 02_Unicorn | 評価額$1B+創業者 | 100人 | 高成長スタートアップの検証プロセス |
| 03_VC_Backed | シリーズA-C調達済み | 150人 | VC調達に至る標準的検証手法 |
| 04_IPO_Japan | 日本上場企業創業者 | 50人 | 日本市場特有の成功パターン |
| 05_IPO_Global | 海外上場企業創業者 | 50人 | グローバル上場の検証水準 |
| 06_Pivot_Success | ピボット成功事例 | 30人 | 方向転換の意思決定プロセス |
| 07_Failure_Study | 失敗事例研究 | 30人 | 避けるべき落とし穴の特定 |
| 08_Emerging | 新興起業家（2020年以降創業） | 40人 | 最新トレンドと手法の把握 |

#### 収集データ項目
- **需要発見**: 創業のきっかけ、課題発見方法、初期仮説
- **CPF検証**: インタビュー実施数、課題共通性（%）、支払い意思確認
- **PSF検証**: 10倍優位性（軸と倍率）、MVPタイプ、競合差別化
- **ピボット/失敗**: 転換点、失敗からの学び、意思決定基準

#### 成果物
1. **ケーススタディDB**: 500人分の標準化されたYAML/Markdown形式データ
2. **横断分析レポート**:
   - CPFスコア分布分析
   - 10倍優位性パターン分析
   - ピボット成功要因分析
3. **索引システム**:
   - 業界別索引（by_industry.md）
   - ステージ別索引（by_stage.md）
   - 失敗パターン索引（by_failure_pattern.md）
4. **調査ガイドライン**: 品質担保のための標準調査手順書（research_guidelines.md）

## 9. 他プロジェクトとの関係性

### 9.1 Founder_Agent_ForStartupとの関係

#### 現状の課題
- **ファイル数**: Founder_Research（646ファイル）≒ Founder_Agent_ForStartup（647ファイル）
- **重複懸念**: 両プロジェクトのディレクトリ構造が酷似しており、役割分担が不明確

#### 関係性の整理

| プロジェクト | 主な目的 | 成果物タイプ | 対象ユーザー |
|-------------|---------|------------|------------|
| **Founder_Research** | **データ収集・分析** | ケーススタディDB、横断分析レポート | Founder Agentの開発者・データ利用者 |
| **Founder_Agent_ForStartup** | **エージェント実装** | システムプロンプト、スキル定義、ピッチデッキテンプレート | スタートアップ創業者（エンドユーザー） |

#### 統合検討の必要性
**推奨アクション**: 以下の2つの選択肢を検討

##### オプション1: データ統合（推奨）
- **Founder_Researchを上位プロジェクト化**: データレイヤーとしてFounder_Agent_ForStartupが参照
- **ディレクトリ構造**:
  ```
  Founder_Research/
  ├── documents/           # 500人のケーススタディ（データレイヤー）
  ├── analysis/            # 横断分析レポート
  ├── _index/              # 索引
  └── _templates/          # 調査テンプレート

  Founder_Agent_ForStartup/
  ├── system_prompts/      # AIエージェント定義
  ├── mvp/                 # スキル実装
  └── documents/
      └── fundraising/     # ピッチデッキ等のテンプレート
      └── _reference/ -> ../../Founder_Research/  # シンボリックリンク
  ```

##### オプション2: 完全統合
- 2つのプロジェクトを1つに統合し、以下の構造に再編
  ```
  Founder_Agent_Complete/
  ├── research/            # 旧Founder_Researchのデータ
  ├── system/              # 旧ForStartupのエージェント定義
  └── templates/           # 共通テンプレート
  ```

#### 短期的対応（このプロジェクト憲章の範囲内）
- **Founder_Researchのスコープ**: データ収集・分析に特化（500件完成まで）
- **重複回避**: Founder_Agent_ForStartupのdocuments配下にケーススタディを重複作成しない
- **統合判断**: プロジェクト完了時（2026-03-31）に正式な統合可否を判断

### 9.2 Solopreneur_Researchとの関係
- **データ共有**: ソロプレナー事例（517件）をFounder_Researchの補完データとして活用
- **手法の流用**: Solopreneur_Researchで確立した調査手法・品質基準を適用
- **差別化**: Founder_Researchは「VC調達・急成長型」に焦点、Solopreneurは「自己資金・持続成長型」

### 9.3 startup_scienceプロジェクトとの関係
- **理論フレームワーク提供**: 「起業の科学」の検証項目をデータ収集に活用
- **逆フィードバック**: Founder_Researchの実証データで「起業の科学」理論を検証
