# Brian Chesky - Airbnb

## 1. 基本情報

| 項目 | 内容 |
|------|------|
| 創業者 | Brian Chesky |
| 生年 | 1981年8月29日 |
| 国籍 | アメリカ人 |
| 学歴 | Rhode Island School of Design (RISD)、インダストリアルデザイン学士 |
| 創業前経験 | インダストリアルデザイナー |
| 企業名 | Airbnb |
| 創業年 | 2008年 |
| 業界 | ホスピタリティ / シェアリングエコノミー |
| 現在の状況 | 上場（NASDAQ: ABNB、2020年12月IPO） |
| 評価額/時価総額 | IPO時$100B+ |

## 2. 創業ストーリー

### 2.1 課題発見（需要発見）

**着想源**:
- 2007年10月、CheskyはRISD時代の友人Joe Gebbiaとサンフランシスコでルームメイトになった
- 家賃を払うお金がなく、経済的に困窮していた
- ちょうどデザインカンファレンスが開催され、市内のホテルが満室状態だった
- 自宅のリビングにエアマットレスを3つ並べ、1泊$80で貸し出すアイデアを思いついた

**初期のサービス名**:
- 「Air Bed and Breakfast」（エアベッド＆ブレックファスト）

**共同創業者の参加**:
- 2008年2月、Cheskyの元ルームメイトNathan Blecharczyk（技術担当）がCTOとして参加
- 3人で正式に「AirBed & Breakfast」を立ち上げ

### 2.2 CPF検証（Customer Problem Fit）

**課題の明確化**:
- カンファレンス時などホテルが満室の際、宿泊先がない
- ホテルは画一的で、ローカルな体験ができない
- 旅行者は「その土地に住むような体験」を求めている

**3U検証**:
- **Unworkable**: ホテルのキャパシティには物理的限界がある
- **Unavoidable**: 旅行需要は増加し続ける
- **Urgent**: カンファレンス・イベント時に即座に顕在化

**支払い意思（WTP）**:
- 最初の3人のゲストは1泊$80を支払った
- 見知らぬ人の家に泊まることへの抵抗は当初大きかった

**初期の顧客獲得戦略**:
- 100人ずつ確実にコンバートする戦略（大量獲得より深い価値提供）
- 口コミによる自然な拡大を重視

### 2.3 PSF検証（Problem Solution Fit）

**10倍優位性**:

| 軸 | 従来の解決策（ホテル） | Airbnbソリューション | 倍率 |
|---|------------------|-------------------|------|
| 体験の独自性 | 画一的な部屋 | ローカルな住居・地元体験 | 10倍 |
| コスト | 高額なホテル代 | 手頃な価格 | 2-3倍 |
| 在庫の柔軟性 | 固定キャパシティ | 無限に拡張可能な供給 | 10倍 |

**MVP**:
- 最初のMVPは文字通り「リビングのエアマットレス」
- シンプルなウェブサイトで予約を受け付け

**UVP（独自の価値提案）**:
- 「どこにいても、そこに住むように旅をする」
- ホストによるパーソナライズされた地元レストラン・観光情報の提供

**競合との差別化**:
- 信頼性を担保するレビューシステム
- デザイン重視のユーザー体験（RISDデザイナーとしての強み）
- 両面マーケットプレイスによるネットワーク効果

**Cheskyの「10倍ルール」**:
- チームに対して「10%の改善ではなく、10倍の改善を考えろ」と促す
- この哲学から「Airbnb Experiences」などの新サービスが誕生

## 3. 初期の失敗と学び

### 3.1 投資家からの拒絶

- 最初の15人のエンジェル投資家全員から拒否された
- 8人は明確に拒否、7人は返信すらしなかった
- $150,000を$1.5M評価額で調達しようとしたが、誰も興味を示さなかった
- Cheskyの母親は「つまり、健康保険付きの仕事はなくなったのね。あなたは起業家じゃない、失業者よ」と言った

### 3.2 資金難とシリアル販売

- 複数のクレジットカードを限度額まで使い切った
- 2008年の大統領選挙に便乗し、「Obama O's」と「Cap'N McCain's」というシリアルを製作
- 1箱$40で販売し、約$30,000を調達
- この資金で会社を存続させ、借金の一部を返済

### 3.3 トラクション不足

- 2008年民主党全国大会（デンバー）でのローンチは失敗
- ほとんど予約が入らなかった
- 友人や親戚以外の最初の100人のユーザーを獲得するのに苦労

### 3.4 写真品質問題の発見と解決

- ニューヨークでのトラクションが伸びなかった
- 創業者3人が直接24人のホストを訪問して原因を調査
- 問題は写真の品質だった（スマートフォンで撮影したCraigslistレベルの写真）
- Y Combinator Paul Grahamの提案で、$5,000のカメラをレンタルし、創業者自らがニューヨークのリスティングを撮影
- 結果：ニューヨークの予約が2-3倍に増加、月間収益が2倍に
- 2010年から正式な無料写真撮影プログラムを開始
- プロ写真付きリスティングは予約率が2.5倍高い

## 4. 成長戦略

### 4.1 Y Combinatorと初期資金調達

**Y Combinator参加（2009年1月）**:
- Paul Grahamは当初ビジネスモデルに懐疑的だった
- しかしシリアル販売のストーリーに創業者の情熱と粘り強さを見出した
- Y Combinatorから$20,000の出資（6%の株式と引き換え）
- プログラム終了後、Sequoia Capitalから$585,000のシード投資を獲得

**マイルストーン**:
- 2009年3月：ユーザー10,000人、リスティング2,500件
- 2009年3月：社名を「AirBed & Breakfast」から「Airbnb」に変更

### 4.2 初期トラクション獲得

**口コミとリファラル**:
- 満足した顧客からの口コミが主要な成長ドライバー
- 紹介ごとにトラベルクレジットを提供するリファラルプログラム

**カンファレンス・イベント参加**:
- カンファレンスやイベントへの直接参加でユーザー獲得

### 4.3 スケール戦略

**国際展開**:
- 2011年：米国ゲストの割合が50%から30%未満に減少（欧州が過半数に）
- 重点市場：スペイン、イタリア、ドイツ、メキシコ、ブラジル、中国、インド、韓国、日本
- 拡張市場はコア市場の2倍以上の成長率

**プロ写真プログラムのスケール**:
- 2011年：無料写真撮影プログラムを正式にスケール
- 2012年2月：月間約5,000件の撮影を実施

### 4.4 COVID-19危機と回復

**危機対応（2020年）**:
- 8週間で予約が80%減少
- 月間$2.5億の損失（burn rate）
- Cheskyは「魚雷を受けた船の船長のようだった」と表現
- 1,900人（従業員の約25%）をレイオフ
- 10部門を1部門に統合
- 輸送、旅行コンテンツ、ホテル、ラグジュアリーサービスなど多くの事業を縮小

**評価額の変動**:
- 2017年：$31B（最後のプライベート資金調達）
- 2020年4月：$18Bに下落
- 2020年12月IPO：$68/株で5,000万株を売り出し、$47B評価
- 初日終値：$148/株、時価総額$100B超

**回復の要因**:
- ロックダウン緩和後、旅行者がホテルより住宅を好む傾向
- 都市から離れた郊外・地方への宿泊需要の増加

### 4.5 フライホイール

```
供給増加（ホスト）→ 選択肢の多様化 → ゲスト満足度向上 → 予約増加
    ↑                                                      ↓
信頼性向上 ← レビュー蓄積 ← 取引量増加 ← ホスト収益増加
```

## 6. 成功要因分析

### 6.1 主要成功要因

1. **デザイン思考の適用**: RISDでの訓練がUX・プロダクトデザインに反映
2. **粘り強さ**: 15回の投資家拒絶、シリアル販売での資金調達
3. **顧客中心主義**: 100人ずつ深くコンバートする戦略
4. **スケールしない施策の実行**: 創業者自らの写真撮影
5. **信頼のプラットフォーム構築**: 見知らぬ人同士の信頼を可能にしたシステム

### 6.2 タイミング要因

- シェアリングエコノミーの台頭
- スマートフォン普及による即時予約の実現
- 2008年金融危機による副収入ニーズの増加
- ソーシャルメディアによる口コミの加速

### 6.3 差別化要因

- デザイナー出身CEOによるUX重視
- 両面マーケットプレイスのネットワーク効果
- 「旅行」から「体験」への価値提案の転換

## 8. orchestrate-phase1への示唆

### 8.1 需要発見（/discover-demand）

- **示唆**: 自身の経済的困窮という切実な課題から着想
- **適用**: 創業者自身が体験した「痛み」から始める

### 8.2 CPF検証（/validate-cpf）

- **示唆**: 最初の顧客は「カンファレンス参加者」という明確なセグメント
- **適用**: 課題が顕在化する特定のタイミング・セグメントを狙う

### 8.3 PSF検証（/validate-10x）

- **示唆**: 「体験の独自性」で10倍優位性を達成
- **適用**: コスト以外の軸（体験、感情、つながり）での10倍を探す

### 8.4 スコアカード（/startup-scorecard）

- **示唆**: 「スケールしないことをやれ」（Paul Graham）
- **適用**: 初期は創業者自らが泥臭く顧客と向き合う

## 9. 事業アイデア候補

この事例から着想を得られる日本向けビジネスアイデア:

1. **体験型宿泊プラットフォーム**: 農泊・古民家など日本独自の宿泊体験
2. **スキル・時間シェアリング**: 地元住民との交流体験マッチング
3. **イベント連動宿泊サービス**: 大型イベント時の宿泊ソリューション
4. **空きスペース活用**: オフィス・店舗の時間帯別シェアリング
