## 使用ガイドライン

### このディレクトリを使用すべき場合

以下のドキュメントは `Stock/Common/project_management/` に配置：

1. **プロジェクト横断的な計画・評価**
   - 複数プロジェクトの優先順位決定
   - 全体ロードマップ
   - リソース配分計画

2. **ポートフォリオレベルの意思決定**
   - プログラム全体の評価マトリクス
   - 戦略的判断の根拠文書

3. **組織全体のKPI管理**
   - 月次/四半期レビュー
   - 全体進捗レポート

### 個別プロジェクトのdocuments/に配置すべき場合

以下は各プロジェクトの `Stock/programs/[プロジェクト名]/documents/` に配置：

1. **プロジェクト固有の計画**
   - プロジェクト憲章（project_charter.md）
   - WBS、PRD、バックログ

2. **個別プロジェクトの成果物**
   - 競合調査（competitor_analysis.md）
   - 検証結果（phase0_validation_result.md）

3. **実行フェーズのドキュメント**
   - ステータスレポート
   - 変更要求

## 移行履歴

### 2026-01-10: 初回作成

**移行元**: `Flow/202601/2026-01-10/`

**移行理由**:
- Q1ロードマップは複数プロジェクト（副業、Note メディア、資産運用、創業支援）に跨る
- プロジェクト優先順位マトリクスは6プログラム全体の評価
- リソース配分計画は組織全体のリソース管理

**移行判断**:
- 単一プロジェクトに帰属させると、他プロジェクトから参照しにくい
- Stock/Common/で一元管理することで、横断的な意思決定を支援

## 更新ルール

### 週次更新（毎週月曜）
- Q1ロードマップの進捗状況を反映
- 優先順位マトリクスのスコア見直し（必要時）

### 月次更新（月初）
- リソース配分計画の実績と計画の比較
- 次月のリソース配分調整

### 四半期更新（3ヶ月毎）
- 全ドキュメントの包括的レビュー
- 次四半期のロードマップ作成
