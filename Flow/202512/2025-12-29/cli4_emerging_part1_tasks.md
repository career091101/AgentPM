## 実行方法

### ステップ1: 並列バッチ実行開始

```
このタスクリストの全66件を並列バッチで実行してください。

特別な注意点（新興企業）:
- 情報が限定的 → 保守的推定を使用
- Twitter、Product Hunt、Crunchbaseを優先ソース
- 創業者のLinkedIn、Medium記事も活用
- Nullではなく推定値+コメントで補完
- 最新トレンドを反映

保守的推定ガイドライン:
- interview_count: 0 (データなしは0)
- problem_commonality: 20-30% (保守的)
- wtp_confirmed: false (不明はfalse)
- initial_cvr: null → 0 (データなし)
- employees: Crunchbase/LinkedInで推定

条件:
- 並列数: 10-15エージェント
- ソース: 最低10件（新興企業は12件が難しい）
- Fact Check: 全件PASS必須
```

### ステップ2: 情報収集戦略

**優先ソース順**:
1. **公式サイト** - About, Blog, Press
2. **Crunchbase** - 資金調達、従業員数
3. **Product Hunt** - プロダクト詳細、レビュー
4. **Twitter** - 創業者ツイート、発表
5. **TechCrunch** - 資金調達ニュース
6. **LinkedIn** - 創業者経歴、従業員数
7. **GitHub** - オープンソースの場合
8. **Discord/Telegram** - コミュニティ情報（Web3）
9. **AngelList** - 初期情報
10. **Forbes/Fortune** - 特集記事

**データ不足時の対応**:
- interview_count: 0 (推定不可)
- problem_commonality: 25% (保守的推定)
- ソース10件未満: 可能な限り収集、コメント記載
