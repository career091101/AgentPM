### ステップ1: 並列バッチ実行開始

```
このタスクリストの全75件を並列バッチで実行してください。

特別な注意点（新興企業 Part2）:
- Part1と同じ保守的推定ガイドライン適用
- 多様な分野 → ソース選定に工夫が必要
- DeepTech/BioTechは技術理解が重要
- Developer Toolsはコミュニティ反応を重視

保守的推定ガイドライン（Part1と同じ）:
- interview_count: 0 (データなしは0)
- problem_commonality: 20-30% (保守的)
- wtp_confirmed: false (不明はfalse)
- initial_cvr: null → 0 (データなし)
- employees: Crunchbase/LinkedInで推定

条件:
- 並列数: 15-20エージェント（Part2は件数多い）
- ソース: 最低10件
- Fact Check: 全件PASS必須
```
