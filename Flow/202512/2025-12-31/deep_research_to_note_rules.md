# Deep Research to Note Rules

ディープリサーチからnote記事作成のルール。

## トリガー
- 「ディープリサーチ」

## 実行手順

### 1. 入力受付と初期設定
- 論文URL/PDFファイルを受け取る
- 読解モード（quick/standard/deep）の確認
- 出力形式（A4スライド/Markdown/Notion）の選択
- オプションパラメータ（研究テーマ、言語、図表抽出数等）の設定
- 使用ツール: ユーザーインタラクション、パラメータバリデーション

**注意点**:
- 論文URLが無効な場合はエラーハンドリング
- PDFアップロードの場合はファイル形式検証
- デフォルト値の適切な設定（読解モード: standard、出力形式: Markdown）

### 2. 論文構造解析
- 論文PDFのダウンロード（URL指定の場合）
- セクション構造の特定（Abstract, Introduction, Methods, Results, Discussion, Referencesの自動検出）
- 図表の抽出と重要度順ソート
- 引用論文リストの抽出（References欄の解析）
- 使用ツール: PDF解析ライブラリ、OCR（必要に応じて）、図表認識AI

**注意点**:
- スキャン画像PDFの場合はOCR実行
- セクション構造が標準と異なる場合のフォールバック処理
- 図表抽出失敗時はテキストのみで継続

### 3. 戦略的読解（逆順アプローチ）
- **Abstract解析**: リサーチクエスチョンと全体像の把握
- **Conclusion解析**: 達成内容と主要結論の理解
- **Results/Experiments解析**: 検証方法、実験条件、結果データの確認
- **Related Work解析**: 先行研究との位置づけと差別化ポイントの理解
- 使用ツール: LLM（Claude/ChatGPT）による自然言語理解、セマンティック抽出

**注意点**:
- 頭から几帳面に読まず、必ず逆順（Abstract→Conclusion→Results→Related Work）を徹底
- 1論文1時間（最終的に10-20分）の時間制限を設定
- 重要論文と判断した場合のみ全セクションの精読フラグを立てる

### 4. 6つの質問への回答生成
1. **「どんなもの？」**: Abstractを凝縮した3行要約（研究テーマ、提案手法、主要結果）
2. **「先行研究と比べてどこがすごい？」**: Related Work + Introductionの分析から学術的価値と差別化ポイントを抽出
3. **「技術や手法のキモはどこ？」**: Methodsセクション分析 + 重要図表（1-3点）の抽出と説明
4. **「どうやって有効だと検証した？」**: Experiments/Resultsセクション分析（被験者数、実験条件、評価指標、統計的有意性）
5. **「議論はある？」**: Discussionセクション + 批判的視点（限界点、今後の課題、適用範囲の制約）
6. **「次に読むべき論文は？」**: References分析 + 引用頻度ランキング + ユーザーの研究テーマとの関連度スコアリング
- 使用ツール: LLMプロンプトチェーン、批判的思考モデル、引用ネットワーク解析

**注意点**:
- 各質問への回答は簡潔に（1-3段落以内）
- 理論論文の場合は質問のバリエーション適用（批判されている理論、文脈・理路の追跡等）
- 必ず批判的視点（限界点、今後の課題）を含める

### 5. サマリー生成
- **A4 1枚スライド生成**: PowerPoint/PDF形式、6つの質問を見出しとした構造化レイアウト
- **Markdown形式生成**: 階層化された見出し、箇条書き、図表埋め込み
- **図表の配置**: 重要図表1-3点を視覚的に配置
- **視覚的デザイン**: 色分け、箇条書き、図解の活用
- 使用ツール: スライド生成API、Markdownテンプレートエンジン、図表埋め込みツール

**注意点**:
- A4半分〜1枚への圧縮を徹底（情報優先度付けの訓練）
- 完璧主義を避け、フィードバックループ重視
- 図表が抽出できない場合はテキストのみで構成

### 6. データベース登録
- Notionデータベースへの自動登録（Notion API使用）
- プロパティの設定:
  - Title（論文タイトル）
  - Authors（著者リスト）
  - Year（発表年）
  - Conference/Journal（発表場所）
  - PDF（PDFリンク）
  - Tags（研究分野、手法、応用領域の自動生成）
  - Summary（6つの質問への回答）
  - Next Papers（次に読むべき論文へのRelation）
  - Read Date（読了日）
  - Read Time（読解時間）
  - Relevance（High/Medium/Low）
- 引用ネットワークへのノード追加（グラフデータベース構築）
- 使用ツール: Notion API、グラフデータベース（Neo4j等）、タグ自動生成AI

**注意点**:
- Notion API失敗時はローカルファイルとして保存し、後で手動登録を促す
- タグの一貫性を保つため、既存タグとのマッチング優先
- 引用ネットワークは段階的に構築（初回は単独ノードでOK）

### 7. 次の論文推薦
- Referencesから重要論文を3-5本抽出（引用頻度、年代、著者の権威性でスコアリング）
- 既読論文リストと照合（重複除外）
- ユーザーの研究テーマとの関連度スコアリング（セマンティック類似度計算）
- 推薦理由の生成（「この論文は〇〇の先行研究として重要」等）
- 使用ツール: 引用解析エンジン、セマンティック検索、推薦アルゴリズム

**注意点**:
- 次の論文候補数はユーザー指定またはデフォルト3本
- 引用関係を辿る連続性を保つため、必ずReferencesから選定
- 推薦理由は具体的に（「図表が優れている」ではなく「提案手法の理論的背景を詳述」）

### 8. 出力返却と統計表示
- サマリーファイル（PDF/Markdown）のダウンロードリンク提供
- Notionページへのリンク提供
- 次に読むべき論文リスト（タイトル、著者、年、推薦理由）の表示
- 読解時間と統計情報（累計読了論文数、今週の進捗、分野別分布等）の可視化
- 使用ツール: ファイルホスティング、統計ダッシュボード、進捗管理ツール

**注意点**:
- 出力形式が複数選択されている場合は全形式を生成
- 統計情報は励みになるよう、前回比や目標達成率を含める
- エラーが発生した場合も部分的な成果物は返却

## 入力パラメータ

### 必須パラメータ
- `paper_source`: 論文URL（arXiv, DOI等）またはPDFファイルパス/アップロード
- `output_format`: サマリー形式（`slide` / `markdown` / `notion` / 複数選択可）

### オプションパラメータ
- `research_theme`: ユーザーの研究テーマ（自分との関連性評価に使用）デフォルト: なし
- `reading_mode`: 読解モード（`quick` - 10分速読 / `standard` - 30分標準 / `deep` - 1時間精読）デフォルト: `standard`
- `language`: 出力言語（`ja` / `en`）デフォルト: `ja`
- `existing_papers`: 既読論文リスト（重複チェック、関連論文推薦に使用）デフォルト: 空リスト
- `next_paper_count`: Referencesから推薦する論文数 デフォルト: 3
- `figure_count`: サマリーに含める図表数 デフォルト: 1-3

## 出力形式

### 形式1: A4 1枚スライド形式（PowerPoint/PDF）
```
┌─────────────────────────────────────┐
│ [論文タイトル]                       │
│ [著者], [Conference/Journal], [Year] │
├─────────────────────────────────────┤
│ 1. どんなもの？                      │
│ [3行要約]                            │
├─────────────────────────────────────┤
│ 2. 先行研究と比べてどこがすごい？    │
│ [差別化ポイント]                     │
├─────────────────────────────────────┤
│ 3. 技術や手法のキモはどこ？          │
│ [手法説明] [図表1]                   │
├─────────────────────────────────────┤
│ 4. どうやって有効だと検証した？      │
│ [実験条件・結果]                     │
├─────────────────────────────────────┤
│ 5. 議論はある？                      │
│ [限界点・今後の課題]                 │
├─────────────────────────────────────┤
│ 6. 次に読むべき論文は？              │
│ - [論文1]                            │
│ - [論文2]                            │
│ - [論文3]                            │
└─────────────────────────────────────┘
```
- 視覚的レイアウト: 色分け、箇条書き、図解の活用
- PDF出力: スライドPDFとして保存し、論文PDFの最初のページに貼り付け可能な形式

### 形式2: Markdown形式
```markdown
# [論文タイトル]

**著者**: [Authors]
**発表**: [Conference/Journal], [Year]
**URL**: [arXiv/DOI Link]

## 1. どんなもの？
[3行要約]

## 2. 先行研究と比べてどこがすごい？
[差別化ポイント]

## 3. 技術や手法のキモはどこ？
[手法説明]

![図1](image_path)

## 4. どうやって有効だと検証した？
- **被験者**: [N人]
- **実験条件**: [条件]
- **評価指標**: [指標]
- **結果**: [主要結果]

## 5. 議論はある？
- **限界点**: [限界]
- **今後の課題**: [課題]

## 6. 次に読むべき論文は？
1. [論文1] - [選定理由]
2. [論文2] - [選定理由]
3. [論文3] - [選定理由]

---
**読了日**: 2025-12-31
**読解時間**: 25分
**自分のテーマとの関連**: [関連性メモ]
```
- Notionへのインポート、GitHubへのコミット、個人ブログへの投稿等に利用可能

## 注意点

### 効率化の原則
- **戦略的読解の徹底**: 頭から几帳面に読まず、Abstract→Conclusion→Experiments→Introの逆順アプローチを徹底
- **時間制限の厳守**: 1論文1時間（最終的に10-20分）の時間制限を守り、だらだら読みを避ける
- **重要論文のみ全文読み**: 大半の論文は6つの質問への回答のみで十分

### アウトプット駆動の学習
- **必ずA4 1枚サマリーを作成**: 読むだけでは定着しないため、必ずアウトプット
- **批判的視点の組み込み**: 単なる要約ではなく、限界点・今後の課題を必ず含める
- **フィードバックループ構築**: まとめ→振り返り→次の論文への反復サイクル

### 自分軸の読解
- **著者の意図通りに読まない**: 自分の研究テーマにとっての意義を問い続ける
- **問題解決軸での判断**: この論文が自分の課題解決にどう役立つかを常に考える
- **次のアクションを明確化**: 次に読むべき論文を必ず特定

### データベース管理の徹底
- **論文PDFの最初にサマリー貼り付け**: 後から見返す際の参照性向上
- **Notionデータベースで一元管理**: タグ付け、全文検索、引用関係の可視化
- **次に読むべき論文を明記**: 引用ネットワークを辿る連続性を保つ

### 完璧主義の回避
- **全ての論文を完璧に理解しようとしない**: 6つの質問に答えられればOK
- **美しいサマリーを作ろうとしない**: A4 1枚に収まれば十分（見た目は二の次）
- **本を書くのではなく、フィードバックループを作る**: 完璧主義を避け、反復改善を重視

### LLM活用の注意
- **LLMの出力をそのまま鵜呑みにしない**: 必ず自分で批判的に評価
- **ハイブリッドアプローチ**: LLM自動要約をベースに、人間が批判的視点を追加
- **人間の役割のシフト**: 要約作成から批判的評価・自分のテーマとの関連付けへ
