## 4. 価値判定の7つの軸

### 軸1: 実践的価値（実用性）

**判定基準**:
- ✅ 具体的な手順・コード例がある
- ✅ 実装方法が明記されている
- ✅ 再現可能なチュートリアル
- ✅ ベストプラクティス・失敗事例

**例**:
- "Claude Codeで〇〇を実装する方法"
- "Skills作成の3つのポイント"
- "ENABLE_TOOL_SEARCH=true でトークン削減"

### 軸2: 最新性（トレンド）

**判定基準**:
- ✅ 新機能リリース情報
- ✅ アップデート・変更点
- ✅ 業界ニュース（OpenAI, Anthropic, Google等）
- ✅ トレンド技術（Claude Code, Gemini 2.0等）

**例**:
- "Claude Code v1.5リリース"
- "OpenAIがGPT-5を発表"
- "Geminiのシェアが急伸"

### 軸3: 具体的数値データ（定量性）

**判定基準**:
- ✅ パフォーマンス改善（"39,000 → 2,800トークン削減"）
- ✅ 成長率・シェア（"87%→68%"）
- ✅ 収益・成果（"45,000円の印税"）
- ✅ 時間短縮（"1時間→5分"）

**例**:
- "トークン使用量を93%削減"
- "ChatGPTシェア87%→68%に急落"

### 軸4: 構造化された情報（可読性）

**判定基準**:
- ✅ リスト形式（番号付き、箇条書き）
- ✅ 比較表・マトリクス
- ✅ ステップバイステップ
- ✅ Before/After明示

**例**:
- "Claude Codeの5つの特徴"
- "Cursor vs Claude Code 比較表"

### 軸5: 信頼性（発信者の専門性）

**判定基準**:
- ✅ 実践者・専門家の発信
- ✅ 公式アカウント
- ✅ 第一次情報（海外記事の翻訳含む）
- ✅ 実装経験に基づく知見

**例**:
- @Anthropic公式アカウント
- "実際に使ってみた結果"
- "〇〇の開発者が語る"

### 軸6: 深い洞察（思考の深さ）

**判定基準**:
- ✅ 本質的な理解・解説
- ✅ 批判的思考・反論
- ✅ 独自の視点・分析
- ✅ 将来予測

**例**:
- "やはりお前らのSkillsの解釈はまちがっている"
- "AGI実現の鍵はアナロジー"

### 軸7: 行動可能性（アクションアイテム）

**判定基準**:
- ✅ すぐに実行できる
- ✅ リンク・リソース提供
- ✅ 具体的な次のステップ
- ✅ ツール・テンプレート提供

**例**:
- "こちらからダウンロード可能"
- "完全ガイドはこちら"
- "設定方法はREADME参照"

## 5. ブックマーク判定アルゴリズム

### 5.1 スコアリングモデル

各投稿を以下の基準でスコアリング（0-100点）：

```python
bookmark_score = (
    実践的価値 * 0.25 +
    最新性 * 0.20 +
    具体的数値データ * 0.15 +
    構造化情報 * 0.15 +
    信頼性 * 0.10 +
    深い洞察 * 0.10 +
    行動可能性 * 0.05
) * 100
```

**閾値**:
- **80点以上**: 高確率でブックマーク
- **60-79点**: 状況に応じてブックマーク
- **59点以下**: ブックマークしない

### 5.2 必須条件（足切り）

以下のいずれかを満たさない場合、スコアに関わらずブックマークしない：

❌ **除外条件**:
1. 広告・宣伝のみ（実質的価値なし）
2. 誹謗中傷・感情的なコメント
3. 信頼性が低い情報源（未確認の噂話等）
4. 自分の専門外で関連性なし（AI・技術以外）

✅ **最低限の必要条件**:
1. AI・生成AI関連（または密接に関連）
2. 実践的価値 OR 最新性 OR 深い洞察のいずれか
3. 発信者の専門性が確認可能

## 6. 典型的なブックマーク投稿パターン

### パターン1: チュートリアル型（実践的価値MAX）

**特徴**:
- ステップバイステップの手順
- コード例・設定方法
- 完全ガイド・網羅的解説

**例**:
```
「Claude Code完全ガイド」が最高だった。
12個のアプリを実際に作りながら学べる実践的な動画。
初心者向けの最初のプロンプトから、編集の仕方まで丁寧に解説。
```

**スコア**: 85-95点

---

### パターン2: 最新ニュース型（最新性MAX）

**特徴**:
- 新機能リリース
- 業界ニュース・発表
- トレンド情報

**例**:
```
【速報】ChatGPTとSlackの連携が大幅強化
Slack上で直接ChatGPTに質問や要約、文章作成を依頼可能に
ChatGPTがSlack内の会話を直接、参照可能に
```

**スコア**: 80-90点

---

### パターン3: データ分析型（定量性MAX）

**特徴**:
- 具体的な数値データ
- 比較・成長率
- パフォーマンス改善結果

**例**:
```
ENABLE_TOOL_SEARCH=true で Claude Code のツール検索を有効にしたら
MCPツールのトークン使用量を 39,000 から 2,800 に抑えられた
（93%削減）
```

**スコア**: 75-85点

---

### パターン4: ベストプラクティス型（実践+洞察）

**特徴**:
- 成功パターン・失敗パターン
- ノウハウ・Tips
- 本質的理解

**例**:
```
Anthropicのベストプラクティスだと、Skillsは一度作って終わりではなく、
「継続的に評価して改良する」必要がある。
そのため、最初に評価を構築することが重要。
```

**スコア**: 80-90点

---

### パターン5: 深い分析型（洞察MAX）

**特徴**:
- 本質的な理解・批判的思考
- 独自の視点・分析
- 将来予測・トレンド解説

**例**:
```
「やはりお前らのSkillsの解釈はまちがっている」

Claude Skillsは単なる関数定義ではなく、
エージェントの振る舞いを制御する「振る舞いテンプレート」。
```

**スコア**: 75-90点

## 7. ブックマークしない投稿の特徴

### 除外パターン

❌ **パターンA: 抽象的な感想のみ**
```
「AIすごい！」
「これは革命的」
「時代が変わる」
```
→ 具体性がない、実用性なし

❌ **パターンB: 単なるリンク共有**
```
https://example.com/article
```
→ コメント・解説なし、付加価値なし

❌ **パターンC: 信頼性が低い情報**
```
「噂によると〇〇らしい」
「未確認情報だが△△」
```
→ ソース不明、検証不能

❌ **パターンD: 関連性が低い**
```
「今日のランチ美味しかった」
「週末の予定」
```
→ AI・技術と無関係

### 8.1 スキルの機能

**Skill名**: `analyze-bookmark-worthiness`

**入力**:
- ツイートテキスト
- 投稿者情報（username, フォロワー数等）
- エンゲージメント指標（いいね、RT、返信）

**出力**:

- アクション推奨（"今すぐブックマーク", "後で確認", "スキップ"）

2. **Phase 2: 機械学習モデル（将来）**

3. **Phase 3: リアルタイム推薦**
