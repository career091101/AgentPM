19個の未実装フレームワークを4軸で評価し、実装優先順位を定量化しました。

**評価結果サマリー**:

**推奨アクション**: Quick Wins 5個の実装により、28-42時間でカバレッジ68.9% → **77.0%（+8.1%）** に向上可能。

## 評価軸の定義

### 1. 技術的複雑性（1-10点）

**定義**: アルゴリズム、統計処理、外部API統合の難易度

| スコア | 説明 | 例 |
|--------|------|-----|
| 1-3 | 単純な計算・集計のみ | 基本的な算術計算、フォーマット変換 |
| 4-6 | 中程度のロジック・統計処理 | Cohort分析、成長率計算、スコアリング |
| 7-10 | 複雑なアルゴリズム・統計検定 | A/Bテスト統計検定、機械学習、複雑な最適化 |

### 2. データ依存性（1-10点）

**定義**: 実データ収集・管理の必要性

| スコア | 説明 | 例 |
|--------|------|-----|
| 1-3 | ユーザー入力のみで完結 | 手動入力データ、仮説ベース計算 |
| 4-6 | 部分的な実データ必要 | ログデータ参照、CSVインポート |
| 7-10 | リアルタイム実データ必須 | トラッキングシステム、DB連携、API統合 |

### 3. 外部ツール依存（1-10点）

**定義**: WebSearch、サードパーティAPI、データソースの依存度

| スコア | 説明 | 例 |
|--------|------|-----|
| 1-3 | 外部ツール不要 | 完全にローカルで完結 |
| 4-6 | WebSearch程度 | 業界ベンチマーク収集、参考記事取得 |
| 7-10 | 複数の外部API必須 | 有料データソース、認証API、複数サービス統合 |

### 4. 実装工数（1-10時間単位）

**定義**: 実装完了までの推定時間（設計・実装・テスト含む）

| スコア | 時間 |
|--------|------|
| 1-2 | 1-2時間 |
| 3-4 | 3-4時間 |
| 5-6 | 5-6時間 |
| 7-8 | 7-8時間 |
| 9-10 | 9-10時間 |
| 11-12 | 11-12時間 |
| 13-15 | 13-15時間 |
| 16-20 | 16-20時間 |

### 5. 総合難易度スコア（1-10点）

**計算式**:
```
総合難易度 = (技術的複雑性 + データ依存性 + 外部ツール依存) / 3
```

**難易度分類**:
- **Easy**: 1-4点
- **Medium**: 4-7点
- **Hard**: 7-10点

## リスク分析

### 高難易度フレームワークのリスク

#### 1. 市場規模検証（TAM/SAM/SOM）

**リスク**:

- 業界レポートが有料・非公開

**対策**:
- 複数データソースのクロスチェック
- トップダウン/ボトムアップ両方で検証
- 仮説ベースの推定も許容

**リスク**:
- 業界ベンチマークが取得できない
- トラッキングデータが不足

**対策**:
- 一般的なベンチマーク（SaaS平均）を使用
- 段階的実装（まず3ステージから）

**リスク**:
- 前提となるKPI測定が未実装
- 可視化ライブラリの制約

**対策**:
- Phase 1-2のKPI実装完了後に着手
- Markdown + Mermaidで最小実装
