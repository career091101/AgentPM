# 落合陽一のディープリサーチ→ノート作成手法（Web抽出版）

## 注記
YouTube動画（https://www.youtube.com/watch?v=gi_EDUK8J2w）への直接アクセスができなかったため、落合陽一氏の論文読解・リサーチ手法に関する複数のWeb情報源から手法を抽出しました。

## 1. リサーチフェーズ

### 1.1 論文選定と管理
- 週次で読む論文数を決定（例：週25本、集中コースでは週100本）
- 興味のあるカンファレンスを絞り込み、引用関係を辿る
- 研究分野の約10年分の知見を把握することを目標とする
- 論文PDFとサマリーをセットで管理する体制を構築
- 後から検索しやすいようにNotion等のデータベースツールを活用

### 1.2 効率的な読解順序
- アブストラクト → 結論 → 実験 → 関連研究の順で読む
- 頭から几帳面に読むのではなく、「その論文がしたかったこと」を真っ先に掴む
- リサーチクエスチョンを最初に把握する
- 細部は後から確認する戦略的アプローチを採用
- 自分のテーマにとっての意義を問題解決軸で判断

### 1.3 時間管理
- 1本の論文につき1時間以内と決める（だらだら読みを防止）
- 時間制限により集中力を維持
- 「1時間だけ」と決めて、重要と思ったら全文読む
- 読了時間を数時間から10-20分程度に短縮
- 反復トレーニングを通じて習得効率を向上

### 1.4 情報収集の観点
- 先行研究との比較視点を常に持つ
- 技術・手法の核心（キモ）を見極める
- 有効性検証の方法論を確認
- 議論の限界点を把握
- 次に読むべき論文を特定

### 1.5 批判的思考の適用
- 批判されている理論を特定
- 文脈・理路を追跡
- スコープ内での網羅性と整合性を評価
- 著者の想定した読み方ではなく、自分の問題解決のための読み方を実践
- 学術的価値と取り組むべき課題を明確化

## 2. 情報整理フェーズ

### 2.1 落合フォーマット（6つの質問）
実験論文用：
1. **どんなもの？** - アブストラクトを凝縮したまとめ
2. **先行研究と比べてどこがすごい？** - 学術的価値と差別化ポイント
3. **技術や手法のキモはどこ？** - 使用した方法論の整理
4. **どうやって有効だと検証した？** - 被験者、実験内容、得られたデータ
5. **議論はある？** - 結果解釈と批判的視点、限界点
6. **次に読むべき論文は？** - 次の学習対象の特定

### 2.2 理論研究向けバリエーション
- 批判されている理論の特定
- 文脈・理路の追跡
- スコープ内での網羅性と整合性評価
- 自動的に文献批判の骨組みを構築

### 2.3 図表の活用
- 重要な図表を含める
- 視覚的な情報整理を重視
- A4 1枚にまとめる際に図表を必ず含める
- 論文PDFの最初にサマリーを貼り付けて参照しやすくする

### 2.4 データベース管理
- Notionデータベースで論文サマリーを蓄積
- 検索性を高めるためのタグ付け・分類
- 論文PDFとサマリーをセットで管理
- 後から参照しやすい構造化

### 2.5 サマリー圧縮
- 1本の論文をA4半分〜1枚に圧縮
- 重要な情報のみを抽出
- 視覚的な整理により理解を深化
- 文献批判の骨組みを自動構築

## 3. ノート作成フェーズ

### 3.1 1枚スライドまとめ
- 内容を視覚的に整理したスライドを作成
- 6項目（概要、差別化、技術、検証、議論、次の論文）を含める
- A4 1枚にまとめて論文PDFの最初に貼り付け
- 図表を必ず含めて視覚的理解を促進
- まとめ作成により理解と定着を強化

### 3.2 執筆スタイル
- 本を書くのではなく、個人的なフィードバックループを作る
- 下の斜め位置から書き始める自由なスタイル
- 好きなところから書く（順序にこだわらない）
- 反復を通じた習得効率の向上を重視
- 学ぶ = トレーニング = 調練という定義

### 3.3 構造化された要約項目
落合陽一先生のサーベイ手法での要約項目：
1. 概要（三行）
2. 研究背景および先行研究との相違点
3. 提案法
4. 実験条件
5. 実験結果および考察

### 3.4 次のアクションへの接続
- 次に読むべき論文を明記
- 引用関係を辿る道筋を残す
- 研究の文脈を連続的に把握
- 自分のテーマとの関連性を記録

### 3.5 継続的な改善
- まとめ習慣により理解を定着
- 反復トレーニングで習得効率向上
- フィードバックループの構築
- 時間短縮を実現（数時間→10-20分）

## 5. 注意点・ベストプラクティス

### 5.1 効率化の原則
- 頭から几帳面に読むと時間が足りない → 戦略的読解順序を採用
- だらだら読みを避ける → 1時間の時間制限設定
- 読むだけでは定着しない → 必ず1枚スライドにまとめる
- 単なる要約ではない → 批判的視点を含める
- 著者の意図通りに読まない → 自分の問題解決のための読み方

### 5.2 情報整理の鉄則
- A4 1枚に圧縮（半分〜1枚）
- 図表を必ず含める
- 論文PDFの最初に貼り付けて参照性向上
- Notionデータベースで検索性を確保
- 次に読むべき論文を明記して連続性を保つ

### 5.3 学習スタイル
- 学ぶ = トレーニング = 調練
- 反復を通じた習得効率向上
- フィードバックループの構築
- 文系理系の区別なく、好きなことと得意なことを組み合わせる
- ロジックと感性の相互補完

### 5.4 時間投資の最適化
- 集中コース（週100本）で研究分野の10年分を把握
- 1論文10-20分への読了時間短縮を目指す
- 重要な論文のみ全文読みする選択眼を養う
- 時間制限により集中力を最大化

### 5.5 継続的改善
- まとめ習慣により理解と定着を強化
- データベース蓄積により過去の知見を活用
- 引用関係を辿り続けることで体系的理解を構築
- 2024年以降はLLMを活用した新しい読み方を模索
- 自分のテーマにとっての意義を常に問い続ける

### 5.6 4つの主要な学習アプローチ（ゼロヒャク教科書より）
1. **言語的アプローチ**: ロジカルシンキング能力を育成、家庭内での習慣化
2. **物理的思考**: 万物のことわりを理解、日常観察と五感を養う
3. **数学脳**: 統計的思考と解析的思考の組み合わせ、データ収集と法則性発見
4. **アート（感性）**: 個の見地と感性を重視、ロジックと感性の相互補完

### 5.7 現代的アップデート（2024年以降）
- 落合陽一自身が2024年にXで言及：「LLM優秀なので違う読み方があるのでは」
