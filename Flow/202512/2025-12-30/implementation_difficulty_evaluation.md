## I. 評価基準の定義

### 1. 実装要素数（1-3点）

| スコア | 説明 | 例 |
|-------|------|---|
| 3点 | 5個以上の要素を実装 | PMF（5要素）、CS（4要素） |
| 2点 | 3-4個の要素を実装 | GTM（3要素）、AARRR（5要素だが類似） |
| 1点 | 1-2個の要素を実装 | MoSCoW（1要素）、Must-Have Test（1要素） |

### 2. データ複雑度（1-3点）

| スコア | 説明 | 例 |
|-------|------|---|
| 3点 | 複数の外部データソースが必要 | PMF（サーベイ、成長率、NPS等） |
| 2点 | 1-2個の外部データソースが必要 | Cohort Analysis（LTV推移データ） |
| 1点 | 内部データのみで完結 | MoSCoW（機能リスト） |

### 3. 計算複雑度（1-3点）

| スコア | 説明 | 例 |
|-------|------|---|
| 3点 | 複雑なアルゴリズムが必要 | Cohort Analysis（時系列分析）、PMF（総合スコア） |
| 2点 | 中程度の計算ロジックが必要 | AARRR（ファネル分析）、GTM（ROI計算） |
| 1点 | 単純な計算で完結 | MoSCoW（分類のみ）、Must-Have Test（判定のみ） |

### 4. 既存資産の再利用性（-2～0点）

| スコア | 説明 | 例 |
|-------|------|---|
| -2点 | 既存スキルを大幅に再利用可能 | MoSCoW（`/validate-psf`拡張） |
| -1点 | 既存スキルを部分的に再利用可能 | Cohort Analysis（`/validate-unit-economics`拡張） |
| 0点 | 新規実装が必要 | PMF Validation（新規スキル） |

### 総合難易度スコア計算式

```
総合難易度スコア = 実装要素数 + データ複雑度 + 計算複雑度 - 既存資産再利用性
```

## II. 詳細評価（上位10フレームワーク）

### 1位: PMF Validation - 9点（Very Hard）

**スコア内訳**:
- 実装要素数: 3点（5要素: Sean Ellis, NPS, Churn, 成長率, LTV/CAC）
- データ複雑度: 3点（複数の外部データソース: サーベイ、成長率、NPS、Churn）
- 計算複雑度: 3点（PMF総合スコア算出ロジック、Sean Ellis率計算）
- 既存資産再利用性: 0点（新規スキル作成が必要）
- **総合難易度**: 3 + 3 + 3 - 0 = **9点**

**実装時間**: 8-12日

**難易度詳細**:
1. **Sean Ellisサーベイ実装**（3-5日）
   - サーベイテンプレート作成
   - N=100以上の回答収集機構
   - 「非常に残念」率の計算エンジン

2. **PMF補助指標実装**（3-4日）
   - NPS調査ツール
   - Churn Rate分析ツール
   - 成長率計算ツール（月次、YoY）

3. **PMF総合スコア算出**（2-3日）
   - 5指標の統合ロジック
   - PMF達成判定基準（Sean Ellis 40%以上等）
   - ダッシュボード表示

**既存資産の再利用**:
- LTV/CAC計算は既存スキル（`/validate-unit-economics`）で実装済み
- しかし、再利用率は20%程度（残り80%は新規実装）

**難易度が高い理由**:
1. 複数のデータソースが必要（サーベイ、成長率、NPS、Churn）
2. PMF総合スコア算出ロジックが複雑
3. Sean Ellisサーベイの回答収集機構が必要

### 2位: Growth Hacking (AARRR) - 8点（Very Hard）

**スコア内訳**:
- 実装要素数: 2点（5ステージだが類似パターン）
- データ複雑度: 3点（複数のデータソース: 訪問数、登録数、継続率、K-factor、ARPU）
- 計算複雑度: 3点（ファネル分析、コンバージョン率計算、K-factor計算）
- 既存資産再利用性: 0点（Acquisitionのみ実装済み、残り80%は新規）
- **総合難易度**: 2 + 3 + 3 - 0 = **8点**

**実装時間**: 5-8日

**難易度詳細**:
1. **Activation実装**（1-2日）
   - アクティベーション率測定
   - Aha! Moment特定

2. **Retention実装**（1-2日）
   - 継続率測定
   - Retention Curve分析

3. **Referral実装**（1-2日）
   - K-factor計算
   - NPS調査（PMFと統合可能）

4. **Revenue実装**（1-2日）
   - ARPU測定（既存スキルで部分実装済み）
   - アップセル・クロスセル分析

5. **ダッシュボード統合**（1-2日）
   - 5ステージ統合表示
   - ファネル分析ツール

**既存資産の再利用**:
- Acquisition（SNS）は実装済み（`/create-sns-content`）
- ARPU測定は部分実装済み（`/validate-unit-economics`）

**難易度が高い理由**:
1. 5ステージそれぞれに異なるデータソースが必要
2. ファネル分析ロジックが複雑
3. ダッシュボード統合の工数が大きい

### 3位: Customer Success - 7点（Hard）

**スコア内訳**:
- 実装要素数: 3点（4要素: CS vs サポート、ライフサイクル、ヘルススコア、チャーン対策）
- データ複雑度: 2点（2-3個のデータソース: ログイン頻度、機能利用率、NPS）
- 計算複雑度: 2点（ヘルススコア計算、チャーン予測モデル）
- 既存資産再利用性: 0点（新規スキル作成が必要）
- **総合難易度**: 3 + 2 + 2 - 0 = **7点**

**実装時間**: 2-3日

**難易度詳細**:
1. **ライフサイクル管理**（1日）
