## 6. Must-Have Test達成基準

| 指標 | 目標値 | 測定方法 |
|------|--------|----------|
| **Must-Have率** | 40%以上 | 「非常に残念」回答者数 / 総回答者数 |
| **セグメント別Must-Have率** | 特定セグメントで50%以上 | アーリーアダプターセグメント特定 |
| **支払い意思率** | 50%以上（既存CPF指標） | 「月XXX円払う」回答者数 / 総回答者数 |

**CPF v3総合判定基準**:
- ✅ **CPF達成**: 3U+1スコア28点以上 **かつ** Must-Have率40%以上
- ⚠️ **要改善**: 3U+1達成 **または** Must-Have率30-40%
- ❌ **見直し**: 3U+1スコア28点未満 **かつ** Must-Have率30%未満

## 7. Instructions（自動実行フロー）

**STEP 1-4**: 既存`/validate-cpf`（STEP 1-4）と同じ

**STEP 5（v3追加）: Must-Have Test実行**
```
1. Must-Have質問テンプレート生成

   質問文例:
   "もし[プロダクト名]が使えなくなったら、どう感じますか？"

   選択肢:
   A. 非常に残念（このプロダクトは私にとって必須）
   B. やや残念（代替品を探す）
   C. 残念ではない（特に困らない）

2. インタビュー結果の集計
   - 回答A（非常に残念）の人数カウント
   - Must-Have率 = A人数 / 総回答者数

3. セグメント別分析
   - ペルソナ別にMust-Have率を算出
   - 特定セグメント（例: 個人事業主、中小企業経営者）でのMust-Have率を確認

4. 支払い意思とのクロス分析
   - Must-Have回答者の支払い意思率（通常高い）
   - Must-Have率高 + 支払い意思率高 = アーリーアダプター確定
```

**STEP 6（v3追加）: アーリーアダプター特定**
```
1. Must-Have率50%以上のセグメント抽出
2. そのセグメントのペルソナ特徴を記録
3. 初期ターゲット顧客として推奨
```

**STEP 7: 総合判定（v3拡張）**
```
CPF v3判定基準:
- 3U+1スコア28点以上 ✅
- Must-Have率40%以上 ✅
→ CPF達成

- 3U+1スコア28点以上 ✅ / Must-Have率30-40% ⚠️
→ 要改善（アーリーアダプター再特定）

- 3U+1スコア28点未満 ❌ / Must-Have率30%未満 ❌
→ 見直し（課題再定義）
```

```
   - プロダクトを2回以上使用したユーザー
   - 直近30日以内にアクティブなユーザー

2. Sean Ellis質問テンプレート生成

   質問文:
   "もし[プロダクト名]が明日から使えなくなったら、どう感じますか？"

   選択肢:
   A. 非常に残念（このプロダクトは私にとって必須）
   B. やや残念（代替品を探すが困る）
   C. 残念ではない（特に困らない）
   D. すでに使っていない

3. サーベイ実施方法提案
   - メール配信（Mailchimp, SendGrid等）
   - アプリ内ポップアップ（Intercom, Typeform等）
   - 手動フォーム（Google Forms）

4. 回答収集
   - N=100以上の回答を収集
   - 回答率30%以上を目指す（送信数300-400）

5. Sean Ellis率計算

   Sean Ellis率 = A回答者数 / (A+B+C回答者数) × 100
   ※ D（すでに使っていない）は除外
```

### STEP 2: PMF補助指標測定

```
1. 月次成長率測定

   計算式:
   月次成長率 = (今月MRR - 前月MRR) / 前月MRR × 100

   スコア化:
   - 10%以上: 100点
   - 5-10%: 50点
   - 5%未満: 0点

2. NPS調査実施

   質問文:
   "このプロダクトを友人や同僚に勧める可能性は、0-10点でどのくらいですか？"

   NPS計算:
   - 推奨者（9-10点）比率
   - 批判者（0-6点）比率
   - NPS = 推奨者比率 - 批判者比率

   スコア化:
   - 50以上: 100点
   - 30-49: 50点
   - 30未満: 0点

3. Churn Rate測定

   計算式:
   Churn Rate = 今月解約ユーザー数 / 前月アクティブユーザー数 × 100

   スコア化:
   - 5%以下: 100点
   - 5-10%: 50点
   - 10%以上: 0点

4. LTV/CAC測定

   計算式:
   LTV/CAC = LTV / CAC
   ※ `/validate-unit-economics`から取得

   スコア化:
   - 3倍以上: 100点
   - 1.5-3倍: 50点
   - 1.5倍未満: 0点
```

### STEP 3: PMF総合スコア算出

```
1. 各指標のスコア化（0-100点）
   - Sean Ellis率: （実測値 / 40%） × 100（上限100点）
   - 月次成長率スコア: 上記ルールでスコア化
   - NPSスコア: 上記ルールでスコア化
   - Churnスコア: 上記ルールでスコア化
   - LTV/CACスコア: 上記ルールでスコア化

2. 重み付け合計

   PMF総合スコア =
     (Sean Ellisスコア × 0.4) +
     (成長率スコア × 0.2) +
     (NPSスコア × 0.15) +
     (Churnスコア × 0.15) +
     (LTV/CACスコア × 0.1)

   結果: 0-100点スケール

3. PMF総合判定
   - 70点以上: ✅ PMF達成
   - 50-69点: ⚠️ PMF未達成（改善中）
   - 50点未満: ❌ PMF未達成（見直し）
```
