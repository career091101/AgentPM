### 1.3 CPF Gap詳細

#### Gap 1.1: インタビュー数15%（3/20人）❌

**公式定義要求**: 20人以上（統計的有意性確保）

**Phase1実績**: 仮想インタビュー3人のみ

**問題**:
- サンプル数3人では統計的に信頼性低い（標準誤差大）
- 仮想インタビューは実インタビューの代替にならない（バイアスリスク）
- CPFスコア76%は「仮想3人では76%」であり、実20人では60%を下回る可能性

**影響**: Phase2で実インタビュー実施時にCPF未達成（60%未満）の確率30-40%

**対策**:
1. `/validate-cpf` v2: 実インタビューロードマップ自動生成
   - ターゲットリスト20-30人（IT部長10人、CTO 5人、経営層5人、その他10人）
   - インタビューガイド（3U Validation質問リスト）
   - スケジュール（Week 1-4で20人完了）
2. Phase2開始後1ヶ月以内に実20人完了を必須化

**優先度**: **P0**（Phase2開始前に対応必須）

**公式定義要求**: `startup_science/01_stages/cpf/3u_validation.md`

### 2.4 MVP選択フレームワーク整合性

**MVP 10類型**:
1. **Landing Page**: 説明ページ+メール登録（Dropbox）
2. **Concierge**: 手動でサービス提供（Food on the Table）
3. **Wizard of Oz**: 自動化のフリ（Zappos）

8. **Crowdfunding**: 事前注文（Pebble Watch）

- コスト: $0-$5,000

## Part 4: Unit Economicsフレームワーク整合性分析

### 4.1 Unit Economics公式定義（起業の科学）

**出典**: `startup_science/03_tactics/unit_economics/unit_eco_overview.md`

**Key Metrics**:

| 指標 | 計算式 | 目標値 |
|------|--------|--------|
| **LTV** | ARPU × 粗利率 / Churn率 | - |
| **CAC** | S&Mコスト / 新規顧客数 | - |
| **LTV/CAC比率** | LTV / CAC | **3.0以上**（健全） |
| **Payback Period** | CAC / (MRR × 粗利率) | **12ヶ月以内** |
| **Gross Margin** | (売上 - 変動費) / 売上 | **70%以上**（SaaS） |

**判定基準**:

| LTV/CAC比率 | 判定 | アクション |
|------------|------|-----------|
| **5.0+** | 優秀 | 積極投資 |
| **3.0-5.0** | 健全 | スケール開始 |
| **1.0-3.0** | 要改善 | 効率化優先 |
| **<1.0** | 危険 | Pivot検討 |

---

### 4.2 Phase1実装の詳細

**現状**: Unit Economics検証**不在** ❌

**Lean Canvasでの財務予測**:
- 年間売上: $9.65M
- 営業利益率: 56%
- ただし、**LTV/CAC/Payback Period未計算**

**問題**:
- LTV/CAC比率が1.0未満の可能性
  - 例: LTV $500K、CAC $600K → 比率0.83 ❌
- 不採算ビジネスがPSF通過するリスク

### 4.3 Unit Economics Gap詳細

#### Gap 4.1: LTV/CAC未検証 ❌ **NEW**

**公式定義要求**: LTV/CAC比率 3.0以上で健全

**Phase1実績**: 未計算

**試算**（保守的シナリオ）:

```
【LTV計算】
ARPU: $40K/月（年間契約$480K/年）
粗利率: 70%（SaaSベンチマーク）
Churn率: 5%/月（Enterprise B2B標準）
LTV = $40K × 0.7 / 0.05 = $560K

【CAC計算】（保守的）
S&Mコスト: $200K/顧客（営業3人体制、3ヶ月営業サイクル）
CAC = $200K

【LTV/CAC比率】
$560K / $200K = 2.8 ⚠️ 要改善（目標3.0未満）
```

**リスク**: LTV/CAC 2.8は「要改善」レベル、スケール投資判断困難

**対策**:
1. 新スキル `/validate-unit-economics` 作成
2. 3シナリオ分析（Conservative/Base/Optimistic）
3. LTV/CAC 3.0未満の場合、改善施策自動提案
   - CAC削減: 紹介プログラム導入（CAC -40%）
   - LTV向上: Churn率5%→3%へ低減（LTV +67%）
   - ARPU向上: 高単価プラン追加（$40K→$50K/月、+25%）

**優先度**: **P0**（Phase2開始前に必須、VC調達の前提条件）

---

#### Gap 4.2: Payback Period未計算 ⚠️ **NEW**

**公式定義要求**: Payback 12ヶ月以内（Enterprise B2Bは18ヶ月OK）

**Phase1実績**: 未計算

**試算**:

```
CAC: $200K
MRR: $40K/月
粗利率: 70%

Payback = $200K / ($40K × 0.7) = 7.1ヶ月 ✅
```

**評価**: 7.1ヶ月は**優秀**（目標12ヶ月以内を大幅クリア）

**ただし**: 年間前払い促進でPayback 1ヶ月以下も可能

#### Gap 4.3: Rule of 40未測定 ⚠️ **NEW**

**公式定義**: Rule of 40 = YoY成長率（%） + EBITDA Margin（%） ≥ 40%

**Phase1実績**: 未計算

**試算**（Year 2予測）:

```

Year 1: 20社 × $480K = $9.6M

```

**評価**: Rule of 40が250%は**超優秀**（目標40%を大幅超過）

## Part 5: Flywheelフレームワーク整合性分析

### 5.1 Flywheel公式定義（起業の科学）

**出典**: `startup_science/03_tactics/flywheel/flywheel_design.md`

**良いFlywheelの条件**:
- [ ] **自己強化する**: Output が次の Input を自動的に増やす
- [ ] **加速する**: 回るたびに速度が上がる
- [ ] **シンプル**: 3-5ステップで表現できる
- [ ] **測定可能**: 各ステップの指標が明確
- [ ] **持続可能**: 外部投入（広告費等）に依存しない

---

### 5.2 Phase1実装の詳細

**Phase1 Flywheel**: 4ループ設計

1. **Loop 1（Sticky Engine）**: 顧客成功 → 口コミ → 新規顧客 → データ蓄積 → 品質向上 → さらなる成功
2. **Loop 2（Data Network Effect）**: データ蓄積 → AIモデル精度向上 → 定着率UP → データ増加
3. **Loop 3（Brand Flywheel）**: 成功事例 → メディア露出 → ブランド確立 → 信頼性UP
4. **Loop 4（Talent Flywheel）**: 人材成長 → 採用力UP → チーム強化 → サービス品質UP

**評価**: ✅ **高品質**（Sticky + Viral + Data効果の組み合わせは理想的）

---

### 5.3 Flywheel改善提案

**公式定義との比較**:
- Loop 1-2は **Sticky Engine**（高Retention、低Churn）
- Loop 3は **Viral Engine**（口コミ・紹介）
- Loop 4は **Growth Support**（人材・組織）

**改善提案**:
1. **K-factor（バイラル係数）測定追加**
   - 定義: 1人の顧客が何人を招待するか
   - 目標: K > 1.0（指数関数的成長）
   - 計算: (招待数) × (招待受諾率)
2. **Retention Cohort分析追加**
   - Day 1: 40%以上
   - Day 7: 20%以上
   - Day 30: 10%以上

**優先度**: **P2**（Phase2で測定開始）
