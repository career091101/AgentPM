## 🚨 緊急対応が必要な問題（優先度: 高）

### 問題1: SNS数値データの禁止事項違反

**現状**: v2の全3案でSNS数値（いいね、リツイート、ビュー数）を記載

**具体例**:
```

```

**影響**:
- 信頼性の欠如（読者が検証不可）

### 問題2: 「想定エンゲージメント」セクションの記載

**現状**: v1の一部案で根拠のないエンゲージメント予測値を記載

```
❌ ### 想定エンゲージメント

    - **理由**: 根拠のない予測値は信頼性を損なう
```

## 具体的数値データの品質管理

### 最低基準
- **絶対最低ライン**: 5つ以上
- **推奨基準**: 7-10つ

### 有効な数値データ（推奨）
- ✅ 財務データ（売上、投資額、評価額、年収、資金調達額等）
- ✅ 市場データ（市場規模、成長率、シェア、TAM/SAM等）

- ✅ 研究データ（実験結果、統計値、サンプル数、信頼区間等）
- ✅ 時系列データ（年、月、日、期間、成長率、CAGR等）
- ✅ 人員データ（社員数、ユーザー数、利用者数等）

### 無効な数値データ（禁止）
- ❌ SNS数値（いいね、リツイート、ビュー、シェア、インプレッション等）
- ❌ 個人アカウントのフォロワー数、登録者数
- ❌ エンゲージメント予測値（"予測いいね500-800"等）
- ❌ 根拠のない予測（"おそらく〜万人"等）

### 数値データ不足時の対処法

SNS数値を削除した結果、数値データが5つ未満になった場合:

#### Step 1: ソースデータの再精査
- 元ツイート/記事/プレスリリースを再読み込み
- 見落とした数値データを発掘
- 引用元の公式発表を確認

#### Step 2: 関連市場データの追加
- 該当トピックの市場規模を検索（例: "3Dモデリング市場規模 2025"）
- 業界成長率、シェア率を調査
- 調査会社レポート（Gartner、IDC等）を参照

#### Step 3: 技術仕様・公開データの追加
- GitHub統計（スター数、フォーク数、コミット数）
- 公式ドキュメントの技術仕様
- 公開された実験結果、ベンチマーク

#### Step 4: 時系列データの明示化
- 「最近」→「2026年1月」
- 「先日」→「1月5日」
- 「数ヶ月前」→「2025年10月」

### 自動検証ロジック（推奨実装）

```python
def validate_numeric_data_quality(body_text):
    """数値データの数と質を検証"""
    import re

    # 有効な数値パターン
    valid_patterns = [
        r'\d+億ドル', r'\d+兆円', r'\d+億円', r'\d+万円',  # 財務
        r'\d+%', r'年率\d+%', r'CAGR\s?\d+%',  # 成長率
        r'バージョン\s?\d+\.?\d*', r'v\d+\.?\d*',  # バージョン
        r'\d{4}年', r'\d+月', r'\d+日',  # 時系列
        r'\d+人', r'\d+社', r'\d+件',  # 数量
    ]

    # 無効な数値パターン
    invalid_patterns = [
        r'いいね\s?\d+', r'リツイート\s?\d+',
        r'\d+ビュー', r'\d+インプレッション',
        r'@\w+', r'フォロワー\s?\d+',
    ]

    # 有効数値をカウント
    valid_count = sum(len(re.findall(p, body_text)) for p in valid_patterns)

    # 無効数値を検出
    invalid_matches = []
    for p in invalid_patterns:
        matches = re.findall(p, body_text)
        invalid_matches.extend(matches)

    return {
        "valid_count": valid_count,
        "invalid_matches": invalid_matches,
        "status": "pass" if valid_count >= 5 and len(invalid_matches) == 0 else "fail"
    }
```
