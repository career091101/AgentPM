## 1. Xパターン概要（現行）

### 1.1 基本仕様

**ソースファイル**: `takano_patterns_config.json`

| 項目 | 仕様 |
|------|------|
| **パターン数** | 7パターン（動的選択） |
| **文字数** | 700-1500字（標準型700-900字、ロング型1000-1500字） |
| **ハッシュタグ** | 最大2個まで |
| **絵文字** | 控えめ（2-4%程度） |
| **トーン** | 断定型+口語体ミックス（"マジで"、"ヤバい"、"〜の件"） |
| **数値データ** | 最低5個以上（推奨7-10個） |
| **固有名詞** | 企業名・人名を最低3回使用 |
| **問いかけ終結** | 100%義務化（7種のテンプレート） |

### 1.2 7パターン詳細

1. **数字 × 断定型** - 具体的数値・倍率・金額を最優先
2. **衝撃 × 企業名** - 著名人の発言や業界への衝撃を強調
3. **比較 × 驚異的数字** - 倍率・変化率の比較で驚きを演出
4. **問題提起 × 具体例** - 読者の課題や業界の変化を問いかける
5. **逆張り × 権威引用** - 一般論の逆を権威者の言葉で裏付け
6. **高野式「〜の件」型** - カジュアルかつインパクトのある口語体
7. **リスト型・衝撃ファクト連打** - 複数の事実を列挙してインパクト最大化

### 1.3 禁止事項（11項目）

- ハッシュタグ過多（3個以上）
- 絵文字の過度な使用
- 伝聞型表現（"〜と述べた"）
- 曖昧な表現（"〜かもしれない"）
- 呼びかけ表現（"経営者のあなたへ:"）
- 不要な@メンション
- 本文内のURL
- 文字数700字未満
- SNS数値データの言及
- 個人SNSアカウントの具体的言及
- メタ情報・制作過程の露出

## 3. Xパターン流用可能な部分

| 要素 | X仕様 | Threads調整案 | 理由 |
|------|-------|--------------|------|
| **文字数** | 700-1500字 | **300-500字**（本文）+ 添付テキスト（詳細） | 500字制限に対応 |

### 3.3 削除すべき要素（❌）

| 要素 | 理由 |
|------|------|
| **「〜の件」型タイトル** | 500字制限でタイトル+本文は非効率 |
| **リスト型・衝撃ファクト連打** | 改行制約で箇条書きが困難 |
| **ロング投稿（1000-1500字）** | 本文500字制限、添付テキストで代替 |

## 4. Threads特化の調整ポイント

### 4.1 文字数戦略

| パターン | 文字数 | 段落数 | 用途 |
|---------|--------|--------|------|
| **標準型** | 300-400字 | 2-3段落 | 時事ニュース・速報 |
| **フル活用型** | 450-500字 | 3-4段落 | データ分析・深掘り |
| **添付テキスト併用型** | 本文400字 + 添付1000-2000字 | 本文3段落 + 添付自由 | 詳細レポート・トレンド分析 |

### 4.2 改行・段落戦略

**制約**: 空白2行で自動ツリー化 → **1行改行のみ使用**

#### ✅ 推奨構成（標準型: 300-400字）

```
【Hook】（100-150字）
[衝撃的数字 × 企業名 × 断定型]

【Data】（100-150字）
[データ・事例の簡潔な展開]

【Insight + CTA】（100-150字）
[自己解釈 + 問いかけ]
```

#### ✅ 推奨構成（フル活用型: 450-500字）

```
【Hook】（120-150字）
[衝撃的数字 × 企業名 × 断定型]

【Data 1】（100-120字）
[データポイント1]

【Data 2】（100-120字）
[データポイント2]

【Insight + CTA】（130-150字）
[自己解釈 + 問いかけ]
```

### 4.3 ハッシュタグ戦略

**基本原則**: 1個（トピックタグ）のみ使用

| シーン | 推奨タグ | 理由 |
|--------|---------|------|
| AI/テクノロジー | #AI、#テクノロジー、#ChatGPT | トレンドトピックと連動 |
| ビジネス | #ビジネス、#スタートアップ、#経営 | CEO/経営者層へリーチ |
| 投資 | #投資、#スタートアップ投資、#VC | 財務データ重視投稿に適合 |

**禁止**: 複数ハッシュタグの羅列（#AI #ChatGPT #OpenAI #テクノロジー）

### 4.4 絵文字戦略

**Xパターン**: 控えめ（2-4%程度）
**Threads調整**: **やや多め（5-8%程度）**

#### 推奨配置

- **Hook**: 絵文字1個（注目喚起）
  - 例: 「🚨 OpenAIが30億ドル投資を発表。」
- **Data**: 絵文字なし（データの信頼性優先）
- **Insight**: 絵文字1個（感情表現）
  - 例: 「つまり、AI開発競争が激化している💡」
- **CTA**: 絵文字1個（親しみやすさ）
  - 例: 「あなたの会社、生き残れる？🤔」

### 4.5 トーン＆マナー戦略

| 要素 | X仕様 | Threads調整 | 理由 |
|------|-------|------------|------|
| **口語体** | "マジで"、"ヤバい"、"〜の件" | **より多用（3-5回）** | 若年層ユーザー層に合わせる |
| **断定型** | 必須 | **必須（変更なし）** | 思考リーダーシップ確立 |
| **問いかけ** | 100%義務化 | **100%義務化（変更なし）** | コメント促進でアルゴリズム優遇 |
| **権威引用** | 企業名・人名3回以上 | **2-3回**（字数制約） | 信頼性確保の最低限 |

### 4.6 視覚的要素の活用

**X**: 画像4枚、動画140秒
**Threads**: **画像10枚、動画5分** ✅

#### 推奨活用法

- **データ可視化**: グラフ・チャートを2-3枚添付
- **スライド形式**: 重要ポイントを画像5-10枚で展開
- **動画解説**: 3-5分の短編解説動画（データ・事例のビジュアル化）

## 7. 実装ガイドライン

### 7.1 文字数バリデーション（Threads版）

| パターン | 最小文字数 | 最大文字数 | 段落数 |
|---------|----------|----------|--------|
| **標準型** | 300字 | 400字 | 2-3段落 |
| **フル活用型** | 450字 | 500字 | 3-4段落 |
| **添付テキスト併用型** | 本文400字 + 添付1000-2000字 | 本文500字 + 添付10000字 | 本文3段落 + 添付自由 |

### 7.2 禁止事項（Threads版）

**Xから継承**:
- ハッシュタグ過多（❌ 3個以上 → ✅ 1個推奨）
- 伝聞型表現（❌ "〜と述べた"）
- 曖昧な表現（❌ "〜かもしれない"）
- 本文内のURL（❌ 500字圧迫）
- 文字数不足（❌ 300字未満）
- SNS数値データの言及（❌ いいね数、リツイート数）
- メタ情報の露出（❌ "想定エンゲージメント"）

**Threads特有の追加禁止事項**:
- ❌ 空白2行以上の改行（自動ツリー化）
- ❌ 複数ハッシュタグの羅列（#AI #ChatGPT #OpenAI）
- ❌ 絵文字過多（10個以上）
- ❌ タイトル+本文の二重構造（500字制限で非効率）

### 7.3 チェックリスト（Threads版）

**必須要素**:
- [ ] 文字数: 300-500字（添付テキスト併用可）
- [ ] 段落数: 2-4段落（改行1行のみ）
- [ ] ハッシュタグ: 1個（トピックタグ）
- [ ] 絵文字: 3-5個（Hook、Insight、CTAに配置）
- [ ] 口語体: 3-5回使用
- [ ] 断定型表現: 必須
- [ ] 問いかけ終結: 100%義務化
- [ ] 数値データ: 3-5個（字数制約考慮）
- [ ] 固有名詞: 2-3回
- [ ] 視覚的要素: 画像2-10枚または動画1本

**品質基準**:
- [ ] 冒頭1行で引き込み（絵文字+衝撃的数字+企業名）
- [ ] データポイントの簡潔な展開（各100字以内）
- [ ] 自己解釈の明確化（"つまり"、"ポイントは"使用）
- [ ] コメント促進の問いかけ（具体的・挑発的）
- [ ] 視覚的補完（画像・動画でデータ可視化）

## 9. まとめ

### 9.1 主要な差異

| 項目 | X | Threads |
|------|---|---------|
| **文字数** | 700-1500字 | 300-500字（+添付10000字） |
| **段落** | 4-8段落 | 2-4段落 |
| **ハッシュタグ** | 2個 | 1個 |
| **絵文字** | 2-4% | 5-8% |
| **口語体** | 2回 | 3-5回 |
| **トーン** | フォーマル〜カジュアル | ややカジュアル寄り |
| **視覚的要素** | 画像4枚、動画140秒 | 画像10枚、動画5分 |

### 9.2 最重要ポイント

1. **文字数削減**: 700-1500字 → 300-500字（最大67%削減）
2. **視覚的補完**: 画像10枚・動画5分でデータ可視化を強化
3. **カジュアル化**: 絵文字3-5個、口語体3-5回で若年層に訴求
4. **コメント促進**: 問いかけ終結100%義務化（アルゴリズム優遇）
5. **トピックタグ**: ハッシュタグ1個でトレンドトピック連動
