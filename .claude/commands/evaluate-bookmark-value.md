# Evaluate Bookmark Value

X投稿のブックマーク価値を7軸評価モデルで判定するコマンド。

## 使用方法

```bash
/evaluate-bookmark-value
```

実行後、以下のいずれかの方法で投稿を入力:

1. **テキスト直接入力**: 投稿本文を貼り付け
2. **JSONファイル指定**: 投稿データのJSONファイルパス
3. **バッチ評価**: 複数投稿を一括評価

## 評価軸（7軸、100点満点）

| 評価軸 | 配点 | 判定基準 |
|--------|------|----------|
| **実践的価値** | 20点 | 概念的理解の深さ、理論・原理の説明、比較分析の質 |
| **最新性** | 15点 | 技術トレンド、新機能リリース、最新ベストプラクティス |
| **データドリブン** | 15点 | 数値データ、実験結果、統計情報の有無 |
| **引用・参照性** | 15点 | 引用文化、参照リンク、情報源の明示 |
| **集合知評価** | 15点 | いいね数、RT数（高エンゲージメント = 1000+いいね） |
| **発信者専門性** | 10点 | 著者の専門性、過去の実績、一貫した発信 |
| **情報の深さ** | 10点 | 表面的でない洞察、批判的視点、今後の課題提示 |

## ユーザーの学習スタイル（823件分析結果）

- **97.1%が概念的学習**: 深い理解重視、実装は自分で
- **70.7%がAI・生成AI**: Claude/GPT/LLM特化
- **36.7%が引用あり**: 文脈重視、引用文化
- **いいね・RT相関0.89**: 集合知への強い信頼

## 評価基準詳細

### 1. 実践的価値（20点）

**概念的学習の質を重視**:

- **理論・原理（10点）**: 「〜とは」「仕組み」「原理」「理論」「解説」
- **比較・分析（6点）**: 「vs」「より」「比べて」「違い」「比較」
- **トレンド・ニュース（4点）**: 「発表」「リリース」「公開」「ローンチ」
- **すぐ実装可能（ボーナス+10点）**: URL + コードブロック/ダウンロードリンク（0.5%の稀少価値）
- **ベストプラクティス（3点）**: 「Tips」「コツ」「ポイント」

### 2. 最新性（15点）

投稿日からの経過日数:
- 1週間以内: 15点
- 1ヶ月以内: 10点
- 3ヶ月以内: 5点
- それ以上: 0点

### 3. データドリブン（15点）

数値データの有無:
- パーセンテージ（「〇%」）: 5点
- 倍数（「〇倍」）: 5点
- 件数（「〇件」）: 3点
- 実験・検証キーワード: 5点
- 具体的数値（2桁以上）: 2点

### 4. 引用・参照性（15点）

- 引用符「」あり: 10点（36.7%が該当）
- URL参照: 1つにつき3点（最大5点）

### 5. 集合知評価（15点）

いいね数（実データ分布）:
- 2000+いいね: 15点（超高エンゲージメント、TOP 5%）
- 1000+いいね: 12点（高エンゲージメント、TOP 15%）
- 500+いいね: 8点
- 100+いいね: 5点
- それ以下: 2点

### 6. 発信者専門性（10点）

**TOP 20専門性著者**（実データから抽出）:
- @tetumemo: 10点（avg 2,761 likes）
- @kenn: 10点（avg 1,535 likes）
- @shinyaaa_code, @AIBoom4: 9点
- @ai_database, @kumarobo: 8点
- @kamome885, @_kentakodama, @gomi_ningen: 7点
- その他TOP 20: 4-6点
- それ以外: 3点（デフォルト）

### 7. 情報の深さ（10点）

- 深い洞察（「洞察」「考察」「本質」「背景」「文脈」）: 5点
- 批判的視点（「課題」「限界」「問題」「しかし」）: 3点
- 今後の展望（「今後」「将来」「可能性」「期待」）: 2点

## 判定基準

| スコア範囲 | 判定 | アクション |
|-----------|------|-----------|
| 80-100点 | **VERY HIGH** | 即ブックマーク＋Notion登録＋要精読 |
| 60-79点 | **HIGH** | ブックマーク推奨 |
| 40-59点 | **MEDIUM** | 興味あれば保存 |
| 0-39点 | **LOW** | スキップ推奨 |

## カテゴリ判定

- **AI・生成AI（70.7%）**: claude, gpt, ai, llm, openai, anthropic, gemini
- **ビジネス・起業**: ビジネス、起業、スタートアップ、経営
- **開発**: 開発、エンジニア、プログラミング、コード
- **デザイン・UX**: デザイン、UX、UI
- **生産性**: 生産性、ツール、効率
- **その他**: 上記以外

## 実行フロー

1. **投稿データ取得**: ユーザーから投稿本文、著者名、エンゲージメントデータを取得
2. **7軸評価実行**: 上記評価基準に基づき各軸をスコアリング
3. **文脈分析**: LLMの強みを活かし、文脈・ニュアンス・隠れた価値を評価
4. **結果出力**: JSON形式で評価結果を出力
5. **（オプション）保存**: 結果をファイルまたはNotionに保存
