## 1. Financial KPI（財務指標）

### MRR（Monthly Recurring Revenue）★

**定義**: 月次経常収益（継続的な収益）

| 項目 | 値 | 単位 | データ出典 |
|-----|-----|------|----------|
| **当月MRR** | | ¥ / $ | [入力例: 決算データ、請求システム] |
| **前月MRR** | | ¥ / $ | |
| **前々月MRR** | | ¥ / $ | |

**月次成長率（計算値）**: `(当月 - 前月) / 前月 × 100` = ___%

**評価**: 目標値 10%/月成長 → □達成 / □要改善 / □未達成

---

### Burn Rate（月次資金消費額）★

**定義**: 毎月消費する現金額（収益を差し引いた純支出）

| 項目 | 値 | 単位 |
|-----|-----|------|
| **当月総支出** | | ¥ / $ |
| **当月総収益** | | ¥ / $ |
| **Net Burn Rate** | | ¥ / $ |

**計算式**: `総支出 - 総収益`

**評価**: 目標値 -¥50K以下 → □達成 / □要改善 / □未達成

---

### Runway（資金残存期間）★

**定義**: 現在の資金で何ヶ月経営できるか

| 項目 | 値 | 単位 |
|-----|-----|------|
| **現在の銀行残高** | | ¥ / $ |
| **月平均Burn Rate** | | ¥ / $ |
| **Runway** | | ヶ月 |

**計算式**: `銀行残高 / 月平均Burn Rate`

**評価**:
- 18ヶ月以上 = ✅ 安全
- 12-18ヶ月 = ⚠️ 警告（資金調達準備開始）
- 6-12ヶ月 = 🚨 緊急（調達活動開始）
- 6ヶ月以下 = 🔴 危機的（緊急対応）

---

### Gross Margin（粗利率）

**定義**: （収益 - 直接コスト）/ 収益 × 100

| 項目 | 値 | パーセンテージ |
|-----|-----|----------|
| **当月収益** | | |
| **当月直接コスト** | | |
| **Gross Margin** | | % |

**計算式**: `(収益 - 直接コスト) / 収益 × 100`

**評価**:
- SaaS: 70%以上 = ✅
- E-commerce: 30-40%以上 = ✅

## 2. Customer KPI（顧客指標）

### CAC（Customer Acquisition Cost）★

**定義**: 1人の顧客を獲得するのに要した費用

| 項目 | 値 | 単位 |
|-----|-----|------|
| **当月マーケティング支出** | | ¥ / $ |
| **当月新規顧客数** | | 人 |
| **CAC** | | ¥ / $ |

**計算式**: `当月マーケティング支出 / 当月新規顧客数`

**評価**:
- B2C目標: $100以下
- B2B目標: $1,000以上

---

### LTV（Customer Lifetime Value）★

**定義**: 1人の顧客から得られる生涯価値

| 項目 | 値 | 単位 |
|-----|-----|------|
| **平均顧客寿命** | | ヶ月 |
| **月当たりの平均利益** | | ¥ / $ |
| **LTV** | | ¥ / $ |

**計算式**: `平均顧客寿命 × 月当たりの平均利益`

**評価**:
- B2C目標: $300以上
- LTV/CAC比率: 3:1以上

---

### LTV/CAC Ratio（顧客生涯価値 / 獲得コスト）★

**定義**: ユニットエコノミクスの健全性を示す比率

| 項目 | 値 | 計算 |
|-----|-----|------|
| **LTV** | | ¥ / $ |
| **CAC** | | ¥ / $ |
| **LTV/CAC Ratio** | | :1 |

**計算式**: `LTV / CAC`

**評価**:
- 3:1以上 = ✅ 健全
- 1-3:1 = ⚠️ 要改善
- 1:1未満 = 🔴 ビジネスモデル見直し必須

---

### Churn Rate（月次解約率）★

**定義**: 月間で失われた顧客の割合

| 項目 | 値 | パーセンテージ |
|-----|-----|----------|
| **月初顧客数** | | 人 |
| **月末顧客数** | | 人 |
| **Churn Rate** | | %/月 |

**計算式**: `(月初 - 月末) / 月初 × 100`

**解約定義**（選択）:
- □ SaaS：契約解除
- □ Freemium：30日非アクティブ
- □ アプリ：アンインストール

**評価**:
- B2B目標: 5%/月以下 = ✅
- B2C目標: 7%/月以下 = ✅
- 要改善: 5-10%/月 = ⚠️

---

### NPS（Net Promoter Score）

**定義**: 顧客推奨度スコア

| 項目 | 値 |
|-----|-----|
| **アンケート対象数** | 人 |
| **Promoter（9-10点）** | 人（ _%） |
| **Passive（7-8点）** | 人（ _%） |
| **Detractor（0-6点）** | 人（ _%） |
| **NPS** | スコア |

**計算式**: `(Promoter数 / 総回答数 × 100) - (Detractor数 / 総回答数 × 100)`

**評価**:
- 50以上 = ✅ 優良
- 30-49 = ⚠️ 要改善
- 30未満 = ❌ 改善必須

---

### Retention Rate（月次継続率）

**定義**: ユーザーが月をまたいで利用を継続する割合

| 項目 | 値 | パーセンテージ |
|-----|-----|----------|
| **月初ユーザー数** | | 人 |
| **翌月も利用したユーザー数** | | 人 |
| **Retention Rate** | | % |

**計算式**: `翌月も利用したユーザー / 月初ユーザー × 100`

**評価**: 95%以上 = ✅

## 4. Growth KPI（成長指標）

### Monthly Growth Rate（月次成長率）★

**定義**: 主要指標の月次成長率（ビジネスモデルで選択）

**ビジネスモデル別選択**:

| ビジネスモデル | 指標 | 値 |
|---------|------|-----|
| □ SaaS | MRR成長率 | __% / 月 |
| □ コンシューマー | MAU成長率 | __% / 月 |
| □ マーケットプレイス | GMV成長率 | __% / 月 |

**3ヶ月移動平均**:
```
直近3ヶ月の成長率:
1ヶ月目: __%
2ヶ月目: __%
3ヶ月目: __%
平均: __% / 月
```

**評価**: 10%/月以上 = ✅

---

### Sean Ellis Test（PMF指標）★

**定義**: 「このプロダクトが使えなくなったら非常に残念か」に「非常に残念」と答えたユーザー割合

| 項目 | 値 | パーセンテージ |
|-----|-----|----------|
| **有効回答数（「よくわからない」除外）** | | 人 |
| **「非常に残念」回答数** | | 人 |
| **Sean Ellis スコア** | | % |

**計算式**: `「非常に残念」数 / 有効回答数 × 100`

**最小サンプル数**: 40人以上（理想: 50人以上）

**評価**:
- 40%以上 = ✅ PMF達成の兆候
- 30-39% = ⚠️ もう少しで達成
- 30%未満 = ❌ PMF未達成

---

### Viral Coefficient（バイラル係数）

**定義**: 1ユーザーが紹介する平均人数 × 紹介先の登録率

| 項目 | 値 | 計算 |
|-----|-----|------|
| **1ユーザーが紹介する平均人数** | | 人 |
| **紹介先の登録率** | | % |
| **Viral Coefficient** | | × |

**計算式**: `紹介人数 × 登録率`

**評価**:
- 1.0以上 = ✅ 指数関数的成長
- 0.5-1.0 = ⚠️ 緩やかな成長
- 0.5未満 = ❌ 自然成長限定的

---

### Activation Rate（アクティベーション率）

**定義**: 訪問者に対するサインアップ数の割合

| 項目 | 値 |
|-----|-----|
| **月間訪問者数** | 人 |
| **サインアップ数** | 人 |
| **Activation Rate** | % |

**計算式**: `サインアップ数 / 訪問者数 × 100`

**評価**: 2-5% = ✅

---

### Referral Rate（紹介率）

**定義**: 新規ユーザー中、紹介経由の割合

| 項目 | 値 |
|-----|-----|
| **新規ユーザー数** | 人 |
| **紹介経由新規ユーザー数** | 人 |
| **Referral Rate** | % |

**計算式**: `紹介経由数 / 新規ユーザー数 × 100`

**評価**:
- 15%以上 = ✅ 良好
- 10-15% = ⚠️ 平均的
- 10%未満 = ❌ 改善必要
