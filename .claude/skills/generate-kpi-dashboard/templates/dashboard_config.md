## ステップ1: ビジネスステージの選択

現在のビジネスステージを選択してください。Top 5 KPI や目標値が自動的に調整されます。

```
選択: □ Phase 2（PMF検証）
      □ Phase 3（初期スケール）
      □ Phase 4（スケール）
```

### ステージ別推奨設定

**Phase 2（PMF検証）の場合**:
- 優先指標：Sean Ellis、成長率、Churn、NPS、Burn Rate
- グラフ期間：3-6ヶ月（初期段階のため短期トレンド）
- アラート感度：高（閾値を厳しめに設定）
- ベンチマーク：初期段階標準値

**Phase 3（初期スケール）の場合**:
- 優先指標：MRR、LTV/CAC、成長率、Activation、Runway
- グラフ期間：6ヶ月（スケール開始時の推移確認）
- アラート感度：中（経営判断の基準に）
- ベンチマーク：業界平均値

**Phase 4（スケール）の場合**:
- 優先指標：MRR、LTV/CAC、Gross Margin、Viral Coefficient、Retention
- グラフ期間：12ヶ月（長期トレンド確認）
- アラート感度：低（安定期のため標準感度）
- ベンチマーク：業界Top 25%值

- Churn Rate: 5%/月

**その他（カスタム）**の場合:

## ステップ5: グラフ期間の設定

トレンドグラフの表示期間を設定してください。

```
選択: □ 3ヶ月（短期トレンド、初期段階向け）
      □ 6ヶ月（標準、推奨） ★デフォルト
      □ 12ヶ月（長期トレンド、スケール段階向け）
      □ カスタム：___ ヶ月
```

### グラフ期間別推奨用途

| 期間 | 用途 | ステージ |
|------|------|---------|
| **3ヶ月** | 短期的な改善効果の確認 | Phase 2 |
| **6ヶ月** | 一般的な経営レビュー | Phase 2-3 |
| **12ヶ月** | 長期トレンド確認、投資家報告 | Phase 3-4 |

## ステップ6: グラフタイプの選択

各KPIの表示形式をカスタマイズします（デフォルトで最適な形式が選択されています）。

```
デフォルト グラフタイプ:

KPI | グラフ形式 | 変更
----|-----------|-----
MRR | 折れ線 | □ 棒グラフに変更
CAC | 棒グラフ | □ 折れ線に変更
Churn Rate | 折れ線 | □ 変更なし
NPS | ドーナツ | □ 棒グラフに変更
Growth Rate | 棒グラフ | □ 折れ線に変更
Retention | 折れ線 | □ 変更なし
LTV/CAC | ゲージ | □ 棒グラフに変更
```

**推奨タイプ説明**:
- **折れ線グラフ**: 時系列推移（MRR、MAU、成長率等）
- **棒グラフ**: 月次比較、増減幅（CAC、LTV等）
- **ドーナツグラフ**: 割合表示（Churn分布、NPS分布）
- **ゲージチャート**: 目標達成度（Sean Ellis、LTV/CAC等）

## ステップ8: 比較設定

前期比・前年比等の比較を設定します。

```
□ 前月比を表示（デフォルト：有効）
□ 前年同月比を表示
□ 目標値との乖離を表示
□ 比較表示なし
```

### 比較表示の詳細

**前月比表示**:
```
□ 有効 - 毎月の増減を確認
  └─ 数値と %の両方を表示
  └─ 上昇時：📈、下降時：📉 記号を追加
```

**前年同月比表示**:
```
□ 有効 - 季節変動を考慮
  └─ 初年度の場合は表示なし
  └─ Phase 1-2向け
```

**目標値との乖離**:
```
□ 有効 - 達成度を可視化
  □ 達成時：✅
  □ 90%以上：⚠️
  □ 90%未満：❌
```

## ステップ10: 出力フォーマット

最終的な出力形式を選択します。

```
□ Markdown（デフォルト、推奨）
  └─ kpi_dashboard.md として出力
  └─ Flow/{YYYYMM}/{YYYY-MM-DD}/ に自動保存
```

## ステップ11: 自動更新スケジュール（オプション）

毎月ダッシュボードを自動生成・更新するスケジュール設定です（任意）。

```
□ 自動更新なし（手動実行のみ）
□ 週次自動更新を有効化
□ 月次自動更新を有効化
```

### 週次更新設定（選択時）

```yaml
頻度: 毎週月曜日
時間: 09:00
タイムゾーン: Asia/Tokyo

処理内容:
- KPIデータを自動収集
- ダッシュボードを再生成
- Slack通知を送信
```

### 月次更新設定（選択時）

```yaml
頻度: 毎月1日（月初）
時間: 09:00
タイムゾーン: Asia/Tokyo

処理内容:
- 前月のKPI確定
- ダッシュボード更新
- 投資家レポート生成
- 前月ファイルをアーカイブ
```

**更新時の自動通知**:

```
□ メール通知（to: _____________）
□ Slack通知（channel: ____________）
```
