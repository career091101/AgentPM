# 無料note記事365日分生成スキル（generate-note-free-pro）

## 概要

このスキルは、365日分の無料note記事（1万文字/記事）を自動生成し、有料noteや商品への戦略的な導線を設計します。

### 主要機能

1. **365日テーマカレンダー生成**
   - 曜日別カテゴリ分け（月曜: マインド系、火曜: スキル系、水曜: 事例系、木曜: ツール系、金曜: 実践系、土日: 総合・レビュー系）
   - ジャンル・ターゲット層に最適化されたテーマ自動生成
   - 重複回避、段階的な深化設計

2. **1万文字高品質記事生成**（full_article モード）
   - ペルソナの痛み・不安・恥・焦りを深く言語化
   - 6部構成（フック → 共感 → 問題提起 → 解決策 → 実践ステップ → まとめ＋CTA）
   - 読了率80%以上を目標とした構成

3. **有料noteへの戦略的CTA**
   - 無料記事で信頼構築、有料記事でマネタイズ
   - 自然な導線設計（押し売り感ゼロ）
   - 商品・サービスへの誘導

4. **品質検証**
   - 文字数チェック（9,000-11,000文字）
   - 読みやすさスコア算出
   - ペルソナ共感度評価

### 制約事項

- **文字数**: 9,000-11,000文字（目標: 10,000文字）
- **構成**: 6部構成固定（フック → 共感 → 問題提起 → 解決策 → 実践ステップ → まとめ＋CTA）
- **トーン**: 読者に寄り添う、共感型、専門性のある親しみやすさ
- **CTA**: 1記事あたり1-2箇所（過度な押し売り禁止）

## 入力仕様

### 必須パラメータ

| パラメータ | 型 | 説明 | 例 |
|-----------|---|------|---|
| `genre` | string | 発信ジャンル | "副業", "恋愛", "育児", "ビジネス", "自己啓発" |
| `product` | string | 販売したい商品・サービス | "副業コンサル", "有料note", "オンライン講座" |
| `target_audience` | string | ターゲット層の詳細 | "30代会社員、副業初心者、月5万円稼ぎたい" |

### オプションパラメータ

| パラメータ | 型 | デフォルト | 説明 |
|-----------|---|----------|------|
| `output_mode` | string | `"themes_only"` | `"themes_only"`: テーマ一覧のみ、`"full_article"`: 記事本文生成 |
| `article_index` | int | `null` | 記事番号（1-365）、`output_mode: "full_article"`時に必須 |
| `start_date` | string | `"2026-01-08"` | カレンダー開始日（ISO 8601形式） |
| `tone` | string | `"empathetic"` | トーン（`"empathetic"`: 共感型、`"professional"`: 専門家型、`"casual"`: カジュアル型） |

**目的**: ターゲット層の心理を深く理解し、共感できる記事を設計する。

**痛み（Pain）**:
- 「給料が上がらない」
- 「将来のお金が不安」
- 「子供の教育費が心配」
- 「このままだと老後破産する」

**不安（Anxiety）**:
- 「副業を始めても失敗するかも」
- 「詐欺に遭わないか不安」
- 「本業に支障が出ないか心配」
- 「家族との時間が減るのが怖い」

**恥（Shame）**:
- 「周りはもっと稼いでいるのに自分だけ」
- 「同期が昇進しているのに自分は」
- 「友人に副業の相談ができない」
- 「家族に心配かけたくない」

**焦り（Urgency）**:
- 「今すぐ行動しないと手遅れになる」
- 「子供の進学まであと〇年しかない」
- 「40代になる前に何とかしたい」
- 「同じ悩みを持つ人はもう始めている」

### STEP 2: テーマカレンダー生成（365日分）

**目的**: 曜日別カテゴリで戦略的にテーマを配信し、読者の継続的な関心を維持。

#### 2-1. 曜日別カテゴリの定義

LLMが以下のカテゴリルールを適用：

```markdown
**月曜日: マインド系**
- 目的: 1週間の始まりにモチベーション向上
- テーマ例:
  - 「副業で成功する人の共通点」
  - 「お金のマインドブロックを外す方法」
  - 「失敗を恐れない思考法」

**火曜日: スキル系**
- 目的: 具体的なスキル・知識の提供
- テーマ例:
  - 「Webライティングの基本5ステップ」
  - 「副業に必要なタイムマネジメント術」
  - 「初心者向けSEO対策入門」

**水曜日: 事例系**
- 目的: 成功事例・失敗事例で共感と学びを提供
- テーマ例:
  - 「副業で月10万円稼いだAさんの実例」
  - 「私が副業で失敗した3つの理由」
  - 「会社員が副業で成功する5つのパターン」

**木曜日: ツール系**
- 目的: 便利なツール・サービスの紹介
- テーマ例:
  - 「副業初心者におすすめのツール10選」
  - 「無料で使える便利なサービス」
  - 「時間を節約するアプリ活用法」

**金曜日: 実践系**
- 目的: 週末に実践できる具体的アクションプラン
- テーマ例:
  - 「今週末にできる副業準備3ステップ」
  - 「初心者が最初にやるべきこと」
  - 「副業の第一歩を踏み出す方法」

**土日: 総合・レビュー系**
- 目的: 週のまとめ、振り返り、長文深掘り
- テーマ例:
  - 「副業を始める前に知っておくべき10のこと」
  - 「今週の学びを振り返る」
  - 「副業の全体像を理解する完全ガイド」
```

#### 2-2. 365日テーマ生成ロジック

LLMが以下のルールでテーマを生成：

```markdown
**段階的な深化設計**:
- 第1四半期（1-90日目）: 基礎知識、マインドセット構築
- 第2四半期（91-180日目）: 実践スキル、具体的手法
- 第3四半期（181-270日目）: 応用テクニック、収益最大化
- 第4四半期（271-365日目）: 総まとめ、高度な戦略

**重複回避ロジック**:
1. 生成済みテーマリストをメモリ保持
2. 新規テーマ生成時に類似度チェック
3. 類似度70%以上の場合は別のテーマに変更
4. キーワードの完全一致は禁止

**テーマの多様性確保**:
- 同じカテゴリ内でも表現・切り口を変える
- 数値・固有名詞を積極的に使用（「5つの方法」「3ステップ」など）
- 問いかけ型・逆説型を織り交ぜる
```

**6. まとめ＋CTA（1,000-1,500文字）**

#### 3-2. 文字数配分の最適化

LLMが以下のルールで文字数を調整：

```markdown
**目標文字数**: 10,000文字（許容範囲: 9,000-11,000文字）

**各部の文字数配分**:
1. フック: 500-800文字（5-8%）
2. 共感パート: 1,500-2,000文字（15-20%）
3. 問題提起: 1,500-2,000文字（15-20%）
4. 解決策: 3,000-3,500文字（30-35%）← 最もボリュームを割く
5. 実践ステップ: 2,000-2,500文字（20-25%）
6. まとめ＋CTA: 1,000-1,500文字（10-15%）

**調整ルール**:
- 文字数不足の場合:
  - 解決策パートに具体例を追加
  - 実践ステップに詳細な手順を追加
  - 図解・箇条書きを文章で補足

- 文字数超過の場合:
  - 冗長な表現を削減
  - 重複内容を統合
  - 補足説明を簡略化
```

#### 3-3. 読みやすさの最適化

LLMが以下のテクニックを適用：

```markdown
**文章構造**:
- 1文は60文字以内（読点2つまで）
- 段落は3-5文程度
- 見出し（##、###）を5-10箇所挿入
- 箇条書きを積極的に使用

**強調表現**:
- 太字（**重要ポイント**）を10-15箇所
- 引用（> 〇〇）で読者の声を代弁
- 数値データを多用（「3つの方法」「80%の人が」など）

**読者エンゲージメント**:
- 問いかけを5-10箇所挿入
- 「あなたは〇〇ですか？」
- 「もし〇〇だとしたら…」

**視覚的な工夫**:
- セクション間に「---」（水平線）を挿入
- 重要な図解・チャート・表を3-5箇所提案（テキストで説明）
```

#### 3-4. ペルソナ共感度チェック

LLMが記事生成後に以下を検証：

```markdown
**チェック項目**:
1. ペルソナの痛み・不安・恥・焦りが5箇所以上言語化されているか
2. 読者の具体的なシチュエーションが3箇所以上描写されているか
3. 「あなただけじゃない」という安心感を与えているか
4. 押し売り感のない自然なCTAになっているか

**スコアリング（100点満点）**:
- 痛み・不安の言語化: 25点
- 共感表現の豊富さ: 25点
- 具体的な解決策の提示: 30点
- 自然なCTA: 20点

**合格基準**: 70点以上
```

### STEP 4: 品質検証

**目的**: 記事の品質を定量的に評価し、基準を満たしているか確認。

#### 4-1. 文字数チェック

LLMが以下を実行：

```markdown
1. **文字数カウント**:
   - 総文字数をカウント（空白・改行除く）
   - 目標: 9,000-11,000文字
   - 最適値: 10,000文字

2. **各部の文字数チェック**:
   - フック: 500-800文字
   - 共感パート: 1,500-2,000文字
   - 問題提起: 1,500-2,000文字
   - 解決策: 3,000-3,500文字
   - 実践ステップ: 2,000-2,500文字
   - まとめ＋CTA: 1,000-1,500文字

3. **不足・超過時の調整**:
   - 不足: 解決策パートに具体例を追加
   - 超過: 冗長な表現を削減
```

#### 4-2. 読みやすさスコア算出

LLMが以下の指標で評価：

```markdown
**読みやすさスコア（100点満点）**:

1. **文章の長さ（20点）**:
   - 1文が60文字以内: 20点
   - 1文が61-80文字: 15点
   - 1文が81-100文字: 10点
   - 1文が100文字超: 5点

2. **段落の長さ（20点）**:
   - 1段落3-5文: 20点
   - 1段落2文または6-7文: 15点
   - 1段落1文または8文以上: 10点

3. **見出しの適切性（20点）**:
   - 見出しが5-10箇所: 20点
   - 見出しが3-4箇所または11-15箇所: 15点
   - 見出しが2箇所以下または16箇所以上: 10点

4. **箇条書きの活用（20点）**:
   - 箇条書きが10-20箇所: 20点
   - 箇条書きが5-9箇所または21-30箇所: 15点
   - 箇条書きが4箇所以下または31箇所以上: 10点

5. **強調表現の適切性（20点）**:
   - 太字が10-15箇所: 20点
   - 太字が5-9箇所または16-25箇所: 15点
   - 太字が4箇所以下または26箇所以上: 10点

**合格基準**: 70点以上
```

#### 4-3. ペルソナ共感度評価

LLMが以下をチェック：

```markdown
**ペルソナ共感度スコア（100点満点）**:

1. **痛み・不安の言語化（25点）**:
   - 5箇所以上: 25点
   - 3-4箇所: 20点
   - 1-2箇所: 10点
   - 0箇所: 0点

2. **共感表現の豊富さ（25点）**:
   - 「あなたは〇〇ですか？」が5箇所以上: 25点
   - 3-4箇所: 20点
   - 1-2箇所: 10点
   - 0箇所: 0点

3. **具体的な解決策の提示（30点）**:
   - 5ステップ以上: 30点
   - 3-4ステップ: 25点
   - 1-2ステップ: 15点
   - 0ステップ: 0点

4. **自然なCTA（20点）**:
   - 押し売り感ゼロ: 20点
   - やや押し売り感あり: 10点
   - 明らかに押し売り: 0点

**合格基準**: 70点以上
```
