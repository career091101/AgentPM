# Create MVV Skill (ForSolo Edition)

MVV（Mission/Vision/Values）を早期定義する自律実行型Skill。ソロプレナー向けにMission/Vision定義（自由・収益性・社会貢献のバランス）を統合。

---

## このSkillでできること

1. **Mission定義**: 何のために存在するか（存在意義）を明確化
2. **Vision定義**: どこを目指すか（目指す未来）を描画
3. **Values定義**: どう行動するか（行動指針）を策定
4. **整合性検証**: MVV 3要素の相互整合性を確認
5. **Lean Canvas整合**: リーンキャンバスとの整合性を検証
6. **ForSolo特化**: 自由・収益性・社会貢献のバランス、1人実行可能性、ライフスタイル重視

5. Vision草案作成（3-5年後の理想状態、定量目標含む、ForSolo視点）

## 【ForSolo新機能】3軸バランス分析

**実装日**: 2026-01-02
**目的**: ソロプレナーが重視する3軸（自由・収益性・社会貢献）のバランスを明確化

### 3軸の定義

#### 1. 自由（Freedom）

**内容**:
- **場所の自由**: リモートワーク、デジタルノマド可能
- **時間の自由**: 自分のペースで働ける、週10-20時間運用
- **決定の自由**: 外部投資なし、自分で意思決定

**Vision例**:
- "5年後、世界中どこでも働ける環境を実現"
- "3年後、週15時間運用で月$10K収益達成"

**Values例**:
- "自律性を最優先、外部依存を最小化"
- "柔軟性を重視、固定スケジュールは避ける"

#### 2. 収益性（Profitability）

**内容**:
- **高LTV/CAC**: 10.0以上、超低CAC前提
- **MRR成長**: 月20%成長率、$1K → $5K → $10K
- **自動化収益**: 運用最小化、収益の自動化

**Vision例**:
- "3年後、MRR $10K達成、本業辞めて独立"
- "5年後、ARR $500K、複数プロダクト運営"

**Values例**:
- "収益性を追求、LTV/CAC 10.0+維持"
- "持続可能性を重視、スケールより利益率"

#### 3. 社会貢献（Impact）

**内容**:
- **課題解決**: 自分と同じ課題を持つ人を助ける
- **透明性**: Build in Public、学びを共有
- **コミュニティ**: IndieHackers、X（Twitter）で貢献

**Mission例**:
- "副業ソロプレナーの時間を10倍効率化し、独立を支援する"
- "ノーコード開発者の限界を突破するツールを提供する"

**Values例**:
- "透明性を重視、Build in Publicで進捗共有"
- "コミュニティファースト、成功を独占しない"

---

### 3軸バランススコア

**各軸の重要度（10点満点）**:

| 軸 | 重要度 | 典型的な配分 |
|----|:------:|------------|
| **自由** | XX/10 | 7-9点（最重要、ソロプレナーの核心動機） |
| **収益性** | XX/10 | 7-9点（独立・生活のため必須） |
| **社会貢献** | XX/10 | 5-7点（重要だが、自由・収益の後） |

**バランス判定**:
- **理想型**: 自由8点、収益性8点、社会貢献6点（合計22点）
- **自由重視型**: 自由9点、収益性7点、社会貢献5点（合計21点）
- **収益重視型**: 自由7点、収益性9点、社会貢献5点（合計21点）

**アンバランス警告**:
- 自由10点、収益性3点 → ⚠️ 収益性不足、ビジネス持続性リスク
- 収益性10点、自由2点 → ⚠️ 自由度不足、ソロプレナーの意義喪失
- 社会貢献10点、収益性3点 → ⚠️ 非営利活動、ビジネスではない

### MVV草案作成（ForSolo視点）

**フォーマット**:
```

```


- 収益性: 7/10（"独立"示唆）

**例2（収益重視）**:
```
ノーコード開発者の限界を突破するツールを提供し、
月$10K収益を実現可能にする
```
- 自由: 6/10（暗黙）
- 収益性: 9/10（"月$10K収益"明記）
- 社会貢献: 6/10（"提供し"）

**例3（社会貢献重視）**:
```
Build in Publicを実践し、失敗・学びを共有することで
次世代ソロプレナーの成功確率を2倍にする
```
- 自由: 7/10（Build in Public実践）
- 収益性: 5/10（暗黙）
- 社会貢献: 9/10（"次世代支援"明記）

**フォーマット**:

**ForSolo例**:

- ライフスタイル: デジタルノマド、週10時間

```markdown
**内容**: Build in Public、進捗・失敗を共有、学びを隠さない

**具体的行動**:
- [ ] X（Twitter）で毎日進捗共有（1-3投稿/日）

---

**内容**: LTV/CAC 10.0+維持、持続可能な成長、利益率重視

**具体的行動**:
- [ ] CAC最小化（広告なし、Viral成長）
- [ ] LTV最大化（継続課金、Churn Rate <5%）
- [ ] 利益率優先（売上成長 < 利益率改善）

---

### 4. 自動化推進

**内容**: 手動作業最小化、ツール活用、週20時間以内運用

**具体的行動**:
- [ ] 自動化率80%以上維持
- [ ] ノーコード/ローコードツール優先
- [ ] 週20時間以内運用（本業維持可能）

---

### 5. コミュニティファースト

**内容**: IndieHackers貢献、成功を独占しない、次世代支援

**具体的行動**:
```
