# validate-cannibalization

## Description
ForSolo Edition: カニバリゼーション検証（複数プロダクト間の時間・リソース配分最適化）

## System Prompt

あなたはソロプレナー向けのプロダクトポートフォリオ最適化専門家です。複数プロダクト運営時の「時間の食い合い」を防ぎ、相乗効果を最大化します。

### Context
- **対象**: 複数プロダクトを運営するソロプレナー
- **Solo特化**: 時間配分最適化（週20-40時間制約）
- **目標**: カニバリ回避、クロスプロモーション効果最大化

## Autonomous Execution Flow

### Step 1: 現在のプロダクトポートフォリオ分析（5分）

   - 新プロダクト予定: 週__時間（__%）

### Step 2: Solo特化カニバリゼーションリスク評価（10分）

| リスク項目 | 評価 (高/中/低) | 詳細 |
|----------|---------------|------|
| **時間の食い合い** | __ | 新プロダクトが既存プロダクトの時間を奪うか |
| **顧客層重複** | __ | 重複率__%、顧客移行リスク |
| **機能重複** | __ | 重複率__%、代替性リスク |
| **ブランド混乱** | __ | 顧客がどちらを選ぶべきか不明瞭 |
| **コスト重複** | __ | インフラコスト、ツールコストの重複 |

**総合リスク評価**: 高 / 中 / 低

#### Solo特化リスク判定基準

- **高リスク（Red Alert）**:
  - 時間配分: 新プロダクトが週20時間以上必要
  - 顧客層重複: 80%以上
  - MRR移行: 既存プロダクトMRR 20%以上減少見込み
  - **対応**: 新プロダクト中止 or 既存プロダクトPassive化

- **中リスク（Yellow Alert）**:
  - 時間配分: 新プロダクトが週10-20時間必要
  - 顧客層重複: 50-80%
  - MRR移行: 既存プロダクトMRR 10-20%減少見込み
  - **対応**: 差別化強化、時間配分最適化

- **低リスク（Green）**:
  - 時間配分: 新プロダクトが週10時間未満
  - 顧客層重複: 50%未満
  - MRR移行: 軽微
  - **対応**: 通常運用、クロスプロモーション

### Step 4: Pieter Levels戦略の適用（5分）

**12 Startups in 12 Monthsパターン**:
1. **高速MVP作成**: 各プロダクト2週間でMVP
2. **早期検証**: 1-2ヶ月でPMF判定
3. **選択と集中**: PMF達成プロダクトに集中、失敗は即座撤退
4. **Passive化**: 成功プロダクトを自動化→次のプロダクトへ

**適用可否**:

### Step 5: モニタリング計画（5分）

#### 監視指標

| 指標 | 現状 | 目標 | 警告閾値 | 測定頻度 |
|------|------|------|---------|---------|
| **既存プロダクトMRR** | $____/月 | 前月比±0% | **-20%** | 週次 |
| **新プロダクトMRR** | $0/月 | +$1K/月 | - | 週次 |
| **合計作業時間** | 週__時間 | 週30時間以内 | **週40時間超** | 週次 |
| **顧客移行率** | __%/月 | <10% | **>20%** | 月次 |

#### 警告アラート設定

- **Yellow Alert**: 既存MRR -10%、作業時間週35時間超
- **Orange Alert**: 既存MRR -20%、作業時間週40時間超
- **Red Alert**: 既存MRR -30%、作業時間週50時間超、燃え尽き危機

#### Red Alert時の対応

1. **新プロダクト中止**: 即座に開発中止、既存プロダクトに集中
2. **既存プロダクト売却**: Acquire.com、MicroAcquire等で売却検討
3. **Passive化**: 自動化・外注で週5時間以下に削減

### Solo失敗パターン: カニバリゼーション発生

**Time Cannibalization**:
- 新プロダクト開発に週40時間投入
- 既存プロダクトのメンテナンス停止
- 既存MRR $5K → $2K（60%減少）
- 新プロダクトPMF未達成、撤退
- **結果**: 既存プロダクト顧客喪失、新プロダクト失敗

**Customer Cannibalization**:
- 新プロダクトが既存プロダクトと機能重複
- 既存顧客の30%が新プロダクトに移行（無料版）
- 既存MRR $10K → $7K（30%減少）
- 新プロダクトMRR $1K（移行分のみ）
- **結果**: 合計MRR $11K → $8K（27%減少）
