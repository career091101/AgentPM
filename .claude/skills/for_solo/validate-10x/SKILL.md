# Validate 10x Skill (ForSolo Edition)

既存代替案と比較して10倍優位性を検証する自律実行型Skill。ソロプレナー向けに1人実行可能性チェック、ニッチ市場での6点許容基準を統合。

---

## このSkillでできること

1. **競合・代替案の特定**: 直接競合/間接競合/既存対処法を洗い出し
2. **5軸比較**: 時間/コスト/使いやすさ/成果/導入障壁で定量比較
3. **10倍判定**: 各軸で10倍以上の優位性があるか判定（ForSolo: 6点許容）
4. **改善アクション策定**: 未達成軸の改善案を提示
5. **ベンチマーク調査**: 成功企業の10倍優位性を参照
6. **ForSolo特化**: 1人実行可能性チェック、コスト制約評価、ニッチ市場対応

## Instructions

**実行モード**: 自律実行（対話なし）
**推定所要時間**: 40-70分

### 自動実行ステップ

1. リーンキャンバス読み込み
2. 競合・代替案の特定（Web調査）
3. **【v2新規】競合リサーチ成果物読込**（`competitor_research.md`）
4. 既存最良ソリューション選定
5. **【v2新規】Top 3競合特定**（`/research-competitors`成果物から）
6. 5軸比較表作成
7. **【v2新規】競合対比10倍検証**（vs. Top 3 Competitors）
8. **【ForSolo新規】1人実行可能性評価**
9. 10倍判定（各軸、ForSolo基準: 6点許容）
10. 総合判定
11. 改善アクション策定
12. ベンチマーク調査
13. 成果物出力

### 5軸比較

| 軸 | 計算式 | ForSolo重視度 |
|----|--------|--------------|
| 時間効率 | 既存の所要時間 / 新の所要時間 | 高（1人実行時間制約） |
| コスト | 既存の総コスト / 新の総コスト | 最高（$100/月制約） |
| 使いやすさ | 既存の学習コスト / 新の学習コスト | 高（技術サポートなし） |
| 成果・効果 | 新の成果 / 既存の成果 | 中 |
| 導入障壁 | 既存の導入ステップ数 / 新の導入ステップ数 | 高（1人セットアップ） |

### 判定基準（ForSolo調整版）

**10倍達成軸数による総合判定**:
- **2軸以上**: ✅ 10倍達成 → PSF検証/LP構築へ進む
- **1軸のみ**: ⚠️ 部分達成 → 強化軸を特定し再設計
- **0軸**: ❌ 未達成 → ソリューション根本見直し

**10倍優位性スコア基準（ForSolo調整）**:
- **ForStartup**: 7点以上（広い市場での優位性）
- **ForSolo**: **6点以上**（ニッチ市場での優位性許容）

**理由**: ソロプレナーはニッチ市場を狙うため、絶対的な10倍優位性は6点でも十分。重要なのは「特定セグメントでの圧倒的優位性」。

## 【ForSolo新機能】1人実行可能性評価

**実装日**: 2026-01-02
**目的**: 10倍優位性を達成しても、1人で実行不可能なら意味がない。実行可能性を明示的に評価。

### 評価項目（6点満点）

| 項目 | 評価基準 | スコア |
|------|---------|:------:|
| **技術的難易度** | ノーコード/ローコードで実装可能 | 2点 |
| **運用負荷** | 週40時間以内で運用可能 | 2点 |
| **初期投資** | $5,000以下で開始可能 | 1点 |
| **月額コスト** | $100以下で維持可能 | 1点 |

**総合判定**:
- **6点**: ✅ 完全に1人実行可能 → 即実行推奨
- **4-5点**: ✅ 条件付き実行可能 → 外部ツール活用
- **2-3点**: ⚠️ 困難 → スコープ縮小検討
- **0-1点**: ❌ 実行不可能 → ビジネスモデル見直し

### 実行可能性チェックリスト

**技術的難易度（2点）**:
- [ ] ノーコードツール（Bubble, Webflow等）で構築可能（2点）
- [ ] ローコード（Supabase, Firebase等）で構築可能（1点）
- [ ] フルコーディング必須（0点）

**運用負荷（2点）**:
- [ ] 自動化率80%以上、週10時間以内（2点）
- [ ] 自動化率50%以上、週20時間以内（1点）
- [ ] 手動作業中心、週40時間超（0点）

**初期投資（1点）**:
- [ ] $1,000以下（1点）
- [ ] $1,001-$5,000（0.5点）
- [ ] $5,001以上（0点）

**月額コスト（1点）**:
- [ ] $50以下（1点）
- [ ] $51-$100（0.5点）
- [ ] $101以上（0点）

## ドメイン知識

### ニッチ市場での10倍優位性（6点許容）

**ForStartup vs ForSolo基準の違い**:

| 項目 | ForStartup | ForSolo |
|------|-----------|---------|
| **ターゲット市場** | 広い市場（TAM $1B+） | ニッチ市場（TAM $10-100M） |
| **10倍優位性スコア** | 7点以上 | **6点以上**（ニッチ許容） |
| **必要な優位性** | 複数軸で圧倒的優位 | 特定セグメントで圧倒的優位 |

**ニッチ市場成功パターン**:
- **Marc Lou（ShipFast）**: Indie Hacker向けボイラープレート
  - ターゲット: Next.js × Supabase開発者（ニッチ）
  - 10倍優位性: 時間効率（1週間 → 1日、7倍）、コスト（$200 vs $0、∞倍）
  - **総合スコア**: 6.5/10点 → ForSolo基準で合格、MRR $40K達成

- **Tony Dinh（BlackMagic.so）**: X（Twitter）分析ツール
  - ターゲット: X（Twitter）クリエイター（ニッチ）
  - 10倍優位性: 使いやすさ（1クリック vs 10クリック、10倍）
  - **総合スコア**: 6.0/10点 → ForSolo基準で合格、MRR $50K達成

**結論**: ニッチ市場では6点でも「特定セグメントで圧倒的」なら成功可能。

### 1人実行可能性チェック

**成功ソロプレナーの共通パターン**:
1. **技術スタック**: ノーコード/ローコード優先（Bubble, Webflow, Supabase）
2. **自動化率**: 80%以上（Zapier, Make.com活用）
3. **運用時間**: 週10-20時間（本業維持可能）
4. **月額コスト**: $50-100（固定費最小化）
5. **初期投資**: $1,000-3,000（最小限でスタート）

**チェック項目**:
- [ ] ノーコードツールで90%構築可能
- [ ] 週20時間以内で運用可能（自動化後）
- [ ] 初期投資$3,000以下
- [ ] 月額コスト$100以下

**不合格時の対応**:
- スコープ縮小（MVP最小化）
- 外部ツール活用（自前開発 → SaaS活用）
- ビジネスモデル変更（受託 → プロダクト → SaaS）
