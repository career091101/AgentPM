# Simulate Interview Skill (ForSolo Edition)

仮想ペルソナとのインタビューをシミュレートする自律実行型Skill。ソロプレナー向けにSNS観察による検証手法（X/Twitter分析、IndieHackers活用）を統合。

---

## このSkillでできること

1. **ペルソナ役を演じる**: LLMがペルソナになりきってインタビューに回答
2. **UVP刺さり度測定**: 10の質問で40点満点のスコアリング
3. **課題深堀り**: 4U（Unworkable/Unavoidable/Urgent/Underserved）の検証
4. **既存代替案評価**: 何を使っていて、何が不満か
5. **WTP（支払い意思）確認**: いくらなら払うか
6. **ForSolo特化**: SNS観察による検証手法、X/Twitter分析、IndieHackers投稿分析

## Instructions

**実行モード**: 自律実行（対話なし）
**推定所要時間**: 25-45分

### 自動実行ステップ

1. ペルソナ・リーンキャンバス読み込み
2. ペルソナ役のキャラクター設定（詳細な人物像構築）
3. **【ForSolo新規】SNS行動パターン分析**（X/Twitter、IndieHackers）
4. インタビュー質問生成（10問）
5. ペルソナ役として回答生成
6. UVP刺さり度スコアリング（40点満点）
7. **【v2拡張】4U検証**（40点満点、Underserved追加）
8. 既存代替案評価
9. WTP（支払い意思）確認
10. **【ForSolo新規】SNS観察による裏付け**
11. インサイト抽出
12. 成果物出力

### 質問カテゴリ（10問、ForSolo調整）

1-2問: 課題認識（どんな困りごとがあるか、副業時間での制約）
3-4問: 現状対処法（今どうしているか、月$100以下制約）
5-6問: UVP提示（このソリューションどう思うか、1人実行可能性）
7-8問: 機能・価格感（何があればいいか、$10-50/月なら払うか）
9-10問: 行動意思（使いたいか、X/IndieHackersで紹介するか）

### 判定基準（ForSolo調整版）

**UVP刺さり度（40点満点）**:
- 32-40点: ✅ 良好 → CPF検証/実インタビューへ
- 24-31点: ⚠️ 要改善 → UVP/Solutionを調整
- 0-23点: ❌ 要見直し → Problem/ペルソナから再検討

**4U検証スコア（40点満点、ForSolo基準）**:
- **35-40点**: ✅ 優秀 → 4U完全合格、即CPF検証合格、PSF検証へ
- **28-34点**: ✅ 良好 → 3-4U合格、CPF検証合格、PMF達成可能性高
- **21-27点**: ⚠️ 要改善 → 2U程度合格、ペルソナ/Problem絞込み、ニッチ化検討
- **0-20点**: ❌ 不合格 → 1U以下、Problem再定義、Pivot検討

**CPF閾値**:
- **ForStartup**: 70%
- **ForSolo**: **60%**（ニッチ市場、仮説検証重視）

## 【ForSolo新機能】SNS観察による検証手法

**実装日**: 2026-01-02
**目的**: 仮想インタビューだけでなく、実際のSNS投稿から課題・ペインを裏付ける

### STEP 1: X（Twitter）分析

**検索クエリ例**:
```
"[課題キーワード] 困っている" OR "[課題キーワード] 時間かかる"
"[課題キーワード] 自動化したい" OR "[課題キーワード] ツールない"
"[課題キーワード] 副業" OR "[課題キーワード] ソロプレナー"
```

**分析項目**:
- [ ] 課題言及ツイート数（過去30日）
- [ ] エンゲージメント率（いいね、RT、リプライ）
- [ ] ペインの強度（感情表現: "最悪"、"無理"、"やばい"等）
- [ ] 既存ツールへの不満ツイート
- [ ] WTP示唆（"$XX払ってもいい"等）

**スコアリング**:
- 言及100+ツイート/月: 5点
- 言及50-99ツイート/月: 3点
- 言及10-49ツイート/月: 1点
- 言及10未満/月: 0点

### STEP 2: IndieHackers分析

**検索範囲**:
- フォーラム投稿（過去3ヶ月）
- Product Hunt投稿（過去6ヶ月）
- ニュースレター（過去1年）

**分析項目**:
- [ ] 同じ課題を持つスレッド数
- [ ] 既存ツールへの不満投稿
- [ ] "誰かこのツール作って"リクエスト
- [ ] 競合ツールの評価・レビュー
- [ ] WTP（価格議論、"$XX/月なら払う"）

**スコアリング**:
- 関連スレッド10+: 5点
- 関連スレッド5-9: 3点
- 関連スレッド1-4: 1点
- 関連スレッド0: 0点

### STEP 3: 総合SNS裏付けスコア

**計算式**:
```
SNS裏付けスコア = X（Twitter）スコア + IndieHackersスコア
```

**判定**:
- **8-10点**: ✅ 強い裏付けあり（実需確認）
- **5-7点**: ✅ 中程度の裏付けあり（検証継続）
- **2-4点**: ⚠️ 弱い裏付け（仮説検証重視）
- **0-1点**: ❌ 裏付けなし（Problem再定義）

### 必須検証項目（起業の科学準拠、4U完全版）

| 要素 | 質問例 | 評価基準 | スコア（10点満点） |
|------|--------|----------|-------------------|
| **Unworkable（現状不可能）** | 「今この課題を解決しないと何が起こるか?」 | 具体的損失を言語化<br>（売上減・工数増・機会損失等） | 9-10点: 致命的損失<br>7-8点: 大きな損失<br>5-6点: 中程度損失<br>0-4点: 損失軽微 |
| **Unavoidable（回避不可）** | 「ターゲットの何%がこの課題に直面するか?」 | 70%以上が直面<br>外部要因で不可避 | 9-10点: 90%以上<br>7-8点: 70-89%<br>5-6点: 50-69%<br>0-4点: 50%未満 |
| **Urgent（緊急性）** | 「今日解決したいか、来年でいいか?」 | 「今すぐ」「今期中必須」回答 | 9-10点: 今日・今週<br>7-8点: 今月・今期<br>5-6点: 半年以内<br>0-4点: 1年以降 |
| **Underserved（代替不足）** | 「現在の代替手段は十分か? 不満点は?」 | 既存代替案に強い不満<br>「満足な解決策なし」明言 | 9-10点: 代替なし<br>7-8点: 代替あるが不満大<br>5-6点: 代替あり不満中<br>0-4点: 代替で満足 |

### 4U改善チェックリスト（ForSolo視点）

**各U軸で7/10未満の場合の改善施策**:

#### Unworkable（現状不可能）が低い場合（<7/10）
- [ ] Problem定義を再検討（本当に「不可能」な課題か?）
- [ ] ペルソナを絞込み（より切実な課題を持つセグメントへ）
- [ ] 「損失の定量化」を強化（時間減・コスト減を数値化）
- [ ] **ForSolo**: 副業時間圧迫、本業影響を明確化

#### Unavoidable（回避不可）が低い場合（<7/10）
- [ ] ペルソナの「外部要因」を明確化（技術トレンド、競争激化等）
- [ ] TAM（市場規模）再計算（課題を持つ人が少ない可能性）
- [ ] Customer Segment Pivot検討（回避不可な課題を持つ別セグメントへ）
- [ ] **ForSolo**: X/IndieHackersで課題言及調査

#### Urgent（緊急性）が低い場合（<7/10）
- [ ] 「今すぐ解決したい理由」を深堀り（期限・外圧等）
- [ ] Customer Need Pivot検討（より緊急性高い別課題へ）
- [ ] ペルソナ除外（緊急性低いセグメントは優先度下げる）
- [ ] **ForSolo**: X（Twitter）で"今すぐ"言及ツイート調査

#### Underserved（代替不足）が低い場合（<7/10）
- [ ] 既存代替案の「不満点」を詳細調査
- [ ] 10倍優位性の再定義（代替案より10倍優れているか?）
- [ ] Value Capture Pivot検討（既存代替案と異なる収益モデルへ）
- [ ] **ForSolo**: IndieHackersで代替ツール評価調査

## ドメイン知識

### SNS観察による検証手法

**X（Twitter）高度検索テクニック**:

```
検索クエリパターン:
1. 課題言及: "[課題] 困っている" OR "[課題] 時間かかる"
2. 自動化ニーズ: "[課題] 自動化したい" OR "[課題] ツール探してる"
3. 既存ツール不満: "[ツール名] 高い" OR "[ツール名] 使いにくい"
4. WTP示唆: "$XX 払ってもいい" OR "[課題] お金払う"
5. ソロプレナー限定: "副業" OR "ソロプレナー" OR "indie hacker"

フィルター:
- 言語: 日本語 OR 英語
- 期間: 過去30日
- エンゲージメント: 最小10 likes

分析項目:
- ツイート数: 課題の広がり
- エンゲージメント率: 共感度
- 感情表現: ペインの強度
- 具体的数字: WTP、時間、コスト
```

**IndieHackers検索テクニック**:

```
検索範囲:
1. フォーラム投稿（全カテゴリ）
2. Product投稿（関連ツール）
3. ニュースレター（月次収益レポート）

検索キーワード:
- "[課題キーワード]" + "struggling" / "pain point" / "frustrating"
- "[既存ツール名]" + "expensive" / "complicated" / "alternatives"
- "[ターゲット職種]" + "side project" / "solopreneur" / "indie maker"

分析項目:
- スレッド数: 課題の普遍性
- コメント数: 関心度
- Upvote数: コミュニティ共感
- 価格議論: WTP、支払い根拠
```

**成功パターン例**:

- **Marc Lou（ShipFast）**: X（Twitter）で"Next.js boilerplate expensive"検索 → 50+ツイート/月 → $200 vs $0の不満確認 → ShipFast開発（MRR $40K）
- **Tony Dinh（BlackMagic.so）**: IndieHackersで"Twitter analytics tool"検索 → 30+スレッド → "簡単なツールない"要望 → BlackMagic.so開発（MRR $50K）

### X（Twitter）/IndieHackers分析ツール

**無料ツール（$100制約内）**:
- **X（Twitter）高度検索**: 無料、リアルタイム
- **IndieHackers検索**: 無料、コミュニティ評判
- **Google Trends**: 無料、検索ボリューム推移
- **Reddit Search**: 無料、サブレディット特化

**有料ツール（オプション）**:
- **Tweet Hunter**（$29/月）: X分析、エンゲージメント追跡
- **Social Blade**（無料〜$4/月）: フォロワー推移
