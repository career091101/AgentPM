# PMF検証スキル（Validate PMF）

Product Market Fit（PMF）達成を4つの定量指標で総合判定する自律実行型スキル。Sean Ellisテスト、月次成長率、Churn Rate、NPSを統合評価し、スケール準備の可否を判定します。

## スキル概要

### 対象ステージ
- **ステージ**: Phase2（PMF検証）
- **前提条件**: PSF達成済み、MVP稼働開始、アクティブユーザー40人以上
- **所要時間**: 20-40分（アンケート設計含む）

### このスキルでできること

1. **4指標統合評価** - Sean Ellisテスト、月次成長率、Churn Rate、NPSを一元判定
2. **PMF達成判定** - 起業の科学準拠の定量基準で達成/要改善/未達成を判断
3. **自動アンケート設計** - Sean Ellisテスト・NPSのアンケートテンプレート自動生成
4. **ボトルネック特定** - どの指標が不足しているか、改善アクションを提案
5. **Phase3移行判断** - スケールへ進むか、改善すべきかを提示

## PMF達成の4指標

### 1. Sean Ellisテスト（必須40%以上）

**定義**: ユーザーに「このプロダクトが使えなくなったら非常に残念か」を聞き、「非常に残念」回答率が40%以上なら PMF達成の兆候。

**計算式**:
```
Sean Ellisスコア = （「非常に残念」回答数 / 有効回答数） × 100
```

**基準値**:
- ✅ 40%以上 = 合格（PMF達成の兆候）
- ⚠️ 30-39% = 要改善（もう少しで達成）
- ❌ 30%未満 = 不合格（PMF未達成）

**最低サンプル数**: 40人以上（理想は50人以上）
**測定方法**: アンケート配信 → 1週間で回答収集 → リマインダー2回

**意義**: コアバリュー（顧客にとって無くてはならない機能）が明確か、ユーザーが本当に必要としているか を測定

---

### 2. 月次成長率（必須10%以上/月）

**定義**: MRR（SaaS）、MAU（コンシューマー）、GMV（マーケットプレイス）などの主要指標が月10%以上の成長を維持しているか。

**計算式**（3ヶ月移動平均）:
```
成長率 = （当月値 - 前月値） / 前月値 × 100

例: 10月→11月: 120 → 132  = +10%
   11月→12月: 132 → 145  = +10%
   3ヶ月平均: （10% + 10%） / 2 = 10%
```

**基準値**:
- ✅ 10%/月以上 = 合格（健全な成長）
- ⚠️ 5-9%/月 = 要改善（やや遅い）
- ❌ 5%/月未満 = 不合格（成長不足）

**選択指標**（ビジネスモデルで選択）:
- SaaS: MRR（月次経常収益）
- コンシューマー: MAU（月間アクティブユーザー）
- マーケットプレイス: GMV（取引総額）

**測定期間**: 直近3ヶ月データ必須

**意義**: 市場が本当に必要とし、需要が成長しているか を測定

---

### 3. Churn Rate（必須5%以下/月）

**定義**: ユーザー解約率が月5%以下（B2B基準）または7%以下（B2C基準）に抑えられているか。高い解約率は顧客満足度が低いサイン。

**計算式**:
```
月次Churn Rate = （月初顧客数 - 月末顧客数） / 月初顧客数 × 100

例: 100人から月末85人に → Churn Rate = 15% (NG)
    100人から月末97人に → Churn Rate = 3% (OK)
```

**解約定義**（プロダクトタイプで選択）:
- SaaS（有料）: 契約解除
- Freemium: 30日間非アクティブ
- アプリ: アンインストール

**基準値**:
- ✅ 5%/月以下 = 合格（B2B基準）
- ✅ 7%/月以下 = 合格（B2C基準）
- ⚠️ 6-10%/月 = 要改善（やや高い）
- ❌ 10%/月以上 = 不合格（離脱が多すぎ）

**測定方法**: Cohort分析で登録月別の残存率を追跡

**意義**: ユーザーが使い続けているか、プロダクトが期待に応えているか を測定

---

### 4. NPS（必須50以上）

**定義**: Net Promoter Score。「このプロダクトを友人・同僚に薦める可能性」を0-10点で測定し、推奨者（9-10点）と批判者（0-6点）の割合差でスコア化。

**計算式**:
```
NPS = （Promoter数 / 総回答数 × 100） - （Detractor数 / 総回答数 × 100）

例: 100人中、Promoter 60人、Passive 30人、Detractor 10人
NPS = 60% - 10% = 50
```

**分類**:
| スコア | 分類 | 意味 |
|-------|------|------|
| 9-10点 | Promoter（推奨者） | 熱烈なファン、口コミで広げる |
| 7-8点 | Passive（中立者） | 満足しているが受動的 |
| 0-6点 | Detractor（批判者） | 不満を持っている、悪口も言う |

**基準値**:
- ✅ 50以上 = 合格（優良スコア）
- ⚠️ 30-49 = 要改善（平均的）
- ❌ 30未満 = 不合格（改善必須）

**最低サンプル数**: 40人以上（理想は50人以上）

**意義**: ユーザー満足度と口コミ効果（オーガニック成長）を測定

## 総合判定基準

| 判定 | 条件 | 意味 | 次のアクション |
|------|------|------|---------------|
| ✅ **PMF達成** | 4指標すべて✅ | 市場ニーズが明確で成長・満足度が安定 | Phase3（スケール）へ移行 |
| ⚠️ **要改善** | 2-3指標✅ | 部分的には成功だが改善余地あり | 1-2ヶ月改善後に再診断 |
| ❌ **PMF未達成** | 0-1指標✅ | 根本的に市場との適合性に課題 | Pivot検討（/pivot-decision） |

## 出力サンプル

### エグゼクティブサマリー

**総合判定**: ⚠️ 要改善

| 指標 | 目標 | 実績 | 判定 |
|------|------|------|:----:|
| Sean Ellisテスト | ≥40% | 38% | ⚠️ |
| 月次成長率 | ≥10%/月 | 12.3%/月 | ✅ |
| Churn Rate | ≤5%/月 | 4.8%/月 | ✅ |
| NPS | ≥50 | 48 | ⚠️ |

**判定理由**: 4指標中2個が目標未達成。Sean Ellisテストが38%（目標40%）、NPSが48（目標50）とともに要改善。特にコアバリュー（Sean Ellis）と推奨意図（NPS）の強化が急務。

### 改善アクション（未達成指標）

| 指標 | 現状 | 改善アクション | 優先度 |
|------|------|---------------|--------|
| Sean Ellisテスト | 38% | 「非常に残念」と回答したユーザーへのインタビュー（10人） → コアバリュー特定 | 高 |
| NPS | 48 | Detractorへのインタビュー → 不満点（バグ or 機能不足）の特定と改善 | 高 |

### 次のステップ

**1-2ヶ月の改善期間**:
1. Sean Ellis「非常に残念」ユーザーへのインタビューで、価値を感じている機能を特定
2. その機能に絞って強化、不要な機能は削減
3. Detractorの不満点（例：バグ、UI/UX改善）に対応
4. 1-2ヶ月後に `/validate-pmf` を再実行

## よくある質問（FAQ）

### Q1: Sean Ellisテストは何ですか？
**A**: Sean Ellis氏が提唱した簡単なPMFテスト。40%以上のユーザーが「使えなくなったら非常に残念」と答えればPMF達成と判断する基準です。複雑な分析は不要で、単純な比率で判定でき、スタートアップ向きです。

### Q2: 「有効回答数」に「よくわかからない」を除外するのはなぜ？
**A**: 「よくわかからない」はプロダクトをまだ本気で使っていないユーザー。PMFテストはプロダクトへの真の執着度を測るため、ほぼ使っていないユーザーは判定から除外します。

### Q3: 月次成長率が5-9%でも満足すべき？
**A**: 初期段階では10%以上が目安ですが、業界・顧客セグメントで異なります。SaaS（法人向け）は成長が遅めで5-8%でも良好、アプリ（BtoC）は10%以上が標準。判定時に業界基準を確認し、解釈を柔軟に。

### Q4: Churn Rateの「30日非アクティブ」定義は必須？
**A**: いいえ。プロダクトタイプで定義を選択します。SaaS（有料）なら「契約解除」、Freemiumなら「30日非アクティブ」、アプリなら「アンインストール」など、重要なのは一貫性。定義を決めたら毎月同じ定義で測定してください。

### Q5: NPSが35でも「平均的」と判定されますが、本当に改善が必要？
**A**: はい。NPS 35は業界平均（30-40）レベルですが、PMF達成には最低50が必要。NPS 50は「顧客が積極的に口コミする」段階で、オーガニック成長につながります。50未満では改善が必須です。

### Q6: 再診断は何ヶ月後が目安？
**A**: 改善施策の規模による。小さな改善（既存機能強化）なら1ヶ月、大きな改善（新機能追加など）なら2ヶ月が目安。改善期間終了後すぐに再診断し、4指標を再測定してください。

### Q7: PMF達成後もこのスキルを使う？
**A**: はい。PMF達成後も四半期ごとに再診断し、指標が維持されているか確認します。スケール過程で顧客満足度が低下することもあるため、継続的なモニタリングが重要です。

---

## 注意事項

1. **最低サンプル数の遵守**: Sean Ellisテスト・NPSは最低40人、成長率・Churn Rateは3ヶ月データ必須。不足時は判定を遅延。
2. **PMF達成前のスケール禁止**: PMF未確認でのマーケティング投資は資金の無駄。必ず4指標達成後にスケール開始。
3. **早期判定の危険性**: データ不足での判定は誤判断につながる。データが揃うまで待つこと。
4. **継続的な再測定**: PMF達成後も月次 or 四半期で再測定し、維持を確認。
