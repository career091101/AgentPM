## 6. 有意差なし の場合の分析

### 原因分析

有意差が検出されなかった場合、以下を確認してください：

#### 1. サンプルサイズは十分か？

```
達成率 = 実績サンプル ÷ 目標サンプル
       = _____ ÷ _____
       = _____%
```

- [ ] **達成率 > 90%**: 十分 → 原因は別
- [ ] **達成率 70-90%**: やや不足 → サンプルサイズ追加か再テスト推奨
- [ ] **達成率 < 70%**: 不足 → **再テスト必須**（結果無効）

#### 2. バリアント設計は適切か？

**コントロール vs バリアント B の差分**:
- 変更点: ________________________________________________________________
- 差別化度合い: ★☆☆ ★★☆ ★★★ （選択）
- 評価: 実装が仮説通り反映されているか、デザイン品質に問題ないか

**評価結果**: ✅ **適切** / ⚠️ **一部不適切** / ❌ **不適切**

#### 3. ターゲット・期待効果は現実的か？

```
期待効果: CVR ___% → ___% (+___%)
実績差: ___% → ___% (+___%)

期待 vs 実績の乖離度: _____%
```

- [ ] **乖離度 < 10%**: 期待値は適切、別の要因あり
- [ ] **乖離度 10-30%**: 期待値が大きすぎた → 小規模改善で検証
- [ ] **乖離度 > 30%**: 根本的に期待値が不適切 → 仮説再検討

#### 4. テスト実装・実行に問題はないか？

- [ ] バリアント B が想定通り配置されている（実装確認）
- [ ] トラフィック分配がランダムである（ロジック確認）
- [ ] データ取得がGAに正確に記録されている（タグ確認）
- [ ] テスト期間中、バリアント B に変更加えていない

**実装品質判定**: ✅ **問題なし** / ⚠️ **軽微な問題** / ❌ **重大な問題**
