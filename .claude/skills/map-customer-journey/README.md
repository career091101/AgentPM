# カスタマージャーニーマップ - UX改善スキル

## 概要

**Map Customer Journey** は、ペルソナの6段階（認知→興味→検討→購入→利用→推薦）カスタマージャーニーをマッピングし、タッチポイント、感情曲線、ペインポイント、改善施策を体系的に設計する自律実行型スキルです。UX改善の優先順位を可視化し、Quick Wins（即座実行可能な施策）を自動特定します。

### このスキルを使うべき段階

- **CPF（顧客問題適合）達成後**、UX設計を始める時点
- **新規機能リリース前**の顧客体験設計時
- **カスタマーサクセス戦略**策定時
- **Phase2-3（Discovery-Executing）**でチャネル戦略を立案する際

### 所要時間

自動実行：30-40分（ペルソナ入力から改善施策提案まで）

### 出力成果物

- 顧客ジャーニーマップ（6段階フルマッピング）
- 感情曲線スコアリング（-5～+5スケール）
- ペインポイント・改善機会の自動検出
- 改善施策リスト（優先度付き）
- Quick Wins実装ロードマップ

|-----------|--------|
| **PSF達成後** | 顧客ニーズに基づくUX設計開始 |
| **新規プロダクトローンチ前** | 初期顧客体験の設計 |

| **チャネル戦略検討時** | 各段階での最適な接点設計 |
| **3ヶ月ごとの定期見直し** | ユーザー行動の変化に対応 |

## 6段階ジャーニー解説

| # | 段階 | 主な行動 | 主要タッチポイント | 典型的な感情 |
|:--:|------|--------|-------------------|-----------|
| **1** | 認知（Awareness） | プロダクト発見 | 広告/SNS/検索/口コミ | 中立～軽い興味 |
| **2** | 興味（Interest） | 詳細調査 | LP/ブログ/資料請求 | 期待～疑念 |
| **3** | 検討（Consideration） | 比較・評価 | デモ/比較表/レビュー | 迷い～納得 |
| **4** | 購入（Purchase） | 決断・契約 | 申込フォーム/決済 | 不安～満足 |
| **5** | 利用（Usage） | 実装・習熟 | オンボーディング/機能利用 | 戸惑い～満足 |
| **6** | 推薦（Advocacy） | 他者推薦 | 口コミ/SNS投稿/紹介 | 喜び～愛着 |

## スキル実行フロー

```
┌─────────────────────────────────────────┐
│ STEP 1: ペルソナ読込                    │
│ → persona.md から顧客特性を抽出         │
└──────────────┬──────────────────────────┘
               ↓
┌──────────────────────────────────────────┐
│ STEP 2: 6段階ジャーニー定義              │
│ → 各段階での行動・課題・感情を分析       │
└──────────────┬──────────────────────────┘
               ↓
┌──────────────────────────────────────────┐
│ STEP 3: タッチポイント特定              │
│ → 各段階で3-5個のタッチポイントを抽出   │
│   （Web/Email/Product/Support等）       │
└──────────────┬──────────────────────────┘
               ↓
┌──────────────────────────────────────────┐
│ STEP 4: 感情曲線スコアリング            │
│ → -5～+5スケールで各段階の感情を数値化   │
│ → ペインポイント（-1以下）を特定        │
└──────────────┬──────────────────────────┘
               ↓
┌──────────────────────────────────────────┐
│ STEP 5: 課題・強化ポイント分析          │
│ → 問題の内容・発生理由・影響度を分類    │
└──────────────┬──────────────────────────┘
               ↓
┌──────────────────────────────────────────┐
│ STEP 6: 改善施策提案                     │
│ → 各課題に対して3-5個の具体的施策を提案 │
│ → Impact/Effortスコアリング              │
└──────────────┬──────────────────────────┘
               ↓
┌──────────────────────────────────────────┐
│ STEP 7: Quick Wins特定                  │
│ → Impact ≥7 かつ Effort ≤3 の施策を抽出 │
│ → 優先度スコア（Impact/Effort）で順位   │
└──────────────┬──────────────────────────┘
               ↓
┌──────────────────────────────────────────┐
│ STEP 8: 実装ロードマップ作成            │
│ → Week 1-2 / 1ヶ月 / 3ヶ月の計画        │
│ → 成果物を Flow に出力                   │
└──────────────────────────────────────────┘
```

## 実際の使用例

### 例1：SaaS プロダクトの顧客ジャーニー

**入力**: ITエンジニア向けプロジェクト管理ツールのペルソナ

**出力（概要）**:
```
カスタマージャーニーマップ: 佐藤太郎（32歳、スタートアップCTO）

| 段階 | タッチポイント数 | 感情スコア | 主要課題 |
|------|----------|--------|--------|
| 認知 | 4個 | +2 | 情報の見つけやすさ |
| 興味 | 4個 | +3 | LP内容の充実度 |
| 検討 | 3個 | 0 | 価格透明性 🚨 |
| 購入 | 4個 | -2 | 決済フロー 🚨 |
| 利用 | 3個 | +1 | オンボーディング |
| 推薦 | 2個 | +4 | 紹介インセンティブ |

Quick Wins:
1. 価格ページの改善（Impact: 8, Effort: 2, Score: 4.0）
2. 決済フロー簡素化（Impact: 9, Effort: 3, Score: 3.0）
3. オンボーディング動画追加（Impact: 7, Effort: 2, Score: 3.5）
```

**実装ロードマップ**:
- Week 1-2: 決済フロー簡素化 + 価格ページ改善
- Month 1: オンボーディング動画制作
- Month 3: 紹介プログラム設計

### 例2：B2B エンタープライズプロダクト

**ペルソナ**: 企業のIT部門マネージャー

**発見**:
- 認知段階: LinkedInとメディア記事が主流（SNS広告が不要）
- 検討段階: セキュリティ・コンプライアンス情報が重要
- 購入段階: 複数利害関係者（CFO/CEO/セキュリティ担当）の承認プロセス
- 利用段階: 導入支援・トレーニング需要が高い

**優先施策**:
1. セキュリティ認証資料の充実（Gartner、SOC2等の掲載）
2. 複数利害関係者向けのテンプレート提供（ビジネス/技術/セキュリティ別）
3. 専任オンボーディング担当者の配置

## よくある質問（FAQ）

### Q1. 複数ペルソナがある場合、全員分のジャーニーマップを作成すべき？

**A.** はい、推奨します。理由：

1. **メインペルソナ**: 全6段階で詳細マッピング
2. **サブペルソナ**: 異なる部分（例：検討段階）に絞った軽量版
3. **比較分析**: 各ペルソナ間でのタッチポイント・感情の違いを可視化

異なるペルソナは異なるカスタマージャーニーを持つため、施策も分岐する場合があります。

---

### Q2. 感情スコアは主観的では？データで検証すべきか？

**A.** 初期版は「仮説」として主観的に設定し、後でユーザーインタビューで検証します。

**フロー**:
1. **ジャーニーマップ作成**（仮説）→ 感情スコア設定
2. **ユーザーインタビュー**（5-10人） → 実際の感情をヒアリング
3. **スコア調整** → インタビュー結果に基づき更新
4. **改善施策検証** → A/Bテストで効果測定

このスキルで生成したマップはあくまで「初期仮説」として使用してください。

---

### Q3. 改善施策の「Impact」「Effort」をどう判定する？

**A.** 以下の基準で定量化してください：

**Impact（影響度）**:
- 10: 売上直結（CVR大幅改善）
- 7-9: 複数段階に影響
- 4-6: 特定段階への影響
- 1-3: 限定的な影響

**Effort（実装難易度）**:
- 1-2: 即座実行可能（設定変更、テンプレート追加等）
- 3-4: 1-2週間で実装可能
- 5-7: 1-2ヶ月必要
- 8-10: 大規模プロジェクト必要

---

### Q4. Quick Wins実装後、効果の測定方法は？

**A.** 以下のKPIで測定してください：

| 施策 | 測定KPI |
|-----|--------|
| ページ改善 | CVR（コンバージョン率）変化 |
| メール施策 | 開封率、クリック率、返信率 |
| オンボーディング | Aha Moment達成率、初週継続率 |
| 紹介施策 | 紹介率、バイラル係数（K-factor） |

実装から2-4週間後に効果を測定し、期待値との乖離を分析してください。

---

### Q5. ジャーニーマップは何ヶ月ごとに更新すべき？

**A.** **3ヶ月ごと**の定期見直し推奨：

**更新トリガー**:
- 3ヶ月の定期更新（通常）
- 新機能リリース直後（2週間後）
- ユーザー行動の大きな変化検知時（随時）
- Quick Wins施策の効果測定後（1ヶ月後）

プロダクト進化に合わせて、ジャーニーマップも常に最新に保つことで、UX改善の方向性がぶれません。

3. `map-customer-journey` を再実行し、ペルソナファイルを指定
