## フックテンプレート7種

### パターン1: 数字 × 断定型

**構造**:
```
[企業名・人名]は「[数値1]の[事象]が[数値2]の速さで進む」と断言した。

[自己解釈]だ。
```

**例**:
```
「OpenAI への 1400 億円の投稿は 18 兆円の価値になった」とマイクロソフトCEO のナデラ氏は明かした。

わずか 6 年で 135 倍という驚異的なリターンだ。
```

**使用シーン**: 数値データが豊富な入力に最適

---

### パターン2: 衝撃 × 企業名

**構造**:
```
「[衝撃的発言]」。

[企業名]の[役職][氏名]の[断言/警告/予測]が業界に衝撃を与えている。
```

**例**:
```
「AI を一度導入すれば、もう二度と人を雇う必要はない」。

インドの新興 AI スタートアップ、ライムチャット共同創業者のニキル・グプタ氏（28歳）の断言が業界に衝撃を与えている。
```

**使用シーン**: 著名人の発言がある入力に最適

---

### パターン3: 比較 × 驚異的数字

**構造**:
```
[企業名]の[指標]は[数値1]から[数値2]へと[倍率]倍に拡大した。

これは[比較対象]を上回る[評価]だ。
```

**例**:
```
ピーター・ティール氏は保有するエヌビディア株を全売却し、ポートフォリオを2 億 1200 万ドルから 7440 万ドルへと 3 分の 1 に縮小させた。

衝撃的な判断だ。
```

**使用シーン**: 数値比較データがある入力に最適

---

### パターン4: 問題提起 × 具体例

**構造**:
```
[問題提起]。

[企業名]では既に[具体例]が現実になっている。
```

**例**:
```
年間売上 1 兆円規模の欧州小売大手が、AI エージェント導入に失敗し、イスラエル企業のエンジニア支援を受けて初めて安定稼働を実現した。
```

**使用シーン**: 業界変化・トレンドに関する入力に最適

---

### パターン5: 逆張り × 権威引用

**構造**:
```
[一般的見解]と言われるが、[権威者]は真逆の見方を示す。

「[逆張り発言]」。
```

**例**:
```
「AI バブルと騒ぐ人たちは、重要な事実を見落としている」と米投資アナリストのダン・ベントン氏は指摘する。
```

**使用シーン**: 権威者の逆張り発言がある入力に最適

---

### パターン6: 高野式「〜の件」型

**構造**:
```
[企業名・話題]が[数値]の[アクション]、[断定的評価]の件。

[詳細1行目]。

これ、[解釈]。
```

**例**:
```
SoftBank が 3.5 兆円ぶち込んだ OpenAI 投稿、マジでヤバい件。

孫正義、完全に本気モード入ってる。

これ、日本企業史上見たことない規模だ。
```

**使用シーン**: カジュアルなトーンが適合する入力に最適

---

### パターン7: リスト型・衝撃ファクト連打

**構造**:
```
[年号]、[常識]が完全に壊れる衝撃の事実

[見出し1]: [データ]
[詳細]
→ [解釈・結論]

[見出し2]: [データ]
[詳細]
→ [解釈・結論]
```

**例**:
```
2026 年、採用の常識が完全に壊れる衝撃の事実

ES で差がつかない時代が来た: 就活生の 90%が AI、ChatGPT など使ってエントリーシート書いてる
→ もう誰も紙の文章なんか信じてない
```

**使用シーン**: 複数の事実・統計がある入力に最適

---

## パターン選択の基本原則

### 入力内容の分析基準

1. **数値の有無**: 具体的な数値・倍率・金額が含まれているか
2. **著名人の有無**: 企業名・人名・役職が含まれているか
3. **トレンドの有無**: 業界変化・市場動向が含まれているか
4. **統計の量**: 複数のデータポイントが含まれているか
5. **トーン**: フォーマル/カジュアルのどちらが適合するか

### 3案への割り当て戦略

- **案1**: 入力特徴に最も適合するパターン（推奨度★★★★★）
- **案2**: 異なる切り口でアプローチするパターン（推奨度★★★★☆）
- **案3**: 補完的な第3のパターン（推奨度★★★☆☆）

**重要**: 3案は必ず異なる3パターンから選択（重複禁止）
