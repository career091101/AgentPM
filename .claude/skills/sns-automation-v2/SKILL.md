## この Skill でできること（v2版）

1. **Phase 1: データ収集**（5 並列最適化版、12-22 分）

   - X タイムライン収集（200 件）
   - Top 10 ツイート抽出
   - ツイート詳細取得（リンク・リプライ、5 並列処理）

2. **Phase 2: 分析・調査**（ハイブリッド実行、20-30 分）

   - STEP 2.1: コンテンツ抽出（記事・YouTube・PDF、LLM 判定ワンパス化）
   - STEP 2.2 + 2.3: リプライ分析 + Web 調査（並列実行）

3. **Phase 3: 投稿生成**（順次実行、20-30 分）

   - 高野式 7 パターンから AI が最適 3 パターン自動選択
   - LinkedIn 投稿 3 案生成（各案独立）
   - **X 投稿生成（全新規化）**:
     - Top 1 スレッド（5-7 ツイート深掘り型）
     - Top 2 スレッド（5-7 ツイート深掘り型）
     - Top 3 スレッド（5-7 ツイート深掘り型）
   - **Threads 投稿生成（全新規化＋文字数制御）**:
     - Top 1 投稿（LinkedIn ≤500文字→単一、>500文字→2-3スレッド）
     - Top 2 投稿（LinkedIn ≤500文字→単一、>500文字→2-3スレッド）
     - Top 3 投稿（LinkedIn ≤500文字→単一、>500文字→2-3スレッド）

4. **Phase 4: マルチプラットフォーム予約投稿**（Late API、10-15 分）

   - **LinkedIn 3日分散投稿**（Late API 空き日自動検出）:
     - Top 1 → 1日目 8:00 JST
     - Top 2 → 2日目 8:00 JST
     - Top 3 → 3日目 8:00 JST
   - **X/Threads 同時刻投稿**（各Top記事を同時刻に配信）:
     - Top 1: X 7:30 + Threads 7:30
     - Top 2: X 12:00 + Threads 12:00
     - Top 3: X 20:00 + Threads 20:00
   - 失敗投稿のフォールバック（Markdown 生成）

5. **最終サマリー生成**

   - 各フェーズの実行時間
   - 成果物一覧
   - 投稿結果（URL、ステータス、予約日時）

## v2の主要変更点

| 項目 | v1（従来版） | v2（新仕様） | 理由 |
|------|-------------|-------------|------|
| **LinkedIn投稿** | 3案全て同日8:00 | 3日間に分散（空き日検出） | 投稿密度の分散、Late API衝突回避 |
| **X投稿タイプ** | 派生1+スレッド2（Top 1-3） | 全て新規スレッド（Top 1-3） | 各投稿の独立性、スタンドアロン化 |
| **X投稿構成** | 派生（280文字）+スレッド（5-7） | 全て5-7ツイートスレッド | 深掘り型統一、品質安定化 |
| **Threads投稿タイプ** | 派生1+新規1（Top 1-2） | 全て新規（Top 1-3） | 各投稿の独立性、スタンドアロン化 |
| **Threads投稿構成** | 固定フォーマット | 文字数制御（≤500単一、>500分割） | LinkedIn投稿長に応じた最適化 |
| **投稿時間帯** | X 7:30/12:00/20:00、Threads 7:30/20:00 | X/Threads共通 7:30/12:00/20:00 | 同一Top記事の同時刻配信 |
| **総投稿数** | 6投稿/日（LinkedIn 1 + X 3 + Threads 2） | 7投稿（LinkedIn 1×3日 + X 3 + Threads 3） | Threads投稿数増加 |

## Instructions

**実行モード**: 完全自律実行（人間介入なし）
**推定所要時間**: 62-97分（Phase 3 +15分、Phase 4 +5分）
**日次投稿数**: 7投稿（LinkedIn 1 + X 3 + Threads 3）
**LinkedIn分散**: 3日間（Late API自動検出）

---

### Phase 1: データ収集（5 並列最適化版、12-22 分）

**概要**: X タイムラインから 200 件を収集し、AI 関連度と高野式適合度で Top 10 を抽出。詳細リンク・リプライを 5 並列処理で取得。

**主要ステップ**:

- STEP 1.1: X タイムライン収集（5-10 分）
- STEP 1.2: Top 10 抽出・並列評価（5-8 分）
- STEP 1.3: ツイート詳細取得・5 並列処理（2-4 分）

**詳細手順**: [@phases/phase1_data_collection.md](./phases/phase1_data_collection.md)

**エラーハンドリング**: 即時停止
（[@\_shared/error_handling_patterns.md](../_shared/error_handling_patterns.md)参照）

---

### Phase 2: 分析・調査（ハイブリッド実行、20-30 分）

**概要**: Phase 1 データから記事・YouTube・PDF コンテンツを LLM 判定でワンパス抽出し、リプライ分析と Web 調査を並列実行。

**主要ステップ**:

- STEP 2.1: コンテンツ抽出 + LLM 判定ワンパス化（7-14 分、従来比 ▲3-6 分短縮）
- STEP 2.2 + 2.3: リプライ分析 + Web 調査（15-20 分、並列実行で ▲10-15 分短縮）

**詳細手順**: [@phases/phase2_detailed.md](./phases/phase2_detailed.md)

**エラーハンドリング**: STEP 2.1, 2.3 成功必須
（[@\_shared/error_handling_patterns.md](../_shared/error_handling_patterns.md)参照）

**最適化内容**:

- LLM 判定ワンパス化: extract-content スキルがフィルタリング統合、filter-extracted-content 廃止
- 2 並列化: STEP 2.2 と STEP 2.3 を独立処理として並列実行

---

### Phase 3: 投稿生成（順次実行、20-30 分）

**概要**: 高野式 7 パターンから AI が最適 3 パターン選択。LinkedIn 3 案＋ X/Threads 全新規投稿を生成。

**主要ステップ**:

- STEP 3.1: 高野式 7 パターン投稿 3 案生成（LinkedIn、10-15 分）
- STEP 3.2: X 全新規スレッド生成（10-15 分、v2変更点）
  - Top 1 スレッド: 5-7 ツイート深掘り型（完全独立）
  - Top 2 スレッド: 5-7 ツイート深掘り型（完全独立）
  - Top 3 スレッド: 5-7 ツイート深掘り型（完全独立）
- STEP 3.3: Threads 全新規投稿生成（5-10 分、v2変更点＋文字数制御）
  - Top 1 投稿: LinkedIn似スタイル（≤500文字→単一、>500文字→2-3分割）
  - Top 2 投稿: LinkedIn似スタイル（≤500文字→単一、>500文字→2-3分割）
  - Top 3 投稿: LinkedIn似スタイル（≤500文字→単一、>500文字→2-3分割）

**詳細手順**:
[@phases/phase3_content_generation_v2.md](./phases/phase3_content_generation_v2.md)、[@generate-sns-posts-takano/SKILL.md](../sns-automation/generate-sns-posts-takano/SKILL.md)

**エラーハンドリング**: チェックリスト未達時は自動再生成
（[@\_shared/error_handling_patterns.md](../_shared/error_handling_patterns.md)参照）

**v2変更点**:

- **派生投稿廃止**: X/Threads は全て独立した新規投稿
- **X深掘り型統一**: Top 1-3 全てが 5-7 ツイートスレッド
- **Threads文字数制御**: LinkedIn投稿長に応じて単一/スレッド自動選択
- **完全独立性**: 各投稿は他投稿を参照しない

---

### Phase 4: マルチプラットフォーム予約投稿（Late API、競合回避型、10-15 分）

**概要**: Late API から既存予約を検出し、LinkedIn は 3 日間に分散、X/Threads は同時刻投稿。

**主要ステップ**:

- STEP 4.0: 既存予約の競合検出（全プラットフォーム、1 分）
- STEP 4.1: LinkedIn 空き日検索（7日間スキャン、1 分）
- STEP 4.2: コンテンツ抽出（7 投稿分、2-3 分）
- STEP 4.3: Late API 予約投稿（7 投稿個別 POST、5-10 分）

**投稿スケジュール（v2仕様）**:

| 時刻  | プラットフォーム | 投稿タイプ              | 対象記事 | 分散 |
| ----- | ---------------- | ----------------------- | -------- | ---- |
| 7:30  | X                | スレッド（Top 1）       | Top 1    | 当日 |
| 7:30  | Threads          | 新規（Top 1）           | Top 1    | 当日 |
| 8:00  | LinkedIn         | 高野式（Top 1）         | Top 1    | 1日目 |
| 12:00 | X                | スレッド（Top 2）       | Top 2    | 当日 |
| 12:00 | Threads          | 新規（Top 2）           | Top 2    | 当日 |
| 8:00  | LinkedIn         | 高野式（Top 2）         | Top 2    | 2日目 |
| 20:00 | X                | スレッド（Top 3）       | Top 3    | 当日 |
| 20:00 | Threads          | 新規（Top 3）           | Top 3    | 当日 |
| 8:00  | LinkedIn         | 高野式（Top 3）         | Top 3    | 3日目 |

**投稿スクリプト**: `scripts/late_api_multi_post_v2.py`

**スキル参照**: [@post-multiplatform/SKILL.md](../post-multiplatform/SKILL.md)

**詳細手順**: [@phases/phase4_api_integration_v2.md](./phases/phase4_api_integration_v2.md)

**エラーハンドリング**: Late API 失敗投稿は Markdown ファイル生成
（[@\_shared/error_handling_patterns.md](../_shared/error_handling_patterns.md)参照）

**v2変更点**:

- **LinkedIn 3日分散**: Late APIで空き日自動検出、Top 1→1日目、Top 2→2日目、Top 3→3日目
- **X/Threads同時刻投稿**: 各Top記事を同じ時間帯に配信（7:30, 12:00, 20:00）
- **投稿数増加**: 6投稿/日 → 7投稿（LinkedIn 1×3日 + X 3 + Threads 3）

## エラーハンドリング

### 基本方針

| フェーズ    | エラー時の対応                   | 理由                           |
| ----------- | -------------------------------- | ------------------------------ |
| **Phase 1** | **即時停止**                     | 後続処理が全て依存するため必須 |
| **Phase 2** | **即時停止（全タスク成功必須）** | 品質重視                       |
| **Phase 3** | **即時停止**                     | 投稿生成は最重要               |
