# build-financial-model

## 概要

VCピッチ用の3年財務予測モデルを自動生成するスキル。

Pre-Seed〜Series Aの起業家向けに、VC評価基準に準拠した財務モデルを提供します。ユニットエコノミクス分析、3年収益予測、資金計画、感度分析を含みます。

## 使用タイミング

- VCピッチ資料作成時
- 資金調達計画策定時
- ビジネスモデル検証時
- PMF達成後のスケール計画立案時

## 入力要件

### 必須情報

1. **事業概要**
   - ビジネスモデル（SaaS, Marketplace, Transaction等）
   - 収益構造（サブスク, 手数料, 広告等）
   - 主要プロダクト/サービス

2. **価格設定**
   - 単価（月額/年額）
   - プラン構成（Free/Basic/Pro/Enterprise）
   - アップセル/クロスセル構造

3. **現在のトラクション**（あれば）
   - 顧客数
   - MRR/ARR
   - 成長率（MoM, YoY）
   - チャーン率

### 任意情報

- 市場規模推定結果（`estimate-market-size`出力）
- 競合の価格帯・収益構造
- 調達予定額・タイミング

## 出力構成

### 1. Unit Economics分析

VCが最重視する指標を算出：

#### CAC（Customer Acquisition Cost）

```
CAC = (マーケティング費用 + セールス費用) / 新規顧客獲得数
```

**チャネル別CAC**:
- Organic（SEO/SNS）: $X
- Paid（広告）: $Y
- Sales（営業）: $Z
- ブレンドCAC: $A

#### LTV（Lifetime Value）

```
LTV = ARPU × Gross Margin × (1 / Churn Rate)
```

**コホート別LTV**:
- Month 0-6: $X
- Month 7-12: $Y
- Month 13+: $Z

#### LTV/CAC比率

```
LTV/CAC比率 = LTV / CAC
```

**VC評価基準**:
- ✅ ≥3.0: 優良
- ⚠️ 2.0-3.0: 改善余地あり
- ❌ <2.0: 要改善

#### Payback Period

```
Payback Period = CAC / (ARPU × Gross Margin)
```

**VC評価基準**:
- ✅ ≤12ヶ月: 優良
- ⚠️ 12-18ヶ月: 許容範囲
- ❌ >18ヶ月: 要改善

## 実装上の注意

### 1. データソース

- **トラクションデータ**: ユーザー入力
- **市場データ**: `estimate-market-size`スキル出力
- **ベンチマーク**: Tier 2ナレッジベース
- **VC基準**: `domain_config.yaml`

### 2. 計算精度

- **成長率**: 現実的な範囲（SaaS平均: YoY 100-300%）
- **チャーン率**: 段階的改善（Year 1: 5% → Year 3: 2%）
- **CAC**: チャネル最適化による改善想定
- **Gross Margin**: 業界標準（SaaS 70-80%）

### 3. VCコミュニケーション

- **保守的予測**: 楽観ケースより悲観ケースを重視
- **根拠明示**: 全ての数値に計算式と前提条件
- **比較可能性**: 業界ベンチマークとの対比
- **ストーリー**: 数値だけでなく、なぜその成長が実現可能かを説明

### 4. アップデート

- **四半期ごと**: 実績値との乖離チェック
- **ピボット時**: モデル全体の見直し
- **調達前**: 最新データで再計算
