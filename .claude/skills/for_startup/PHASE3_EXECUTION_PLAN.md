## Phase 3.1: 品質保証（QA）フェーズ 詳細

### タスク1: 全19スキルの参照パス妥当性確認

**目的**: Domain-Specific Knowledgeセクションの参照パスがすべて実在するファイルを指しているか確認

**実行方法**:
1. 全19スキルのSKILL.mdを読み込み
2. Domain-Specific Knowledge > Reference セクションから参照パスを抽出
3. 各パスの実在確認（`@`記法のパスを実際のファイルパスに変換して確認）
4. リンク切れがあれば修正または削除

**成果物**: `PHASE3_QA_REFERENCE_PATH_VALIDATION.md`

**想定される問題**:
- 相対パス記法（`@Founder_Research/...`）の解決
- ファイル名の変更・移動によるリンク切れ
- タイポによる参照エラー

**品質基準**: 参照パス妥当性100%達成

---

### タスク2: 定量的評価基準の整合性確認

**目的**: スキル間で定量的評価基準が整合しているか確認

**実行方法**:
1. 全19スキルのQuantitative Benchmarksセクションを抽出
2. スキル間で共通する評価基準（CPFスコア、PMF Sean Ellis等）を特定
3. 基準値の整合性を確認（例: CPFスコア70%がすべてのスキルで統一されているか）
4. 不整合があれば修正

**成果物**: `PHASE3_QA_QUANTITATIVE_BENCHMARKS_CONSISTENCY.md`

**確認対象の評価基準**:
- CPFスコア基準（70%以上）
- PMF Sean Ellisテスト基準（50%以上）
- 月次成長率基準（20%以上）
- Scorecard総合基準（40点以上/50点）
- Pitch Deck投資家説得力基準（110点以上/130点）
- タイミングスコア基準（35点以上/50点）
- レビュー品質スコア基準（70点以上/100点）

**品質基準**: 基準値の整合性100%達成

---

### タスク3: タイポ・誤字脱字の確認

**目的**: SKILL.mdおよび統合レポートのタイポ・誤字脱字を修正

**実行方法**:
1. 全19スキルのSKILL.mdを読み込み
2. Domain-Specific Knowledgeセクションを中心にタイポ・誤字脱字をチェック
3. 統合レポート（tier2_integration_report.md）も同様にチェック
4. 発見したエラーを修正

**成果物**: `PHASE3_QA_TYPO_CORRECTION_REPORT.md`

**チェック対象**:
- 企業名・製品名の表記揺れ（例: "Airbnb" vs "AirBnB"）
- 数値の誤記（例: 85% vs 58%の取り違え）
- 単位の不統一（例: "$1B" vs "10億ドル"）
- 日本語の誤字脱字
- 英語のスペルミス

**品質基準**: 主要なタイポ・誤字脱字の修正率90%以上

---

### タスク4: QAレポート作成

**目的**: Phase 3.1の品質保証結果を総括

**実行方法**:
1. タスク1-3の結果を統合
2. 発見した問題の総数と修正率を記録
3. 未修正の問題と今後の対応方針を明記
4. Phase 3.2への移行条件を確認

**成果物**: `PHASE3_QA_FINAL_REPORT.md`

**品質基準**:
- 参照パス妥当性100%達成
- 評価基準整合性100%達成
- タイポ修正率90%以上
- 総合品質スコア95点以上/100点
