# Monitor Burn Rate - ForStartup Edition

## 目的

スタートアップの資金状況を常時監視し、**バーンレート（月次消費金額）**と**ランウェイ（資金が尽きるまでの期間）**を自動計算。**24ヶ月ルール**（VC投資推奨基準）に基づき、資金ショートを未然に防ぎ、最適な資金調達タイミングを提案します。

**重要な原則**（起業の科学 + VC投資基準）:
> 「VC投資を受けるスタートアップは、ランウェイが24ヶ月を切ったら、資金調達の準備を開始すべきである」
> 「ランウェイが12ヶ月以下になったら、事業継続が危機的状況にある」
> 「資金調達には平均4-6ヶ月かかるため、余裕を持った計画が必須」
> — David Skok, a16z, Y Combinator

**ForStartup版の追加機能**:
1. **ユニットエコノミクス統合分析**: LTV/CAC比率とバーンレートの関連性評価
2. **資金調達ロードマップ自動生成**: Pre-Seed → Seed → Series Aのマイルストーン連携
3. **VC投資基準準拠チェック**: Founder_Researchから抽出した成功事例との比較
4. **月次成長率との連携**: 成長率20%/月達成時の資金効率評価

### Common Pitfalls (失敗パターン)

1. **ランウェイ6ヶ月以下での資金調達開始** → 交渉力低下、不利な条件受諾
2. **バーンレート増加とトラクション停滞の同時発生** → VCから投資見送り
3. **ユニットエコノミクス未確立のまま調達** → 次ラウンドで資金枯渇
4. **調達所要期間の過小評価** → 実際は4-6ヶ月（想定2-3ヶ月）

### Best Practices

1. **毎月バーンレート・ランウェイを計測**: 経営判断の最重要指標
2. **ランウェイ24ヶ月を切ったら資金調達準備開始**: ピッチデッキ作成、投資家リスト
3. **ユニットエコノミクス確立を優先**: LTV/CAC 5.0以上を達成してから調達
4. **月次成長率20%以上を実証**: VCへの説得力が劇的に向上

## 実行ステップ

### STEP 1: 現在残高の入力（1分）

**質問**:
1. 現在の手元現金残高（銀行口座残高）
2. 最終更新日
3. **[ForStartup追加]** 前回調達日・調達額（初回の場合は自己資金額）

**入力例**:
```
現在残高: ¥30,000,000
更新日: 2026-01-02
前回調達: 2025-07-01、Seed ¥50,000,000
```

---

### STEP 2: Gross Burn Rate計算（2分）

**計算方法**: 月次総支出の算出

**必要データ**:
- 人件費（給与、社会保険）
- オフィス・設備費
- マーケティング費
- 開発費（外注、ツール費）
- その他経費

**計算式**:
```
Gross Burn Rate = 月次総支出 / 月
```

**計算例**:
```
人件費: ¥2,000,000/月
オフィス費: ¥300,000/月
マーケティング費: ¥500,000/月
開発費: ¥800,000/月
その他: ¥100,000/月

Gross Burn Rate = ¥3,700,000/月
```

**業種別目安**（ForStartup基準）:
| ステージ | 目安バーンレート | チーム規模 | 資金調達レンジ |
|---------|----------------|----------|--------------|
| Pre-Seed（MVP開発） | ¥500,000-1,500,000/月 | 2-3人 | ¥30-50M |
| Seed（初期トラクション） | ¥2,000,000-5,000,000/月 | 5-10人 | ¥50-200M |
| Series A（スケール） | ¥10,000,000-30,000,000/月 | 20-50人 | ¥200M-1B |
| Series B（グロース） | ¥30,000,000-100,000,000/月 | 50-150人 | ¥1-5B |

---

### STEP 3: Net Burn Rate計算（2分）

**計算方法**: 収益を差し引いた実質消費額

**必要データ**:
- 月次収益（売上、契約金）
- 月次総支出（STEP 2から）

**計算式**:
```
Net Burn Rate = (月次総支出 - 月次収益) / 月
```

**計算例**:
```
月次収益: ¥800,000
月次総支出: ¥3,700,000

Net Burn Rate = ¥2,900,000/月
```

**注意**:
- Net Burn Rateがマイナス（収益 > 支出）の場合 → キャッシュフロー黒字 ✅
- Net Burn Rateがプラスの場合 → 赤字運営、資金枯渇リスクあり ⚠️

---

### STEP 4: ランウェイ計算（1分）

**計算方法**: 現在の消費ペースで資金が何ヶ月持つか

**計算式**:
```
Runway（月） = 現在残高 / Net Burn Rate
```

**計算例**:
```
現在残高: ¥30,000,000
Net Burn Rate: ¥2,900,000/月

Runway = 30,000,000 / 2,900,000 = 10.3ヶ月 ⚠️
```

**判定基準**（ForStartup版: 24ヶ月ルール適用）:

| ランウェイ | 判定 | 状態 | 推奨アクション |
|-----------|------|------|--------------|
| **24ヶ月以上** | ✅ 安全 | 余裕あり | 成長投資、月次監視継続 |
| **18-24ヶ月** | ⚠️ 注意 | 資金調達検討開始 | 投資家リスト作成、財務モデル更新 |
| **12-18ヶ月** | 🚨 警告 | 資金調達準備開始 | ピッチデッキ作成、ウォームイントロ |
| **6-12ヶ月** | 🔴 緊急 | 資金調達必須 | 即座に調達活動開始、コスト削減 |
| **6ヶ月以下** | 🔴🔴 危機的 | 倒産リスク | 緊急ブリッジ調達、ドラスティックなコスト削減 |

**VC投資判断への影響**:
- ランウェイ < 12ヶ月 → 交渉力低下、Valuation下方圧力
- ランウェイ < 6ヶ月 → 投資見送りリスク大（「資金繰りに困っている」と判断）

### STEP 6: シナリオ分析（3シナリオ）（3分）

**目的**: 楽観・現実・悲観の3パターンでランウェイを試算

#### シナリオ1: 楽観シナリオ（収益+30%、支出-10%）

```
月次収益: ¥800,000 × 1.30 = ¥1,040,000
月次支出: ¥3,700,000 × 0.90 = ¥3,330,000
Net Burn Rate: ¥2,290,000/月
Runway: 30,000,000 / 2,290,000 = 13.1ヶ月 ⚠️
```

#### シナリオ2: 現実シナリオ（現状維持）

```
Net Burn Rate: ¥2,900,000/月
Runway: 30,000,000 / 2,900,000 = 10.3ヶ月 🚨
```

#### シナリオ3: 悲観シナリオ（収益-20%、支出+10%）

```
月次収益: ¥800,000 × 0.80 = ¥640,000
月次支出: ¥3,700,000 × 1.10 = ¥4,070,000
Net Burn Rate: ¥3,430,000/月
Runway: 30,000,000 / 3,430,000 = 8.7ヶ月 🔴
```

**シナリオ比較表**:

| シナリオ | Net Burn Rate | Runway | 判定 |
|---------|--------------|--------|------|
| 楽観 | ¥2,290,000 | 13.1ヶ月 | ⚠️ |
| 現実 | ¥2,900,000 | 10.3ヶ月 | 🚨 |
| 悲観 | ¥3,430,000 | 8.7ヶ月 | 🔴 |

**インサイト**:
- いずれのシナリオでも12ヶ月以下 → **即座に資金調達活動を開始すべき**
- 悲観シナリオでは9ヶ月で資金枯渇 → **コスト削減も並行実施**

### STEP 7: 資金調達タイミング・調達額提案（4分）

**判定ロジック**:
- ランウェイ < 24ヶ月 → 資金調達検討開始
- ランウェイ < 18ヶ月 → 資金調達準備開始
- ランウェイ < 12ヶ月 → 資金調達必須
- ランウェイ < 6ヶ月 → 緊急ブリッジ調達

#### 現状判定（例）

```
現在Runway: 10.3ヶ月 🚨 緊急
→ 即座に資金調達活動を開始すべき
→ 調達所要期間4-6ヶ月を考慮すると、資金枯渇リスク高い
```

#### 目標調達額の自動計算

**計算式**（24ヶ月ルール適用）:
```
目標調達額 = Net Burn Rate × 24ヶ月
```

**計算例**:
```
Net Burn Rate: ¥2,900,000/月
目標調達額 = ¥2,900,000 × 24 = ¥69,600,000
```

**調達方法の提案**（Founder_Research統合）:

| 調達額 | 推奨方法 | 適した投資家 | 期待評価額 | 調達期間 |
|-------|---------|------------|----------|---------|
| ~¥50,000,000 | Pre-Seed/Seed | エンジェル、小規模VC | ¥100-300M | 2-3ヶ月 |
| ¥50,000,000~¥200,000,000 | Seed | シードステージVC、YC | ¥300M-1B | 3-4ヶ月 |
| ¥200,000,000~¥1,000,000,000 | Series A | アーリーステージVC | ¥1-5B | 4-6ヶ月 |
| ¥1,000,000,000~ | Series B以降 | グロースファンド、CVC | ¥5-20B | 6-9ヶ月 |

**本ケースの提案**:
```
目標調達額: ¥69,600,000

```

### STEP 8: ユニットエコノミクス統合分析（3分）

**目的**: バーンレートと顧客獲得効率の関連性を評価

#### LTV/CAC 比率の計算

**必要データ**:
- LTV（顧客生涯価値）: 平均顧客利益 × 平均契約期間（月）
- CAC（顧客獲得コスト）: マーケティング費用 ÷ 新規顧客数

**計算例**:
```
月次新規顧客数: 20人
マーケティング費: ¥500,000/月
CAC = ¥500,000 / 20 = ¥25,000/顧客

平均顧客単価: ¥5,000/月
平均契約期間: 24ヶ月
Gross Margin: 70%
LTV = ¥5,000 × 24 × 0.70 = ¥84,000

LTV/CAC = ¥84,000 / ¥25,000 = 3.36
```

**VC投資基準判定**:

| LTV/CAC比率 | 判定 | VC評価 |
|------------|------|--------|
| **5.0以上** | ✅ 優秀 | 投資推奨レベル |
| **3.0-5.0** | ⚠️ 標準 | 改善余地あり |
| **2.0-3.0** | 🚨 要改善 | 投資見送りリスク |
| **2.0未満** | 🔴 危険 | ビジネスモデル再設計必須 |

**本ケース判定**: LTV/CAC 3.36 → ⚠️ 標準（VC基準5.0以上を目指すべき）

#### CAC回収期間

**計算式**:
```
CAC回収期間（月） = CAC / (月次顧客単価 × Gross Margin)
```

**計算例**:
```
CAC回収期間 = ¥25,000 / (¥5,000 × 0.70) = 7.1ヶ月 ✅
```

**VC投資基準判定**:
- 12ヶ月以内 ✅ → 合格
- 12-18ヶ月 ⚠️ → 改善必要
- 18ヶ月以上 🔴 → ビジネスモデル再検討

#### 成長率とバーンレートの関係

**月次成長率**: 新規顧客数の前月比
```
今月新規顧客: 20人
前月新規顧客: 18人
月次成長率 = (20 - 18) / 18 × 100 = 11.1%
```

**VC投資判断への影響**:
- 月次成長率 20%以上 ✅ → バーンレート増加も許容される
- 月次成長率 10-20% ⚠️ → バーンレート管理が重要
- 月次成長率 10%未満 🔴 → バーンレート削減必須

**本ケース**: 月次成長率11.1% → ⚠️ 20%目標に未達、バーンレート削減検討

## 注意事項

### ForStartup版の厳格化ポイント

1. **ランウェイ基準の厳格化**: 18ヶ月 → **24ヶ月以上**
   - VC調達には平均4-6ヶ月かかるため、余裕を持った計画が必須
   - 交渉力維持のため、資金枯渇リスクを最小化

2. **ユニットエコノミクス基準の厳格化**: LTV/CAC 3.0 → **5.0以上**
   - VC投資基準に準拠
   - スケーラビリティの証明が必須

3. **月次成長率基準の明確化**: 10-15% → **20%以上**
   - YC基準（月次成長率5-7%/週 ≒ 20-30%/月）に準拠
   - トラクション実証がVC投資の前提

4. **資金調達ロードマップの統合**:
   - Pre-Seed → Seed → Series Aの各マイルストーンを明示
   - Founder_Researchから抽出した成功パターンを参照

### 効率化の原則

- **月次監視の徹底**: バーンレート・ランウェイを毎月計測
- **早期警告システム**: ランウェイ24ヶ月を切ったらアラート
- **ユニットエコノミクス優先**: LTV/CAC 5.0以上達成を優先
- **成長率とのバランス**: 月次成長率20%以上維持がバーンレート増加の前提

### データベース管理の徹底

- **月次レポート作成**: 毎月バーンレートレポートを作成
- **VCミーティング前の確認**: 財務状況を常に最新化
- **トレンド分析**: 3ヶ月移動平均でバーンレート傾向を把握

### 完璧主義の回避

- **完璧なコスト削減を求めない**: 即効性のある施策から実行
- **全ての支出を削減しない**: 成長投資は維持（ただし、ROI > 3.0が前提）
- **調達額は柔軟に**: 24ヶ月分が原則だが、成長率次第で18-30ヶ月分も選択肢
