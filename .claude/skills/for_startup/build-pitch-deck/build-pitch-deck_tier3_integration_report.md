#### ピッチデッキの特徴

##### 1. Problem スライド: 業界共通課題の明確化

**課題定義**:
- 「AIにはデータ、計算能力、アルゴリズムの3要素が必要だが、データ（特に高品質なラベル付きデータ）が最大のボトルネック」

**3Uスコア**:
- **Unworkable**: 既存ソリューション（Amazon MTurk）は3D非対応、品質60-70%で実用不可
- **Unavoidable**: 教師あり学習の本質として、ラベル付きデータは100%不可避
- **Urgent**: 自動運転ブームで「今すぐ」必要（緊急性9/10）

**定量データ**:
- TAM: データラベリング市場$0.5B（2016年）→ $4.9B（2024年、CAGR 35%+）

**投資家説得力**: 10/10点（最高レベル）

|---|------|---------|:----:|
| コスト | $50/h | $5/h | 10x |

| スケーラビリティ | 10-20人 | 240K人 | 50x |
| 品質 | 70-80% | 95%+ | 3x |
| 対応データ型 | 画像のみ | 全型対応 | 6x |
| 導入障壁 | 専門チーム必要 | API 1本 | 15x |

**技術的差別化**:
- AI自動ラベリング（90%）+ 人間検証（10%）のハイブリッド
- Wizard of Oz型MVP: 外向きは完全自動化API、内部は手作業（初期）

#### ピッチデッキの特徴

**課題定義**:
- 「防衛産業は数十年間停滞している。従来の調達プロセス（5-10年）では、AI・自律システムの迅速な導入が不可能」

- **Unworkable**: 従来の防衛企業はレガシーシステムに縛られ、商業AI技術の迅速な導入が物理的に不可能
- **Unavoidable**: 国家安全保障に直結、中国等との技術競争という背景から回避不可能
- **Urgent**: 中国の軍事AI開発、ドローン脅威の急増（緊急性9/10）

**定量データ**:
- 米国防総省全体が認識する課題（SOCOM、CBP、Marine Corps等、課題共通性80%）

**UVP**: "Software-Defined Defense" - AI・自律システムで従来の防衛調達を根本から変革

**10倍優位性**（5軸達成）:

| 軸 | 従来（Lockheed Martin等） | Anduril | 倍率 |
|---|--------------------------|---------|:----:|
| 開発速度 | 5-10年 | 1-2年 | 10x |
| コスト | 数億ドル | 推定1/5 | 5x |
| 導入障壁 | 複雑な調達プロセス | パイロット→本格契約 | 8x |
| 柔軟性 | ハードウェア固定 | ソフトウェア定義型 | 6x |
| 自律性 | 人間オペレーター依存 | AI・自律24/7監視 | 12x |

**技術的差別化**:
- Lattice AIプラットフォーム: 複数センサー統合、自律運用、リアルタイム意思決定
- 商業技術（AI、機械学習、コンピュータビジョン）を防衛応用
- 自己資金先行開発 → 完成品販売（従来企業は前払い受領後に開発）

```
自己資金先行開発 → パイロット契約 → 実績構築 → VC大型調達
    ↓                                              ↑
より大型契約獲得 ← 製品拡充＋R&D強化 ← ネットワーク効果 ← |
```

- 利益率: ラベリングAPI 60%、政府契約50%（セキュリティコスト高）
