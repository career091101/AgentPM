### 2. orchestrate-review-loop

**統合日**: 2026-01-03
**ケーススタディ数**: 13件（4社から抽出）

#### 統合事例

**Success Patterns（5件）**:
1. Superhuman - 反復的PMF測定（22% → 32% → 58%）
2. Airbnb - ボトルネック発見と即座の対応（予約2-3倍増）
3. Airbnb Pitch Deck - 視覚的ストーリーテリング（投資家説得力120/130点）
4. Airbnb Scorecard - 包括的評価（6カテゴリ50点満点、45点で即Series A推奨）
5. Airbnb CPF検証 - 統計的有意性（100人インタビュー、課題共通率85%）

**Common Pitfalls（4件）**:
1. 不十分なサンプルサイズによる誤判定（10-20人では統計的有意性不足）
2. 抽象的な表現によるエビデンス不足（「市場が大きい」等の根拠なき主張）
3. 論理的矛盾によるVC不信（「市場が小さい」×「急成長市場」の矛盾）
4. セグメンテーション不足によるPMF誤判定（全ユーザー平均値のみで判断）

**Quantitative Benchmarks（12指標）**:
- CPFスコア: 70%以上（Airbnb実績85%）
- PMF Sean Ellisテスト: 50%以上（Superhuman実績58%）
- 月次成長率: 20%以上（Airbnb実績25-30%）
- 総合Scorecard: 40点以上/50点満点（Airbnb 45点）
- Pitch Deck投資家説得力: 110点以上/130点満点（Airbnb 120点）
- レビュー品質スコア: 70点以上/100点満点で合格
- 反復改善の効果: 1イテレーションあたり+10-20ポイント

**Best Practices（5件）**:
1. 早期レビューの実施（SubAgent完了直後、統合前）
2. 定量的評価基準の明示（5観点100点満点評価）
3. セグメンテーション分析の徹底（「非常に残念」グループに焦点）
4. Human-in-the-Loopの適切なタイミング（3回リトライ後も70点未満で停止）
5. 証拠記録による学習ループ（イテレーション履歴の完全トレース）

#### 主要な発見

- **品質管理の重要性**: Airbnb 100人インタビューで85%達成、Superhuman 22% → 58%への段階的向上
- **早期レビューの効果**: ボトルネック発見と即座の対応で予約2-3倍増、月次成長率25-30%回復
- **セグメンテーション分析の重要性**: 「非常に残念」グループ特定でSean Ellis 58%達成
- **定量的評価基準の明示**: 各カテゴリの配点と合格ラインを事前明示で透明性確保

### 3. validate-market-timing

**統合日**: 2026-01-03
**ケーススタディ数**: 7件

#### 統合事例

**成功事例（適切なタイミング）- 3件**:

1. **Groupon (2008)**
   - タイミングスコア: 10/10
   - 2008年金融危機という完璧なタイミングで参入
   - 16ヶ月で$1B評価到達（史上最速Unicorn）
   - キーインサイト: 外部環境の危機が新ソリューションへの需要を爆発させる

2. **Stripe (2013)**
   - タイミングスコア: 8.8/10
   - タブレットPOS市場の黎明期に参入
   - 技術成熟度10点、顧客準備度10点、市場成長率10点
   - 2016年までに10万店舗達成

3. **LinkedIn (2003)**
   - ドットコムバブル崩壊直後の2003年参入
   - 失業率上昇でネットワーキングの緊急性増大
   - 8年でIPO達成、2016年Microsoft買収$26.2B

**失敗事例（早すぎる Too Early）- 3件**:

4. **Quibi (2020)**
   - COVID-19パンデミックと完全に矛盾（「移動中にモバイル視聴」 ↔ ロックダウン）
   - 6ヶ月で撤退、$1.75B焼却
   - キーインサイト: 予期しない外部環境変化への柔軟性欠如

5. **Jawbone (2011)**
   - ウェアラブル市場の顧客準備度が低い（課題認識30%未満）
   - 初代UPを大規模リコール→ブランド信頼失墜
   - 2017年清算、$930M焼却

6. **Segway (2001)**
   - 技術的には20倍優位だが、顧客準備度3点、規制環境3点
   - 価格$4,950は大衆市場に高すぎる
   - 19年間で14万台（目標50万台/年の0.3%）
   - キーインサイト: 技術優位性≠顧客価値

**失敗事例（遅すぎる Too Late）- 1件**:

7. **CAREER CARVER (2012)**
   - リクルートエージェントが既に市場制圧（シェア60%以上）
   - 人材紹介市場成長率5%/年（既に成熟期）
   - 2017年統合、累損18億円

#### 主要な発見

- **タイミング成功の共通要素**: 外部環境の追い風、技術成熟度とインフラ整備、顧客課題認識率60%以上、市場成長率20%/年以上
- **タイミング失敗の共通要素**: 顧客準備度3点以下、規制環境の未整備、予期しない外部環境変化、競合先行市場
- **5次元評価の重要性**: 3点以下が1つでもあれば警告サイン、総合スコア70点以上が目安

## 統合による改善効果

### 1. 定量的評価基準の具体化

**改善前**: 抽象的な評価基準（「シナジーが高い」「タイミングが良い」等）
**改善後**: 具体的な数値基準と実績値の明記

**事例**:
- **シナジースコア**: 45点以上/60点で成功率100%、ROI 6,300%
- **クロスセル率**: 57%（Figma）、40%（リクルートAgent）
- **Sean Ellisテスト**: 50%以上（Superhuman実績58%）
- **タイミングスコア**: 総合70点以上（Groupon 10点、Stripe 8.8点）

### 2. 失敗パターンの体系化

**改善前**: 抽象的な失敗要因（「タイミングが早すぎた」等）
**改善後**: 具体的な失敗パターンと教訓の明記

**事例**:
- **不十分なサンプルサイズ**: インタビュー10-20人では統計的有意性不足 → 30人以上必須（Airbnb 100人実施）
- **顧客準備度の見誤り**: Jawbone（課題認識30%）、Segway（課題認識15%） → 60%以上が目安
- **予期しない外部環境変化**: Quibi COVID-19対応失敗 → 柔軟性確保の重要性

### 3. ベストプラクティスの明示

**改善前**: 一般的なアドバイス
**改善後**: 実証された具体的な手法の明記

**事例**:
- **早期レビューの実施**: SubAgent完了直後にReview Agent起動（Airbnb事例: 予約2-3倍増）
- **セグメンテーション分析**: 「非常に残念」グループに焦点（Superhuman: 22% → 58%）
- **営業効率向上**: AI活用アタックリスト（Figma QR: 成約率3倍）

### 4. 参照パスの整備

**改善前**: 参照元が不明確
**改善後**: 具体的なファイルパスを明記

**事例**:
- `@Founder_Agent_ForStartup/Founder_Research/documents/SUCCESS/TIER3_SAAS/CORP_S012_airpay.md`
