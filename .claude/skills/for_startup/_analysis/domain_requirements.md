## ターゲットユーザー

### プライマリユーザー
- **VC資金調達を目指すスタートアップ創業者**
- **Seed/Series A-C資金調達準備中の起業家**
- **急成長（3倍以上/年）を目指す事業の創業者**
- **ピッチデッキ作成・投資家対応が必要な方**

### ユーザーの典型的なシナリオ
1. **MVP完成直後**: プロダクト市場適合を検証し、投資家向けに整理したい
2. **初期トラクション獲得**: ユーザー数・売上の初期実績をもとに、Seed/Series A調達を目指している
3. **資金調達準備**: ピッチデッキ作成、VC対応Q&A準備など、投資家応対の準備が必要
4. **スケール検証**: ビジネスモデルの急成長可能性をVC基準で検証したい

### ユーザーの課題
- **調達成功への不確実性**: Seed調達成功率は約5-10%と低い
- **体系的検証の不足**: CPF/PSF検証を実施している創業者は10%未満
- **VC対応の負担**: VC対応に月40-80時間を費やすが、体系化されていない
- **ピッチング準備の時間**: 従来10-20時間かかるピッチデッキ作成を効率化したい

## 評価基準の変更（Originとの差分）

### 1. 厳格化されたステージゲート基準

| 評価項目 | Origin | ForStartup | 変更理由 |
|---------|--------|------------|---------|
| **CPFスコア（課題の切実度）** | 60%以上 | **70%以上** | VC投資水準では高い課題の切実度が必須。解くべき問題の重大性がVC投資判断の重要な基準 |
| **10倍優位性の軸数** | 2軸以上 | **3軸以上** | スケーラビリティ重視。単一軸では競合参入時に優位性を失う可能性が高い |
| **インタビュー件数** | 20人 | **30人** | より確実な市場検証。VC基準では統計的有意性が求められる |
| **UVP刺さり度（ユニークバリュープロポジション）** | 28/40以上 | **35/40以上** | 投資家への説得力強化。プレゼン品質がVC投資判断に大きく影響 |

### 2. スコアカード評価基準の変更（40点満点）

| 評価項目 | 配点 | Origin基準 | ForStartup基準 | 変更理由 |
|---------|------|----------|---------------|---------|
| **市場機会** | 8点 | 6点以上 | **7点以上** | 市場規模の最小要件が高い（TAM > $100M推奨） |
| **課題の切実度（CPF）** | 8点 | 6点以上 | **7点以上** | CPFスコア70%以上の達成が必須 |
| **ソリューション（PSF）** | 8点 | 6点以上 | **7点以上** | ソリューションの実装完成度（MVP以上）が必須 |
| **ビジネスモデル** | 6点 | 4点以上 | **5点以上** | Unit Economics（LTV/CAC 5.0以上）が必須 |
| **実行可能性** | 6点 | 4点以上 | **5点以上** | チーム構成、資本効率が重視 |
| **独自性・競争優位** | 4点 | 3点以上 | **4点以上** | 3軸以上の10倍優位性が必須 |

### 3. 合格判定基準

| スコア範囲 | 判定 | 次のアクション |
|-----------|------|------------------|
| **35-40点** | ✅ VC調達推奨レベル | ピッチデッキ作成へ進行。Series A調達も検討可能 |
| **28-34点** | ⚠️ 要改善 | 低スコア項目の強化必須。Seed調達を見送り、さらなる検証を実施 |
| **0-27点** | ❌ 要再設計 | Originレベルから再検証。ビジネスモデルの根本的見直しが必要 |

## 必須機能

### 1. ピッチデッキ自動生成 (`/build-pitch-deck`)

**目的**: 10-15スライドの投資家向けピッチデッキを自動生成

**スライド構成**:
1. **Problem（課題）** - 3Uスコアを可視化、課題の大きさを定量化
2. **Solution（ソリューション）** - UVP明示、MVP機能の説明
3. **Market Size（市場規模）** - TAM/SAM/SOM の明示
4. **Product（プロダクト）** - MVP完成状況、ロードマップ
5. **Traction（トラクション）** - KPI、成長率、初期ユーザー数
6. **Business Model（収益モデル）** - Unit Economics、LTV/CAC 比率
7. **Competition（競合分析）** - 10倍優位性の明示、参入障壁
8. **Go-to-Market（販売戦略）** - グロースハック施策、フライホイール
9. **Team（チーム）** - FIF（Founder-Issue-Fit）、チーム経験
10. **Financials（財務計画）** - 3年間の予測、資金使途計画
11. **Ask（調達額・用途）** - ラウンド計画、資金配分

**出力形式**: Markdown + PDF（Presentation対応）

### 2. VC対応質問リスト (`/prepare-vc-meeting`)

**目的**: 典型的なVC質問への回答を事前に準備

**対応すべきVC質問**:
- **Timing**: "なぜ今なのか？" - 市場タイミングの説得力
- **FIF**: "なぜあなたなのか？" - Founder-Issue-Fit の説明
- **10倍優位性**: "10倍どこで勝つのか？" - 競争優位性の明示
- **成長率**: "1年後のKPIは？" - トラクション計画の現実性
- **参入障壁**: "競合が真似したらどうするのか？" - 防御可能性

**出力形式**: Q&A集（各質問に対する回答案を提供）

### 3. 資金調達ロードマップ (`/create-fundraising-plan`)

**目的**: Pre-Seed → Series B までの資金調達計画を策定

| ラウンド | 調達額目安 | 評価額目安 | 達成すべきマイルストーン | 期間 |
|---------|----------|----------|----------------------|------|
| **Pre-Seed** | ¥30-50M | ¥100-300M | CPF達成（70%）、MVP完成 | 3-6ヶ月 |
| **Seed** | ¥50-200M | ¥300M-1B | PSF達成、初期トラクション（10-50ユーザー） | 6-9ヶ月 |
| **Series A** | ¥200M-1B | ¥1-5B | PMF達成、成長率20%以上/月 | 9-12ヶ月 |
| **Series B** | ¥1-5B | ¥5-20B | スケール確立、営業組織構築、収益化開始 | 12-18ヶ月 |

**出力形式**: Markdown（タイムライン、マイルストーン、資金使途を明示）

### 4. ユニットエコノミクス厳格検証 (`/validate-unit-economics-strict`)

**目的**: LTV/CAC 比率を厳格に検証し、VC基準を満たしているか確認

**検証項目**:
- **LTV（顧客生涯価値）**: 平均顧客利益 × 平均契約期間（月）
- **CAC（顧客獲得コスト）**: マーケティング費用 ÷ 新規顧客数
- **LTV/CAC比率**: 5.0以上を目標（Origin基準の3.0以上から引き上げ）
- **Payback期間**: CAC回収までの期間が12ヶ月以内
- **GrossMargin**: SaaS > 70%、E-commerce > 30% 以上
- **月次成長率**: 20%以上、年次3倍以上を必須化

**出力形式**: Excel + Markdown レポート

## スコアカード評価体系

### 詳細評価ルーブリック

#### 市場機会（8点）

| 点数 | 基準 |
|------|------|
| 8点 | TAM > $1B、急成長市場（年20%以上）、最低でも $100M規模の確認ユーザーニーズ |
| 7点 | TAM > $300M、市場成長率年10-20%、$100M未満だが確実なニーズ確認 |
| 6点 | TAM > $100M、市場成長率年5-10%、限定的なニーズ確認 |
| 5点以下 | TAM < $100M、または市場成長の見込み低い |

#### 課題の切実度（CPF: Critical Problem Fit）（8点）

| 点数 | 基準（CPFスコア） |
|------|-----------------|
| 8点 | CPFスコア 80%以上、ユーザーがお金を払う意思がある（実際に支払い例あり） |
| 7点 | CPFスコア 70-79%、高い課題への実感、複数ユーザーから確認（ただし未支払い） |
| 6点 | CPFスコア 60-69%、課題の確認、単一ユーザーグループのみ |
| 5点以下 | CPFスコア 60%未満、仮説段階 |

#### ソリューション（PSF: Problem-Solution Fit）（8点）

| 点数 | 基準 |
|------|------|
| 8点 | MVP完成、複数ユーザーが実際に使用、NPS 50以上 |
| 7点 | MVP完成、プロトタイプテスト完了、ユーザーフィードバック統合済み |
| 6点 | プロトタイプ段階、初期テスト実施、改善ロードマップ明確 |
| 5点以下 | 概念段階、テスト未実施 |

#### ビジネスモデル（6点）

| 点数 | 基準（SaaS基準） |
|------|-----------------|
| 5点 | LTV/CAC > 5.0、GrossMargin > 70%、成長率月20%以上 |
| 4点 | LTV/CAC 3.0-5.0、GrossMargin 60-70%、成長率月10-20% |
| 3点 | LTV/CAC 2.0-3.0、GrossMargin < 60%、成長率月5-10% |
| 2点以下 | LTV/CAC < 2.0、経済性が未確認 |

#### 実行可能性（6点）

| 点数 | 基準 |
|------|------|
| 5点 | チーム人数 3人以上、技術・営業・ビジネス全分野カバー、最低資本6ヶ月分確保 |
| 4点 | チーム人数 2人、複数分野の経験者含む、資本3-6ヶ月分 |
| 3点 | チーム人数 1-2人、経験にギャップ、資本1-3ヶ月分 |
| 2点以下 | ソロ、経験不足、資本不足 |

#### 独自性・競争優位（4点）

| 点数 | 基準 |
|------|------|
| 4点 | 3軸以上の10倍優位性（技術、ネットワーク効果、ブランド、顧客関係等） |
| 3点 | 2軸の10倍優位性（単一軸では不十分） |
| 2点 | 1軸の優位性（模倣可能性が高い） |
| 1点以下 | 競合との差別化不明確 |

## 資金調達ロードマップ

### タイムライン（概要）

```
Month 1-3: Pre-Seed Check
  - CPF 70%達成確認
  - MVP完成・初期テスト
  - チーム構成確定
  - 資金需要額確定

Month 3-6: Seed 準備
  - PSF達成（初期ユーザー 10-50）
  - 初期トラクション（月成長率 10%以上）
  - ピッチデッキ作成完了
  - VC接触開始

Month 6-9: Seed 調達
  - 投資先決定
  - 資金受け取り
  - 月成長率 15%以上を実証

Month 9-12: Series A 準備
  - PMF達成の兆候（NPS 40+、Retention 80%+）
  - 成長率月20%以上を継続
  - 初期営業チーム構築
  - Series A ピッチ準備

Month 12-18: Series A 調達
  - 資金受け取り
  - スケール体制構築
  - Series B 準備開始
```

### 各ラウンドのチェックリスト

#### Pre-Seed（自己資金またはエンジェル投資 ¥30-50M）

- [ ] CPFスコア 70%以上
- [ ] MVP完成・複数ユーザーでテスト
- [ ] 3人以上のチーム体制
- [ ] 最低 6ヶ月の資本確保
- [ ] Founder-Issue-Fit の明確化

#### Seed（¥50-200M）

- [ ] PSF達成（初期ユーザー 10-50、NPS > 30）
- [ ] 初期トラクション（月成長率 10%以上、ARR ¥1-10M想定）
- [ ] ピッチデッキ完成
- [ ] チーム拡大計画（営業/マーケティング採用予定）
- [ ] 12ヶ月の資本計画

#### Series A（¥200M-1B）

- [ ] PMF達成の明確化（NPS 50+、Retention 80%+）
- [ ] 月成長率 20%以上を 3-6ヶ月継続実証
- [ ] ARR ¥50M-500M レベル
- [ ] Unit Economics：LTV/CAC > 5.0
- [ ] GTMスケーラビリティの実証

#### Series B（¥1-5B）

- [ ] 成長率月20%以上を 12ヶ月以上継続
- [ ] ARR > ¥500M
- [ ] 営業組織の確立（営業チーム規模 5人以上）
- [ ] ローカリゼーション / グローバル展開の準備
- [ ] 利益化への道筋（経営効率化）

## 実装ガイドライン

### スキル開発時の注意点

1. **Founder_Research との連携**
   - 597件のケーススタディから業種別成功基準を自動抽出
   - 失敗パターンの警告をユーザーに提示
   - 類似事例からのベストプラクティス引用

   - 四半期ごとのVC業界トレンド調査
   - 投資基準の変動に対応した設定値の動的調整
   - 新興市場（AI、Web3等）への対応

3. **ユーザーサポート体系**
   - Originレベルから ForStartup基準への段階的なガイダンス
   - Originレベルでの合格ユーザーへのアップセル推奨
   - ForStartupレベル合格後の Series A・B サポート（今後実装）
