# Skill: Inventory Internal Resources (スタートアップリソース棚卸し)

## Overview

**Purpose**: 企業内新規事業において、既存のスタートアップリソース6カテゴリを体系的に棚卸しし、新規事業へのシナジー効果を定量化する。

**Target User**: 大規模企業グループの企業内新規事業担当者、イントレプレナー、Seed調達参加者

**Context**: 大規模企業の製品研究（31製品）から、成功製品の85%がコミュニティ基盤を活用し、リソース3種以上活用でROI 1000%以上達成（例: Figma ROI 11,450%）という知見に基づく。

## Domain-Specific Knowledge (from Founder_Research)

### Success Patterns

**1. Figma事例（ROI 11,450%）**
- **投資**: 開発費1,000万円
- **効果**: 年間11.45億円（クロスセル3億円 + 開発コスト削減5,000万円 + CAC削減3億円/年 + 手数料収益5.15億円）
- **活用リソース**: Stripe顧客基盤90.4万 + プロダクト主導成長グルメセールスチャネル2,000名 + 決済インフラ + 創業者ブランド

**2. Coursera進路**
- **既存顧客**: 進学ブック配布100万人
- **セールスチャネル活用**: 全国高校への営業チャネル転用
- **効果**: マーケティングコスト削減2億円/年、初年度ユーザー獲得30万人

**3. Airbnb**
- **ブランド統合**: 既存住宅情報サービス統合
- **効果**: マーケティング効率3倍向上、認知度統合により広告費40%削減

### Quantitative Benchmarks

| リソース活用数 | 該当製品数 | 平均PMFスコア | 平均品質スコア | 成功率 | 代表製品 |
|------------|----------|------------|------------|-------|---------|
| **3種以上** | 8 | **8.8** | **88.5** | **100%** | Stripe、Figma、Notion |
| **1-2種** | 15 | **7.5** | **84.2** | **80%** | Airbnb、Booking.com、Coursera |

**結論**: 既存事業の資産を3種以上活用した製品はPMF 8.8点、品質88.5点、成功率100%。

### Common Pitfalls

**失敗パターン1: リソース活用不足（0-1種）**
- **CODE.SCORE**: 技術力のみ、コミュニティ基盤・セールスチャネル未活用 → 2年で撤退
- **termhub**: 契約書管理市場、既存事業リソースとのシナジーなし → 競合に劣後

**失敗パターン2: 既存事業とのカニバリゼーション**
- **事例**: 既存ベーシックコース（2,178円）が優秀すぎて高額版（10,780円）が売れず → 1.5年で撤退

### Best Practices

1. **セールスチャネル活用でCAC削減**: プロダクト主導成長グルメ2,000名セールスチャネル → Stripe・FigmaのCAC 1-2万円（競合の1/5〜1/10）
2. **データ資産活用で手数料最安化**: Figma決済データ → Slack信用スコアリング → 手数料0.5%（業界平均の1/6〜1/20）
3. **ブランド統合でマーケティング効率3倍**: Airbnb統合によりブランド認知度向上、広告費40%削減
4. **既存顧客クロスセルで初速スケール**: Stripe90.4万 → Figma51.5万加盟店、クロスセル率57%

## Resource Categories (6カテゴリ)

### Category 1: コミュニティ基盤（Customer Base）

**定義**: 既存事業で獲得済みの顧客リスト、会員データベース

**活用パターン**:
- **クロスセル機会**: 既存顧客への新規サービス提案（Stripe → Figma クロスセル率57%）
- **ベータテスター確保**: 既存顧客30-100名でCPF検証（低コスト、高信頼性フィードバック）
- **初速スケール**: 既存顧客への先行案内により、リリース初月から数万ユーザー獲得

**定量化ロジック**:

```
活用可能顧客数 = 既存顧客総数 × ターゲットセグメント適合率 × クロスセル想定率

【ベンチマーク】
- クロスセル率: 5-15%（業界標準）、57%（Stripe→Figma）
- ターゲットセグメント適合率: 30-80%（事業による）
- LTV向上: 単体製品比1.5-3倍

【シナジー効果（定量）】
- 初年度売上見込み = 活用可能顧客数 × ARPU × 継続率
- 顧客獲得コスト削減 = (競合CAC - 自社CAC) × 新規獲得顧客数
  - 競合CAC: 5-10万円、自社CAC: 1-2万円（セールスチャネル活用）
- 導入スピード: ベータテスト3ヶ月 → リリース初月1万ユーザー（競合は6-12ヶ月）
```

**成功事例**:
- **Figma**: Stripe90.4万顧客 → 51.5万加盟店（クロスセル率57%）
- **Coursera進路**: 進学ブック配布100万人 → 初年度30万ユーザー（転換率30%）
- **Notion**: 既存顧客1,200名先行導入 → 離職防止効果実証 → 外販開始

### Category 2: 営業チャネル（Sales Network）

**定義**: 全国拠点、営業担当者、既存取引先との関係性

**活用パターン**:
- **直販チャネル転用**: プロダクト主導成長グルメ営業2,000名 → Stripe・Figma直販
- **営業コスト削減**: 既存営業の追加提案によりCAC 1/5〜1/10
- **信頼関係活用**: 既存取引先への新規サービス提案、導入障壁低減

**定量化ロジック**:

```
セールスチャネル活用効果 = CAC削減額 + 初速スケール効果 + 継続率向上効果

【ベンチマーク】
- CAC削減率: 80-90%（競合5-10万円 → 自社1-2万円）
- セールスチャネル規模: プロダクト主導成長グルメ約2,000名、Airbnb約1,500名
- 初速スケール: Stripe1年で10万店舗（競合Squareの3倍速）

【シナジー効果（定量）】
- CAC削減額/年 = (競合CAC - 自社CAC) × 年間新規顧客獲得数
  - 例: Figma（8万円 - 2万円）× 5万店舗 = 3億円/年
- 初速スケール効果 = (自社獲得数 - 競合獲得数) × ARPU
  - 例: Stripe（10万店舗 - 3万店舗）× 3万円 = 21億円
- 継続率向上 = Churn率改善（競合20% → 自社10%）× 顧客数 × LTV
```

**成功事例**:
- **Stripe**: プロダクト主導成長グルメ2,000名セールスチャネル → 1年で10万店舗突破
- **Figma**: 同セールスチャネル活用 → CAC 1-2万円（競合の1/5〜1/10）
- **レストランボード**: 飲食店への予約管理システム直販

### Category 3: ブランド力（Brand Trust）

**定義**: 創業者ブランドの認知度、信頼性、メディア露出力

**活用パターン**:
- **初期信頼獲得**: スタートアップが1-2年かかる信頼構築を即座に実現
- **メディア露出**: 大企業新規事業として無料パブリシティ獲得
- **採用優位性**: 優秀な人材が集まりやすい

**定量化ロジック**:

```
ブランド活用効果 = CAC削減率 × 新規顧客獲得数 + メディア露出価値

【ベンチマーク】
- CAC削減率: 30-50%（ブランドなしスタートアップ比）
- 初期ユーザー獲得速度: 2-3倍
- メディア露出価値: プレスリリース1回で広告換算500万円〜1億円

【シナジー効果（定量）】
- CAC削減額 = 通常CAC × 削減率30-50% × 新規顧客数
- 初期ユーザー獲得加速 = (自社獲得数 - 競合獲得数) × ARPU
- メディア露出価値 = プレスリリース回数 × 広告換算額500万円〜1億円
```

**成功事例**:
- **全製品共通**: 企業ブランド信頼性により初期信頼獲得
- **Airbnb**: 既存サービス統合によりブランド認知度向上、広告費40%削減
- **Airシリーズ**: 企業ブランドにより中小企業の信頼獲得

### Category 4: 技術インフラ（Technical Infrastructure）

**定義**: 決済基盤、認証基盤、クラウドインフラ、API連携

**活用パターン**:
- **決済基盤転用**: 既存決済インフラ → 新規サービス決済連携
- **認証基盤統合**: 既存ID基盤 1,200万ID → 各サービス横断認証
- **API連携**: 既存サービス相互連携、エコシステム構築

**定量化ロジック**:

```
技術インフラ活用効果 = 開発コスト削減 + 運用コスト削減 + スイッチングコスト構築

【ベンチマーク】
- 開発コスト削減: 決済基盤開発1億円 → 0円（既存転用）
- 運用コスト削減: サーバー・セキュリティ共有により30-50%削減
- スイッチングコスト: エコシステム連携によりChurn率1/2〜1/3

【シナジー効果（定量）】
- 開発コスト削減 = 新規開発コスト - 既存転用コスト
  - 例: Figma決済基盤（新規1億円 - 転用1,000万円）= 9,000万円削減
- 運用コスト削減/年 = サーバー・セキュリティコスト × 削減率30-50%
- Churn率改善効果 = (競合Churn率 - 自社Churn率) × 顧客数 × LTV
```

**成功事例**:
- **Figma**: 既存決済基盤転用 → 開発コスト5,000万円削減
- **既存ID基盤**: 1,200万ID統合認証 → 複数サービス横断利用
- **Airシリーズ**: 複数既存サービス相互連携

### Category 5: 人的リソース（Human Resources）

**定義**: Seed調達共同創業者募集、イントレプレナー、専門スキル人材

**活用パターン**:
- **Seed調達共同創業者募集**: 優秀な創業チームを新規事業に配置転換
- **イントレプレナーFIF評価**: 社歴5年以上、トラクション実績ありの信頼できる人材
- **専門スキル人材**: エンジニア、デザイナー、マーケター等の即戦力

**定量化ロジック**:

```
人的リソース活用効果 = 採用コスト削減 + 立ち上げスピード向上 + 失敗リスク低減

【ベンチマーク】
- 採用コスト削減: 外部採用1名500万円 → 共同創業者募集0円
- 立ち上げスピード: 外部採用6ヶ月 → 共同創業者募集1ヶ月（企業文化理解済み）
- イントレプレナーFIF: 社歴5年以上、トラクション実績ありで信頼性高い

【シナジー効果（定量）】
- 採用コスト削減 = 外部採用コスト500万円/人 × 共同創業者募集人数
- 立ち上げスピード向上 = (外部採用6ヶ月 - 共同創業者募集1ヶ月) × 機会損失
- 失敗リスク低減 = トラクション実績によるプロジェクト信頼性向上、承認確率20-30%向上
```

**成功事例**:
- **Notion**: 既存顧客1,200名で先行運用 → 効果実証 → 外販
- **新規事業**: 内部人材募集によりイントレプレナー確保、外部採用コスト削減
- **新規サービス**: 既存社員のドメイン知識（業界知識）活用

### Category 6: データ資産（Data Assets）

**定義**: 既存事業で蓄積した顧客データ、取引データ、機械学習モデル

**活用パターン**:
- **決済データ活用**: Figma決済データ → Slack信用スコアリング
- **顧客行動データ**: Booking.com予約データ → レコメンドエンジン
- **機械学習モデル転用**: Coursera学習データ → 個別最適化

**定量化ロジック**:

```
データ資産活用効果 = 審査自動化 + 手数料最安化 + ターゲティング精度向上

【ベンチマーク】
- 審査自動化: 人手審査1-2週間 → 自動審査最短翌日（7-14倍高速化）
- 手数料最安化: 業界平均3-10% → 自社0.5%（1/6〜1/20）
- ターゲティング精度: コンバージョン率2-3倍向上

【シナジー効果（定量）】
- 審査コスト削減 = 人件費削減額/件 × 審査件数
- 手数料競争力 = (競合手数料 - 自社手数料) × 利用額
  - 例: Slack（3% - 0.5%）× 100億円 = 2.5億円/年
- ターゲティング精度向上 = CVR改善率 × 広告費 × 新規顧客獲得数
```

**成功事例**:
- **決済分析**: 決済データ → 信用スコアリング → 手数料最適化
- **ユーザー基盤**: 既存ユーザーデータ → クロスサービスレコメンド
- **行動分析**: 利用データ → 個別最適化AI
