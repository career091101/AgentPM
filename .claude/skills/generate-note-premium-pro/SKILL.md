# 有料note生成スキル（generate-note-premium-pro）

## 概要

このスキルは、有料販売向けの高品質note記事（3万文字以上）を自動生成します。

### 主要機能

1. **3万文字以上の本格コンテンツ生成**
   - 情報密度を保った自然文
   - 冗長な水増しなし、具体策・手順・テンプレート必須
   - 読了率を意識した構成

2. **10セクション構成**
   - 表紙〜追伸まで完全網羅
   - 各章に具体例・失敗例・ワーク・チェックリスト
   - 読者の「次の一歩」を明確化

3. **ニーズ推定エンジン**
   - 入力から読者の悩み・理想・障害・購入動機を自動推定
   - ターゲット心理に刺さる構成を自動設計

4. **品質保証**
   - 有料級品質（無料記事レベルで終わらせない）
   - 共感・再現性・読了率を最大化
   - 同じ表現の連呼・冗長な水増し禁止

### 制約事項

- **文字数**: 3万文字以上（情報密度で達成、水増し禁止）
- **構成**: 10セクション必須
- **品質基準**: 有料級チェックリスト全項目クリア必須
- **出力形式**: Markdown形式、noteにそのまま貼れる整形済み

## 入力仕様

### 必須パラメータ

| パラメータ | 型 | 説明 | 例 |
|-----------|---|------|---|
| `product_title` | string | 商品タイトル | `"AI副業で月30万円稼ぐ完全ロードマップ"` |
| `target_audience` | string | ターゲット層（属性・悩み） | `"副業を始めたい30代サラリーマン、時間がない、何から始めればいいかわからない"` |
| `genre` | string | ジャンル | `"副業・AI活用"` |

### オプションパラメータ

| パラメータ | 型 | デフォルト | 説明 |
|-----------|---|----------|------|
| `output_type` | string | `"full_text"` | 出力タイプ（`"full_text"`: 本文生成、`"template_only"`: プロンプトテンプレートのみ） |
| `chapter_count` | integer | `10` | 章数（8/10/12から選択） |
| `tone` | string | `"professional"` | トーン（`"professional"`, `"casual"`, `"motivational"`） |
| `include_appendix` | boolean | `true` | 付録を含めるか |

### 入力例

```json
{
  "product_title": "AI副業で月30万円稼ぐ完全ロードマップ",
  "target_audience": "副業を始めたい30代サラリーマン、時間がない、何から始めればいいかわからない",
  "genre": "副業・AI活用",
  "output_type": "full_text",
  "chapter_count": 10,
  "tone": "motivational"
}
```

## 処理フロー

### STEP 1: ニーズ推定（LLM直接推論）

LLMへの指示：
```
入力された「商品タイトル」「ターゲット層」「ジャンル」から、以下を推定：

1. **悩み（Pain Points）**
   - 表面的な悩み（3-5個）
   - 深層の悩み（2-3個）
   - 言語化されていない悩み（1-2個）

2. **理想の状態（Desired State）**
   - 短期的な理想（1-3ヶ月後）
   - 中期的な理想（6ヶ月-1年後）
   - 長期的な理想（3年後）

3. **障害（Obstacles）**
   - 内的障害（マインドセット、スキル不足）
   - 外的障害（時間、お金、環境）
   - 情報障害（何から始めればいいかわからない）

4. **購入動機（Buying Triggers）**
   - 緊急性（今すぐ解決したい理由）
   - 信頼性（なぜこのnoteを信じるか）
   - 期待値（購入後に得られる具体的成果）

出力形式：
{
  "pain_points": {
    "surface": ["..."],
    "deep": ["..."],
    "unspoken": ["..."]
  },
  "desired_state": {
    "short_term": "...",
    "mid_term": "...",
    "long_term": "..."
  },
  "obstacles": {
    "internal": ["..."],
    "external": ["..."],
    "information": ["..."]
  },
  "buying_triggers": {
    "urgency": "...",
    "credibility": "...",
    "expectation": "..."
  }
}
```

### STEP 2: 章設計（8-12章構成、3万文字配分）

LLMへの指示：
```
STEP 1のニーズ推定に基づき、3万文字を超えるための章設計を行う：

1. **章数の決定**
   - 8章構成: シンプル・入門者向け（各章3,000文字）
   - 10章構成: 標準・中級者向け（各章2,500文字）★推奨
   - 12章構成: 網羅的・上級者向け（各章2,200文字）

2. **各章の役割定義**
   章ごとに以下を設計：
   - 章タイトル（読者の興味を引く）
   - 章の目的（この章で読者が得る変化）
   - 必須要素（具体例、ワーク、チェックリスト等）
   - 文字数目安

3. **文字数配分**
   - 表紙〜導入: 3,000文字（10%）
   - 全体マップ: 1,000文字（3%）
   - 本文（8-12章）: 22,000文字（73%）
   - FAQ: 2,000文字（7%）
   - まとめ〜追伸: 2,000文字（7%）
   - 合計: 30,000文字以上

4. **章構成パターン選択**
   Read toolで `structure_templates.md` を読み込み、
   ジャンルに最適なパターンを選択
```

章設計の出力例：
```json
{
  "chapter_count": 10,

    {

      "required_elements": ["市場データ", "成功事例", "失敗例"],
      "target_chars": 2500
    },
    // ... 章2-10
  ]
}
```

### STEP 3: 本文生成（10セクション構成）

**重要**: 本文生成は全てLLMの自然言語能力を活用。情報密度を保ち、冗長な水増しは禁止。

#### セクション1: 表紙（300文字）

LLMへの指示：
```
1. **タイトル案**（3パターン）
   - 数字を含む具体的なタイトル
   - 感情を刺激するタイトル
   - 問いかけ型タイトル

2. **サブタイトル案**
   - ターゲット層を明示
   - 得られる成果を明示

3. **一言ベネフィット**
   - 「このnoteを読めば、あなたは○○できるようになります」形式
```

#### セクション2: このnoteで得られること（500文字）

LLMへの指示：
```
以下を箇条書きで3-7項目：

1. **スキル系**
   - 「○○ができるようになる」形式
   - 具体的な動作を明示

2. **成果系**
   - 「○○を達成できる」形式
   - 数値目標を含める

3. **変化系**
   - 「○○から○○に変わる」形式
   - ビフォーアフターを明示
```

#### セクション3: こんな人向け（500文字）

LLMへの指示：
```
ターゲット層を具体化：

1. **属性**（職業、年齢、状況）
2. **悩み**（具体的な言葉で3-5個）
3. **状態**（今まさに○○な人）
4. **除外条件**（こんな人には向いていません）
```

#### セクション4: 導入・共感ストーリー（2,000文字）

LLMへの指示：
```
読者の悩み→希望を描く共感ストーリー：

1. **問題提起**（500文字）
   - 読者が抱える具体的な悩みを言語化
   - 「あなたもこんな経験ありませんか？」形式

2. **共感**（500文字）
   - 筆者自身の失敗体験・苦労を開示
   - 「私も以前は○○でした」形式

3. **転機**（500文字）
   - 何がきっかけで変わったか
   - 「ある日、○○に気づいたのです」形式

4. **希望提示**（500文字）
   - このnoteを読めばどうなるか
   - 「このnoteでは、○○を完全公開します」形式
```

#### セクション5: 全体マップ（1,000文字）

LLMへの指示：
```
章立ての概要と読み方ガイド：

1. **全体の流れを図示**
   - 章番号とタイトル一覧
   - 各章の狙いを1行で

2. **読み方の提案**
   - 「時間がない人は第○章から」
   - 「初心者は順番通りに」
   - 「経験者は第○章を重点的に」
```

#### セクション6: 本文（8-12章、22,000文字）

LLMへの指示：
```
各章の構成（章あたり2,000-3,000文字）：

1. **導入**（200文字）
   - この章で学ぶことの予告
   - 読者のメリットを明示

2. **本論**（1,500-2,000文字）
   - 具体的なノウハウ・手順
   - 「なぜ」「何を」「どうやって」を明記
   - 箇条書きと文章を組み合わせる

3. **具体例・事例**（300-500文字）
   - 成功事例（数字を含む）
   - 失敗事例（回避方法付き）

4. **ワーク・チェックリスト**（200-300文字）
   - 読者が今すぐ実践できるアクション
   - チェックボックス形式

5. **章のまとめ**（100-200文字）
   - 要点を3つに絞る
   - 次章への橋渡し

**禁止事項**:
- 抽象論のみで具体策なし
- 同じ表現の繰り返し
- 「〜と思います」の多用
- 情報密度の低い水増し
```

#### セクション7: よくある質問（2,000文字）

LLMへの指示：
```
読者が抱きそうな疑問をQ&A形式で5-10個：

1. **購入前の疑問**
   - 「本当に効果ありますか？」
   - 「初心者でも大丈夫ですか？」

2. **実践中の疑問**
   - 「○○で詰まったらどうすれば？」
   - 「どのくらいの期間で成果が出ますか？」

3. **応用の疑問**
   - 「他のジャンルにも使えますか？」
   - 「さらにレベルアップするには？」

各回答は：
- 具体的かつ安心感を与える
- 「〜してください」と行動を促す
- 200-300文字程度
```

#### セクション8: まとめ（1,000文字）

LLMへの指示：
```
要点再掲と次のアクション：

1. **要点再掲**（500文字）
   - 各章のエッセンスを1行ずつ
   - 「このnoteで学んだこと」形式

2. **次のアクション**（500文字）
   - 「今日からできる3つのこと」形式
   - 優先順位をつける
   - 具体的な期限を設定
```

#### セクション9: 付録（1,000文字）

LLMへの指示：
```
読者の実践をサポートするツール：

1. **テンプレート集**
   - そのままコピペで使える形式
   - カスタマイズ方法付き

2. **チェックリスト**
   - 進捗確認用
   - 印刷可能な形式

3. **30日プラン**
   - 日別のToDoリスト
   - マイルストーン設定
```

#### セクション10: 追伸（500文字）

LLMへの指示：
```
背中を押す一文：

1. **エモーショナルな締め**
   - 筆者の想いを伝える
   - 読者への応援メッセージ

2. **最後のCTA**
   - 「まずは○○から始めてください」
   - 連絡先・サポート情報
```

### STEP 4: 品質検証（有料級チェックリスト）

LLMへの指示：
```
生成した本文を以下の5観点で検証：

1. **文字数チェック**
   - 総文字数: 30,000文字以上か？
   - 各セクションの文字数配分は適切か？

2. **有料級チェック**
   - [ ] 具体的な手順・ステップが含まれているか
   - [ ] 数値データ・事例が3つ以上あるか
   - [ ] テンプレート・チェックリストが含まれているか
   - [ ] 読者が「今日から実践できる」内容か
   - [ ] 無料記事では得られない情報があるか

3. **共感チェック**
   - [ ] 読者の痛み・悩みを言語化しているか
   - [ ] 「自分のことだ」と思わせる具体例があるか
   - [ ] 安心感・希望を与えているか

4. **再現性チェック**
   - [ ] 「何を」「いつ」「どうやって」が明記されているか
   - [ ] 抽象論だけで終わっていないか
   - [ ] 初心者でも実行可能か

5. **読了率チェック**
   - [ ] 見出しは適切な間隔か（500-800文字ごと）
   - [ ] 箇条書きと文章のバランスは適切か
   - [ ] 1文が長すぎないか（60文字以内推奨）

不合格項目がある場合は修正して再検証。
```

## 品質基準

### 有料級の定義

| 項目 | 無料記事レベル | 有料級 |
|------|--------------|--------|
| 具体性 | 「頑張りましょう」 | 「毎日30分、この3ステップを実行」 |
| 数値 | なし | 「成功率87%」「3ヶ月で月収30万円」 |
| ツール | 言及のみ | テンプレート・チェックリスト付き |
| 事例 | 一般論 | 具体的な成功/失敗事例 |
| 行動 | 「試してみてください」 | 「今日から始める5ステップ」 |

### 品質スコア計算

```
品質スコア =
  文字数評価（20点）
  + 有料級評価（30点）
  + 共感評価（20点）
  + 再現性評価（20点）
  + 読了率評価（10点）

合格基準: 70点以上
推奨基準: 85点以上
```
