# CPF Stage Validation Skill

CPF（Customer Problem Fit）達成度診断。

---

## Skill情報

- **コマンド**: `/validate-cpf`
- **目的**: CPF達成度の定量診断
- **所要時間**: 20-30分

---

## このSkillでできること

1. **CPFチェックリスト判定**: 必須項目の達成度確認
2. **3Uスコア分析**: 課題の切実度評価
3. **インタビュー品質チェック**: 適切にヒアリングできているか
4. **Next Action**: PSFへ進むか、CPF強化か判断

---

## プロンプト

### セッション開始

```markdown
# CPF達成度診断

CPF（Customer Problem Fit）の達成度を診断します。

## CPFとは？

顧客が抱える課題が、解決する価値があるほど切実かを検証するフェーズ。

**判定基準**:
- 3U平均スコア 7.0+
- インタビュー30件、肯定50%+
- ペルソナ明確

---

## 現状を教えてください

### 1. ペルソナ
- 誰の課題を解決しますか？（具体的に）
- 年齢、職業、業界、企業規模など

### 2. 課題
- どんな課題を解決しますか？
- 現在、どうやって解決していますか？

### 3. 3Uスコア（自己評価 1-10点）
- **Unworkable（機能不全）**: 現在の解決策、うまくいっていない？
- **Unavoidable（回避不可）**: 避けて通れない課題？
- **Urgent（緊急）**: 今すぐ解決したい？
- **Underserved（未充足）**: 既存ソリューション不十分？

### 4. 顧客インタビュー
- 何件実施しましたか？
- そのうち、何人が「これは課題だ」と肯定しましたか？
```

---

### データ収集後 → 診断

```markdown
## CPF診断結果

### 1. ペルソナ明確度
- **スコア**: [5段階評価]
- **評価**:
  - 明確: ✅ 年齢・職業・行動パターンが具体的
  - 曖昧: ❌ 「中小企業」のみ、抽象的

### 2. 3Uスコア

| 項目 | スコア | 判定 |
|------|--------|------|
| Unworkable | [X]/10 | [OK/NG] |
| Unavoidable | [X]/10 | [OK/NG] |
| Urgent | [X]/10 | [OK/NG] |
| Underserved | [X]/10 | [OK/NG] |
| **平均** | [X]/10 | [OK/NG] |

**目標**: 平均7.0+

### 3. インタビュー実績

- 実施数: [X]件
- 肯定数: [Y]件
- 肯定率: [Y/X × 100]%

**目標**: 30件、肯定50%+

---

## 総合判定

[全項目クリアの場合]
### ✅ CPF達成！

PSFフェーズへ進みましょう。

Next Skill: `/validate-psf`

---

[1つでも未達の場合]
### ⚠️ CPF未達成

以下を改善してから再診断してください。

#### 改善が必要な項目
[未達項目と改善策]
```

---

### 改善提案

```markdown
## 改善アクションプラン

### [ペルソナ不明確の場合]

#### 問題
ペルソナが抽象的すぎます。

#### 改善策
1. インタビュー肯定者の共通点を分析
2. 具体的な1人をイメージ
3. テンプレート埋める

参考: @startup_science/01_stages/cpf/persona_creation.md

#### Tomorrow Action
- [ ] 肯定者[X]人の属性リスト化
- [ ] 共通パターン抽出
- [ ] ペルソナ1つ作成

---

### [3Uスコア低い場合]

#### 問題
3Uスコア平均[X]点（目標7.0+）

特に低い項目:
- [項目名]: [X]点

#### 改善策（2つの選択肢）

**選択肢A: 課題の再定義**
- より切実な課題を探す
- インタビューで「一番困っていること」を深掘り

**選択肢B: ペルソナの変更**
- よりpainfulな顧客セグメントに変える
- 例: 「中小企業」→「従業員5人以下の零細企業」

#### Tomorrow Action
- [ ] インタビュー[X]件追加（深掘り質問）
- [ ] 3U再評価
- [ ] ペルソナ/課題の見直し検討

---

### [インタビュー不足の場合]

#### 問題
インタビュー[X]件（目標30件）
肯定率[Y]%（目標50%+）

#### 改善策
1. 目標30件まで継続
2. 肯定率低い → ペルソナ/課題見直し

インタビューガイド: @startup_science/01_stages/cpf/customer_interview.md

#### Tomorrow Action
- [ ] インタビュー候補[X]人リストアップ
- [ ] アポ取得[X]件
- [ ] インタビュー実施[X]件

期限: 2週間以内に30件達成
```

---

### CPF達成時のNext Action

```markdown
## ✅ CPF達成！おめでとうございます

### Next: PSFフェーズへ

PSF（Problem Solution Fit）では:
1. 10倍価値提供の検証
2. MVP作成・テスト
3. UVP（Unique Value Proposition）明確化

#### Tomorrow Action

1. **10倍検証**（2時間）
   - 既存ソリューションのリサーチ
   - 10倍の定義（時間、コスト、品質）
   - フレームワーク: @startup_science/01_stages/psf/10x_validation.md

2. **MVP計画**（3時間）
   - MVP10類型から選択
   - 作成期間・コスト見積もり
   - 参考: @startup_science/01_stages/psf/mvp_types.md

3. **UVP仮説**（1時間）
   - UVPキャンバス埋める
   - テンプレート: @startup_science/01_stages/psf/uvp_canvas.md

#### 2週間後

`/validate-psf` でPSF達成度を診断しましょう。
```

```

## CPF診断結果

平均: 7.5/10 ✅ 目標達成

### 3. インタビュー
30件、肯定60% ✅ 目標達成

---

## ✅ CPF達成！

PSFフェーズへ進みましょう。

[Next Action提示...]
```
