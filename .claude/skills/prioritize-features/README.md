# Prioritize Features - README

## スキル概要

### 目的

機能候補をICEスコア（Impact × Confidence × Ease）とRICEスコア（Reach × Impact × Confidence / Effort）で定量評価し、実装優先順位を自動化します。Quick Wins（少ない工数で大きな成果）を特定し、3ステージ（Now/Next/Later）の実装ロードマップを生成します。

### このスキルでできること

1. **機能候補リストアップ** - ユーザーフィードバック、仮説マップ、競合分析から抽出
2. **ICEスコアリング** - Impact/Confidence/Ease各1-10点で定量評価
3. **RICEスコアリング** - Reach/Impact/Confidence/Effortで多角評価
4. **優先順位ランキング** - ICEスコア降順で機能をランク付け
5. **Quick Wins特定** - Impact ≥ 7 かつ Ease ≥ 7 の機能を自動検出
6. **実装ロードマップ生成** - 0-3ヶ月/3-6ヶ月/6-12ヶ月の段階計画
7. **依存関係マッピング** - 機能間の前提条件を可視化

- 仮説マップ or 競合分析結果

## スコアリング基準

### ICEスコア（Impact × Confidence × Ease）

#### Impact（ビジネスインパクト）1-10点

| 点数 | 基準 | 例 |
|------|------|-----|
| **10点** | ビジネスモデルの根幹、LTV 2倍以上改善 | ユーザー認証、決済機能 |
| **7-9点** | 主要KPI大幅改善（20-50%） | Retention 30%向上、LTV 50%向上 |
| **4-6点** | 重要だが補助的 | UX改善、新コンテンツタイプ |
| **1-3点** | Nice-to-have、改善幅小 | ダークモード、UI調整 |

#### Confidence（確信度）1-10点

| 点数 | 基準 | 根拠 |
|------|------|------|
| **10点** | データで実証済み、確実に効果あり | A/Bテスト成功、成功事例あり |
| **7-9点** | A/Bテスト検証済み、成功事例あり | インタビュー支持率70%以上 |
| **4-6点** | 仮説段階、ユーザーインタビュー支持あり | インタビュー支持率30-70% |
| **1-3点** | 推測、データなし | 想像・社内推測のみ |

#### Ease（実装容易性）1-10点

| 点数 | 基準 | 実装期間 | 複雑度 |
|------|------|---------|--------|
| **10点** | 設定変更のみ | 1日以内 | 最小 |
| **7-9点** | 既存コード軽微修正 | 1週間以内 | 低 |
| **4-6点** | 新規開発必要 | 2-4週間 | 中 |
| **1-3点** | 大規模アーキテクチャ変更 | 2ヶ月以上 | 高 |

#### 計算式

```
ICEスコア = (Impact × Confidence × Ease) / 100
```

**例**: Impact 8 × Confidence 7 × Ease 9 = 5.04

### RICEスコア（Reach × Impact × Confidence / Effort）

- **Reach**: 月次ユーザー到達数（人数）
- **Impact**: ユーザーインパクト（0.25-3点）
- **Confidence**: 確信度（20-100%）
- **Effort**: 実装工数（人月）

```
RICEスコア = (Reach × Impact × Confidence%) / Effort
```

## Quick Wins特定

**定義**: Impact ≥ 7 かつ Ease ≥ 7 の機能

**優先アクション**: 次スプリントで即座実装

**ビジネス価値**: 少ない工数で大きな成果

---

## 実装ロードマップの3ステージ

### Stage 1: Now（0-3ヶ月） - MVP強化

- **選定基準**: ICEスコア上位5-10機能 + Quick Wins全て
- **目的**: PMF達成、Retention向上
- **成功基準**: Retention Rate 5%向上、NPS 10点向上

### Stage 2: Next（3-6ヶ月） - スケール準備

- **選定基準**: ICEスコア中位10-20機能
- **目的**: Activation向上、MAU増加
- **成功基準**: MAU 30%向上、Activation Rate 20%向上

### Stage 3: Later（6-12ヶ月） - 差別化

- **選定基準**: ICEスコア下位機能 + 長期ビジョン
- **目的**: 競合優位性獲得
- **成功基準**: 市場シェア拡大、メディア露出増加

## 使用例

### 例1: SaaSプロダクト（顧客管理ツール）

**入力**: 20機能候補（ユーザーフィードバック + 競合分析）

**実行**:
```
1. ICEスコアリング → トップ5機能を特定
2. Quick Wins → 3機能を自動検出
3. ロードマップ →
   - Now: CRM統合（スコア5.76）、レポート機能（5.04）、Quick Wins 3機能
   - Next: データ分析、自動化ワークフロー
   - Later: AI推奨エンジン
```

**出力**: 優先順位付きBacklog、実装ロードマップ、依存関係図

### 例2: コンシューマーアプリ（SNS機能）

**入力**: 30機能候補（ユーザーインタビュー + 社内案）

**実行**:
```
1. RICEスコアリング → Reach大きい機能を優先
2. Quick Wins → 2機能（DM改善、通知制御）
3. ロードマップ →
   - Now: リアルタイム通知（RICE 1,200）、ワンクリック共有（640）
   - Next: AIレコメンデーション、ユーザー検索
   - Later: ライブストリーミング
```

**出力**: MAU成長戦略、スプリント計画、技術負債整理リスト

## よくある質問（FAQ）

### Q1: 機能候補が100個以上ある場合？

**A**: 機能をカテゴリ別にグループ化してから評価してください。

```
例: CRM機能グループ
  - ユーザー管理系（5機能）
  - レポーティング系（8機能）
  - インテグレーション系（6機能）
```

または、投票やスコアリングで候補を30-40に絞ってからこのスキルを実行します。

### Q2: Easeの評価が難しい場合？

**A**: 技術チームと一緒に評価してください。以下の視点で判定：

- 既存コードベースの活用度
- 外部依存性（API、サービス連携）
- テスト工数
- ドキュメント整備工数

初回評価後、実装時に精査し、次の優先順位付けに反映します。

### Q3: Quick Winsが0件の場合？

**A**: 以下の対応を検討：

1. **Ease軸の見直し** - 過小評価していないか確認
2. **Impact軸の見直し** - 過大評価していないか確認
3. **機能分割の検討** - 大きすぎる機能を小さく分割
4. **MVP絞込み** - スコアが4-6点の「中程度」機能に集中

### Q4: このスキル実行後、誰が機能を選択する？

**A**: 推奨優先順位は生成されますが、最終判断はPMとビジネスチームが担当。理由：

- 市場機会の変動（競合対応）
- 顧客ニーズの急変
- 内部リソース制約

出力結果は「意思決定の根拠」として活用します。

### Q5: 3ヶ月以上先の機能を詳細に計画すべき？

**A**: 不要です。Later（6-12ヶ月）は「方向性の指針」レベル。詳細は以下のタイミングで実行：

- Now完了後 → Next詳細化
- 3ヶ月後 → Later再評価

市場環境は大きく変わるため、固定計画は避けます。
