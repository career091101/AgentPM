## 使用ガイドライン

### ソース分類

機能候補の出所を明確にします：

| ソース種別 | 例 | 重み |
|----------|-----|-----|
| **PSF検証結果** | MVP検証で確認された必須機能 | 高 |
| **ユーザーフィードバック** | 既存ユーザーからの直接要望 | 高 |
| **インタビュー（N=XX）** | ユーザーインタビューで複数人から言及 | 高 |
| **仮説マップ** | プロダクト戦略に基づく仮説 | 中 |
| **競合分析** | 競合プロダクトにある機能 | 中 |
| **内部提案** | チーム内からのアイデア | 低 |

### 初期評価の入力方法

**初期Impact評価**: ビジネスへの推定インパクト（1-10点、目安のみ）
- 「非常に重要」と思う機能 → 8-10点
- 「重要」と思う機能 → 6-7点
- 「あるといい」と思う機能 → 4-5点

**初期Ease評価**: 実装の推定容易さ（1-10点、目安のみ）
- 「すぐにできそう」→ 8-10点
- 「2-4週間で実装できそう」→ 5-7点
- 「2ヶ月以上かかる」→ 1-4点

詳細な評価（Impact/Confidence/Ease）は、スキル実行時に ice_rice_scoring.md で実施します。

### 最低推奨数

- **最小**: 10機能
- **標準**: 15-25機能
- **詳細分析**: 30機能以上の場合は、カテゴリ分けしてグループ化推奨
