# generate-article スキル

Note.com向けの記事コンテンツを自動生成するスキル。

## 概要

- **目的**: SEO最適化されたNote記事を生成
- **出力**: 2,000〜5,000文字のMarkdown記事
- **特徴**: 画像挿入マーカーを自動配置

## 入力パラメータ

| パラメータ | 必須 | 説明 | デフォルト |
|-----------|------|------|-----------|
| `topic` | ○ | 記事のトピック/テーマ | - |
| `target_audience` | △ | ターゲット読者層 | 一般 |
| `tone` | △ | 文体（カジュアル/フォーマル/専門的） | カジュアル |
| `word_count` | △ | 目標文字数 | 3,000 |
| `image_count` | △ | 挿入する画像数 | 3 |
| `keywords` | △ | SEOキーワード（カンマ区切り） | - |

## 出力

### ファイル出力

```
Flow/YYYYMM/YYYY-MM-DD/note_article/
├── draft_article.md      # 生成された記事
└── article_metadata.json # メタデータ（SEO情報等）
```

```markdown
# [タイトル（20〜30文字）]

<!--IMAGE:アイキャッチ画像の説明-->

## [見出し1]

[本文...]

<!--IMAGE:セクション1の説明画像-->

## [見出し2]

[本文...]

## [見出し3]

[本文...]

<!--IMAGE:セクション3の説明画像-->

## [見出し4]

<!-- metadata
title: タイトル
description: 記事の説明（120文字以内）
keywords: キーワード1, キーワード2
estimated_read_time: 5分
-->
```

## 実行手順

### STEP 1: ナレッジベース参照

以下のファイルを読み込み、Noteのベストプラクティスを把握：

```
Stock/programs/副業/projects/SNS/knowledge/Note/algorithm.md
Stock/programs/副業/projects/SNS/knowledge/Note/best_practices.md
```

**重要な最適化ポイント**:

| 指標 | 最適化方法 |
|------|-----------|
| 読了率（最重要） | 2,000〜3,000文字が最適、見出しで区切り |
| 滞在時間 | 画像・引用で視覚的変化を作る |
| CTR | タイトル20〜30文字、数字を含める |
| いいね率 | 感情に訴える結論、CTA配置 |

### STEP 2: 記事構成設計

1. **タイトル設計**（20〜30文字）
   - 数字を含める（例：「3つの方法」「5分で分かる」）
   - 読者のベネフィットを明示
   - 疑問形または断定形

2. **リード文設計**（150〜300文字）
   - 読者の悩み・課題を提示
   - 解決策があることを示唆
   - 「この記事では〜」で内容を予告

3. **本文構成**（4セクション推奨）
   - 各セクション500〜800文字
   - 見出しは具体的かつ興味を引く表現
   - 箇条書き・太字で読みやすく

4. **画像マーカー配置**
   - 形式: `<!--IMAGE:画像の説明-->
   - 配置位置:
     - リード文直後（アイキャッチ）
     - 本文中（1〜2箇所）
     - まとめ前（オプション）

### STEP 3: 記事生成

以下の観点で記事を生成：

**SEO最適化**:
- キーワードを自然に散りばめる（密度2〜3%）
- H2見出しにキーワードを含める
- メタディスクリプション用の要約を作成

**読了率最適化**:
- 冒頭で読者の興味を引く
- 適度な改行と空白
- 専門用語は解説を添える

**エンゲージメント最適化**:
- 問いかけを含める
- 具体例・事例を挙げる
- 最後に行動喚起（CTA）

### STEP 4: メタデータ生成

`article_metadata.json` を生成：

```json
{
  "title": "記事タイトル",
  "description": "SEO用説明文（120文字以内）",
  "keywords": ["キーワード1", "キーワード2"],
  "word_count": 3000,
  "image_markers": [
    {"position": 1, "description": "アイキャッチ画像の説明"},
    {"position": 2, "description": "セクション1の説明画像"},
    {"position": 3, "description": "セクション3の説明画像"}
  ],
  "estimated_read_time": "5分",
  "generated_at": "2026-01-08T12:00:00+09:00"
}
```

### STEP 5: 品質チェック

生成した記事を以下の基準でセルフチェック：

| チェック項目 | 基準 |
|-------------|------|
| 文字数 | 2,000〜5,000文字 |
| タイトル文字数 | 20〜30文字 |
| リード文字数 | 150〜300文字 |
| 見出し数 | 4〜6個 |
| 画像マーカー数 | 3〜5個 |
| 誤字脱字 | なし |
| 論理的整合性 | 破綻なし |

## 画像マーカー形式

画像挿入位置を示すマーカー形式：

```html
<!--IMAGE:説明文-->
```

**例**:
```html
<!--IMAGE:AIを活用したビジネス効率化のイメージ図-->
<!--IMAGE:データ分析のグラフ-->
<!--IMAGE:成功事例のインフォグラフィック-->
```

## エラーハンドリング

| エラー | 対応 |
|--------|------|
| トピックが不明確 | ユーザーに具体化を依頼 |
| 文字数超過 | 自動で要約・圧縮 |
| キーワード過多 | SEOスパムを避け、自然な密度に調整 |
