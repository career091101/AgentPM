# Measure AARRR - 成長ファネル分析スキル

## 概要

**Measure AARRR** は、AARRR（Pirate Metrics）フレームワークを用いて、スタートアップの成長ファネルを体系的に測定・分析するスキルです。Acquisition（獲得）→ Activation（活性化）→ Retention（継続）→ Referral（紹介）→ Revenue（収益）の5段階で、各ステップの転換率を業界ベンチマークと比較し、成長のボトルネックを自動検出します。

### このスキルを使うべき段階

- **PMF達成後**、スケール段階に入った時点
- **月次/週次の成長レビュー**で進捗確認が必要な場合
- **Phase3（スケール）**を実行中のプロジェクト

### 所要時間

初回：40-50分（データ収集・入力含む）
継続実行：20-30分/月

### 出力成果物

- AARRR分析レポート（ファネル分析、ボトルネック検出）
- 改善施策優先順位リスト
- 実装ロードマップ（1ヶ月・3ヶ月計画）

---

## AARRR フレームワーク解説

AARRR（Pirate Metrics）は、スタートアップの成長を5つの段階で測定するフレームワークです。Dave McClureによって提唱され、多くの成功したスタートアップで採用されています。

### 1. Acquisition（獲得）

**何を測定するのか**
- ユーザーがあなたのプロダクトにどのようにして到達するか

**主要KPI**
- 月間訪問者数（Unique Visitors）
- 流入チャネル別内訳（SEO、SNS、広告、リファラル等）
- 前月比成長率

**良好な状態**
- 訪問者数が毎月着実に増加している
- 複数のチャネルからの流入がある

**ボトルネックの場合**
- チャネル単一依存（SEOに90%依存など）
- 訪問者数が停滞または減少

---

### 2. Activation（活性化）

**何を測定するのか**
- 訪問者が実際にサインアップし、初回体験で価値を実感したか

**主要KPI**
- **サインアップ率**：訪問者数に対するサインアップ数（目標：2-5%）
- **Aha Moment達成率**：サインアップからオンボーディング完了率（目標：40%+）

**良好な状態**
- サインアップ率が業界ベンチマーク（2-5%）以上
- Aha Moment達成率が40%を超える
- オンボーディングプロセスが明確に設計されている

**ボトルネックの場合**
- サインアップ率が1%未満（訪問者への訴求力不足）
- Aha Moment達成率が20%未満（初回体験が価値を示していない）
- ユーザーがサインアップ後すぐに離脱

---

### 3. Retention（継続）

**何を測定するのか**
- ユーザーが継続的にプロダクトを利用し続けているか

**主要KPI**
- **DAU/MAU比率**：日次アクティブユーザー / 月間アクティブユーザー（利用習慣度）
- **D1継続率**：初日から翌日も利用したユーザー比率
- **D7継続率**：初日から7日後も利用したユーザー比率
- **D30継続率**：初日から30日後も利用したユーザー比率（月次継続率）
- **Churn Rate**：月次解約率（B2B：<5%/月、B2C：<7%/月）

**良好な状態**
- D30継続率が85-95%以上
- Churn Rateが目標値以下
- DAU/MAU比率が50%以上

**ボトルネックの場合**
- D30継続率が70%未満（短期で多くのユーザーが離脱）
- Churn Rateが高すぎる（プロダクト価値が不足）

---

### 4. Referral（紹介）

**何を測定するのか**
- ユーザーが他のユーザーを紹介する傾向

**主要KPI**
- **紹介率**：紹介経由の新規ユーザー / 総ユーザー（目標：10-30%）
- **バイラル係数（K-factor）**：1ユーザーが紹介する平均人数 × 紹介先の登録率（目標：>1.0）
- **NPS（Net Promoter Score）**：推奨度スコア（目標：50以上）

**良好な状態**
- 紹介率が10%を超える
- バイラル係数が1.0以上（指数関数的成長の可能性）
- NPSが50以上（ユーザーが満足）

**ボトルネックの場合**
- 紹介率が5%未満（口コミが発生していない）
- バイラル係数が0.5未満（自然成長が限定的）
- NPSが30未満（ユーザー満足度が低い）

---

### 5. Revenue（収益）

**何を測定するのか**
- プロダクトが収益を生んでいるか、またその効率性

**主要KPI**
- **課金転換率**：無料ユーザーに対する有料ユーザー比率（目標：2-5%）
- **ARPU（Average Revenue Per User）**：ユーザーあたりの平均月間売上
- **MRR（Monthly Recurring Revenue）**：月次経常収益
- **LTV（Customer Lifetime Value）**：顧客生涯価値
- **LTV/CAC比率**：顧客生涯価値 / 顧客獲得コスト（目標：3.0以上）

**良好な状態**
- 課金転換率が2%以上
- LTV/CAC比率が3.0以上（健全な単位経済学）
- MRRが毎月着実に増加

**ボトルネックの場合**
- 課金転換率が1%未満（プロダクトの価値が価格に見合っていない）
- LTV/CAC比率が1.0未満（顧客獲得コストが高い、または顧客寿命が短い）

## 使用タイミング

### 必要な前提条件

✅ **PMF（Product-Market Fit）を達成している必要があります**

- プロダクト・マーケット適合性を検証してください（validate-pmf スキルを参照）
- PMF未達成の場合は、このスキルを実行する前に PMF達成に注力してください

### 推奨実行タイミング

| タイミング | 使用目的 |
|----------|--------|
| **月次レビュー** | 成長トレンドの監視、ボトルネック変化の追跡 |
| **四半期レビュー** | 改善施策の効果測定、戦略修正 |
| **新機能リリース後** | 各段階への影響測定 |
| **マーケティング施策実行後** | チャネル別効果測定 |
| **スケール段階開始時** | 基準値の確立、目標設定 |

## スキル実行フロー

```
┌─────────────────────────────────────────────┐
│ STEP 1: PMF判定確認                          │
│ → PMF達成を確認、未達成なら処理中止         │
└──────────────┬──────────────────────────────┘
               ↓
┌──────────────────────────────────────────┐
│ STEP 2: 実績データ入力                     │
│ → 5段階のKPIデータを入力                   │
│   (GA4, Mixpanel, 決済システム等から)      │
└──────────────┬──────────────────────────────┘
               ↓
┌──────────────────────────────────────────┐
│ STEP 3: 業界ベンチマーク取得              │
│ → WebSearchで最新ベンチマークを自動取得    │
└──────────────┬──────────────────────────────┘
               ↓
┌──────────────────────────────────────────┐
│ STEP 4: ファネル分析                       │
│ → 転換率を計算、ビジュアル化              │
└──────────────┬──────────────────────────────┘
               ↓
┌──────────────────────────────────────────┐
│ STEP 5: ボトルネック自動検出              │
│ → 実績 vs ベンチマークで差分分析          │
│ → Impact × Ease でスコアリング             │
└──────────────┬──────────────────────────────┘
               ↓
┌──────────────────────────────────────────┐
│ STEP 6: 改善施策提案                       │

└──────────────┬──────────────────────────────┘
               ↓
┌──────────────────────────────────────────┐
│ STEP 7: 実装ロードマップ作成              │
│ → 1ヶ月・3ヶ月の具体的な計画を策定       │
│ → 目標KPI設定                             │
└──────────────────────────────────────────┘
```

## 実際の使用例

### 例：SaaS スタートアップの AARRR 分析

**入力データ**（11月分）
```
Acquisition:
- 月間訪問者数: 10,000人

Activation:
- サインアップ数: 300人（3.0%）
- Aha Moment達成数: 120人（40%）

Retention:
- MAU: 280人
- DAU: 80人（DAU/MAU: 28.6%）
- D30継続率: 75%
- Churn Rate: 8%/月

Referral:
- 紹介経由新規: 45人
- 紹介率: 8.5%
- K-factor: 0.35
- NPS: 35

Revenue:
- 有料ユーザー数: 42人（15%）
- ARPU: ¥15,000
- MRR: ¥630,000
- LTV: ¥450,000
- CAC: ¥200,000
- LTV/CAC: 2.25
```

**分析結果**

| 段階 | 実績値 | ベンチマーク | 差分 | 判定 |
|-----|--------|-----------|------|------|
| Activation（サインアップ率） | 3.0% | 2-5% | ✅ | 良好 |
| Activation（Aha Moment） | 40% | 40%+ | ✅ | 良好 |
| Retention（D30継続率） | 75% | 85-95% | ⚠️ | ボトルネック |
| Referral（紹介率） | 8.5% | 10-30% | ⚠️ | ボトルネック |
| Referral（K-factor） | 0.35 | >1.0 | ❌ | 重大ボトルネック |
| Revenue（課金転換率） | 15% | 2-5% | ✅ | 優秀 |
| Revenue（LTV/CAC比率） | 2.25 | 3.0+ | ⚠️ | 要改善 |

**優先度の高いボトルネック TOP 3**

1. **🚨 バイラル係数（K-factor）が0.35**
   - 理由：1ユーザーの紹介ループが弱い
   - Impact: 8/10（上流からの影響大）
   - Ease: 4/10（紹介仕組みの再設計必要）
   - Priority Score: 32

2. **⚠️ D30継続率が75%**
   - 理由：ユーザーが30日以内に離脱している
   - Impact: 7/10（全段階への影響）
   - Ease: 6/10（エンゲージメント施策で改善可能）
   - Priority Score: 42

3. **⚠️ LTV/CAC比率が2.25**
   - 理由：顧客獲得コストに対し、顧客価値が低い
   - Impact: 5/10（収益性への影響）
   - Ease: 7/10（価格戦略・アップセル施策で改善）
   - Priority Score: 35

**推奨改善施策（Quick Wins）**

1. **エンゲージメントメール施策**（Retention改善）
   - 対象：D7未達成ユーザー
   - 施策：毎週金曜に成果サマリーメール配信
   - 期待効果：D30継続率 75% → 85%
   - 実装期間：2週間

2. **紹介インセンティブプログラム**（Referral改善）
   - 対象：有料ユーザー
   - 施策：紹介者に30日間プラン割引、被紹介者に初月50%割引
   - 期待効果：K-factor 0.35 → 0.6
   - 実装期間：1週間

## よくある質問（FAQ）

### Q1. 業界ベンチマークはどこから取得するのか？

**A.** このスキルは自動的に WebSearch を実行して、最新の業界ベンチマークデータを取得します。

- 検索対象：「SaaS AARRR metrics benchmarks 2025」など
- 出典の記載：レポートに参考ソースURLを記載

ただし、業界・ビジネスモデル別にベンチマークが異なるため、参考値としています。あなたのプロダクト固有の競合他社データも併せて収集することをお勧めします。

---

### Q2. データが不完全な場合、実行できるのか？

**A.** 部分的なデータでも実行可能ですが、以下の点に注意してください：

- **必須データ**：
  - 訪問者数（Acquisition）
  - サインアップ数（Activation）
  - 継続率（Retention）

- **推奨データ**：
  - Aha Moment達成数
  - D30継続率
  - 課金転換率

データが不足している場合は、そのステップの分析が限定的になることを理解した上で、実行してください。

---

### Q3. ボトルネックが複数ある場合、何から着手すべきか？

**A.** 以下の優先順位で判断してください：

1. **Impact（影響度） × Ease（容易性）** で Priority Score を計算
2. **Quick Wins**（Impact ≥7、Ease ≥7、Score ≥49）を最優先実行
3. **ファネル上流** のボトルネックから対応（Acquisition > Activation > Retention > Referral > Revenue）

通常、Quick Wins を1-2週間で実装し、効果を測定した上で、中期・長期施策に移行します。

---

### Q4. 月次で AARRR を再測定する場合、何に注意すべきか？

**A.** 継続測定時の注意点：

- **新規ユーザーベースの分離**：前月のユーザーと新規ユーザーを分けて測定
- **セグメント分析**：チャネル別・プラン別に分析（全体と個別で異なる場合がある）
- **季節変動の考慮**：年末年始、GW等の影響を考慮
- **施策の効果測定**：前月実装した改善施策の効果を定量的に測定

---

### Q5. AARRR 分析の後は、何をするべきか？

**A.** 推奨される次のステップ：

| ステップ | 概要 |
|---------|------|
| **1. Quick Wins実装** | 最初の1-2週間で優先度の高い改善施策を実装 |
| **2. A/Bテスト** | 各施策の効果を定量的に検証 |
| **3. 週次モニタリング** | KPIダッシュボードで進捗を追跡 |
| **4. 月次再測定** | AARRR 分析を繰り返し、改善効果を確認 |
| **5. 施策調整** | 効果が出ていない施策は中止・修正 |

スケール段階では、AARRR 分析と改善施策の実装を継続的に回すことが重要です。

## トラブルシューティング

### AARRR分析が実行できない

**症状**: 「PMF達成が確認できません」というエラー

**解決策**:
1. `validate-pmf` スキルを実行し、PMF達成判定を取得
2. PMF判定レポートが Flow に生成されたことを確認
3. measure-aarrr を再実行

---

### ボトルネックの判定が不明確

**症状**: 複数のステップが「ボトルネック」と判定されている

**解決策**:
1. 各ステップの実績値が正確か再確認
2. 業界ベンチマークが適切か確認（SaaS vs エンタープライズ vs モバイルアプリ）
3. セグメント別分析を追加実施（チャネル別・デバイス別等）
