## 4. API価格変更分析

### 月次コスト試算 (5M tokens/月の場合)

#### Gemini 1.0 Proでの月間コスト
```
入力: 3M tokens × $0.50 = $1,500
出力: 2M tokens × $1.50 = $3,000
合計: $4,500/月
```

#### Gemini 1.5 Proでの月間コスト
```
入力: 3M tokens × $0.075 = $225
出力: 2M tokens × $0.30 = $600
合計: $825/月
```

#### 削減効果
```
月間削減額: $4,500 - $825 = $3,675 (82%削減)
年間削減額: $44,100
```

### ビデオ処理での新規収益機会

```
従来: ビデオ処理未対応 (外部サービス利用)
外部コスト: $3/分単位 × 480分/月 = $1,440/月

Gemini 1.5 Pro対応:
ビデオ処理コスト: $0.12/分 × 480分 = $57.60/月
削減効果: $1,382.40/月 (96%削減)

新規サービス化可能:

- ビデオ内容検索
```

## 7. 移行判断・移行計画

### 移行判定: ✅ **即時推奨**

理由:

- 62.5倍のコンテキスト拡張

### 段階的移行計画

#### Phase 1: 準備 (2日)
```bash
# API確認
curl https://generativelanguage.googleapis.com/v1/models/gemini-1.5-pro:generateContent \
  -H "x-goog-api-key: $GOOGLE_API_KEY" \
  -d '{
    "contents": [{
      "role": "user",
      "parts": [{"text": "test"}]
    }]
  }'
```

#### Phase 2: A/Bテスト (3日)
```
30% トラフィック: Gemini 1.5 Pro
70% トラフィック: Gemini 1.0 Pro

測定項目:
- テキスト精度
- 応答速度
- エラー率
- コスト
- ビデオ処理品質 (該当タスク)
```

#### Phase 3: 段階的ロールアウト (4日)
```
Day 1: 30% → 50%
Day 2: 50% → 70%
Day 3: 70% → 85%
Day 4: 85% → 100%
```

#### Phase 4: 完全移行 (1日)
```bash
# 設定更新
config/gemini.yaml:
  primary_model: "gemini-1.5-pro"
  context_window: 2000000
  enable_video_processing: true
  fallback_model: "gemini-1.0-pro"
```

### ロールバック計画

**トリガー条件**:
- テキストエラー率 > 1.5%
- ビデオ処理エラー > 5%
- 精度低下 > 1.5%

**ロールバック手順**:
```bash
1. Gemini 1.0 Proに即座に切り替え
2. キャッシュをクリア
3. 詳細ログ分析
4. ユーザー通知
```
