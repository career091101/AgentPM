## 学び

### 成功要因

1. **オープンソース・コミットメント**:
   - 「Truly open. Truly yours.」という明確なメッセージング
   - Apache 2.0、CreativeML等のライセンス明示で透明性

2. **商用利用の明確化**:
   - 「No licensing fees」「No royalties」を冒頭で明記
   - DALL-E 2（商用利用制限）、Midjourney（サブスク必須）との差別化

3. **セルフホスティング訴求**:
   - 「Run anywhere」「No cloud lock-in」でクラウド依存回避
   - 開発者・研究者の「データプライバシー」「コスト削減」ニーズに対応

4. **コミュニティ成長の可視化**:
   - GitHub、HuggingFace、Discordの統計を前面に出す
   - コミュニティ作成モデル50K+で「民主化」を証明

### 教訓

- **オープンソースLPでは「商用利用可」が最重要**: ライセンス、ロイヤリティの明確化
- **コミュニティ統計**: GitHub Stars、HuggingFaceダウンロード数を社会的証明に活用
- **セルフホスティング訴求**: クラウドAPIとの併存（Self-hosted無料、API有料）
- **民主化メッセージング**: 「AI by the people, for the people」という理念訴求

### 適用可能性

- **オープンソースAI全般**: 商用利用可、セルフホスティング、コミュニティ統計をLPのコアに配置
- **AI民主化**: GitHub、HuggingFace統合、Discordコミュニティ訴求
- **クラウドAPI併存**: Self-hosted無料 + クラウドAPI有料のハイブリッド戦略
