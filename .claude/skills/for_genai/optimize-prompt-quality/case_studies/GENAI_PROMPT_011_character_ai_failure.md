# Character.AI - Prompt Optimization Insufficient (Failure Case)

## プロンプト最適化サマリー

| 指標 | 現状 | ForGenAI基準 | 改善余地 |
|------|------|------------|---------|
| AI精度 | 78% | 90%以上 | -12% |
| ハルシネーション率 | 15% | 5%以下 | -10% |
| 一貫性 | 65% | 85%以上 | -20% |
| ユーザー満足度 | 82%（エンタメ特化） | - | - |

## 背景

Character.AIは、エンタメ特化のAIチャットボット製品として、若年層を中心に高いエンゲージメント（DAU/MAU 0.55）を誇る。しかし、プロンプト最適化が不十分で、ハルシネーション率15%（ForGenAI基準5%以下の3倍）と高い。

## 問題点

### 1. プロンプト最適化不十分

**現状プロンプト（推定）**:
```
あなたは[キャラクター名]です。ユーザーとチャットしてください。
```

**問題**:
- Chain-of-Thoughtなし → AI精度低下
- Few-shot examplesなし → 一貫性低下
- ハルシネーション防止ルールなし → ハルシネーション率15%

### 2. エンタメ特化でハルシネーション許容

Character.AIは、エンタメ目的のため、事実確認よりもキャラクター一貫性を重視。ハルシネーション率15%は「キャラクター創造性」として許容されている。

**例**:
```
ユーザー: 「昨日のニュース教えて」
Character.AI: 「昨日、火星で新しい生命体が発見されたらしいよ！」

[ハルシネーション: 火星で生命体発見は事実でない]
[ただし、エンタメ目的では許容範囲]
```

## 改善提案

### Few-shot Examples導入

**改善後プロンプト**:
```
あなたは[キャラクター名]です。以下の例に倣ってユーザーとチャットしてください。

例1:
ユーザー: 「今日の天気は？」
キャラクター: 「今日は晴れだよ！散歩日和だね。」

例2:
ユーザー: 「昨日のニュース教えて」
キャラクター: 「ごめん、最新ニュースは分からないんだ。でも、君の話なら聞けるよ！」

[ハルシネーション回避例を明示]
```

**期待効果**:
- ハルシネーション率: 15% → 8%（-7%）
- 一貫性: 65% → 80%（+15%）

### Constitutional AI導入

**改善後プロンプト**:
```
憲法:
1. 事実確認が必要な情報は「分からない」と答える
2. 不確実な情報は「推測だけど...」と明記する
3. ハルシネーション（誤情報生成）を避ける

キャラクター性と事実確認のバランスを取る。
```

**期待効果**:
- ハルシネーション率: 8% → 5%（-3%、ForGenAI基準達成）

## 失敗教訓

1. **エンタメ特化でもハルシネーション対策必須**: ユーザー信頼性低下リスク
2. **プロンプト最適化コスト回避は短期的**: 長期的にユーザー離脱リスク
3. **DAU/MAU高い ≠ AI品質高い**: エンゲージメントとAI品質は別指標

## 成功要因（エンタメ特化）

1. **若年層バイラル**: DAU/MAU 0.55（最高）
2. **キャラクター一貫性**: エンタメ目的では十分
3. **低コスト運営**: プロンプト最適化コスト削減
