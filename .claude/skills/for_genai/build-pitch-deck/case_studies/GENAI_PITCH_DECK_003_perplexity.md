## スライド構成分析

### 1. Problem (10/10点)
**課題定義**: 検索エンジンの限界（リンク羅列、広告、時間浪費）
- Google検索: 平均7-8クリック必要、10分以上の時間浪費
- 広告優先: オーガニック検索結果の信頼性低下
- 情報の分散: 複数サイトを横断、統合作業が必要

**定量データ**:
- 検索→回答取得時間: Google 10分 vs Perplexity 2秒（**80%短縮**）
- クリック数: Google 7-8クリック vs Perplexity 0クリック
- 広告比率: Google検索結果の30%が広告、Perplexity 0%

### 2. Solution (10/10点)
**UVP**: Answer Engine（検索 → 即座に回答 + Citation）
- リアルタイム検索AI（最新情報を即座に統合）
- Citation機能（情報源明記、信頼性担保）
- 対話型インターフェース（追加質問で深堀り）

**Before → After**:
| Before（Google検索） | After（Perplexity） |
|---------------------|---------------------|
| リンク羅列（7-8クリック） | 即座に回答（0クリック） |
| 10分以上の時間浪費 | 2秒で回答取得（80%短縮） |
| 広告30%、信頼性低下 | 広告0%、Citation付き |

### 3. Market (10/10点)
**TAM/SAM/SOM**:
- TAM: $200B（検索広告市場、2024年）
- SAM: $50B（AI-powered検索市場、2026年）

**Why Now**:

### 4. AI Technology (15/15点)

- Multi-source aggregation: 10-20サイト並列検索、統合回答生成

**ファインチューニング**:
- Citation抽出（情報源特定、信頼性評価）
- Ranking algorithm（回答品質スコアリング）
- リアルタイム検索最適化（速度 vs 精度トレードオフ）

**プロプライエタリデータ**:

- Citation評価データ: 人間フィードバック10万+
- リアルタイム検索ログ: 検索精度向上、Ranking最適化
- データフライホイール: ユーザー検索 → クエリ蓄積 → 検索精度向上 → ユーザー増加
