# case_009_cohere_enterprise_developer_persona.md

## 概要
- **製品名**: Cohere
- **カテゴリ**: Enterprise LLM Platform
- **URL**: https://cohere.ai
- **関連性**: Enterprise開発者向けペルソナ設計により、プライベートデプロイ・カスタマイズ需要を捉え、エンタープライズAPI契約1,000+を達成。

## 背景
Cohereは2019年にGoogle Brain元メンバーが創業。OpenAI、Anthropicとの差別化として「エンタープライズ特化」「プライベートデプロイ」「カスタムモデル」を打ち出し、金融、医療、製造業で導入拡大。

## エンタープライズ開発者・ペルソナ

### プライマリペルソナ "Enterprise ML Engineer"
- **デモグラフィック**:
  - 年齢: 28-45歳
  - 職業: MLエンジニア、AI/MLリードエンジニア
  - 組織規模: 1,000-50,000名企業のAI/MLチーム
  - 経験: 5-15年（機械学習、NLP）
  - 予算権限: $100K-$1M/年（API費用、インフラ）

- **サイコグラフィック**:
  - **最大の課題**:
    - ChatGPT/Claude等のクラウドAPIでは「データプライバシー」「カスタマイズ性」に制限
    - 自社データでのファインチューニングが困難
    - レイテンシ、可用性のSLA保証が不十分
  - **意思決定基準**:
    1. プライベートデプロイ可能性（VPC、オンプレミス）
    2. カスタムモデル作成（自社データでファインチューニング）
    3. API SLA保証（99.9%以上、レイテンシ<200ms）
    4. 価格透明性（トークン単価、ボリュームディスカウント）
  - **行動パターン**:
    - HuggingFace、AWS SageMaker等を既に使用
    - arXiv、NeurIPS等で最新研究をフォロー
    - 社内AI/MLコミュニティで技術共有
  - **情報源**: arXiv、NeurIPS/ICML、HuggingFace、AI/MLエンジニアSlackコミュニティ

- **Jobs-to-be-Done**:
  1. **Functional Job**: 自社データでカスタムLLMを3ヶ月で構築し、社内業務に統合
  2. **Emotional Job**: 「クラウドAPIにデータを送る」不安の解消、「自社専用AI」の優越感
  3. **Social Job**: 社内で「エンタープライズAIの専門家」として認知される

### セカンダリペルソナ "Enterprise Architect"
- **特徴**: システムアーキテクト、インフラ・セキュリティ設計責任者
- **ニーズ**:
  - 既存システム統合（SAP、Salesforce、Snowflake等）
  - セキュリティ認証（SOC 2、ISO 27001、HIPAA）
  - マルチクラウド対応（AWS、Azure、GCP）
- **購買行動**: PoC（3-6ヶ月） → アーキテクチャ承認 → 本番導入

### ターシャリペルソナ "Data Scientist (Internal)"
- **特徴**: 社内データサイエンスチーム、カスタムユースケース開発
- **ニーズ**: Jupyter Notebook統合、実験環境、モデル評価ツール
- **購買行動**: 無料API枠 → PoC → エンタープライズ契約

## 定量データ
- **エンタープライズAPI契約**: 1,000+企業（2023年10月時点）
- **プライベートデプロイ比率**: 全契約の47%（VPC/オンプレミス）
- **カスタムモデル作成**: 月間200+モデル（顧客データでファインチューニング）
- **API SLA**: 99.95%（レイテンシ中央値 165ms）
- **顧客業界**: 金融38%、医療24%、製造18%、その他20%
- **平均契約額**: $250K/年（エンタープライズAPI + カスタムモデル）

## 学び

### 成功要因
1. **プライベートデプロイ重視**: OpenAI、Anthropicと異なり、VPC/オンプレミス展開を標準提供
2. **カスタマイズ性**: 自社データでのファインチューニング、プロンプトエンジニアリング支援
3. **エンタープライズSLA**: 99.95%可用性、レイテンシ保証、24/7サポート
4. **業界特化**: 金融、医療等の規制業界向けにコンプライアンス対応

### 教訓
- **エンタープライズML Engineerは「データプライバシー」が最優先**: クラウドAPIへのデータ送信を嫌い、プライベートデプロイを要求
- **カスタマイズ性の重要性**: 汎用LLMではなく、自社データで最適化したモデルが必要
- **SLA保証**: エンタープライズでは「99.9%以上」「レイテンシ<200ms」が契約条件
- **3層ペルソナ設計**: 実装者（ML Engineer） → アーキテクチャ承認者 → 購買権限者

### 適用可能性
- **エンタープライズAI全般**: ペルソナに「プライベートデプロイ要求」「カスタマイズ性」「SLA要求水準」を含める
- **規制業界AI**: 金融、医療等でコンプライアンス要件をペルソナに反映
- **AI検証**: プライベートデプロイ比率、カスタムモデル作成数をKPIとして測定
