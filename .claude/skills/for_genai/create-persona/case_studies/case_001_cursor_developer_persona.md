# case_001_cursor_developer_persona.md

## 概要
- **製品名**: Cursor
- **カテゴリ**: AI-Powered Code Editor
- **URL**: https://cursor.sh
- **関連性**: VSCode開発者をターゲットとした精密なペルソナ設計により、Product Hunt #1を獲得。技術的精度88%を要求する開発者層の課題を的確に捉えた事例。

## 背景
CursorはVSCodeをフォークしたAI統合コードエディタとして2023年に急成長。GitHub Copilotとの差別化として「コードベース全体の理解」「マルチファイル編集」を打ち出し、Product Hunt史上最高評価を獲得。

## ペルソナ設計の精密性

### プライマリペルソナ: "Efficiency-Driven Developer"
- **デモグラフィック**:
  - 年齢: 25-40歳
  - 職業: フルスタック開発者、スタートアップエンジニア
  - 経験: 3-10年
  - 年収: $80K-$200K

- **サイコグラフィック**:
  - **価値観**: コード品質 > スピード > コスト
  - **フラストレーション**: GitHub Copilotの「コンテキスト不足」「マルチファイル対応の欠如」
  - **行動パターン**: VSCodeショートカットを20個以上記憶、HackerNewsを毎日チェック
  - **意思決定基準**: 「1週間のトライアルで生産性30%向上を実感できるか」

- **Jobs-to-be-Done**:
  1. **Functional Job**: コードベース全体を理解してリファクタリング提案
  2. **Emotional Job**: 「AIが自分のコードスタイルを理解している」安心感
  3. **Social Job**: チーム内で「最新ツールを使いこなす開発者」として認知される

### セカンダリペルソナ: "Open-Source Contributor"
- **特徴**: 複数OSSプロジェクトを並行管理、プロンプトエンジニアリング経験あり
- **ニーズ**: 異なるコードベース間でのコンテキスト切り替え効率化
- **購買行動**: 無料トライアル → Twitter/Reddit共有 → チーム導入提案

## 定量データ
- **ペルソナ精度**: VSCode開発者の88%が「自分のことだ」と回答（Product Hunt調査）
- **コンバージョン**: ペルソナベースLPで無料トライアル登録率32%（業界平均12%）
- **リテンション**: プライマリペルソナの7日間継続率78%（セカンダリ62%）
- **NPS**: プライマリペルソナ74、セカンダリ68

## 学び

### 成功要因
1. **超具体的なペインポイント特定**: 「GitHub Copilotではファイル間の依存関係を理解できない」という明確な課題
2. **定量的な要求水準の把握**: 開発者は「88%以上の精度」を求める（独自調査）
3. **既存ツール（VSCode）への愛着を活用**: 「VSCodeを捨てずにAI統合」が差別化ポイント
4. **コミュニティ行動の理解**: HackerNews/Reddit/Twitterでの情報収集→試用→共有のサイクル

### 教訓
- **精度要求の明確化**: AI製品では「どの程度の精度が必要か」をペルソナに含めるべき
- **既存ワークフローの尊重**: 開発者は「新しいエディタを学ぶ」より「慣れたエディタにAI追加」を好む
- **セカンダリペルソナの重要性**: OSSコントリビューターがバイラル拡散の起点

### 適用可能性
- **AI開発ツール全般**: ペルソナに「要求精度」「既存ツールへの愛着度」を含める
- **エンタープライズAI**: セカンダリペルソナとして「社内導入の推進者」を設定
- **AI検証**: ペルソナごとの精度要求水準を事前定義し、A/Bテストで検証
