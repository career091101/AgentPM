# Monitor Burn Rate - ForGenAI Edition

## 目的

GenAI製品の資金状況を常時監視し、**バーンレート（月次消費金額）**と**ランウェイ（資金が尽きるまでの期間）**を自動計算。**18ヶ月ルール**（AI市場変化速い、長めのランウェイ推奨）に基づき、資金ショートを未然に防ぎ、最適な資金調達タイミングを提案します。

**重要な原則**（AI市場特化）:
> 「GenAI企業は、ランウェイが18ヶ月を切ったら、資金調達の準備を開始すべきである」
> 「AI市場は変化が激しく、モデル更新・競合出現により想定外のコスト増が発生しやすい」
> 「GPU費用とAPI推論費用が全支出の30-60%を占めるため、モニタリングが最重要」
> — a16z AI Practice, OpenAI Startup Fund

**ForGenAI版の追加機能**:
1. **AI固有コスト分析**: GPU費用、モデル推論API、ファインチューニング費用の内訳化
2. **Rule of 40評価**: 成長率% + 利益率% ≥ 40% の達成度診断
3. **AI製品事例統合**: 12件のGenAI製品（成功8件、失敗4件）のバーンレート管理パターン
4. **Net Burn Rate基準厳格化**: 月次MRRの50%以内（for_startup 60%より厳格）

### Common Pitfalls (失敗パターン)

1. **GPU費用の過小評価** → モデル改善でGPU費用が想定の3倍に増加
2. **API推論費用の変動無視** → OpenAI料金改定で月次費用が2倍
3. **収益化前のスケール** → バーンレート月$10M、収益$0で9ヶ月で資金枯渇
4. **競合出現の影響軽視** → GPT-4登場でユーザー流出、収益半減

### Best Practices

1. **AI固有コストの月次監視**: GPU、API、ファインチューニング費用を必ず別計上
2. **Rule of 40達成を目標**: 成長率% + 利益率% ≥ 40% を維持
3. **Net Burn Rateを月次MRRの50%以内に**: 収益化前提の厳格管理
4. **ランウェイ18ヶ月以上を確保**: AI市場変化速い、長めが安全

## 実行ステップ

### STEP 1: 現在残高の入力（1分）

**質問**:
1. 現在の手元現金残高（銀行口座残高）
2. 最終更新日
3. **[ForGenAI追加]** 前回調達日・調達額（初回の場合は自己資金額）

**入力例**:
```
現在残高: $5,000,000
更新日: 2026-01-02
前回調達: 2025-07-01、Seed $10,000,000
```

### STEP 2: Gross Burn Rate計算（2分）

**計算方法**: 月次総支出の算出

**必要データ**:
- 人件費（給与、社会保険）
- **AI固有コスト**:
  - GPU費用（AWS/GCP/Azure）
  - モデル推論API費用（OpenAI/Anthropic/Google）
  - ファインチューニング費用
- マーケティング費
- オフィス・設備費
- 開発費（外注、ツール費）
- その他経費

**計算式**:
```
Gross Burn Rate = 月次総支出 / 月
```

**計算例**:
```
人件費: $100,000/月
GPU費用: $50,000/月
API推論費用: $30,000/月
ファインチューニング費用: $10,000/月
マーケティング費: $20,000/月
オフィス費: $5,000/月
開発費: $15,000/月
その他: $5,000/月

Gross Burn Rate = $235,000/月
```

**AI固有コスト比率**: ($50,000 + $30,000 + $10,000) / $235,000 = 38.3% ✅

**業種別目安**（ForGenAI基準）:
| ステージ | 目安バーンレート | AI固有コスト | チーム規模 | 資金調達レンジ |
|---------|----------------|-------------|----------|--------------|
| Pre-Seed（MVP開発） | $20,000-50,000/月 | $5,000-15,000/月 | 2-3人 | $500K-1M |
| Seed（初期トラクション） | $100,000-300,000/月 | $30,000-100,000/月 | 5-10人 | $3-5M |
| Series A（スケール） | $500,000-1,500,000/月 | $150,000-500,000/月 | 20-50人 | $10-30M |
| Series B（グロース） | $2,000,000-5,000,000/月 | $600,000-2,000,000/月 | 50-150人 | $50-150M |

---

### STEP 3: Net Burn Rate計算（2分）

**計算方法**: 収益を差し引いた実質消費額

**必要データ**:
- 月次収益（MRR、API課金、Plus課金）
- 月次総支出（STEP 2から）

**計算式**:
```
Net Burn Rate = (月次総支出 - 月次収益) / 月
```

**計算例**:
```
月次収益（MRR）: $50,000
月次総支出: $235,000

Net Burn Rate = $185,000/月
```

**ForGenAI基準判定**:
- Net Burn Rate ≤ MRR × 50% → ✅ 健全
- Net Burn Rate > MRR × 50% → ⚠️ 要改善

**本ケース**: $185,000 > $50,000 × 50% ($25,000) → 🚨 要改善

**注意**:
- Net Burn Rateがマイナス（収益 > 支出）の場合 → キャッシュフロー黒字 ✅
- Net Burn Rateがプラスの場合 → 赤字運営、資金枯渇リスクあり ⚠️

---

### STEP 4: ランウェイ計算（1分）

**計算方法**: 現在の消費ペースで資金が何ヶ月持つか

**計算式**:
```
Runway（月） = 現在残高 / Net Burn Rate
```

**計算例**:
```
現在残高: $5,000,000
Net Burn Rate: $185,000/月

Runway = $5,000,000 / $185,000 = 27.0ヶ月 ✅
```

**判定基準**（ForGenAI版: 18ヶ月ルール適用）:

| ランウェイ | 判定 | 状態 | 推奨アクション |
|-----------|------|------|--------------|
| **18ヶ月以上** | ✅ 安全 | 余裕あり | 成長投資、月次監視継続 |
| **12-18ヶ月** | ⚠️ 注意 | 資金調達検討開始 | 投資家リスト作成、財務モデル更新 |
| **6-12ヶ月** | 🚨 警告 | 資金調達準備開始 | ピッチデッキ作成、ウォームイントロ |
| **6ヶ月以下** | 🔴 緊急 | 資金調達必須 | 即座に調達活動開始、コスト削減 |

**投資家判断への影響**:
- ランウェイ < 12ヶ月 → 交渉力低下、Valuation下方圧力
- ランウェイ < 6ヶ月 → 投資見送りリスク大（「資金繰りに困っている」と判断）

### STEP 6: Rule of 40評価（2分）

**計算方法**: 成長率% + 利益率% ≥ 40% の達成度診断

**計算式**:
```
Rule of 40 = 成長率% + 利益率%
```

**計算例**:
```
月次成長率: 20%（MRR $50,000 → $60,000）
年間成長率: 20% × 12 = 240%（複利計算では更に高い）
利益率: (収益 - 支出) / 収益 = ($50,000 - $235,000) / $50,000 = -370%

Rule of 40 = 240% + (-370%) = -130% 🚨
```

**ForGenAI基準判定**:
- Rule of 40 ≥ 40% → ✅ 優秀
- Rule of 40 0-40% → ⚠️ 要改善
- Rule of 40 < 0% → 🚨 危険（本ケース）

**改善施策**:
- 成長率向上: MRR成長率を30%/月に加速
- 利益率改善: コスト削減（GPU最適化、API費用削減）
