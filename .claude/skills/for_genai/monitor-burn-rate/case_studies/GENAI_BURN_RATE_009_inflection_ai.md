# Inflection AI - バーンレート高すぎ、Microsoft吸収

| 指標 | 実績 | ForGenAI基準 | 判定 |
|------|------|-------------|:----:|

| Net Burn Rate | $167M/月 | MRRの50%以内 | 🔴 |
| 総調達額 | $1.5B | N/A | - |

## 詳細分析

### バーンレート内訳（2024年推定）

| 項目 | 月次コスト | 構成比 |
|-----|-----------|-------|
| 人件費（200人） | $30M | 18% |
| GPU費用（Azure） | $100M | 60% |
| API推論費用 | $20M | 12% |
| マーケティング費 | $10M | 6% |
| その他 | $7M | 4% |
| **合計支出** | **$167M/月** | **100%** |
| **月次収益（MRR）** | **$0** | - |
| **Net Burn Rate** | **$167M/月** | - |

**AI固有コスト**: $100M + $20M = $120M（全体の72%、異常値）

### 資金調達タイミング

- **2023年6月**: Series A $1.5B（Microsoft, Reid Hoffman主導、評価額$4B）
- **調達理由**: Pi開発、超大規模GPU投資
- **ランウェイ**: $1.5B ÷ $167M/月 = 9.0ヶ月（危機的）

### 失敗要因

1. **GPU費用異常値**:
   - 月$100M（全体の60%）
   - 22,000個のNVIDIA H100 GPUを購入・運用
   - 収益$0で月$100M支出は持続不可能

2. **収益化失敗**:
   - Pi無料提供継続
   - 月次収益$0（課金モデル未確立）

3. **ランウェイ9ヶ月**:
   - $1.5B調達でも9ヶ月で資金枯渇リスク
   - ForGenAI基準18ヶ月の半分以下

4. **Microsoft吸収**（2024年6月）:
   - 創業者・主要メンバーはMicrosoft AI部門に参加
   - IP・モデルはMicrosoftにライセンス供与
   - 実質的な失敗（独立企業として継続できず）

### 成長 vs 効率のバランス

**Rule of 40評価**:
```
年間成長率: 0%（MRR $0）
利益率: ($0 - $167M) / $0 = -∞
Rule of 40 = 0% + (-∞) = -∞ 🔴
```

**成長効率**:
- CAC: 計測不能（収益$0）
- LTV: $0
- LTV/CAC: 0（最悪）

## 教訓

### 失敗要因

1. **GPU費用異常値**: 月$100M（全体の60%）は持続不可能
2. **収益化失敗**: 月次収益$0で$167M支出は論外
3. **ランウェイ9ヶ月**: $1.5B調達でも9ヶ月で枯渇
4. **Rule of 40 -∞**: 成長率0%、利益率-∞は最悪

### ForGenAI製品への示唆

- **GPU費用は全体の30-60%以内**: 60%超過は危険（Inflectionは72%）
- **収益化最優先**: GPU費用$100M/月なら、最低でも月$50M収益必要
- **ランウェイ18ヶ月以上確保**: 9ヶ月は危機的、早期資金ショート
- **Rule of 40**: 成長率0%は論外、最低でも100%成長必要

### 注意点

- Inflection AIは極端な失敗事例（GPU $100M/月、収益$0は異常）
- 一般的なGenAI製品はGPU費用$5-20M/月が現実的
- 収益化前提のバーンレート管理が必須
