# GenAI製品 Burn Rate管理 - 12ケーススタディサマリー

## 概要

本ディレクトリには、GenAI製品のバーンレート管理に関する12件のケーススタディ（成功8件、失敗・警告4件）が収録されています。

**ForGenAI版の評価基準**:
- **ランウェイ**: 18ヶ月以上（AI市場変化速い、長めが安全）
- **Net Burn Rate**: 月次MRRの50%以内
- **AI固有コスト比率**: 30-60%（GPU+API費用）
- **Rule of 40**: 成長率% + 利益率% ≥ 40%

## ForGenAI製品への重要教訓

### 1. ランウェイ管理

| 基準 | 推奨値 | 根拠 |
|-----|-------|------|
| **最低ライン** | 18ヶ月 | AI市場変化速い、長めのランウェイが安全 |
| **理想** | 24ヶ月 | 余裕を持った調達活動が可能 |
| **危険** | 12ヶ月以下 | 早期資金調達必須 |
| **危機的** | 6ヶ月以下 | 緊急ブリッジ調達 |

**成功事例**: Jasper AI（24ヶ月）、Cohere（24ヶ月）、Anthropic（18ヶ月）
**失敗事例**: Inflection AI（9ヶ月）、Vicuna（6ヶ月）

### 2. Net Burn Rate管理

| 基準 | 推奨値 | 根拠 |
|-----|-------|------|
| **ForGenAI基準** | MRRの50%以内 | 収益化前提の厳格管理 |
| **許容範囲** | MRRの60-80% | 高成長期は一時的に許容 |
| **危険** | MRRの100%超 | 収益化加速が必須 |
| **異常値** | MRRの200%超 | ビジネスモデル再設計必須 |

**成功事例**: ChatGPT Plus（-5%、黒字）、Midjourney（-20%、黒字）
**失敗事例**: AI21 Labs（355%）、Adept（348%）、Inflection AI（∞）

### 3. AI固有コスト管理

| 基準 | 推奨値 | 根拠 |
|-----|-------|------|
| **健全範囲** | 30-60% | GPU+API費用、健全範囲 |
| **警告** | 60-70% | コスト削減検討 |
| **異常値** | 70%超 | 持続不可能、即座に削減必要 |

**成功事例**: ChatGPT Plus（42%）、Midjourney（50%）、Anthropic（54%）
**失敗事例**: Inflection AI（72%）、Vicuna（80%）

**GPU費用最適化施策**:
- AWS Spot Instance活用: 40-50%削減
- Google TPU活用: 30%削減
- モデル量子化（INT8）: 40%削減
- バッチ推論最適化: 効率2倍

### 4. Rule of 40管理

| 基準 | 推奨値 | 根拠 |
|-----|-------|------|
| **ForGenAI基準** | 40%以上 | 成長率% + 利益率% ≥ 40% |
| **優秀** | 100%以上 | 高成長・高効率 |
| **警告** | 0-40% | 改善必要 |
| **危険** | 0%未満 | ビジネスモデル再検討 |

**成功事例**: ChatGPT Plus（305%）、Midjourney（170%）、Anthropic（150%）
**失敗事例**: AI21 Labs（-315%）、Adept（-248%）、Inflection AI（-∞）

### 5. 資金調達タイミング

| ランウェイ | 推奨アクション | 所要期間 |
|-----------|--------------|---------|
| **18ヶ月以上** | 成長投資継続、月次監視 | - |
| **12-18ヶ月** | 資金調達検討開始 | 3-6ヶ月 |
| **6-12ヶ月** | 資金調達準備開始 | - |
| **6ヶ月以下** | 緊急ブリッジ調達 | - |

**資金調達目標額**: Net Burn Rate × 18ヶ月
