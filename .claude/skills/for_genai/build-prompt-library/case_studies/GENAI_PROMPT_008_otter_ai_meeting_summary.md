## Otter.ai会議要約プロンプトの詳細分析

### 1. Otter.aiとは

**定義**: AI音声認識と会議要約ツール。リアルタイム文字起こし、話者識別、自動要約、アクションアイテム抽出を提供。

**3つの特徴**:
1. **リアルタイム文字起こし**: 話しながら即座にテキスト化（遅延1秒以内）
2. **OtterPilot**: 会議に自動参加し、記録・要約（Zoom、Teams、Google Meet対応）
3. **AI Summary**: 会議終了後30秒で要約生成

**アーキテクチャ**:
```
音声入力（マイク or 会議ツール）
  ↓
リアルタイム文字起こし（Whisperベース）
  ↓
話者識別（Diarization）
  ↓
GPT-4で要約生成
  ↓
アクションアイテム抽出
  ↓
出力（Transcript + Summary）
```

### 3. 技術的キモ

#### Speaker Identification（話者識別）

Otter.aiは自動的に話者を識別：

```markdown
# Transcript Example

[00:00:12] 山田: 本日はプロダクトレビューを行います。
[00:00:18] 佐藤: ダークモードの実装状況を報告します。
[00:00:45] 鈴木: デザインモックを共有します。

→ 話者ごとに発言を整理
→ 要約時に「山田が決定」「佐藤が提案」と明確化
```

**効果**: 手動話者識別時間 15分 → 0秒（-100%）、精度92%

#### Live Summary（ライブ要約）

会議中にリアルタイムで要約を更新：

```markdown
# 会議開始（0分）
## Summary
[まだ空]

# 会議中（15分経過）
## Summary
### 議論中のトピック
- ダークモード実装の優先順位

# 会議終了（60分）
## Summary
### Key Discussion Points
- ダークモード実装を優先
- リリース日: 2/15
...（完全な要約）
```

**効果**: 会議後の整理時間 20分 → 30秒（-98%）

### 5. 適用事例

#### 事例1: リモート会議の議事録自動化

**課題**: 週5回の会議で議事録作成に毎回30分、週2.5時間の負担

**Otter.ai活用**:
```markdown
# Workflow
1. Zoom会議にOtterPilotを招待（自動参加）
2. 会議中、リアルタイム文字起こし
3. 会議終了後30秒で要約生成
4. アクションアイテムをSlackに自動投稿（Zapier連携）
```

**結果**:
- 議事録作成時間: 30分 → 2分（-93%、確認のみ）
- アクションアイテム抽出: 手動 → 自動（精度91%）
- ステークホルダー満足度: 3.2 → 4.5（+41%）
- 年間削減時間: 130時間（週2.5時間 × 52週）

#### 事例2: カスタマーインタビュー分析

**課題**: 1時間のインタビューを文字起こし・分析に3-4時間

**Otter.ai活用**:
```markdown
# Workflow
1. インタビュー中、Otter.aiで録音・文字起こし
2. 会議後、カスタムプロンプトで要約:
   - ユーザーのペインポイント
   - 要望機能
   - 満足度評価
3. Transcriptをタグ付け（#pain-point, #feature-request等）
```

**結果**:
- 文字起こし時間: 2時間 → 0分（-100%）
- 分析時間: 1-2時間 → 15分（-88%）
- ペインポイント抽出: 手動5-8個 → AI 12個（+50%）
- インサイト発見: 定性的 → 定量的（頻出キーワード分析）

### 7. 限界と課題

#### 限界

1. **多人数会議**: 10人以上で話者識別精度低下（92% → 78%）
2. **重複発話**: 同時発話時の文字起こし精度低下
3. **方言・訛り**: 標準語以外で精度低下（94% → 85%）

#### 対策

| 課題 | 対策 |
|------|------|
| **多人数会議** | マイクを近づける、発言者を名乗る |
| **重複発話** | ファシリテーターが交通整理 |
| **方言・訛り** | カスタムボキャブラリーで補正 |

## Key Learnings

### 成功要因

1. **リアルタイム文字起こし**: 遅延0.8秒で会議中に確認可能、精度94%
2. **自動話者識別**: 手動識別時間100%削減、精度92%
3. **AI Summary**: 会議終了後30秒で要約生成、議事録作成時間93%削減

### 適用推奨シーン

- **リモート会議**: 議事録自動化（週2.5時間削減）
- **顧客インタビュー**: ペインポイント自動抽出
- **ブレインストーミング**: アイデア一覧化
- **1on1**: 進捗・課題の記録

### 適用非推奨シーン

- **多人数会議（10人以上）**: 話者識別精度低下
- **専門用語多い会議**: カスタムボキャブラリー登録必須
- **非英語会議**: 日本語は対応だが精度やや低い
